1. Thinking in 2D DP

When a problem involves two sequences, two indices, or a grid, a 2D DP table is the natural choice. The cell dp[i][j] represents some property of the first i elements of one sequence and the first j elements of another.

1D DP: dp[i] = answer for input of size i 2D DP: dp[i][j] = answer for first i items of A AND first j items of B Common 2D DP setups: ┌──────────────────────────────────────────────────────────┐ │ Problem Type │ dp[i][j] represents │ ├──────────────────────────────────────────────────────────┤ │ Two sequences (LCS) │ best result for A[0..i-1], │ │ │ B[0..j-1] │ ├──────────────────────────────────────────────────────────┤ │ Grid problems │ answer to reach cell (i,j) │ ├──────────────────────────────────────────────────────────┤ │ Knapsack │ dp[i][w] = best value using │ │ │ first i items with capacity w │ └──────────────────────────────────────────────────────────┘
Space Optimization for 2D DP: If dp[i][j] only depends on the previous row (dp[i-1][...]), you can reduce space from O(m×n) to O(n) by using just two 1D arrays (current row and previous row). Sometimes even one row with careful updates.

2. Longest Common Subsequence (LCS)

Problem: Given two strings, find the length of their longest common subsequence (characters don't need to be contiguous). (LeetCode 1143)

Example: text1 = "abcde", text2 = "ace" → LCS = "ace" → length 3

text1 = "abcde" (rows) text2 = "ace" (cols) "" a c e "" [0 0 0 0] a [0 1 1 1] ← 'a'=='a': dp[i-1][j-1]+1=1 b [0 1 1 1] ← 'b'!='a','c','e': max(left,up) c [0 1 2 2] ← 'c'=='c': dp[i-1][j-1]+1=2 d [0 1 2 2] e [0 1 2 3] ← 'e'=='e': dp[i-1][j-1]+1=3 Answer: dp[5][3] = 3 → "ace" Recurrence: if text1[i-1] == text2[j-1]: dp[i][j] = dp[i-1][j-1] + 1 else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])

Implementation

public int LongestCommonSubsequence(string text1, string text2)
{
    int m = text1.Length, n = text2.Length;
    int[,] dp = new int[m + 1, n + 1];
    // dp[0,*] = dp[*,0] = 0 (empty string LCS = 0)

    for (int i = 1; i <= m; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            if (text1[i - 1] == text2[j - 1])
                dp[i, j] = dp[i - 1, j - 1] + 1; // Match: extend
            else
                dp[i, j] = Math.Max(dp[i - 1, j], dp[i, j - 1]); // Skip one
        }
    }
    return dp[m, n];
}
// Time: O(m*n), Space: O(m*n)  → optimize to O(n) with 1D rolling array

// Space-Optimized O(n):
public int LCSOptimized(string text1, string text2)
{
    int m = text1.Length, n = text2.Length;
    int[] prev = new int[n + 1];
    int[] curr = new int[n + 1];

    for (int i = 1; i <= m; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            if (text1[i - 1] == text2[j - 1])
                curr[j] = prev[j - 1] + 1;
            else
                curr[j] = Math.Max(prev[j], curr[j - 1]);
        }
        Array.Copy(curr, prev, n + 1);
        Array.Clear(curr, 0, n + 1);
    }
    return prev[n];
}

Reconstructing the Actual LCS

public string ReconstructLCS(string text1, string text2)
{
    int m = text1.Length, n = text2.Length;
    int[,] dp = new int[m + 1, n + 1];

    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            dp[i, j] = text1[i - 1] == text2[j - 1]
                ? dp[i - 1, j - 1] + 1
                : Math.Max(dp[i - 1, j], dp[i, j - 1]);

    // Backtrack from dp[m,n] to find the actual characters
    var sb = new System.Text.StringBuilder();
    int r = m, c = n;
    while (r > 0 && c > 0)
    {
        if (text1[r - 1] == text2[c - 1])
        { sb.Insert(0, text1[r - 1]); r--; c--; }
        else if (dp[r - 1, c] > dp[r, c - 1])
            r--;
        else
            c--;
    }
    return sb.ToString();
}

3. Edit Distance (Levenshtein Distance)

Problem: Given two words, find the minimum number of operations (insert, delete, replace) to transform word1 into word2. (LeetCode 72)

Example: "horse" → "ros" in 3 operations: replace h→r, delete r, delete e → "ros"

word1 = "horse", word2 = "ros" "" r o s "" [ 0 1 2 3 ] ← cost to build word2 from empty = insertions h [ 1 1 2 3 ] o [ 2 2 1 2 ] r [ 3 2 2 2 ] s [ 4 3 3 2 ] e [ 5 4 4 3 ] ← answer = 3 Recurrence: if word1[i-1] == word2[j-1]: dp[i][j] = dp[i-1][j-1] (free — chars match) else: dp[i][j] = 1 + min( dp[i-1][j], ← delete from word1 dp[i][j-1], ← insert into word1 dp[i-1][j-1] ← replace in word1 )
public int MinDistance(string word1, string word2)
{
    int m = word1.Length, n = word2.Length;
    int[,] dp = new int[m + 1, n + 1];

    // Base cases: convert to/from empty string
    for (int i = 0; i <= m; i++) dp[i, 0] = i; // Delete all of word1
    for (int j = 0; j <= n; j++) dp[0, j] = j; // Insert all of word2

    for (int i = 1; i <= m; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            if (word1[i - 1] == word2[j - 1])
                dp[i, j] = dp[i - 1, j - 1]; // No operation needed
            else
                dp[i, j] = 1 + Math.Min(dp[i - 1, j - 1],  // Replace
                           Math.Min(dp[i - 1, j],       // Delete
                                    dp[i, j - 1]));      // Insert
        }
    }
    return dp[m, n];
}
// Time: O(m*n), Space: O(m*n)
Real-World Use: Edit distance is used in spell-checkers ("did you mean?"), DNA sequence alignment, diff tools (git diff), and natural language processing.

4. 0/1 Knapsack

Problem: Given items with weights and values, and a knapsack of capacity W, find the maximum value you can carry. Each item can be taken at most once (0/1 = take or leave).

Items: weight=[1,3,4,5], value=[1,4,5,7], capacity=7 0 1 2 3 4 5 6 7 (capacity) none[0 0 0 0 0 0 0 0] i=0 [0 1 1 1 1 1 1 1] (item wt=1,val=1) i=1 [0 1 1 4 5 5 5 5] (item wt=3,val=4) i=2 [0 1 1 4 5 6 6 9] (item wt=4,val=5) i=3 [0 1 1 4 5 7 8 9] (item wt=5,val=7) Answer: dp[4][7] = 9 (items: wt=3,val=4 + wt=4,val=5) Recurrence: dp[i][w] = max( dp[i-1][w], // skip item i dp[i-1][w - weight[i]] + value[i] // take item i (if w >= weight[i]) )
public int Knapsack01(int[] weights, int[] values, int capacity)
{
    int n = weights.Length;
    int[,] dp = new int[n + 1, capacity + 1];

    for (int i = 1; i <= n; i++)
    {
        for (int w = 0; w <= capacity; w++)
        {
            // Option 1: skip item i
            dp[i, w] = dp[i - 1, w];

            // Option 2: take item i (only if it fits)
            if (weights[i - 1] <= w)
                dp[i, w] = Math.Max(dp[i, w],
                    dp[i - 1, w - weights[i - 1]] + values[i - 1]);
        }
    }
    return dp[n, capacity];
}
// Time: O(n*W), Space: O(n*W)

// Space-Optimized O(W): iterate capacity RIGHT-TO-LEFT when using 1D array
public int Knapsack01Optimized(int[] weights, int[] values, int capacity)
{
    int[] dp = new int[capacity + 1];

    for (int i = 0; i < weights.Length; i++)
    {
        // RIGHT-TO-LEFT: ensures each item is taken at most once
        // (prevents using item i multiple times within same row)
        for (int w = capacity; w >= weights[i]; w--)
            dp[w] = Math.Max(dp[w], dp[w - weights[i]] + values[i]);
    }
    return dp[capacity];
}
// Time: O(n*W), Space: O(W)
Warning — 0/1 vs Unbounded Knapsack Loop Direction:
0/1 Knapsack (each item used at most once): inner loop goes RIGHT TO LEFT (capacity down to weight).
Unbounded Knapsack (each item reusable): inner loop goes LEFT TO RIGHT.
This is a common interview gotcha — always state which you're solving first.

5. Unique Paths — Grid DP

Problem: A robot starts at top-left of an m×n grid. It can only move right or down. How many unique paths to reach bottom-right? (LeetCode 62)

3×3 grid: paths[i][j] = paths from (0,0) to (i,j) 0 1 2 0 [ 1 1 1 ] ← top row: only 1 way (all right) 1 [ 1 2 3 ] 2 [ 1 3 6 ] ← answer = 6 Recurrence: dp[i][j] = dp[i-1][j] + dp[i][j-1] (came from above OR came from left) Base case: dp[0][j] = 1, dp[i][0] = 1 (edges have 1 path)
public int UniquePaths(int m, int n)
{
    int[,] dp = new int[m, n];

    // Base cases: first row and first column = 1
    for (int i = 0; i < m; i++) dp[i, 0] = 1;
    for (int j = 0; j < n; j++) dp[0, j] = 1;

    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[i, j] = dp[i - 1, j] + dp[i, j - 1];

    return dp[m - 1, n - 1];
}
// Time: O(m*n), Space: O(m*n) → O(n) with 1D array

// Space-Optimized O(n): only need previous row
public int UniquePathsOptimized(int m, int n)
{
    int[] dp = new int[n];
    Array.Fill(dp, 1); // First row all 1s

    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[j] += dp[j - 1]; // dp[j] = above + left

    return dp[n - 1];
}

Unique Paths II — With Obstacles (LeetCode 63)

public int UniquePathsWithObstacles(int[][] obstacleGrid)
{
    int m = obstacleGrid.Length, n = obstacleGrid[0].Length;
    if (obstacleGrid[0][0] == 1) return 0; // Start blocked

    int[] dp = new int[n];
    dp[0] = 1;

    for (int i = 0; i < m; i++)
    {
        for (int j = 0; j < n; j++)
        {
            if (obstacleGrid[i][j] == 1)
                dp[j] = 0; // Obstacle: no paths through here
            else if (j > 0)
                dp[j] += dp[j - 1];
        }
    }
    return dp[n - 1];
}

Minimum Path Sum (LeetCode 64)

Find the path from top-left to bottom-right with minimum sum (can only go right or down).

public int MinPathSum(int[][] grid)
{
    int m = grid.Length, n = grid[0].Length;
    // Recurrence: dp[i][j] = grid[i][j] + min(from above, from left)

    // Fill first row and column as cumulative sums
    for (int j = 1; j < n; j++) grid[0][j] += grid[0][j - 1];
    for (int i = 1; i < m; i++) grid[i][0] += grid[i - 1][0];

    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            grid[i][j] += Math.Min(grid[i - 1][j], grid[i][j - 1]);

    return grid[m - 1][n - 1];
}
// Time: O(m*n), Space: O(1) — modify grid in-place

6. Longest Increasing Subsequence (LIS)

Problem: Find the length of the longest strictly increasing subsequence. (LeetCode 300)

Example: [10, 9, 2, 5, 3, 7, 101, 18] → LIS = [2, 3, 7, 101] → length 4

nums = [10, 9, 2, 5, 3, 7, 101, 18] idx: 0 1 2 3 4 5 6 7 dp[i] = length of LIS ending at index i dp[0] = 1 (just [10]) dp[1] = 1 (just [9]) dp[2] = 1 (just [2]) dp[3] = 2 ([2,5]) nums[3]=5 > nums[2]=2 → dp[2]+1=2 dp[4] = 2 ([2,3]) nums[4]=3 > nums[2]=2 → dp[2]+1=2 dp[5] = 3 ([2,3,7]) nums[5]=7 > nums[4]=3 → dp[4]+1=3 dp[6] = 4 ([2,3,7,101]) dp[7] = 4 ([2,3,7,18]) Answer: max(dp) = 4

O(n²) DP Solution

public int LengthOfLIS(int[] nums)
{
    int n = nums.Length;
    int[] dp = new int[n];
    Array.Fill(dp, 1); // Every element alone is an LIS of length 1

    int maxLen = 1;

    for (int i = 1; i < n; i++)
    {
        for (int j = 0; j < i; j++)
        {
            if (nums[j] < nums[i]) // Strictly increasing
                dp[i] = Math.Max(dp[i], dp[j] + 1);
        }
        maxLen = Math.Max(maxLen, dp[i]);
    }
    return maxLen;
}
// Time: O(n²), Space: O(n)

O(n log n) — Patience Sorting

Maintain a tails array where tails[i] is the smallest tail element of all increasing subsequences of length i+1. Use binary search to update.

public int LengthOfLISFast(int[] nums)
{
    var tails = new List<int>();

    foreach (int num in nums)
    {
        // Find leftmost position where tails[i] >= num
        int lo = 0, hi = tails.Count;
        while (lo < hi)
        {
            int mid = (lo + hi) / 2;
            if (tails[mid] < num) lo = mid + 1;
            else hi = mid;
        }
        if (lo == tails.Count)
            tails.Add(num);  // Extend LIS
        else
            tails[lo] = num; // Replace to maintain smallest tail
    }
    return tails.Count; // Length of LIS
}
// Time: O(n log n), Space: O(n)
// Note: tails is NOT the actual LIS, but its length is correct
nums = [10, 9, 2, 5, 3, 7, 101, 18] tails evolution: 10: [10] 9: [9] (replace 10 — smaller tail for len=1) 2: [2] (replace 9) 5: [2, 5] (extend) 3: [2, 3] (replace 5 — smaller tail for len=2) 7: [2, 3, 7] (extend) 101: [2, 3, 7, 101](extend) 18: [2, 3, 7, 18] (replace 101) Length = 4 ✓ (tails = [2,3,7,18] is NOT the LIS, but length is correct)

7. Palindrome Problems

Longest Palindromic Substring (LeetCode 5)

Find the longest palindromic contiguous substring. Classic approach: expand around center O(n²) time, O(1) space.

public string LongestPalindrome(string s)
{
    int start = 0, maxLen = 1;

    for (int center = 0; center < s.Length; center++)
    {
        // Odd-length palindromes (e.g., "aba")
        ExpandAndCheck(s, center, center, ref start, ref maxLen);
        // Even-length palindromes (e.g., "abba")
        ExpandAndCheck(s, center, center + 1, ref start, ref maxLen);
    }
    return s.Substring(start, maxLen);
}

private void ExpandAndCheck(string s, int l, int r,
                              ref int start, ref int maxLen)
{
    while (l >= 0 && r < s.Length && s[l] == s[r])
    { l--; r++; }

    int len = r - l - 1; // Length after expansion
    if (len > maxLen) { start = l + 1; maxLen = len; }
}
// Time: O(n²), Space: O(1)

Longest Palindromic Subsequence (LeetCode 516)

Find the longest palindromic subsequence (characters need not be contiguous). Key: LPS(s) = LCS(s, reverse(s)).

public int LongestPalindromeSubseq(string s)
{
    int n = s.Length;
    // dp[i][j] = length of LPS in s[i..j]
    int[,] dp = new int[n, n];

    // Base: single chars are palindromes of length 1
    for (int i = 0; i < n; i++) dp[i, i] = 1;

    // Fill by increasing substring length
    for (int len = 2; len <= n; len++)
    {
        for (int i = 0; i <= n - len; i++)
        {
            int j = i + len - 1;
            if (s[i] == s[j])
                dp[i, j] = dp[i + 1, j - 1] + 2;
            else
                dp[i, j] = Math.Max(dp[i + 1, j], dp[i, j - 1]);
        }
    }
    return dp[0, n - 1];
}
// Time: O(n²), Space: O(n²)

8. Partition Equal Subset Sum

Problem: Can you partition an array into two subsets with equal sums? (LeetCode 416)

This is a 0/1 knapsack decision problem: can we pick items that sum to exactly totalSum / 2?

nums = [1, 5, 11, 5] total = 22, target = 11 dp[w] = can we achieve sum w with nums seen so far? dp[0] = true (empty subset sums to 0) After num=1: dp[1]=true After num=5: dp[5]=true, dp[6]=true After num=11: dp[11]=true, dp[12]=true, dp[16]=true, dp[17]=true After num=5: dp[11]=true ← answer! (5+1+5=11, other side=11)
public bool CanPartition(int[] nums)
{
    int total = nums.Sum();
    if (total % 2 != 0) return false; // Odd sum — impossible

    int target = total / 2;
    bool[] dp = new bool[target + 1];
    dp[0] = true; // Empty subset

    foreach (int num in nums)
    {
        // RIGHT-TO-LEFT: 0/1 knapsack (each num used at most once)
        for (int w = target; w >= num; w--)
        {
            if (dp[w - num])
                dp[w] = true;
        }
        if (dp[target]) return true; // Early exit
    }
    return dp[target];
}
// Time: O(n * target), Space: O(target)

9. Matrix Chain Multiplication — Interval DP

Problem: Given dimensions of matrices in a chain, find the minimum number of scalar multiplications to compute their product. This introduces interval DP — the subproblem is a range [i, j].

Matrix A: 10×30, B: 30×5, C: 5×60 Dims array: [10, 30, 5, 60] Cost(A×B) = 10*30*5 = 1500, result is 10×5 Cost((AB)×C) = 1500 + 10*5*60 = 1500+3000 = 4500 Cost(B×C) = 30*5*60 = 9000, result is 30×60 Cost(A×(BC)) = 9000 + 10*30*60 = 9000+18000 = 27000 Best: (A×B)×C with cost 4500 dp[i][j] = min cost to multiply matrices i..j dp[i][j] = min over k in [i,j-1]: dp[i][k] + dp[k+1][j] + dims[i-1]*dims[k]*dims[j]
public int MatrixChainOrder(int[] dims)
{
    int n = dims.Length - 1; // Number of matrices
    int[,] dp = new int[n + 1, n + 1];
    // dp[i][i] = 0 (single matrix costs nothing)

    // Fill by chain length 2, 3, ..., n
    for (int len = 2; len <= n; len++)
    {
        for (int i = 1; i <= n - len + 1; i++)
        {
            int j = i + len - 1;
            dp[i, j] = int.MaxValue;

            for (int k = i; k < j; k++)
            {
                int cost = dp[i, k] + dp[k + 1, j]
                           + dims[i - 1] * dims[k] * dims[j];
                dp[i, j] = Math.Min(dp[i, j], cost);
            }
        }
    }
    return dp[1, n];
}
// Time: O(n³), Space: O(n²)
Interval DP Pattern: Problems like Burst Balloons, Strange Printer, and Remove Boxes all use this pattern where dp[i][j] represents the answer for a subarray and you try all split points k.

10. Burst Balloons — Advanced Interval DP

Problem: n balloons, each with a number. Burst a balloon and earn left * balloon * right. Maximize total coins. (LeetCode 312)

Key insight: Instead of thinking about which balloon to burst first, think about which to burst last in range [i,j].

public int MaxCoins(int[] nums)
{
    int n = nums.Length;
    // Add virtual balloons 1 at both ends
    int[] balloons = new int[n + 2];
    balloons[0] = balloons[n + 1] = 1;
    for (int i = 0; i < n; i++) balloons[i + 1] = nums[i];

    int N = n + 2;
    int[,] dp = new int[N, N];

    // dp[i][j] = max coins from open interval (i, j) — not including i,j
    for (int len = 2; len < N; len++)
    {
        for (int i = 0; i < N - len; i++)
        {
            int j = i + len;
            for (int k = i + 1; k < j; k++)
            {
                // k is the LAST balloon burst in (i, j)
                int coins = balloons[i] * balloons[k] * balloons[j]
                           + dp[i, k] + dp[k, j];
                dp[i, j] = Math.Max(dp[i, j], coins);
            }
        }
    }
    return dp[0, N - 1];
}
// Time: O(n³), Space: O(n²)

12. DP Pattern Recognition & Summary

Which DP Dimension to Use?

Problem StructureDimensionExamples
Single sequence, state at position i1D: dp[i]Fibonacci, House Robber, Climbing Stairs
Two sequences or 2 indices2D: dp[i][j]LCS, Edit Distance, Knapsack
Grid/matrix navigation2D grid: dp[r][c]Unique Paths, Min Path Sum
Contiguous interval [i..j]Interval 2D: dp[i][j]Burst Balloons, MCM, LPS
With remaining budget/capacity2D: dp[i][remaining]Knapsack, Coin Change, Trips Within K

Space Optimization Checklist

  • 1D: 2 variables — when dp[i] depends only on dp[i-1] and dp[i-2] (Fibonacci, House Robber)
  • 2D → 1D array — when dp[i][j] depends only on dp[i-1][...] (previous row)
  • Right-to-left sweep — for 0/1 knapsack with 1D array (avoid using item twice)
  • Left-to-right sweep — for unbounded knapsack (allow using item multiple times)
Interview Strategy:
  1. Identify it's a DP problem (overlapping subproblems + optimal substructure)
  2. Define the state: what does dp[i] or dp[i][j] mean?
  3. Write the recurrence relation (the "how does a bigger problem relate to smaller ones?")
  4. Code the base cases
  5. Code the loops (check direction!)
  6. Mention space optimization if asked
