Section 1 — Why Linked Lists? Arrays vs Linked Lists

You already know arrays — a block of contiguous memory. Arrays are great for random access (jump to index 5 instantly — O(1)) but terrible at insertion and deletion in the middle because every element after the insertion point must shift.

A Linked List solves exactly that. Instead of one big memory block, a linked list is a chain of nodes scattered anywhere in memory, each holding a piece of data and a pointer (a reference in C#) to the next node. Insert and delete at the front: O(1). No shifting needed.

🌍 Real-World Analogy — A Train:
Think of each train car as a node. Each car is coupled to the next one via a hook (the pointer). To add a new car at the front, you just attach it and point it to the old first car — O(1). To add it in the middle, you unhook two cars and re-hook with the new one — O(1) once you're there, but you had to walk from the front to find the spot — O(n).

💡 When to choose Linked List over Array:
  • You insert/delete frequently at the front (e.g., implementing a stack or queue)
  • You don't know the size in advance and don't want array resizing overhead
  • You need O(1) merge of two lists (just point last node of list A to head of list B)
  • Implementing other data structures: stacks, queues, adjacency lists for graphs
Section 2 — The Node: Building Block

Everything in a linked list is built from a single concept: the Node. A node is a tiny object that holds two things:

  1. Data — the value you want to store (an int, a string, an object — anything)
  2. Next — a reference (pointer) to the next node in the chain, or null if there is no next node
// The simplest possible Node for integers
public class Node
{
    public int Data;          // The value stored in this node
    public Node? Next;         // Reference to the NEXT node (null if last)

    public Node(int data)
    {
        Data = data;
        Next = null;           // New nodes start disconnected
    }
}

// Generic version (preferred — works with any type)
public class Node<T>
{
    public T Data;
    public Node<T>? Next;

    public Node(T data)
    {
        Data = data;
        Next = null;
    }
}
A SINGLE NODE in Memory ┌─────────────────────────────┐ │ Node Object │ │ ┌──────────────────────┐ │ │ │ Data: 42 │ │ ← The actual value │ ├──────────────────────┤ │ │ │ Next: ──────────────────────▶ (points to another Node, or null) │ └──────────────────────┘ │ └─────────────────────────────┘ When Next = null, this node is the LAST node in the list (the TAIL).
// Creating nodes manually to understand how they link:
Node<int> node1 = new Node<int>(10);
Node<int> node2 = new Node<int>(20);
Node<int> node3 = new Node<int>(30);

// Link them together: 10 → 20 → 30 → null
node1.Next = node2;
node2.Next = node3;
// node3.Next is already null

// Traverse: walk from node1 to the end
Node<int>? current = node1;
while (current != null)
{
    Console.Write(current.Data + " → ");
    current = current.Next;
}
// Output: 10 → 20 → 30 →
📌 C# References are Pointers: In C#, when you write Node? Next, the variable Next does not contain another Node object — it contains a memory address (reference) telling the runtime where to find the next Node. This is exactly what a pointer is in other languages. When you set node1.Next = node2, you're storing the address of node2 inside node1.
Section 3 — Singly Linked List Structure

A Singly Linked List is a collection that manages a chain of nodes using:

  • Head — a reference to the first node. This is your entry point to the entire list. If Head is null, the list is empty.
  • Tail — a reference to the last node. Keeping a tail reference makes adding to the end O(1) instead of O(n).
  • Count — the number of nodes (cached for O(1) size lookup).
SINGLY LINKED LIST — Full Structure SinglyLinkedList object ┌──────────────────────────────────────────────┐ │ Head ──────▶ ┌────┬───┐ ┌────┬───┐ │ │ │ 10 │ ──┼───▶│ 20 │ ──┼──▶ │ │ └────┴───┘ └────┴───┘ │ │ Tail ───────────────────────────────▶┌────┬───┐ │ │ 30 │ X │ ← null │ Count = 3 └────┴───┘ └──────────────────────────────────────────────┘ X = null (no next node — this is the tail) EMPTY LIST: ┌──────────────────────────────────────────────┐ │ Head = null │ │ Tail = null │ │ Count = 0 │ └──────────────────────────────────────────────┘
public class SinglyLinkedList<T>
{
    private Node<T>? head;    // first node
    private Node<T>? tail;    // last node
    public int Count { get; private set; }

    public bool IsEmpty => Count == 0;

    // We'll add methods in the sections below
}
Section 4 — Traversal

Traversal means visiting every node from head to tail. Since there is no index to jump to, you always start at head and follow the Next pointer until you reach null.

⚠️ The Golden Rule of Traversal: NEVER modify head while traversing. Always create a separate variable (conventionally called current) to walk the list. Losing head means losing access to the entire list! 
// Print all nodes: O(n) — must visit every node
public void Print()
{
    Node<T>? current = head;  // DO NOT touch 'head' itself!
    while (current != null)
    {
        Console.Write(current.Data + " → ");
        current = current.Next;  // Move to next node
    }
    Console.WriteLine("null");
}

// Get element at index i: O(n)
public T Get(int index)
{
    if (index < 0 || index >= Count)
        throw new ArgumentOutOfRangeException(nameof(index));

    Node<T>? current = head;
    for (int i = 0; i < index; i++)
        current = current!.Next;

    return current!.Data;
}
TRAVERSAL TRACE — List: 10 → 20 → 30 → null Step 1: current = head (points to node 10) current != null → print "10 → " current = current.Next (now points to node 20) Step 2: current = node 20 current != null → print "20 → " current = current.Next (now points to node 30) Step 3: current = node 30 current != null → print "30 → " current = current.Next (now null) Step 4: current = null Loop exits → print "null" OUTPUT: 10 → 20 → 30 → null
Section 5 — Insertion Operations

5.1 — Insert at Head — O(1)

Adding a node to the front of the list. This is the fastest insertion — no traversal needed. Steps: create new node → set its Next to the current head → update head to be the new node.

INSERT AT HEAD — Adding node (5) to list: 10 → 20 → 30 → null BEFORE: head ──▶ [10|──▶] [20|──▶] [30|null] Step 1: Create newNode(5). newNode.Next = null Step 2: newNode.Next = head → newNode.Next = ──▶ [10|──▶] Step 3: head = newNode AFTER: head ──▶ [5|──▶] [10|──▶] [20|──▶] [30|null] Edge case (empty list): head = null, tail = null → newNode.Next = null (already null) → head = newNode → tail = newNode (the only node is both head AND tail)
public void AddFirst(T value)
{
    Node<T> newNode = new Node<T>(value);

    if (IsEmpty)
    {
        head = newNode;
        tail = newNode;      // Only node — both head and tail
    }
    else
    {
        newNode.Next = head;  // New node points to old head
        head = newNode;       // Update head to new node
    }
    Count++;
}

5.2 — Insert at Tail — O(1)

Adding a node to the end of the list. With a tail reference, this is O(1) — no traversal needed. Without tail, you'd have to walk the entire list.

INSERT AT TAIL — Adding node (99) to list: 10 → 20 → 30 → null BEFORE: head ──▶ [10|──▶] [20|──▶] [30|null] ◀── tail Step 1: Create newNode(99). newNode.Next = null Step 2: tail.Next = newNode → [30|──▶] → [99|null] Step 3: tail = newNode AFTER: head ──▶ [10|──▶] [20|──▶] [30|──▶] [99|null] ◀── tail
public void AddLast(T value)
{
    Node<T> newNode = new Node<T>(value);

    if (IsEmpty)
    {
        head = newNode;
        tail = newNode;
    }
    else
    {
        tail!.Next = newNode;  // Old tail now points to new node
        tail = newNode;        // Update tail
    }
    Count++;
}

5.3 — Insert at Index — O(n)

Adding a node at a specific position requires traversal to the node before the target position, then relinking pointers. The key insight: you need the node before the insertion point, not the node at the insertion point.

INSERT AT INDEX 2 — Adding node (99) to: 10 → 20 → 30 → 40 → null BEFORE: [10|▶] [20|▶] [30|▶] [40|null] idx0 idx1 idx2 idx3 We want: 10 → 20 → 99 → 30 → 40 → null Step 1: Traverse to node BEFORE index 2 = node at index 1 (value=20) prev = node(20) Step 2: newNode(99).Next = prev.Next newNode.Next ──▶ [30|▶] Step 3: prev.Next = newNode [20|──▶] now points to [99] AFTER: [10|▶] [20|▶] [99|▶] [30|▶] [40|null] idx0 idx1 idx2 idx3 idx4 KEY: You always wire the NEW node to point FORWARD first (step 2), THEN update the PREVIOUS node (step 3). Reversing these steps would lose the rest of the list!
public void AddAt(int index, T value)
{
    if (index < 0 || index > Count)
        throw new ArgumentOutOfRangeException(nameof(index));

    if (index == 0)   { AddFirst(value); return; }
    if (index == Count) { AddLast(value);  return; }

    Node<T> newNode = new Node<T>(value);

    // Traverse to the node BEFORE the target index
    Node<T>? prev = head;
    for (int i = 0; i < index - 1; i++)
        prev = prev!.Next;

    // Wire new node forward FIRST, then update prev
    newNode.Next = prev!.Next;   // Step 2: new → old next
    prev.Next = newNode;          // Step 3: prev → new
    Count++;
}
💡 The Two-Step Rule for Middle Insertion: Always:
1. Set newNode.Next = prev.Next (connect new node to the rest of the list)
2. Set prev.Next = newNode (connect previous node to new node)
Never do step 2 first — you'd lose your reference to the rest of the list!
Section 6 — Deletion Operations

6.1 — Delete First Node — O(1)

Simply move head forward to the second node. The old head node becomes unreachable and is garbage collected by C#'s runtime automatically.

DELETE FIRST — List: 10 → 20 → 30 → null BEFORE: head ──▶ [10|──▶] [20|──▶] [30|null] Step 1: head = head.Next → head now points to node(20) Step 2: The old node(10) has no more references → GC collects it AFTER: head ──▶ [20|──▶] [30|null] Edge: One-element list (head == tail): head = head.Next (= null) tail = null ← must also clear tail!
public T RemoveFirst()
{
    if (IsEmpty) throw new InvalidOperationException("List is empty");

    T value = head!.Data;
    head = head.Next;
    Count--;

    if (Count == 0) tail = null;  // Was one node — clear tail too
    return value;
}

6.2 — Delete Last Node — O(n)

Even with a tail reference, deleting the last node requires O(n) traversal. Why? You need to find the second-to-last node to update its Next to null and reassign tail. Since the singly linked list has no backward pointer, you must walk from head.

This is one of the key limitations of singly linked lists — it's why doubly linked lists (Part B) are often preferred.

DELETE LAST — List: 10 → 20 → 30 → null BEFORE: head ──▶ [10|──▶] [20|──▶] [30|null] ◀── tail Step 1: Traverse to the node BEFORE tail prev = node(20) Step 2: prev.Next = null → [20|null] Step 3: tail = prev → tail now points to node(20) node(30) has no references → GC collects it AFTER: head ──▶ [10|──▶] [20|null] ◀── tail
public T RemoveLast()
{
    if (IsEmpty) throw new InvalidOperationException("List is empty");

    // Only one node?
    if (head == tail)
    {
        T only = head!.Data;
        head = tail = null;
        Count--;
        return only;
    }

    // Traverse to the second-to-last node
    Node<T>? prev = head;
    while (prev!.Next != tail)
        prev = prev.Next;

    T value = tail!.Data;
    prev.Next = null;    // Disconnect last node
    tail = prev;           // Update tail to second-to-last
    Count--;
    return value;
}

6.3 — Delete by Value — O(n)

Find the first occurrence of a value and remove it. The pattern: use two pointers (prev and current) — when you find the node, rewire prev.Next to skip over current.

public bool Remove(T value)
{
    if (IsEmpty) return false;

    // Special case: removing the head
    if (head!.Data!.Equals(value))
    {
        RemoveFirst();
        return true;
    }

    Node<T>? prev = head;
    Node<T>? current = head.Next;

    while (current != null)
    {
        if (current.Data!.Equals(value))
        {
            prev!.Next = current.Next;  // Skip over current node
            if (current == tail)
                tail = prev;             // Update tail if removed last
            Count--;
            return true;
        }
        prev = current;
        current = current.Next;
    }
    return false;  // Value not found
}
DELETE BY VALUE — Remove node(20) from: 10 → 20 → 30 → null Start: prev=node(10), current=node(20) Does current.Data == 20? YES! → prev.Next = current.Next → node(10).Next = node(30) AFTER: head ──▶ [10|──▶] [30|null] node(20) is unreferenced → GC collects it
Section 7 — Search

Since a linked list is not indexed, searching requires traversal — O(n) in all cases (we cannot do binary search because we can't jump to a middle position).

// Returns true if value exists in list — O(n)
public bool Contains(T value)
{
    Node<T>? current = head;
    while (current != null)
    {
        if (current.Data!.Equals(value)) return true;
        current = current.Next;
    }
    return false;
}

// Returns the zero-based index of value, or -1 if not found — O(n)
public int IndexOf(T value)
{
    Node<T>? current = head;
    int index = 0;
    while (current != null)
    {
        if (current.Data!.Equals(value)) return index;
        current = current.Next;
        index++;
    }
    return -1;
}
Section 8 — Cycle Detection: Floyd's Algorithm

A cycle (or loop) occurs when a node's Next pointer points back to a previous node in the list, forming an infinite loop. This is a classic bug — your traversal code would loop forever. Floyd's Cycle Detection Algorithm (also called the "fast and slow pointer" or "tortoise and hare" algorithm) detects this in O(n) time and O(1) space.

🌍 Real-World Analogy — The Race Track:
Imagine two runners on a circular track. One runs at 1x speed, the other at 2x speed. If the track is circular (cycle exists), the fast runner will eventually lap the slow runner — they will be at the same position. If the track is a straight line (no cycle), the fast runner simply reaches the end.
FLOYD'S ALGORITHM — Two Pointer Technique List WITH a cycle: 1 → 2 → 3 → 4 → 5 ↑ ↓ └─── 6 ←───┘ (node 6's Next points back to node 3) slow pointer: moves 1 step at a time: 1, 2, 3, 4, 5, 6, 3, 4, 5, 6... fast pointer: moves 2 steps at time: 1, 3, 5, 3, 5, 3, 5, 3... Both in the cycle — they WILL meet! Steps: Start: slow=1, fast=1 Step 1: slow=2, fast=3 Step 2: slow=3, fast=5 Step 3: slow=4, fast=3 (fast wrapped in cycle) Step 4: slow=5, fast=5 ← THEY MET! Cycle detected!
// Floyd's Cycle Detection — O(n) time, O(1) space
public bool HasCycle()
{
    Node<T>? slow = head;  // moves 1 step
    Node<T>? fast = head;  // moves 2 steps

    while (fast != null && fast.Next != null)
    {
        slow = slow!.Next;          // 1 step
        fast = fast.Next.Next;     // 2 steps

        if (slow == fast) return true;  // They met — CYCLE!
    }
    return false;  // fast reached null — no cycle
}

// Find the START of the cycle (where the loop begins)
public Node<T>? FindCycleStart()
{
    Node<T>? slow = head;
    Node<T>? fast = head;

    // Phase 1: Detect if a cycle exists
    while (fast != null && fast.Next != null)
    {
        slow = slow!.Next;
        fast = fast.Next.Next;
        if (slow == fast) break;
    }

    if (fast == null || fast.Next == null) return null;  // No cycle

    // Phase 2: Reset slow to head, keep fast at meeting point.
    // Advance both 1 step at a time — they meet at cycle start.
    slow = head;
    while (slow != fast)
    {
        slow = slow!.Next;
        fast = fast!.Next;
    }
    return slow;  // Start of cycle
}
📌 Why Does Phase 2 Work? This is a beautiful math proof: Let F = distance from head to cycle start, C = cycle length, K = distance from cycle start to meeting point. When slow and fast meet: slow has traveled F+K steps. Fast has traveled F+K+n×C steps (for some n). Since fast travels 2× slow: 2(F+K) = F+K+n×C → F = n×C - K. This means: if you put one pointer at head and keep the other at the meeting point, advancing both by 1 step each time, they meet exactly at the cycle start after F steps!
Section 9 — Full SinglyLinkedList<T> Class

Here is the complete, production-quality implementation putting everything together:

public class Node<T>
{
    public T Data;
    public Node<T>? Next;
    public Node(T data) { Data = data; Next = null; }
}

public class SinglyLinkedList<T>
{
    private Node<T>? head;
    private Node<T>? tail;
    public int Count { get; private set; }
    public bool IsEmpty => Count == 0;

    // ── Add Operations ──────────────────────────────────
    public void AddFirst(T value)
    {
        Node<T> n = new(value);
        if (IsEmpty) { head = tail = n; }
        else         { n.Next = head; head = n; }
        Count++;
    }

    public void AddLast(T value)
    {
        Node<T> n = new(value);
        if (IsEmpty) { head = tail = n; }
        else         { tail!.Next = n; tail = n; }
        Count++;
    }

    public void AddAt(int index, T value)
    {
        if (index < 0 || index > Count) throw new ArgumentOutOfRangeException(nameof(index));
        if (index == 0)     { AddFirst(value); return; }
        if (index == Count) { AddLast(value);  return; }
        Node<T> newNode = new(value);
        Node<T>? prev = head;
        for (int i = 0; i < index - 1; i++) prev = prev!.Next;
        newNode.Next = prev!.Next;
        prev.Next = newNode;
        Count++;
    }

    // ── Remove Operations ───────────────────────────────
    public T RemoveFirst()
    {
        if (IsEmpty) throw new InvalidOperationException("List is empty");
        T val = head!.Data;
        head = head.Next;
        Count--;
        if (Count == 0) tail = null;
        return val;
    }

    public T RemoveLast()
    {
        if (IsEmpty) throw new InvalidOperationException("List is empty");
        if (head == tail) { T o = head!.Data; head = tail = null; Count--; return o; }
        Node<T>? prev = head;
        while (prev!.Next != tail) prev = prev.Next;
        T val = tail!.Data;
        prev.Next = null;
        tail = prev;
        Count--;
        return val;
    }

    public bool Remove(T value)
    {
        if (IsEmpty) return false;
        if (head!.Data!.Equals(value)) { RemoveFirst(); return true; }
        Node<T>? prev = head, curr = head.Next;
        while (curr != null)
        {
            if (curr.Data!.Equals(value))
            {
                prev!.Next = curr.Next;
                if (curr == tail) tail = prev;
                Count--;
                return true;
            }
            prev = curr; curr = curr.Next;
        }
        return false;
    }

    // ── Search & Access ─────────────────────────────────
    public bool  Contains(T value) => IndexOf(value) != -1;
    public int   IndexOf(T value)
    {
        Node<T>? c = head; int i = 0;
        while (c != null) { if (c.Data!.Equals(value)) return i; c = c.Next; i++; }
        return -1;
    }
    public T Get(int i)
    {
        if (i < 0 || i >= Count) throw new ArgumentOutOfRangeException();
        Node<T>? c = head;
        for (int j = 0; j < i; j++) c = c!.Next;
        return c!.Data;
    }
    public T PeekFirst() { if (IsEmpty) throw new InvalidOperationException(); return head!.Data; }
    public T PeekLast()  { if (IsEmpty) throw new InvalidOperationException(); return tail!.Data; }

    // ── Utilities ───────────────────────────────────────
    public void Print()
    {
        Node<T>? c = head;
        while (c != null) { Console.Write(c.Data + " → "); c = c.Next; }
        Console.WriteLine("null");
    }

    public bool HasCycle()
    {
        Node<T>? slow = head, fast = head;
        while (fast != null && fast.Next != null)
        {
            slow = slow!.Next; fast = fast.Next.Next;
            if (slow == fast) return true;
        }
        return false;
    }

    public void Reverse()
    {
        Node<T>? prev = null, current = head;
        tail = head;
        while (current != null)
        {
            Node<T>? next = current.Next;
            current.Next = prev;
            prev = current;
            current = next;
        }
        head = prev;
    }
}

// ── Usage Example ────────────────────────────────────────
var list = new SinglyLinkedList<int>();
list.AddLast(10);
list.AddLast(20);
list.AddLast(30);
list.AddFirst(5);
list.AddAt(2, 15);
list.Print();          // 5 → 10 → 15 → 20 → 30 → null
list.Remove(15);
list.Print();          // 5 → 10 → 20 → 30 → null
list.Reverse();
list.Print();          // 30 → 20 → 10 → 5 → null

9.1 — List Reversal — O(n)

Reversing a singly linked list in-place using three pointers is one of the most common interview questions. The key insight: you process one node at a time, always pointing it backward.

REVERSE LIST: 1 → 2 → 3 → 4 → null becomes 4 → 3 → 2 → 1 → null Use three pointers: prev, current, next Initial: prev=null, current=node(1), head→[1→2→3→4→null] Iteration 1: next = current.Next = node(2) [save next before overwriting] current.Next = prev = null [1 now points backward to null] prev = current = node(1) [advance prev] current = next = node(2) [advance current] State: null ←[1] [2→3→4→null] Iteration 2: next = node(3) current.Next = prev = node(1) [2 now points backward to 1] prev = node(2), current = node(3) State: null ←[1]←[2] [3→4→null] Iteration 3: next = node(4) current.Next = prev = node(2) [3 now points backward to 2] prev = node(3), current = node(4) Iteration 4: next = null current.Next = prev = node(3) [4 now points backward to 3] prev = node(4), current = null Loop ends (current == null) head = prev = node(4) RESULT: [4→3→2→1→null] ✓

🌍 Where Singly Linked Lists Are Used in Real Software:
  • Undo stacks — Text editors (VS Code, Word) store edit history as a linked list. Undo = RemoveFirst.
  • Hash table chaining — Each bucket in a HashMap stores a linked list of key-value pairs with the same hash.
  • Memory allocators — Free memory blocks are often managed as a linked list of available chunks.
  • Adjacency lists — Graph representations in algorithms (BFS, DFS) use lists of neighbor nodes.
  • Music playlists — A playlist where each song knows the "next song" to play.
