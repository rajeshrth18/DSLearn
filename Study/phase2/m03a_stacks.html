Section 1 — What is a Stack? LIFO Explained

A Stack is an ordered collection where items are added and removed from the same end, called the top. This gives it the property: Last In, First Out (LIFO) — the most recently added item is always the first to be removed.

🌍 Real-World Analogies:
  • Stack of plates: You always put a new plate on top and always take from the top. The first plate you put down is the last one you'll reach.
  • Browser back button: Every page you visit gets pushed onto a history stack. Clicking Back pops the most recent page.
  • Ctrl+Z (Undo): Every action you take is pushed onto an undo stack. Ctrl+Z pops and reverses the most recent action.
  • Function call stack: When function A calls function B, B is pushed. When B returns, it's popped — control returns to A.
STACK — LIFO Structure (Last In, First Out) PUSH 10: PUSH 20: PUSH 30: POP: ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │ 10 │ ← TOP │ 20 │ ← TOP │ 30 │ ← TOP │ 20 │ ← TOP └──────┘ ├──────┤ ├──────┤ ├──────┤ │ 10 │ │ 20 │ │ 10 │ └──────┘ ├──────┤ └──────┘ │ 10 │ Returns 30 └──────┘ Add to TOP (Push). Remove from TOP (Pop). No access to the middle — LIFO only.
📌 Key Property — Restricted Access: A stack intentionally limits your access. You cannot access an arbitrary element — you can only see and touch the top. This constraint is what makes stacks useful: it models scenarios where order matters and the most recent context is what you need.

A stack has exactly four core operations, all O(1):

💡 TryPop / TryPeek: In production code, prefer TryPop(out T value) and TryPeek(out T value) which return false instead of throwing on an empty stack. C#'s built-in Stack<T> provides both patterns.
Section 3 — Array-Based Stack Implementation

The simplest implementation uses an array. The "top" is tracked by an index. Push increments the index and stores the value; Pop retrieves the value and decrements the index.

ARRAY-BASED STACK — Internal Structure backing array: [ 10 | 20 | 30 | | ] index: 0 1 2 3 4 ↑ topIndex = 2 Push(40): topIndex → 3, arr[3] = 40 Pop(): returns arr[3]=40, topIndex → 2 Peek(): returns arr[topIndex=2] = 30 (no index change)
public class ArrayStack<T>
{
    private T[] items;
    private int topIndex = -1;  // -1 means empty
    private int capacity;

    public int Count => topIndex + 1;
    public bool IsEmpty => topIndex == -1;
    public bool IsFull  => topIndex == capacity - 1;

    public ArrayStack(int capacity = 16)
    {
        this.capacity = capacity;
        items = new T[capacity];
    }

    public void Push(T item)
    {
        if (IsFull) Resize();  // Double capacity
        items[++topIndex] = item;
    }

    public T Pop()
    {
        if (IsEmpty) throw new InvalidOperationException("Stack underflow");
        T value = items[topIndex];
        items[topIndex--] = default!;  // Clear reference for GC
        return value;
    }

    public T Peek()
    {
        if (IsEmpty) throw new InvalidOperationException("Stack is empty");
        return items[topIndex];
    }

    public bool TryPop(out T value)
    {
        if (IsEmpty) { value = default!; return false; }
        value = Pop(); return true;
    }

    public bool TryPeek(out T value)
    {
        if (IsEmpty) { value = default!; return false; }
        value = Peek(); return true;
    }

    private void Resize()
    {
        capacity *= 2;
        T[] newArr = new T[capacity];
        Array.Copy(items, newArr, topIndex + 1);
        items = newArr;
    }
}

// Usage
var stack = new ArrayStack<int>();
stack.Push(10); stack.Push(20); stack.Push(30);
Console.WriteLine(stack.Peek());  // 30
Console.WriteLine(stack.Pop());   // 30
Console.WriteLine(stack.Pop());   // 20
Console.WriteLine(stack.Count);   // 1
Section 4 — Linked-List Stack Implementation

A stack can also be built on top of a singly linked list. The head node becomes the top of the stack — Push and Pop both operate on the head, giving O(1) operations with no capacity limit.

public class LinkedStack<T>
{
    private class Node
    {
        public T Data;
        public Node? Next;
        public Node(T data) { Data = data; }
    }

    private Node? top;
    public int Count { get; private set; }
    public bool IsEmpty => Count == 0;

    public void Push(T item)
    {
        Node node = new(item) { Next = top };
        top = node;
        Count++;
    }

    public T Pop()
    {
        if (IsEmpty) throw new InvalidOperationException("Stack underflow");
        T value = top!.Data;
        top = top.Next;
        Count--;
        return value;
    }

    public T Peek()
    {
        if (IsEmpty) throw new InvalidOperationException();
        return top!.Data;
    }
}
// The top of the stack IS the head of the linked list.
// Push = AddFirst (O(1)), Pop = RemoveFirst (O(1)).
Section 5 — C#'s Built-in Stack<T>

In production C# code, use System.Collections.Generic.Stack<T>. It is array-based, auto-resizing, and provides all the operations you need. There is no reason to implement your own stack unless it's for learning or a very specific constraint.

using System.Collections.Generic;

Stack<int> stack = new Stack<int>();

// Push — add to top
stack.Push(10);
stack.Push(20);
stack.Push(30);

// Peek — view top without removing
Console.WriteLine(stack.Peek());   // 30

// Pop — remove and return top
Console.WriteLine(stack.Pop());    // 30
Console.WriteLine(stack.Count);    // 2

// TryPop / TryPeek — safe versions (no exception)
if (stack.TryPop(out int val))
    Console.WriteLine($"Popped: {val}");  // Popped: 20

if (!stack.TryPeek(out int top))
    Console.WriteLine("Stack is empty");

// Iteration (top-to-bottom order, does NOT remove items)
Stack<string> history = new([ "page1", "page2", "page3" ]);
foreach (var page in history)
    Console.WriteLine(page);  // page3, page2, page1 (top-first)

// Contains — O(n) linear search
bool has = history.Contains("page2");  // true

// ToArray — snapshot from top to bottom
string[] arr = history.ToArray();  // ["page3","page2","page1"]

// Clear
history.Clear();
Section 6 — Application: Balanced Parentheses

One of the most classic stack problems: given a string of brackets, determine if every opening bracket has a matching closing bracket in the correct order.

🌍 Real-World: Every compiler and IDE uses this exact algorithm to check syntax. When you type if (x > (a + b) and the IDE tells you "missing closing parenthesis", it used a stack to find the mismatch.
BALANCED PARENTHESES — Stack Trace Input: "({[]})" PUSH '(' → stack: ['('] PUSH '{' → stack: ['(', '{'] PUSH '[' → stack: ['(', '{', '['] READ ']' → peek='[', matches! POP → stack: ['(', '{'] READ '}' → peek='{', matches! POP → stack: ['('] READ ')' → peek='(', matches! POP → stack: [] End of string, stack is empty → BALANCED ✓ Input: "([)]" PUSH '(' → ['('] PUSH '[' → ['(', '['] READ ')' → peek='[', does NOT match ')' → INVALID ✗
public static bool IsBalanced(string s)
{
    Stack<char> stack = new();

    foreach (char c in s)
    {
        switch (c)
        {
            case '(': case '[': case '{':
                stack.Push(c);  // Opening bracket — push
                break;

            case ')':
                if (!stack.TryPop(out char open) || open != '(') return false;
                break;
            case ']':
                if (!stack.TryPop(out open) || open != '[') return false;
                break;
            case '}':
                if (!stack.TryPop(out open) || open != '{') return false;
                break;
        }
    }
    return stack.Count == 0;  // All opened must be closed
}

// Test cases:
Console.WriteLine(IsBalanced("({[]})"));   // True
Console.WriteLine(IsBalanced("([)]"));     // False
Console.WriteLine(IsBalanced("((("));       // False (stack not empty)
Console.WriteLine(IsBalanced(""));          // True (empty string)
Section 7 — Application: Infix to Postfix (Shunting Yard)

We normally write expressions in infix notation: 3 + 4 * 2. Computers evaluate them more easily in postfix (Reverse Polish Notation): 3 4 2 * +. The Shunting Yard algorithm by Edsger Dijkstra converts between them using a stack.

📌 Postfix (RPN) Rules:
  • Operands appear in the same order as infix
  • Operators appear after their operands
  • No parentheses needed — precedence is implicit in position
  • 3 + 4 * 2 = 3 4 2 * + = 3 + 8 = 11
  • (3 + 4) * 2 = 3 4 + 2 * = 7 * 2 = 14
// Shunting Yard Algorithm — Infix to Postfix
public static string InfixToPostfix(string infix)
{
    Stack<char> opStack = new();
    StringBuilder output = new();

    static int Prec(char op) => op switch
    {
        '+' or '-' => 1,
        '*' or '/' => 2,
        '^'         => 3,
        _            => 0
    };

    foreach (char c in infix)
    {
        if (char.IsDigit(c) || char.IsLetter(c))
        {
            output.Append(c).Append(' ');  // Operand → output
        }
        else if (c == '(')
        {
            opStack.Push(c);
        }
        else if (c == ')')
        {
            // Pop until matching '('
            while (opStack.Count > 0 && opStack.Peek() != '(')
                output.Append(opStack.Pop()).Append(' ');
            opStack.Pop();  // Discard '('
        }
        else  // Operator
        {
            while (opStack.Count > 0 && Prec(opStack.Peek()) >= Prec(c))
                output.Append(opStack.Pop()).Append(' ');
            opStack.Push(c);
        }
    }

    while (opStack.Count > 0)
        output.Append(opStack.Pop()).Append(' ');

    return output.ToString().Trim();
}

Console.WriteLine(InfixToPostfix("3+4*2"));       // "3 4 2 * +"
Console.WriteLine(InfixToPostfix("(3+4)*2"));     // "3 4 + 2 *"
Console.WriteLine(InfixToPostfix("a+b*(c-d)/e")); // "a b c d - * e / +"
Section 8 — Application: Postfix Evaluation

Once you have a postfix expression, evaluating it is clean: scan left to right — when you see a number, push it; when you see an operator, pop two numbers, apply the operator, push the result.

EVALUATE "3 4 2 * +" (answer should be 11) Token "3" → PUSH 3 stack: [3] Token "4" → PUSH 4 stack: [3, 4] Token "2" → PUSH 2 stack: [3, 4, 2] Token "*" → POP 2, POP 4, compute 4*2=8, PUSH 8 stack: [3, 8] Token "+" → POP 8, POP 3, compute 3+8=11, PUSH 11 stack: [11] End: POP 11 → ANSWER = 11 ✓
public static double EvaluatePostfix(string expr)
{
    Stack<double> stack = new();
    string[] tokens = expr.Split(' ', StringSplitOptions.RemoveEmptyEntries);

    foreach (string token in tokens)
    {
        if (double.TryParse(token, out double num))
        {
            stack.Push(num);
        }
        else
        {
            double b = stack.Pop();  // Right operand (popped first)
            double a = stack.Pop();  // Left operand
            double result = token switch
            {
                "+" => a + b,
                "-" => a - b,
                "*" => a * b,
                "/" => a / b,
                "^" => Math.Pow(a, b),
                _   => throw new ArgumentException($"Unknown op: {token}")
            };
            stack.Push(result);
        }
    }
    return stack.Pop();  // Final result
}

Console.WriteLine(EvaluatePostfix("3 4 2 * +"));   // 11
Console.WriteLine(EvaluatePostfix("3 4 + 2 *"));   // 14
Console.WriteLine(EvaluatePostfix("5 1 2 + 4 * + 3 -")); // 14
Section 9 — Application: The Call Stack

The most important stack you'll never directly touch: the call stack. Every time a function is called, the runtime creates a stack frame containing the function's local variables, parameters, and return address, and pushes it onto the call stack. When the function returns, its frame is popped.

THE CALL STACK — Function A() calls B() calls C() void A() { → void B() { → void C() { B(); C(); // work } } } CALL STACK (grows downward): ┌─────────────────────────┐ │ A's frame │ ← pushed first │ (local vars, ret addr) │ ├─────────────────────────┤ │ B's frame │ ← pushed when A calls B │ (local vars, ret addr) │ ├─────────────────────────┤ │ C's frame │ ← pushed when B calls C (TOP = currently executing) └─────────────────────────┘ C() returns → C's frame POPPED → execution returns to B B() returns → B's frame POPPED → execution returns to A A() returns → A's frame POPPED → program done
🌍 StackOverflowException: When you write infinite recursion in C#, you see StackOverflowException. This happens because each recursive call pushes a new frame — and the call stack has a finite size (~1 MB by default). When it's full, the runtime throws the exception. This is why iterative algorithms (using an explicit stack) are often preferred over deep recursion. 
// Converting recursion to iteration using an explicit stack
// Example: DFS traversal of a tree (avoids stack overflow on deep trees)

public static void DFSIterative(TreeNode? root)
{
    if (root == null) return;

    Stack<TreeNode> stack = new();
    stack.Push(root);

    while (stack.Count > 0)
    {
        TreeNode node = stack.Pop();
        Console.Write(node.Val + " ");

        // Push right first so left is processed first (LIFO)
        if (node.Right != null) stack.Push(node.Right);
        if (node.Left  != null) stack.Push(node.Left);
    }
}
// This exactly simulates the recursive call stack with an explicit one.
