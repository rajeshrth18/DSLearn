Section 1 вҖ” DFS (Depth-First Search)

DFS dives as deep as possible along each branch before backtracking. It's naturally recursive вҖ” the call stack is the "path so far". Unlike BFS, DFS does not guarantee shortest paths, but it's better for:

  • Detecting cycles
  • Topological ordering
  • Finding all paths / connected components
  • Tree-like traversals on graphs
DFS on graph: 0в”Җ1, 0в”Җ2, 1в”Җ3, 1в”Җ4, 2в”Җ5 [0] / \ [1] [2] / \ \ [3] [4] [5] DFS from node 0 (visiting smaller neighbor first): Visit 0 вҶ’ recurse into 1 Visit 1 вҶ’ recurse into 3 Visit 3 вҶ’ no unvisited neighbors вҶ’ return (post-order: process 3) recurse into 4 Visit 4 вҶ’ no unvisited neighbors вҶ’ return (post-order: process 4) post-order: process 1 recurse into 2 Visit 2 вҶ’ recurse into 5 Visit 5 вҶ’ return (post-order: process 5) post-order: process 2 post-order: process 0 Pre-order (visit when entering): 0, 1, 3, 4, 2, 5 Post-order (visit when leaving): 3, 4, 1, 5, 2, 0 вҶҗ useful for topological sort
// Recursive DFS
public void DFS(List<int>[] graph, int node, bool[] visited)
{
    visited[node] = true;
    // Pre-order action: Console.Write(node + " ");
    foreach (int nb in graph[node])
        if (!visited[nb]) DFS(graph, nb, visited);
    // Post-order action: result.Add(node); (for topological sort)
}

// Iterative DFS (using a stack)
public void DFSIterative(List<int>[] graph, int start)
{
    bool[] visited = new bool[graph.Length];
    var stack = new Stack<int>();
    stack.Push(start);
    while (stack.Count > 0)
    {
        int node = stack.Pop();
        if (visited[node]) continue;
        visited[node] = true;
        // Process node here
        foreach (int nb in graph[node])
            if (!visited[nb]) stack.Push(nb);
    }
}
// Time: O(V+E), Space: O(V) for visited + recursion stack
// Note: iterative DFS visits in reverse neighbor order vs recursive
Section 2 вҖ” Cycle Detection

2.1 вҖ” Undirected Graph: Parent Tracking

In an undirected graph, a cycle exists if DFS visits an already-visited neighbor that is not the parent of the current node.

UNDIRECTED cycle detection: 0в”Җ1, 1в”Җ2, 2в”Җ0 (triangle) DFS from 0, parent=-1: Visit 0 (parent=-1), neighbors: [1, 2] Visit 1 (parent=0), neighbors: [0, 2] 0 is visited AND 0 != parent(1)? No, parent=0, skip. Visit 2 (parent=1), neighbors: [1, 0] 1 is visited AND 1 != parent(2)? No, parent=1, skip. 0 is visited AND 0 != parent(2)? YES (parent is 1) вҶ’ CYCLE FOUND вң“
public bool HasCycleUndirected(List<int>[] graph)
{
    bool[] visited = new bool[graph.Length];
    for (int i = 0; i < graph.Length; i++)
        if (!visited[i] && DFSCycleUndirected(graph, i, -1, visited))
            return true;
    return false;
}
private bool DFSCycleUndirected(List<int>[] g, int node, int parent, bool[] vis)
{
    vis[node] = true;
    foreach (int nb in g[node])
    {
        if (!vis[nb]) { if (DFSCycleUndirected(g, nb, node, vis)) return true; }
        else if (nb != parent) return true; // visited non-parent вҶ’ cycle
    }
    return false;
}

2.2 вҖ” Directed Graph: 3-Color State (White/Gray/Black)

In a directed graph, "visited" isn't enough вҖ” we need to know if a node is on the current DFS path. Use 3 states:

  • 0 (White) вҖ” unvisited
  • 1 (Gray) вҖ” currently in the DFS stack (in progress)
  • 2 (Black) вҖ” fully processed (all descendants explored)

A cycle exists when we reach a Gray node (we're already exploring it вҖ” we've come back around).

DIRECTED cycle detection: 0вҶ’1вҶ’2вҶ’0 (cycle) State: 0=White, 1=Gray, 2=Black DFS from 0: 0: WhiteвҶ’Gray, explore neighbor 1 1: WhiteвҶ’Gray, explore neighbor 2 2: WhiteвҶ’Gray, explore neighbor 0 0 is GRAY вҶ’ CYCLE DETECTED вң“ No cycle example: 0вҶ’1вҶ’2 (no back edge) 0: WhiteвҶ’Gray вҶ’ 1: WhiteвҶ’Gray вҶ’ 2: WhiteвҶ’Gray вҶ’ no gray neighbors 2: GrayвҶ’Black вҶ’ 1: GrayвҶ’Black вҶ’ 0: GrayвҶ’Black All black = no cycle
// Directed cycle detection using 3-state coloring
public bool HasCycleDirected(List<int>[] graph)
{
    int[] state = new int[graph.Length]; // 0=white, 1=gray, 2=black
    for (int i = 0; i < graph.Length; i++)
        if (state[i] == 0 && DFSCycleDirected(graph, i, state))
            return true;
    return false;
}
private bool DFSCycleDirected(List<int>[] g, int node, int[] state)
{
    state[node] = 1; // gray = in current path
    foreach (int nb in g[node])
    {
        if (state[nb] == 1) return true;  // gray neighbor вҶ’ back edge вҶ’ cycle
        if (state[nb] == 0 && DFSCycleDirected(g, nb, state)) return true;
    }
    state[node] = 2; // black = fully explored
    return false;
}
Section 3 вҖ” Topological Sort

Topological sort orders nodes of a DAG (Directed Acyclic Graph) such that for every directed edge uвҶ’v, u comes before v in the ordering. This is the fundamental algorithm behind: dependency resolution, build systems, course prerequisites.

TOPOLOGICAL SORT вҖ” Course Schedule example Prerequisites: 0вҶ’2, 1вҶ’2, 1вҶ’3, 2вҶ’4, 3вҶ’4 (вҶ’ means "is prerequisite of") Dependency graph: 0 в”Җв”ҖвҶ’ 2 в”Җв”ҖвҶ’ 4 вҶ‘ вҶ‘ 1 в”Җв”Җв”Җв”ҖвҶ’в”ҳ в”Ӯ в””в”Җв”Җв”Җв”Җв”ҖвҶ’ 3 в”Җв”Җв”ҖвҶ’в”ҳ Valid topological orders: [0,1,2,3,4] or [1,0,3,2,4] etc. Invalid: 2 before 0 (violates 0вҶ’2) KAHN'S ALGORITHM (BFS-based): 1. Compute in-degree of every node 2. Enqueue all nodes with in-degree = 0 (no prerequisites) 3. BFS: dequeue node, add to result, decrease in-degree of neighbors 4. When neighbor's in-degree reaches 0, enqueue it 5. If result.Count == n, valid DAG. Otherwise вҶ’ cycle exists.
// Kahn's Algorithm вҖ” Topological Sort (BFS)
public List<int> TopologicalSort(int n, int[][] edges)
{
    List<int>[] adj = new List<int>[n];
    int[] inDeg = new int[n];
    for (int i = 0; i < n; i++) adj[i] = new();

    foreach (var e in edges) { adj[e[0]].Add(e[1]); inDeg[e[1]]++; }

    var q = new Queue<int>();
    for (int i = 0; i < n; i++) if (inDeg[i] == 0) q.Enqueue(i);

    var order = new List<int>();
    while (q.Count > 0)
    {
        int node = q.Dequeue();
        order.Add(node);
        foreach (int nb in adj[node])
        {
            inDeg[nb]--;
            if (inDeg[nb] == 0) q.Enqueue(nb);
        }
    }
    return order.Count == n ? order : new(); // empty = cycle exists
}
// Time: O(V+E), Space: O(V+E)

// DFS post-order Topological Sort (alternative)
public List<int> TopologicalSortDFS(int n, List<int>[] adj)
{
    int[] state = new int[n]; // 0=unvisited,1=visiting,2=done
    var result = new Stack<int>();
    bool hasCycle = false;

    void Dfs(int v) {
        if (hasCycle) return;
        state[v] = 1;
        foreach (int nb in adj[v])
        {
            if (state[nb] == 1) { hasCycle = true; return; }
            if (state[nb] == 0) Dfs(nb);
        }
        state[v] = 2;
        result.Push(v); // push AFTER all descendants вҖ” post-order
    }
    for (int i = 0; i < n; i++) if (state[i] == 0) Dfs(i);
    return hasCycle ? new() : result.ToList();
}
// Post-order gives REVERSE topological order вҶ’ pop stack for correct order
Kahn's vs DFS Topo Sort:
вҖў Kahn's naturally detects cycles (result.Count < n) and processes in topological order directly.
вҖў DFS post-order uses the 3-color trick; you get the order by reversing the post-order list.
вҖў Both are O(V+E). Prefer Kahn's for clarity in interviews.
Section 4 вҖ” Connected Components
CONNECTED COMPONENTS Graph (undirected, 7 nodes, 3 components): 0в”Җ1в”Җ2 3в”Җ4 5 6в”Җ7 Component 1: {0,1,2} Component 2: {3,4} Component 3: {5} Component 4: {6,7} Count = 4
// Count connected components in undirected graph
public int CountComponents(int n, int[][] edges)
{
    List<int>[] adj = new List<int>[n];
    for (int i = 0; i < n; i++) adj[i] = new();
    foreach (var e in edges) { adj[e[0]].Add(e[1]); adj[e[1]].Add(e[0]); }

    bool[] vis = new bool[n]; int count = 0;
    for (int i = 0; i < n; i++)
    {
        if (!vis[i]) { count++; DFS(adj, i, vis); }
    }
    return count;
}

// "Graph Valid Tree" (valid tree = n-1 edges + 1 component + no cycle)
Section 5 вҖ” Dijkstra's Algorithm

Dijkstra's finds the shortest path from one source to all other nodes in a weighted graph with non-negative edge weights. It is the backbone of GPS routing, network routing protocols, and game pathfinding.

DIJKSTRA on weighted graph (source = 0): 1 в”Җв”Җ5в”Җв”Җ 3 / \ 0в”Җв”Җ2в”Җв”Җ 2в”Җв”Җ1в”Җв”Җ4 \ 1 в”Җв”Җ6в”Җв”Җ ... Edges: (0,1,4), (0,2,2), (1,3,5), (2,1,1), (2,3,8), (3,4,2), (2,4,10) Initial: dist=[0,вҲһ,вҲһ,вҲһ,вҲһ], pq=[(0,src=0)] Step 1: pop (dist=0, node=0). Relax: dist[1]=min(вҲһ, 0+4)=4 вҶ’ push (4,1) dist[2]=min(вҲһ, 0+2)=2 вҶ’ push (2,2) pq=[(2,2),(4,1)] Step 2: pop (dist=2, node=2). Relax: dist[1]=min(4, 2+1)=3 вҶ’ push (3,1) dist[3]=min(вҲһ, 2+8)=10 вҶ’ push (10,3) dist[4]=min(вҲһ, 2+10)=12 вҶ’ push (12,4) pq=[(3,1),(4,1),(10,3),(12,4)] Step 3: pop (dist=3, node=1). Relax: dist[3]=min(10, 3+5)=8 вҶ’ push (8,3) pq=[(4,1-stale),(8,3),(10,3-stale),(12,4)] Step 4: pop (dist=4, node=1) вҖ” stale (dist[1]=3 < 4) вҶ’ SKIP Step 5: pop (dist=8, node=3). Relax: dist[4]=min(12, 8+2)=10 вҶ’ push (10,4) Step 6: pop stale (10,3) вҶ’ skip. pop (10,4) вҶ’ done. Final distances: [0, 3, 2, 8, 10]
// Dijkstra's shortest path вҖ” O((V+E) log V)
public int[] Dijkstra(List<(int to, int w)>[] graph, int src)
{
    int n = graph.Length;
    int[] dist = new int[n];
    Array.Fill(dist, int.MaxValue);
    dist[src] = 0;

    // PriorityQueue<node, dist> вҖ” min-heap on distance
    var pq = new PriorityQueue<int, int>();
    pq.Enqueue(src, 0);

    while (pq.Count > 0)
    {
        pq.TryDequeue(out int node, out int d);
        if (d > dist[node]) continue; // stale entry вҖ” skip

        foreach (var (nb, w) in graph[node])
        {
            int newDist = dist[node] + w;
            if (newDist < dist[nb])
            {
                dist[nb] = newDist;
                pq.Enqueue(nb, newDist);
            }
        }
    }
    return dist; // dist[i] = shortest distance from src to i (int.MaxValue = unreachable)
}
// Time: O((V+E) log V)  Space: O(V)
// CRITICAL: Skip stale heap entries with (d > dist[node]) check
// Works only with non-negative weights!

// Example: Network Delay Time (LC 743)
public int NetworkDelayTime(int[][] times, int n, int k)
{
    var g = new List<(int,int)>[n + 1];
    for (int i = 0; i <= n; i++) g[i] = new();
    foreach (var t in times) g[t[0]].Add((t[1], t[2]));
    var dist = Dijkstra(g, k);
    int max = dist.Skip(1).Max();
    return max == int.MaxValue ? -1 : max;
}
вҡ пёҸ Dijkstra Limitation: Dijkstra fails with negative weight edges. For graphs with negative weights, use Bellman-Ford (next section).
Section 6 вҖ” Bellman-Ford Algorithm

Bellman-Ford handles negative edge weights and can detect negative-weight cycles. It relaxes all edges VвҲ’1 times. Trade-off: slower than Dijkstra вҖ” O(VE) vs O((V+E) log V).

BELLMAN-FORD вҖ” Why relax V-1 times? In the worst case, the shortest path visits all V vertices вҶ’ at most V-1 edges. Each iteration of relaxing all E edges "extends" shortest paths by one more edge. Example: path 0вҶ’1вҶ’2вҶ’3 Iteration 1: dist[1] = dist[0]+w(0,1) = correct dist to 1 Iteration 2: dist[2] = dist[1]+w(1,2) = correct dist to 2 Iteration 3: dist[3] = dist[2]+w(2,3) = correct dist to 3 After V-1=3 iterations, all shortest paths are found. NEGATIVE CYCLE DETECTION: Do a V-th relaxation. If any dist changes вҶ’ negative cycle exists (distances would decrease infinitely around the cycle)
public int[] BellmanFord(int n, int[][] edges, int src)
{
    int[] dist = new int[n];
    Array.Fill(dist, int.MaxValue);
    dist[src] = 0;

    for (int i = 0; i < n - 1; i++) // relax V-1 times
    {
        bool updated = false;
        foreach (var e in edges)  // e = [u, v, weight]
        {
            int u=e[0], v=e[1], w=e[2];
            if (dist[u] != int.MaxValue && dist[u] + w < dist[v])
            { dist[v] = dist[u] + w; updated = true; }
        }
        if (!updated) break; // early termination optimization
    }

    // Check for negative cycle: do one more relaxation
    foreach (var e in edges)
    {
        int u=e[0], v=e[1], w=e[2];
        if (dist[u] != int.MaxValue && dist[u] + w < dist[v])
            throw new InvalidOperationException("Negative cycle detected");
    }
    return dist;
}
// Time: O(VГ—E), Space: O(V)
// LC 787 вҖ” Cheapest Flights Within K Stops:
//   Use Bellman-Ford with exactly K+1 iterations (K+1 edges = K stops)
//   Copy dist array before each iteration to avoid using current-iteration updates
Section 7 вҖ” Union-Find (Disjoint Set Union)

Union-Find is a data structure that tracks a collection of elements partitioned into disjoint (non-overlapping) sets. It supports two operations in near-O(1) time: Find (which set does element x belong to?) and Union (merge two sets).

Critical uses: cycle detection, Kruskal's MST, number of connected components, network connectivity.

UNION-FIND example вҖ” processing edges: (0,1),(2,3),(0,2),(1,4) Initial: each node is its own root parent = [0, 1, 2, 3, 4] rank = [0, 0, 0, 0, 0] Union(0,1): root(0)=0, root(1)=1 вҶ’ merge вҶ’ parent[1]=0 parent = [0, 0, 2, 3, 4] Union(2,3): root(2)=2, root(3)=3 вҶ’ merge вҶ’ parent[3]=2 parent = [0, 0, 2, 2, 4] Union(0,2): root(0)=0, root(2)=2 вҶ’ merge вҶ’ parent[2]=0 parent = [0, 0, 0, 2, 4] Union(1,4): root(1)=0, root(4)=4 вҶ’ merge вҶ’ parent[4]=0 parent = [0, 0, 0, 2, 0] After path compression вҖ” all point to root 0: parent = [0, 0, 0, 0, 0] All 5 nodes in ONE component. No cycle (4 unions, 5 nodes).
public class UnionFind
{
    private int[] _parent, _rank;
    public int Components { get; private set; }

    public UnionFind(int n)
    {
        _parent = Enumerable.Range(0, n).ToArray(); // parent[i] = i initially
        _rank = new int[n];
        Components = n;
    }

    // Find with path compression: make all nodes point directly to root
    public int Find(int x)
    {
        if (_parent[x] != x) _parent[x] = Find(_parent[x]); // path compression
        return _parent[x];
    }

    // Union by rank: attach smaller tree under larger tree
    public bool Union(int x, int y)
    {
        int rx = Find(x), ry = Find(y);
        if (rx == ry) return false; // already same set вҖ” adding edge creates cycle!
        if (_rank[rx] < _rank[ry]) (rx, ry) = (ry, rx); // ensure rx has higher rank
        _parent[ry] = rx;
        if (_rank[rx] == _rank[ry]) _rank[rx]++;
        Components--;
        return true;
    }

    public bool Connected(int x, int y) => Find(x) == Find(y);
}
// With path compression + union by rank:
// Find/Union вүҲ O(Оұ(n)) вүҲ O(1) amortized
// Оұ = inverse Ackermann function вҖ” grows so slowly it's effectively constant

// Kruskal's Minimum Spanning Tree:
public int KruskalMST(int n, int[][] edges)
{
    Array.Sort(edges, (a,b) => a[2] - b[2]); // sort by weight ascending
    var uf = new UnionFind(n);
    int totalWeight = 0, edgesUsed = 0;
    foreach (var e in edges)
    {
        if (uf.Union(e[0], e[1]))  // if not same component (no cycle)
        {
            totalWeight += e[2];
            if (++edgesUsed == n-1) break; // MST complete: n-1 edges
        }
    }
    return edgesUsed == n-1 ? totalWeight : -1; // -1 = disconnected graph
}
// Time: O(E log E) for sort + O(E Оұ(V)) for union-find вүҲ O(E log E)

Key Takeaways:
вҖў DFS = recursion/stack. Pre-order (visiting) vs post-order (returning) serve different purposes.
вҖў Cycle detection: undirected = parent tracking; directed = gray/black state coloring.
вҖў Topological sort (Kahn's) = BFS with in-degree counting. Detects cycles if result < n.
вҖў Dijkstra = greedy min-heap. Always skip stale entries. Non-negative weights only.
вҖў Bellman-Ford = relax all edges V-1 times. Handles negatives. Detects negative cycles.
вҖў Union-Find = near-O(1) connectivity. Path compression + union by rank = best practice.
