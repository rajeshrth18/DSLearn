Easy Problems

#1 Binary Search Easy

Problem: Given a sorted array of distinct integers and a target, return the index of the target or -1 if not found. Must run in O(log n).

Example: nums = [-1,0,3,5,9,12], target = 9 → 4

Hint: Classic binary search. Use lo = 0, hi = n-1. Check mid, move lo or hi. Loop while lo <= hi.
public int Search(int[] nums, int target)
{
    int lo = 0, hi = nums.Length - 1;
    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;
        if      (nums[mid] == target) return mid;
        else if (nums[mid] <  target) lo = mid + 1;
        else                          hi = mid - 1;
    }
    return -1;
}
// Time: O(log n), Space: O(1)
#2 Search Insert Position Easy

Problem: Given a sorted array and a target, return the index if found. If not found, return the index where it would be inserted to keep the array sorted.

Example: [1,3,5,6], target=5 → 2; target=2 → 1; target=7 → 4

Hint: This is the lower bound operation — first index i where nums[i] >= target. Use hi = nums.Length (exclusive) and lo < hi loop.
public int SearchInsert(int[] nums, int target)
{
    int lo = 0, hi = nums.Length; // hi is exclusive — can be at end
    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] < target) lo = mid + 1;
        else                    hi = mid;
    }
    return lo; // Lower bound = insertion point
}
// When loop ends: lo == hi == first index where nums[i] >= target
#3 First Bad Version Easy

Problem: You have n versions [1..n]. Once a version is bad, all subsequent versions are bad. Given API bool IsBadVersion(int version), find the first bad version with minimum API calls.

Example: n=5, bad=4. IsBadVersion(3)=false, IsBadVersion(4)=true → answer: 4

Hint: The versions form: [F,F,F,...,T,T,T]. You need the leftmost T. Binary search: if mid is bad, answer might be mid or earlier (hi=mid). If good, answer must be after (lo=mid+1).
public int FirstBadVersion(int n)
{
    int lo = 1, hi = n;
    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (IsBadVersion(mid))
            hi = mid;       // mid is bad, but might not be first
        else
            lo = mid + 1; // mid is good, first bad is after mid
    }
    return lo; // lo == hi == first bad version
}
// Time: O(log n), Space: O(1). Minimum API calls!
#4 Squares of a Sorted Array Easy

Problem: Given an integer array sorted in non-decreasing order (may contain negatives), return an array of the squares of each number, also sorted in non-decreasing order. Do it in O(n).

Example: [-4,-1,0,3,10] → [0,1,9,16,100]

Hint: Largest squares are at both ends (largest negative or largest positive). Use two pointers converging from both ends, filling result from the back (largest to smallest).
public int[] SortedSquares(int[] nums)
{
    int n = nums.Length;
    int[] result = new int[n];
    int lo = 0, hi = n - 1, pos = n - 1; // Fill from end

    while (lo <= hi)
    {
        int lSq = nums[lo] * nums[lo];
        int rSq = nums[hi] * nums[hi];
        if (lSq > rSq)
        { result[pos--] = lSq; lo++; }
        else
        { result[pos--] = rSq; hi--; }
    }
    return result;
}
// Time: O(n), Space: O(n) for output

Medium Problems

#5 Search in Rotated Sorted Array Medium

Problem: A sorted array was rotated at some unknown pivot. Given the rotated array and a target, return the index or -1. Must be O(log n).

Example: [4,5,6,7,0,1,2], target=0 → 4; target=3 → -1

Hint: At mid, one of the two halves is always sorted. Check which half by comparing arr[lo] to arr[mid]. Then check if target falls in the sorted half. If yes, search there; otherwise search the other half.
public int Search(int[] nums, int target)
{
    int lo = 0, hi = nums.Length - 1;
    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target) return mid;

        if (nums[lo] <= nums[mid]) // Left half is sorted
        {
            if (nums[lo] <= target && target < nums[mid])
                hi = mid - 1;
            else
                lo = mid + 1;
        }
        else // Right half is sorted
        {
            if (nums[mid] < target && target <= nums[hi])
                lo = mid + 1;
            else
                hi = mid - 1;
        }
    }
    return -1;
}
// Time: O(log n), Space: O(1)
#6 Find Peak Element Medium

Problem: A peak element is strictly greater than its neighbors. Given nums where nums[-1] = nums[n] = -infinity, find any peak index in O(log n).

Example: [1,2,3,1] → 2 (nums[2]=3 is peak); [1,2,1,3,5,6,4] → 5

Hint: If nums[mid] < nums[mid+1], then a peak exists in the right half (the slope is going up, so either mid+1 is a peak or there's a higher peak further right). Otherwise, a peak exists in the left half (including mid).
public int FindPeakElement(int[] nums)
{
    int lo = 0, hi = nums.Length - 1;
    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] < nums[mid + 1])
            lo = mid + 1; // Ascending slope → peak is to the right
        else
            hi = mid;     // Descending slope → peak is mid or to the left
    }
    return lo; // lo == hi == a peak
}
// Key insight: you don't need to find THE peak, just ANY peak.
// If climbing right, go right. If descending, go left.
#7 Three Sum Medium

Problem: Find all unique triplets in nums that sum to zero. The solution set must not contain duplicate triplets.

Example: [-1,0,1,2,-1,-4] → [[-1,-1,2],[-1,0,1]]

Hint: Sort first. Fix one element nums[i], then use two pointers for the remaining pair. Skip duplicates at every level to avoid duplicate triplets.
public IList<IList<int>> ThreeSum(int[] nums)
{
    Array.Sort(nums);
    var result = new List<IList<int>>();

    for (int i = 0; i < nums.Length - 2; i++)
    {
        if (i > 0 && nums[i] == nums[i - 1]) continue; // Skip dup
        int lo = i + 1, hi = nums.Length - 1;

        while (lo < hi)
        {
            int sum = nums[i] + nums[lo] + nums[hi];
            if (sum == 0)
            {
                result.Add(new[] { nums[i], nums[lo], nums[hi] });
                while (lo < hi && nums[lo] == nums[lo + 1]) lo++;
                while (lo < hi && nums[hi] == nums[hi - 1]) hi--;
                lo++; hi--;
            }
            else if (sum < 0) lo++;
            else              hi--;
        }
    }
    return result;
}
// Time: O(n²), Space: O(1) excluding output
#8 Container With Most Water Medium

Problem: Given n vertical lines at x=0..n-1 with heights in array height, find two lines that, together with the x-axis, form a container that holds the most water.

Example: [1,8,6,2,5,4,8,3,7] → 49

Hint: Water = min(height[lo], height[hi]) * (hi - lo). Start at both ends. Moving the taller wall can only make things worse (distance decreases, height bounded by shorter). So always move the shorter wall inward.
public int MaxArea(int[] height)
{
    int lo = 0, hi = height.Length - 1, maxWater = 0;
    while (lo < hi)
    {
        int water = Math.Min(height[lo], height[hi]) * (hi - lo);
        maxWater = Math.Max(maxWater, water);
        if (height[lo] < height[hi]) lo++;
        else                          hi--;
    }
    return maxWater;
}
// Proof: if we move the taller wall, new water ≤ old water (shorter wall still limits)
// Moving the shorter wall gives a chance of finding a taller wall to improve
#9 Longest Substring Without Repeating Characters Medium

Problem: Given a string s, find the length of the longest substring without repeating characters.

Example: "abcabcbb" → 3 ("abc"); "pwwkew" → 3 ("wke")

Hint: Sliding window with a dictionary tracking the last seen index of each character. When you see a repeated character, jump left pointer to just after the previous occurrence (if it's inside the window).
public int LengthOfLongestSubstring(string s)
{
    var lastIdx = new Dictionary<char, int>();
    int maxLen = 0, left = 0;

    for (int right = 0; right < s.Length; right++)
    {
        if (lastIdx.TryGetValue(s[right], out int prev) && prev >= left)
            left = prev + 1; // Jump left past previous occurrence

        lastIdx[s[right]] = right;
        maxLen = Math.Max(maxLen, right - left + 1);
    }
    return maxLen;
}
// Time: O(n), Space: O(min(n, alphabet_size))
// Key: prev >= left check ensures we don't jump left backward
#10 Minimum in Rotated Sorted Array Medium

Problem: Given a sorted array rotated between 1 and n times (all elements unique), find the minimum element in O(log n).

Example: [3,4,5,1,2] → 1; [4,5,6,7,0,1,2] → 0

Hint: If nums[mid] > nums[hi], the minimum is in the right half (the rotation point is there). Otherwise it's in the left half (including mid). Loop until lo == hi.
public int FindMin(int[] nums)
{
    int lo = 0, hi = nums.Length - 1;
    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] > nums[hi])
            lo = mid + 1; // Rotation is in right half, min is there
        else
            hi = mid;     // Min is mid or to the left of mid
    }
    return nums[lo];
}
// Why compare to hi (not lo)? Because nums[lo] <= nums[mid] when NOT rotated.
// Comparing to hi always correctly identifies which side is "broken".
#11 Subarray Sum Equals K Medium

Problem: Given an integer array nums and an integer k, return the total number of continuous subarrays whose sum equals k.

Example: [1,1,1], k=2 → 2; [1,2,3], k=3 → 2

Hint: Prefix sum + hashmap. If prefix[j] - prefix[i] = k, then subarray [i..j-1] sums to k. For each j, count how many previous prefix sums equal prefix[j] - k.
public int SubarraySum(int[] nums, int k)
{
    var prefixCount = new Dictionary<int, int> { [0] = 1 }; // Empty prefix
    int sum = 0, count = 0;

    foreach (int num in nums)
    {
        sum += num;
        // How many prefixes had value (sum - k)?
        count += prefixCount.GetValueOrDefault(sum - k);
        prefixCount[sum] = prefixCount.GetValueOrDefault(sum) + 1;
    }
    return count;
}
// Time: O(n), Space: O(n)
// Works with negative numbers (unlike sliding window)

Hard Problems

#12 Median of Two Sorted Arrays Hard

Problem: Given two sorted arrays nums1 and nums2, find the median of the two arrays combined. Must be O(log(m+n)).

Example: [1,3], [2] → 2.0; [1,2], [3,4] → 2.5

Hint: Binary search on the partition point in the smaller array. We want to partition both arrays so the left half has (m+n)/2 elements total, and max(left halves) <= min(right halves). Binary search on partition1, derive partition2 = (m+n)/2 - partition1.
public double FindMedianSortedArrays(int[] nums1, int[] nums2)
{
    // Always binary search on the smaller array
    if (nums1.Length > nums2.Length) return FindMedianSortedArrays(nums2, nums1);

    int m = nums1.Length, n = nums2.Length;
    int lo = 0, hi = m;
    int half = (m + n + 1) / 2; // Left half size (rounds up for odd)

    while (lo <= hi)
    {
        int p1 = lo + (hi - lo) / 2; // Elements from nums1 in left half
        int p2 = half - p1;            // Elements from nums2 in left half

        int l1 = (p1 == 0) ? int.MinValue : nums1[p1 - 1];
        int r1 = (p1 == m) ? int.MaxValue : nums1[p1];
        int l2 = (p2 == 0) ? int.MinValue : nums2[p2 - 1];
        int r2 = (p2 == n) ? int.MaxValue : nums2[p2];

        if (l1 <= r2 && l2 <= r1) // Correct partition found
        {
            if ((m + n) % 2 == 1) return Math.Max(l1, l2); // Odd total
            return (Math.Max(l1, l2) + Math.Min(r1, r2)) / 2.0;  // Even total
        }
        else if (l1 > r2)
            hi = p1 - 1; // Too many from nums1
        else
            lo = p1 + 1; // Too few from nums1
    }
    throw new InvalidOperationException();
}
// Time: O(log(min(m,n))), Space: O(1)
// Key: find partition where max(left) <= min(right) across both arrays
#13 Minimum Window Substring Hard

Problem: Given strings s and t, return the minimum window in s that contains all characters of t. If no such window exists, return "".

Example: s="ADOBECODEBANC", t="ABC" → "BANC"

Hint: Variable sliding window. Track what you need (from t) and what you have (in current window). Expand right until all needed chars are covered. Then shrink left while still covered, updating minimum. Repeat.
public string MinWindow(string s, string t)
{
    var need = new Dictionary<char, int>();
    foreach (char c in t)
        need[c] = need.GetValueOrDefault(c) + 1;

    int have = 0, required = need.Count;
    var window = new Dictionary<char, int>();
    int left = 0, minLen = int.MaxValue, minStart = 0;

    for (int right = 0; right < s.Length; right++)
    {
        char rc = s[right];
        window[rc] = window.GetValueOrDefault(rc) + 1;
        if (need.ContainsKey(rc) && window[rc] == need[rc]) have++;

        while (have == required)
        {
            if (right - left + 1 < minLen)
            { minLen = right - left + 1; minStart = left; }

            char lc = s[left++];
            window[lc]--;
            if (need.ContainsKey(lc) && window[lc] < need[lc]) have--;
        }
    }
    return minLen == int.MaxValue ? "" : s.Substring(minStart, minLen);
}
// Time: O(|s| + |t|), Space: O(|s| + |t|)
#14 Sliding Window Maximum Hard

Problem: Given array nums and integer k, return the maximum value in each sliding window of size k. There are n-k+1 windows.

Example: [1,3,-1,-3,5,3,6,7], k=3 → [3,3,5,5,6,7]

Hint: Use a monotonic deque (decreasing) that stores indices. When adding a new element: 1) Remove from back while back element <= new element (they can never be max). 2) Remove from front if it's outside the window. 3) Front always holds the max index.
public int[] MaxSlidingWindow(int[] nums, int k)
{
    int n = nums.Length;
    int[] result = new int[n - k + 1];
    var deq = new LinkedList<int>(); // Stores INDICES, decreasing values

    for (int i = 0; i < n; i++)
    {
        // Remove indices outside current window
        while (deq.Count > 0 && deq.First!.Value < i - k + 1)
            deq.RemoveFirst();

        // Remove smaller elements from back (they'll never be max)
        while (deq.Count > 0 && nums[deq.Last!.Value] < nums[i])
            deq.RemoveLast();

        deq.AddLast(i);

        // Front of deque is the max of current window
        if (i >= k - 1)
            result[i - k + 1] = nums[deq.First!.Value];
    }
    return result;
}
// Time: O(n) — each element added/removed from deque at most once
// Space: O(k) for the deque
#15 Koko Eating Bananas — Binary Search on Answer Hard

Problem: Koko has piles of bananas. She eats at speed k bananas/hour. She has h hours. Find the minimum integer speed k so she finishes in h hours. Each hour she picks one pile and eats up to k bananas from it.

Example: piles=[3,6,7,11], h=8 → 4

Hint: Binary search on answer (speed k). Lower bound = 1, upper bound = max(piles). For a given speed k, hours needed = sum of ceil(pile/k). Binary search for minimum k where this sum ≤ h.
public int MinEatingSpeed(int[] piles, int h)
{
    int lo = 1, hi = piles.Max(); // Answer is in [1, max_pile]

    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        long hours = piles.Sum(p => (long)(p + mid - 1) / mid); // ceil(p/mid)

        if (hours <= h)
            hi = mid;       // Feasible — try slower (smaller k)
        else
            lo = mid + 1; // Too slow — must eat faster
    }
    return lo;
}
// Time: O(n log(max_pile)), Space: O(1)
// Why: feasibility is monotone (higher k is always feasible if lower k is)
// Tip: (p + mid - 1) / mid = ceil(p/mid) using integer arithmetic
