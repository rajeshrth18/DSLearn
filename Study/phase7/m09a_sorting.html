1. Why Sorting Matters

Sorting is foundational. It unlocks faster algorithms — once data is sorted, you can binary search in O(log n), find duplicates in O(n), and simplify two-pointer problems. Many "hard" problems become "medium" if you sort first.

Key Properties of Sorting Algorithms

  • Stable: Equal elements retain original relative order
  • In-place: Sorts within the array, O(1) extra space
  • Adaptive: Faster on nearly-sorted input
  • Comparison-based: Compares pairs of elements

Comparison Sort Lower Bound

Any comparison-based sort must make at least Ω(n log n) comparisons in the worst case. This is a mathematical lower bound — you cannot do better with comparisons alone.

Non-comparison sorts (Counting, Radix, Bucket) break this barrier using extra assumptions about input.

2. O(n²) Sorts — The Basics

These sorts are simple to implement and fine for small arrays (n < 1000), but become impractical for large inputs. They're important for interview explanations and as building blocks.

Bubble Sort

Repeatedly compare adjacent elements and swap if they're in the wrong order. Each pass "bubbles" the largest unsorted element to its correct position.

Array: [5, 3, 8, 1, 2] Pass 1: compare pairs left-to-right, swap if needed [5,3] → swap → [3,5,8,1,2] [5,8] → ok → [3,5,8,1,2] [8,1] → swap → [3,5,1,8,2] [8,2] → swap → [3,5,1,2,8] ← 8 bubbled to end Pass 2: [3,5] ok, [5,1]→swap, [1,2] ok → [3,1,2,5,8] ← wait, recheck ...continue until no swaps in a pass Optimization: if no swaps in a pass, array is sorted → break early
public static void BubbleSort(int[] arr)
{
    int n = arr.Length;
    for (int i = 0; i < n - 1; i++)
    {
        bool swapped = false;
        for (int j = 0; j < n - 1 - i; j++) // Last i elements already in place
        {
            if (arr[j] > arr[j + 1])
            {
                (arr[j], arr[j + 1]) = (arr[j + 1], arr[j]); // C# tuple swap
                swapped = true;
            }
        }
        if (!swapped) break; // Already sorted — adaptive optimization
    }
}
// Time: O(n²) worst/avg, O(n) best (already sorted). Space: O(1). Stable: Yes

Selection Sort

Find the minimum element in the unsorted portion, swap it to the front. Repeat for each position.

Array: [5, 3, 8, 1, 2] Pass 0: min in [0..4] = 1 at idx 3 → swap [0] and [3] → [1, 3, 8, 5, 2] Pass 1: min in [1..4] = 2 at idx 4 → swap [1] and [4] → [1, 2, 8, 5, 3] Pass 2: min in [2..4] = 3 at idx 4 → swap [2] and [4] → [1, 2, 3, 5, 8] Pass 3: min in [3..4] = 5 at idx 3 → no swap needed Done: [1, 2, 3, 5, 8] Note: NOT stable (swapping can move equal elements past each other)
public static void SelectionSort(int[] arr)
{
    int n = arr.Length;
    for (int i = 0; i < n - 1; i++)
    {
        int minIdx = i;
        for (int j = i + 1; j < n; j++)
            if (arr[j] < arr[minIdx]) minIdx = j;

        if (minIdx != i)
            (arr[i], arr[minIdx]) = (arr[minIdx], arr[i]);
    }
}
// Time: O(n²) all cases. Space: O(1). Stable: No (unstable swaps)
// Pro: minimum number of swaps (O(n)) — useful when writes are expensive

Insertion Sort

Build a sorted sub-array one element at a time. Take the next element, shift sorted elements right until correct position is found, insert.

Array: [5, 3, 8, 1, 2] i=1: key=3. 5>3, shift 5 right → [_, 5, 8, 1, 2]. Insert 3 → [3, 5, 8, 1, 2] i=2: key=8. 5<8, stop. Insert in place → [3, 5, 8, 1, 2] i=3: key=1. 8>1,5>1,3>1, shift all → [_, 3, 5, 8, 2]. Insert 1 → [1, 3, 5, 8, 2] i=4: key=2. 8>2,5>2,3>2, shift → [1, _, 3, 5, 8]. Insert 2 → [1, 2, 3, 5, 8] BEST CASE: already sorted array → O(n) (no shifts needed) Great for small arrays and nearly-sorted data
public static void InsertionSort(int[] arr)
{
    for (int i = 1; i < arr.Length; i++)
    {
        int key = arr[i];
        int j = i - 1;

        // Shift elements greater than key one position right
        while (j >= 0 && arr[j] > key)
        {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key; // Insert key in correct position
    }
}
// Time: O(n²) worst, O(n) best (sorted). Space: O(1). Stable: Yes. Adaptive: Yes
// Used by Timsort for small sub-arrays (n < 32)

3. Merge Sort — Divide & Conquer

Split array in half recursively until single elements, then merge sorted halves back together. The merge step maintains sorted order.

Array: [38, 27, 43, 3, 9, 82, 10] Split: [38, 27, 43, 3, 9, 82, 10] ┌────────────┐ ┌──────────┐ [38, 27, 43, 3] [9, 82, 10] ┌─────┐ ┌─────┐ ┌──┐ ┌──┐ [38,27] [43,3] [9,82] [10] ┌──┐┌──┐┌──┐┌──┐ ┌──┐┌──┐ [38][27][43][3] [9][82] Merge up: [27,38] [3,43] [9,82] → [9,82,10] → [9,10,82] [3,27,38,43] [9,10,82] [3,9,10,27,38,43,82] ← final
public static void MergeSort(int[] arr, int left, int right)
{
    if (left >= right) return; // Base case: 0 or 1 element

    int mid = left + (right - left) / 2; // Avoid integer overflow
    MergeSort(arr, left, mid);
    MergeSort(arr, mid + 1, right);
    Merge(arr, left, mid, right);
}

private static void Merge(int[] arr, int left, int mid, int right)
{
    // Copy both halves into temp arrays
    int[] L = arr[left..(mid + 1)];   // C# range syntax
    int[] R = arr[(mid + 1)..(right + 1)];

    int i = 0, j = 0, k = left;

    while (i < L.Length && j < R.Length)
    {
        if (L[i] <= R[j])     // <= keeps equal elements in order (stable)
            arr[k++] = L[i++];
        else
            arr[k++] = R[j++];
    }
    while (i < L.Length) arr[k++] = L[i++]; // Remaining left elements
    while (j < R.Length) arr[k++] = R[j++]; // Remaining right elements
}
// Time: O(n log n) always. Space: O(n) auxiliary. Stable: Yes
// Usage: MergeSort(arr, 0, arr.Length - 1);
Merge Sort Recurrence: T(n) = 2T(n/2) + O(n) By Master Theorem: T(n) = O(n log n) Depth of recursion: log₂(n) levels Work per level: O(n) total merging Total: O(n) × O(log n) = O(n log n) Memory: O(n) for temp arrays at each level (not O(n log n) — temp arrays reused)
Real-World Use: Merge Sort is used for external sorting (sorting data too large to fit in memory — merge sorted chunks from disk). Also basis for counting inversions and Timsort (Python/Java default sort).

4. Quick Sort — In-Place Divide & Conquer

Pick a pivot, partition the array so all elements less than the pivot are on the left, all greater on the right. Recursively sort both sides.

Array: [3, 6, 8, 10, 1, 2, 1] Pivot = last element = 1 Lomuto Partition: pivot = 1, i = -1 (tracks "less than" boundary) j=0: arr[0]=3, 3>1, skip j=1: arr[1]=6, 6>1, skip j=2: arr[2]=8, 8>1, skip j=3: arr[3]=10, 10>1, skip j=4: arr[4]=1, 1≤1, i++ → i=0, swap arr[0] and arr[4]: [1, 6, 8, 10, 3, 2, 1] j=5: arr[5]=2, 2>1, skip End: swap arr[i+1]=arr[1] with pivot arr[6]: [1, 1, 8, 10, 3, 2, 6] pivot (1) is now at index 1 Left [1] is sorted. Right [8,10,3,2,6] recursed.
public static void QuickSort(int[] arr, int low, int high)
{
    if (low < high)
    {
        int pivotIdx = Partition(arr, low, high);
        QuickSort(arr, low, pivotIdx - 1);
        QuickSort(arr, pivotIdx + 1, high);
    }
}

// Lomuto partition scheme — pivot = last element
private static int Partition(int[] arr, int low, int high)
{
    int pivot = arr[high];
    int i = low - 1; // Boundary of "less than pivot" region

    for (int j = low; j < high; j++)
    {
        if (arr[j] <= pivot)
        {
            i++;
            (arr[i], arr[j]) = (arr[j], arr[i]);
        }
    }
    // Place pivot in its final position
    (arr[i + 1], arr[high]) = (arr[high], arr[i + 1]);
    return i + 1;
}
// Time: O(n log n) avg, O(n²) worst (sorted array with last-element pivot)
// Space: O(log n) avg (recursion stack). Stable: No. In-place: Yes
// Usage: QuickSort(arr, 0, arr.Length - 1);

Randomized Quick Sort — Avoiding O(n²)

private static Random _rand = new Random();

private static int PartitionRandom(int[] arr, int low, int high)
{
    // Swap a random element with last element before partitioning
    int randIdx = _rand.Next(low, high + 1);
    (arr[randIdx], arr[high]) = (arr[high], arr[randIdx]);
    return Partition(arr, low, high);
}
// Random pivot: O(n log n) expected time even on sorted/reverse-sorted input

Three-Way Partition (Dutch National Flag)

Handles many duplicate elements efficiently — groups array into three parts: <pivot, =pivot, >pivot. Reduces to O(n) when all elements are equal.

public static void QuickSort3Way(int[] arr, int low, int high)
{
    if (low >= high) return;

    int pivot = arr[low + _rand.Next(high - low + 1)];
    int lt = low, gt = high, i = low;

    while (i <= gt)
    {
        if      (arr[i] < pivot) (arr[lt++], arr[i++]) = (arr[i], arr[lt]);
        else if (arr[i] > pivot) (arr[i],   arr[gt--]) = (arr[gt], arr[i]);
        else                      i++;
    }
    // arr[low..lt-1] < pivot, arr[lt..gt] == pivot, arr[gt+1..high] > pivot
    QuickSort3Way(arr, low, lt - 1);
    QuickSort3Way(arr, gt + 1, high);
}
// O(n log n) avg, O(n) when all elements equal

5. Heap Sort

Build a max-heap from the array (O(n)), then repeatedly extract the max and place at the end (O(log n) each). Achieves O(n log n) in-place with no extra memory.

Array: [4, 10, 3, 5, 1] Step 1 — Build Max-Heap (Floyd's algorithm, O(n)): Start from last non-leaf = n/2-1 = 1: Heapify down index 1: arr[1]=10 > arr[3]=5, arr[4]=1 → already max Heapify down index 0: arr[0]=4 < arr[1]=10 → swap → [10, 4, 3, 5, 1] Continue: arr[1]=4 < arr[3]=5 → swap → [10, 5, 3, 4, 1] Max-Heap: [10, 5, 3, 4, 1] Step 2 — Extract max repeatedly (n-1 times): Swap arr[0] (max=10) with arr[4], heap size--: [1, 5, 3, 4, | 10] → heapify → [5, 4, 3, 1, | 10] Swap arr[0]=5 with arr[3], heap size--: [1, 4, 3, | 5, 10] → heapify → [4, 1, 3, | 5, 10] ... Final: [1, 3, 4, 5, 10] ✓
public static void HeapSort(int[] arr)
{
    int n = arr.Length;

    // Build max-heap (heapify all non-leaf nodes bottom-up)
    for (int i = n / 2 - 1; i >= 0; i--)
        Heapify(arr, n, i);

    // Extract elements from heap one by one
    for (int i = n - 1; i > 0; i--)
    {
        (arr[0], arr[i]) = (arr[i], arr[0]); // Move max to end
        Heapify(arr, i, 0);                 // Restore heap of size i
    }
}

private static void Heapify(int[] arr, int heapSize, int root)
{
    int largest = root;
    int left  = 2 * root + 1;
    int right = 2 * root + 2;

    if (left  < heapSize && arr[left]  > arr[largest]) largest = left;
    if (right < heapSize && arr[right] > arr[largest]) largest = right;

    if (largest != root)
    {
        (arr[root], arr[largest]) = (arr[largest], arr[root]);
        Heapify(arr, heapSize, largest);
    }
}
// Time: O(n log n) always. Space: O(1) in-place. Stable: No

6. Non-Comparison Sorts — Breaking O(n log n)

These sorts don't compare elements — they use mathematical properties of the keys to achieve O(n) time. They require specific assumptions about the input.

Counting Sort — O(n + k)

Count occurrences of each value, then reconstruct sorted output. Works when values are integers in a known range [0, k].

Array: [1, 4, 1, 2, 7, 5, 2] k = 7 (max value) Step 1 — Count: count[1]=2, count[2]=2, count[4]=1, count[5]=1, count[7]=1 count: [0, 2, 2, 0, 1, 1, 0, 1] Step 2 — Prefix sum (cumulative): gives ending position of each value count: [0, 2, 4, 4, 5, 6, 6, 7] Step 3 — Build output (traverse INPUT right-to-left for stability): output[count[arr[6]]-1] = output[6] = arr[6]=2 → output[3]=2, count[2]--=3 ... output: [1, 1, 2, 2, 4, 5, 7] ✓
public static int[] CountingSort(int[] arr, int maxVal)
{
    int[] count = new int[maxVal + 1];
    foreach (int x in arr) count[x]++;

    // Build prefix sums (each count[i] = # elements ≤ i)
    for (int i = 1; i <= maxVal; i++)
        count[i] += count[i - 1];

    // Fill output right-to-left (stable sort)
    int[] output = new int[arr.Length];
    for (int i = arr.Length - 1; i >= 0; i--)
        output[--count[arr[i]]] = arr[i];

    return output;
}
// Time: O(n + k), Space: O(n + k). Stable: Yes
// Use when: integer keys in small range [0, k], k = O(n)

Radix Sort — O(d * (n + k))

Sort integers digit by digit from least significant to most significant, using counting sort at each digit position.

public static void RadixSort(int[] arr)
{
    int maxVal = arr.Max();

    // Sort by each digit position: 1s, 10s, 100s, ...
    for (int exp = 1; maxVal / exp > 0; exp *= 10)
        CountingSortByDigit(arr, exp);
}

private static void CountingSortByDigit(int[] arr, int exp)
{
    int n = arr.Length;
    int[] output = new int[n];
    int[] count = new int[10]; // Digits 0-9

    foreach (int x in arr)
        count[(x / exp) % 10]++;

    for (int i = 1; i < 10; i++)
        count[i] += count[i - 1];

    for (int i = n - 1; i >= 0; i--)
    {
        int digit = (arr[i] / exp) % 10;
        output[--count[digit]] = arr[i];
    }
    Array.Copy(output, arr, n);
}
// Time: O(d*(n+10)) = O(d*n) where d = number of digits. Space: O(n). Stable: Yes

Bucket Sort — O(n) average

Distribute elements into buckets, sort each bucket (usually with insertion sort), then concatenate. Works well when input is uniformly distributed.

public static void BucketSort(float[] arr)
{
    int n = arr.Length;
    var buckets = new List<float>[n];
    for (int i = 0; i < n; i++) buckets[i] = new List<float>();

    // Distribute elements into buckets [0,1) range
    foreach (float x in arr)
        buckets[(int)(x * n)].Add(x);

    // Sort each bucket and concatenate
    int idx = 0;
    foreach (var bucket in buckets)
    {
        bucket.Sort(); // Insertion sort internally for small buckets
        foreach (float x in bucket)
            arr[idx++] = x;
    }
}
// Time: O(n) avg (uniform distribution), O(n²) worst. Space: O(n). Stable: Yes

7. C# Built-In Sorting

In real code and in interviews where you just need sorted data, use C#'s built-in sort which uses Introsort (quicksort + heapsort + insertion sort hybrid — always O(n log n)).

// Sort an array (ascending)
int[] nums = { 5, 2, 8, 1, 9 };
Array.Sort(nums);                         // [1, 2, 5, 8, 9]
Array.Sort(nums, (int a, int b) => b - a); // Descending: [9, 8, 5, 2, 1]

// Sort a List<T>
var list = new List<int> { 5, 2, 8, 1 };
list.Sort();
list.Sort((a, b) => b.CompareTo(a)); // Descending

// Sort by custom property (e.g., sort intervals by start time)
int[][] intervals = { new[] { 3,5 }, new[] { 1,4 }, new[] { 2,7 } };
Array.Sort(intervals, (a, b) => a[0] - b[0]); // Sort by start: [[1,4],[2,7],[3,5]]

// LINQ OrderBy (returns new IEnumerable, lazy evaluation)
var sorted = nums.OrderBy(x => x);
var sortedDesc = nums.OrderByDescending(x => x);
var multi = words.OrderBy(w => w.Length).ThenBy(w => w); // Multi-key sort

// Custom IComparer for complex objects
class IntervalComparer : IComparer<int[]>
{
    public int Compare(int[]? a, int[]? b)
        => a![0] != b![0] ? a[0] - b[0] : a[1] - b[1];
}
Interview Convention: In coding interviews, you can always call Array.Sort() or list.Sort() — never implement a sort from scratch unless explicitly asked. The built-in sorts are heavily optimized and well-known to interviewers.

8. Sorting in Problem Solving

Sorting is often a preprocessing step that unlocks a simpler algorithm. Recognizing when to sort first is a key interview skill.

Merge Intervals (LeetCode 56)

Given a list of intervals, merge all overlapping ones. Key step: sort by start time.

public int[][] Merge(int[][] intervals)
{
    // Sort by start time
    Array.Sort(intervals, (a, b) => a[0] - b[0]);

    var result = new List<int[]> { intervals[0] };

    foreach (var interval in intervals.Skip(1))
    {
        var last = result[^1]; // C# index-from-end
        if (interval[0] <= last[1])                         // Overlap
            last[1] = Math.Max(last[1], interval[1]);       // Extend
        else
            result.Add(interval);                               // New interval
    }
    return result.ToArray();
}
// Time: O(n log n), Space: O(n)

Meeting Rooms II (LeetCode 253)

Find minimum number of meeting rooms needed. Sort start times and end times separately, then use two pointers.

public int MinMeetingRooms(int[][] intervals)
{
    int n = intervals.Length;
    int[] starts = intervals.Select(i => i[0]).OrderBy(x => x).ToArray();
    int[] ends   = intervals.Select(i => i[1]).OrderBy(x => x).ToArray();

    int rooms = 0, endPtr = 0;

    for (int i = 0; i < n; i++)
    {
        if (starts[i] < ends[endPtr])
            rooms++;        // New meeting starts before any ends → need room
        else
            endPtr++;       // A meeting ended — reuse that room
    }
    return rooms;
}
// Time: O(n log n), Space: O(n)

Sort Colors / Dutch National Flag (LeetCode 75)

Sort an array of 0s, 1s, 2s in one pass without extra space.

public void SortColors(int[] nums)
{
    // Three pointers: lo (0s), mid (current), hi (2s)
    int lo = 0, mid = 0, hi = nums.Length - 1;

    while (mid <= hi)
    {
        if      (nums[mid] == 0) (nums[lo++], nums[mid++]) = (nums[mid], nums[lo]);
        else if (nums[mid] == 2) (nums[mid],  nums[hi--])  = (nums[hi], nums[mid]);
        else                       mid++;
    }
}
// Time: O(n), Space: O(1) — single pass!

9. Quick Select — Find kth Smallest in O(n)

A variant of Quick Sort that only recurses into one side of the partition, finding the kth smallest element in expected O(n) time.

public int FindKthSmallest(int[] nums, int k)
{
    return QuickSelect(nums, 0, nums.Length - 1, k - 1); // k-1 = 0-indexed
}

private int QuickSelect(int[] nums, int lo, int hi, int k)
{
    if (lo == hi) return nums[lo];

    // Random pivot to avoid worst case
    int randIdx = lo + _rand.Next(hi - lo + 1);
    (nums[randIdx], nums[hi]) = (nums[hi], nums[randIdx]);
    int pivotIdx = Partition(nums, lo, hi);

    if      (pivotIdx == k) return nums[pivotIdx]; // Found!
    else if (pivotIdx < k)  return QuickSelect(nums, pivotIdx + 1, hi, k);
    else                    return QuickSelect(nums, lo, pivotIdx - 1, k);
}
// Time: O(n) avg, O(n²) worst. Space: O(log n) avg recursion stack
// Alternative: use a min-heap of size k → O(n log k)
Kth Smallest — Compare approaches: ──────────────────────────────────────────────────────── Sort + index: O(n log n) time, O(1) extra (in-place) Min-heap size k: O(n log k) time, O(k) space Quick Select: O(n) avg time, O(log n) stack space ──────────────────────────────────────────────────────── For k << n: heap is best (O(n log k)) For general k: quick select is fastest on avg
