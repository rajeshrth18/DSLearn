1. Binary Search — The Canonical Template

Binary search finds a target in a sorted array in O(log n) by halving the search space each step. It's deceptively simple but has many subtle variants that trip up interviewers.

Array: [1, 3, 5, 7, 9, 11, 13, 15] Target: 7 idx: 0 1 2 3 4 5 6 7 Round 1: lo=0, hi=7, mid=3. arr[3]=7 == target → FOUND at index 3! If target = 6: Round 1: lo=0, hi=7, mid=3. arr[3]=7 > 6 → search left: hi=2 Round 2: lo=0, hi=2, mid=1. arr[1]=3 < 6 → search right: lo=2 Round 3: lo=2, hi=2, mid=2. arr[2]=5 < 6 → lo=3 > hi → NOT FOUND Key: mid = lo + (hi - lo) / 2 ← avoids integer overflow

The Two Most Common Bugs

Bug 1: Infinite Loop

Using lo <= hi with hi = mid (instead of hi = mid - 1) can loop forever when lo == hi. Always check: what happens when lo == hi?

Bug 2: Integer Overflow

mid = (lo + hi) / 2 can overflow for large indices. Always use: mid = lo + (hi - lo) / 2

// Standard binary search — exact match
public int BinarySearch(int[] nums, int target)
{
    int lo = 0, hi = nums.Length - 1;

    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;

        if      (nums[mid] == target) return mid;  // Found
        else if (nums[mid] <  target) lo = mid + 1; // Search right
        else                          hi = mid - 1; // Search left
    }
    return -1; // Not found
}
// Time: O(log n), Space: O(1)

// C# built-in: Array.BinarySearch(arr, target)
// Returns index if found, ~insertionPoint if not (negative value)

2. Finding Boundaries — First and Last Occurrence

When duplicates exist, you often need the first or last position of a target. Standard binary search stops at any occurrence.

Array: [1, 2, 2, 2, 3, 4, 5] Find first '2': index 1 Find last '2': index 3 For first '2': lo=0,hi=6 → mid=3, arr[3]=2 (match! but keep searching LEFT) → hi=2 lo=0,hi=2 → mid=1, arr[1]=2 (match! keep searching LEFT) → hi=0 lo=0,hi=0 → mid=0, arr[0]=1 < 2 → lo=1 lo=1 > hi=0 → stop. Answer = 1 (last match position recorded)
// Find FIRST occurrence of target (leftmost index)
public int FindFirst(int[] nums, int target)
{
    int lo = 0, hi = nums.Length - 1, result = -1;

    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target)
        {
            result = mid;     // Record this match
            hi = mid - 1;  // Keep searching LEFT for earlier occurrence
        }
        else if (nums[mid] < target) lo = mid + 1;
        else                          hi = mid - 1;
    }
    return result;
}

// Find LAST occurrence of target (rightmost index)
public int FindLast(int[] nums, int target)
{
    int lo = 0, hi = nums.Length - 1, result = -1;

    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target)
        {
            result = mid;     // Record this match
            lo = mid + 1;  // Keep searching RIGHT for later occurrence
        }
        else if (nums[mid] < target) lo = mid + 1;
        else                          hi = mid - 1;
    }
    return result;
}
// Find Range: [FindFirst, FindLast] — LeetCode 34

Lower Bound and Upper Bound

The lower bound is the first position ≥ target. The upper bound is the first position > target. These are STL-style and very useful in practice.

// Lower bound: first index i where nums[i] >= target
// If all elements < target, returns nums.Length
public int LowerBound(int[] nums, int target)
{
    int lo = 0, hi = nums.Length; // hi is EXCLUSIVE here

    while (lo < hi)               // Note: strict < (not <=)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] < target) lo = mid + 1;
        else                    hi = mid;     // Keep mid as candidate
    }
    return lo; // lo == hi when loop ends
}

// Upper bound: first index i where nums[i] > target
public int UpperBound(int[] nums, int target)
{
    int lo = 0, hi = nums.Length;

    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] <= target) lo = mid + 1; // Note: <= (not <)
        else                     hi = mid;
    }
    return lo;
}
// Count of target in sorted array = UpperBound - LowerBound

3. Binary Search on a Rotated Sorted Array

A sorted array rotated at some pivot: [4,5,6,7,0,1,2]. Binary search still works — identify which half is sorted, then check if target is in that half.

Array: [4, 5, 6, 7, 0, 1, 2] Target: 0 lo=0, hi=6, mid=3. arr[3]=7 Left half [4..7] is sorted (arr[lo]=4 ≤ arr[mid]=7). Is 0 in [4,7]? No → search RIGHT: lo=4 lo=4, hi=6, mid=5. arr[5]=1 Left half [0..1] is sorted (arr[lo]=0 ≤ arr[mid]=1). Is 0 in [0,1)? Yes (0 ≥ 0 and 0 < 1) → search LEFT: hi=4 lo=4, hi=4, mid=4. arr[4]=0 == target → FOUND at 4!
public int SearchRotated(int[] nums, int target)
{
    int lo = 0, hi = nums.Length - 1;

    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target) return mid;

        // Left half is sorted
        if (nums[lo] <= nums[mid])
        {
            // Target within left sorted half?
            if (nums[lo] <= target && target < nums[mid])
                hi = mid - 1;
            else
                lo = mid + 1;
        }
        else // Right half is sorted
        {
            // Target within right sorted half?
            if (nums[mid] < target && target <= nums[hi])
                lo = mid + 1;
            else
                hi = mid - 1;
        }
    }
    return -1;
}
// Time: O(log n), Space: O(1) — LeetCode 33

Find Minimum in Rotated Sorted Array (LeetCode 153)

public int FindMinRotated(int[] nums)
{
    int lo = 0, hi = nums.Length - 1;

    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        // If mid > hi, minimum is in right half
        if (nums[mid] > nums[hi])
            lo = mid + 1;
        else
            hi = mid; // mid could be the minimum
    }
    return nums[lo];
}
// Time: O(log n). When loop ends, lo == hi == index of minimum

4. Binary Search in a 2D Matrix

Search a 2D Matrix (LeetCode 74)

Matrix is sorted row-by-row, and first element of each row is greater than last of previous row. Treat it as a 1D sorted array.

public bool SearchMatrix(int[][] matrix, int target)
{
    int m = matrix.Length, n = matrix[0].Length;
    int lo = 0, hi = m * n - 1; // Treat as 1D array of size m*n

    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;
        int val = matrix[mid / n][mid % n]; // Convert 1D index to 2D

        if      (val == target) return true;
        else if (val <  target) lo = mid + 1;
        else                    hi = mid - 1;
    }
    return false;
}
// Time: O(log(m*n)) = O(log m + log n)

Search a 2D Matrix II (LeetCode 240)

Each row and each column is sorted independently (first element of row is NOT necessarily greater than last of previous row). Use staircase search starting from top-right.

public bool SearchMatrixII(int[][] matrix, int target)
{
    int row = 0, col = matrix[0].Length - 1; // Start at top-right

    while (row < matrix.Length && col >= 0)
    {
        if      (matrix[row][col] == target) return true;
        else if (matrix[row][col] >  target) col--; // Too big: go left
        else                                 row++; // Too small: go down
    }
    return false;
}
// Time: O(m + n) — at most m+n steps (eliminate row or column each step)

5. Binary Search on the Answer

This is the most powerful binary search technique. Instead of searching for a value in an array, you binary search over the answer space. Use when you can efficiently check "is answer X feasible?"

Key insight: If a function f(x) is monotone (once feasible, always feasible), binary search finds the minimum/maximum x where f(x) is true. f(x) = feasible? [F, F, F, T, T, T, T] ↑ Find this boundary Pattern: lo = min_possible_answer hi = max_possible_answer while (lo < hi): mid = (lo + hi) / 2 if feasible(mid): hi = mid (or lo = mid+1 for "last feasible") else: lo = mid + 1

Example: Capacity to Ship Packages Within D Days (LeetCode 1011)

Find minimum ship capacity so all packages can be shipped in ≤ D days.

public int ShipWithinDays(int[] weights, int days)
{
    // Answer is between max single weight (minimum possible) and total sum
    int lo = weights.Max(), hi = weights.Sum();

    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (CanShip(weights, days, mid))
            hi = mid;       // Feasible — try smaller capacity
        else
            lo = mid + 1; // Not feasible — need more capacity
    }
    return lo;
}

private bool CanShip(int[] weights, int days, int capacity)
{
    int daysNeeded = 1, currentLoad = 0;
    foreach (int w in weights)
    {
        if (currentLoad + w > capacity)
        {
            daysNeeded++;       // Start new day
            currentLoad = 0;
        }
        currentLoad += w;
    }
    return daysNeeded <= days;
}
// Time: O(n log(sum)), Space: O(1)

Example: Koko Eating Bananas (LeetCode 875)

public int MinEatingSpeed(int[] piles, int h)
{
    int lo = 1, hi = piles.Max();

    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        long hoursNeeded = piles.Sum(p => (long)Math.Ceiling((double)p / mid));
        if (hoursNeeded <= h)
            hi = mid;       // Feasible — try eating slower
        else
            lo = mid + 1; // Too slow — eat faster
    }
    return lo;
}
// Time: O(n log(max_pile)), Space: O(1)
Binary Search on Answer Template: The key is defining a monotone feasibility function. Ask yourself: "If capacity X works, does capacity X+1 also work?" If yes, binary search applies.

6. Two Pointers Technique

Use two pointers to traverse a sorted array (or linked list) from both ends, or to maintain a window. Reduces O(n²) brute force to O(n).

Two Sum in Sorted Array

Array: [1, 2, 3, 4, 6] Target: 6 lo→ ←hi [1, 2, 3, 4, 6] ↑ ↑ sum=1+6=7 > 6 → move hi left [1, 2, 3, 4, 6] ↑ ↑ sum=1+4=5 < 6 → move lo right [1, 2, 3, 4, 6] ↑ ↑ sum=2+4=6 == 6 → FOUND! (indices 1,3)
public int[] TwoSumSorted(int[] numbers, int target)
{
    int lo = 0, hi = numbers.Length - 1;

    while (lo < hi)
    {
        int sum = numbers[lo] + numbers[hi];
        if      (sum == target) return new[] { lo + 1, hi + 1 }; // 1-indexed
        else if (sum <  target) lo++; // Need larger sum
        else                    hi--; // Need smaller sum
    }
    return new[] { -1, -1 };
}
// Time: O(n), Space: O(1) — LeetCode 167

Three Sum (LeetCode 15)

Find all unique triplets summing to zero. Sort first, then for each element, use two pointers for the remaining pair.

public IList<IList<int>> ThreeSum(int[] nums)
{
    Array.Sort(nums);
    var result = new List<IList<int>>();

    for (int i = 0; i < nums.Length - 2; i++)
    {
        // Skip duplicates for the first element
        if (i > 0 && nums[i] == nums[i - 1]) continue;

        int lo = i + 1, hi = nums.Length - 1;

        while (lo < hi)
        {
            int sum = nums[i] + nums[lo] + nums[hi];
            if (sum == 0)
            {
                result.Add(new[] { nums[i], nums[lo], nums[hi] });
                // Skip duplicates for lo and hi
                while (lo < hi && nums[lo] == nums[lo + 1]) lo++;
                while (lo < hi && nums[hi] == nums[hi - 1]) hi--;
                lo++; hi--;
            }
            else if (sum < 0) lo++;
            else              hi--;
        }
    }
    return result;
}
// Time: O(n²), Space: O(1) excluding output

Container With Most Water (LeetCode 11)

public int MaxArea(int[] height)
{
    int lo = 0, hi = height.Length - 1, maxWater = 0;

    while (lo < hi)
    {
        int water = Math.Min(height[lo], height[hi]) * (hi - lo);
        maxWater = Math.Max(maxWater, water);

        // Move the shorter wall (it's the limiting factor)
        if (height[lo] < height[hi]) lo++;
        else                          hi--;
    }
    return maxWater;
}
// Greedy: moving taller wall can't increase water; move shorter to maybe find taller

7. Sliding Window Technique

Use a window (subarray) defined by two pointers that slide forward. Expand right, shrink left when a constraint is violated. Achieves O(n) for many substring/subarray problems.

Two types: ───────────────────────────────────────────────────────── Fixed-size window: slide window of size k across array Variable-size window: expand right, shrink left to maintain condition ───────────────────────────────────────────────────────── Variable window pattern: left = 0 for right = 0 to n-1: add nums[right] to window while window is invalid: remove nums[left] from window left++ update answer with window [left..right]

Longest Substring Without Repeating Characters (LeetCode 3)

public int LengthOfLongestSubstring(string s)
{
    var charIndex = new Dictionary<char, int>();
    int maxLen = 0, left = 0;

    for (int right = 0; right < s.Length; right++)
    {
        // If char is in window, shrink left past its previous position
        if (charIndex.TryGetValue(s[right], out int prevIdx) && prevIdx >= left)
            left = prevIdx + 1;

        charIndex[s[right]] = right;
        maxLen = Math.Max(maxLen, right - left + 1);
    }
    return maxLen;
}
// Time: O(n), Space: O(k) where k = charset size (≤128 for ASCII)

Minimum Window Substring (LeetCode 76)

Find the smallest window in s containing all characters of t.

public string MinWindow(string s, string t)
{
    if (s.Length < t.Length) return "";

    var need = new Dictionary<char, int>();
    foreach (char c in t) need[c] = need.GetValueOrDefault(c) + 1;

    int have = 0, required = need.Count;
    int left = 0, minLen = int.MaxValue, minStart = 0;
    var window = new Dictionary<char, int>();

    for (int right = 0; right < s.Length; right++)
    {
        char c = s[right];
        window[c] = window.GetValueOrDefault(c) + 1;
        if (need.ContainsKey(c) && window[c] == need[c]) have++;

        while (have == required) // Window is valid — try to shrink
        {
            if (right - left + 1 < minLen)
            { minLen = right - left + 1; minStart = left; }

            char lc = s[left++];
            window[lc]--;
            if (need.ContainsKey(lc) && window[lc] < need[lc]) have--;
        }
    }
    return minLen == int.MaxValue ? "" : s.Substring(minStart, minLen);
}
// Time: O(s + t), Space: O(s + t)

Maximum Sum Subarray of Size K (Fixed Window)

public int MaxSumSubarrayK(int[] nums, int k)
{
    // Initialize first window
    int windowSum = nums[..k].Sum();
    int maxSum = windowSum;

    for (int i = k; i < nums.Length; i++)
    {
        // Slide: add new right element, remove leftmost element
        windowSum += nums[i] - nums[i - k];
        maxSum = Math.Max(maxSum, windowSum);
    }
    return maxSum;
}
// Time: O(n), Space: O(1)

8. Fast and Slow Pointers (Floyd's Cycle Detection)

Two pointers at different speeds. Used for cycle detection, finding middle of linked lists, and more.

Find the Duplicate Number (LeetCode 287)

Array of n+1 integers in [1,n]. One number appears twice. Find it without modifying the array and O(1) space. Treat array as linked list where arr[i] is next pointer.

public int FindDuplicate(int[] nums)
{
    // Phase 1: Detect cycle (Floyd's algorithm)
    int slow = nums[0], fast = nums[0];
    do
    {
        slow = nums[slow];
        fast = nums[nums[fast]];
    } while (slow != fast);

    // Phase 2: Find cycle entry (= duplicate number)
    slow = nums[0];
    while (slow != fast)
    {
        slow = nums[slow];
        fast = nums[fast];
    }
    return slow;
}
// Time: O(n), Space: O(1)

9. Prefix Sum — O(1) Range Queries

Precompute cumulative sums so any subarray sum query takes O(1). Essential for range sum problems.

Array: [3, 1, 4, 1, 5, 9, 2, 6] prefix:[0, 3, 4, 8, 9,14,23,25,31] ↑ prefix[0] = 0 (empty prefix) Sum of arr[2..5] = prefix[6] - prefix[2] = 23 - 4 = 19 General: sum(i,j) = prefix[j+1] - prefix[i]
public class PrefixSum
{
    private readonly int[] _prefix;

    public PrefixSum(int[] nums)
    {
        _prefix = new int[nums.Length + 1];
        for (int i = 0; i < nums.Length; i++)
            _prefix[i + 1] = _prefix[i] + nums[i];
    }

    // Sum of nums[left..right] inclusive — O(1)
    public int RangeSum(int left, int right)
        => _prefix[right + 1] - _prefix[left];
}

// Subarray Sum Equals K (LeetCode 560): count subarrays summing to k
public int SubarraySum(int[] nums, int k)
{
    var prefixCount = new Dictionary<int, int> { [0] = 1 };
    int sum = 0, count = 0;

    foreach (int num in nums)
    {
        sum += num;
        // If (sum - k) was seen before, those subarrays sum to k
        count += prefixCount.GetValueOrDefault(sum - k);
        prefixCount[sum] = prefixCount.GetValueOrDefault(sum) + 1;
    }
    return count;
}
// Time: O(n), Space: O(n)

Binary Search Decision Tree:
  1. Is the search space sorted (or can the answer be checked monotonically)? → Binary search
  2. Do you need exact value? → Standard template with lo <= hi
  3. Do you need first/last occurrence? → Record result, bias search direction
  4. Are you searching for the minimum feasible answer? → lo < hi with hi = mid on feasible
  5. Are you searching for the maximum feasible answer? → lo < hi with lo = mid+1 on not-feasible, careful mid rounding
Common Binary Search Mistakes:
  • Using (lo + hi) / 2 instead of lo + (hi - lo) / 2 (overflow)
  • Wrong loop condition: lo < hi vs lo <= hi (depends on variant)
  • Infinite loop: when hi = mid, ensure lo strictly increases
  • Off-by-one: forgetting +1 or -1 in boundary updates
  • Not handling empty input or single element
