Section 1 — Traversal Overview

A traversal visits every node in a tree exactly once. The order matters — different orderings solve different problems. Two broad categories:

CategoryStrategyVariantsData Structure
DFS — Depth-First SearchGo deep first; backtrackPre-order, In-order, Post-orderStack (implicit call stack, or explicit)
BFS — Breadth-First SearchVisit level by levelLevel-order, Zigzag, Right-side viewQueue
REFERENCE TREE for all examples in this module: [1] Level 0 / \ [2] [3] Level 1 / \ \ [4] [5] [6] Level 2 / \ [7] [8] Level 3 Pre-order (Root→L→R): 1, 2, 4, 5, 7, 8, 3, 6 In-order (L→Root→R): 4, 2, 7, 5, 8, 1, 3, 6 Post-order (L→R→Root): 4, 7, 8, 5, 2, 6, 3, 1 Level-order: 1, 2, 3, 4, 5, 6, 7, 8
🌍 When to Use Which:
  • Pre-order: Copy/serialize a tree, print directory structure (parent before children)
  • In-order: BST sorted output, expression tree evaluation (infix)
  • Post-order: Delete a tree, calculate directory size (children before parent), postfix evaluation
  • Level-order: Shortest path in unweighted tree, print by levels, find right-side view
Section 2 — Pre-Order: Root → Left → Right

Pre-order processes the root first, then recursively the left subtree, then the right subtree. Think: "I see you before I look at your children."

PRE-ORDER TRACE on reference tree: Visit 1 → go left Visit 2 → go left Visit 4 → leaf → back to 2 go right Visit 5 → go left Visit 7 → leaf → back to 5 go right Visit 8 → leaf → back to 5 → back to 2 → back to 1 go right Visit 3 → go right Visit 6 → leaf → done Result: [1, 2, 4, 5, 7, 8, 3, 6]
// ── Recursive Pre-Order ────────────────────────────────────
public List<int> PreOrder()
{
    var result = new List<int>();
    PreRec(_root, result);
    return result;
}
private void PreRec(TreeNode? n, List<int> r)
{
    if (n == null) return;
    r.Add(n.Val);     // ROOT first
    PreRec(n.Left,  r);
    PreRec(n.Right, r);
}

// ── Iterative Pre-Order (explicit stack) ───────────────────
public List<int> PreOrderIterative()
{
    var result = new List<int>();
    if (_root == null) return result;
    var stack = new Stack<TreeNode>();
    stack.Push(_root);
    while (stack.Count > 0)
    {
        var node = stack.Pop();
        result.Add(node.Val);             // process immediately
        if (node.Right != null) stack.Push(node.Right); // push right FIRST
        if (node.Left  != null) stack.Push(node.Left);  // push left second (LIFO)
    }
    return result;
}
// Why push right first? Stack is LIFO, so left is processed first → correct order
// Time: O(n), Space: O(h)
Section 3 — In-Order: Left → Root → Right
// ── Recursive In-Order ─────────────────────────────────────
public List<int> InOrder()
{
    var result = new List<int>();
    InRec(_root, result);
    return result;
}
private void InRec(TreeNode? n, List<int> r)
{
    if (n == null) return;
    InRec(n.Left, r);
    r.Add(n.Val); // ROOT in middle
    InRec(n.Right, r);
}

// ── Iterative In-Order ─────────────────────────────────────
// Pattern: push left chain, pop and process, move right
public List<int> InOrderIterative()
{
    var result = new List<int>();
    var stack  = new Stack<TreeNode>();
    var curr   = _root;
    while (curr != null || stack.Count > 0)
    {
        while (curr != null)      // push entire left chain
        {
            stack.Push(curr);
            curr = curr.Left;
        }
        curr = stack.Pop();
        result.Add(curr.Val);    // process node
        curr = curr.Right;         // explore right subtree
    }
    return result;
}
// Memorize this iterative in-order — very common in interviews!
💡 Iterative In-Order Mnemonic: "Push left as far as possible. When you can't go left, process the node, then try going right. The popped node is always the next in-order element."
Section 4 — Post-Order: Left → Right → Root

Post-order processes children before the parent. Classic use: delete a tree (you must delete children before the parent) or compute directory sizes bottom-up.

// ── Recursive Post-Order ───────────────────────────────────
public List<int> PostOrder()
{
    var result = new List<int>();
    PostRec(_root, result);
    return result;
}
private void PostRec(TreeNode? n, List<int> r)
{
    if (n == null) return;
    PostRec(n.Left,  r);
    PostRec(n.Right, r);
    r.Add(n.Val); // ROOT last
}

// ── Iterative Post-Order ───────────────────────────────────
// Trick: reverse of modified pre-order (Root→Right→Left)
// Pre-order (Right-biased): 1,3,6,2,5,8,7,4
// Reversed:                 4,7,8,5,2,6,3,1 = Post-order ✓
public List<int> PostOrderIterative()
{
    var result = new List<int>();
    if (_root == null) return result;
    var stack = new Stack<TreeNode>();
    stack.Push(_root);
    while (stack.Count > 0)
    {
        var node = stack.Pop();
        result.Add(node.Val);              // collect (will be reversed)
        if (node.Left  != null) stack.Push(node.Left);  // push left first
        if (node.Right != null) stack.Push(node.Right); // push right second
    }
    result.Reverse(); // reverse to get post-order
    return result;
}
// Time: O(n), Space: O(h)
📝 Summary — DFS Traversal Orders:
Pre: [Root] Left Right → process root, then recurse
In: Left [Root] Right → recurse left, process, recurse right
Post: Left Right [Root] → recurse both sides, then process
Section 5 — Level-Order BFS

Level-order traversal visits all nodes level by level (left to right). Uses a Queue — the natural FIFO structure for BFS. This gives the shortest-path tree property.

LEVEL-ORDER BFS TRACE on reference tree: Queue: [1] Dequeue 1 → output 1 → enqueue children [2, 3] Queue: [2, 3] Dequeue 2 → output 2 → enqueue children [4, 5] Queue: [3, 4, 5] Dequeue 3 → output 3 → enqueue children [6] Queue: [4, 5, 6] Dequeue 4 → output 4 → no children Queue: [5, 6] Dequeue 5 → output 5 → enqueue children [7, 8] Queue: [6, 7, 8] Dequeue 6 → output 6 → no children Dequeue 7, 8 → output 7, 8 Result: [1, 2, 3, 4, 5, 6, 7, 8] ✓ By levels: [[1], [2,3], [4,5,6], [7,8]]
// ── Simple Level-Order (flat list) ────────────────────────
public List<int> LevelOrder()
{
    var result = new List<int>();
    if (_root == null) return result;
    var queue = new Queue<TreeNode>();
    queue.Enqueue(_root);
    while (queue.Count > 0)
    {
        var node = queue.Dequeue();
        result.Add(node.Val);
        if (node.Left  != null) queue.Enqueue(node.Left);
        if (node.Right != null) queue.Enqueue(node.Right);
    }
    return result;
}

// ── Level-Order with level grouping (LeetCode 102) ────────
public IList<IList<int>> LevelOrderGrouped()
{
    var result = new List<IList<int>>();
    if (_root == null) return result;
    var queue = new Queue<TreeNode>();
    queue.Enqueue(_root);
    while (queue.Count > 0)
    {
        int levelSize = queue.Count;      // snapshot: nodes on this level
        var level = new List<int>();
        for (int i = 0; i < levelSize; i++)
        {
            var node = queue.Dequeue();
            level.Add(node.Val);
            if (node.Left  != null) queue.Enqueue(node.Left);
            if (node.Right != null) queue.Enqueue(node.Right);
        }
        result.Add(level);
    }
    return result;
}
// [[1],[2,3],[4,5,6],[7,8]]
// Key trick: snapshot queue.Count BEFORE the inner loop — that's the level size
Section 6 — Zigzag Level-Order

Zigzag traversal alternates direction: level 0 goes left→right, level 1 goes right→left, level 2 left→right, etc. Used in the "Binary Tree Zigzag Level Order Traversal" problem (LeetCode 103).

ZIGZAG LEVEL ORDER on reference tree: Level 0 (L→R): [1] Level 1 (R→L): [3, 2] ← reversed Level 2 (L→R): [4, 5, 6] Level 3 (R→L): [8, 7] ← reversed
public IList<IList<int>> ZigzagLevelOrder()
{
    var result  = new List<IList<int>>();
    if (_root == null) return result;
    var  queue  = new Queue<TreeNode>();
    queue.Enqueue(_root);
    bool leftToRight = true;
    while (queue.Count > 0)
    {
        int size = queue.Count;
        var level = new int[size]; // fixed-size array — fill by index
        for (int i = 0; i < size; i++)
        {
            var node = queue.Dequeue();
            int idx  = leftToRight ? i : size - 1 - i; // write index
            level[idx] = node.Val;
            if (node.Left  != null) queue.Enqueue(node.Left);
            if (node.Right != null) queue.Enqueue(node.Right);
        }
        result.Add(level);
        leftToRight = !leftToRight; // flip direction
    }
    return result;
}
// No need to reverse — use write index trick for O(1) per element
Section 7 — Right-Side View

The "right-side view" of a tree shows only the rightmost node at each level — what you would see looking at the tree from the right side. (LeetCode 199)

RIGHT-SIDE VIEW of reference tree: [1] ← visible from right: 1 / \ [2] [3] ← rightmost: 3 / \ \ [4] [5] [6] ← rightmost: 6 / \ [7] [8] ← rightmost: 8 Output: [1, 3, 6, 8]
// BFS approach — last node of each level
public IList<int> RightSideView()
{
    var result = new List<int>();
    if (_root == null) return result;
    var queue  = new Queue<TreeNode>();
    queue.Enqueue(_root);
    while (queue.Count > 0)
    {
        int size = queue.Count;
        for (int i = 0; i < size; i++)
        {
            var node = queue.Dequeue();
            if (i == size - 1) result.Add(node.Val); // last = rightmost
            if (node.Left  != null) queue.Enqueue(node.Left);
            if (node.Right != null) queue.Enqueue(node.Right);
        }
    }
    return result;
}

// DFS approach — depth-first, keep track of max depth seen so far
public IList<int> RightSideViewDFS()
{
    var result = new List<int>();
    RightDFS(_root, 0, result);
    return result;
}
private void RightDFS(TreeNode? n, int depth, List<int> r)
{
    if (n == null) return;
    if (depth == r.Count) r.Add(n.Val); // first time at this depth = rightmost
    else r[depth] = n.Val;               // overwrite — rightmost seen last
    RightDFS(n.Left,  depth + 1, r);
    RightDFS(n.Right, depth + 1, r);   // right child always overwrites left
}
Section 8 — Vertical Order Traversal

Vertical order groups nodes by their column index: root is column 0, left child is column-1, right child is column+1. Within the same column, sort by depth (and by value if same depth).

VERTICAL ORDER — reference tree with column assignments: [1] col=0 / \ [2] [3] col=-1,+1 / \ \ [4] [5] [6] col=-2,0,+2 / \ [7] [8] col=-1,+1 Column grouping: col=-2: [4] col=-1: [2, 7] (depth 1, depth 3) col= 0: [1, 5] (depth 0, depth 2) — both in same col, sorted by depth col=+1: [3, 8] col=+2: [6] Output: [[4],[2,7],[1,5],[3,8],[6]]
public IList<IList<int>> VerticalOrder()
{
    var colMap = new SortedDictionary<int, List<(int depth, int val)>>();
    var queue  = new Queue<(TreeNode node, int col, int depth)>();
    if (_root != null) queue.Enqueue((_root, 0, 0));
    while (queue.Count > 0)
    {
        var (node, col, depth) = queue.Dequeue();
        if (!colMap.ContainsKey(col)) colMap[col] = new();
        colMap[col].Add((depth, node.Val));
        if (node.Left  != null) queue.Enqueue((node.Left,  col - 1, depth + 1));
        if (node.Right != null) queue.Enqueue((node.Right, col + 1, depth + 1));
    }
    var result = new List<IList<int>>();
    foreach (var entries in colMap.Values)
    {
        entries.Sort(); // sort by (depth, val)
        result.Add(entries.Select(e => e.val).ToList());
    }
    return result;
}
// SortedDictionary keeps columns in order automatically
// Time: O(n log n), Space: O(n)

💡 General Rule of Thumb:
  • Need to process children before parent? → Post-order
  • Need sorted output from a BST? → In-order
  • Need to process parent before children? → Pre-order
  • Need level-by-level info or shortest path? → BFS (queue)

Key Takeaways:
• DFS has 3 orders: Pre (root first), In (root middle), Post (root last)
• Iterative in-order pattern: push left chain → pop → process → go right
• BFS uses a Queue; snapshot queue.Count before each level loop
• Zigzag trick: use write index to avoid reversing lists
• Right-side view: last node per level in BFS, or DFS always overwriting with right
