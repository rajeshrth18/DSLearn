1. Binary Representation & Bitwise Operators

Every integer is stored as bits in memory. Bitwise operators work directly on those bits — often giving O(1) solutions to problems that seem to require O(n).

Integer 13 in binary (32-bit int): 0000 0000 0000 0000 0000 0000 0000 1101 ↑↑ ↑ 8+4+1 = 13 Bit positions (0-indexed from right): Bit 3: 1 (value 8) Bit 2: 1 (value 4) Bit 1: 0 (value 2) Bit 0: 1 (value 1) Negative numbers use Two's Complement: -1 = 1111...1111 (all ones) -13 = flip all bits of 13, add 1

The Six Bitwise Operators

// a = 0b1100 = 12, b = 0b1010 = 10
int a = 12, b = 10;

// AND: 1 only if BOTH bits are 1
int andResult  = a & b;   // 1100 & 1010 = 1000 = 8

// OR: 1 if EITHER bit is 1
int orResult   = a | b;   // 1100 | 1010 = 1110 = 14

// XOR: 1 if bits are DIFFERENT
int xorResult  = a ^ b;   // 1100 ^ 1010 = 0110 = 6

// NOT: flip all bits (produces ~a = -(a+1) for signed ints)
int notResult  = ~a;      // ~12 = -13

// Left Shift: multiply by 2 for each shift
int leftShift  = a << 2; // 1100 → 110000 = 48 = 12 * 4

// Right Shift (arithmetic): divide by 2, preserves sign bit
int rightShift = a >> 1; // 1100 → 0110 = 6 = 12 / 2

// Unsigned right shift (C# 11+): fills with 0, ignores sign
int unsignedRS = a >>> 1; // Same as >> for positive numbers

2. Essential Bit Tricks

These patterns appear in interviews constantly. Know them by heart.

// ─── Check if bit i is set ───────────────────────────────
bool IsBitSet(int n, int i) => (n & (1 << i)) != 0;
// 13 = 1101, i=2: (1101 & 0100) = 0100 ≠ 0 → true

// ─── Set bit i (turn it ON) ──────────────────────────────
int SetBit(int n, int i) => n | (1 << i);

// ─── Clear bit i (turn it OFF) ───────────────────────────
int ClearBit(int n, int i) => n & ~(1 << i);

// ─── Toggle bit i ────────────────────────────────────────
int ToggleBit(int n, int i) => n ^ (1 << i);

// ─── Extract lowest set bit (LSB) ────────────────────────
int LowestSetBit(int n) => n & (-n);
// 12 = 1100, -12 = 0100 (two's complement), 1100 & 0100 = 0100 = 4

// ─── Remove lowest set bit ───────────────────────────────
int RemoveLowestBit(int n) => n & (n - 1);
// 12 = 1100, 11 = 1011, 1100 & 1011 = 1000 = 8

// ─── Check if n is a power of 2 ──────────────────────────
bool IsPowerOf2(int n) => n > 0 && (n & (n - 1)) == 0;
// Power of 2 has exactly one bit set: 8=1000, 8-1=0111, 1000&0111=0

// ─── Count set bits (popcount) ───────────────────────────
int CountBits(int n)
{
    int count = 0;
    while (n != 0) { n &= n - 1; count++; } // Each step removes lowest set bit
    return count;
}
// C# built-in: System.Numerics.BitOperations.PopCount((uint)n)

// ─── Swap two integers without temp variable ──────────────
void XorSwap(ref int a, ref int b) { a ^= b; b ^= a; a ^= b; }
// Clever but rarely used in practice (C# tuple swap is cleaner)

3. XOR Magic — Finding Missing / Unique Numbers

XOR has a magical property: x ^ x = 0 (any value XOR itself is 0) and x ^ 0 = x. This lets you "cancel out" pairs.

XOR Properties: x ^ x = 0 (self-cancellation) x ^ 0 = x (identity) x ^ y = y ^ x (commutative) (x ^ y) ^ z = x ^ (y ^ z) (associative) Finding the single element in [4,1,2,1,2]: XOR all: 4 ^ 1 ^ 2 ^ 1 ^ 2 = 4 ^ (1^1) ^ (2^2) = 4 ^ 0 ^ 0 = 4 ← pairs cancel out!

Single Number (LeetCode 136)

public int SingleNumber(int[] nums)
{
    int xor = 0;
    foreach (int n in nums) xor ^= n;
    return xor; // All pairs cancel; only unique remains
}
// Time: O(n), Space: O(1)

Single Number III — Two Non-Duplicate Numbers (LeetCode 260)

public int[] SingleNumberIII(int[] nums)
{
    // Step 1: XOR all → get xor of the two unique numbers (a ^ b)
    int xorAB = 0;
    foreach (int n in nums) xorAB ^= n;

    // Step 2: Find any bit where a and b differ (lowest set bit of a^b)
    int diffBit = xorAB & (-xorAB);

    // Step 3: Split numbers into two groups based on that bit
    // Each group contains one unique number + its own pairs
    int a = 0, b = 0;
    foreach (int n in nums)
    {
        if ((n & diffBit) != 0) a ^= n;
        else                     b ^= n;
    }
    return new[] { a, b };
}
// Time: O(n), Space: O(1) — brilliant XOR divide-and-conquer!

Missing Number (LeetCode 268)

public int MissingNumber(int[] nums)
{
    // XOR all indices [0..n] with all values. Matching pairs cancel.
    int xor = nums.Length; // Start with n (the last index)
    for (int i = 0; i < nums.Length; i++)
        xor ^= i ^ nums[i];
    return xor;
    // Alternative: math approach: sum = n*(n+1)/2 - actual sum
}

4. Bit Counting — Counting Bits 1 to N

Count the number of 1-bits for every number from 0 to n efficiently using DP and bit tricks.

// Counting Bits (LeetCode 338): return array where ans[i] = popcount(i)
public int[] CountBits(int n)
{
    int[] dp = new int[n + 1];
    for (int i = 1; i <= n; i++)
        // dp[i] = dp[i with LSB removed] + 1
        dp[i] = dp[i >> 1] + (i & 1);
        // i >> 1 = i/2 (removes last bit), (i&1) = is last bit 1?
    return dp;
}
// Time: O(n), Space: O(n)
// Example: dp[5] = dp[2] + 1 = dp[1] + 0 + 1 = 1 + 0 + 1 = 2 ✓ (5=101 has two 1s)

// Hamming Distance (LeetCode 461): count differing bits between x and y
public int HammingDistance(int x, int y)
{
    int xorVal = x ^ y; // Bits differ where XOR is 1
    int count = 0;
    while (xorVal != 0) { xorVal &= xorVal - 1; count++; }
    return count;
}

5. Bitmask for Subsets

An integer with n bits can represent all 2ⁿ subsets of a set of n elements. Bit i being set means element i is in the subset.

Set: [a, b, c] (n=3, so bitmasks 0..7) 000 = {} 001 = {a} 010 = {b} 011 = {a,b} 100 = {c} 101 = {a,c} 110 = {b,c} 111 = {a,b,c} Enumerate all subsets of [a,b,c]: for mask in 0..7: subset = elements where bit is set in mask
// Enumerate all subsets using bitmasks
public IList<IList<int>> SubsetsBitmask(int[] nums)
{
    int n = nums.Length;
    var result = new List<IList<int>>();

    for (int mask = 0; mask < (1 << n); mask++) // 1<
    {
        var subset = new List<int>();
        for (int i = 0; i < n; i++)
            if ((mask & (1 << i)) != 0)
                subset.Add(nums[i]);
        result.Add(subset);
    }
    return result;
}
// Time: O(n * 2^n), Space: O(n * 2^n)

6. Fast Exponentiation (Binary Exponentiation)

Compute base^exp in O(log exp) instead of O(exp) by squaring repeatedly.

2^10 naively: 2 * 2 * ... * 2 = 10 multiplications Binary exponentiation: 10 in binary = 1010 2^10 = 2^8 * 2^2 = 256 * 4 = 1024 (only 4 multiplications!) Algorithm: at each step, if current bit is 1, multiply result by base always square the base (for next bit) Step: exp=10(1010), base=2, result=1 bit0=0: result=1, base=2^2=4 bit1=1: result=4, base=4^2=16 bit2=0: result=4, base=16^2=256 bit3=1: result=4*256=1024, done ✓
// Fast power: base^exp mod modulus
public long FastPow(long baseVal, long exp, long mod)
{
    long result = 1;
    baseVal %= mod;

    while (exp > 0)
    {
        if ((exp & 1) == 1)           // If current bit is set
            result = result * baseVal % mod;
        baseVal = baseVal * baseVal % mod; // Square the base
        exp >>= 1;                        // Move to next bit
    }
    return result;
}
// Time: O(log exp), Space: O(1)
// Used in: LeetCode 50 (Pow(x,n)), cryptography, number theory problems

// LeetCode 50 — Pow(x, n) with negative exponent support
public double MyPow(double x, int n)
{
    long N = n; // Use long to handle int.MinValue negation
    if (N < 0) { x = 1.0 / x; N = -N; }

    double result = 1.0;
    while (N > 0)
    {
        if ((N & 1) == 1) result *= x;
        x *= x;
        N >>= 1;
    }
    return result;
}

7. Greatest Common Divisor & Least Common Multiple

GCD and LCM are fundamental number theory operations that appear frequently in math-heavy interview problems.

// Euclidean algorithm for GCD
public int Gcd(int a, int b)
{
    while (b != 0) { (a, b) = (b, a % b); }
    return a;
}
// Recursive version: int Gcd(int a, int b) => b == 0 ? a : Gcd(b, a % b);
// Time: O(log(min(a,b))), Space: O(1)

// LCM from GCD: LCM(a,b) = a * b / GCD(a,b)
public long Lcm(long a, long b) => a / Gcd((int)a, (int)b) * b; // Divide first to avoid overflow

// C# built-in (since .NET 5):
// int gcd = (int)Math.Abs(a * b) / (int)MathF.Max(...) // no direct GCD in BCL
// Use your own Gcd helper
Euclidean algorithm trace: GCD(48, 18) GCD(48, 18) → GCD(18, 48%18=12) → GCD(12, 18%12=6) → GCD(6, 12%6=0) → 6 Intuition: GCD(a,b) = GCD(b, a mod b) Because any divisor of a and b also divides a mod b

8. Prime Numbers & Sieve of Eratosthenes

Finding all primes up to n using the Sieve of Eratosthenes — one of the oldest efficient algorithms.

Sieve of Eratosthenes for n=20: Mark all numbers 2..20 as potentially prime. Start at p=2 (first prime): Mark 4,6,8,10,12,14,16,18,20 as composite (multiples of 2) Next unmarked = 3: Mark 9,15 as composite (6,12,18 already marked; start at p²=9) Next unmarked = 5: Mark 25 > 20, stop inner loop Next prime = 7, 7²=49 > 20, stop Primes: [2, 3, 5, 7, 11, 13, 17, 19] Optimization: start marking at p², not 2p (smaller multiples already marked)
public bool[] SieveOfEratosthenes(int n)
{
    bool[] isPrime = new bool[n + 1];
    Array.Fill(isPrime, true);
    isPrime[0] = isPrime[1] = false;

    for (int p = 2; (long)p * p <= n; p++)
    {
        if (!isPrime[p]) continue;
        // Mark multiples of p starting from p² (p*2..p*(p-1) already marked)
        for (int multiple = p * p; multiple <= n; multiple += p)
            isPrime[multiple] = false;
    }
    return isPrime;
}
// Time: O(n log log n), Space: O(n)
// Count of primes ≤ n ≈ n / ln(n) (prime number theorem)

// Check if a single number is prime: O(√n)
public bool IsPrime(int n)
{
    if (n < 2) return false;
    if (n < 4) return true;
    if (n % 2 == 0 || n % 3 == 0) return false;
    for (int i = 5; (long)i * i <= n; i += 6) // Check 6k±1 form
        if (n % i == 0 || n % (i + 2) == 0) return false;
    return true;
}

9. Integer Overflow & Modular Arithmetic

Many problems ask for answers modulo 10⁹+7. This is a prime — using a prime modulus enables modular inverses (for division under modulo).

const int MOD = 1_000_000_007; // 10^9 + 7 (prime)

// Safe addition mod MOD
long AddMod(long a, long b) => (a + b) % MOD;

// Safe multiplication mod MOD (use long to avoid overflow)
long MulMod(long a, long b) => a % MOD * (b % MOD) % MOD;

// Modular inverse (for division: a/b mod m = a * b^(m-2) mod m)
// Only valid when m is prime (Fermat's little theorem)
long ModInverse(long b, long mod) => FastPow(b, mod - 2, mod);

// Common overflow traps:
int a = int.MaxValue;
// WRONG: int overflow
// int bad = a * 2;  
// CORRECT: cast to long first
long good = (long)a * 2;

// Overflow-safe midpoint (binary search):
int mid = lo + (hi - lo) / 2; // NOT (lo + hi) / 2 !
Overflow Rules to Memorize:
  • int * int can overflow — cast to long before multiplying
  • n * (n+1) / 2 for large n — use (long)n * (n+1) / 2
  • When doing a + b under modulo, the result could be up to 2*(MOD-1) which fits in int, but a * b needs long
  • Math.Abs(int.MinValue) overflows! Use long: Math.Abs((long)n)

10. Important Math Patterns in Problems

Roman Numerals / Number Conversion

public string IntToRoman(int num)
{
    int[]    vals  = { 1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1 };
    string[] syms  = { "M","CM","D","CD","C","XC","L","XL","X","IX","V","IV","I" };
    var sb = new StringBuilder();

    for (int i = 0; i < vals.Length; i++)
        while (num >= vals[i]) { sb.Append(syms[i]); num -= vals[i]; }

    return sb.ToString();
}

Digit Sum & Digit Manipulation

// Sum of digits until single digit (digital root)
public int AddDigits(int num)
{
    // Math formula: digital root = 1 + (n-1) % 9 for n > 0
    if (num == 0) return 0;
    return 1 + (num - 1) % 9;
}

// Get all digits of a number
var digits = new List<int>();
while (num > 0) { digits.Add(num % 10); num /= 10; }

Happy Number (LeetCode 202)

public bool IsHappy(int n)
{
    // Fast/slow pointer cycle detection on digit-square sequence
    int slow = n, fast = SumSquares(n);
    while (fast != 1 && slow != fast)
    {
        slow = SumSquares(slow);
        fast = SumSquares(SumSquares(fast));
    }
    return fast == 1;
}

private int SumSquares(int n)
{
    int sum = 0;
    while (n > 0) { int d = n % 10; sum += d * d; n /= 10; }
    return sum;
}
