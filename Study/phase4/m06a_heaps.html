Section 1 — What is a Heap?

A heap is a complete binary tree satisfying the heap property: every parent is either ≤ all its children (min-heap) or ≥ all its children (max-heap). The crucial word is complete — all levels are fully filled except possibly the last, which is filled from left to right.

MIN-HEAP MAX-HEAP [1] [9] / \ / \ [3] [2] [7] [8] / \ / \ / \ / \ [5][4][6] [8] [4][6][2] [5] Parent <= both children Parent >= both children Root = minimum element Root = maximum element Key property: Heap is NOT fully sorted! Min-heap: parent <= children, but siblings can be in any order. [3] < [5] and [4], but [5] vs [4] is not constrained.
🌍 Real-World Use Cases:
• OS schedulers — run the highest-priority process next
• Dijkstra's algorithm — extract min-distance node efficiently
• Event simulation — process events in time order
• Merge K sorted files — streaming data merge
• Median maintenance — two-heap trick for running median
Section 2 — Array Representation

A heap is stored in a flat array — no node objects, no pointers. The tree structure is implied by index arithmetic. This makes heaps cache-friendly and memory-efficient.

Min-heap stored as array: Tree: [1] / \ [3] [2] / \ / \ [5][4][6] [8] Array: [ 1, 3, 2, 5, 4, 6, 8 ] Index: 0 1 2 3 4 5 6 Index formulas (0-based): ┌────────────────────────────────────────────┐ │ Parent of i = (i - 1) / 2 │ │ Left child of i = 2 * i + 1 │ │ Right child of i= 2 * i + 2 │ └────────────────────────────────────────────┘ Verification: Node at index 1 (value=3): parent = (1-1)/2 = 0 ✓ (value=1) Node at index 1 (value=3): left = 2*1+1 = 3 ✓ (value=5) Node at index 1 (value=3): right = 2*1+2 = 4 ✓ (value=4) Root is ALWAYS at index 0. Last element in heap is at index (size - 1).
📝 Why array representation? A complete binary tree with n nodes can be stored in an array of size n with zero wasted space. Accessing parent/child is O(1) arithmetic — no pointer chasing. This is why heaps are much faster in practice than they appear on paper.
Section 3 — Core Operations

3.1 — Heapify-Up (Bubble Up)

Used after inserting a new element at the end. The new element bubbles up by swapping with its parent until the heap property is restored.

INSERT 0 into min-heap [1, 3, 2, 5, 4, 6, 8]: Step 1: Append 0 at end Array: [1, 3, 2, 5, 4, 6, 8, 0] Index: 0 1 2 3 4 5 6 7 Step 2: Heapify-up from index 7 Parent of 7 = (7-1)/2 = 3 → array[3]=5. 0 < 5 → SWAP Array: [1, 3, 2, 0, 4, 6, 8, 5] Step 3: Now at index 3 Parent of 3 = (3-1)/2 = 1 → array[1]=3. 0 < 3 → SWAP Array: [1, 0, 2, 3, 4, 6, 8, 5] Step 4: Now at index 1 Parent of 1 = (1-1)/2 = 0 → array[0]=1. 0 < 1 → SWAP Array: [0, 1, 2, 3, 4, 6, 8, 5] Done! 0 is now the root (minimum). 3 swaps → O(log n)

3.2 — Heapify-Down (Bubble Down / Sift Down)

Used after extracting the root (min/max). The last element is moved to root, then bubbles down by swapping with the smaller child until heap property is restored.

EXTRACT-MIN from [1, 3, 2, 5, 4, 6, 8]: Step 1: Remove root (value=1), move last element (8) to root Array: [8, 3, 2, 5, 4, 6] (size shrinks to 6) Step 2: Heapify-down from index 0 (value=8) Children: left=array[1]=3, right=array[2]=2 Smallest child = 2 (index 2). 8 > 2 → SWAP Array: [2, 3, 8, 5, 4, 6] Step 3: Now at index 2 (value=8) Children: left=array[5]=6, right=array[6] (doesn't exist, size=6) Only left child. 8 > 6 → SWAP Array: [2, 3, 6, 5, 4, 8] Step 4: Now at index 5 (value=8) — no children (2*5+1=11 >= size=6) Done! New min = 2. 2 swaps → O(log n)
// Generic MinHeap implementation (simplified for learning)
public class MinHeap
{
    private List<int> _data = new();
    public int Count => _data.Count;

    private int Parent(int i) => (i - 1) / 2;
    private int Left(int i)   => 2 * i + 1;
    private int Right(int i)  => 2 * i + 2;
    private void Swap(int a, int b) => (_data[a], _data[b]) = (_data[b], _data[a]);

    public void Insert(int val)
    {
        _data.Add(val);         // append at end
        HeapifyUp(_data.Count - 1);
    }
    private void HeapifyUp(int i)
    {
        while (i > 0 && _data[i] < _data[Parent(i)])
        {
            Swap(i, Parent(i));
            i = Parent(i);
        }
    }

    public int PeekMin() => _data[0];

    public int ExtractMin()
    {
        int min = _data[0];
        int last = _data.Count - 1;
        _data[0] = _data[last];  // move last to root
        _data.RemoveAt(last);    // shrink
        if (_data.Count > 0) HeapifyDown(0);
        return min;
    }
    private void HeapifyDown(int i)
    {
        int n = _data.Count;
        while (true)
        {
            int smallest = i;
            int l = Left(i), r = Right(i);
            if (l < n && _data[l] < _data[smallest]) smallest = l;
            if (r < n && _data[r] < _data[smallest]) smallest = r;
            if (smallest == i) break;
            Swap(i, smallest);
            i = smallest;
        }
    }
}
// Insert: O(log n), ExtractMin: O(log n), PeekMin: O(1)
Section 4 — Build Heap in O(n)

If you have an unsorted array and want to turn it into a heap, you could insert each element one-by-one in O(n log n). But there's a smarter way: Floyd's heapify algorithm runs heapify-down on every non-leaf node from bottom to top, achieving O(n) total time.

BUILD MIN-HEAP from [9, 4, 7, 1, 6, 3, 8] Array: [9, 4, 7, 1, 6, 3, 8] Index: 0 1 2 3 4 5 6 Tree visualization: [9] / \ [4] [7] / \ / \ [1][6][3] [8] Last non-leaf = floor((n/2)-1) = floor(7/2-1) = 2 Process from index 2 down to 0: Index 2 (value=7): children are 3(index 5) and 8(index 6) Smallest child = 3. 7 > 3 → swap → [9, 4, 3, 1, 6, 7, 8] Index 1 (value=4): children are 1(index 3) and 6(index 4) Smallest child = 1. 4 > 1 → swap → [9, 1, 3, 4, 6, 7, 8] Continue down: index 3 (value=4): children 7,8 (out of range) → done Index 0 (value=9): children are 1(index 1) and 3(index 2) Smallest child = 1. 9 > 1 → swap → [1, 9, 3, 4, 6, 7, 8] Continue: index 1 (value=9): children 4(idx 3) and 6(idx 4) Smallest = 4. 9 > 4 → swap → [1, 4, 3, 9, 6, 7, 8] Final heap: [1, 4, 3, 9, 6, 7, 8] ✓ (root=1=min)
// Build heap in O(n) — Floyd's algorithm
public static void BuildHeap(int[] arr)
{
    int n = arr.Length;
    // Start from last non-leaf: floor((n/2) - 1) = (n/2 - 1)
    for (int i = n / 2 - 1; i >= 0; i--)
        HeapifyDown(arr, i, n);
}
private static void HeapifyDown(int[] arr, int i, int n)
{
    while (true)
    {
        int smallest = i;
        int l = 2 * i + 1, r = 2 * i + 2;
        if (l < n && arr[l] < arr[smallest]) smallest = l;
        if (r < n && arr[r] < arr[smallest]) smallest = r;
        if (smallest == i) break;
        (arr[i], arr[smallest]) = (arr[smallest], arr[i]);
        i = smallest;
    }
}
// Time: O(n) — counterintuitively, not O(n log n)
// Analysis: Most nodes are near leaves and need few swaps.
// Total work = sum over all levels of (nodes at level × swaps needed)
//            = n/2 × 1 + n/4 × 2 + n/8 × 3 + ... = O(n)
Section 5 — Heap Sort

Heap sort uses a max-heap to sort an array in-place in O(n log n). First build a max-heap. Then repeatedly extract the max (swap root with last, shrink heap, heapify-down) — resulting in ascending sorted order.

HEAP SORT on [4, 10, 3, 5, 1]: Phase 1 — Build max-heap: Build max-heap → [10, 5, 3, 4, 1] Tree: [10] / \ [5] [3] / \ [4][1] Phase 2 — Extract max repeatedly: Swap arr[0] with arr[4]: [1, 5, 3, 4, 10] heap_size=4 Heapify-down index 0 : [5, 4, 3, 1, | 10] Swap arr[0] with arr[3]: [1, 4, 3, 5, | 10] heap_size=3 Heapify-down index 0 : [4, 1, 3, | 5, 10] Swap arr[0] with arr[2]: [3, 1, 4, | 5, 10] heap_size=2 Heapify-down index 0 : [3, 1, | 4, 5, 10] Swap arr[0] with arr[1]: [1, 3, | 4, 5, 10] heap_size=1 Done! Result: [1, 3, 4, 5, 10] ✓ Sorted ascending!
public static void HeapSort(int[] arr)
{
    int n = arr.Length;
    // Phase 1: Build max-heap (note: use ">" for max-heap)
    for (int i = n / 2 - 1; i >= 0; i--)
        SiftDown(arr, i, n);
    // Phase 2: Extract max one by one
    for (int end = n - 1; end > 0; end--)
    {
        (arr[0], arr[end]) = (arr[end], arr[0]); // swap max to end
        SiftDown(arr, 0, end); // restore heap property (heap_size = end)
    }
}
private static void SiftDown(int[] arr, int i, int n)
{
    while (true)
    {
        int largest = i;
        int l = 2 * i + 1, r = 2 * i + 2;
        if (l < n && arr[l] > arr[largest]) largest = l;
        if (r < n && arr[r] > arr[largest]) largest = r;
        if (largest == i) break;
        (arr[i], arr[largest]) = (arr[largest], arr[i]);
        i = largest;
    }
}
// Time: O(n log n)  Space: O(1) — in-place!
// Not stable (equal elements may swap relative order)
// Good: guaranteed O(n log n), no extra memory
// Bad: poor cache performance vs quicksort in practice
Section 6 — Custom Generic MinHeap<T> in C#
public class MinHeap<T> where T : IComparable<T>
{
    private List<T> _heap = new();
    public int  Count    => _heap.Count;
    public bool IsEmpty  => _heap.Count == 0;
    public T    Peek()   => _heap[0];

    private int  Par(int i)   => (i - 1) / 2;
    private int  L(int i)     => 2 * i + 1;
    private int  R(int i)     => 2 * i + 2;
    private bool Less(int a, int b) => _heap[a].CompareTo(_heap[b]) < 0;
    private void Swap(int a, int b) => (_heap[a], _heap[b]) = (_heap[b], _heap[a]);

    public void Push(T val)
    {
        _heap.Add(val);
        BubbleUp(_heap.Count - 1);
    }
    public T Pop()
    {
        T top = _heap[0];
        int last = _heap.Count - 1;
        _heap[0] = _heap[last];
        _heap.RemoveAt(last);
        if (_heap.Count > 0) SinkDown(0);
        return top;
    }
    private void BubbleUp(int i)
    {
        while (i > 0 && Less(i, Par(i))) { Swap(i, Par(i)); i = Par(i); }
    }
    private void SinkDown(int i)
    {
        int n = _heap.Count;
        while (true)
        {
            int m = i, l = L(i), r = R(i);
            if (l < n && Less(l, m)) m = l;
            if (r < n && Less(r, m)) m = r;
            if (m == i) break;
            Swap(i, m); i = m;
        }
    }
}
// Usage:
// var minH = new MinHeap<int>();
// minH.Push(5); minH.Push(2); minH.Push(8);
// minH.Pop() → 2 (smallest)
// For MaxHeap: reverse the comparator: "Greater" instead of "Less"
💡 MaxHeap trick: To get a max-heap using the same class, wrap your values: Push(-val) and negate on Pop(). Or pass a custom comparer. In interviews, this trick is very common with C#'s PriorityQueue.
Section 7 — C# PriorityQueue<TElement, TPriority>

Since .NET 6, C# ships a built-in PriorityQueue<TElement, TPriority>. It is a min-heap on priority — the element with the lowest priority value dequeues first.

// Basic usage — PriorityQueue is a min-heap on priority
var pq = new PriorityQueue<string, int>();

pq.Enqueue("low task",    10);
pq.Enqueue("urgent task",  1);
pq.Enqueue("normal task",  5);

string next = pq.Dequeue(); // "urgent task" (priority 1 = lowest = highest urgency)

// ── Common API ──────────────────────────────────────────
pq.Enqueue(element, priority);        // O(log n)
string e = pq.Dequeue();              // O(log n) — removes min-priority
bool ok = pq.TryDequeue(out var el, out var pri); // safe dequeue
pq.Peek();                            // O(1)   — view without removing
bool ok2 = pq.TryPeek(out var el2, out var pri2); // safe peek
pq.Count;                             // O(1)
pq.EnqueueRange(collection);          // bulk insert O(n) to O(n log n)

// ── Max-heap trick: negate priority ─────────────────────
var maxPQ = new PriorityQueue<int, int>();
maxPQ.Enqueue(5, -5);  // negate to turn min-heap into max-heap
maxPQ.Enqueue(3, -3);
maxPQ.Enqueue(8, -8);
int max = maxPQ.Dequeue(); // 8 (priority -8 = minimum = extracted first)

// ── Custom object priority ───────────────────────────────
record Task(string Name, int Deadline);
var taskQ = new PriorityQueue<Task, int>();
taskQ.Enqueue(new Task("Report", 3), 3);
taskQ.Enqueue(new Task("Email",  1), 1);
Task first = taskQ.Dequeue(); // Task "Email" (deadline=1)
⚠️ Important: C#'s PriorityQueue does NOT support DecreaseKey (changing the priority of an existing element). If you need that (e.g., Dijkstra's with updates), use lazy deletion: re-insert with new priority and skip stale entries when dequeuing.
Section 8 — Top-K Pattern

One of the most common heap interview patterns: "Find the K largest/smallest elements." The trick is to maintain a size-K heap and discard elements outside the top-K as you stream through data.

FIND K-TH LARGEST ELEMENT — example: [3,2,1,5,6,4], k=2 Strategy 1 — Min-heap of size K: Keep a min-heap of the K largest elements seen so far. If new element > heap top, pop and push new element. Process 3: heap=[3] Process 2: heap=[2,3] (size 2 = k, heap min = 2) Process 1: 1 < 2 (heap top) → skip Process 5: 5 > 2 → pop 2, push 5 → heap=[3,5] Process 6: 6 > 3 → pop 3, push 6 → heap=[5,6] Process 4: 4 < 5 → skip Answer: heap.Peek() = 5 ✓ (2nd largest is 5) Time: O(n log k) Space: O(k) Strategy 2 — Sort: O(n log n), OK for small n Strategy 3 — QuickSelect: O(n) average, harder to implement
// LeetCode 215 — Kth Largest Element in an Array
public int FindKthLargest(int[] nums, int k)
{
    // Min-heap of size k — top = kth largest
    var pq = new PriorityQueue<int, int>();
    foreach (int n in nums)
    {
        pq.Enqueue(n, n);
        if (pq.Count > k) pq.Dequeue(); // remove smallest, keep K largest
    }
    pq.TryPeek(out int result, out _);
    return result;
}
// Time: O(n log k), Space: O(k)

// Top K Frequent Elements (LeetCode 347)
public int[] TopKFrequent(int[] nums, int k)
{
    var freq = new Dictionary<int, int>();
    foreach (int n in nums) freq[n] = freq.GetValueOrDefault(n) + 1;

    // Min-heap on frequency, keep top-K
    var pq = new PriorityQueue<int, int>();
    foreach (var (num, cnt) in freq)
    {
        pq.Enqueue(num, cnt);           // priority = frequency
        if (pq.Count > k) pq.Dequeue(); // drop least frequent
    }
    var result = new int[k];
    for (int i = k - 1; i >= 0; i--) result[i] = pq.Dequeue();
    return result;
}
// Time: O(n log k), Space: O(n) for freq map
Section 9 — Merge K Sorted Lists

Given K sorted lists, merge them into one sorted list. Naive: merge two at a time O(nK²). Optimal: use a min-heap holding one element from each list — always extract the current minimum and advance that list's pointer.

MERGE K SORTED LISTS Lists: L1: 1 → 4 → 7 L2: 2 → 5 → 8 L3: 0 → 3 → 6 Initial heap: [(0,L3), (1,L1), (2,L2)] (value, source list) Extract min 0 (L3) → output: [0] Advance L3 → push (3,L3) Heap: [(1,L1), (2,L2), (3,L3)] Extract min 1 (L1) → output: [0,1] Advance L1 → push (4,L1) Heap: [(2,L2), (3,L3), (4,L1)] Extract min 2 (L2) → output: [0,1,2] Advance L2 → push (5,L2) Heap: [(3,L3), (4,L1), (5,L2)] ... continue until all lists exhausted Final: [0,1,2,3,4,5,6,7,8] ✓ Time: O(n·K · log K) where n = avg list length The log K comes from the heap holding at most K elements
// LeetCode 23 — Merge K Sorted Lists
public ListNode? MergeKLists(ListNode?[] lists)
{
    // pq: (nodeValue, listIndex, node)
    var pq = new PriorityQueue<ListNode, int>();

    foreach (var head in lists)
        if (head != null) pq.Enqueue(head, head.Val);

    var dummy = new ListNode();
    var cur   = dummy;

    while (pq.Count > 0)
    {
        var node = pq.Dequeue();
        cur.Next = node;
        cur = cur.Next;
        if (node.Next != null)
            pq.Enqueue(node.Next, node.Next.Val); // advance this list
    }
    return dummy.Next;
}
// Time: O(N log K) where N = total nodes, K = number of lists
// Space: O(K) for the heap

// BONUS: Find Median from Data Stream (Two-Heap approach)
public class MedianFinder
{
    // maxH: lower half — max-heap (negate priority)
    private PriorityQueue<int, int> maxH = new(); // stores -val as priority
    // minH: upper half — min-heap
    private PriorityQueue<int, int> minH = new();

    public void AddNum(int num)
    {
        maxH.Enqueue(num, -num);  // push to lower half
        // Balance: all lower ≤ all upper
        maxH.TryPeek(out int maxTop, out _);
        if (minH.Count > 0)
        {
            minH.TryPeek(out int minTop, out _);
            if (maxTop > minTop)  // lower max > upper min → rebalance
            {
                minH.Enqueue(maxH.Dequeue(), maxH.Count > 0 ? maxH.Peek() : 0);
                // simpler: just re-enqueue with correct priority
            }
        }
        // Size balance: maxH.Count == minH.Count or maxH.Count == minH.Count+1
        if (maxH.Count > minH.Count + 1)
            minH.Enqueue(maxH.Dequeue(), 0); // (re-push with correct priority)
        else if (minH.Count > maxH.Count)
            maxH.Enqueue(minH.Peek(), -minH.Dequeue());
    }
    public double FindMedian()
    {
        if (maxH.Count > minH.Count) { maxH.TryPeek(out int v, out _); return v; }
        maxH.TryPeek(out int a, out _);
        minH.TryPeek(out int b, out _);
        return (a + b) / 2.0;
    }
}

Key Takeaways:
• Heap = complete binary tree, stored as array. Root = min (or max).
• Array math: parent=(i-1)/2, left=2i+1, right=2i+2
• Insert: O(log n) heapify-up. Extract: O(log n) heapify-down.
• Build heap: O(n) Floyd's algorithm
• Top-K problems: maintain size-K min-heap → O(n log k)
• C# PriorityQueue is min-heap on priority. Negate for max-heap.
