Section 1 — What is a Trie?

A trie (pronounced "try", from retrieval) is a tree where each node represents one character. Words are formed by paths from root to a marked end node. The key property: all words sharing a common prefix share the same path.

Trie storing: "cat", "car", "card", "care", "dog" [root] / \ [c] [d] | \ [a] [o] / \ \ [t*] [r*] [g*] / \ [d*] [e*] * = isEndOfWord marker Paths: root → c → a → t* = "cat" ✓ root → c → a → r* = "car" ✓ root → c → a → r* → d* = "card" ✓ root → c → a → r* → e* = "care" ✓ root → d → o → g* = "dog" ✓ Key observations: 1. "ca" prefix is shared by "cat", "car", "card", "care" — O(prefix_len) lookup 2. 14 nodes instead of storing 5 separate strings 3. Search for prefix "car" = O(3), not O(n_words)
🌍 Real-World Use Cases:
• Autocomplete — suggest words as user types
• Spell checker — is this a valid word?
• IP routing — longest prefix match for routing tables
• Browser history — address bar suggestions
• Search engines — prefix indexing for fast lookup
📝 Why not use a HashMap? A HashMap can check exact words in O(1), but cannot efficiently answer "give me all words starting with 'pre'" without scanning all keys. A trie answers prefix queries in O(prefix_length), regardless of how many words are stored.
Section 2 — TrieNode & Trie Class
// TrieNode — represents one character in the trie
public class TrieNode
{
    public Dictionary<char, TrieNode> Children = new();
    public bool IsEnd = false;   // marks end of a valid word
}
// Alternative: use array of size 26 for lowercase letters only
// public TrieNode[] Children = new TrieNode[26];
// Access: node.Children[c - 'a']
// Array version: O(26) space per node (faster, uses more memory)
// Dictionary version: O(distinct chars) space (slower lookup, less memory)

public class Trie
{
    private readonly TrieNode _root = new();

    public void Insert(string word)
    {
        var cur = _root;
        foreach (char c in word)
        {
            if (!cur.Children.ContainsKey(c))
                cur.Children[c] = new TrieNode();
            cur = cur.Children[c];
        }
        cur.IsEnd = true;
    }

    public bool Search(string word)
    {
        var node = GetNode(word);
        return node != null && node.IsEnd;
    }

    public bool StartsWith(string prefix)
    {
        return GetNode(prefix) != null;
    }

    // Helper: traverse to end of given string, return node (or null)
    private TrieNode? GetNode(string s)
    {
        var cur = _root;
        foreach (char c in s)
        {
            if (!cur.Children.TryGetValue(c, out cur!)) return null;
        }
        return cur;
    }
}
// Time: O(m) for all operations where m = word/prefix length
// Space: O(total_characters_inserted)
Section 3 — Insert & Search Step-by-Step
INSERT "apple" into empty trie: Start at root []. c='a': root has no 'a' child → create TrieNode 'a' root → [a] c='p': 'a' has no 'p' child → create TrieNode 'p' root → [a] → [p] c='p': 'p' has no 'p' child → create TrieNode 'p' root → [a] → [p] → [p] c='l': create TrieNode 'l' root → [a] → [p] → [p] → [l] c='e': create TrieNode 'e' root → [a] → [p] → [p] → [l] → [e] Mark last node: IsEnd = true root → [a] → [p] → [p] → [l] → [e*] INSERT "app" into the same trie: c='a': root has 'a' → follow it (no new node) c='p': 'a' has 'p' → follow it c='p': 'p' has 'p' → follow it Mark current node [p] IsEnd = true root → [a] → [p] → [p*] → [l] → [e*] Now both "app" and "apple" are stored. They share root→a→p! SEARCH "app": Traverse a→p→p → found node with IsEnd=true → FOUND SEARCH "ap": Traverse a→p → found node with IsEnd=false → NOT FOUND (not a word) STARSWITH "app": Traverse a→p→p → node exists → PREFIX EXISTS
Section 4 — StartsWith: Autocomplete

With StartsWith, we can also retrieve all words with a given prefix — the foundation of autocomplete systems.

// Get all words in the trie starting with prefix
public IList<string> GetWordsWithPrefix(string prefix)
{
    var results = new List<string>();
    var node = GetNode(prefix);
    if (node == null) return results;
    DFS(node, new System.Text.StringBuilder(prefix), results);
    return results;
}
private void DFS(TrieNode node, System.Text.StringBuilder sb,
                  List<string> results)
{
    if (node.IsEnd) results.Add(sb.ToString());
    foreach (var (c, child) in node.Children)
    {
        sb.Append(c);
        DFS(child, sb, results);
        sb.Remove(sb.Length - 1, 1); // backtrack
    }
}
// Example:
// trie.Insert("apple"); trie.Insert("app"); trie.Insert("apply");
// GetWordsWithPrefix("app") → ["app", "apple", "apply"]
💡 Complexity: GetWordsWithPrefix is O(prefix_len + total output chars). In the worst case (prefix is root = empty string), it traverses the entire trie O(total chars). This is still optimal — you have to visit every returned word character anyway.
Section 5 — Delete from Trie

Deletion is the trickiest trie operation. We must only remove nodes that are not shared by other words. Three cases:

CASES for deleting "care" from trie containing "cat","car","card","care": Case 1: Word is a prefix of another word Delete "car" → unmark IsEnd only. "card" still exists. [c][a][r*] → [c][a][r] ← just clear IsEnd flag Case 2: Word shares prefix with another word Delete "care" → remove [e] node (leaf). "card" still intact. [r*][d*] stays, just remove [e*] branch Case 3: Word is unique (no shared prefix with others) Delete "dog" → remove d→o→g nodes (all become garbage) Algorithm: Post-order recursion 1. Reach end of word → unset IsEnd 2. On way back up: if node has no children AND is not end of another word → delete This naturally handles shared prefix cases.
public bool Delete(string word)
    => DeleteHelper(_root, word, 0);

private bool DeleteHelper(TrieNode node, string word, int depth)
{
    if (depth == word.Length)
    {
        if (!node.IsEnd) return false; // word doesn't exist
        node.IsEnd = false;
        return node.Children.Count == 0; // can delete if leaf
    }
    char c = word[depth];
    if (!node.Children.TryGetValue(c, out var child)) return false;

    bool shouldDelete = DeleteHelper(child, word, depth + 1);
    if (shouldDelete)
    {
        node.Children.Remove(c);
        return !node.IsEnd && node.Children.Count == 0;
    }
    return false;
}
// Returns true when a node should be deleted by its parent
// A node is deletable only when: IsEnd=false AND no remaining children
Section 6 — Count Words with Prefix
// Track word count at each node for O(1) prefix count queries
public class TrieNodeWithCount
{
    public Dictionary<char, TrieNodeWithCount> Children = new();
    public bool IsEnd = false;
    public int  PassCount = 0;  // how many words pass through this node
    public int  EndCount  = 0;  // how many words end at this node
}
public class TrieWithCount
{
    private TrieNodeWithCount _root = new();

    public void Insert(string word)
    {
        var cur = _root;
        foreach (char c in word)
        {
            if (!cur.Children.ContainsKey(c))
                cur.Children[c] = new TrieNodeWithCount();
            cur = cur.Children[c];
            cur.PassCount++;    // increment on every pass
        }
        cur.IsEnd = true;
        cur.EndCount++;
    }

    // How many words start with this prefix? O(prefix_len)
    public int CountPrefix(string prefix)
    {
        var cur = _root;
        foreach (char c in prefix)
        {
            if (!cur.Children.TryGetValue(c, out cur!)) return 0;
        }
        return cur.PassCount;
    }

    // How many times was this exact word inserted?
    public int CountWord(string word)
    {
        var cur = _root;
        foreach (char c in word)
        {
            if (!cur.Children.TryGetValue(c, out cur!)) return 0;
        }
        return cur.EndCount;
    }
}
// Insert "apple" 3 times:
// CountWord("apple") → 3
// CountPrefix("app")  → 3
Section 7 — Wildcard Search (LeetCode 211)

Design a data structure that supports adding words and searching with . as a wildcard that matches any single character.

WordDictionary — add and search with wildcards Add "bad", "dad", "mad" Trie: [root] / | \ [b] [d] [m] | | | [a] [a] [a] | | | [d*][d*][d*] search("pad") → false (no 'p' child from root) search(".ad") → '.' can be b, d, or m → check all branches → true search("b..") → b → any char → any char → check all 2-deep paths from b → true
public class WordDictionary
{
    private TrieNode _root = new();

    public void AddWord(string word)
    {
        var cur = _root;
        foreach (char c in word)
        {
            if (!cur.Children.ContainsKey(c))
                cur.Children[c] = new TrieNode();
            cur = cur.Children[c];
        }
        cur.IsEnd = true;
    }

    public bool Search(string word)
        => SearchHelper(_root, word, 0);

    private bool SearchHelper(TrieNode node, string word, int i)
    {
        if (i == word.Length) return node.IsEnd;
        char c = word[i];
        if (c != '.')
        {
            return node.Children.TryGetValue(c, out var next)
                && SearchHelper(next, word, i + 1);
        }
        // '.' matches any character — try all children
        foreach (var child in node.Children.Values)
            if (SearchHelper(child, word, i + 1)) return true;
        return false;
    }
}
// Time: O(m) for exact search, O(26^dots × m) worst case with wildcards
// In practice '.ad' only branches at depth 1 → O(26 × 2) = very fast
Section 8 — Word Search II (Trie + Board DFS)

Given a 2D board and a list of words, find all words that can be formed by adjacent cells (up/down/left/right, no cell reused in one path). Brute force: DFS from every cell for every word = O(words × 4^L). Optimal: build a trie of all words, do ONE DFS from each cell and prune using the trie.

WORD SEARCH II board example: Board: Words: ["oath", "pea", "eat", "rain"] o a a n e t a e i h k r i f l v Strategy: 1. Build trie from word list: ["oath", "pea", "eat", "rain"] 2. Start DFS from EACH cell (i,j) 3. At each cell, follow the trie. If current path IS a word → record it. 4. If no trie child matches → prune (don't go deeper) 5. Optimization: after finding a word, remove it from trie (avoid duplicates)
public IList<string> FindWords(char[][] board, string[] words)
{
    var root = new TrieNode();
    foreach (var w in words)
    {
        var cur = root;
        foreach (var c in w)
        {
            if (!cur.Children.ContainsKey(c)) cur.Children[c] = new TrieNode();
            cur = cur.Children[c];
        }
        cur.Word = w;  // store the complete word at end node
    }

    var result = new List<string>();
    int rows = board.Length, cols = board[0].Length;
    for (int r = 0; r < rows; r++)
        for (int c = 0; c < cols; c++)
            DFS(board, r, c, root, result);
    return result;
}
private static int[][] _dirs = { new[]{0,1},new[]{0,-1},new[]{1,0},new[]{-1,0} };
private void DFS(char[][] board, int r, int c, TrieNode node, List<string> res)
{
    char ch = board[r][c];
    if (ch == '#' || !node.Children.ContainsKey(ch)) return; // visited or not in trie
    var next = node.Children[ch];
    if (next.Word != null)
    {
        res.Add(next.Word);
        next.Word = null;  // avoid duplicates — remove word from trie
    }
    board[r][c] = '#';  // mark visited
    foreach (var d in _dirs)
    {
        int nr = r + d[0], nc = c + d[1];
        if (nr >= 0 && nr < board.Length && nc >= 0 && nc < board[0].Length)
            DFS(board, nr, nc, next, res);
    }
    board[r][c] = ch;  // restore
}
// Time: O(M × 4^L) where M = board cells, L = max word length
// With trie pruning, drastically faster in practice
📝 Note on TrieNode.Word: For Word Search II, we add a string? Word field to TrieNode to store the complete word at the end node. Setting it to null after finding prevents adding duplicates when multiple DFS paths reach the same word.

Key Takeaways:
• Trie = tree where each path root→leaf represents a string
• All words sharing prefix share the same path nodes
• Insert/Search/StartsWith: O(m) where m = word length
• Use array[26] for lowercase-only (fast), Dictionary for Unicode (flexible)
• Wildcard search: at '.' recurse all children
• Word Search II: build trie from word list, prune DFS early
