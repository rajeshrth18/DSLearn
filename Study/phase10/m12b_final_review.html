2. Pattern Recognition Guide

When you see these clues in a problem — reach for the corresponding algorithm.

🔍 "Find two numbers that sum to target" / "Pair with difference k"

→ Two Pointers (sorted array) or HashSet for O(n) lookup. If array is sorted: l++/r--. If unsorted: store complements.

🔍 "Subarray with property X" / "Longest window where..."

→ Sliding Window. Use two pointers l, r. Expand r, shrink l when constraint violated. Track running count/sum with a frequency map or running variable.

🔍 "k-th largest / smallest" / "Top k elements" / "Median of stream"

→ Heap (PriorityQueue). Min-heap of size k for k-th largest. Two heaps (max + min) for median. QuickSelect for O(n) average k-th element.

🔍 "Can you reach from A to B?" / "Are they connected?" / "Count groups"

→ BFS/DFS or Union-Find. BFS for shortest path count. DFS for reachability. Union-Find for dynamic connectivity / grouping queries.

🔍 "Shortest path" / "Minimum steps"

→ BFS (unweighted/equal weights) or Dijkstra (weighted, non-negative). Bellman-Ford if negative weights. Floyd-Warshall if all-pairs needed.

🔍 "Is there a valid ordering?" / "Detect circular dependency"

→ Topological Sort (Kahn's BFS). If all nodes can be processed → no cycle → valid order exists. If remaining nodes with in-degree > 0 → cycle exists.

🔍 "Maximum profit / minimum cost over time" / "State changes (buy/sell/hold)"

→ DP with states. Model each day's state as hold/empty/cooldown. Transition between states by price action. O(n * states) time.

🔍 "How many ways to..." / "Minimum operations to reach..." / "Is it possible?"

→ Dynamic Programming. Define sub-problem state, write recurrence, check overlapping sub-problems. Memoize top-down or tabulate bottom-up.

🔍 "All subsets / all permutations / all combinations"

→ Backtracking. Build choices recursively with undo. Use index to avoid duplicates in combinations. Sort + skip duplicates for unique results.

🔍 "Word search / prefix autocomplete / count words with prefix"

→ Trie. Each node stores children and isEnd. For wildcard matching, DFS with '.' matching any child. Prefix count uses an integer at each node.

🔍 "Binary search on the answer" / "Find minimum k such that condition holds"

→ Binary Search on value space. If a predicate f(k) is monotone (false → true), binary search on k in range [lo, hi]. Check mid with f(mid) in O(n) or O(n log n).

🔍 "n is small (≤ 20)" + subset enumeration + optimal assignment

→ Bitmask DP. State = subset represented as integer. Iterate over all 2^n masks. For each mask, compute best transition from sub-masks.

Common C# Gotchas

Watch Out For:
  • Integer overflow: Use long when multiplying two int values that could exceed 2^31-1 ≈ 2.1 billion. Common in graph weights, DP sums.
  • int.MaxValue + anything = negative: When checking dp[u] + w < dp[v], guard with dp[u] != int.MaxValue.
  • string is immutable: String concatenation in a loop is O(n²). Use StringBuilder instead.
  • List.Remove(value) vs RemoveAt(index): Remove(x) removes the first occurrence of value x. RemoveAt(0) removes by index.
  • Dictionary default: Accessing a missing key throws. Use GetValueOrDefault(key, 0) or check ContainsKey first.
  • char arithmetic: s[i] - 'a' gives 0-25 index. (char)('a' + i) goes back to char.
  • Array.Sort on 2D: Sort int[][] with Array.Sort(arr, (a,b) => a[0] - b[0]).

4. Code Templates

Binary Search Template (find leftmost)

int lo = 0, hi = n; // [lo, hi) — hi is exclusive
while (lo < hi)
{
    int mid = lo + (hi - lo) / 2; // Prevents overflow vs (lo+hi)/2
    if (condition(mid)) hi = mid;   // mid could be the answer — shrink right
    else               lo = mid + 1; // mid is definitely not — skip it
}
// lo == hi == first index where condition is true (or n if none)

DFS Backtracking Template

List<List<int>> result = new();

void Backtrack(int start, List<int> current)
{
    result.Add(new List<int>(current)); // Snapshot current state (if collecting all)
    for (int i = start; i < nums.Length; i++)
    {
        if (i > start && nums[i] == nums[i-1]) continue; // Skip duplicates (sorted array)
        current.Add(nums[i]);   // Choose
        Backtrack(i + 1, current); // Explore (i+1 for combinations, i for repetition)
        current.RemoveAt(current.Count - 1); // Undo
    }
}
Backtrack(0, new List<int>());

BFS Shortest Path Template

var queue   = new Queue<int>();
var visited = new HashSet<int>();
queue.Enqueue(start);
visited.Add(start);
int steps = 0;

while (queue.Count > 0)
{
    int size = queue.Count; // Process entire level at once
    while (size-- > 0)
    {
        int node = queue.Dequeue();
        if (node == target) return steps;
        foreach (int neighbor in GetNeighbors(node))
            if (visited.Add(neighbor)) // Add returns true if new
                queue.Enqueue(neighbor);
    }
    steps++;
}
return -1; // Not reachable

Dijkstra Template

int[] dist = Enumerable.Repeat(int.MaxValue, n).ToArray();
dist[src] = 0;
var pq = new PriorityQueue<int, int>(); // (node, dist)
pq.Enqueue(src, 0);

while (pq.Count > 0)
{
    pq.TryDequeue(out int u, out int d);
    if (d > dist[u]) continue; // Stale entry — skip
    foreach (var (v, w) in adj[u])
        if (dist[u] + w < dist[v])
        {
            dist[v] = dist[u] + w;
            pq.Enqueue(v, dist[v]);
        }
}

Union-Find Template (Path Compression + Union by Rank)

int[] parent = Enumerable.Range(0, n).ToArray();
int[] rank   = new int[n];

int Find(int x) => parent[x] == x ? x : (parent[x] = Find(parent[x]));

bool Union(int a, int b)
{
    int ra = Find(a), rb = Find(b);
    if (ra == rb) return false; // Already connected (cycle!)
    if (rank[ra] < rank[rb]) (ra, rb) = (rb, ra);
    parent[rb] = ra;
    if (rank[ra] == rank[rb]) rank[ra]++;
    return true;
}

Rolling Hash (Substring Comparison in O(1))

const long BASE = 31, MOD = 1_000_000_007;
long[] hash = new long[n + 1], pw = new long[n + 1];
pw[0] = 1;
for (int i = 0; i < n; i++)
{
    hash[i+1] = (hash[i] * BASE + (s[i] - 'a' + 1)) % MOD;
    pw[i+1]   = pw[i] * BASE % MOD;
}
// Hash of s[l..r] (0-indexed, inclusive)
long GetHash(int l, int r) => (hash[r+1] - hash[l] * pw[r-l+1] % MOD + MOD * MOD) % MOD;

5. Interview Strategy

The 5-Minute Approach Framework

  1. Clarify (1 min): Edge cases — empty input? Single element? Negative numbers? Duplicates? What to return if no answer?
  2. Pattern identify (1 min): Map the problem to a known pattern (see Section 2). Say it out loud: "This looks like a sliding window problem because we need a contiguous subarray…"
  3. State the approach (1 min): Explain your algorithm at a high level before coding. Confirm complexity is acceptable.
  4. Code (5-10 min): Write clean, readable code. Use meaningful variable names. Comment non-obvious parts.
  5. Test (2 min): Trace through examples — including edge cases. Fix bugs. Re-confirm complexity.
