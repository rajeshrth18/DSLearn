1. Bitmask DP

When n is small (≤ 20), you can use a bitmask to represent a subset of elements as a DP state. This turns exponential enumeration into manageable DP.

Problem: Assign n tasks to n workers. Cost[i][j] = cost for worker i to do task j. Minimize total cost. (n ≤ 15) State: dp[mask] = min cost to complete the set of tasks in 'mask' The number of set bits = number of workers used so far. Transition: if mask has k set bits, worker k does the next task. // Minimum Cost to Assign Tasks (Hungarian-like via bitmask DP)
public int AssignTasks(int[][] cost)
{
    int n = cost.Length;
    int[] dp = Enumerable.Repeat(int.MaxValue, 1 << n).ToArray();
    dp[0] = 0;

    for (int mask = 0; mask < (1 << n); mask++)
    {
        if (dp[mask] == int.MaxValue) continue;
        int worker = Integer.BitCount(mask); // = how many tasks assigned = which worker is next
        if (worker == n) continue;

        for (int task = 0; task < n; task++)
        {
            if ((mask & (1 << task)) != 0) continue; // Task already assigned
            int next = mask | (1 << task);
            dp[next] = Math.Min(dp[next], dp[mask] + cost[worker][task]);
        }
    }
    return dp[(1 << n) - 1]; // All tasks assigned
}
// Time: O(n² * 2^n), Space: O(2^n)

Traveling Salesman Problem (TSP)

// Visit all n cities exactly once and return to start — minimize total distance
// dp[mask][i] = min cost to visit all cities in mask, ending at city i
public int TSP(int[,] dist, int n)
{
    int[,] dp = new int[1 << n, n];
    for (int i = 0; i < (1 << n); i++)
        for (int j = 0; j < n; j++)
            dp[i, j] = int.MaxValue / 2;

    dp[1, 0] = 0; // Start at city 0, only city 0 visited (mask=0001)

    for (int mask = 1; mask < (1 << n); mask++)
    for (int u = 0; u < n; u++)
    {
        if ((mask & (1 << u)) == 0) continue; // u not in current path
        if (dp[mask, u] == int.MaxValue / 2) continue;

        for (int v = 0; v < n; v++)
        {
            if ((mask & (1 << v)) != 0) continue; // v already visited
            int next = mask | (1 << v);
            dp[next, v] = Math.Min(dp[next, v], dp[mask, u] + dist[u, v]);
        }
    }

    int full = (1 << n) - 1;
    int ans = int.MaxValue;
    for (int u = 1; u < n; u++) // Return to city 0
        ans = Math.Min(ans, dp[full, u] + dist[u, 0]);
    return ans;
}
// Time: O(n² * 2^n), Space: O(n * 2^n) — feasible for n ≤ 20

2. Interval DP

Interval DP solves problems on contiguous subarrays. The key pattern: enumerate the split point k within interval [i, j] and combine results from [i,k] and [k+1,j] (or [i,k] and [k,j]).

General pattern: for length = 2 to n: ← build from small to large intervals for i = 0 to n-length: j = i + length - 1 for k = i to j-1: ← try all split points dp[i][j] = min/max(dp[i][j], combine(dp[i][k], dp[k+1][j], cost)) Base case: dp[i][i] = cost of single element (or 0)

Matrix Chain Multiplication

// Minimum multiplications to chain-multiply matrices
// dims[i] = rows of matrix i, dims[i+1] = cols of matrix i
public int MatrixChain(int[] dims)
{
    int n = dims.Length - 1; // number of matrices
    int[,] dp = new int[n, n];

    for (int len = 2; len <= n; len++)
    for (int i = 0; i <= n - len; i++)
    {
        int j = i + len - 1;
        dp[i, j] = int.MaxValue;
        for (int k = i; k < j; k++)
            dp[i, j] = Math.Min(dp[i, j],
                dp[i, k] + dp[k+1, j] + dims[i] * dims[k+1] * dims[j+1]);
    }
    return dp[0, n - 1];
}
// Time: O(n³), Space: O(n²)

Burst Balloons (LeetCode 312)

// Burst all balloons to maximize coins. Bursting balloon i gives nums[i-1]*nums[i]*nums[i+1]
// Key insight: think about which balloon is burst LAST in interval [i..j]
public int MaxCoins(int[] nums)
{
    // Add boundary balloons of value 1
    int n = nums.Length;
    int[] arr = new int[n + 2];
    arr[0] = arr[n + 1] = 1;
    for (int i = 0; i < n; i++) arr[i + 1] = nums[i];

    int m = arr.Length;
    int[,] dp = new int[m, m]; // dp[i][j] = max coins from open interval (i,j)

    for (int len = 2; len < m; len++)   // interval length
    for (int i = 0; i < m - len; i++)
    {
        int j = i + len;
        for (int k = i + 1; k < j; k++) // k = last balloon burst in (i,j)
            dp[i, j] = Math.Max(dp[i, j],
                dp[i, k] + arr[i] * arr[k] * arr[j] + dp[k, j]);
    }
    return dp[0, m - 1];
}
// Time: O(n³), Space: O(n²)

3. Digit DP

Digit DP counts integers in range [0, n] that satisfy a digit-based condition. The key state variables are: position, tight (are we still bounded by n's digits?), and any problem-specific state (digit sum, count of specific digits, etc.).

Count numbers ≤ 1234 with digit sum = 10 State: dp[pos][sum][tight] - pos: which digit position we're filling (left to right) - sum: current digit sum so far - tight: if true, digit at pos must be ≤ n's digit at pos At each position, try digits 0..limit (limit = n[pos] if tight, else 9) If tight AND we pick digit < n[pos]: subsequent positions are "free" (tight=false) If tight AND we pick n[pos]: still tight in next position Build memoized DFS → count satisfying numbers efficiently
// Count numbers from 1 to n that do NOT contain digit 4 (example)
public int CountNoFour(int n)
{
    string s = n.ToString();
    int len = s.Length;
    // memo[pos][tight] — tight is bool → 2 states
    int[,] memo = new int[len, 2];
    for (int i = 0; i < len; i++) memo[i, 0] = memo[i, 1] = -1;

    return Dp(0, true);

    int Dp(int pos, bool tight)
    {
        if (pos == len) return 1; // Fully built a valid number
        int t = tight ? 1 : 0;
        if (memo[pos, t] != -1) return memo[pos, t];

        int limit = tight ? (s[pos] - '0') : 9;
        int count = 0;
        for (int d = 0; d <= limit; d++)
        {
            if (d == 4) continue; // Skip digit 4
            count += Dp(pos + 1, tight && d == limit);
        }
        return memo[pos, t] = count;
    }
}
// Time: O(10 * len * 2) = O(len), Space: O(len)

4. Tree DP

Tree DP computes values bottom-up from leaves to root. Each node aggregates answers from its children. Common patterns: maximum path sum, diameter, independent set on tree.

Binary Tree Maximum Path Sum (LeetCode 124)

// Each node contributes: value + max(0, best from left) + max(0, best from right)
// But a path can only go in ONE direction from each node (no branching upward)
int maxSum = int.MinValue;

public int MaxPathSum(TreeNode root)
{
    DFS(root);
    return maxSum;
}

private int DFS(TreeNode node)
{
    if (node == null) return 0;
    int left  = Math.Max(0, DFS(node.left));  // Ignore negative branches
    int right = Math.Max(0, DFS(node.right));
    maxSum = Math.Max(maxSum, node.val + left + right); // Path through this node
    return node.val + Math.Max(left, right); // Best single-direction path upward
}
// Time: O(n), Space: O(h) recursion stack

House Robber III — Robbery on a Tree (LeetCode 337)

// Can't rob parent AND child. dp returns (rob_this_node, skip_this_node)
public int Rob(TreeNode root)
{
    var (withRoot, withoutRoot) = DFS(root);
    return Math.Max(withRoot, withoutRoot);
}

private (int rob, int skip) DFS(TreeNode node)
{
    if (node == null) return (0, 0);

    var (lr, ls) = DFS(node.left);
    var (rr, rs) = DFS(node.right);

    int rob  = node.val + ls + rs;              // Rob this: skip both children
    int skip = Math.Max(lr, ls) + Math.Max(rr, rs); // Skip this: best from each child
    return (rob, skip);
}
// Time: O(n), Space: O(h)

5. Stock Trading Problems

The stock problems form a family of DP questions. Master the state transitions and they all become easy.

Unified state machine: States per day: HOLD (own stock), EMPTY (no stock), COOLDOWN (sold yesterday) Transitions: HOLD[i] = max(HOLD[i-1], EMPTY[i-1] - price) ← hold or buy EMPTY[i] = max(EMPTY[i-1], COOLDOWN[i-1]) ← stay empty or end cooldown COOLDOWN[i] = HOLD[i-1] + price ← sold today Answer: max(EMPTY[last], COOLDOWN[last])
// Best Time to Buy and Sell Stock with Cooldown (LeetCode 309)
public int MaxProfit(int[] prices)
{
    int hold = -prices[0], empty = 0, cooldown = 0;

    for (int i = 1; i < prices.Length; i++)
    {
        int newHold     = Math.Max(hold, empty - prices[i]);
        int newEmpty    = Math.Max(empty, cooldown);
        int newCooldown = hold + prices[i]; // Sold today
        hold = newHold; empty = newEmpty; cooldown = newCooldown;
    }
    return Math.Max(empty, cooldown);
}
// Time: O(n), Space: O(1)
// Best Time to Buy with Transaction Fee (LeetCode 714)
public int MaxProfitFee(int[] prices, int fee)
{
    int hold = -prices[0], empty = 0;
    for (int i = 1; i < prices.Length; i++)
    {
        int newHold  = Math.Max(hold,  empty - prices[i]);
        int newEmpty = Math.Max(empty, hold + prices[i] - fee); // Pay fee on sell
        hold = newHold; empty = newEmpty;
    }
    return empty;
}

// At Most K Transactions (LeetCode 188)
public int MaxProfitK(int k, int[] prices)
{
    int n = prices.Length;
    if (k >= n / 2) // Unlimited transactions
        return prices.Zip(prices.Skip(1), (a, b) => Math.Max(0, b - a)).Sum();

    // buy[j] = max profit after buying in at most j transactions
    // sell[j] = max profit after selling in at most j transactions
    int[] buy = Enumerable.Repeat(int.MinValue, k + 1).ToArray();
    int[] sell = new int[k + 1];

    foreach (int price in prices)
    for (int j = k; j >= 1; j--)
    {
        sell[j] = Math.Max(sell[j], buy[j] + price);
        buy[j]  = Math.Max(buy[j],  sell[j - 1] - price);
    }
    return sell[k];
}
// Time: O(n*k), Space: O(k)

6. DP Optimization — Monotonic Deque

When a DP transition requires a sliding window maximum/minimum, a monotonic deque converts O(n²) DP to O(n).

Jump Game VI (LeetCode 1696): dp[i] = nums[i] + max(dp[j]) for j in [i-k, i-1] Naively: O(n*k) — for each i, look back k positions With deque: O(n) — maintain deque of indices with decreasing dp values Deque invariant: front = best index in current window When dp[i] computed: remove from back any index with dp ≤ dp[i] (they're useless)
// Jump Game VI: reach end in max score, jump up to k steps each time
public int MaxResult(int[] nums, int k)
{
    int n = nums.Length;
    int[] dp = new int[n];
    dp[0] = nums[0];
    var dq = new LinkedList<int>(); // Stores indices; front = max dp index in window
    dq.AddLast(0);

    for (int i = 1; i < n; i++)
    {
        // Remove indices outside window [i-k, i-1]
        while (dq.Count > 0 && dq.First.Value < i - k)
            dq.RemoveFirst();

        dp[i] = nums[i] + dp[dq.First.Value]; // Best from window

        // Maintain decreasing monotone: remove worse indices from back
        while (dq.Count > 0 && dp[dq.Last.Value] <= dp[i])
            dq.RemoveLast();
        dq.AddLast(i);
    }
    return dp[n - 1];
}
// Time: O(n), Space: O(n)

DP Problem-Solving Framework:
  1. Define state: What information uniquely describes a sub-problem?
  2. Write recurrence: How does dp[i] depend on smaller sub-problems?
  3. Identify base cases: What's the smallest valid sub-problem?
  4. Determine order: Which direction to fill the table (top-down memoization or bottom-up)?
  5. Optimize space: Can you reduce from O(n²) to O(n) by only keeping previous row?
