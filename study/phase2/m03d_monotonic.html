Section 1 — What is a Monotonic Stack?

A Monotonic Stack is a regular stack with one extra constraint: the elements inside it are always kept in monotonically increasing or decreasing order. Whenever a new element would violate this order, you pop elements until the order is restored.

The power of this constraint: when an element gets popped because a new element is larger (or smaller), that new element is the answer for the popped element — the "next greater" or "previous smaller" value. This transforms O(n²) brute-force searches into O(n) single-pass scans.

🌍 Real-World Analogy — Stock Market: Imagine you're tracking stock prices day by day. For each day, you want to know: "How many days from now until the price is higher than today's?" A brute force approach checks every future day for every day — O(n²). The monotonic stack does it in one pass — O(n). The same pattern appears in temperature forecasting, histogram analysis, water storage calculations, and compiler expression parsing.
📌 Key Insight: Every element enters the stack once and leaves the stack once. Therefore all push and pop operations combined are O(n), giving us O(n) total time complexity even though it looks like there's a loop inside a loop.
Section 2 — Monotonically Increasing vs Decreasing Stack

Section 3 — Pattern: Next Greater Element

For every element in an array, find the first element to the right that is larger than it. Return -1 if none exists. This is the canonical monotonic stack problem.

NEXT GREATER ELEMENT — nums=[2, 1, 2, 4, 3] Use DECREASING monotonic stack (stores indices). When we pop index i because current element is larger, current = NGE for i. i=0, nums[0]=2: stack empty, push 0. stack=[0] i=1, nums[1]=1: 1 < nums[0]=2, just push. stack=[0,1] i=2, nums[2]=2: 2 > nums[1]=1 → POP 1, result[1]=2. 2 = nums[0]=2 → stop. push 2. stack=[0,2] i=3, nums[3]=4: 4 > nums[2]=2 → POP 2, result[2]=4. 4 > nums[0]=2 → POP 0, result[0]=4. push 3. stack=[3] i=4, nums[4]=3: 3 < nums[3]=4 → just push. stack=[3,4] End: remaining in stack have no greater element → result = -1 result[3]=-1, result[4]=-1 FINAL: result = [4, 2, 4, -1, -1]
public static int[] NextGreaterElement(int[] nums)
{
    int n = nums.Length;
    int[] result = new int[n];
    Array.Fill(result, -1);  // Default: no greater element

    Stack<int> stack = new();  // Stores INDICES, not values

    for (int i = 0; i < n; i++)
    {
        // Pop all indices whose value is less than current
        while (stack.Count > 0 && nums[stack.Peek()] < nums[i])
        {
            int idx = stack.Pop();
            result[idx] = nums[i];  // nums[i] is the NGE for idx
        }
        stack.Push(i);
    }
    // Indices still in stack have result=-1 (already filled)
    return result;
}

// [2,1,2,4,3] → [4,2,4,-1,-1]
// [1,3,2,4]   → [3,4,4,-1]
Section 4 — Pattern: Previous Smaller Element

For every element, find the nearest element to the left that is smaller. Use a monotonically increasing stack — the top of the stack when you push element i is the previous smaller element for i.

public static int[] PreviousSmallerElement(int[] nums)
{
    int n = nums.Length;
    int[] result = new int[n];
    Stack<int> stack = new();  // stores indices, increasing values

    for (int i = 0; i < n; i++)
    {
        // Pop elements >= current (they can't be "previous smaller")
        while (stack.Count > 0 && nums[stack.Peek()] >= nums[i])
            stack.Pop();

        // Whatever's on top NOW is the previous smaller element
        result[i] = stack.Count == 0 ? -1 : nums[stack.Peek()];
        stack.Push(i);
    }
    return result;
}

// [2,1,2,4,3] → [-1,-1,1,2,2]
// Meaning:
//   PSE of index 0 (val=2): -1  (no smaller to the left)
//   PSE of index 1 (val=1): -1  (no smaller to the left)
//   PSE of index 2 (val=2):  1  (index 1, val=1)
//   PSE of index 3 (val=4):  2  (index 2, val=2)
//   PSE of index 4 (val=3):  2  (index 2, val=2)
Section 5 — Next Greater Element II (Circular Array)

Same as Next Greater Element, but the array wraps around — the next greater element of the last element might be in the beginning. The trick: process the array twice by looping from 0 to 2n-1 using modulo, but only record results in the first pass (indices 0..n-1).

public static int[] NextGreaterElements(int[] nums)
{
    int n = nums.Length;
    int[] result = new int[n];
    Array.Fill(result, -1);
    Stack<int> stack = new();

    // Loop TWICE — simulates circular array
    for (int i = 0; i < 2 * n; i++)
    {
        int idx = i % n;  // Wrap around
        while (stack.Count > 0 && nums[stack.Peek()] < nums[idx])
            result[stack.Pop()] = nums[idx];
        if (i < n) stack.Push(idx);  // Only push in first pass
    }
    return result;
}

// [1,2,1] → [2,-1,2]
//   Index 0 (val=1): next greater is 2 (index 1)
//   Index 1 (val=2): no greater element in the whole array → -1
//   Index 2 (val=1): wraps around, next greater is 2 (index 1)
Section 6 — Application: Daily Temperatures

Given daily temperatures, for each day find how many days until a warmer temperature. Return 0 if no warmer day exists. This is the "Next Greater Element" with distances.

DAILY TEMPERATURES — temps=[73,74,75,71,69,72,76,73] result[i] = distance to next warmer day (0 if none) Stack stores indices. When we find a warmer day, distance = i - popped_index. i=0 T=73: push 0. stack=[0] i=1 T=74: 74>73 → pop0, result[0]=1-0=1. push 1. stack=[1] i=2 T=75: 75>74 → pop1, result[1]=2-1=1. push 2. stack=[2] i=3 T=71: 71<75 → push. stack=[2,3] i=4 T=69: 69<71 → push. stack=[2,3,4] i=5 T=72: 72>69→pop4,result[4]=5-4=1. 72>71→pop3,result[3]=5-3=2. 72<75→push5. stack=[2,5] i=6 T=76: 76>72→pop5,result[5]=6-5=1. 76>75→pop2,result[2]=6-2=4. push6. stack=[6] i=7 T=73: 73<76→push. stack=[6,7] End: stack=[6,7] → result[6]=0, result[7]=0 RESULT: [1, 1, 4, 2, 1, 1, 0, 0]
public static int[] DailyTemperatures(int[] temps)
{
    int[] result = new int[temps.Length];
    Stack<int> stack = new();  // Indices

    for (int i = 0; i < temps.Length; i++)
    {
        while (stack.Count > 0 && temps[stack.Peek()] < temps[i])
        {
            int j = stack.Pop();
            result[j] = i - j;  // Days until warmer
        }
        stack.Push(i);
    }
    return result;
}

// [73,74,75,71,69,72,76,73] → [1,1,4,2,1,1,0,0]
Section 7 — Application: Largest Rectangle in Histogram

Given bar heights in a histogram, find the largest rectangle area. This is one of the hardest and most elegant monotonic stack problems — O(n).

🌍 Real-World: This pattern appears in: finding the largest contiguous region of equal-height buildings, max area sub-matrix in an all-ones binary matrix, and signal processing (largest amplitude window that fits under a threshold curve).
LARGEST RECTANGLE IN HISTOGRAM heights = [2, 1, 5, 6, 2, 3] The largest rectangle is the one spanning indices 2–3 (heights 5,6) with height limited to min(5,6)=5, width=2, area=10. STRATEGY: For each bar i, what's the widest rectangle where bar i is the SHORTEST bar? The width extends from (previous smaller left) to (next smaller right). Using monotonic stack to find boundaries in one pass: Stack stores indices. When we pop bar i because current bar is shorter, current bar's index (right) and new top (left) give us the width. i=0 h=2: push 0. [0] i=1 h=1: 1<h[0]=2 → pop 0. Width=i-(-1)-1=1. Area=2*1=2. stack empty, push 1. [1] i=2 h=5: push. [1,2] i=3 h=6: push. [1,2,3] i=4 h=2: 2<h[3]=6 → pop 3. Left=stack.top=2. Width=4-2-1=1. Area=6*1=6. 2<h[2]=5 → pop 2. Left=stack.top=1. Width=4-1-1=2. Area=5*2=10. ← MAX 2>h[1]=1 → stop. push 4. [1,4] i=5 h=3: push. [1,4,5] End: pop 5. Left=4. Width=6-4-1=1. Area=3*1=3. pop 4. Left=1. Width=6-1-1=4. Area=2*4=8. pop 1. Left=-1. Width=6-(-1)-1=6. Area=1*6=6. MAX AREA = 10
public static int LargestRectangleArea(int[] heights)
{
    int n = heights.Length;
    int maxArea = 0;
    Stack<int> stack = new();  // Increasing stack of indices

    for (int i = 0; i <= n; i++)
    {
        // Use height=0 at i=n to flush remaining stack
        int h = i == n ? 0 : heights[i];

        while (stack.Count > 0 && heights[stack.Peek()] > h)
        {
            int height = heights[stack.Pop()];
            // Left boundary: new top of stack (or -1 if empty)
            int left = stack.Count == 0 ? -1 : stack.Peek();
            int width = i - left - 1;
            maxArea = Math.Max(maxArea, height * width);
        }
        stack.Push(i);
    }
    return maxArea;
}

// [2,1,5,6,2,3] → 10
// [2,4]         → 4
// [1]           → 1
Section 8 — Application: Trapping Rain Water

Given an elevation map (bar heights), compute how much water is trapped after rain. The classic O(n) stack approach: water fills up between bars whenever a taller bar is found to the right.

TRAPPING RAIN WATER heights = [0,1,0,2,1,0,1,3,2,1,2,1] 3 | | 2 | | | | | | 1 | | | | | | | | | | | | | 0 1 0 2 1 0 1 3 2 1 2 1 Water trapped (shown as ~): 3 | | 2 | | |~|~| |~| | 1 | |~|~| |~|~| |~| | | | | |~| Total trapped water = 6 APPROACH: Stack stores indices in decreasing height order. When current bar is taller than stack top: - Pop bottom of pool - New water height = min(current, new_top) - bottom - Width = current_index - new_top_index - 1
public static int Trap(int[] height)
{
    int water = 0;
    Stack<int> stack = new();  // Decreasing heights

    for (int i = 0; i < height.Length; i++)
    {
        while (stack.Count > 0 && height[stack.Peek()] < height[i])
        {
            int bottom = stack.Pop();
            if (stack.Count == 0) break;

            int left = stack.Peek();
            int boundedHeight = Math.Min(height[left], height[i]) - height[bottom];
            int width = i - left - 1;
            water += boundedHeight * width;
        }
        stack.Push(i);
    }
    return water;
}

// [0,1,0,2,1,0,1,3,2,1,2,1] → 6
// [4,2,0,3,2,5]              → 9
Section 9 — Monotonic Queue (Deque) — Sliding Window Maximum

The Monotonic Queue (deque variant) keeps elements in decreasing order. It answers: "What is the maximum in the current window?" in O(1) per query — the front of the deque is always the current window's maximum. Total time: O(n).

SLIDING WINDOW MAXIMUM — nums=[1,3,-1,-3,5,3,6,7], k=3 Deque stores INDICES. Maintains DECREASING values. i=0 v=1: dq=[0] i=1 v=3: 3>1→remove 0. dq=[1] i=2 v=-1: -1<3, just add. dq=[1,2] Window[0..2]: max=nums[1]=3 i=3 v=-3: -3<-1, add. dq=[1,2,3] Window[1..3]: max=nums[1]=3 (check front: 1 still in window [1..3] ✓) i=4 v=5: 5>-3→remove 3. 5>-1→remove 2. 5>3→remove 1. dq=[4] Window[2..4]: max=nums[4]=5 i=5 v=3: 3<5, add. dq=[4,5] Window[3..5]: max=nums[4]=5 i=6 v=6: 6>3→remove 5. 6>5→remove 4. dq=[6] Window[4..6]: max=nums[6]=6 i=7 v=7: 7>6→remove 6. dq=[7] Window[5..7]: max=nums[7]=7 RESULT: [3, 3, 5, 5, 6, 7]
public static int[] MaxSlidingWindow(int[] nums, int k)
{
    int n = nums.Length;
    if (n == 0 || k == 0) return [];

    int[] result = new int[n - k + 1];
    LinkedList<int> dq = new();  // Decreasing monotonic queue — stores indices

    for (int i = 0; i < n; i++)
    {
        // Remove indices that are outside the window
        while (dq.Count > 0 && dq.First!.Value < i - k + 1)
            dq.RemoveFirst();

        // Maintain decreasing order: remove smaller elements from back
        while (dq.Count > 0 && nums[dq.Last!.Value] < nums[i])
            dq.RemoveLast();

        dq.AddLast(i);

        // Record result once we have a full window
        if (i >= k - 1)
            result[i - k + 1] = nums[dq.First!.Value];
    }
    return result;
}

// [1,3,-1,-3,5,3,6,7], k=3 → [3,3,5,5,6,7]
// [1], k=1                  → [1]

💡 Recognition Pattern — When to Use Monotonic Stack:
  • Problem asks for "next/previous greater/smaller" relationships
  • Problem involves a histogram or bar-chart structure
  • Brute force is O(n²) and involves comparing each element to all others
  • The phrase "nearest", "closest", "immediately before/after" appears
  • Sliding window problems that need max/min efficiently
📌 O(n) Why? — Amortized Analysis: The nested while-loop looks like O(n²), but it's O(n) amortized. Each element is pushed once and popped at most once. Therefore, the total number of push + pop operations across the entire algorithm is bounded by 2n — giving O(n) overall.
