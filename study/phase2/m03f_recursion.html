Section 1 — What is Recursion?

Recursion is when a function calls itself to solve a smaller version of the same problem. The key insight is divide and conquer: break a hard problem into identical but smaller sub-problems until you reach a trivially solvable case.

Real world analogy: "To understand recursion, you must first understand recursion." More practically: ┌─────────────────────────────────────────────────────┐ │ To count stairs from step N down to 0: │ │ → Count 1 + (stairs from step N-1 down to 0) │ │ → Count 1 + (stairs from step N-2 down to 0) │ │ → ... │ │ → When N == 0, you're done! Return 0. │ └─────────────────────────────────────────────────────┘

Every recursive solution has exactly two parts:

  • Base case: The stopping condition — a problem small enough to answer directly.
  • Recursive case: Break the problem into a smaller version and call yourself.
📌 Key Insight: Recursion is not magic — it's just the call stack doing the work for you. Every recursive function can be rewritten with an explicit stack, but recursion is often far more readable.
Section 2 — The Call Stack in Memory

Every function call pushes a stack frame onto the call stack. When a function returns, its frame is popped. Recursion just means pushing many frames of the same function.

Example: Factorial(4) — Call Stack Trace

int Factorial(int n)
{
    if (n == 0) return 1;          // base case
    return n * Factorial(n - 1);   // recursive case
}
WINDING DOWN (pushing frames): UNWINDING (returning values): Factorial(4) ← called first Factorial(4) = 4 * 6 = 24 ✅ │ │ └─ Factorial(3) └─ Factorial(3) = 3 * 2 = 6 │ │ └─ Factorial(2) └─ Factorial(2) = 2 * 1 = 2 │ │ └─ Factorial(1) └─ Factorial(1) = 1 * 1 = 1 │ │ └─ Factorial(0) = 1 ← base case └─ Factorial(0) = 1 Stack grows downward → Stack unwinds upward ← Each call PAUSES waiting Each return RESUMES the caller
💡 Mental Model: Think of recursion as a stack of sticky notes. Each call writes a note and pins it on a spike. When the base case is hit, you pull notes off one by one, reading each answer on the way back up.

Stack Frame Contents

Each frame on the call stack stores:

  • The function's local variables (e.g., n)
  • The return address — where to resume in the caller
  • The return value slot
⚠️ Stack Overflow: The call stack has a fixed size (~1MB in .NET by default). If recursion goes too deep without hitting a base case, you get a StackOverflowException. This is always caused by a missing or unreachable base case.
Section 3 — The Three Rules of Recursion

A correct recursive function must satisfy all three rules:

RuleWhat It MeansExample Violation
1. Must have a base caseAt least one case that returns directly without recursingForgetting if (n == 0)
2. Must change state towards base caseEach call must make progress — get closer to the base caseCalling Factorial(n) instead of Factorial(n-1)
3. Must call itselfThe function must invoke itself (otherwise it's not recursive)—
 
// CORRECT — all three rules satisfied
int Sum(int n)
{
    if (n == 0) return 0;        // Rule 1: base case
    return n + Sum(n - 1);       // Rule 2: n decreases  Rule 3: calls itself
}

// BROKEN — missing base case → StackOverflowException
int BadSum(int n)
{
    return n + BadSum(n - 1);   // goes to -∞, never stops
}
Section 4 — Classic Recursion Examples in C#

4.1 — Factorial

// n! = n × (n-1) × (n-2) × ... × 1
int Factorial(int n)
{
    if (n <= 1) return 1;
    return n * Factorial(n - 1);
}
// Factorial(5) → 5*4*3*2*1 = 120   O(n) time, O(n) space (stack frames)

4.2 — Fibonacci

// Fib(n) = Fib(n-1) + Fib(n-2)    base: Fib(0)=0, Fib(1)=1
int Fib(int n)
{
    if (n <= 1) return n;
    return Fib(n - 1) + Fib(n - 2);
}
// WARNING: O(2^n) time — exponential! Each call spawns two children.
// We fix this with memoization in Section 8.

4.3 — Power (x^n)

double Power(double x, int n)
{
    if (n == 0) return 1;
    if (n < 0)  return 1.0 / Power(x, -n);

    double half = Power(x, n / 2);           // fast exponentiation
    return (n % 2 == 0) ? half * half
                          : half * half * x;
}
// O(log n) — halving the problem each time is far better than O(n)

4.4 — Reverse a String

string Reverse(string s)
{
    if (s.Length <= 1) return s;
    return Reverse(s[1..]) + s[0];
}
// Reverse("hello") → Reverse("ello") + 'h' → ... → "olleh"

4.5 — Binary Search (Recursive)

int BinarySearch(int[] arr, int target, int lo, int hi)
{
    if (lo > hi) return -1;               // base: not found

    int mid = lo + (hi - lo) / 2;
    if (arr[mid] == target) return mid;   // base: found
    if (arr[mid] < target)
        return BinarySearch(arr, target, mid + 1, hi);
    return BinarySearch(arr, target, lo, mid - 1);
}
// O(log n) — halves the search space each call

4.6 — Flatten a Nested List

// Works on arbitrarily deep nesting — perfect for recursion
IEnumerable<int> Flatten(IEnumerable<object> items)
{
    foreach (var item in items)
    {
        if (item is int n)
            yield return n;
        else if (item is IEnumerable<object> nested)
            foreach (var x in Flatten(nested))
                yield return x;
    }
}

Section 6 — Tree-Shaped Recursion

Linear recursion makes one recursive call. Tree-shaped recursion makes multiple calls per invocation, creating an exponential call tree.

Fib(5) call tree — each node spawns two children: Fib(5) / \ Fib(4) Fib(3) / \ / \ Fib(3) Fib(2) Fib(2) Fib(1) / \ / \ / \ Fib(2) Fib(1) ... ... ... Fib(3) is computed TWICE, Fib(2) THREE TIMES → Wasted work! Total calls: O(2^n) — exponential growth

This is why naive Fibonacci is O(2^n). The fix is memoization — cache results so each subproblem is solved only once.

Section 7 — Common Pitfalls

7.1 — Missing Base Case

// BUG: no base case → infinite recursion → StackOverflowException
int CountDown(int n)
{
    return CountDown(n - 1);  // ← never stops
}

// FIX
int CountDown(int n)
{
    if (n == 0) return 0;   // ← base case added
    return CountDown(n - 1);
}

7.2 — Not Making Progress

// BUG: calling with same argument → infinite loop
int Bad(int n)
{
    if (n == 0) return 0;
    return Bad(n);           // ← should be Bad(n - 1)
}

7.3 — Off-by-One in Base Case

// BUG: base case at n==1 misses n==0 → crashes on Factorial(0)
int Factorial(int n)
{
    if (n == 1) return 1;   // ← should be n <= 1
    return n * Factorial(n - 1);
}

7.4 — Accidental Mutation of Shared State

int count = 0;
void CountNodes(TreeNode node)
{
    if (node == null) return;
    count++;                       // mutating outer state — be careful!
    CountNodes(node.Left);
    CountNodes(node.Right);
}
// Prefer: return int from the function instead of mutating an outer variable
Section 8 — Memoization Preview

Memoization = cache the result of a recursive call so if it's called again with the same input, return the cached answer instantly. This converts tree recursion from O(2^n) to O(n).

// Naive Fibonacci — O(2^n)
int Fib(int n) =>
    n <= 1 ? n : Fib(n - 1) + Fib(n - 2);

// Memoized Fibonacci — O(n) time, O(n) space
int[] _memo = new int[100];

int FibMemo(int n)
{
    if (n <= 1) return n;
    if (_memo[n] != 0) return _memo[n];  // cache hit!
    _memo[n] = FibMemo(n - 1) + FibMemo(n - 2);
    return _memo[n];
}

// Using Dictionary for general memoization
Dictionary<int, int> _cache = new();

int FibDict(int n)
{
    if (n <= 1) return n;
    if (_cache.TryGetValue(n, out int cached)) return cached;
    return _cache[n] = FibDict(n - 1) + FibDict(n - 2);
}
Memoized Fib(5) — each subproblem solved ONCE: Fib(5) Fib(4) Fib(3) Fib(2) Fib(1) = 1 [cached] Fib(0) = 0 [cached] Fib(2) = 1 ← cache hit! No recomputation. Fib(3) = 2 ← cache hit! Fib(4) = 3 ← cache hit! Fib(5) = 5 Only 5 unique calls instead of 15 in naive version.
📌 Coming Up: Memoization is the gateway to Dynamic Programming (Phase 6). In DP we convert memoized recursion into bottom-up table filling — no call stack needed at all.

Easy Problems

Problem 1 — Sum of Digits

Write a recursive function that returns the sum of digits of a positive integer.
SumDigits(1234) → 10

Show Hint
Last digit = n % 10. Remaining number = n / 10. Base case: n < 10.
Show Solution 
int SumDigits(int n)
{
    if (n < 10) return n;
    return (n % 10) + SumDigits(n / 10);
}
// SumDigits(1234) → 4 + SumDigits(123) → 4+3+SumDigits(12) → 4+3+2+1 = 10

Problem 2 — Count Occurrences in Array

Recursively count how many times target appears in arr[lo..hi].

Show Hint
Base case: lo > hi → return 0. Check arr[lo], then recurse on lo+1.
Show Solution 
int Count(int[] arr, int target, int lo, int hi)
{
    if (lo > hi) return 0;
    int found = arr[lo] == target ? 1 : 0;
    return found + Count(arr, target, lo + 1, hi);
}

Problem 3 — Is Palindrome?

Recursively check if a string is a palindrome (reads same forwards and backwards).

Show Hint
Compare first and last characters. If equal, recurse on the middle substring.
Show Solution 
bool IsPalindrome(string s)
{
    if (s.Length <= 1) return true;
    if (s[0] != s[^1]) return false;
    return IsPalindrome(s[1..^1]);
}

Problem 4 — Power of 2?

Recursively determine if a positive integer n is a power of 2.

Show Hint
If n == 1 → true. If n is odd → false. Otherwise recurse on n / 2.
Show Solution 
bool IsPowerOf2(int n)
{
    if (n == 1) return true;
    if (n <= 0 || n % 2 != 0) return false;
    return IsPowerOf2(n / 2);
}

Problem 5 — GCD (Greatest Common Divisor)

Implement the Euclidean algorithm recursively: GCD(a, b) = GCD(b, a % b)

Show Solution 
int GCD(int a, int b)
{
    if (b == 0) return a;
    return GCD(b, a % b);
}
// GCD(48, 18) → GCD(18, 12) → GCD(12, 6) → GCD(6, 0) → 6

Medium Problems

Problem 6 — Tower of Hanoi

Move n discs from peg A to peg C using peg B as auxiliary. Print each move.

Show Hint
Move top n-1 discs to B, move disc n from A to C, move n-1 discs from B to C.
Show Solution 
void Hanoi(int n, char from, char to, char aux)
{
    if (n == 1) { Console.WriteLine($"Move disc 1 from {from} to {to}"); return; }
    Hanoi(n - 1, from, aux, to);
    Console.WriteLine($"Move disc {n} from {from} to {to}");
    Hanoi(n - 1, aux, to, from);
}
// Hanoi(3, 'A', 'C', 'B') → 7 moves (2^n - 1 moves for n discs)

Problem 7 — Flatten Nested Array

Given a List<object> where each element is either an int or another List<object>, return a flat list of all integers.

Show Solution 
List<int> Flatten(List<object> nested)
{
    var result = new List<int>();
    foreach (var item in nested)
    {
        if (item is int n) result.Add(n);
        else if (item is List<object> inner) result.AddRange(Flatten(inner));
    }
    return result;
}

Problem 8 — All Permutations of a String

Print all permutations of a given string. Perms("abc") → "abc", "acb", "bac", "bca", "cab", "cba"

Show Hint
Fix each character at position 0 and recurse on the remaining characters. Use string slicing or char swapping.
Show Solution 
void Permutations(string prefix, string remaining)
{
    if (remaining.Length == 0) { Console.WriteLine(prefix); return; }
    for (int i = 0; i < remaining.Length; i++)
    {
        char ch = remaining[i];
        string rest = remaining[..i] + remaining[(i + 1)..];
        Permutations(prefix + ch, rest);
    }
}
// Call: Permutations("", "abc")

Problem 9 — Count Paths in a Grid

How many unique paths are there from top-left to bottom-right of an m×n grid, moving only right or down?

Show Hint
At each cell you came from either the left or above. Base case: row 0 or col 0 has exactly 1 path.
Show Solution 
// Naive recursive — O(2^(m+n)) without memo
int CountPaths(int m, int n)
{
    if (m == 1 || n == 1) return 1;
    return CountPaths(m - 1, n) + CountPaths(m, n - 1);
}

// With memoization — O(m*n)
int CountPathsMemo(int m, int n, Dictionary<(int,int), int> memo)
{
    if (m == 1 || n == 1) return 1;
    if (memo.TryGetValue((m, n), out int v)) return v;
    return memo[(m, n)] = CountPathsMemo(m-1, n, memo) + CountPathsMemo(m, n-1, memo);
}

Problem 10 — Merge Sort

Implement merge sort recursively on int[]. Classic divide & conquer pattern.

Show Solution 
void MergeSort(int[] arr, int lo, int hi)
{
    if (lo >= hi) return;                      // base: 1 element
    int mid = lo + (hi - lo) / 2;
    MergeSort(arr, lo, mid);                    // sort left half
    MergeSort(arr, mid + 1, hi);               // sort right half
    Merge(arr, lo, mid, hi);                    // merge sorted halves
}

void Merge(int[] arr, int lo, int mid, int hi)
{
    int[] temp = arr[lo..(hi + 1)];
    int i = 0, j = mid - lo + 1, k = lo;
    while (i <= mid - lo && j < temp.Length)
        arr[k++] = temp[i] <= temp[j] ? temp[i++] : temp[j++];
    while (i <= mid - lo) arr[k++] = temp[i++];
    while (j < temp.Length) arr[k++] = temp[j++];
}
// O(n log n) time — the gold standard for comparison-based sorting

Key Takeaways:
• Every recursive function needs a base case (stops recursion) and a recursive case (makes progress)
• The call stack stores one frame per active call — deep recursion can cause StackOverflowException
• Tree-shaped recursion (multiple calls per level) is often O(2^n) — memoization brings it to O(n)
• Recursion shines for trees, graphs, divide & conquer, and backtracking
• Memoization is the bridge between recursion and Dynamic Programming (Phase 6)
