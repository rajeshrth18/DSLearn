Section 1 — The Problem: Why Naive Detection Fails

A cycle in a linked list occurs when a node's Next pointer points back to a node already visited — the traversal loops forever and never reaches null.

LINKED LIST WITH A CYCLE head │ ▼ [1] → [2] → [3] → [4] → [5] ▲ │ └──────────────┘ cycle entry = node(3) If you traverse: 1 → 2 → 3 → 4 → 5 → 3 → 4 → 5 → 3 → ... (infinite loop)

1.1 — Naive Approach: HashSet

The obvious solution: record every visited node in a HashSet. If you see a node again — cycle detected.

C# 
public static bool HasCycleHashSet(ListNode? head)
{
    var seen = new HashSet<ListNode>();
    ListNode? cur = head;
    while (cur != null)
    {
        if (!seen.Add(cur)) return true;  // Add returns false if already present
        cur = cur.Next;
    }
    return false;
}
Python 
def has_cycle_hash_set(head: ListNode) -> bool:
    seen = set()
    curr = head
    while curr:
        if id(curr) in seen: return True   # identity = pointer address
        seen.add(id(curr))
        curr = curr.next
    return False

Time: O(n)  |  Space: O(n) — stores every node reference.

⚠️ The O(n) Space Problem: For a list with 10 million nodes, that's 10 million object references in memory just to answer a yes/no question. Floyd's algorithm solves it in O(1) space — just two pointer variables.
Section 2 — Floyd's Algorithm: Phase 1 — Detect the Cycle

Robert W. Floyd discovered a brilliant insight: use two pointers moving at different speeds through the list. The slow pointer (tortoise) advances one step at a time. The fast pointer (hare) advances two steps at a time.

  • If there is no cycle: the fast pointer reaches null and exits.
  • If there is a cycle: the fast pointer laps the slow pointer — they must eventually meet inside the cycle.
PHASE 1 — TORTOISE & HARE MEET (cycle: 3→4→5→3) Step 0: slow=1, fast=1 Step 1: slow=2, fast=3 (fast moves 2 steps) Step 2: slow=3, fast=5 Step 3: slow=4, fast=4 ← MEET! Cycle confirmed. head │ ▼ [1] → [2] → [3] → [4] → [5] ▲ ▲ │ │ slow fast meet here (step 3)

2.1 — Why Do They Always Meet?

Once both pointers are inside the cycle, think of them as two runners on a circular track. The fast runner gains 1 step per iteration on the slow runner. If the cycle length is C, the gap closes by 1 each step — they meet within at most C steps.

C# 
public class ListNode
{
    public int Val;
    public ListNode? Next;
    public ListNode(int val) { Val = val; }
}

// Phase 1: Detect if a cycle exists — O(n) time, O(1) space
public static bool HasCycle(ListNode? head)
{
    ListNode? slow = head;
    ListNode? fast = head;

    while (fast != null && fast.Next != null)
    {
        slow = slow!.Next;          // Tortoise: 1 step
        fast = fast.Next.Next;      // Hare: 2 steps

        if (slow == fast) return true;  // They met — cycle exists!
    }

    return false;  // fast hit null — no cycle
}
// LeetCode 141 — Linked List Cycle
Python 
class ListNode:
    def __init__(self, val=0): self.val = val; self.next = None

def has_cycle(head: ListNode) -> bool:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next           # 1 step
        fast = fast.next.next      # 2 steps
        if slow == fast: return True
    return False
💡 Why check fast.Next != null? The hare moves two steps: fast.Next.Next. If the list has even length, fast lands on null. If odd, fast.Next is null. Both must be checked to avoid a NullReferenceException.
Section 3 — Phase 2: Find the Cycle Entry Point

Once slow and fast have met, we can find exactly where the cycle begins with a clever mathematical trick — no extra memory needed.

3.1 — The Math

Define these distances:

  • F = distance from head to cycle entry
  • C = cycle length
  • a = distance from cycle entry to meeting point (inside cycle)

When they meet:

  • Slow has traveled: F + a
  • Fast has traveled: F + a + k·C (lapped k times)
  • Fast traveled 2× slow: 2(F + a) = F + a + k·C
  • Simplify: F = k·C − a — meaning F ≡ C − a (mod C)
WHAT DOES F = C - a MEAN? Distance from head → entry = F Distance from meeting point → entry (going forward) = C - a So: if one pointer starts at head, another at the meeting point, and BOTH move 1 step at a time, they will meet exactly at the entry! head meeting point │ │ ▼ F ▼ [1] → [2] → [3] → [4] → [5] ▲ C-a │ entry ──┘ │ └────────────────┘ ptr1 starts at head (travels F to reach entry) ptr2 starts at meeting point (travels C-a to reach entry) F = C-a → they arrive at the SAME time!
C# 
// Phase 2: Find the START of the cycle — O(n) time, O(1) space
public static ListNode? DetectCycleEntry(ListNode? head)
{
    ListNode? slow = head;
    ListNode? fast = head;

    // Phase 1: Find meeting point
    while (fast != null && fast.Next != null)
    {
        slow = slow!.Next;
        fast = fast.Next.Next;
        if (slow == fast) break;  // Met inside cycle
    }

    // No cycle
    if (fast == null || fast.Next == null) return null;

    // Phase 2: Find entry point
    // Reset one pointer to head; both now move at speed 1
    slow = head;
    while (slow != fast)
    {
        slow = slow!.Next;
        fast = fast!.Next;
    }

    return slow;  // Both arrive at the cycle entry simultaneously
}
// LeetCode 142 — Linked List Cycle II
Python 
def detect_cycle_entry(head: ListNode) -> ListNode:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next; fast = fast.next.next
        if slow == fast: break
    else: return None   # no cycle
    slow = head             # Phase 2: reset slow to head
    while slow != fast:
        slow = slow.next; fast = fast.next
    return slow             # cycle entry node
📝 Key Insight: The second phase doesn't need the fast pointer to be fast anymore. After the meeting point, reset one pointer to head. Move BOTH at speed 1. They'll converge at the cycle entry due to the F = C - a relationship.
Section 4 — Find the Cycle Length

After detecting the meeting point in Phase 1, you can measure the cycle length by counting steps from the meeting point back to itself.

C# 
public static int CycleLength(ListNode? head)
{
    ListNode? slow = head, fast = head;

    // Phase 1 — find meeting point
    while (fast != null && fast.Next != null)
    {
        slow = slow!.Next;
        fast = fast.Next.Next;
        if (slow == fast)
        {
            // Count cycle length from meeting point
            int length = 1;
            ListNode? current = slow!.Next;
            while (current != slow)
            {
                length++;
                current = current!.Next;
            }
            return length;
        }
    }
    return 0;  // No cycle
}

// Example usage
// List: 1→2→3→4→5→3 (cycle of length 3: 3→4→5→3)
// CycleLength(...) = 3
Python 
def cycle_length(head: ListNode) -> int:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next; fast = fast.next.next
        if slow == fast:
            length = 1; curr = slow.next
            while curr != slow:
                curr = curr.next; length += 1
            return length
    return 0
Section 5 — The Happy Number Application

Floyd's algorithm works on any sequence that might loop, not just linked lists. The "Happy Number" problem (LeetCode 202) is a perfect example.

A happy number: repeatedly sum the squares of digits. If the sequence reaches 1 — happy. If it enters a cycle that doesn't include 1 — not happy. Without Floyd's, you'd use a HashSet.

HAPPY NUMBER SEQUENCE — n = 19 19 → 1²+9² = 82 → 8²+2² = 68 → 6²+8² = 100 → 1²+0²+0² = 1 ✓ HAPPY! UNHAPPY NUMBER SEQUENCE — n = 4 4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4 → 16 → ... (CYCLE!)
C# 
public class HappyNumber
{
    private static int SumSquaredDigits(int n)
    {
        int sum = 0;
        while (n > 0)
        {
            int d = n % 10;
            sum += d * d;
            n /= 10;
        }
        return sum;
    }

    public static bool IsHappy(int n)
    {
        int slow = n;
        int fast = SumSquaredDigits(n);  // fast starts 1 step ahead

        while (fast != 1 && slow != fast)
        {
            slow = SumSquaredDigits(slow);
            fast = SumSquaredDigits(SumSquaredDigits(fast));
        }

        return fast == 1;  // Reached 1 = happy; slow==fast (cycle) = not happy
    }

    static void Main()
    {
        Console.WriteLine(IsHappy(19));  // True
        Console.WriteLine(IsHappy(4));   // False
        Console.WriteLine(IsHappy(7));   // True
    }
}
Python 
def sum_squared_digits(n: int) -> int:
    total = 0
    while n:
        d = n % 10; total += d * d; n //= 10
    return total

def is_happy(n: int) -> bool:
    slow = fast = n
    while True:
        slow = sum_squared_digits(slow)
        fast = sum_squared_digits(sum_squared_digits(fast))
        if slow == fast: break
    return slow == 1
Section 6 — Find the Duplicate Number (Array as Implicit Linked List)

LeetCode 287 is one of the most elegant applications of Floyd's algorithm. Given an array of n+1 integers where each value is in [1, n], find the duplicate — using O(1) extra space.

The key insight: treat each array value as a "next" pointer. Index i points to index nums[i]. Since one value is duplicated, two indices map to the same "next" — creating a cycle whose entry point is the duplicate value.

ARRAY AS IMPLICIT LINKED LIST — nums = [1,3,4,2,2] Index: 0 1 2 3 4 Value: 1 3 4 2 2 Traversal following nums[i]: start at index 0 0 → nums[0]=1 → nums[1]=3 → nums[3]=2 → nums[2]=4 → nums[4]=2 → nums[2]=4 → ... Written as pointer chain: 0 → 1 → 3 → 2 → 4 ▲ │ └───────┘ Cycle entry = index 2, which means nums[2]=2 is visited twice → duplicate is 2!
C# 
public static int FindDuplicate(int[] nums)
{
    // Phase 1: Find meeting point inside cycle
    int slow = nums[0];
    int fast = nums[0];

    do
    {
        slow = nums[slow];          // 1 step: follow one pointer
        fast = nums[nums[fast]];    // 2 steps: follow two pointers
    } while (slow != fast);

    // Phase 2: Find cycle entry = duplicate number
    slow = nums[0];               // Reset slow to start
    while (slow != fast)
    {
        slow = nums[slow];
        fast = nums[fast];
    }

    return slow;                    // The duplicate value
}
// LeetCode 287 — Find the Duplicate Number
// O(n) time, O(1) space — no array modification!

static void Main()
{
    Console.WriteLine(FindDuplicate(new[]{1,3,4,2,2}));  // 2
    Console.WriteLine(FindDuplicate(new[]{3,1,3,4,2}));  // 3
}
Python 
def find_duplicate(nums: list) -> int:
    slow = fast = nums[0]
    while True:
        slow = nums[slow]
        fast = nums[nums[fast]]
        if slow == fast: break
    slow = nums[0]               # Phase 2: reset slow
    while slow != fast:
        slow = nums[slow]; fast = nums[fast]
    return slow
🌍 Real-World Context: This technique appears in memory-constrained embedded systems where you need to detect pointer corruption or circular references without allocating tracking structures. The "array as implicit graph" mental model also appears in problems like "Next Permutation" and "Rearrange Array" problems.

7.1 — When to Reach for Floyd's

  • Constraint says O(1) extra space — Floyd's is almost always the answer
  • Detect a loop in a linked list, sequence, or pointer chain
  • Find where a loop starts — use Phase 2 after Phase 1
  • Array values in [1..n] with n+1 elements — treat as implicit linked list
  • Repeated sequence detection — Happy Number, function iteration cycles
📝 What Floyd's Cannot Do: It finds a duplicate but not all duplicates (for that, use a counting array or sort). It also only works when the structure truly has a rho (ρ) shape — a tail leading into a cycle. Random graphs with multiple cycles need different approaches.

3
Happy Number (LC 202)
Easy

Write an algorithm to determine if a number n is happy. A happy number eventually reaches 1 when replaced repeatedly by the sum of the square of each digit.

💡 Hint: Model as a sequence; if it loops without hitting 1, it's not happy. Floyd detects the loop.
C# 
public static bool IsHappy(int n)
{
    static int Next(int x)
    {
        int s = 0;
        while (x > 0) { int d = x%10; s += d*d; x /= 10; }
        return s;
    }
    int slow = n, fast = Next(n);
    while (fast != 1 && slow != fast)
    {
        slow = Next(slow);
        fast = Next(Next(fast));
    }
    return fast == 1;
}
// Time: O(log n) | Space: O(1)
Python 
def is_happy(n: int) -> bool:
    def nxt(x):
        s = 0
        while x: x, d = divmod(x, 10); s += d * d
        return s
    slow = fast = n
    while True:
        slow = nxt(slow); fast = nxt(nxt(fast))
        if slow == fast: return slow == 1

Find the Duplicate Number (LC 287)
Medium

Given an array of n+1 integers where each value is in [1, n], find the one duplicate. Must not modify the array. Must use O(1) extra space.

💡 Hint: Treat indices as a linked list where index i → index nums[i]. The duplicate creates a cycle.
C# 
public static int FindDuplicate(int[] nums)
{
    int slow = nums[0], fast = nums[0];
    do { slow = nums[slow]; fast = nums[nums[fast]]; } while (slow != fast);
    slow = nums[0];
    while (slow != fast) { slow = nums[slow]; fast = nums[fast]; }
    return slow;
}
// Time: O(n) | Space: O(1) | Does NOT modify the array
Python 
def find_duplicate(nums) -> int:
    s = f = nums[0]
    while True:
        s = nums[s]; f = nums[nums[f]]
        if s == f: break
    s = nums[0]
    while s != f: s = nums[s]; f = nums[f]
    return s

Find the Middle of a Linked List (LC 876)
Medium

Return the middle node of a linked list. If two middle nodes exist, return the second one. Bonus: do it in a single pass without counting length first.

💡 Hint: Same two-pointer mechanic — when fast reaches the end, slow is at the middle.
C# 
public static ListNode MiddleNode(ListNode head)
{
    ListNode slow = head, fast = head;
    while (fast != null && fast.Next != null)
    {
        slow = slow.Next!;
        fast = fast.Next.Next!;
    }
    return slow;  // When fast is at end, slow is at middle
}
// 1→2→3→4→5 → returns node(3)
// 1→2→3→4   → returns node(3) (second middle)
// Time: O(n) | Space: O(1)
Python 
def middle_node(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next; fast = fast.next.next
    return slow
