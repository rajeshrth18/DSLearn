Section 1 — What is a Priority Queue?

A Priority Queue is a queue where each element has a priority. When you dequeue, you always get the element with the highest priority — not the element that has been waiting the longest. Priority can mean "smallest number", "largest number", or any custom ordering.

🌍 Real-World Examples:
  • Emergency room: A patient with a heart attack jumps the queue over a patient with a minor cut — severity is the priority, not arrival time.
  • CPU scheduling: The OS picks the highest-priority process to run next. A system process beats a background update.
  • Dijkstra's shortest path: Always explore the unvisited city with the smallest known distance first — guaranteed to find optimal paths.
  • Huffman compression: Always merge the two least-frequent characters first to build the optimal encoding tree.
📌 Priority Queue vs Sorted Array:
  • Sorted array: Insert is O(n), Extract-Min is O(1). Bad for many insertions.
  • Priority Queue (heap): Insert is O(log n), Extract-Min is O(log n). Excellent balance.
  • Unsorted array: Insert O(1), Extract-Min O(n). Bad for many extractions.
Section 2 — The Binary Heap

The data structure that efficiently implements a priority queue is the Binary Heap. A binary heap is a complete binary tree stored in an array, satisfying the heap property: in a min-heap, every parent is smaller than or equal to its children.

MIN-HEAP as a TREE STORED as an ARRAY 1 Index: 0 1 2 3 4 5 6 / \ Value: [1, 3, 2, 7, 4, 5, 6] 3 2 / \ / \ Parent of i = (i-1) / 2 (integer division) 7 4 5 6 Left child = 2*i + 1 Right child = 2*i + 2 RULE: parent ≤ both children at ALL levels The MINIMUM is always at the ROOT (index 0) COMPLETE TREE: filled level by level, left to right. This guarantees height = O(log n), making all operations O(log n).
📌 Array Index Math — The Key Insight: Because heaps are complete trees, we can store them in a flat array without pointers. The parent-child relationships are computed with simple arithmetic. This makes heaps cache-friendly and memory-efficient.
  • Node at index i
  • Parent: (i - 1) / 2
  • Left child: 2 * i + 1
  • Right child: 2 * i + 2
Section 3 — Heap Operations in Detail

Insert (Enqueue) — O(log n)

Add the new element at the end of the array (maintaining the "complete tree" shape). Then "bubble up" (sift up): compare with parent, swap if smaller, repeat until the heap property is restored.

INSERT 1 into heap [2, 3, 5, 7, 4, 6]: Step 1 — Add at end: [2, 3, 5, 7, 4, 6, 1] 2 / \ 3 5 / \ / \ 7 4 6 1 ← new node at index 6 Step 2 — Bubble Up: 1 vs parent at (6-1)/2=2 → parent is 5 → 1 < 5, SWAP [2, 3, 1, 7, 4, 6, 5] 2 / \ 3 1 / \ / \ 7 4 6 5 Step 3 — Continue: 1 vs parent at (2-1)/2=0 → parent is 2 → 1 < 2, SWAP [1, 3, 2, 7, 4, 6, 5] 1 ← NEW ROOT (minimum) / \ 3 2 / \ / \ 7 4 6 5 Heap property restored! ✓

ExtractMin (Dequeue) — O(log n)

Remove the root (minimum). Replace it with the last element (maintains complete tree shape). Then "bubble down" (sift down): compare with children, swap with the smaller child if needed, repeat until heap property is restored.

EXTRACT MIN from [1, 3, 2, 7, 4, 6, 5]: Step 1 — Remove root (1). Move last element (5) to root: [5, 3, 2, 7, 4, 6] 5 / \ 3 2 / \ / 7 4 6 Step 2 — Bubble Down: 5 vs children 3 and 2. Smaller child=2. 5>2, SWAP: [2, 3, 5, 7, 4, 6] 2 / \ 3 5 / \ / 7 4 6 Step 3 — Continue: 5 vs children 6 (only one). 5<6, STOP. Heap property restored! ✓ Returns: 1 (the extracted minimum)
Section 4 — Min-Heap Implementation from Scratch
C# 
public class MinHeap<T> where T : IComparable<T>
{
    private List<T> heap = new();

    public int Count => heap.Count;
    public bool IsEmpty => heap.Count == 0;
    public T Peek() => IsEmpty ? throw new InvalidOperationException() : heap[0];

    // Index helpers
    private static int Parent(int i) => (i - 1) / 2;
    private static int Left  (int i) => 2 * i + 1;
    private static int Right (int i) => 2 * i + 2;

    private void Swap(int a, int b)
        => (heap[a], heap[b]) = (heap[b], heap[a]);

    // Insert — O(log n)
    public void Insert(T value)
    {
        heap.Add(value);
        BubbleUp(heap.Count - 1);
    }

    private void BubbleUp(int i)
    {
        while (i > 0)
        {
            int p = Parent(i);
            if (heap[i].CompareTo(heap[p]) < 0)  // child < parent
            {
                Swap(i, p);
                i = p;
            }
            else break;
        }
    }

    // ExtractMin — O(log n)
    public T ExtractMin()
    {
        if (IsEmpty) throw new InvalidOperationException("Heap is empty");
        T min = heap[0];
        int last = heap.Count - 1;
        heap[0] = heap[last];  // Move last to root
        heap.RemoveAt(last);
        if (!IsEmpty) BubbleDown(0);
        return min;
    }

    private void BubbleDown(int i)
    {
        int n = heap.Count;
        while (true)
        {
            int smallest = i;
            int l = Left(i), r = Right(i);
            if (l < n && heap[l].CompareTo(heap[smallest]) < 0) smallest = l;
            if (r < n && heap[r].CompareTo(heap[smallest]) < 0) smallest = r;
            if (smallest == i) break;
            Swap(i, smallest);
            i = smallest;
        }
    }

    // Build heap from array — O(n) using bottom-up heapify
    public static MinHeap<T> FromArray(T[] arr)
    {
        var h = new MinHeap<T>();
        h.heap.AddRange(arr);
        // Start from last non-leaf node, sift down each
        for (int i = (arr.Length / 2) - 1; i >= 0; i--)
            h.BubbleDown(i);
        return h;
    }
}
Python 
import heapq

class MinHeap:
    def __init__(self): self._h = []

    def insert(self, val):  heapq.heappush(self._h, val)
    def extract_min(self): return heapq.heappop(self._h)
    def peek_min(self):    return self._h[0]
    def count(self):       return len(self._h)

    @staticmethod
    def from_array(arr: list) -> 'MinHeap':
        h = MinHeap()
        h._h = arr[:]
        heapq.heapify(h._h)   # O(n) heapify
        return h
💡 Build Heap in O(n): The FromArray method builds a heap in O(n) — not O(n log n). The trick is starting from the last non-leaf and sifting down. Most elements are near the bottom of the tree and travel only a short distance, giving O(n) total work.
Section 5 — Max-Heap & The Inversion Trick

A max-heap always puts the largest element at the root. You can build one in two ways:

Option 1 — Negate the values

If using a min-heap, insert -value. When you extract, negate the result back. This is the classic interview trick for "K largest elements using a min-heap".

Option 2 — Reverse comparator

C# 
// Using C# PriorityQueue with inverted priority for max-heap behavior
var maxHeap = new PriorityQueue<int, int>(Comparer<int>.Create((a, b) => b - a));
maxHeap.Enqueue(5, 5);
maxHeap.Enqueue(1, 1);
maxHeap.Enqueue(8, 8);
maxHeap.Enqueue(3, 3);
Console.WriteLine(maxHeap.Dequeue());  // 8 (largest first)
Console.WriteLine(maxHeap.Dequeue());  // 5
Python 
import heapq

# Python heapq is min-heap only; negate values for max-heap
max_heap = []
for v in [3, 1, 4, 1, 5, 9]:
    heapq.heappush(max_heap, -v)    # push negated

max_val = -heapq.heappop(max_heap)  # negate back on extract → 9
peek    = -max_heap[0]               # peek at max
print(max_val, peek)
Section 6 — C# PriorityQueue<TElement, TPriority>

Starting with .NET 6, C# provides a built-in PriorityQueue<TElement, TPriority>. It is a min-heap by default — the element with the smallest priority value is dequeued first. The element and its priority are separate — you can store any object and rank it by a separate priority value.

C# 
// Basic usage
var pq = new PriorityQueue<string, int>();  // (task name, priority)

pq.Enqueue("Low priority cleanup",   10);
pq.Enqueue("Medium: update cache",    5);
pq.Enqueue("HIGH: server is down!",   1);
pq.Enqueue("Medium: send email",      5);

// Dequeue — always gets lowest priority NUMBER (highest urgency)
Console.WriteLine(pq.Dequeue());  // "HIGH: server is down!" (priority=1)
Console.WriteLine(pq.Dequeue());  // one of the priority=5 items

// Peek — view without removing
Console.WriteLine(pq.Peek());     // front item (doesn't remove)

// TryDequeue / TryPeek — safe versions
while (pq.TryDequeue(out string? task, out int pri))
    Console.WriteLine($"[P{pri}] {task}");

// ── Enqueue with custom object ─────────────────────────────
record Patient(string Name, int Severity);

var er = new PriorityQueue<Patient, int>();
er.Enqueue(new Patient("Alice", 3), 3);  // moderate
er.Enqueue(new Patient("Bob",   1), 1);  // critical
er.Enqueue(new Patient("Carol", 5), 5);  // mild

while (er.TryDequeue(out var p, out _))
    Console.WriteLine($"Treating: {p.Name} (severity {p.Severity})");
// Treating: Bob (severity 1)
// Treating: Alice (severity 3)
// Treating: Carol (severity 5)
Python 
import heapq

# (priority, element) tuples — lower number = higher priority
pq = []
heapq.heappush(pq, (3, "Alice"))    # priority 3 (MEDIUM)
heapq.heappush(pq, (1, "Bob"))      # priority 1 (CRITICAL)
heapq.heappush(pq, (2, "Carol"))    # priority 2 (HIGH)

priority, patient = heapq.heappop(pq)   # Bob (1, CRITICAL)
print(f"Serving: {patient} (priority {priority})")
📌 .NET 6+ Only: PriorityQueue<TElement, TPriority> was added in .NET 6. If you're on an older framework, use a third-party library or implement your own heap. For interview code that uses LeetCode's C# runner, it supports this class.
Section 7 — Application: K Largest / K Smallest Elements

Finding the K largest or K smallest elements from an array of N elements is one of the most common interview problems. A heap gives us O(N log K) — far better than O(N log N) sorting when K is small.

K Smallest Elements — Use a Max-Heap of size K

Maintain a max-heap of size K. For each new element, if it's smaller than the heap's max, replace the max. At the end, the heap contains the K smallest elements.

C# 
// Find K smallest elements — O(N log K)
public static int[] KSmallest(int[] nums, int k)
{
    // Max-heap using inverted priority (negative values)
    var maxHeap = new PriorityQueue<int, int>(
        Comparer<int>.Create((a, b) => b.CompareTo(a)));  // reversed

    foreach (int n in nums)
    {
        maxHeap.Enqueue(n, n);
        if (maxHeap.Count > k)
            maxHeap.Dequeue();  // Evict the current maximum
    }

    return [.. maxHeap.UnorderedItems().Select(x => x.Element)];
}

// Find K largest elements — O(N log K)
public static int[] KLargest(int[] nums, int k)
{
    // Min-heap of size K; evict minimum when over K
    var minHeap = new PriorityQueue<int, int>();

    foreach (int n in nums)
    {
        minHeap.Enqueue(n, n);
        if (minHeap.Count > k)
            minHeap.Dequeue();  // Evict current minimum
    }

    return [.. minHeap.UnorderedItems().Select(x => x.Element)];
}

// Example
// nums = [3,1,4,1,5,9,2,6,5,3], k=3
// K Smallest: [1,1,2]   K Largest: [6,9,5]
Python 
import heapq

def k_smallest(nums: list, k: int) -> list:
    return heapq.nsmallest(k, nums)    # built-in — uses heap internally

def k_largest(nums: list, k: int) -> list:
    return heapq.nlargest(k, nums)

# Manual approach with fixed-size heap:
def k_largest_manual(nums: list, k: int) -> list:
    min_heap = []
    for n in nums:
        heapq.heappush(min_heap, n)
        if len(min_heap) > k: heapq.heappop(min_heap)  # evict smallest
    return min_heap
WHY MAX-HEAP FOR K SMALLEST? Array: [3, 1, 4, 1, 5, 9, 2, 6], k=3 Strategy: keep a max-heap of size 3 (the 3 smallest seen so far) Process 3: heap=[3] size=1 Process 1: heap=[3,1] size=2 Process 4: heap=[4,3,1] size=3 Process 1: heap=[4,3,1,1]→evict max(4)=[3,1,1] Process 5: heap=[5,3,1,1]→evict max(5)=[3,1,1] ← 5 not in K smallest Process 9: heap=[9,3,1,1]→evict max(9)=[3,1,1] ← 9 not in K smallest Process 2: heap=[3,2,1,1]→evict max(3)=[2,1,1] ← 3 evicted, 2 is smaller Process 6: heap=[6,2,1,1]→evict max(6)=[2,1,1] RESULT: [2,1,1] = 3 smallest ✓
Section 8 — Application: Merge K Sorted Lists

Given K sorted linked lists, merge them into one sorted list. The heap approach processes the "current minimum" across all K lists efficiently.

C# 
public class ListNode
{
    public int Val; public ListNode? Next;
    public ListNode(int v, ListNode? n = null) { Val = v; Next = n; }
}

public static ListNode? MergeKLists(ListNode?[] lists)
{
    // Min-heap: stores (value, list-index) pairs, ordered by value
    var pq = new PriorityQueue<ListNode, int>();

    // Seed with the head of each list
    foreach (var node in lists)
        if (node != null) pq.Enqueue(node, node.Val);

    ListNode dummy = new(0);
    ListNode curr = dummy;

    while (pq.Count > 0)
    {
        var min = pq.Dequeue();       // Get node with smallest value
        curr.Next = min;
        curr = curr.Next;
        if (min.Next != null)
            pq.Enqueue(min.Next, min.Next.Val);  // Add next node from same list
    }
    return dummy.Next;
}
// Time: O(N log K) — N total nodes, K lists in heap at most
// Space: O(K) — heap holds at most one node per list
Python 
import heapq

class ListNode:
    def __init__(self, val=0): self.val = val; self.next = None
    def __lt__(self, other): return self.val < other.val

def merge_k_lists(lists: list) -> ListNode:
    heap = []
    for node in lists:
        if node: heapq.heappush(heap, node)
    dummy = cur = ListNode(0)
    while heap:
        node = heapq.heappop(heap)
        cur.next = node; cur = cur.next
        if node.next: heapq.heappush(heap, node.next)
    return dummy.next
Section 9 — Application: Task Scheduler

Given tasks with frequencies, schedule them such that the same task must wait at least n cooling intervals between executions. Minimize total time (including idle slots).

C# 
// LeetCode 621 — Task Scheduler
// tasks=['A','A','A','B','B','B'], n=2
// Optimal: A B _ A B _ A B  = 8 intervals

public static int LeastInterval(char[] tasks, int n)
{
    int[] freq = new int[26];
    foreach (char t in tasks) freq[t - 'A']++;

    // Max-heap by frequency
    var maxHeap = new PriorityQueue<int, int>(
        Comparer<int>.Create((a, b) => b - a));

    foreach (int f in freq)
        if (f > 0) maxHeap.Enqueue(f, f);

    // Queue: (remaining frequency, time when it can run again)
    var cooldown = new Queue<(int count, int availAt)>();
    int time = 0;

    while (maxHeap.Count > 0 || cooldown.Count > 0)
    {
        time++;

        // Release tasks that finished cooling
        if (cooldown.Count > 0 && cooldown.Peek().availAt == time)
        {
            var (cnt, _) = cooldown.Dequeue();
            maxHeap.Enqueue(cnt, cnt);
        }

        if (maxHeap.Count > 0)
        {
            int remaining = maxHeap.Dequeue() - 1;
            if (remaining > 0)
                cooldown.Enqueue((remaining, time + n + 1));
        }
        // else: IDLE slot (time still increments)
    }
    return time;
}
// tasks=['A','A','A','B','B','B'], n=2 → 8
Python 
import heapq
from collections import Counter, deque

def least_interval(tasks: list, n: int) -> int:
    counts   = Counter(tasks)
    max_heap = [-c for c in counts.values()]
    heapq.heapify(max_heap)
    cooldown = deque()   # (count_remaining, available_at_time)
    time = 0
    while max_heap or cooldown:
        time += 1
        if max_heap:
            cnt = heapq.heappop(max_heap) + 1  # decrement (negated)
            if cnt: cooldown.append((cnt, time + n))
        if cooldown and cooldown[0][1] == time:
            heapq.heappush(max_heap, cooldown.popleft()[0])
    return time
Section 10 — Heap Sort

Heap Sort uses a max-heap to sort an array in-place in O(n log n) time and O(1) extra space — better space than merge sort's O(n).

HEAP SORT — Two Phases Phase 1 — BUILD MAX-HEAP from array in O(n): Input: [4, 10, 3, 5, 1] After build: [10, 5, 3, 4, 1] (max at root) Phase 2 — EXTRACT MAX n times, each O(log n): Swap root (max) with last element, reduce heap size by 1, sift down. Step 1: Swap arr[0]=10 with arr[4]=1 → [1,5,3,4,|10] → sift down → [5,4,3,1,|10] Step 2: Swap arr[0]=5 with arr[3]=1 → [1,4,3,|5,10] → sift down → [4,1,3,|5,10] Step 3: Swap arr[0]=4 with arr[2]=3 → [3,1,|4,5,10] → sift down → [3,1,|4,5,10] Step 4: Swap arr[0]=3 with arr[1]=1 → [1,|3,4,5,10] → sift down → trivial Result: [1, 3, 4, 5, 10] SORTED ✓
C# 
public static void HeapSort(int[] arr)
{
    int n = arr.Length;

    // Phase 1: Build max-heap in O(n)
    for (int i = n / 2 - 1; i >= 0; i--)
        SiftDown(arr, n, i);

    // Phase 2: Extract max one by one
    for (int i = n - 1; i > 0; i--)
    {
        // Swap current root (max) to sorted position
        (arr[0], arr[i]) = (arr[i], arr[0]);
        // Restore heap property on the reduced heap
        SiftDown(arr, i, 0);
    }
}

private static void SiftDown(int[] arr, int heapSize, int i)
{
    while (true)
    {
        int largest = i;
        int l = 2 * i + 1, r = 2 * i + 2;
        if (l < heapSize && arr[l] > arr[largest]) largest = l;
        if (r < heapSize && arr[r] > arr[largest]) largest = r;
        if (largest == i) break;
        (arr[i], arr[largest]) = (arr[largest], arr[i]);
        i = largest;
    }
}

int[] arr = [4, 10, 3, 5, 1];
HeapSort(arr);
Console.WriteLine(string.Join(", ", arr));  // 1, 3, 4, 5, 10
Python 
def heap_sort(arr: list) -> list:
    def sift_down(a, root, end):
        while True:
            largest = root
            l, r    = 2 * root + 1, 2 * root + 2
            if l < end and a[l] > a[largest]: largest = l
            if r < end and a[r] > a[largest]: largest = r
            if largest == root: break
            a[root], a[largest] = a[largest], a[root]
            root = largest

    n = len(arr)
    for i in range(n // 2 - 1, -1, -1):  # build max-heap O(n)
        sift_down(arr, i, n)
    for i in range(n - 1, 0, -1):           # extract max n times
        arr[0], arr[i] = arr[i], arr[0]
        sift_down(arr, 0, i)
    return arr

🌍 When to Choose a Priority Queue:
  • You need to repeatedly access the minimum or maximum element
  • Elements have varying priorities and you process the most important next
  • Problems with the pattern: "K largest", "K smallest", "K closest", "K most frequent"
  • Graph algorithms: Dijkstra's shortest path, Prim's MST
  • Merge multiple sorted data sources
