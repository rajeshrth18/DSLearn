Section 1 — Doubly Linked List: The Concept

In a singly linked list, each node only knows its next neighbor. This creates one critical weakness: to delete a node or traverse backwards, you need O(n) traversal from the head.

A Doubly Linked List fixes this by adding a second pointer to each node: Prev — a reference to the previous node. Now every node knows its two neighbors, enabling O(1) deletion of any node you already have a reference to and bidirectional traversal.

SINGLY vs DOUBLY — Structural Difference SINGLY — Each node has ONE link (forward only): head --? [10|?] --? [20|?] --? [30|null] DOUBLY — Each node has TWO links (forward AND backward): head --? [null|10|?] ? [?|20|?] ? [?|30|null] ?-- tail prev data next prev data next prev data next Each node: +--------------------+ ¦ Prev ¦ Data ¦ Next ¦ +--------------------+ ? ¦ ¦ ? previous node next node
?? The Key Advantage — O(1) Delete: In a singly list, deleting a node requires finding its previous node (O(n)). In a doubly list, the node already has a Prev reference — just re-link its neighbors and you're done in O(1). This is why C#'s LinkedList<T> is a doubly linked list.
OperationSinglyDoubly
Insert at head/tailO(1)O(1)
Insert at middle (by ref)O(n) — find prevO(1) — node has Prev
Delete given a node referenceO(n) — find prevO(1)
Delete tailO(n)O(1)
Traverse forwardO(n)O(n)
Traverse backwardNot possibleO(n)
Memory per nodeData + 1 pointerData + 2 pointers

public class DNode<T>

Section 3 — Insert & Delete in a Doubly List

3.1 — Insert at Head — O(1)

Four pointer updates are needed (vs two in singly). Always update the Prev of the old head first.

INSERT AT HEAD — Adding (5) to: 10 ? 20 ? 30 ? null BEFORE: head --? [null|10|?] ? [?|20|?] ? [?|30|null] Step 1: Create newNode(5). newNode.Prev=null, newNode.Next=null Step 2: newNode.Next = head ? newNode --? [null|10|?] Step 3: head.Prev = newNode ? [null|10|?].Prev = newNode ? BIDIRECTIONAL Step 4: head = newNode AFTER: head --? [null|5|?] ? [?|10|?] ? [?|20|?] ? [?|30|null]

3.2 — Delete a Node (given reference) — O(1)

The crown jewel of doubly linked lists. If you already have a reference to the node you want to remove, you can delete it in O(1) without any traversal. Just rewire its two neighbors.

DELETE NODE with value 20 — List: 10 ? 20 ? 30 ? 40 We have a reference to node(20). No traversal needed! node(20).Prev = node(10) node(20).Next = node(30) Step 1: node(20).Prev.Next = node(20).Next ? node(10).Next = node(30) Step 2: node(20).Next.Prev = node(20).Prev ? node(30).Prev = node(10) Step 3: node(20).Prev = null (optional cleanup) node(20).Next = null (optional cleanup) AFTER: 10 ? 30 ? 40 (node(20) garbage collected) Edge cases: If deleting HEAD: no node(20).Prev ? skip step 1, update head If deleting TAIL: no node(20).Next ? skip step 2, update tail
C# 
// Delete a specific node reference — O(1)!
public T RemoveNode(DNode<T> node)
{
    T value = node.Data;

    if (node.Prev != null)
        node.Prev.Next = node.Next;  // wire prev ? next
    else
        head = node.Next;            // deleting head

    if (node.Next != null)
        node.Next.Prev = node.Prev;  // wire next ? prev
    else
        tail = node.Prev;            // deleting tail

    node.Prev = node.Next = null;   // cleanup (optional, helps GC)
    Count--;
    return value;
}
Python 
def remove_node(self, node: DNode):
    if node.prev: node.prev.next = node.next
    else:         self.head = node.next   # removing head
    if node.next: node.next.prev = node.prev
    else:         self.tail = node.prev   # removing tail
    self.count -= 1
Section 4 — Full DoublyLinkedList<T> Implementation
C# 
public class DoublyLinkedList<T>
{
    private DNode<T>? head;
    private DNode<T>? tail;
    public int Count { get; private set; }
    public bool IsEmpty => Count == 0;

    // -- Add Operations ----------------------------------
    public void AddFirst(T value)
    {
        DNode<T> n = new(value);
        if (IsEmpty) { head = tail = n; }
        else
        {
            n.Next = head;
            head!.Prev = n;   // ? bidirectional link
            head = n;
        }
        Count++;
    }

    public void AddLast(T value)
    {
        DNode<T> n = new(value);
        if (IsEmpty) { head = tail = n; }
        else
        {
            n.Prev = tail;
            tail!.Next = n;   // ? bidirectional link
            tail = n;
        }
        Count++;
    }

    public void AddBefore(DNode<T> node, T value)
    {
        if (node == head) { AddFirst(value); return; }
        DNode<T> n = new(value)
        {
            Next = node,
            Prev = node.Prev
        };
        node.Prev!.Next = n;
        node.Prev = n;
        Count++;
    }

    public void AddAfter(DNode<T> node, T value)
    {
        if (node == tail) { AddLast(value); return; }
        DNode<T> n = new(value)
        {
            Prev = node,
            Next = node.Next
        };
        node.Next!.Prev = n;
        node.Next = n;
        Count++;
    }

    // -- Remove Operations -------------------------------
    public T RemoveFirst()
    {
        if (IsEmpty) throw new InvalidOperationException("List is empty");
        T val = head!.Data;
        head = head.Next;
        if (head != null) head.Prev = null;  // clear backward link
        else tail = null;
        Count--;
        return val;
    }

    public T RemoveLast()
    {
        if (IsEmpty) throw new InvalidOperationException("List is empty");
        T val = tail!.Data;
        tail = tail.Prev;
        if (tail != null) tail.Next = null;  // ? O(1)! No traversal needed
        else head = null;
        Count--;
        return val;
    }

    public T RemoveNode(DNode<T> node)  // O(1) given reference
    {
        T val = node.Data;
        if (node.Prev != null) node.Prev.Next = node.Next;
        else head = node.Next;
        if (node.Next != null) node.Next.Prev = node.Prev;
        else tail = node.Prev;
        node.Prev = node.Next = null;
        Count--;
        return val;
    }

    public bool Remove(T value)
    {
        DNode<T>? node = Find(value);
        if (node == null) return false;
        RemoveNode(node);
        return true;
    }

    // -- Search ------------------------------------------
    public DNode<T>? Find(T value)
    {
        DNode<T>? c = head;
        while (c != null)
        {
            if (c.Data!.Equals(value)) return c;
            c = c.Next;
        }
        return null;
    }

    // -- Traversal ---------------------------------------
    public void PrintForward()
    {
        DNode<T>? c = head;
        while (c != null) { Console.Write(c.Data + " ? "); c = c.Next; }
        Console.WriteLine("null");
    }

    public void PrintBackward()  // ? Unique to doubly!
    {
        DNode<T>? c = tail;
        while (c != null) { Console.Write(c.Data + " ? "); c = c.Prev; }
        Console.WriteLine("null");
    }
}

// -- Usage ------------------------------------------------
var dl = new DoublyLinkedList<int>();
dl.AddLast(10); dl.AddLast(20); dl.AddLast(30);
dl.AddFirst(5);
dl.PrintForward();   // 5 ? 10 ? 20 ? 30 ? null
dl.PrintBackward();  // 30 ? 20 ? 10 ? 5 ? null

DNode<int>? node20 = dl.Find(20);
dl.RemoveNode(node20!);  // O(1) — no traversal!
dl.PrintForward();   // 5 ? 10 ? 30 ? null
Python 
class DoublyLinkedList:
    def __init__(self):
        self.head = self.tail = None; self.count = 0

    def add_first(self, data):
        n = DNode(data)
        if not self.head: self.head = self.tail = n
        else: n.next = self.head; self.head.prev = n; self.head = n
        self.count += 1

    def add_last(self, data):
        n = DNode(data)
        if not self.head: self.head = self.tail = n
        else: self.tail.next = n; n.prev = self.tail; self.tail = n
        self.count += 1

    def remove_first(self):
        if not self.head: return
        self.remove_node(self.head)

    def remove_last(self):
        if not self.tail: return
        self.remove_node(self.tail)

    def find(self, data):
        c = self.head
        while c:
            if c.data == data: return c
            c = c.next
        return None

    def print_forward(self):
        c = self.head
        while c: print(c.data, end=' '); c = c.next; print()

    def print_backward(self):
        c = self.tail
        while c: print(c.data, end=' '); c = c.prev; print()
Section 5 — C#'s Built-in LinkedList<T>

C# provides a built-in LinkedList<T> in System.Collections.Generic. It is a doubly linked list and is the data structure you should use in production code rather than rolling your own. Understanding what we built in Section 4 is exactly what lets you use this class with confidence.

C# 
using System.Collections.Generic;

// LinkedList<T> — C#'s built-in doubly linked list
LinkedList<string> songs = new LinkedList<string>();

// Adding elements — returns a LinkedListNode<T>
LinkedListNode<string> node1 = songs.AddLast("Bohemian Rhapsody");
LinkedListNode<string> node2 = songs.AddLast("Hotel California");
LinkedListNode<string> node3 = songs.AddLast("Stairway to Heaven");
songs.AddFirst("Yesterday");
songs.AddBefore(node2, "Let It Be");    // Insert before "Hotel California"
songs.AddAfter(node3, "Purple Haze");   // Insert after "Stairway..."

// Access
Console.WriteLine(songs.First!.Value);  // "Yesterday" — O(1)
Console.WriteLine(songs.Last!.Value);   // "Purple Haze" — O(1)
Console.WriteLine(songs.Count);         // 6

// Remove — O(1) given a node, O(n) by value
songs.Remove(node2);              // O(1) — remove "Hotel California"
songs.Remove("Let It Be");         // O(n) — find then remove
songs.RemoveFirst();               // O(1)
songs.RemoveLast();                // O(1)

// Traversal — forward and backward
LinkedListNode<string>? current = songs.First;
while (current != null)
{
    Console.WriteLine(current.Value);
    current = current.Next;    // Forward
}

current = songs.Last;
while (current != null)
{
    Console.WriteLine(current.Value);
    current = current.Previous;  // Backward!
}

// Contains / Find
bool has = songs.Contains("Bohemian Rhapsody");       // O(n)
LinkedListNode<string>? found = songs.Find("Bohemian Rhapsody"); // O(n)

from collections import deque

d = deque()

?? Real-World: LRU Cache (Most Common Interview Pattern)
An LRU (Least Recently Used) cache keeps the N most recently accessed items. When full, it evicts the least recently used. The ideal structure: LinkedList<T> + Dictionary.
  • Move accessed item to front ? O(1) (LinkedListNode reference in Dictionary)
  • Evict least recently used ? O(1) (RemoveLast)
  • Look up by key ? O(1) (Dictionary)
This is exactly how browser history, CPU caches, and database buffer pools work.
C# 
// Classic LRU Cache using LinkedList<T> + Dictionary
public class LRUCache<TKey, TValue>
{
    private readonly int capacity;
    private readonly Dictionary<TKey, LinkedListNode<(TKey, TValue)>> map;
    private readonly LinkedList<(TKey, TValue)> list;

    public LRUCache(int capacity)
    {
        this.capacity = capacity;
        map  = new Dictionary<TKey, LinkedListNode<(TKey, TValue)>>();
        list = new LinkedList<(TKey, TValue)>();
    }

    public TValue? Get(TKey key)
    {
        if (!map.TryGetValue(key, out var node)) return default;
        list.Remove(node);        // O(1) — have the node ref
        list.AddFirst(node);      // Move to front (most recent)
        return node.Value.Item2;
    }

    public void Put(TKey key, TValue value)
    {
        if (map.TryGetValue(key, out var existing))
        {
            list.Remove(existing);
            map.Remove(key);
        }
        else if (list.Count == capacity)
        {
            var lru = list.Last!;
            map.Remove(lru.Value.Item1);  // Evict LRU
            list.RemoveLast();
        }
        var node = list.AddFirst((key, value));
        map[key] = node;
    }
}

// Usage
var cache = new LRUCache<int, string>(3);
cache.Put(1, "Alice");
cache.Put(2, "Bob");
cache.Put(3, "Carol");
cache.Get(1);               // Access Alice ? moves to front
cache.Put(4, "Dave");       // Evicts Bob (LRU), adds Dave
Console.WriteLine(cache.Get(2));  // null — Bob was evicted
Python 
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.cap   = capacity
        self.cache = OrderedDict()   # preserves insertion order

    def get(self, key):
        if key not in self.cache: return -1
        self.cache.move_to_end(key)   # mark as most-recently used
        return self.cache[key]

    def put(self, key, value):
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.cap:
            self.cache.popitem(last=False)  # evict LRU (oldest)
Section 6 — Circular Linked Lists

A Circular Linked List is a singly or doubly linked list where the last node's Next points back to the first node (and in the doubly variant, the first node's Prev points to the last node), forming a closed ring with no null terminus.

CIRCULAR SINGLY LINKED LIST +------------------------------------------------------+ ¦ ¦ ? ¦ [1|?] --? [2|?] --? [3|?] --? [4|?] --? [5|?] ------+ ? head (or any designated start node) No null — the tail's Next WRAPS BACK to the head. CIRCULAR DOUBLY LINKED LIST +-----------------------------------------+ ? ¦ ?--[1|?] ? [?|2|?] ? [?|3|?] ? [?|4|?] --? + +----------------------------------------------? Head.Prev = Tail, Tail.Next = Head

6.1 — Circular Singly Linked List Implementation

The core difference in implementation: instead of checking for null to detect the end, you check whether you've looped back to the head.

C# 
public class CircularLinkedList<T>
{
    private Node<T>? head;
    private Node<T>? tail;
    public int Count { get; private set; }

    public void AddLast(T value)
    {
        Node<T> n = new(value);
        if (head == null)
        {
            head = tail = n;
            n.Next = n;         // Points to itself!
        }
        else
        {
            n.Next = head;      // New node points back to head
            tail!.Next = n;     // Old tail points to new node
            tail = n;
        }
        Count++;
    }

    public void AddFirst(T value)
    {
        Node<T> n = new(value);
        if (head == null) { head = tail = n; n.Next = n; }
        else
        {
            n.Next = head;
            tail!.Next = n;  // tail must ALWAYS point to head
            head = n;
        }
        Count++;
    }

    public void Print()
    {
        if (head == null) { Console.WriteLine("(empty)"); return; }
        Node<T>? current = head;
        do
        {
            Console.Write(current.Data + " ? ");
            current = current.Next;
        } while (current != head);  // Stop when back at head!
        Console.WriteLine("(HEAD)");
    }
}
Python 
class CircularLinkedList:
    def __init__(self):
        self.head = self.tail = None; self.count = 0

    def add_last(self, data):
        n = Node(data)
        if not self.head:
            self.head = self.tail = n; n.next = n   # points to itself
        else:
            n.next = self.head; self.tail.next = n; self.tail = n
        self.count += 1

    def print_all(self):
        if not self.head: return
        current = self.head
        while True:
            print(current.data, end=' ')
            current = current.next
            if current == self.head: break   # completed one cycle
        print()
?? Infinite Loop Danger: When traversing a circular list, use a do-while loop with current != head as the stopping condition. A regular while (current != null) loop will run FOREVER because there is no null — the list loops back around!
Section 7 — Circular List Use Cases

7.1 — Round-Robin Scheduling

Operating systems give each process a time slice in a cycle. A circular list is perfect — move to next after each slice, wrapping around forever.

C# 
var processes = new CircularLinkedList<string>();
processes.AddLast("Process A");
processes.AddLast("Process B");
processes.AddLast("Process C");

// Simulate 7 time slices (round-robin)
Node<string>? current = processes.Head;
for (int i = 0; i < 7; i++)
{
    Console.WriteLine($"Slice {i+1}: Running {current!.Data}");
    current = current.Next;
}
// Output: A B C A B C A  (cycles endlessly)
Python 
cll = CircularLinkedList()
for name in ["Alice", "Bob", "Carol"]:
    cll.add_last(name)

current = cll.head
for i in range(7):             # 7 turns, round-robin
    print(f"Turn {i+1}: {current.data}")
    current = current.next

7.2 — Josephus Problem

Classic algorithm puzzle: N people stand in a circle. Starting from position 1, every k-th person is eliminated. Who is the last survivor? A circular linked list models this perfectly.

C# 
// Josephus Problem — N=6 people, every 3rd person eliminated
// People: 1 2 3 4 5 6, k=3 ? eliminated: 3,6,4,2,5 ? survivor: 1
static int Josephus(int n, int k)
{
    var circle = new CircularLinkedList<int>();
    for (int i = 1; i <= n; i++) circle.AddLast(i);

    Node<int>? current = circle.Head;
    while (circle.Count > 1)
    {
        // Move k-1 steps forward
        for (int i = 1; i < k; i++) current = current!.Next;
        Console.Write($"Eliminated: {current!.Data}  ");
        var next = current.Next;
        circle.RemoveNode(current);
        current = next;
    }
    Console.WriteLine($"\nSurvivor: {circle.Head!.Data}");
    return circle.Head.Data;
}

Josephus(6, 3);
// Eliminated: 3  6  4  2  5
// Survivor: 1
Python 
def josephus(n: int, k: int) -> int:
    cll = CircularLinkedList()
    for i in range(1, n + 1):
        cll.add_last(i)

    current = cll.head
    prev    = cll.tail   # need prev to remove nodes
    while cll.count > 1:
        for _ in range(k - 1):   # advance k-1 steps
            prev    = current
            current = current.next
        prev.next = current.next   # skip (remove) current
        if current == cll.head: cll.head = current.next
        if current == cll.tail: cll.tail = prev
        current = prev.next
        cll.count -= 1
    return cll.head.data

print(josephus(7, 3))   # 4
