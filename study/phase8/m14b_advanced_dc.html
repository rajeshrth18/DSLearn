Section 1 — Closest Pair of Points

Given n points in 2D, find the pair with minimum Euclidean distance. Brute force is O(n²). D&C achieves O(n log n).

ALGORITHM: 1. Sort points by x-coordinate. 2. Divide: split at median x into L and R halves. 3. Conquer: recursively find δ_L = min_dist(L), δ_R = min_dist(R). 4. δ = min(δ_L, δ_R). 5. COMBINE: check cross-strip pairs. - Collect points within δ of the dividing line (|x - mid_x| < δ). - For each strip point, compare with at most 7 others above it (sorted by y — it's proven at most 7 points can fit in δ×2δ rectangle). - Update δ if closer pair found. 6. Return δ. WHY AT MOST 7 POINTS? The 2δ × δ strip on each side contains points ≥ δ apart (from sub-results). Pack circles of radius δ/2 around each point — at most 8 fit in 2δ×2δ box. So each point compares with at most 7 others. O(n log n) total. RECURRENCE: T(n) = 2T(n/2) + O(n log n) = O(n log² n) (or O(n log n) with pre-sorted y lists passed to each level)
C# 
using System;
public double ClosestPair((double x, double y)[] pts)
{
    var byX = pts.OrderBy(p => p.x).ToArray();
    return Rec(byX, 0, byX.Length - 1);
}
private double Dist((double x, double y) a, (double x, double y) b)
    => Math.Sqrt((a.x-b.x)*(a.x-b.x)+(a.y-b.y)*(a.y-b.y));

private double Rec((double x, double y)[] pts, int lo, int hi)
{
    if (hi - lo < 3)  // brute force for small inputs
    {
        double d = double.MaxValue;
        for (int i = lo; i <= hi; i++)
            for (int j = i+1; j <= hi; j++)
                d = Math.Min(d, Dist(pts[i], pts[j]));
        return d;
    }
    int mid = (lo + hi) / 2;
    double midX = pts[mid].x;
    double delta = Math.Min(Rec(pts, lo, mid), Rec(pts, mid+1, hi));
    // Collect strip points within delta of midline
    var strip = pts[lo..(hi+1)].Where(p => Math.Abs(p.x - midX) < delta)
                               .OrderBy(p => p.y).ToArray();
    for (int i = 0; i < strip.Length; i++)
        for (int j = i+1; j < strip.Length && strip[j].y - strip[i].y < delta; j++)
            delta = Math.Min(delta, Dist(strip[i], strip[j]));
    return delta;
}
Python 
import math
def closest_pair(points):
    pts = sorted(points, key=lambda p: p[0])
    def dist(a, b): return math.hypot(a[0]-b[0], a[1]-b[1])
    def rec(sub):
        n = len(sub)
        if n <= 3:
            return min(dist(sub[i],sub[j]) for i in range(n) for j in range(i+1,n))
        mid = n // 2; mid_x = sub[mid][0]
        delta = min(rec(sub[:mid]), rec(sub[mid:]))
        strip = sorted([p for p in sub if abs(p[0]-mid_x) < delta], key=lambda p: p[1])
        for i, p in enumerate(strip):
            for q in strip[i+1:]:
                if q[1] - p[1] >= delta: break
                delta = min(delta, dist(p, q))
        return delta
    return rec(pts)
Section 2 — Master Theorem Deep Dive
MASTER THEOREM: T(n) = aT(n/b) + f(n), a≥1, b>1 INTUITION: The recursion tree has log_b(n) levels. Level 0 (root): f(n) Level 1: a × f(n/b) Level k: a^k × f(n/b^k) Leaves (level log_b n): a^(log_b n) = n^(log_b a) leaves, each O(1). THREE CASES compare work at LEAVES vs work at INTERNAL NODES: Case 1 — LEAVES DOMINATE: f(n) = O(n^(log_b(a) - ε)) for some ε>0 → T(n) = Θ(n^log_b(a)) EXAMPLE: T(n) = 8T(n/2) + O(n²) log_2(8)=3, n²=O(n^(3-1)) → Case 1 → T(n)=Θ(n³) Case 2 — TIED (equal at all levels): f(n) = Θ(n^log_b(a) · log^k n) [k≥0] → T(n) = Θ(n^log_b(a) · log^(k+1) n) EXAMPLE: T(n) = 2T(n/2) + O(n) [merge sort, k=0] log_2(2)=1, n=Θ(n¹) → Case 2, k=0 → T(n)=Θ(n log n) ✓ Case 3 — INTERNAL NODES DOMINATE: f(n) = Ω(n^(log_b(a) + ε)) AND regularity: af(n/b) ≤ cf(n) for c<1 → T(n) = Θ(f(n)) EXAMPLE: T(n) = T(n/2) + O(n²) log_2(1)=0, n²=Ω(n^(0+2)) → Case 3 → T(n)=Θ(n²) ✓ COMMON RECURRENCES TABLE: Algorithm Recurrence Complexity Binary Search T=T(n/2)+1 O(log n) Merge Sort T=2T(n/2)+n O(n log n) Strassen T=7T(n/2)+n² O(n^2.807) Karatsuba T=3T(n/2)+n O(n^1.585) FFT T=2T(n/2)+n log n O(n log² n) → O(n log n) with care Closest Pair T=2T(n/2)+n log n O(n log² n) Matrix Multiplication (std) T=8T(n/2)+n² O(n³)
Section 3 — Strassen Matrix Multiplication

Standard matrix multiplication is O(n³). Strassen reduces 8 recursive multiplications to 7, giving O(n^log₂7) ≈ O(n^2.807).

STANDARD 2×2 BLOCK MULTIPLICATION requires 8 multiplications: [A B] × [E F] = [AE+BG AF+BH] [C D] [G H] [CE+DG CF+DH] STRASSEN'S TRICK — 7 multiplications: M₁ = (A+D)(E+H) M₂ = (C+D)E M₃ = A(F-H) M₄ = D(G-E) M₅ = (A+B)H M₆ = (C-A)(E+F) M₇ = (B-D)(G+H) Result: [0,0] = M₁ + M₄ - M₅ + M₇ [0,1] = M₃ + M₅ [1,0] = M₂ + M₄ [1,1] = M₁ - M₂ + M₃ + M₆ RECURRENCE: T(n) = 7T(n/2) + O(n²) log_2(7) ≈ 2.807 > 2, so n² = O(n^(2.807-0.807)) → Case 1 T(n) = Θ(n^log_2(7)) ≈ Θ(n^2.807) NOTE: Strassen is mainly of theoretical interest — the constant factors are large and cache performance is poor vs standard multiplication for typical n. Current best: Coppersmith-Winograd O(n^2.376), AlphaTensor O(n^2.778) (DeepMind, 2022).
Section 4 — Karatsuba Fast Integer Multiplication

Multiplying two n-digit numbers. Standard: O(n²). Karatsuba: O(n^log₂3) ≈ O(n^1.585).

KEY INSIGHT: Represent n-digit numbers as: x = x₁·10^m + x₀, y = y₁·10^m + y₀ where m = n/2. Naively: x·y = x₁y₁·10^2m + (x₁y₀+x₀y₁)·10^m + x₀y₀ [4 multiplications] KARATSUBA: Compute only 3 multiplications: z₂ = x₁·y₁ z₀ = x₀·y₀ z₁ = (x₁+x₀)·(y₁+y₀) - z₂ - z₀ [= x₁y₀ + x₀y₁] x·y = z₂·10^2m + z₁·10^m + z₀ RECURRENCE: T(n) = 3T(n/2) + O(n) log_2(3) ≈ 1.585 > 1, Case 1 → T(n) = Θ(n^log_2(3)) ≈ Θ(n^1.585)
C# 
// Karatsuba multiplication for large integers (simplified, using long)
public long Karatsuba(long x, long y)
{
    if (x < 10 || y < 10) return x * y;
    int n = Math.Max(x.ToString().Length, y.ToString().Length);
    int m = n / 2;
    long pow = (long)Math.Pow(10, m);
    long x1 = x / pow, x0 = x % pow;
    long y1 = y / pow, y0 = y % pow;
    long z2 = Karatsuba(x1, y1);
    long z0 = Karatsuba(x0, y0);
    long z1 = Karatsuba(x1 + x0, y1 + y0) - z2 - z0;
    return z2 * pow * pow + z1 * pow + z0;
}
Python 
def karatsuba(x, y):
    if x < 10 or y < 10: return x * y
    n = max(len(str(x)), len(str(y)))
    m = n // 2; pow10 = 10 ** m
    x1, x0 = x // pow10, x % pow10
    y1, y0 = y // pow10, y % pow10
    z2 = karatsuba(x1, y1)
    z0 = karatsuba(x0, y0)
    z1 = karatsuba(x1+x0, y1+y0) - z2 - z0
    return z2 * pow10**2 + z1 * pow10 + z0

Section 5 — Q&A Review
Q1: Why does the Closest Pair strip check only ~7 points per strip point?
After recursion, δ is the minimum distance within each half. In the strip region (within δ of the midline), all points have already been proven to be at least δ apart from other points on the same side. The strip rectangle on each side is δ wide and 2δ tall (we only look at points within δ in y). In this 2δ × δ rectangle on each side, points must be δ apart. You can pack at most 8 points in a 2δ × 2δ square where each pair is ≥ δ apart. Subtracting the point itself, each strip point compares with at most 7 others. This bounds the strip check at O(n) per level, giving O(n log n) or O(n log² n) total.
Q2: Apply the Master Theorem to T(n) = 4T(n/2) + n².
a=4, b=2, n^(log_b a) = n^(log_2 4) = n^2. The driving function f(n) = n². Compare f(n) = n² vs n^2 = n²: they are equal, so this is Case 2 (with k=0 in the log extension). T(n) = Θ(n² log n). This is the recurrence for standard matrix multiplication where you do 4 recursive multiplications of n/2 × n/2 matrices and O(n²) work to combine. Note: standard naive matrix mult recurrence is T(n)=8T(n/2)+n² → Case 1 → O(n³). The 4 subproblem version here gives O(n² log n), which is still worse than Strassen's O(n^2.807)? No: n^2.807 > n² log n for large n, so Θ(n² log n) would be BETTER. But you can't do standard matmul with only 4 recursive multiplications.
Q3: How does Strassen achieve 7 multiplications from 8? Why does this help asymptotically?
Strassen defines 7 intermediate products (M1...M7) using only 7 multiplications (plus 18 additions). The key observation is that the four output submatrices can be expressed as linear combinations of these 7 products. This substitutes one multiplication (the 8th) for several additions, which at large n costs O(n²) instead of the recursive O(n^c) for multiplication. Why it helps: in the recurrence T(n)=aT(n/2)+n², reducing a from 8 to 7 changes log_2(a) from 3 to ~2.807. Since leaves dominate (Case 1), T(n)=Θ(n^log_2 a). Every time we reduce the number of recursive calls, the leaf count drops, improving the overall bound.
Q4: Karatsuba uses 3 recursive multiplications vs 4 for naive. What is the "trick"?
The trick is that x₁y₀ + x₀y₁ (the "cross term") can be computed without two separate multiplications. Instead: compute (x₁+x₀)(y₁+y₀) = x₁y₁ + x₁y₀ + x₀y₁ + x₀y₀. Then subtract z₂=x₁y₁ and z₀=x₀y₀ (which we've already computed): z₁ = (x₁+x₀)(y₁+y₀) - z₂ - z₀ = x₁y₀ + x₀y₁. This is one extra multiplication instead of two, plus two extra subtractions. The subtractions are cheap (linear), so the net saving is one recursive multiplication per level. This is the same algebraic trick Strassen uses in 2D: exploit the structure of linear algebra to replace multiplications with additions.
Q5: When does the Master Theorem NOT apply?
The Master Theorem does not apply in these cases: (1) Non-uniform subproblems: T(n) = T(n/3) + T(2n/3) + O(n) — different-size subproblems. (2) Subtractive recurrences: T(n) = T(n-1) + f(n) — e.g., factorial, which is T(n) = T(n-1) + O(1) → O(n). (3) f(n) is not polynomially larger/smaller than n^(log_b a): e.g., T(n) = 2T(n/2) + n/log(n). (4) a < 1: meaningless for recursion. For case (1) use Akra-Bazzi method. For case (2) use telescoping or substitution. For case (3), you may need to solve the recurrence directly by expanding the recursion tree and summing the levels.
