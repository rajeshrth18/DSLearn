Section 1 — The Analogy: Traffic Light Rules

Imagine a city with a strict rule: every road that leads to a destination must have no more than 2 red lights in a row. If you install a new traffic light and create 3 consecutive reds, you must flip some lights to green to restore the rule. Only a small, local adjustment is needed — not a city-wide re-plan.

A Red-Black tree works on the same principle. Every node is colored RED or BLACK. Five strict rules define which color configurations are allowed. When you insert a new node (always RED initially), you may violate a rule locally. But the fix is always local — at most a few recolorings and at most 2 rotations per insert.

The beauty of Red-Black trees over AVL: fewer rotations per insert/delete. While AVL may perform O(log n) rotations up the tree, Red-Black trees need at most 2 rotations per insert and at most 3 per delete. The trade-off: Red-Black trees allow slightly more imbalance, so searches are marginally slower.

Section 2 — The 5 Red-Black Properties
?? The 5 Red-Black Invariants — ALL must hold at all times:

1. Every node is either RED or BLACK.
2. The root is always BLACK.
3. Every null leaf (NIL sentinel) is BLACK.
4. If a node is RED, both its children must be BLACK. (No two consecutive RED nodes on any path.)
5. For every node, all paths from that node to its descendant null leaves contain the same number of BLACK nodes (the "black height").

Why these 5 rules guarantee O(log n) height? From rule 5, every root-to-leaf path has the same number of BLACK nodes (black height bh). From rule 4, at most every other node can be RED. So the total height h = 2 × bh. And bh = ?h/2?. Since the tree must have 2^bh - 1 internal BLACK nodes, bh = log2(n+1)/2, giving h = 2log2(n+1).

Section 4 — Formal Definitions
  • Black height (bh): the number of BLACK nodes on any path from a node to a descendant null leaf (not counting the node itself, but counting the null leaf). All paths from a given node must have the same bh.
  • NIL sentinel: A shared BLACK dummy node used as all leaf (null) pointers. Simplifies code by eliminating null checks for color.
  • Height guarantee: h = 2 log2(n+1) where n is the number of internal nodes.
  • Rotation cost per insert: At most 2 (AVL can need O(log n) rotations)
  • Rotation cost per delete: At most 3
Section 5 — Complete C# Implementation
C# 
using System;

public enum Color { Red, Black }

public class RBNode
{
    public int     Value;
    public  Color  Color;
    public  RBNode? Left, Right, Parent;

    public RBNode(int val, Color color) { Value = val; Color = color; }
}

public class RedBlackTree
{
    private readonly RBNode NIL;   // Sentinel — represents all null leaves
    private RBNode Root;

    public RedBlackTree()
    {
        NIL  = new RBNode(0, Color.Black);   // Sentinel: always BLACK
        Root = NIL;
    }

    // -- LEFT ROTATION --------------------------------------------
    private void RotateLeft(RBNode x)
    {
        RBNode y = x.Right!;
        x.Right = y.Left;
        if (y.Left != NIL) y.Left.Parent = x;

        y.Parent = x.Parent;
        if      (x.Parent == null)         Root    = y;
        else if (x == x.Parent.Left)       x.Parent.Left  = y;
        else                               x.Parent.Right = y;

        y.Left   = x;
        x.Parent = y;
    }

    // -- RIGHT ROTATION -------------------------------------------
    private void RotateRight(RBNode x)
    {
        RBNode y = x.Left!;
        x.Left = y.Right;
        if (y.Right != NIL) y.Right.Parent = x;

        y.Parent = x.Parent;
        if      (x.Parent == null)        Root           = y;
        else if (x == x.Parent.Right)      x.Parent.Right = y;
        else                               x.Parent.Left  = y;

        y.Right  = x;
        x.Parent = y;
    }

    // -- INSERT ---------------------------------------------------
    public void Insert(int val)
    {
        var z = new RBNode(val, Color.Red) { Left = NIL, Right = NIL };

        // Standard BST insert
        RBNode? parent = null;
        RBNode  curr   = Root;
        while (curr != NIL)
        {
            parent = curr;
            if      (val < curr.Value) curr = curr.Left!;
            else if (val > curr.Value) curr = curr.Right!;
            else return;  // Duplicate
        }

        z.Parent = parent;
        if      (parent == null)       Root          = z;
        else if (val < parent.Value) parent.Left  = z;
        else                          parent.Right = z;

        if (z.Parent == null) { Root.Color = Color.Black; return; }
        if (z.Parent.Parent == null) return;

        InsertFixup(z);
    }

    // -- INSERT FIXUP — restore RB properties ---------------------
    private void InsertFixup(RBNode z)
    {
        while (z.Parent?.Color == Color.Red)  // Violation: two consecutive reds
        {
            if (z.Parent == z.Parent.Parent!.Left)  // Parent is LEFT child of grandparent
            {
                RBNode uncle = z.Parent.Parent.Right!;

                if (uncle.Color == Color.Red)       // Case 1: Uncle is RED — recolor
                {
                    z.Parent.Color         = Color.Black;
                    uncle.Color            = Color.Black;
                    z.Parent.Parent.Color  = Color.Red;
                    z = z.Parent.Parent;
                }
                else
                {
                    if (z == z.Parent.Right)           // Case 2: Inner child — rotate to Case 3
                    {
                        z = z.Parent;
                        RotateLeft(z);
                    }
                    // Case 3: Outer child — rotate grandparent, swap colors
                    z.Parent!.Color        = Color.Black;
                    z.Parent.Parent!.Color = Color.Red;
                    RotateRight(z.Parent.Parent!);
                }
            }
            else   // Symmetric: parent is RIGHT child of grandparent
            {
                RBNode uncle = z.Parent.Parent!.Left!;

                if (uncle.Color == Color.Red)       // Case 1 (mirror)
                {
                    z.Parent.Color        = Color.Black;
                    uncle.Color           = Color.Black;
                    z.Parent.Parent.Color = Color.Red;
                    z = z.Parent.Parent;
                }
                else
                {
                    if (z == z.Parent.Left)            // Case 2 (mirror)
                    {
                        z = z.Parent;
                        RotateRight(z);
                    }
                    z.Parent!.Color        = Color.Black;  // Case 3 (mirror)
                    z.Parent.Parent!.Color = Color.Red;
                    RotateLeft(z.Parent.Parent!);
                }
            }
        }
        Root.Color = Color.Black;  // Property 2: root always BLACK
    }

    // -- CONTAINS -------------------------------------------------
    public bool Contains(int val)
    {
        var n = Root;
        while (n != NIL && n != null)
        {
            if      (val == n.Value) return true;
            else if (val <  n.Value) n = n.Left!;
            else                     n = n.Right!;
        }
        return false;
    }

    public Color RootColor() => Root.Color;
}

// -- Main ----------------------------------------------------------
class Program
{
    static void Main()
    {
        var rbt = new RedBlackTree();
        foreach (int v in new[] { 10, 20, 30, 15, 25, 5 })
            rbt.Insert(v);

        Console.WriteLine($"Root color (must be Black): {rbt.RootColor()}");
        Console.WriteLine($"Contains 15: {rbt.Contains(15)}");
        Console.WriteLine($"Contains 99: {rbt.Contains(99)}");
    }
}

/* -- Output ------------------------------------------------------
   Root color (must be Black): Black
   Contains 15: True
   Contains 99: False
   ---------------------------------------------------------------- */
Section 6 — Complete Python Implementation
Python 
from enum import Enum
from typing import Optional

class Color(Enum):
    RED   = "RED"
    BLACK = "BLACK"

class RBNode:
    def __init__(self, val: int, color: Color = Color.RED):
        self.val    = val
        self.color  = color
        self.left:   Optional['RBNode'] = None
        self.right:  Optional['RBNode'] = None
        self.parent: Optional['RBNode'] = None

class RedBlackTree:
    def __init__(self):
        self.NIL  = RBNode(0, Color.BLACK)  # Sentinel node
        self.root = self.NIL

    def _rotate_left(self, x: RBNode) -> None:
        y = x.right
        x.right = y.left
        if y.left != self.NIL: y.left.parent = x

        y.parent = x.parent
        if   x.parent is None:       self.root      = y
        elif x == x.parent.left:      x.parent.left  = y
        else:                          x.parent.right = y

        y.left   = x
        x.parent = y

    def _rotate_right(self, x: RBNode) -> None:
        y = x.left
        x.left = y.right
        if y.right != self.NIL: y.right.parent = x

        y.parent = x.parent
        if   x.parent is None:       self.root      = y
        elif x == x.parent.right:     x.parent.right = y
        else:                          x.parent.left  = y

        y.right  = x
        x.parent = y

    def insert(self, val: int) -> None:
        z = RBNode(val)
        z.left = z.right = self.NIL

        # Standard BST insert
        parent, curr = None, self.root
        while curr != self.NIL:
            parent = curr
            if   val < curr.val: curr = curr.left
            elif val > curr.val: curr = curr.right
            else: return          # Duplicate

        z.parent = parent
        if   parent is None:    self.root      = z
        elif val < parent.val:  parent.left    = z
        else:                    parent.right   = z

        if z.parent is None:
            z.color = Color.BLACK; return
        if z.parent.parent is None: return

        self._insert_fixup(z)

    def _insert_fixup(self, z: RBNode) -> None:
        while z.parent and z.parent.color == Color.RED:
            if z.parent == z.parent.parent.left:
                uncle = z.parent.parent.right
                if uncle.color == Color.RED:      # Case 1: uncle RED
                    z.parent.color         = Color.BLACK
                    uncle.color            = Color.BLACK
                    z.parent.parent.color  = Color.RED
                    z = z.parent.parent
                else:
                    if z == z.parent.right:        # Case 2: inner child
                        z = z.parent
                        self._rotate_left(z)
                    z.parent.color         = Color.BLACK   # Case 3
                    z.parent.parent.color  = Color.RED
                    self._rotate_right(z.parent.parent)
            else:
                uncle = z.parent.parent.left
                if uncle.color == Color.RED:      # Case 1 (mirror)
                    z.parent.color         = Color.BLACK
                    uncle.color            = Color.BLACK
                    z.parent.parent.color  = Color.RED
                    z = z.parent.parent
                else:
                    if z == z.parent.left:         # Case 2 (mirror)
                        z = z.parent
                        self._rotate_right(z)
                    z.parent.color         = Color.BLACK   # Case 3 (mirror)
                    z.parent.parent.color  = Color.RED
                    self._rotate_left(z.parent.parent)
        self.root.color = Color.BLACK

    def contains(self, val: int) -> bool:
        n = self.root
        while n != self.NIL:
            if   val == n.val: return True
            elif val <  n.val: n = n.left
            else:              n = n.right
        return False


if __name__ == "__main__":
    rbt = RedBlackTree()
    for v in [10, 20, 30, 15, 25, 5]:
        rbt.insert(v)
    print("Root color:", rbt.root.color)        # BLACK
    print("Contains 15:", rbt.contains(15))   # True
    print("Contains 99:", rbt.contains(99))   # False

# Output:
# Root color: Color.BLACK
# Contains 15: True
# Contains 99: False

Section 8 — Real-World Usage
?? C# — SortedSet<T> and SortedDictionary<TKey,TValue>
Both are backed by a Red-Black tree in .NET. SortedSet<T> gives O(log n) Add/Remove/Contains and O(n) ordered enumeration. The Red-Black tree backing means inserts/deletes are fast even with millions of elements in mixed patterns.
?? C++ std::map / std::set
The C++ STL's map and set containers are implemented as Red-Black trees in all major implementations (GCC libstdc++, MSVC, Clang libc++). Inserting 10M elements into std::map gives O(n log n) total time with at most 2 rotations per insert.
?? Java TreeMap / TreeSet
Java's TreeMap and TreeSet are backed by a Red-Black tree. They provide O(log n) put(), get(), remove(), plus in-order iteration. The subMap(), headMap(), and tailMap() operations return views on the underlying tree — powerful for range queries.
?? Linux Kernel — CFS Scheduler
The Completely Fair Scheduler (CFS) in the Linux kernel uses a Red-Black tree to track runnable processes, keyed by their virtual runtime. The process with the smallest virtual runtime (leftmost node) is always next to run. O(log n) insert/delete and O(1) find-min (cached leftmost node).
Section 9 — Common Mistakes
?? Mistake 1 — Not handling the NIL sentinel correctly: All null pointers in an RB tree should point to the NIL sentinel (which is BLACK). If you mix null and NIL, property 3 checks fail silently and rotations produce incorrect parent pointers. Use a single shared NIL object for the entire tree.
?? Mistake 2 — Forgetting to set root to BLACK after fixup: After InsertFixup, the while loop may leave the root as RED (if the inserted node bubbled up to root). Always add Root.Color = Black at the very end of InsertFixup.
?? Mistake 3 — Case 2 forgetting to update z before Case 3: In Case 2 (inner child), after rotating, z is now the parent (it moved UP). You must update z = z.Parent BEFORE the rotation so that Case 3 sees the correct node as "z" (the outer child after the rotation).
?? Mistake 4 — Confusing parent pointer updates in rotation: When rotating, you must update: (1) y's parent pointer, (2) the grandparent's child pointer (was x, now y), (3) T2's parent pointer (was y, now x). Missing any one of these breaks the tree structure.
Section 10 — Check Your Understanding
Q1. What are the 5 Red-Black tree properties? Give them from memory.
?? Think: node colors, root, leaves, consecutive reds, black height.
1. Every node is RED or BLACK.
2. The root is BLACK.
3. Every null leaf (NIL) is BLACK.
4. If a node is RED, both its children are BLACK (no two consecutive REDs on any path).
5. Every path from any node to any descendant null leaf has the same number of BLACK nodes (constant black-height).
Q2. You insert a new node and its parent is RED and its uncle is RED. What happens?
?? Which fixup case is this? Does it require a rotation?
Case 1: Recolor only, no rotation. Set parent and uncle to BLACK, set grandparent to RED. Then repeat the fixup check with grandparent as the new "current" node — the RED grandparent may itself now violate property 4 with its own parent. This propagation continues up at most O(log n) levels.
Q3. Why is a Red-Black tree preferred over AVL for std::map in C++?
?? Think about typical usage patterns: many inserts/deletes vs. many reads.
Red-Black trees perform at most 2 rotations per insert and 3 per delete, while AVL trees may require O(log n) rotations per insert. For workloads with many inserts and deletions (like map/set in general-purpose code), the lower rotation cost per modification gives Red-Black trees better practical performance. AVL would be better if the tree is built once and then only searched.
Q4. After a Red-Black tree insert fixup, why do we always set root.color = BLACK at the end?
?? What can happen to the root's color during Case 1 propagation?
During Case 1 fixup, the grandparent is recolored RED and becomes the new "current" node. If this propagation reaches the root, the root ends up RED (violating property 2). Setting root.color = BLACK after the while loop always restores property 2 and does NOT violate property 5 — making the root BLACK only ADDS to the black-height on every path (increasing it by 1), keeping black-heights equal.
Q5. What is the maximum height of a Red-Black tree with 15 nodes?
?? Use h = 2 log2(n+1). Compute for n=15.
At most 8. Formula: h = 2 × log2(15+1) = 2 × log2(16) = 2 × 4 = 8. In practice, a Red-Black tree with 15 nodes would typically have height 4-5. Compare with AVL: h = ?1.44 × log2(17)? = ?1.44 × 4? = 5. So for the same n, AVL's maximum height is strictly lower than Red-Black's.
