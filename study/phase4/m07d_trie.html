Section 1 — The Analogy: Filing Cabinet by Letter

Imagine a filing cabinet where the first drawer contains all files starting with 'A', 'B', ... 'Z'. Inside the 'A' drawer are subdrawers for each second letter. Inside "AP" is a subdrawer for each third letter: "APA", "APB", ... "APP", "APR" (which would contain the file "APRIL"). The key insight: every file that starts with "AP" shares the same first two drawers. The path from root to a node IS the prefix.

This is a Trie (pronounced "try", from retrieval). Shared prefixes share nodes. If you have 10,000 words starting with "pre", they all share the same 3-node path through root ? 'p' ? 'r' ? 'e'.

Section 2 — The Problem Tries Solve

Given a dictionary of n words, support:

  • Insert word: Add a word to the dictionary
  • Search word: Does the dictionary contain exactly this word?
  • StartsWith prefix: Does any word in the dictionary start with this prefix?
  • Autocomplete: Return all words starting with a given prefix
OperationHashSet<string>Trie
Insert wordO(L)O(L)
Search wordO(L)O(L)
StartsWith prefixO(n × L)O(L)
Autocomplete k resultsO(n × L)O(L + k)
SpaceO(n × L)O(ALPHABET × n × L) worst case

The Trie completely dominates for prefix-based queries. The trade-off is higher space usage — each node may have up to 26 children (for lowercase a–z), but in practice most nodes have only 1–3 children.

Section 4 — Formal Definitions
  • TrieNode: Contains a dictionary/array of children (one per alphabet character) and a boolean IsEnd flag marking whether the path to this node forms a complete word.
  • Prefix path: The sequence of characters on the path from root to a node. The root represents the empty string.
  • IsEnd flag: A node can be both an end-of-word AND an internal node (e.g., "car" is a word AND a prefix of "card").
  • L: Length of the word or prefix being processed. All insert/search/startsWith operations are O(L) regardless of how many words are stored.
  • Alphabet size (s): Typically 26 for lowercase English. Each node holds up to s children. Can use a Dictionary for sparse alphabets.
Section 5 — Complete C# Implementation
C# 
using System;
using System.Collections.Generic;
using System.Text;

// -- Trie Node -----------------------------------------------------
public class TrieNode
{
    public Dictionary<char, TrieNode> Children = new();
    public bool IsEnd = false;
    public int  WordCount = 0;   // Optional: how many times this word was inserted
}

// -- Trie ----------------------------------------------------------
public class Trie
{
    private readonly TrieNode Root = new();

    // -- INSERT ----------------------------------------------------
    public void Insert(string word)
    {
        var node = Root;
        foreach (char c in word)
        {
            if (!node.Children.ContainsKey(c))
                node.Children[c] = new TrieNode();
            node = node.Children[c];
        }
        node.IsEnd = true;
        node.WordCount++;
    }

    // -- SEARCH — exact word match ---------------------------------
    public bool Search(string word)
    {
        var node = FindNode(word);
        return node != null && node.IsEnd;
    }

    // -- STARTSWITH — any word begins with this prefix? ------------
    public bool StartsWith(string prefix)
        => FindNode(prefix) != null;

    // -- Helper: walk to the node at end of prefix -----------------
    private TrieNode? FindNode(string prefix)
    {
        var node = Root;
        foreach (char c in prefix)
        {
            if (!node.Children.TryGetValue(c, out node!)) return null;
        }
        return node;
    }

    // -- GET ALL WORDS starting with prefix (autocomplete) ---------
    public List<string> Autocomplete(string prefix)
    {
        var results = new List<string>();
        var node = FindNode(prefix);
        if (node == null) return results;

        var sb = new StringBuilder(prefix);
        DFS(node, sb, results);
        return results;
    }

    private void DFS(TrieNode node, StringBuilder sb, List<string> results)
    {
        if (node.IsEnd) results.Add(sb.ToString());
        foreach (var (ch, child) in node.Children)
        {
            sb.Append(ch);
            DFS(child, sb, results);
            sb.Remove(sb.Length - 1, 1);  // Backtrack
        }
    }

    // -- GET ALL WORDS in the entire trie --------------------------
    public List<string> GetAllWords() => Autocomplete("");

    // -- DELETE a word (if it exists) ------------------------------
    public bool Delete(string word)
    {
        return DeleteHelper(Root, word, 0);
    }

    private bool DeleteHelper(TrieNode node, string word, int depth)
    {
        if (depth == word.Length)
        {
            if (!node.IsEnd) return false;  // Word not in trie
            node.IsEnd = false;
            return node.Children.Count == 0;  // Can we delete this node?
        }
        char c = word[depth];
        if (!node.Children.TryGetValue(c, out var child)) return false;

        bool shouldDelete = DeleteHelper(child, word, depth + 1);
        if (shouldDelete) node.Children.Remove(c);  // Remove child if leaf and not end
        return shouldDelete && !node.IsEnd && node.Children.Count == 0;
    }
}

// -- Main ----------------------------------------------------------
class Program
{
    static void Main()
    {
        var trie = new Trie();
        foreach (var w in new[] { "cat", "car", "card", "care", "bat", "ball" })
            trie.Insert(w);

        Console.WriteLine("Search 'car':     "  + trie.Search("car"));     // True
        Console.WriteLine("Search 'ca':      "  + trie.Search("ca"));      // False (prefix only)
        Console.WriteLine("StartsWith 'car': "  + trie.StartsWith("car")); // True
        Console.WriteLine("StartsWith 'xyz': "  + trie.StartsWith("xyz")); // False
        Console.WriteLine("Autocomplete 'car': " + string.Join(", ", trie.Autocomplete("car")));
        // car, card, care
        Console.WriteLine("All words: "          + string.Join(", ", trie.GetAllWords()));

        trie.Delete("car");
        Console.WriteLine("After delete 'car', Search 'car': " + trie.Search("car"));  // False
        Console.WriteLine("After delete 'car', Search 'card': " + trie.Search("card")); // True
    }
}

Section 6 — Complete Python Implementation
Python 
from collections import defaultdict
from typing import List, Optional


class TrieNode:
    def __init__(self):
        # defaultdict auto-creates missing children
        self.children: dict = {}
        self.is_end: bool = False
        self.word_count: int = 0


class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True
        node.word_count += 1

    def search(self, word: str) -> bool:
        node = self._find_node(word)
        return node is not None and node.is_end

    def starts_with(self, prefix: str) -> bool:
        return self._find_node(prefix) is not None

    def _find_node(self, prefix: str) -> Optional[TrieNode]:
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return None
            node = node.children[ch]
        return node

    def autocomplete(self, prefix: str) -> List[str]:
        results: List[str] = []
        node = self._find_node(prefix)
        if node is None: return results
        self._dfs(node, list(prefix), results)
        return results

    def _dfs(self, node: TrieNode, path: List[str], results: List[str]) -> None:
        if node.is_end:
            results.append("".join(path))
        for ch, child in node.children.items():
            path.append(ch)
            self._dfs(child, path, results)
            path.pop()             # Backtrack

    def get_all_words(self) -> List[str]:
        return self.autocomplete("")

    def delete(self, word: str) -> bool:
        return self._delete(self.root, word, 0)

    def _delete(self, node: TrieNode, word: str, depth: int) -> bool:
        if depth == len(word):
            if not node.is_end: return False
            node.is_end = False
            return len(node.children) == 0  # True = safe to remove this node
        ch = word[depth]
        if ch not in node.children: return False
        should_delete = self._delete(node.children[ch], word, depth + 1)
        if should_delete:
            del node.children[ch]
        return should_delete and not node.is_end and len(node.children) == 0


if __name__ == "__main__":
    t = Trie()
    for w in ["cat", "car", "card", "care", "bat", "ball"]:
        t.insert(w)

    print("search 'car':",      t.search("car"))       # True
    print("search 'ca':",       t.search("ca"))        # False
    print("starts_with 'car':", t.starts_with("car"))  # True
    print("autocomplete 'car':", t.autocomplete("car")) # ['car', 'card', 'care']
    print("all words:",          t.get_all_words())      # all 6 words
    t.delete("car")
    print("after delete 'car', search 'car':",  t.search("car"))   # False
    print("after delete 'car', search 'card':", t.search("card"))  # True

Section 8 — Real-World Usage
?? Search Engine Autocomplete
Google's search bar shows suggestions as you type. A trie stores millions of common queries. Each keystroke extends the prefix by one character — the trie walk takes O(L) to reach the prefix node, then a BFS/DFS returns the top-k most popular completions. In practice, each trie node also stores the top-k results cached so completions are returned in O(1) after the O(L) prefix walk.
?? Spell Checker / Dictionary Lookup
IDEs like Visual Studio and editors like VS Code use trie-based structures for code completion. The dictionary is loaded into a trie at startup. As you type identifiers, the autocomplete engine walks the trie to find all symbols matching the current prefix. The O(L) lookup is independent of how many symbols are in the dictionary.
?? IP Routing Tables (Longest Prefix Match)
Network routers use binary tries (Patricia trees) to match IP addresses to routing rules. A destination IP is a 32-bit binary string. The router finds the most specific matching rule via "longest prefix match" — walking the trie bit-by-bit until the deepest matching node. This is O(32) = O(1) regardless of routing table size!
Section 9 — Common Mistakes
?? Mistake 1 — Not setting IsEnd flag on insert: After walking all characters, you must mark the final node with isEnd = true. Without this, searching for the exact word returns false even though the characters are in the trie. "car" without IsEnd means only "card" and "care" exist, not "car" itself.
?? Mistake 2 — Confusing Search and StartsWith: Search("ca") should return false if only "cat" is in the trie (because "ca" is a prefix, not a complete word). StartsWith("ca") returns true. The only difference is whether you check node.IsEnd at the end. Always check IsEnd for Search, never for StartsWith.
?? Mistake 3 — Delete breaks existing longer words: When deleting "car", you must NOT delete the 'r' node if "card" or "care" still exist as children. The correct delete algorithm only removes nodes that are (a) not IsEnd and (b) have no children. This is why recursive delete checks node.Children.Count == 0 before recommending node removal.
?? Mistake 4 — Using array[26] instead of Dictionary for sparse alphabets: If your alphabet includes uppercase, digits, punctuation, or Unicode characters, a fixed array[26] won't work. Always use a Dictionary/HashMap for general-purpose tries unless you know the alphabet is exactly lowercase a–z and memory efficiency is critical.
Section 10 — Check Your Understanding
Q1. After inserting "apple", "app", and "application" into an empty trie, how many nodes total exist (including root)?
?? Count shared prefix nodes carefully. All three words share "app".
12 nodes total (including root).
Shared path: root ? 'a' ? 'p' ? 'p' = 3 nodes (plus root = 4)
"app" ends at the 3rd 'p' node (IsEnd=true)
"apple" adds: 'l' ? 'e' = 2 more nodes
"application" adds: 'l' (shared? No — "apple" uses 'l' but then 'e', while "application" uses 'l'?'i'...)
Wait — "appl" is shared! "apple"=a-p-p-l-e, "application"=a-p-p-l-i-c-a-t-i-o-n
Shared: root,a,p,p,l (5 nodes). "apple" adds 'e' (1). "application" adds i,c,a,t,i,o,n (7).
"app" is a prefix of "apple" and "application", so no extra nodes needed — just mark the 3rd 'p' as IsEnd.
Total: 5 shared + 1 ('e') + 7 ('i','c','a','t','i','o','n') = 13... Actually:
root + a + p + p + l (IsEnd for "app" is at 3rd node; "apple" and "application" share through 'l') = root + 4 = 5.
Then 'e' for apple (1), 'i','c','a','t','i','o','n' for application (7) = 5 + 1 + 7 = 13 nodes.
Q2. Why is a Trie better than a HashSet for the "StartsWith" (prefix search) operation?
?? Think about what a HashSet has to do to find all words with a given prefix.
A HashSet has O(1) exact lookup but O(n × L) prefix search — you'd have to iterate over all n words and check if each starts with the prefix (O(L) per word). A Trie's prefix search is O(L) — just walk L nodes down from the root. This is independent of how many words are stored. For autocomplete over 10 million words, the Trie is millions of times faster than a HashSet iteration.
Q3. You delete the word "car" from a trie that also contains "card". What happens to the 'r' node?
?? Should the 'r' node be removed from memory? Why or why not?
The 'r' node should NOT be removed. It still serves as a prefix node for "card". Only the IsEnd flag on the 'r' node is set to false (indicating "car" is no longer a complete word). The 'r' node remains in the trie because its child 'd' (for "card") still exists. Only nodes that are (1) not IsEnd and (2) have zero children can be safely removed.
Q4. How does a trie compare to a sorted array (binary search) for word lookup?
?? Compare O(L) vs. O(L log n) for search; consider how each handles prefix queries.
Exact search: Trie O(L) vs. Sorted array O(L × log n). Trie wins for large n.
Prefix search: Trie O(L) (walk to prefix node, done) vs. Sorted array O(L × log n) (binary search to find range start) + O(k) to enumerate k results.
Space: Trie uses more space due to child pointers, but shares prefix nodes. Sorted array is compact.
Insert: Trie O(L) vs. Sorted array O(n) (shift elements). Trie wins heavily for dynamic dictionaries.
Overall: Trie wins for prefix queries and dynamic data; sorted arrays win for static data with cache-friendly sequential access.
Q5. What modification to a TrieNode would let you implement a word frequency counter (count how many times each word was inserted)?
?? Which node stores word-level information? What field do you add?
Add an integer field WordCount (or count) to TrieNode. During Insert, after walking all characters to the end node, increment node.WordCount++ in addition to setting node.IsEnd = true. The Search operation can then return node.WordCount instead of just a boolean. A count of 0 means not present; count = 1 means present (and the count itself gives frequency). The C# implementation in this module already includes this field.
