Section 1 — The Analogy: The Phone Book Principle

Open a phone book (or think of a sorted list). Looking for "Smith"? You don't start from "A" — you open somewhere in the middle. If you land on "Miller" you know Smith is to the right. You go halfway into the right half. Land on "Robinson" — Smith is still right. Three jumps and you're there.

That's a BST. At every node, you make a binary decision: go left (smaller) or go right (larger). Each decision eliminates roughly half the remaining tree. With 1,000,000 entries and a balanced tree, you reach any value in at most 20 comparisons (log2 1,000,000 ˜ 20).

But here's the catch: this only works if the tree stays balanced. If you insert 1, 2, 3, 4, 5 in order, the "tree" degenerates into a linked list — every number goes to the right, and search is back to O(n). This is why AVL trees and Red-Black trees exist (Module 6D/6E).

?? The BST Invariant: For every node N in a BST:
  • ALL values in the left subtree of N are strictly less than N.Value
  • ALL values in the right subtree of N are strictly greater than N.Value
  • Both left and right subtrees are themselves valid BSTs (recursive definition)
Consequence: inorder traversal of a BST always produces sorted order.
Section 2 — The Problem BST Solves

Suppose you need a data structure that supports all three at O(log n):

  • Insert a value
  • Search for a value (does it exist?)
  • Delete a value
  • Enumerate in sorted order (inorder traversal)
  • Find min/max (leftmost/rightmost node)
  • Find predecessor/successor (next smaller / next larger)

Sorted arrays give you O(log n) search but O(n) insert/delete (shifting). Linked lists give you O(1) insert but O(n) search. A balanced BST gives all of the above at O(log n) — that's the sweet spot.

Section 5 — Complete C# Implementation
C# 
using System;
using System.Collections.Generic;

// -- BST Node -----------------------------------------------------
public class BSTNode
{
    public int Value;
    public BSTNode? Left, Right;
    public BSTNode(int val) { Value = val; }
}

// -- Binary Search Tree --------------------------------------------
public class BST
{
    public BSTNode? Root;

    // -- INSERT: O(h) average, O(log n) balanced ------------------
    public void Insert(int val) => Root = InsertRec(Root, val);

    private BSTNode InsertRec(BSTNode? node, int val)
    {
        if (node == null) return new BSTNode(val); // Found empty spot
        if (val < node.Value)
            node.Left  = InsertRec(node.Left,  val);  // Go left
        else if (val > node.Value)
            node.Right = InsertRec(node.Right, val);  // Go right
        // If val == node.Value: duplicate, ignore (common policy)
        return node;
    }

    // -- SEARCH: O(h) — returns node or null ----------------------
    public bool Contains(int val) => Search(Root, val) != null;

    private BSTNode? Search(BSTNode? node, int val)
    {
        if (node == null || node.Value == val) return node;
        return val < node.Value
            ? Search(node.Left,  val)   // Go left
            : Search(node.Right, val);  // Go right
    }

    // -- DELETE: O(h) — three cases ------------------------------
    public void Delete(int val) => Root = DeleteRec(Root, val);

    private BSTNode? DeleteRec(BSTNode? node, int val)
    {
        if (node == null) return null;  // Value not found

        if (val < node.Value)
        {
            node.Left = DeleteRec(node.Left, val);   // Go left
        }
        else if (val > node.Value)
        {
            node.Right = DeleteRec(node.Right, val);  // Go right
        }
        else  // Found the node to delete
        {
            // Case 1: Leaf node (no children)
            if (node.Left == null && node.Right == null) return null;

            // Case 2a: Only right child
            if (node.Left == null)  return node.Right;

            // Case 2b: Only left child
            if (node.Right == null) return node.Left;

            // Case 3: Two children — find inorder successor (min of right subtree)
            int successorVal = FindMin(node.Right);        // Smallest in right subtree
            node.Value = successorVal;                       // Copy successor value here
            node.Right = DeleteRec(node.Right, successorVal); // Delete the successor
        }
        return node;
    }

    // -- MIN / MAX ------------------------------------------------
    public int FindMin(BSTNode? node)
    {
        if (node == null) throw new InvalidOperationException("Tree is empty");
        while (node.Left != null) node = node.Left;  // Go leftmost
        return node.Value;
    }

    public int FindMax()
    {
        if (Root == null) throw new InvalidOperationException("Tree is empty");
        var node = Root;
        while (node.Right != null) node = node.Right;  // Go rightmost
        return node.Value;
    }

    // -- INORDER (sorted output) ----------------------------------
    public List<int> InorderSorted()
    {
        var result = new List<int>();
        InorderRec(Root, result);
        return result;
    }

    private void InorderRec(BSTNode? node, List<int> result)
    {
        if (node == null) return;
        InorderRec(node.Left,  result);
        result.Add(node.Value);
        InorderRec(node.Right, result);
    }

    // -- VALIDATE: is this tree a valid BST? ---------------------
    public bool IsValidBST() =>
        IsValid(Root, int.MinValue, int.MaxValue);

    private bool IsValid(BSTNode? node, int min, int max)
    {
        if (node == null) return true;                   // Empty is valid
        if (node.Value <= min || node.Value >= max) return false; // Violates bounds
        return IsValid(node.Left,  min, node.Value) &&   // Left must be < current
               IsValid(node.Right, node.Value, max);      // Right must be > current
    }

    // -- HEIGHT ---------------------------------------------------
    public int Height(BSTNode? n) =>
        n == null ? -1 : 1 + Math.Max(Height(n.Left), Height(n.Right));
}

// -- Main ----------------------------------------------------------
class Program
{
    static void Main()
    {
        var bst = new BST();

        // Insert values — building a balanced tree
        foreach (int v in new[] { 50, 30, 70, 20, 40, 60, 80 })
            bst.Insert(v);

        Console.WriteLine("Sorted output: " + string.Join(", ", bst.InorderSorted()));
        Console.WriteLine($"Contains 40:   {bst.Contains(40)}");
        Console.WriteLine($"Contains 55:   {bst.Contains(55)}");
        Console.WriteLine($"Min: {bst.FindMin(bst.Root)}, Max: {bst.FindMax()}");
        Console.WriteLine($"Height: {bst.Height(bst.Root)}");
        Console.WriteLine($"Valid BST: {bst.IsValidBST()}");

        // Delete operations — test all three cases
        bst.Delete(20);  // Case 1: leaf
        bst.Delete(70);  // Case 3: two children (successor = 80)
        bst.Delete(30);  // Case 2: one child (40)
        Console.WriteLine("After deletes: " + string.Join(", ", bst.InorderSorted()));
    }
}

/* -- Output ------------------------------------------------------

   Contains 40:   True
   Contains 55:   False
   Min: 20, Max: 80
   Height: 2
   Valid BST: True
   After deletes: 40, 50, 60, 80
   ---------------------------------------------------------------- */
Section 6 — Complete Python Implementation
Python 
from typing import Optional, List

# -- BST Node -----------------------------------------------------
class BSTNode:
    def __init__(self, val: int):
        self.val = val
        self.left:  Optional['BSTNode'] = None
        self.right: Optional['BSTNode'] = None

# -- Binary Search Tree --------------------------------------------
class BST:
    def __init__(self):
        self.root: Optional[BSTNode] = None

    # -- INSERT ---------------------------------------------------
    def insert(self, val: int) -> None:
        self.root = self._insert(self.root, val)

    def _insert(self, node: Optional[BSTNode], val: int) -> BSTNode:
        if node is None: return BSTNode(val)  # Found empty slot
        if val < node.val:
            node.left  = self._insert(node.left,  val)   # Go left
        elif val > node.val:
            node.right = self._insert(node.right, val)   # Go right
        # If val == node.val: duplicate, ignore
        return node

    # -- SEARCH ---------------------------------------------------
    def contains(self, val: int) -> bool:
        node = self.root
        while node:
            if   val == node.val: return True
            elif val <  node.val: node = node.left
            else:                 node = node.right
        return False

    # -- DELETE (three cases) -------------------------------------
    def delete(self, val: int) -> None:
        self.root = self._delete(self.root, val)

    def _delete(self, node: Optional[BSTNode], val: int) -> Optional[BSTNode]:
        if node is None: return None             # Not found

        if   val < node.val:
            node.left  = self._delete(node.left,  val)
        elif val > node.val:
            node.right = self._delete(node.right, val)
        else:                                       # Found node
            if not node.left  and not node.right:  # Case 1: leaf
                return None
            if not node.left:  return node.right  # Case 2a: only right child
            if not node.right: return node.left   # Case 2b: only left child
            # Case 3: two children — find inorder successor
            successor = self._find_min(node.right)
            node.val   = successor.val               # Copy successor value
            node.right = self._delete(node.right, successor.val)
        return node

    # -- MIN / MAX ------------------------------------------------
    def _find_min(self, node: BSTNode) -> BSTNode:
        while node.left: node = node.left          # Leftmost = minimum
        return node

    def find_min(self) -> int:
        if not self.root: raise ValueError("Tree is empty")
        return self._find_min(self.root).val

    def find_max(self) -> int:
        if not self.root: raise ValueError("Tree is empty")
        node = self.root
        while node.right: node = node.right        # Rightmost = maximum
        return node.val

    # -- INORDER (sorted output) ----------------------------------
    def inorder_sorted(self) -> List[int]:
        result: List[int] = []
        self._inorder(self.root, result)
        return result

    def _inorder(self, node: Optional[BSTNode], result: List[int]) -> None:
        if not node: return
        self._inorder(node.left,  result)
        result.append(node.val)
        self._inorder(node.right, result)

    # -- VALIDATE -------------------------------------------------
    def is_valid_bst(self) -> bool:
        def valid(node, lo, hi):
            if not node: return True
            if not (lo < node.val < hi): return False
            return valid(node.left, lo, node.val) and valid(node.right, node.val, hi)
        return valid(self.root, float('-inf'), float('inf'))

    def height(self, node: Optional[BSTNode] = None) -> int:
        node = node if node is not None else self.root
        if not node: return -1
        return 1 + max(self.height(node.left), self.height(node.right))


if __name__ == "__main__":
    bst = BST()
    for v in [50, 30, 70, 20, 40, 60, 80]:
        bst.insert(v)

    print("Sorted output:", bst.inorder_sorted())
    print("Contains 40:  ", bst.contains(40))
    print("Contains 55:  ", bst.contains(55))
    print("Min:", bst.find_min(), " Max:", bst.find_max())
    print("Height:       ", bst.height())
    print("Valid BST:    ", bst.is_valid_bst())

    bst.delete(20)  # Case 1: leaf
    bst.delete(70)  # Case 3: two children
    bst.delete(30)  # Case 2: one child
    print("After deletes:", bst.inorder_sorted())

Section 7 — Complexity Analysis
OperationBalanced (h=log n)Degenerate (h=n)Why
SearchO(log n)O(n)Follow one path from root; path length = h
InsertO(log n)O(n)Search for correct position, then link
DeleteO(log n)O(n)Search for node, then find successor if needed
Find min/maxO(log n)O(n)Traverse to leftmost/rightmost
Inorder traversalO(n)O(n)Must visit all n nodes regardless
Space (BST structure)O(n)O(n)One node per element
?? The Degenerate Case: Inserting already-sorted data (1, 2, 3, ..., n) into a plain BST creates a right-skewed tree of height n. All operations degrade to O(n) — the same as a linked list, but with double the memory (two pointers per node instead of one). This is why plain BSTs are almost never used in production — use SortedSet<T> or SortedDictionary<T,V> in C# (backed by a Red-Black tree), or Python's sortedcontainers.SortedList.
Section 8 — Real-World Usage
?? .NET — SortedSet<T> and SortedDictionary<TKey,TValue>
C#'s SortedSet<T> and SortedDictionary<TKey,TValue> are both backed by a Red-Black tree (a self-balancing BST). They give you O(log n) insert/delete/search AND always-sorted iteration.
Use SortedSet when you need unique elements in sorted order (e.g., leaderboard, scheduling events by time). Use SortedDictionary for key-value pairs sorted by key (e.g., ordered index, range queries by date).
?? Database Indexes (MySQL, PostgreSQL) — Sorted index lookups
When you run SELECT * FROM orders WHERE date BETWEEN '2024-01-01' AND '2024-12-31', the database uses a tree index (B-Tree variant — Module 6H) that is conceptually a BST. It finds the first date in O(log n) comparisons, then scans forward. Without the index, every query is O(n) full table scan.
Section 9 — Common Mistakes
?? Mistake 1 — Forgetting to return the modified node in recursive insert/delete: The pattern node.Left = InsertRec(node.Left, val) is critical — you must reassign the child pointer to whatever the recursive call returns. Forgetting this causes the insert to have no effect on the actual tree.
?? Mistake 2 — Validating BST incorrectly: The naive check "left.Value < node.Value < right.Value" is wrong. It doesn't check that ALL values in the subtree satisfy the constraint. Example: root=10, left=5, left.right=15 passes the naive check but 15 > 10 violates the BST property. Always pass min/max bounds down recursively.
?? Mistake 3 — Using == comparison with Delete finding the wrong node: After deleting a node with two children by copying the successor's value, you must delete the successor from the right subtree specifically. Don't call Delete(val) from root — the value might appear higher in the tree and delete the wrong node.
?? Mistake 4 — Using plain BST in production: A plain BST is vulnerable to sorted input. In interview problems, always mention "in practice, we'd use a self-balancing BST (AVL or Red-Black) or C#'s SortedSet/SortedDictionary" to show you understand the limitation.
Section 10 — Check Your Understanding
Q1. You insert the values 10, 5, 15, 3, 7, 12, 20 into an empty BST. What is the inorder traversal output?
?? Inorder on a BST always produces sorted order.
3, 5, 7, 10, 12, 15, 20 — Inorder traversal (Left ? Root ? Right) on a BST always yields values in ascending sorted order. This is the defining property of BSTs — the BST invariant guarantees left subtree < root < right subtree at every level, so inorder naturally produces sorted output.
Q2. In the tree from Q1, how many comparisons are needed to confirm that 6 is NOT in the tree?
?? Trace the search path: which nodes do you visit?
3 comparisons: Start at 10 (6 < 10, go left) ? visit 5 (6 > 5, go right) ? visit 7 (6 < 7, go left) ? null (not found). You compared against 10, 5, and 7 — three nodes.
Q3. You delete the root (10) from the tree in Q1. Which value becomes the new root, and why?
?? Delete Case 3: two children. Find the inorder successor.
12 becomes the new root. Delete with two children: find the inorder successor — the smallest value in the right subtree. The right subtree of 10 is rooted at 15, and its smallest value is 12. Copy 12 to the root, then delete 12 from the right subtree (12 is a leaf, so just remove it). Result: root = 12, right subtree = {15, 20}, left subtree = {5, 3, 7} unchanged.
Q4. What is the height of a perfectly balanced BST with 15 nodes?
?? Use h = floor(log2 n).
Height = 3. For n = 15: h = ?log2(15)? = ?3.9? = 3. A perfect binary tree with height 3 has 24 - 1 = 15 nodes. Alternatively: level 0 = 1, level 1 = 2, level 2 = 4, level 3 = 8. Total = 15 nodes, height = 3.
Q5. You're given a BST and asked to find the kth smallest element. What is the optimal approach and its time complexity?
?? Which traversal produces elements in sorted order?
Inorder traversal, stop at the kth node. O(h + k) time. — Inorder traversal visits nodes in ascending order. Keep a counter; when it reaches k, return that node's value. You don't need to complete the full traversal. For a balanced BST, h = O(log n) and k = n, so worst case is O(n) but average is much better. Space: O(h) for the recursive stack (or O(1) with iterative + Morris traversal).
