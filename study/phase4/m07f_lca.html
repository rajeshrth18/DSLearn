Section 1 — The Analogy: Family Tree Common Ancestor

Given a family tree, the Lowest Common Ancestor (LCA) of two people is their most recent common ancestor — the closest ancestor that both share. "Lowest" means deepest in the tree (closest to the leaves), not lowest in value or position.

For example: Alice and Bob are cousins. Their grandparent is a common ancestor. Their great-grandparent is also a common ancestor. But the grandparent is the lowest (deepest) common ancestor — the most recent one in the family tree.

In a tree with n nodes, finding this efficiently matters when you have millions of queries: "What is the path between node u and node v?" or "What is the distance from u to v?" Both reduce to LCA queries.

Section 2 — The Problem LCA Solves
  • LCA query: Given a rooted tree and two nodes u, v, find their lowest common ancestor.
  • Distance query: dist(u, v) = depth(u) + depth(v) - 2 × depth(LCA(u,v))
  • Path sum query: Sum of weights on path from u to v: uses LCA as the split point
  • Multiple queries: If you have Q queries on a tree with n nodes, naïve O(n) per query ? O(nQ) total. Binary lifting: O(n log n) preprocessing + O(log n) per query ? much better for Q >> 1.

LCA(5, 6) — Naïve approach: Path from 5 to root: 5 ? 2 ? 1 Path from 6 to root: 6 ? 3 ? 1 First common: 1 ? LCA(5,6) = 1 LCA(4, 8) — Naïve approach: Path from 4: 4 ? 2 ? 1 Path from 8: 8 ? 4 ? 2 ? 1 First common: 4 ? LCA(4,8) = 4 Problem: O(n) per query for a skewed tree (chain).

3.3 — Approach 2: Depth Equalization + Parent Walk O(h)

LCA(8, 6) — using depth + parent walk: depth[8] = 3, depth[6] = 2 Step 1: Equalize depths — bring deeper node up 8 is at depth 3, 6 is at depth 2 ? 8 needs to go up 1 step 8 ? parent = 4 (now both at depth 2) Step 2: Walk both nodes up simultaneously until they meet 4 ? 6 ? both go up: 4?2, 6?3 2 ? 3 ? both go up: 2?1, 3?1 1 == 1 ? FOUND: LCA(8,6) = 1 Better, but O(h) in worst case for skewed trees.

3.4 — Approach 3: Binary Lifting O(log n)

BINARY LIFTING PREPROCESSING: up[node][j] = 2^j-th ancestor of node up[node][0] = direct parent up[node][j] = up[up[node][j-1]][j-1] (ancestor of ancestor) For our tree (LOG = 4 levels): up[8][0]=4, up[8][1]=2, up[8][2]=1, up[8][3]=1 (root's ancestor = root) up[4][0]=2, up[4][1]=1, up[4][2]=1 up[6][0]=3, up[6][1]=1 LCA(8, 7) using binary lifting: depth[8]=3, depth[7]=2 Step 1: Equalize — 8 needs to go up 1 (depth diff = 3-2=1) 1 = binary 1 = 2^0 ? jump up[8][0] = 4 Now: node_u = 4 (depth 2), node_v = 7 (depth 2) Step 2: Binary search for LCA Check 2^j jumps from both, largest j where they DON'T meet: j=1: up[4][1]=1, up[7][1]=1 ? same (1==1) ? don't jump (they'd overshoot) j=0: up[4][0]=2, up[7][0]=3 ? different ? JUMP: 4?2, 7?3 Now: 2 ? 3, check their parents: parent[2]=1=parent[3]=1 ? LCA = 1
Section 4 — Formal Definitions
  • LCA(u, v): The deepest node that is an ancestor of both u and v. A node is an ancestor of itself.
  • depth[v]: Number of edges from root to v. depth[root] = 0.
  • up[v][j]: The 2^j-th ancestor of v. up[v][0] = parent[v]. up[v][j] = up[up[v][j-1]][j-1].
  • LOG: ?log2(n)? + 1. The maximum number of jumps needed. For n=105: LOG=17.
  • Binary lifting query: (1) Bring both nodes to the same depth using power-of-2 jumps. (2) Jump both nodes upward in powers of 2, largest first, while they differ. Their parent is the LCA.
Section 5 — Complete C# Implementation
C# 
using System;
using System.Collections.Generic;

// -- LCA with Binary Lifting ---------------------------------------
public class LCA
{
    private readonly int    N;
    private readonly int    LOG;
    private readonly int[]  Depth;
    private readonly int[,] Up;   // up[node, j] = 2^j-th ancestor
    private readonly List<int>[] Adj;

    public LCA(int n)
    {
        N   = n;
        LOG = (int)Math.Log2(n) + 2;
        Depth = new int[n + 1];
        Up    = new int[n + 1, LOG];
        Adj   = new List<int>[n + 1];
        for (int i = 0; i <= n; i++) Adj[i] = new List<int>();

        // Initialize all ancestors to 0 (root sentinel)
        for (int i = 0; i <= n; i++)
            for (int j = 0; j < LOG; j++)
                Up[i, j] = 0;
    }

    public void AddEdge(int u, int v)
    {
        Adj[u].Add(v);
        Adj[v].Add(u);
    }

    // -- PREPROCESS — DFS to fill depth[] and up[,] ---------------
    public void Build(int root = 1)
    {
        DFS(root, 0, root);   // parent of root = root (sentinel)
    }

    private void DFS(int v, int depth, int parent)
    {
        Depth[v]  = depth;
        Up[v, 0] = parent;   // Direct parent

        // Fill binary lifting table for this node
        for (int j = 1; j < LOG; j++)
            Up[v, j] = Up[Up[v, j - 1], j - 1];

        foreach (int neighbor in Adj[v])
            if (neighbor != parent)
                DFS(neighbor, depth + 1, v);
    }

    // -- QUERY — find LCA of u and v -------------------------------
    public int QueryLCA(int u, int v)
    {
        // Step 1: Bring u and v to the same depth
        if (Depth[u] < Depth[v]) (u, v) = (v, u);  // u is deeper
        int diff = Depth[u] - Depth[v];
        for (int j = 0; j < LOG; j++)
            if ((diff >> j & 1) == 1)  // j-th bit set in diff
                u = Up[u, j];

        // Step 2: If they're now the same node, that IS the LCA
        if (u == v) return u;

        // Step 3: Jump both nodes upward, largest power first, while different
        for (int j = LOG - 1; j >= 0; j--)
            if (Up[u, j] != Up[v, j])  // Don't jump if they'd meet
            {
                u = Up[u, j];
                v = Up[v, j];
            }

        return Up[u, 0];  // Parent of u (= parent of v) = LCA
    }

    // -- DISTANCE between two nodes --------------------------------
    public int Distance(int u, int v)
    {
        int lca = QueryLCA(u, v);
        return Depth[u] + Depth[v] - 2 * Depth[lca];
    }
}

class Program
{
    static void Main()
    {
        // Build tree: 1-2-3-4-5-6-7-8 as in the diagram
        var lca = new LCA(8);
        lca.AddEdge(1, 2); lca.AddEdge(1, 3); lca.AddEdge(2, 4);
        lca.AddEdge(2, 5); lca.AddEdge(3, 6); lca.AddEdge(3, 7);
        lca.AddEdge(4, 8);
        lca.Build(1);

        Console.WriteLine($"LCA(5, 6)  = {lca.QueryLCA(5, 6)}");   // 1
        Console.WriteLine($"LCA(4, 8)  = {lca.QueryLCA(4, 8)}");   // 4
        Console.WriteLine($"LCA(8, 7)  = {lca.QueryLCA(8, 7)}");   // 1
        Console.WriteLine($"LCA(5, 4)  = {lca.QueryLCA(5, 4)}");   // 2
        Console.WriteLine($"Dist(5,6)  = {lca.Distance(5, 6)}");   // 4
        Console.WriteLine($"Dist(8,7)  = {lca.Distance(8, 7)}");   // 5
        Console.WriteLine($"Dist(4,8)  = {lca.Distance(4, 8)}");   // 1
    }
}

Section 6 — Complete Python Implementation
Python 
import sys
from typing import List
from math import log2
sys.setrecursionlimit(100000)


class LCA:
    def __init__(self, n: int):
        self.n   = n
        self.LOG = int(log2(n)) + 2 if n > 1 else 2
        self.depth: List[int]       = [0] * (n + 1)
        self.up:    List[List[int]]  = [[0] * self.LOG for _ in range(n + 1)]
        self.adj:   List[List[int]]  = [[] for _ in range(n + 1)]

    def add_edge(self, u: int, v: int) -> None:
        self.adj[u].append(v)
        self.adj[v].append(u)

    def build(self, root: int = 1) -> None:
        self._dfs(root, 0, root)

    def _dfs(self, v: int, d: int, parent: int) -> None:
        self.depth[v]   = d
        self.up[v][0]  = parent
        for j in range(1, self.LOG):
            self.up[v][j] = self.up[self.up[v][j-1]][j-1]
        for nb in self.adj[v]:
            if nb != parent:
                self._dfs(nb, d + 1, v)

    def query_lca(self, u: int, v: int) -> int:
        # Ensure u is the deeper node
        if self.depth[u] < self.depth[v]: u, v = v, u

        # Step 1: Bring u to same depth as v
        diff = self.depth[u] - self.depth[v]
        for j in range(self.LOG):
            if (diff >> j) & 1:
                u = self.up[u][j]

        if u == v: return u

        # Step 2: Jump both up while different
        for j in range(self.LOG - 1, -1, -1):
            if self.up[u][j] != self.up[v][j]:
                u = self.up[u][j]
                v = self.up[v][j]
        return self.up[u][0]

    def distance(self, u: int, v: int) -> int:
        lca = self.query_lca(u, v)
        return self.depth[u] + self.depth[v] - 2 * self.depth[lca]


if __name__ == "__main__":
    lca = LCA(8)
    for u, v in [(1,2),(1,3),(2,4),(2,5),(3,6),(3,7),(4,8)]:
        lca.add_edge(u, v)
    lca.build(1)

    print("LCA(5,6) =",  lca.query_lca(5, 6))   # 1
    print("LCA(4,8) =",  lca.query_lca(4, 8))   # 4
    print("LCA(8,7) =",  lca.query_lca(8, 7))   # 1
    print("LCA(5,4) =",  lca.query_lca(5, 4))   # 2
    print("Dist(5,6)=",  lca.distance(5, 6))    # 4
    print("Dist(8,7)=",  lca.distance(8, 7))    # 5
    print("Dist(4,8)=",  lca.distance(4, 8))    # 1

# LCA(5,6) = 1

Section 7 — Complexity Analysis

Naïve (path to root + common)O(n)O(n)O(n)
Parent pointer + depth equalizationO(n)O(h)O(n)
Binary LiftingO(n log n)O(log n)O(n log n)
Sparse Table (Euler Tour + RMQ)O(n log n)O(1)O(n log n)
?? When to use Binary Lifting: When you have many queries (Q >> 1) and the tree is arbitrary (possibly skewed). For a balanced tree, the parent-pointer approach (O(log n) due to tree height) may be simpler to implement. For competitive programming with Q = 105 queries on n = 105 nodes, always use binary lifting.
Section 8 — Real-World Usage
?? Network Routing — Finding Common Gateway
In hierarchical network topologies (data center networks with spine-leaf architecture), finding the shortest path between two servers often reduces to finding their LCA in the topology tree: the closest common switch/router they share. LCA + depth gives the path length in O(1) after preprocessing.
?? Version Control — Git Merge-Base
The git merge-base A B command finds the most recent common commit ancestor of two branches — this is exactly an LCA query on the commit graph (which is a DAG, not a tree, but the tree LCA concept generalizes). Git uses this to determine what changed in each branch since their divergence.
?? Computational Biology — Phylogenetic Trees
A phylogenetic tree represents evolutionary relationships between species. The LCA of two species is their most recent common ancestor. Questions like "When did humans and chimpanzees diverge?" are LCA queries on the phylogenetic tree, where edge weights represent evolutionary distance (time or mutation count).
Section 9 — Common Mistakes
?? Mistake 1 — Off-by-one in depth equalization: When equalizing depths, you jump the DEEPER node up by (depth[u] - depth[v]) steps. Be careful: you're reading the diff bit-by-bit (diff >> j & 1). After equalization, both nodes should be at the SAME depth, not one step apart. Verify: after the loop, depth[u] should equal depth[v].
?? Mistake 2 — Wrong ancestor for root's parent: The root has no parent. In binary lifting, up[root][j] = root for all j (root points to itself). This is the standard sentinel convention — jumping above the root stays at the root. Initialize this correctly or out-of-bounds access will occur.
?? Mistake 3 — Not handling u == v after depth equalization: If one node is an ancestor of the other, after equalization they'll be the same node. Return immediately — don't enter the second loop, which would incorrectly jump both nodes up one more time.
?? Mistake 4 — Building the up table in wrong order: up[v][j] depends on up[up[v][j-1]][j-1]. This means you must compute up[v][j-1] BEFORE up[v][j] for all nodes. The DFS naturally ensures this — when processing a node, all its ancestors have already been processed (they were on the recursion stack before the current node).
Section 10 — Check Your Understanding
Q1. For a tree with 100,000 nodes, what is LOG (the number of columns in the binary lifting table)? How much space does the full up[][] table use?
?? LOG = ?log2(100000)? + 1. Space = n × LOG entries.
log2(100000) ˜ 16.6, so ?16.6? = 17, LOG = 17 + 1 = 18 (with a safety buffer).
Space: 100000 × 17 = 1,700,000 integers. At 4 bytes each ˜ 6.8 MB. This is well within typical competitive programming memory limits (256MB).
Q2. How do you compute the distance between two nodes u and v using LCA?
?? dist(u,v) = ? Think about the path u ? LCA ? v.
dist(u, v) = depth[u] + depth[v] - 2 × depth[LCA(u,v)]
The path from u to v goes: u ? LCA ? v. The length u?LCA = depth[u] - depth[LCA]. The length LCA?v = depth[v] - depth[LCA]. Total = depth[u] - depth[LCA] + depth[v] - depth[LCA] = depth[u] + depth[v] - 2×depth[LCA].
Q3. What is up[v][2] in binary lifting? Describe it in words.
?? up[v][j] is the 2^j-th ancestor. What is 2^2?
up[v][2] is the 4th ancestor of v (2² = 4 levels up). It's computed as up[up[v][1]][1] — "the 2nd ancestor of my 2nd ancestor". In the DFS, up[v][0] = parent, up[v][1] = grandparent (2 levels), up[v][2] = great-great-grandparent (4 levels), up[v][3] = 8 levels up, and so on. This exponential doubling is what allows us to cover any distance in O(log n) jumps.
Q4. In the binary lifting LCA query, why do we iterate j from LOG-1 down to 0 (not 0 to LOG-1) in Step 2?
?? Think about greedy bit-by-bit construction of the jump distance.
We want to jump both nodes as high as possible WITHOUT making them meet (because once they meet, their common ancestor has been overshot). By starting with the largest power (LOG-1 ? 0), we use a greedy approach: try the biggest jump first, skip if it would cause both nodes to merge, then try the next smaller jump. This is the same as constructing the binary representation of the LCA's depth difference greedily from most-significant bit to least-significant bit.
Q5. You have 105 nodes and 105 LCA queries. Compare binary lifting O(n log n + Q log n) vs. naïve O(n + Qn). When does binary lifting win?
?? Substitute n = Q = 105, log n ˜ 17.
Naïve: n + Q×n = 105 + 105×105 = 105 + 10¹° ˜ 10 billion operations. Impossibly slow.
Binary lifting: n log n + Q log n = 105×17 + 105×17 = 3.4 million operations. Fast.
Binary lifting wins overwhelmingly when Q is large (even Q = 100 queries justifies the preprocessing for large n). The breakeven is roughly Q × n = Q × log n + n × log n ? Q ˜ log n ? Q = 17 queries for n=105. So for any non-trivial number of queries, binary lifting is better.
