Section 1 — The Analogy: The VIP Queue

Imagine a hospital emergency room. Patients don't wait in arrival order — they're treated by severity. The most critical patient is always seen first. When a new patient arrives, a nurse quickly determines their priority and slots them in. When the doctor is ready, the most critical patient steps forward instantly.

This is a heap. A heap is a partially ordered data structure that only guarantees one thing: the minimum (or maximum) is always at the top. It doesn't maintain full sorted order — only the root-to-leaf "parent is always better than children" invariant. This relaxed requirement makes it extremely efficient: O(1) to peek at the best item, O(log n) to insert or remove.

Section 2 — The Problem Heaps Solve

You need to repeatedly:

  • Insert items with priorities
  • Retrieve (and remove) the item with the highest priority

Sorted array: O(1) max but O(n) insert. Unsorted array: O(1) insert but O(n) find-max. Heap: O(log n) both. This makes heaps the backbone of priority queues, Dijkstra's algorithm, Prim's MST, heap sort, and the k-way merge.

📌 Two Types of Heaps:
Min-Heap: Parent ≤ both children. Root = minimum element. Extract-min in O(log n).
Max-Heap: Parent ≥ both children. Root = maximum element. Extract-max in O(log n).
Python's heapq module is a min-heap. C#'s PriorityQueue<TElement,TPriority> is also a min-heap (lowest priority value is dequeued first).
Section 3 — Visual Walkthrough

3.1 — Array Representation of a Complete Binary Tree

A heap is stored as an ARRAY (no pointers needed — perfect for cache performance): Array index (0-based): [0] [1] [2] [3] [4] [5] [6] Values: 1 3 2 7 4 5 6 Tree visualization: 1 ← index 0 (root) / \ 3 2 ← index 1, 2 / \ / \ 7 4 5 6 ← index 3, 4, 5, 6 INDEX ARITHMETIC (0-based): Parent of i: (i - 1) / 2 (integer division) Left child of i: 2 * i + 1 Right child of i: 2 * i + 2 Examples: Parent of index 3 (7): (3-1)/2 = 1 → value 3 ✓ (7 > 3 in min-heap? NO → 3 < 7 ✓) Left child of 1 (3): 2*1+1 = 3 → value 7 ✓ Right child of 1 (3): 2*1+2 = 4 → value 4 ✓

3.2 — Heapify Up (Insert)

INSERT 0 into the min-heap [1, 3, 2, 7, 4, 5, 6]: Step 1: Append to end [1, 3, 2, 7, 4, 5, 6, 0] ← 0 is at index 7 Step 2: Heapify Up — compare with parent, swap if smaller Index 7, parent = (7-1)/2 = 3, value 7 0 < 7 → SWAP [1, 3, 2, 0, 4, 5, 6, 7] Index 3, parent = (3-1)/2 = 1, value 3 0 < 3 → SWAP [1, 0, 2, 3, 4, 5, 6, 7] Index 1, parent = (1-1)/2 = 0, value 1 0 < 1 → SWAP [0, 1, 2, 3, 4, 5, 6, 7] Index 0 = root → STOP Final heap: [0, 1, 2, 3, 4, 5, 6, 7] ← 0 correctly at root!

3.3 — Heapify Down (Extract Min)

EXTRACT MIN from [0, 1, 2, 3, 4, 5, 6, 7]: Step 1: Save root (0), move last element to root, shrink array Remove 0. Move 7 to root. [7, 1, 2, 3, 4, 5, 6] Step 2: Heapify Down — swap with smallest child if violation Index 0 (7): left=1 (val 1), right=2 (val 2). Smallest child: 1 (val 1) 7 > 1 → SWAP(0, 1) [1, 7, 2, 3, 4, 5, 6] Index 1 (7): left=3 (val 3), right=4 (val 4). Smallest child: 3 (val 3) 7 > 3 → SWAP(1, 3) [1, 3, 2, 7, 4, 5, 6] Index 3 (7): left=7 → out of bounds. STOP. Final: [1, 3, 2, 7, 4, 5, 6] ← valid min-heap, 1 is new minimum
Section 4 — Formal Definitions
  • Heap property: In a min-heap, every node ≤ its children. In a max-heap, every node ≥ its children.
  • Complete binary tree: All levels are completely filled except possibly the last, which is filled left to right. This ensures the array representation is compact (no gaps).
  • Heapify Up: After inserting at the end, bubble the new element up until the heap property is restored. O(log n).
  • Heapify Down (Sift Down): After removing the root (swap with last element, shrink), push the new root down until the heap property is restored. O(log n).
  • Build Heap: Given an unsorted array of n elements, build a valid heap in O(n) time by calling Heapify Down on all internal nodes from right to left.
💡 Why Build Heap is O(n) not O(n log n): The insight is that most nodes are near the bottom and have small subtrees to heapify. The total work is ∑(h × n/2^h) for h = 1 to log n, which converges to O(n). In contrast, inserting n elements one-by-one is O(n log n).
Section 5 — Complete C# Implementation
C# 
using System;
using System.Collections.Generic;

// ── Manual Min-Heap ───────────────────────────────────────────────
public class MinHeap
{
    private readonly List<int> _data = new();

    public int Count  => _data.Count;
    public int Peek() => _data.Count > 0 ? _data[0] : throw new InvalidOperationException();

    // ── INSERT (Heapify Up) ──────────────────────────────────────
    public void Push(int val)
    {
        _data.Add(val);              // Step 1: Append at end
        HeapifyUp(_data.Count - 1); // Step 2: Bubble up
    }

    private void HeapifyUp(int i)
    {
        while (i > 0)
        {
            int parent = (i - 1) / 2;
            if (_data[i] < _data[parent])  // Violation: child < parent
            {
                Swap(i, parent);
                i = parent;                  // Move up
            }
            else break;                    // Heap property restored
        }
    }

    // ── EXTRACT MIN (Heapify Down) ───────────────────────────────
    public int Pop()
    {
        if (_data.Count == 0) throw new InvalidOperationException("Heap is empty");

        int min  = _data[0];                          // Save min
        int last = _data.Count - 1;
        Swap(0, last);                                 // Move last to root
        _data.RemoveAt(last);                         // Remove old root
        if (_data.Count > 0) HeapifyDown(0);          // Restore heap
        return min;
    }

    private void HeapifyDown(int i)
    {
        int n = _data.Count;
        while (true)
        {
            int smallest = i;
            int left  = 2 * i + 1;
            int right = 2 * i + 2;

            if (left  < n && _data[left]  < _data[smallest]) smallest = left;
            if (right < n && _data[right] < _data[smallest]) smallest = right;

            if (smallest == i) break;   // Already smallest — stop
            Swap(i, smallest);
            i = smallest;
        }
    }

    private void Swap(int a, int b) =>
        (_data[a], _data[b]) = (_data[b], _data[a]);

    // ── HEAP SORT using this heap ─────────────────────────────────
    public static int[] HeapSort(int[] arr)
    {
        var heap = new MinHeap();
        foreach (var v in arr) heap.Push(v);
        var sorted = new int[arr.Length];
        for (int i = 0; i < sorted.Length; i++) sorted[i] = heap.Pop();
        return sorted;  // O(n log n) time, O(n) space
    }
}

// ── C# Built-in PriorityQueue<TElement, TPriority> (.NET 6+) ────
class Program
{
    static void Main()
    {
        // ── Manual MinHeap demo ──────────────────────────────────
        var heap = new MinHeap();
        foreach (int v in new[] { 5, 3, 7, 1, 9, 2 })
            heap.Push(v);

        Console.Write("Extract min order: ");
        while (heap.Count > 0)
            Console.Write(heap.Pop() + " ");  // 1 2 3 5 7 9
        Console.WriteLine();

        // ── Heap sort ────────────────────────────────────────────
        int[] sorted = MinHeap.HeapSort(new[] { 64, 25, 12, 22, 11 });
        Console.WriteLine("HeapSort: " + string.Join(", ", sorted));

        // ── Built-in PriorityQueue<TElement, TPriority> (min-heap) ─
        // Enqueue(element, priority) — dequeues element with LOWEST priority first
        var pq = new PriorityQueue<string, int>();
        pq.Enqueue("Low",      10);
        pq.Enqueue("Critical", 1);
        pq.Enqueue("Medium",   5);
        pq.Enqueue("High",     2);

        Console.Write("PriorityQueue (by priority): ");
        while (pq.Count > 0)
            Console.Write(pq.Dequeue() + " ");  // Critical High Medium Low
        Console.WriteLine();

        // MAX-heap simulation: negate the priority
        var maxPQ = new PriorityQueue<int, int>();
        foreach (int v in new[] { 3, 1, 4, 1, 5 })
            maxPQ.Enqueue(v, -v);  // Negate → lowest (-5) = highest priority
        Console.Write("Max-heap (negated): ");
        while (maxPQ.Count > 0)
            Console.Write(maxPQ.Dequeue() + " ");  // 5 4 3 1 1
        Console.WriteLine();
    }
}

Section 6 — Complete Python Implementation
Python 
import heapq
from typing import List

# ── Manual MinHeap ────────────────────────────────────────────────
class MinHeap:
    def __init__(self): self._data: List[int] = []

    @property
    def size(self) -> int: return len(self._data)

    def peek(self) -> int:
        if not self._data: raise IndexError("Heap is empty")
        return self._data[0]

    def push(self, val: int) -> None:
        self._data.append(val)
        self._heapify_up(len(self._data) - 1)

    def _heapify_up(self, i: int) -> None:
        while i > 0:
            parent = (i - 1) // 2
            if self._data[i] < self._data[parent]:
                self._data[i], self._data[parent] = self._data[parent], self._data[i]
                i = parent
            else: break

    def pop(self) -> int:
        if not self._data: raise IndexError("Heap is empty")
        self._data[0], self._data[-1] = self._data[-1], self._data[0]
        val = self._data.pop()
        if self._data: self._heapify_down(0)
        return val

    def _heapify_down(self, i: int) -> None:
        n = len(self._data)
        while True:
            smallest, l, r = i, 2*i+1, 2*i+2
            if l < n and self._data[l] < self._data[smallest]: smallest = l
            if r < n and self._data[r] < self._data[smallest]: smallest = r
            if smallest == i: break
            self._data[i], self._data[smallest] = self._data[smallest], self._data[i]
            i = smallest


# ── Python heapq module (min-heap) ───────────────────────────────
# heapq always maintains a min-heap over a regular Python list

if __name__ == "__main__":
    # Manual MinHeap
    h = MinHeap()
    for v in [5, 3, 7, 1, 9, 2]: h.push(v)
    print("Manual heap:", [h.pop() for _ in range(h.size + 6)])  # [1,2,3,5,7,9]

    # heapq min-heap
    nums = [5, 3, 7, 1, 9, 2]
    heapq.heapify(nums)             # Build heap in-place: O(n)
    print("heapq peep:", nums[0])   # 1 — always min at index 0

    sorted_vals = []
    while nums:
        sorted_vals.append(heapq.heappop(nums))
    print("Heap sort:", sorted_vals)  # [1, 2, 3, 5, 7, 9]

    # Max-heap in Python: negate values
    max_heap: List[int] = []
    for v in [3, 1, 4, 5, 9]:
        heapq.heappush(max_heap, -v)   # Push negated
    print("Max-heap order:", [-heapq.heappop(max_heap) for _ in range(len(max_heap)+5)])
    # [9, 5, 4, 3, 1]

    # Priority queue with (priority, item) tuples
    tasks: List = []
    heapq.heappush(tasks, (3, "write report"))
    heapq.heappush(tasks, (1, "fix critical bug"))
    heapq.heappush(tasks, (2, "review PR"))
    print("Tasks by priority:")
    while tasks:
        priority, task = heapq.heappop(tasks)
        print(f"  [{priority}] {task}")

# Output:
# Manual heap: [1, 2, 3, 5, 7, 9]
# heapq peek: 1
# Heap sort: [1, 2, 3, 5, 7, 9]
# Max-heap order: [9, 5, 4, 3, 1]
# Tasks by priority:
#   [1] fix critical bug

Section 8 — Real-World Usage
🌍 Dijkstra's Shortest Path Algorithm
Dijkstra maintains a min-heap of (distance, vertex) pairs. At each step, extract the vertex with the minimum distance, relax its edges, and push updated distances. Without a heap: O(V²). With a heap: O((V+E) log V). C#: use PriorityQueue<int, int>. Python: use heapq with (dist, node) tuples.
🌍 OS Task Scheduling — Thread Priorities
Operating system thread schedulers use priority queues backed by heaps. The thread with the highest priority (or smallest deadline in EDF scheduling) is always at the root and executes next. Linux's CFS uses an RB-tree for O(1) min, but many real-time OSes use min-heaps.
🌍 K-way Merge — Merge K Sorted Lists
Merging k sorted arrays: initialize a min-heap with the first element from each array. Extract the minimum, add the next element from the same array. This processes n total elements in O(n log k) time — much better than O(nk) naive merge.
Section 9 — Common Mistakes
⚠️ Mistake 1 — Max-heap in Python with heapq: Python's heapq is ALWAYS a min-heap. To simulate a max-heap, negate all values before pushing: heapq.heappush(h, -val) and negate when popping: -heapq.heappop(h). Forgetting this negation gives you min-order when you expect max-order.
⚠️ Mistake 2 — Using 1-indexed formulas with 0-indexed arrays: Some textbooks use 1-indexed arrays (root at index 1). Formulas: left = 2i, right = 2i+1, parent = i/2. If you use 0-indexed (root at 0), the formulas change: left = 2i+1, right = 2i+2, parent = (i-1)/2. Mixing the two causes incorrect heap operations.
⚠️ Mistake 3 — HeapifyDown comparing against wrong child: Always pick the SMALLER child to swap with in a min-heap (not just the left child). If you always go left, you may swap with a child that's larger than the right child, violating the heap property.
⚠️ Mistake 4 — C# PriorityQueue priority semantics: C#'s PriorityQueue<TElement, TPriority> dequeues the element with the lowest priority value first (min-heap). Priority 1 comes before Priority 10. For a max-heap behavior, negate the priority (enqueue with priority -value).
Section 10 — Check Your Understanding
Q1. Given the array [4, 8, 6, 9, 7, 5], is this a valid min-heap? If not, which element violates the property?
💡 Check: parent at index i must be ≤ both children at 2i+1 and 2i+2.
Yes, it is a valid min-heap.
Index 0 (4): children = index 1 (8) and 2 (6). 4 ≤ 8 ✓, 4 ≤ 6 ✓
Index 1 (8): children = index 3 (9) and 4 (7). 8 ≤ 9 ✓, 8 ≤ 7? NO — 8 > 7 ✗!
Correction: [4, 8, 6, 9, 7, 5] is NOT a valid min-heap. Index 1 (value 8) has a right child at index 4 (value 7), and 8 > 7 violates the min-heap property (parent must be ≤ both children).
Q2. You insert 5 elements into an empty heap one-by-one. What is the total number of comparisons in the worst case?
💡 Each insert does at most log₂(current_size) comparisons in heapify up.
At most ⌊log₂ 1⌋ + ⌊log₂ 2⌋ + ⌊log₂ 3⌋ + ⌊log₂ 4⌋ + ⌊log₂ 5⌋ = 0+1+1+2+2 = 6 comparisons.
Insert 1st element (size 0→1): 0 comparisons (no parent).
Insert 2nd (size 1→2): up to 1 comparison.
Insert 3rd (size 2→3): up to 1.
Insert 4th (size 3→4): up to 2.
Insert 5th (size 4→5): up to 2.
Total worst case = 6 comparisons.
Q3. Why is building a heap from an array in O(n) faster than inserting n elements one-by-one (O(n log n))?
💡 Think about which elements need more sifting down vs. which need less.
When building a heap by calling heapify-down on all internal nodes (starting from the last internal node going to root), elements near the bottom of the tree have very small subtrees and need at most 1–2 swaps. Only the root needs up to log n swaps. The total work is bounded by the sum ∑(h × n/2^(h+1)) over all heights h, which converges to O(n). By contrast, when inserting elements one-by-one, the first half of inserts into a nearly-full heap all need ~log n swaps, giving O(n log n) total.
Q4. In C#, how do you use PriorityQueue<T,P> as a max-heap (highest numeric priority dequeued first)?
💡 PriorityQueue dequeues the LOWEST priority value first by default.
Negate the priority value: Instead of pq.Enqueue(element, priority), use pq.Enqueue(element, -priority). A priority of 100 becomes -100, which is now the "lowest" value and gets dequeued first. When reading back, you can ignore the negation since you're only using the element. Example: pq.Enqueue("High", -100); pq.Enqueue("Low", -1); — "High" dequeues first.
Q5. You have a heap of 1000 elements. How many comparisons does extract-min take in the worst case?
💡 HeapifyDown starts at the root and goes down one level at a time.
At most 2 × ⌊log₂ 1000⌋ = 2 × 9 = 18 comparisons. HeapifyDown: at each level, you compare the parent with both left and right children (2 comparisons per level), then move down at most ⌊log₂ n⌋ levels. For n=1000: ⌊log₂ 1000⌋ = 9 levels. So at most 9 × 2 = 18 comparisons (though in practice often fewer since the sink may stop early).
