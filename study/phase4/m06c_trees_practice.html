Problems 1–5 — Easy: Core Traversals & Properties
Problem 1 — Maximum Depth of Binary Tree
EasyDFSRecursion
Given the root of a binary tree, return its maximum depth (number of nodes along the longest path from root to the farthest leaf node).
Constraints: 0 ≤ nodes ≤ 10⁴ | −100 ≤ Node.val ≤ 100
Approach: At each node, the depth is 1 + max(depth of left subtree, depth of right subtree). Base case: null node → return 0. Single recursive call — clean and minimal.
C# 
public int MaxDepth(TreeNode? root)
{
    if (root == null) return 0;   // Empty tree → depth 0
    int left  = MaxDepth(root.Left);
    int right = MaxDepth(root.Right);
    return 1 + Math.Max(left, right);  // 1 for current node + deeper child
}
// Input: [3,9,20,null,null,15,7]  Output: 3
Python 
def maxDepth(root: Optional[TreeNode]) -> int:
    if not root: return 0
    return 1 + max(maxDepth(root.left), maxDepth(root.right))
Time: O(n) — visit every node once  |  Space: O(h) — call stack depth = tree height
Problem 2 — Symmetric Tree
EasyDFSMirror
Given the root of a binary tree, check whether it is a mirror of itself (i.e., symmetric around its center).
Constraints: 1 ≤ nodes ≤ 1000
Approach: Two trees are mirrors if: their roots are equal AND the left subtree of one mirrors the right subtree of the other. Use a helper isMirror(left, right) that checks left.val == right.val AND isMirror(left.left, right.right) AND isMirror(left.right, right.left).
C# 
public bool IsSymmetric(TreeNode? root) =>
    IsMirror(root?.Left, root?.Right);

private bool IsMirror(TreeNode? left, TreeNode? right)
{
    if (left == null && right == null) return true;   // Both empty = symmetric
    if (left == null || right == null) return false;  // One empty = asymmetric
    return left.Val == right.Val                         // Values match AND...
        && IsMirror(left.Left,  right.Right)              // outer edges match AND...
        && IsMirror(left.Right, right.Left);              // inner edges match
}
Python 
def isSymmetric(root: Optional[TreeNode]) -> bool:
    def mirror(left, right):
        if not left and not right: return True
        if not left or  not right: return False
        return (left.val == right.val
                and mirror(left.left,  right.right)
                and mirror(left.right, right.left))
    return mirror(root.left, root.right)
Time: O(n)  |  Space: O(h)
Problem 3 — Path Sum
EasyDFSPath
Given the root of a binary tree and an integer targetSum, return true if there is a root-to-leaf path such that the sum of all node values equals targetSum.
Constraints: 0 ≤ nodes ≤ 5000 | −1000 ≤ Node.val, targetSum ≤ 1000
Approach: Subtract the current node's value from targetSum at each step. At a leaf node, check if targetSum == 0. If yes, we found our path. Key: check "is this a leaf?" with left == null && right == null, not just node == null.
C# 
public bool HasPathSum(TreeNode? root, int targetSum)
{
    if (root == null) return false;      // Empty subtree: no path
    targetSum -= root.Val;               // Subtract current node's value
    if (root.Left == null && root.Right == null)
        return targetSum == 0;          // Leaf: did we hit the target?
    return HasPathSum(root.Left,  targetSum) // Check left subtree
        || HasPathSum(root.Right, targetSum); // OR right subtree
}
Python 
def hasPathSum(root: Optional[TreeNode], targetSum: int) -> bool:
    if not root: return False
    targetSum -= root.val
    if not root.left and not root.right:
        return targetSum == 0
    return hasPathSum(root.left, targetSum) or hasPathSum(root.right, targetSum)
Time: O(n)  |  Space: O(h)
Problem 4 — Invert Binary Tree
EasyDFSClassic
Given the root of a binary tree, invert the tree (mirror it left-to-right), and return its root. This is the famous "Google Homebrew" interview question.
Constraints: 0 ≤ nodes ≤ 100 | −100 ≤ Node.val ≤ 100
Approach: At each node, swap left and right children, then recursively invert each child subtree. The recursion naturally handles all levels. Two lines of code once you see the pattern.
C# 
public TreeNode? InvertTree(TreeNode? root)
{
    if (root == null) return null;
    (root.Left, root.Right) = (InvertTree(root.Right), InvertTree(root.Left));
    return root;
    // C# tuple swap: invert right first, then left, swap them in one line
}
Python 
def invertTree(root: Optional[TreeNode]) -> Optional[TreeNode]:
    if not root: return None
    root.left, root.right = invertTree(root.right), invertTree(root.left)
    return root
Time: O(n) — visit every node once  |  Space: O(h)
Problem 5 — Level-Order Zigzag Traversal
MediumBFSDeque
Given the root of a binary tree, return the zigzag level-order traversal: level 0 left-to-right, level 1 right-to-left, level 2 left-to-right, and so on.
Constraints: 0 ≤ nodes ≤ 2000 | −100 ≤ Node.val ≤ 100
Approach: Standard BFS level-order, but alternate the direction of appending values per level. Use a boolean flag leftToRight — flip it after each level. For right-to-left levels, either reverse the list or insert at front using a deque/LinkedList.
C# 
public IList<IList<int>> ZigzagLevelOrder(TreeNode? root)
{
    var result = new List<IList<int>>();
    if (root == null) return result;

    var queue = new Queue<TreeNode>();
    queue.Enqueue(root);
    bool leftToRight = true;

    while (queue.Count > 0)
    {
        int size = queue.Count;
        var level = new LinkedList<int>();  // Allows O(1) add at front AND back

        for (int i = 0; i < size; i++)
        {
            var node = queue.Dequeue();
            if (leftToRight) level.AddLast(node.Val);   // Normal: add at end
            else             level.AddFirst(node.Val);  // Zigzag: add at front
            if (node.Left  != null) queue.Enqueue(node.Left);
            if (node.Right != null) queue.Enqueue(node.Right);
        }
        result.Add(new List<int>(level));
        leftToRight = !leftToRight;          // Flip direction for next level
    }
    return result;
}
Python 
from collections import deque

def zigzagLevelOrder(root: Optional[TreeNode]) -> List[List[int]]:
    if not root: return []
    result, q = [], deque([root])
    left_to_right = True
    while q:
        level = deque()
        for _ in range(len(q)):
            node = q.popleft()
            if left_to_right: level.append(node.val)
            else:             level.appendleft(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
        result.append(list(level))
        left_to_right = not left_to_right
    return result
Time: O(n)  |  Space: O(w) where w = maximum tree width
Problems 6–10 — Medium: Classic Tree Patterns
Problem 6 — Validate Binary Search Tree
MediumBSTBounds
Given the root of a binary tree, determine if it is a valid Binary Search Tree. A valid BST: left subtree contains ONLY nodes with values less than the node's value, right subtree contains ONLY nodes greater than the node's value, and both subtrees are themselves BSTs.
Constraints: 1 ≤ nodes ≤ 10⁴ | −2³¹ ≤ Node.val ≤ 2³¹−1
Approach: Pass min and max bounds down recursively. At each node: value must be strictly between (min, max). For left child: max becomes current node's value. For right child: min becomes current node's value. Initialize with (-∞, +∞) at root.
C# 
public bool IsValidBST(TreeNode? root) =>
    Validate(root, long.MinValue, long.MaxValue);  // Use long to handle int limits

private bool Validate(TreeNode? node, long min, long max)
{
    if (node == null) return true;
    if (node.Val <= min || node.Val >= max) return false;  // Violates bounds
    return Validate(node.Left,  min, node.Val)   // Left: upper bound = current
        && Validate(node.Right, node.Val, max);   // Right: lower bound = current
}
Python 
def isValidBST(root: Optional[TreeNode]) -> bool:
    def validate(node, lo, hi):
        if not node: return True
        if not (lo < node.val < hi): return False
        return validate(node.left, lo, node.val) and validate(node.right, node.val, hi)
    return validate(root, float('-inf'), float('inf'))
Time: O(n)  |  Space: O(h)
💡 Why not just check left.val < root.val < right.val? This misses cases like: root=5, left=3, left.right=7. The local check passes (3 < 5) but 7 is in the left subtree and 7 > 5, violating the BST property.
Problem 7 — Lowest Common Ancestor of BST
MediumBSTLCA
Given a BST and two nodes p and q, find their Lowest Common Ancestor (LCA) — the deepest node that is an ancestor of both p and q.
Constraints: 2 ≤ nodes ≤ 10⁵ | Both p and q exist in the tree
Approach: In a BST, use the ordering property. If both p and q are less than current node → LCA is in left subtree. If both greater → in right subtree. If one on each side (or one equals current) → current node IS the LCA. Much faster than general tree LCA.
C# 
public TreeNode LowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q)
{
    while (root != null)
    {
        if (p.Val < root.Val && q.Val < root.Val)
            root = root.Left;                   // Both left → go left
        else if (p.Val > root.Val && q.Val > root.Val)
            root = root.Right;                  // Both right → go right
        else
            return root;                       // Split point = LCA
    }
    return root!;  // Won't reach here if p,q guaranteed in tree
}
Python 
def lowestCommonAncestor(root: TreeNode, p: TreeNode, q: TreeNode) -> TreeNode:
    node = root
    while node:
        if   p.val < node.val and q.val < node.val: node = node.left
        elif p.val > node.val and q.val > node.val: node = node.right
        else: return node
Time: O(h) — O(log n) balanced, O(n) worst  |  Space: O(1) iterative
Problem 8 — Binary Tree Right Side View
MediumBFSLevel Order
Given the root of a binary tree, imagine yourself standing on the right side. Return the values of the nodes you can see ordered from top to bottom.
Constraints: 0 ≤ nodes ≤ 100 | −100 ≤ Node.val ≤ 100
Approach: Level-order BFS. For each level, the LAST node you process is the rightmost visible node — add it to the result. Process nodes left-to-right so that the last node dequeued per level is always the rightmost.
C# 
public IList<int> RightSideView(TreeNode? root)
{
    var result = new List<int>();
    if (root == null) return result;

    var queue = new Queue<TreeNode>();
    queue.Enqueue(root);

    while (queue.Count > 0)
    {
        int size = queue.Count;
        for (int i = 0; i < size; i++)
        {
            var node = queue.Dequeue();
            if (i == size - 1) result.Add(node.Val);  // Last in level = rightmost
            if (node.Left  != null) queue.Enqueue(node.Left);
            if (node.Right != null) queue.Enqueue(node.Right);
        }
    }
    return result;
}
Python 
def rightSideView(root: Optional[TreeNode]) -> List[int]:
    if not root: return []
    result, q = [], deque([root])
    while q:
        for i in range(len(q)):
            node = q.popleft()
            if i == len(q):       # After popleft, len is size-1 at last node
                result.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
    return result

# Cleaner Python version using level list:
def rightSideViewClean(root: Optional[TreeNode]) -> List[int]:
    if not root: return []
    result, q = [], deque([root])
    while q:
        size = len(q)
        for i in range(size):
            node = q.popleft()
            if i == size - 1: result.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
    return result
Time: O(n)  |  Space: O(w) — max width of tree
Problem 9 — Construct Binary Tree from Preorder and Inorder
MediumTree ConstructionDivide & Conquer
Given two integer arrays preorder and inorder (both representing the same tree), construct and return the binary tree. No duplicates in the tree.
Constraints: 1 ≤ n ≤ 3000 | Both arrays contain same n elements
Approach: Preorder's first element is always the root. Find that value in inorder — everything to its left is the left subtree, everything to its right is the right subtree. Use a HashMap for O(1) inorder index lookup. Recurse on left and right halves of both arrays.
C# 
public TreeNode? BuildTree(int[] preorder, int[] inorder)
{
    // Map inorder values → indices for O(1) lookup
    var inMap = new Dictionary<int, int>();
    for (int i = 0; i < inorder.Length; i++) inMap[inorder[i]] = i;

    int preIdx = 0;  // Global index into preorder array
    return Build(0, inorder.Length - 1);

    TreeNode? Build(int inLeft, int inRight)
    {
        if (inLeft > inRight) return null;   // No elements left

        int rootVal = preorder[preIdx++];      // Next in preorder = current root
        var root = new TreeNode(rootVal);
        int mid = inMap[rootVal];              // Position in inorder

        root.Left  = Build(inLeft,   mid - 1); // Left subtree: inorder[inLeft..mid-1]
        root.Right = Build(mid + 1, inRight);  // Right subtree: inorder[mid+1..inRight]
        return root;
    }
}
Python 
def buildTree(preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
    in_map = {v: i for i, v in enumerate(inorder)}  # val → index
    pre_iter = iter(preorder)                          # Consume preorder in order

    def build(in_left, in_right):
        if in_left > in_right: return None
        root_val = next(pre_iter)
        root = TreeNode(root_val)
        mid = in_map[root_val]
        root.left  = build(in_left,  mid - 1)
        root.right = build(mid + 1, in_right)
        return root

    return build(0, len(inorder) - 1)
Time: O(n) with HashMap  |  Space: O(n) for HashMap + O(h) stack
Problem 10 — Diameter of Binary Tree
MediumDFSPost-order
Given the root of a binary tree, return the length of the diameter — the length of the longest path between any two nodes (path may or may not pass through the root).
Constraints: 1 ≤ nodes ≤ 10⁴ | −100 ≤ Node.val ≤ 100
Approach: At each node, the diameter passing through it = depth(left) + depth(right). Track the global maximum using a reference variable. Return depth (not diameter) upward so parent can compute its own diameter. Use postorder — compute children first.
C# 
public int DiameterOfBinaryTree(TreeNode? root)
{
    int maxDiameter = 0;
    Depth(root);
    return maxDiameter;

    int Depth(TreeNode? node)
    {
        if (node == null) return 0;
        int left  = Depth(node.Left);   // Depth of left subtree
        int right = Depth(node.Right);  // Depth of right subtree
        maxDiameter = Math.Max(maxDiameter, left + right);  // Path through this node
        return 1 + Math.Max(left, right);                  // Return depth upward
    }
}
Python 
def diameterOfBinaryTree(root: Optional[TreeNode]) -> int:
    max_d = [0]  # Use list to allow mutation inside nested function
    def depth(node):
        if not node: return 0
        left, right = depth(node.left), depth(node.right)
        max_d[0] = max(max_d[0], left + right)
        return 1 + max(left, right)
    depth(root)
    return max_d[0]
Time: O(n) — single DFS pass  |  Space: O(h)
💡 Key insight: Return the depth (max height of one path going down), NOT the diameter. The diameter at each node is left_depth + right_depth. We track the global max in an external variable updated at each node.
Problems 11–15 — Hard: Advanced Tree Algorithms
Problem 11 — Binary Tree Maximum Path Sum
HardDFSPath Sum
Given the root of a binary tree, return the maximum path sum. A path goes from any node to any other node through any sequence of parent-child connections. A single node counts as a path. Node values can be negative.
Constraints: 1 ≤ nodes ≤ 3×10⁴ | −1000 ≤ Node.val ≤ 1000
Approach: At each node, compute the max contribution going DOWN (only take a child if it adds positive value). The path through this node = node.val + max(0, left_gain) + max(0, right_gain). Track global max. Return to parent only the one-arm gain: node.val + max(0, max(left_gain, right_gain)) — because a path can't split.
C# 
public int MaxPathSum(TreeNode? root)
{
    int maxSum = int.MinValue;
    Gain(root);
    return maxSum;

    int Gain(TreeNode? node)
    {
        if (node == null) return 0;
        int leftGain  = Math.Max(0, Gain(node.Left));   // Only take if positive
        int rightGain = Math.Max(0, Gain(node.Right));  // Only take if positive
        maxSum = Math.Max(maxSum, node.Val + leftGain + rightGain); // Path THROUGH this node
        return node.Val + Math.Max(leftGain, rightGain);            // Return ONE arm to parent
    }
}
Python 
def maxPathSum(root: Optional[TreeNode]) -> int:
    max_sum = [float('-inf')]
    def gain(node):
        if not node: return 0
        l = max(0, gain(node.left))   # Discard negative contributions
        r = max(0, gain(node.right))
        max_sum[0] = max(max_sum[0], node.val + l + r)
        return node.val + max(l, r)   # Only ONE direction to parent
    gain(root)
    return max_sum[0]
Time: O(n)  |  Space: O(h)
Problem 12 — Serialize and Deserialize Binary Tree
HardBFSSerialization
Design an algorithm to serialize a binary tree to a string and deserialize that string back to the original tree. No specific format required — just make it invertible.
Constraints: 0 ≤ nodes ≤ 10⁴ | −1000 ≤ Node.val ≤ 1000
Approach: Use BFS level-order serialization with null markers ("N"). Serialize: enqueue root, dequeue nodes writing values, enqueue children (or "N" for null children). Deserialize: split by comma, build queue of values, use index tracking to assign children.
C# 
public string Serialize(TreeNode? root)
{
    if (root == null) return "";
    var sb = new System.Text.StringBuilder();
    var queue = new Queue<TreeNode?>();
    queue.Enqueue(root);
    while (queue.Count > 0)
    {
        var node = queue.Dequeue();
        if (node == null) { sb.Append("N,"); continue; }
        sb.Append(node.Val + ",");
        queue.Enqueue(node.Left);
        queue.Enqueue(node.Right);
    }
    return sb.ToString().TrimEnd(',');
}

public TreeNode? Deserialize(string data)
{
    if (string.IsNullOrEmpty(data)) return null;
    var vals = data.Split(',');
    var root = new TreeNode(int.Parse(vals[0]));
    var queue = new Queue<TreeNode>();
    queue.Enqueue(root);
    int i = 1;
    while (i < vals.Length)
    {
        var node = queue.Dequeue();
        if (vals[i] != "N") { node.Left = new TreeNode(int.Parse(vals[i])); queue.Enqueue(node.Left); }
        i++;
        if (i < vals.Length && vals[i] != "N") { node.Right = new TreeNode(int.Parse(vals[i])); queue.Enqueue(node.Right); }
        i++;
    }
    return root;
}
Python 
def serialize(root: Optional[TreeNode]) -> str:
    if not root: return ""
    q, res = deque([root]), []
    while q:
        node = q.popleft()
        if not node: res.append("N")
        else:
            res.append(str(node.val))
            q.append(node.left)
            q.append(node.right)
    return ",".join(res)

def deserialize(data: str) -> Optional[TreeNode]:
    if not data: return None
    vals = data.split(",")
    root = TreeNode(int(vals[0]))
    q, i = deque([root]), 1
    while q and i < len(vals):
        node = q.popleft()
        if vals[i] != "N":
            node.left = TreeNode(int(vals[i]))
            q.append(node.left)
        i += 1
        if i < len(vals) and vals[i] != "N":
            node.right = TreeNode(int(vals[i]))
            q.append(node.right)
        i += 1
    return root
Time: O(n) for both  |  Space: O(n) for the queue and string
Problem 13 — Kth Smallest Element in BST
MediumBSTInorder
Given the root of a BST and an integer k, return the kth smallest value (1-indexed) of all values in the BST.
Constraints: 1 ≤ k ≤ n ≤ 10⁴ | 0 ≤ Node.val ≤ 10⁴
Approach: Inorder traversal of a BST gives sorted ascending order. Count nodes visited — when count reaches k, that's your answer. Use iterative inorder to stop early (no need to traverse the whole tree). Early termination makes this O(h + k) instead of O(n).
C# 
public int KthSmallest(TreeNode? root, int k)
{
    var stack = new Stack<TreeNode>();
    var current = root;

    while (current != null || stack.Count > 0)
    {
        while (current != null) { stack.Push(current); current = current.Left; }
        current = stack.Pop();
        if (--k == 0) return current.Val;  // Decrement k; if 0, this is the kth
        current = current.Right;
    }
    return -1;  // k > tree size (shouldn't happen per constraints)
}
Python 
def kthSmallest(root: Optional[TreeNode], k: int) -> int:
    stack, cur = [], root
    while cur or stack:
        while cur: stack.append(cur); cur = cur.left
        cur = stack.pop()
        k -= 1
        if k == 0: return cur.val
        cur = cur.right
Time: O(h + k) — h to reach leftmost, then k steps  |  Space: O(h)
Problem 14 — Flatten Binary Tree to Linked List
MediumPreorderIn-place
Given the root of a binary tree, flatten it to a linked list in-place. The linked list should follow preorder traversal of the tree, using only right pointers (left pointers all null).
Constraints: 0 ≤ nodes ≤ 2000 | −100 ≤ Node.val ≤ 100 | Must do in-place (O(1) extra space).
Approach: Morris Traversal style. At each node with a left child: find the rightmost node of the left subtree, attach the current right subtree there, then move left subtree to right position. Repeat until no left children remain. O(1) space, no recursion.
C# 
public void Flatten(TreeNode? root)
{
    var curr = root;
    while (curr != null)
    {
        if (curr.Left != null)
        {
            // Find rightmost of left subtree
            var rightmost = curr.Left;
            while (rightmost.Right != null) rightmost = rightmost.Right;

            // Attach curr's right subtree to rightmost's right
            rightmost.Right = curr.Right;

            // Move left subtree to right; clear left
            curr.Right = curr.Left;
            curr.Left  = null;
        }
        curr = curr.Right;  // Move to next node in preorder
    }
}
// Input: 1→(2→(3,4), 5→(null,6))  Output: 1→2→3→4→5→6 (all right links)
Python 
def flatten(root: Optional[TreeNode]) -> None:
    curr = root
    while curr:
        if curr.left:
            rightmost = curr.left
            while rightmost.right: rightmost = rightmost.right
            rightmost.right = curr.right   # Attach right subtree
            curr.right = curr.left         # Left becomes new right
            curr.left  = None
        curr = curr.right
Time: O(n) — each node visited at most twice  |  Space: O(1) — no recursion, no stack
Problem 15 — Recover BST (Two Nodes Swapped)
HardBSTInorderMorris
Two nodes in a BST have been accidentally swapped. Recover the tree without changing its structure (restore the original BST property).
Constraints: 2 ≤ nodes ≤ 1000 | Must do in O(1) space (Morris traversal bonus). Swap back only the VALUES, not the nodes.
Approach: Inorder traversal of a valid BST is strictly increasing. If two nodes are swapped, there are exactly 1 or 2 "inversions" (places where prev.val > curr.val). The first inversion's "prev" node and the last inversion's "curr" node are the two swapped nodes. Swap their values back.
C# 
public void RecoverTree(TreeNode? root)
{
    TreeNode? first = null, second = null, prev = null;

    // Inorder traversal — find the two swapped nodes
    InOrder(root);

    // Swap values back
    if (first != null && second != null)
        (first.Val, second.Val) = (second.Val, first.Val);

    void InOrder(TreeNode? node)
    {
        if (node == null) return;
        InOrder(node.Left);

        if (prev != null && prev.Val > node.Val)
        {
            if (first == null) first = prev;   // First inversion: record prev
            second = node;                       // Always update second (handles adjacent swap)
        }
        prev = node;                             // Advance prev pointer

        InOrder(node.Right);
    }
}
// Example: BST [3,1,4,null,null,2] where 3 and 2 are swapped
// Inorder would be: 1,3,2,4 → inversion at (3,2) → swap 3<→2 → [2,1,4,null,null,3]
Python 
def recoverTree(root: Optional[TreeNode]) -> None:
    first = second = prev = None

    def inorder(node):
        nonlocal first, second, prev
        if not node: return
        inorder(node.left)
        if prev and prev.val > node.val:
            if not first: first = prev   # First violation: candidate 1
            second = node                 # Always update: candidate 2
        prev = node
        inorder(node.right)

    inorder(root)
    first.val, second.val = second.val, first.val  # Restore BST
Time: O(n)  |  Space: O(h) for recursion (O(1) with Morris traversal)
💡 Adjacent vs. Non-adjacent swap: If the two swapped nodes are adjacent in inorder (e.g., [1,3,2,4] — 3 and 2 swapped), there is exactly ONE inversion. If they're non-adjacent (e.g., [1,4,3,2] — 4 and 2 swapped), there are TWO inversions. The algorithm handles both: first is set at the first inversion, second keeps getting updated — so it ends up at the correct second node in both cases.
