1. GPS Navigation — Google Maps / Waze

🌎

Find the Fastest Route Through a City

Google Maps · Waze · Apple Maps · HERE Technologies
Dijkstra's Algorithm A* Search Bidirectional BFS Phase 5 — Graphs Phase 9 — Advanced Graphs Scale: 10⁹ road segments

Google Maps has over 1 billion users requesting routes every day. The road network of a country contains hundreds of millions of intersections (nodes) and roads (edges). A naive Dijkstra would be too slow — real systems use a hierarchy of optimizations.

🔍 Problem Breakdown

The Problem: Find the minimum travel-time path between two intersections in a weighted directed graph with ~300 million nodes and ~1 billion edges.
Naive Approach: BFS ignores edge weights — it minimises hops, not time. Dijkstra is correct but explores every node equally outward in all directions, visiting ~10 million nodes before reaching a distant destination.
Key Insight: A* adds a heuristic h(v) = straight-line distance from v to the destination (always ≤ actual road time, so it's admissible). This guides expansion toward the goal, skipping irrelevant areas of the map. Real systems layer Contraction Hierarchies on top — pre-contracting unimportant nodes into shortcuts reduces queries from millions of node-visits to thousands.
Road Network Graph: ┌─────────────────────────────────────────┐ │ Node = Intersection (lat/lng) │ │ Edge = Road segment (direction, speed) │ │ Weight = travel time (seconds) │ └─────────────────────────────────────────┘ Challenge: London → Edinburgh (700km, millions of nodes in-between) Naive Dijkstra: explores ~10 million nodes → too slow for real-time A* with heuristic: focuses search toward destination → 100x faster Contraction Hierarchies: pre-contract road graph → 1000x faster

Core Algorithm: A* (Dijkstra + Heuristic)

A* extends Dijkstra by adding a heuristic function — an estimate of the remaining distance to the goal (usually straight-line distance). This focuses the search toward the destination and drastically reduces nodes explored.

public class AStarRouter
{
    // Node = intersection, heuristic = straight-line distance to target
    public List<int> FindRoute(int src, int dst,
        List<(int to, double time)>[] adj, double[] lat, double[] lng)
    {
        int n = adj.Length;
        double[] gScore = Enumerable.Repeat(double.MaxValue, n).ToArray();
        int[] prev = Enumerable.Repeat(-1, n).ToArray();
        gScore[src] = 0;

        // Priority queue sorted by f = g (actual) + h (heuristic)
        var pq = new PriorityQueue<int, double>();
        pq.Enqueue(src, Heuristic(src, dst, lat, lng));

        while (pq.Count > 0)
        {
            pq.TryDequeue(out int u, out _);
            if (u == dst) return ReconstructPath(prev, src, dst);

            foreach (var (v, w) in adj[u])
            {
                double tentative = gScore[u] + w;
                if (tentative < gScore[v])
                {
                    gScore[v] = tentative;
                    prev[v] = u;
                    // f = actual cost + straight-line estimate to destination
                    pq.Enqueue(v, tentative + Heuristic(v, dst, lat, lng));
                }
            }
        }
        return new List<int>(); // No path
    }

    private double Heuristic(int a, int b, double[] lat, double[] lng)
    {
        // Haversine distance (straight-line on Earth's surface)
        double dlat = lat[b] - lat[a], dlng = lng[b] - lng[a];
        return Math.Sqrt(dlat*dlat + dlng*dlng) * 111000; // rough meters
    }
}

import heapq

class AStarRouter:
    # Node = intersection, heuristic = straight-line distance to target
    def find_route(self, src, dst, adj, lat, lng):
        n = len(adj)
        g_score = [float('inf')] * n
        prev = [-1] * n
        g_score[src] = 0
        # Priority queue: (f = g + h, node)
        pq = [(self._heuristic(src, dst, lat, lng), src)]
        while pq:
            _, u = heapq.heappop(pq)
            if u == dst:
                return self._reconstruct_path(prev, src, dst)
            for v, w in adj[u]:
                tentative = g_score[u] + w
                if tentative < g_score[v]:
                    g_score[v] = tentative
                    prev[v] = u
                    # f = actual cost + straight-line estimate to destination
                    f = tentative + self._heuristic(v, dst, lat, lng)
                    heapq.heappush(pq, (f, v))
        return []  # No path

        # Haversine distance (straight-line on Earth's surface)
        dlat = lat[b] - lat[a]
        dlng = lng[b] - lng[a]

📖 Solution Walkthrough

1. Initialise: g[src] = 0, all others = ∞. Push src onto a min-heap with priority f = h(src, dst) (straight-line estimate to destination).
2. Main loop: Pop node u with the lowest f-score. If u == dst, path is found — trace back through prev[] to reconstruct it.
3. Relaxation: For each road u→v with travel time w: tentative = g[u] + w. If tentative < g[v]: update g[v], set prev[v] = u, enqueue v with f = tentative + h(v, dst).
4. Heuristic (Haversine): Straight-line Earth-surface distance — always ≤ actual road distance. This admissibility guarantee means A* never overshoots and always finds the optimal path.
5. Reconstruction: Follow prev[] backward from dst to src, then reverse the list.
⏱ Time: O(E log V) worst-case — in practice far fewer edges are relaxed because the heuristic skips unpromising nodes.
💾 Space: O(V) for g[], prev[], and the priority queue.
Why not Dijkstra? Dijkstra expands nodes in concentric circles from the source. A* stretches that frontier toward the destination — provably correct because h is admissible.
Real-World Scale: Google Maps' road network has ~300 million nodes. Standard Dijkstra explores all of them. Contraction Hierarchies reduce this to ~10,000 nodes per query. Live traffic updates modify edge weights — the system re-runs routing every few seconds for active navigations.

2. Social Network — LinkedIn / Facebook

👥

"People You May Know" — Connection Recommendations

LinkedIn · Facebook · Twitter · Instagram
BFS (Degree of Separation) Union-Find Hash Maps Phase 5 — Graphs Phase 3 — Hash Tables Scale: LinkedIn has 950M users

LinkedIn's "People You May Know" finds users who share mutual connections. This is a classic graph problem: find all nodes within 2 hops from the current user. At LinkedIn's scale, the social graph has billions of edges.

🔍 Problem Breakdown

The Problem: For a given user with D direct connections, find all 2nd-degree connections (friends-of-friends not yet directly connected) and rank them by number of shared mutual friends.
Naive Approach: Full BFS to depth 2 works but visits every node in the 2-hop neighbourhood once. For LinkedIn (950M users, avg 200 connections each), a single BFS per page load = 200 × 200 = 40,000 candidates — but BFS has queue overhead. There's a simpler way.
Key Insight: Skip BFS entirely. Iterate over each friend's connection list directly. Every person you reach this way is a friend-of-friend. Count how many different friends led to the same candidate — that count is the mutual connection score. O(D²) where D = average degree, with no queue needed.

public int DegreesOfSeparation(int a, int b, HashSet<int>[] graph)
{
    if (a == b) return 0;
    var visited = new HashSet<int> { a };
    var q = new Queue<int>();
    q.Enqueue(a);
    int degree = 0;
    while (q.Count > 0)
    {
        degree++;
        int sz = q.Count;
        while (sz-- > 0)
        {
            int u = q.Dequeue();
            foreach (int v in graph[u])
            {
                if (v == b) return degree;
                if (visited.Add(v)) q.Enqueue(v);
            }
        }
    }
    return -1; // Not connected
}

def degrees_of_separation(a, b, graph):
    if a == b: return 0
    visited = {a}
    q = deque([a])
    degree = 0
    while q:
        degree += 1
        for _ in range(len(q)):
            u = q.popleft()
            for v in graph[u]:
                if v == b: return degree
                if v not in visited:
                    visited.add(v); q.append(v)
    return -1  # Not connected

1. Store direct connections in a HashSet for O(1) membership checks ("is this person already connected?").
2. For each 1st-degree friend, iterate their connection list — these are the 2nd-degree candidates.
3. Skip if the candidate is the user themselves or already in the direct set.
4. Each time the same candidate appears via a different mutual friend, increment their counter. This count = number of shared connections = recommendation strength.

3. Package Manager — npm / NuGet / Maven

📦

Dependency Resolution — Install Packages in Correct Order

npm · NuGet · Maven · Cargo · pip
Topological Sort Cycle Detection DAG Phase 5 — Graph Algorithms Phase 9 — Advanced Graphs npm registry: 2.5M packages

When you run dotnet add package Newtonsoft.Json, the package manager must install all dependencies in the correct order — a package must be installed after all its dependencies. This is exactly topological sort. If a circular dependency exists, installation must fail with an error.

🔍 Problem Breakdown

The Problem: Given a dependency graph (directed: edge A→B means A depends on B), produce a valid install order where every package appears only after all of its dependencies have been installed. Detect cycles and fail with a clear error.
Naive Approach: Try a random install order, retry when it fails. With n packages this can take O(n!) tries. Recursive "install dependencies first" can revisit the same package exponentially when many packages share a dependency.
Key Insight: Model as a Directed Acyclic Graph (DAG). Kahn's topological sort tracks in-degree (how many uninstalled dependencies each package still has). Packages with in-degree 0 are ready to install. Installing one decrements the in-degree of packages that depended on it — if any reach 0, they become ready. Cycle detection is automatic: if the installed count < total packages, some in-degrees never reached 0.

public class PackageInstaller
{
    // Returns install order or throws if circular dependency exists
    public List<string> ResolveOrder(Dictionary<string, List<string>> deps)
    {
        var inDegree = new Dictionary<string, int>();
        foreach (var pkg in deps.Keys) inDegree[pkg] = 0;
        foreach (var (_, depList) in deps)
            foreach (string dep in depList)
                inDegree[dep] = inDegree.GetValueOrDefault(dep) + 1;

        // Kahn's Algorithm — packages with no remaining dependencies install first
        var ready = new Queue<string>(inDegree.Where(kv => kv.Value == 0).Select(kv => kv.Key));
        var order = new List<string>();

        while (ready.Count > 0)
        {
            string pkg = ready.Dequeue();
            order.Add(pkg);
            foreach (string dependant in deps.GetValueOrDefault(pkg, new()))
                if (--inDegree[dependant] == 0) ready.Enqueue(dependant);
        }

        if (order.Count != deps.Count)
            throw new InvalidOperationException("Circular dependency detected!");
        return order;
    }
}

from collections import deque

class PackageInstaller:
    # Returns install order or raises if circular dependency exists
    def resolve_order(self, deps):
        in_degree = {pkg: 0 for pkg in deps}
        for dep_list in deps.values():
            for dep in dep_list:
                in_degree[dep] = in_degree.get(dep, 0) + 1

        # Kahn's Algorithm — packages with no remaining dependencies install first
        ready = deque(pkg for pkg, d in in_degree.items() if d == 0)
        order = []
        while ready:
            pkg = ready.popleft()
            order.append(pkg)
            for dependant in deps.get(pkg, []):
                in_degree[dependant] -= 1
                if in_degree[dependant] == 0:
                    ready.append(dependant)

        if len(order) != len(deps):
            raise ValueError("Circular dependency detected!")
        return order

📖 Solution Walkthrough

1. Build in-degree map: For every package, count how many others list it as a dependency. in_degree[X] = number of packages that must be installed before X.
2. Seed queue: Any package with in-degree 0 has no dependencies → add to the install queue immediately.
3. Process queue: Dequeue a package, add to the install order. For each package that depended on it, decrement their in-degree. If any reach 0, enqueue them.
4. Repeat until queue is empty. If installed count < total packages → a cycle exists (some in-degrees never reached 0 → those packages are stuck waiting on each other).
⏱ Time: O(V + E) — every node and every edge is processed exactly once.
💾 Space: O(V) for the in-degree map and the queue.
Why Kahn's over DFS? DFS topological sort also works but Kahn's naturally surfaces cycles (remaining unprocessed nodes) and is iterative — no recursion stack overflow on large dependency trees.

Google · Bing · Elasticsearch · Algolia

Google's search box has three algorithmic layers working together: Trie for instant prefix autocomplete, Edit Distance DP for "Did you mean X?" spell correction, and Inverted Index (hash map from word → documents) for finding matching pages.

🔍 Problem Breakdown

The Problem: Three separate challenges: (a) given a typed prefix, return the top-K most-searched completions instantly; (b) given a misspelled query, identify the closest known query; (c) given a multi-word query, find all documents containing those words and rank them.
Naive Approach: (a) Scan all known queries for prefix match = O(n); (b) compute edit distance against every word in the vocabulary = O(n × m × k); (c) scan every word in every document for each query word = O(D × W) per word. All three are too slow at Google's scale.
Key Insight: Pre-build three specialised structures: a Trie reduces prefix lookup to O(prefix_length). An optimised Edit Distance DP uses the space-efficient rolling-array approach and only computes against a curated vocabulary. The Inverted Index — built at crawl time — turns document search into a hash lookup + set intersection.

public int EditDistance(string s, string t)
{

        int[] curr = new int[n + 1];
        curr[0] = i;
        for (int j = 1; j <= n; j++)

        prev = curr;
    }

5. Compiler / IDE — Visual Studio / Roslyn

🛠

Expression Evaluation, Syntax Checking & Build Ordering

Roslyn (.NET) · JVM · Clang · GCC · IntelliJ
Stack (Expression Parsing) Trie (Symbol Table) Topological Sort (Build) Interval DP (Regex) Phase 2 — Stacks Phase 4 — Trie Phase 9 — Graphs Roslyn: compiles millions of lines/sec

When you write int x = (3 + 4) * 2 - 1;, the C# compiler uses a stack to parse and evaluate that expression. The Roslyn compiler also uses a Trie-based symbol table for name resolution and topological sort for build project ordering.

🔍 Problem Breakdown

The Problem: Given an arithmetic expression string with numbers, operators (+−×÷), parentheses, and precedence rules (× before +), compute the correct result. Parentheses override precedence.
Naive Approach: Evaluating left-to-right without precedence gives wrong answers: "2 + 3 × 4" evaluated as (2+3)×4 = 20 instead of the correct 2+(3×4) = 14. Building a full AST (Abstract Syntax Tree) then evaluating it works but requires two passes.
Key Insight: Dijkstra's Shunting-Yard algorithm solves this in a single pass with two stacks — one for operands, one for operators. When an operator arrives, first flush all pending operators of equal or higher precedence from the stack (applying them immediately), then push the new operator. Parentheses act as precedence barriers. This naturally produces correct evaluation order.
C# 
// Evaluate arithmetic expression: "3 + 4 * 2 - 1" → 10
// Uses two stacks: one for numbers, one for operators
public int Evaluate(string expr)
{
    var nums = new Stack<int>();
    var ops  = new Stack<char>();
    var prec = new Dictionary<char, int> { {'+',1},{'-',1},{'*',2},{'/',2} };

    void Apply() {
        int b = nums.Pop(), a = nums.Pop();
        nums.Push(ops.Pop() switch { '+' => a+b, '-' => a-b, '*' => a*b, _ => a/b });
    }

    int i = 0;
    while (i < expr.Length)
    {
        char c = expr[i];
        if (c == ' ') { i++; continue; }
        if (char.IsDigit(c)) {
            int num = 0;
            while (i < expr.Length && char.IsDigit(expr[i])) num = num*10 + expr[i++]-'0';
            nums.Push(num); continue;
        }
        if (c == '(') { ops.Push(c); }
        else if (c == ')') { while (ops.Peek() != '(') Apply(); ops.Pop(); }
        else if (prec.ContainsKey(c)) {
            while (ops.Count > 0 && ops.Peek() != '(' && prec.GetValueOrDefault(ops.Peek()) >= prec[c])
                Apply();
            ops.Push(c);
        }
        i++;
    }
    while (ops.Count > 0) Apply();
    return nums.Pop();
}

# Evaluate arithmetic expression: "3 + 4 * 2 - 1" → 10
# Uses two stacks: one for numbers, one for operators
def evaluate(expr):
    nums, ops = [], []
    prec = {'+': 1, '-': 1, '*': 2, '/': 2}

    def apply():
        b, a = nums.pop(), nums.pop()
        op = ops.pop()
        if   op == '+': nums.append(a + b)
        elif op == '-': nums.append(a - b)
        elif op == '*': nums.append(a * b)
        else:           nums.append(int(a / b))

    i = 0
    while i < len(expr):
        c = expr[i]
        if c == ' ': i += 1; continue
        if c.isdigit():
            num = 0
            while i < len(expr) and expr[i].isdigit():
                num = num * 10 + int(expr[i]); i += 1
            nums.append(num); continue
        if   c == '(': ops.append(c)
        elif c == ')':
            while ops[-1] != '(': apply()
            ops.pop()
        elif c in prec:
            while ops and ops[-1] != '(' and prec.get(ops[-1], 0) >= prec[c]:
                apply()
            ops.append(c)
        i += 1
    while ops: apply()
    return nums[0]
# This is the Shunting-Yard algorithm — how every compiler/interpreter works
📖 Solution Walkthrough

1. Numbers: Push immediately onto the nums stack.
2. Operator (+−×÷): Before pushing, inspect the top of the ops stack. While the top has equal or higher precedence AND is not an open parenthesis, pop it and apply it (pop two nums, compute result, push result back). Then push the new operator.
3. Open parenthesis '(': Push directly — it blocks lower-precedence operators from being flushed through it.
4. Close parenthesis ')': Keep applying operators from the ops stack until '(' is found. Pop and discard the '('.
5. End of expression: Apply all remaining operators in the ops stack left-to-right.
Example — "3 + 4 × 2": Push 3. Push '+'. Push 4. See '×' (higher than '+') — push '×' without flushing '+'. Push 2. End: apply '×' first (4×2=8), then '+' (3+8=11). ✓
⏱ Time: O(n) — each token is pushed and popped at most once from each stack.
💾 Space: O(n) — both stacks hold at most O(n) elements for deeply nested expressions.

An exchange's order book matches buyers and sellers. Buy orders are sorted by descending price (highest bid wins), sell orders by ascending price (lowest ask wins). When the best bid ≥ best ask, a trade executes. This uses two priority queues (max-heap for bids, min-heap for asks).

🔍 Problem Breakdown

The Problem: Maintain two sorted collections of live orders — bids (sorted highest-first) and asks (sorted lowest-first). On each incoming order, check for a match: a buy order at price P matches any ask priced ≤ P; a sell order at price P matches any bid priced ≥ P. Fill as many shares as possible (partial fills allowed), then add the remainder to the book.
Naive Approach: An array requires O(n log n) sort on every insert and O(n) scan for matching — catastrophic at NASDAQ's 30 billion orders/day.
Key Insight: Two priority queues provide O(log n) insert/extract. A max-heap for bids always surfaces the best (highest) bid at the top. A min-heap for asks always surfaces the cheapest ask. Each match cycle is O(log n) — the fill loop runs only for actual fills, not for the full book.

class OrderBook:
    def __init__(self):
        # Bids: max-heap via negated price (highest price first)
        self._bids = []  # (-price, qty, id)
        # Asks: min-heap (lowest price first)
        self._asks = []  # (price, qty, id)

    def add_order(self, order):
        trades = []
        if order['is_buy']:
            # Try to match against asks (sell orders at or below bid price)
            while order['qty'] > 0 and self._asks and self._asks[0][0] <= order['price']:
                ask_price, ask_qty, ask_id = heapq.heappop(self._asks)
                filled = min(order['qty'], ask_qty)
                trades.append({'price': ask_price, 'qty': filled})
                order['qty'] -= filled; ask_qty -= filled
                if ask_qty > 0:  # Push remainder back
                    heapq.heappush(self._asks, (ask_price, ask_qty, ask_id))
            if order['qty'] > 0:
                # Rest goes to book (negate price for max-heap)
                heapq.heappush(self._bids, (-order['price'], order['qty'], order['id']))
        # Symmetric for sell orders...
        return trades

Buy order arrives at price P, quantity Q:
1. Peek the min-ask. If ask_price ≤ P, a trade is possible at the ask price (sellers' guaranteed price).
2. Compute fill_qty = min(order.qty, ask.qty). Record the trade at ask_price. Decrement both quantities.
3. If the ask is fully filled, dequeue it. If partially filled, push the remainder back to the min-heap.
4. Repeat while ask_price ≤ P and remaining order quantity > 0.
5. Any unfilled remainder: enqueue to the bid max-heap (becomes a resting limit order on the book).
Sell orders: Mirror image — match against the max-bid while bid_price ≥ ask_price.
Price-time priority: The C# implementation uses a Queue per price level inside the SortedDictionary — orders at the same price execute FIFO (earliest submitted first).

An order might contain 7 items scattered across a warehouse. A robot must pick all items and return to the packing station in the minimum total distance. With 7 items, there are 7! = 5040 possible orders — manageable with bitmask DP (n ≤ 20 is the sweet spot).

🔍 Problem Breakdown

The Problem: Given n item locations and a start/end position (packing station), find the visit order that minimises total travel distance. Distances between locations are pre-computed by Dijkstra/BFS on the warehouse grid (handling obstacles like shelving units). This is the Travelling Salesman Problem (TSP) — NP-hard in general, but tractable for small n with bitmask DP.
Naive Approach: Try all n! orderings. For n=7: 5,040 — fine. For n=20: 2.4 × 10¹⁸ — completely impossible. Even greedy "nearest unvisited item" is fast but gives a suboptimal path (can be 25% longer than optimal).
Key Insight: Represent the set of visited items as a bitmask (integer). State = (bitmask of visited items, index of the last item visited). With n items there are n × 2ⁿ unique states. For each state, try adding each unvisited item — O(n) transitions. Total: O(n² × 2ⁿ). For n=15: ~30M operations, completes in milliseconds. Amazon limits pick batches to ~15 items specifically because 2¹⁵ = 32K is tractable.

// Minimum distance to pick all items and return to start
// Distances pre-computed via BFS/Dijkstra on the warehouse grid
public int OptimalPickPath(int[] items, int[,] dist, int start)
{
    int n = items.Length;
    // dp[mask][i] = min cost to pick items in mask, currently at item i
    int[,] dp = new int[1 << n, n];
    for (int i = 0; i < (1<<n); i++)
        for (int j = 0; j < n; j++) dp[i, j] = int.MaxValue / 2;

    // Initialize: cost to travel from start to each item
    for (int i = 0; i < n; i++) dp[1 << i, i] = dist[start, items[i]];

    for (int mask = 1; mask < (1<<n); mask++)
    for (int u = 0; u < n; u++)
    {
        if ((mask & (1<<u)) == 0 || dp[mask, u] == int.MaxValue/2) continue;
        for (int v = 0; v < n; v++)
            if ((mask & (1<<v)) == 0)
                dp[mask|(1<<v), v] = Math.Min(dp[mask|(1<<v), v],
                                              dp[mask, u] + dist[items[u], items[v]]);
    }

    int full = (1<<n) - 1, ans = int.MaxValue;
    for (int i = 0; i < n; i++)  // Return to packing station
        ans = Math.Min(ans, dp[full, i] + dist[items[i], start]);
    return ans;
}

def optimal_pick_path(items, dist, start):
    n = len(items)
    INF = float('inf')
    # dp[mask][i] = min cost to pick items in mask, currently at item i
    dp = [[INF] * n for _ in range(1 << n)]
    # Initialize: cost to travel from start to each item
    for i in range(n):
        dp[1 << i][i] = dist[start][items[i]]

    for mask in range(1, 1 << n):
        for u in range(n):
            if not (mask & (1 << u)) or dp[mask][u] == INF: continue
            for v in range(n):
                if not (mask & (1 << v)):
                    new_cost = dp[mask][u] + dist[items[u]][items[v]]
                    new_mask = mask | (1 << v)
                    if new_cost < dp[new_mask][v]:
                        dp[new_mask][v] = new_cost

    full = (1 << n) - 1
    ans = INF
    for i in range(n):  # Return to packing station
        ans = min(ans, dp[full][i] + dist[items[i]][start])
    return ans

📖 Solution Walkthrough

Pre-computation: Run BFS/Dijkstra from the start position and from each item location to compute all pairwise distances (accounting for walls/obstacles on the grid).
1. DP state: dp[mask][i] = minimum distance to visit exactly the items encoded in mask, ending at item i.
2. Base case: dp[1<<i][i] = dist[start][item_i] for each item i — cost to travel directly from start to just that one item.
3. Transition: For every (mask, u) where u is in mask and dp[mask][u] is known, try extending to each unvisited item v (not in mask): dp[mask|(1<<v)][v] = min(current value, dp[mask][u] + dist[item_u][item_v]).
4. Answer: Iterate all last items u in the full mask and find min(dp[full_mask][u] + dist[item_u][start]) — the final leg returning to the packing station.
⏱ Time: O(n² × 2ⁿ) — for n=15: ~30M operations (milliseconds on modern hardware).
💾 Space: O(n × 2ⁿ) for the DP table — for n=15: ~500KB.

Git · GitHub · GitLab · Azure DevOps

When you run git diff, Git computes the Longest Common Subsequence (LCS) between two file versions to find the minimal set of line additions/deletions. The commit history is a Directed Acyclic Graph (DAG) where git rebase is a topological sort.

🔍 Problem Breakdown

The Problem: Given two versions of a file (arrays of lines), produce a compact diff that shows the minimum additions (+) and deletions (-). The foundation is finding the Longest Common Subsequence (LCS) — lines that appear in both files in the same order. Lines NOT in the LCS are either added or deleted.
Naive Approach: Enumerate all possible common subsequences between two files — exponential time. Even checking every pair of lines naively is O(m × n × avg_line_length), prohibitively slow for large source files.
Key Insight: LCS DP solves this in O(m × n) with a 2D table. dp[i][j] = length of LCS of the first i lines of old and first j lines of new. Backtracking through the table produces the diff. Git further uses patience diff — an LCS variant that first anchors on unique lines, producing more human-readable diffs on code.
C# 
// git diff — Longest Common Subsequence shows what changed between versions
public List<string> DiffFiles(string[] oldLines, string[] newLines)
{
    int m = oldLines.Length, n = newLines.Length;
    int[,] dp = new int[m+1, n+1];

    for (int i = 1; i <= m; i++)
    for (int j = 1; j <= n; j++)
        dp[i,j] = oldLines[i-1] == newLines[j-1]
            ? dp[i-1,j-1] + 1
            : Math.Max(dp[i-1,j], dp[i,j-1]);

    // Reconstruct diff by backtracking through dp table
    var diff = new List<string>();
    int r = m, c = n;
    while (r > 0 || c > 0)
    {
        if (r > 0 && c > 0 && oldLines[r-1] == newLines[c-1])
            { diff.Add("  " + oldLines[--r]); c--; }
        else if (c > 0 && (r == 0 || dp[r,c-1] >= dp[r-1,c]))
            { diff.Add("+ " + newLines[--c]); }
        else
            { diff.Add("- " + oldLines[--r]); }
    }
    diff.Reverse();
    return diff;
}

9. Banking — Real-Time Fraud Detection

Detect Fraudulent Transaction Clusters

Fraud detection runs multiple checks simultaneously: a sliding window checks if too many transactions occurred in the last 60 seconds (velocity fraud), Union-Find clusters accounts that share devices/IPs to find fraud rings, and rolling hashes deduplicate retry attempts.

🔍 Problem Breakdown

The Problem: Two independent fraud signals must run in real time on a stream of 65,000 transactions/second: (a) Velocity fraud — flag if a single account makes more than k transactions in the last t seconds; (b) Fraud rings — cluster accounts that share hardware fingerprints (device ID, IP, card number) and flag clusters of 3 or more as suspicious.
Naive Approach: (a) Scan all historical transactions per account = O(n) per check. (b) Build a new graph and run connected-components on every transaction = O(V + E) each time. Both are too slow at Visa's throughput.
Key Insight: (a) A sliding-window deque/queue naturally expires old timestamps in O(1) amortised — just pop the front while oldest timestamp is outside the window. (b) Union-Find is incrementally maintained: each new shared fingerprint triggers a union() call in O(α(V)) ≈ O(1). Fraud rings are detected by grouping accounts that share the same root node.

// Velocity check: flag if > k transactions in last t seconds (sliding window)
public bool IsVelocityFraud(Queue<long> txTimestamps, long nowMs, int maxTx, int windowSec)
{
    long windowMs = windowSec * 1000L;
    while (txTimestamps.Count > 0 && nowMs - txTimestamps.Peek() > windowMs)
        txTimestamps.Dequeue(); // Remove expired transactions
    txTimestamps.Enqueue(nowMs);
    return txTimestamps.Count > maxTx;
}

def find_fraud_clusters(accounts, fingerprint_to_accounts):
    parent = list(range(len(accounts)))
    def find(x):
        while parent[x] != x: parent[x] = parent[parent[x]]; x = parent[x]
        return x

    # Accounts sharing any fingerprint (device, IP, card) get unioned
    for accts in fingerprint_to_accounts.values():
        for i in range(1, len(accts)):
            parent[find(accts[0])] = find(accts[i])

    # Group by root → clusters of 3+ accounts are fraud ring candidates
    groups = defaultdict(list)
    for a in accounts: groups[find(a)].append(a)
    return {root: accts for root, accts in groups.items() if len(accts) >= 3}

📖 Solution Walkthrough

Velocity Check (Sliding Window):
1. Maintain a queue (deque) of timestamps for each account.
2. On each new transaction at time now: pop from the front while (now - front) > window duration. These timestamps are expired — outside the window.
3. Push the new timestamp. If queue size > maxTx → velocity fraud.
Why a queue? Timestamps are always inserted at the back and expired from the front (FIFO) — O(1) amortised, no searching needed.

Fraud Ring (Union-Find):
4. When accounts A and B share any fingerprint (device, IP, card): call union(A, B). Path compression in find() ensures all subsequent find() calls are O(1) amortised.
5. Group all accounts by their root node (the canonical representative of each connected cluster). Clusters with 3+ members → flag for investigation.
Why Union-Find over graph BFS? BFS requires the full graph structure upfront. Union-Find is incremental — it merges clusters one fingerprint match at a time, in real time as transactions arrive.
⏱ Time: O(1) amortised per velocity check. O(α(V)) per union/find ≈ O(1). 💾 Space: O(W) per account for the window; O(V) for Union-Find.

In every open-world game, NPCs (enemies, allies, vehicles) navigate around obstacles in real time. This is A* on a grid or navigation mesh. Games like The Witcher 3 run hundreds of pathfinding queries per frame (60fps), so optimization is critical.

🔍 Problem Breakdown

The Problem: On a 2D grid (0 = walkable, 1 = wall), find the shortest path between two cells. Allow 8-directional movement — diagonal moves cost √2 ≈ 1.414 (longer distance covered). Must run for hundreds of NPCs in under 16ms (one frame at 60fps).
Naive Approach: BFS gives the path with fewest steps but treats all moves as cost 1 — diagonals and cardinal moves are equal, producing paths that are geometrically longer than necessary. No heuristic means BFS expands in all directions equally — slow on large grids.
Key Insight: A* on a grid with Euclidean heuristic (straight-line distance to goal) is optimal for weighted movement. Only nodes closer to the goal are expanded. Real games go further: they pre-bake the walkable terrain into a Navigation Mesh (NavMesh) — a set of convex polygons. // A* pathfinding on a 2D game grid (0=walkable, 1=wall)
public List<(int,int)> FindPath(int[,] grid, (int r, int c) src, (int r, int c) dst)
{
    int rows = grid.GetLength(0), cols = grid.GetLength(1);
    double[,] g = new double[rows, cols];
    for (int i = 0; i < rows; i++) for (int j = 0; j < cols; j++) g[i,j] = double.MaxValue;
    g[src.r, src.c] = 0;

    var prev = new (int,int)?[rows, cols];
    var pq   = new PriorityQueue<(int,int), double>();
    pq.Enqueue(src, H(src, dst));

    int[] dr = {-1,1,0,0,-1,-1,1,1}; // 8-directional movement
    int[] dc = {0,0,-1,1,-1,1,-1,1};
    double[] cost = {1,1,1,1, 1.414,1.414,1.414,1.414}; // √2 for diagonal

    while (pq.Count > 0)
    {
        pq.TryDequeue(out var u, out _);
        if (u == dst) return ReconstructPath(prev, src, dst);
        for (int d = 0; d < 8; d++) {
            int nr = u.Item1+dr[d], nc = u.Item2+dc[d];

            double ng = g[u.Item1,u.Item2] + cost[d];
            if (ng < g[nr,nc]) {
                g[nr,nc] = ng; prev[nr,nc] = u;
                pq.Enqueue((nr,nc), ng + H((nr,nc), dst));
            }
        }
    }
    return new(); // No path
}

def find_path(grid, src, dst):
    rows, cols = len(grid), len(grid[0])
    INF = float('inf')
    g = [[INF] * cols for _ in range(rows)]
    g[src[0]][src[1]] = 0
    prev = [[None] * cols for _ in range(rows)]
    # 8-directional movement
    dr = [-1,1,0,0,-1,-1,1,1]; dc = [0,0,-1,1,-1,1,-1,1]
    cost = [1,1,1,1,1.414,1.414,1.414,1.414]  # √2 for diagonal
    h = lambda a, b: math.sqrt((a[0]-b[0])**2 + (a[1]-b[1])**2)  # Euclidean
    pq = [(h(src, dst), src)]
    while pq:
        _, u = heapq.heappop(pq)
        if u == dst:
            path, cur = [], dst
            while cur is not None:
                path.append(cur); cur = prev[cur[0]][cur[1]]
            return path[::-1]
        for d in range(8):
            nr, nc = u[0] + dr[d], u[1] + dc[d]
            if nr < 0 or nr >= rows or nc < 0 or nc >= cols or grid[nr][nc] == 1: continue
            ng = g[u[0]][u[1]] + cost[d]
            if ng < g[nr][nc]:
                g[nr][nc] = ng; prev[nr][nc] = u
                heapq.heappush(pq, (ng + h((nr, nc), dst), (nr, nc)))
    return []  # No path

📖 Solution Walkthrough

1. Initialise: g[src] = 0, all others = ∞. Push src onto the priority queue with f = h(src, dst).
2. 8-directional movement costs: The 4 cardinal directions cost 1.0; the 4 diagonal directions cost √2 ≈ 1.414. This makes diagonal moves proportionally more expensive since they cover more Euclidean distance.
3. Relaxation: For each walkable neighbour (nr, nc): tentative = g[r][c] + step_cost. If tentative < g[nr][nc]: update g, set prev[nr][nc] = current cell, push with f = tentative + h((nr,nc), dst).
4. Goal check: On dequeue, if current == dst → backtrack through prev[][] to build the path (reversed, then flip).
5. Heuristic h = Euclidean: √((r₂-r₁)² + (c₂-c₁)²) is always ≤ any walkable path — admissible, so A* finds the true optimal path.
⏱ Time: O(E log V) where V = walkable cells, E = 8V edges. In practice, the heuristic prunes large portions of the grid — far faster than Dijkstra.
💾 Space: O(V) for g[][], prev[][], and the priority queue.

Comparing two DNA sequences to find genetic mutations is equivalent to computing edit distance. The Smith-Waterman algorithm (a DP variant) finds the best local alignment — the most similar substring between two sequences, ignoring mismatched ends.

🔍 Problem Breakdown

The Problem: Given two DNA sequences (strings of A/C/G/T characters), find the highest-scoring local alignment — the pair of substrings that are most similar, accounting for matches (+2), mismatches (-1), and gaps/insertions (-1). Unlike global alignment (Needleman-Wunsch, which aligns full sequences), local alignment (Smith-Waterman) finds where the sequences are most similar regardless of mismatched ends.
Naive Approach: Checking every pair of substrings from both sequences is O(m² × n²) — completely infeasible for a genome of 30,000 bases.
Key Insight: Smith-Waterman fills an (m+1)×(n+1) DP table similar to edit distance, but with one critical difference: all negative scores are reset to 0 (local reset). This means the algorithm "forgets" poor-quality matches and restarts. The cell with the globally maximum value anywhere in the table marks the end of the best local alignment. Backtrack from that cell (stopping when you hit a 0) to retrieve the aligned region.

// Smith-Waterman local alignment: find best matching region between two DNA strings
// Used to find where a gene mutation occurred
public (int score, string aligned) SmithWaterman(string seq1, string seq2,
    int matchScore = 2, int mismatch = -1, int gap = -1)
{
    int m = seq1.Length, n = seq2.Length;
    int[,] dp = new int[m+1, n+1];
    int maxScore = 0; (int,int) maxPos = (0,0);

    for (int i = 1; i <= m; i++)
    for (int j = 1; j <= n; j++)
    {
        int match = dp[i-1,j-1] + (seq1[i-1]==seq2[j-1] ? matchScore : mismatch);
        dp[i,j] = Math.Max(0, Math.Max(match, Math.Max(dp[i-1,j]+gap, dp[i,j-1]+gap)));
        if (dp[i,j] > maxScore) { maxScore = dp[i,j]; maxPos = (i,j); }
    }

    // Traceback from maxPos to find the aligned region
    var sb = new StringBuilder();
    var (r, c) = maxPos;
    while (dp[r, c] > 0)
    {
        if      (r > 0 && c > 0 && dp[r,c] == dp[r-1,c-1] + (seq1[r-1]==seq2[c-1] ? matchScore : mismatch))
            { sb.Insert(0, seq1[--r]); c--; }
        else if (r > 0 && dp[r,c] == dp[r-1,c] + gap)
            { sb.Insert(0, '-'); r--; }
        else    { sb.Insert(0, '-'); c--; }
    }
    return (maxScore, sb.ToString());
}

# Smith-Waterman local alignment: find best matching region between two DNA strings
# Used to find where a gene mutation occurred
def smith_waterman(seq1, seq2, match_score=2, mismatch=-1, gap=-1):
    m, n = len(seq1), len(seq2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    max_score, max_pos = 0, (0, 0)
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            s = match_score if seq1[i-1] == seq2[j-1] else mismatch
            dp[i][j] = max(0, dp[i-1][j-1] + s, dp[i-1][j] + gap, dp[i][j-1] + gap)
            if dp[i][j] > max_score:
                max_score = dp[i][j]; max_pos = (i, j)

    # Traceback from max_pos to find the aligned region
    aligned = []
    r, c = max_pos
    while dp[r][c] > 0:
        s = match_score if seq1[r-1] == seq2[c-1] else mismatch
        if   r > 0 and c > 0 and dp[r][c] == dp[r-1][c-1] + s:
            aligned.append(seq1[r-1]); r -= 1; c -= 1
        elif r > 0 and dp[r][c] == dp[r-1][c] + gap:
            aligned.append('-'); r -= 1
        else:
            aligned.append('-'); c -= 1
    aligned.reverse()
    return max_score, ''.join(aligned)

📖 Solution Walkthrough

1. Fill DP table: dp[i][j] = best alignment score when seq1[i-1] aligns to seq2[j-1].
  • Diagonal: dp[i-1][j-1] + score (match_score if chars equal, else mismatch_penalty)
  • Up: dp[i-1][j] + gap_penalty (gap in seq2 — one character from seq1 has no match)
  • Left: dp[i][j-1] + gap_penalty (gap in seq1 — one character from seq2 has no match)
  • Local reset: dp[i][j] = max(0, three options above) — the key difference from global alignment.
2. Track max: Record the cell (maxPos) with the highest score across the entire table.
3. Traceback: From maxPos, follow the choices made (diagonal/up/left) until reaching a cell with value 0 — this gives the best-matching local region.
4. BLAST uses this in two phases: First, fast k-mer hashing finds candidate regions. Then Smith-Waterman runs on those candidates only — reducing the full O(m × n) work to a fraction of the genome.
⏱ Time: O(m × n) — fill the entire table once. 💾 Space: O(m × n) for the table; O(min(m,n)) if only the score is needed.

🔍 Problem Breakdown

The Problem: Route each request to the same server consistently (cache coherence), distribute load evenly across n servers (uniform hashing), and when a server is added or removed, minimise the number of keys that need to be remapped (stability).
Naive Approach: server = hash(requestKey) % n. This is O(1) but adding or removing a server changes n — almost every key maps to a different server. CDN cache miss rate spikes to ~100% — all cached content becomes unreachable instantly, causing a thundering-herd failure.
Key Insight: Place both servers and request keys on a virtual ring (hash space 0 to 2³²). A request maps to the nearest server clockwise. Adding a server only "steals" keys from the next server clockwise (~1/n keys). Removing a server only "gives" its keys to the next server (~1/n keys). public class ConsistentHashRing
{
    private readonly SortedDictionary<int, string> _ring = new();
    private readonly int _virtualNodes;

    public ConsistentHashRing(int virtualNodes = 150) => _virtualNodes = virtualNodes;

    public void AddServer(string server)
    {
        for (int i = 0; i < _virtualNodes; i++)
        {
            int hash = Hash($"{server}#{i}");
            _ring[hash] = server;
        }
    }

    public void RemoveServer(string server)
    {
        for (int i = 0; i < _virtualNodes; i++)

    }

        int hash = Hash(requestKey);
        // Find nearest server clockwise: first key ≥ hash
        foreach (var kv in _ring)
            if (kv.Key >= hash) return kv.Value;
        return _ring.First().Value; // Wrap around the ring
    }

import hashlib, bisect

class ConsistentHashRing:
    def __init__(self, virtual_nodes=150):
        self._ring = {}           # hash → server
        self._sorted_keys = []    # sorted hashes for binary search
        self._vnodes = virtual_nodes

    def _hash(self, key):
        return int(hashlib.md5(key.encode()).hexdigest(), 16) % (2**32)

    def add_server(self, server):
        for i in range(self._vnodes):
            h = self._hash(f"{server}#{i}")
            self._ring[h] = server
            bisect.insort(self._sorted_keys, h)

    def remove_server(self, server):
        for i in range(self._vnodes):
            h = self._hash(f"{server}#{i}")
            del self._ring[h]
            self._sorted_keys.remove(h)

    def get_server(self, request_key):
        if not self._ring: raise ValueError("No servers")
        h = self._hash(request_key)
        # Find nearest server clockwise: binary search for first key ≥ h
        idx = bisect.bisect_left(self._sorted_keys, h)
        if idx == len(self._sorted_keys): idx = 0  # Wrap around the ring
        return self._ring[self._sorted_keys[idx]]

AddServer:
1. For each virtual node i (0 to 149): compute hash("serverName#i") and insert into a SortedDictionary keyed by the hash value. 150 virtual nodes per server spreads its responsibility across the ring, preventing one server from becoming a hotspot.
RemoveServer:
2. Remove all 150 virtual node entries. Only the keys that previously mapped to this server's ring segments are remapped — they go to the successor server on the ring. On average: 1/n of all keys remapped.
GetServer (request routing):
3. Compute hash(requestKey). Scan the SortedDictionary for the first entry whose key ≥ the hash ("first server clockwise"). If none found, wrap around to the first entry in the ring.
4. Why SortedDictionary? It supports O(log n) insert, delete, and successor queries — exactly what "find first clockwise server" requires. A plain HashMap can't do this.

import heapq, math

class DriverDispatch:
    # Spatial index: geohash cell → list of available drivers in that cell
    def __init__(self): self._grid = {}

    def _encode(self, lat, lng, bits=6):
        lat_b = int((lat + 90) / 180.0 * (1 << bits))
        lng_b = int((lng + 180) / 360.0 * (1 << bits))
        return f"{lat_b}:{lng_b}"

    def update_driver(self, driver, lat, lng):
        # Remove from old cell, register in new cell
        old = self._grid.get(driver.get('cell'), [])
        if driver in old: old.remove(driver)
        driver['cell'] = self._encode(lat, lng)
        self._grid.setdefault(driver['cell'], []).append(driver)

    # Find k nearest drivers: geohash lookup → min-heap sort by real distance
    def find_nearest(self, lat, lng, k):
        pq = []  # min-heap: (distance, counter, driver)
        counter = 0
        for cell in self._get_neighbor_cells(self._encode(lat, lng)):
            for d in self._grid.get(cell, []):
                dist = self._haversine_km(lat, lng, d['lat'], d['lng'])
                heapq.heappush(pq, (dist, counter, d)); counter += 1
        result = []
        while len(result) < k and pq:
            _, _, d = heapq.heappop(pq); result.append(d)
        return result

    def _haversine_km(self, lat1, lng1, lat2, lng2):
        R = 6371
        dlat = math.radians(lat2 - lat1); dlng = math.radians(lng2 - lng1)
        a = math.sin(dlat/2)**2 + math.cos(math.radians(lat1)) * math.cos(math.radians(lat2)) * math.sin(dlng/2)**2
        return R * 2 * math.asin(math.sqrt(a))

    public IEnumerable<TRow> RangeScan(TKey lo, TKey hi)
    {
        // Binary search for the first key ≥ lo (B-Tree internal node traversal)
        int left = 0, right = _pages.Count - 1, start = _pages.Count;
        while (left <= right)
        {
            int mid = (left + right) / 2;
            if (_pages.GetKeyAtIndex(mid).CompareTo(lo) >= 0)
                { start = mid; right = mid - 1; }
            else left = mid + 1;
        }
        // Stream forward through leaf pages while key ≤ hi (linked-list scan)
        for (int i = start; i < _pages.Count; i++)
        {
            if (_pages.GetKeyAtIndex(i).CompareTo(hi) > 0) yield break;
            foreach (var row in _pages.GetValueAtIndex(i)) yield return row;
        }
    }

    public MerkleTree(IEnumerable<string> transactions)

    // Build bottom-up: pair-hash each level until one root hash remains
    public byte[] Root()
    {
        var level = new List<byte[]>(_leaves);
        while (level.Count > 1)
        {
            if (level.Count % 2 != 0) level.Add(level[^1]); // duplicate last if odd count
            var next = new List<byte[]>();
            for (int i = 0; i < level.Count; i += 2)
                next.Add(HashPair(level[i], level[i + 1]));
            level = next;
        }
        return level[0];
    }

    // Merkle proof: O(log n) sibling hashes needed to prove leaf at index
    public List<(byte[] sibling, bool siblingOnRight)> GetProof(int idx)
    {
        var proof = new List<(byte[], bool)>();

            if (level.Count % 2 != 0) level.Add(level[^1]);
            bool isLeft = idx % 2 == 0;              // is current node a left child?

            var next = new List<byte[]>();
            for (int i = 0; i < level.Count; i += 2)
                next.Add(HashPair(level[i], level[i + 1]));
            level = next; idx /= 2;
        }
        return proof;
    }

        foreach (var (sib, right) in proof)
            cur = right ? HashPair(cur, sib) : HashPair(sib, cur);
        return cur.SequenceEqual(expectedRoot);

import hashlib

class MerkleTree:
    def __init__(self, transactions):
        self._leaves = [self._hash(tx) for tx in transactions]

    def _hash(self, data):
        if isinstance(data, str): data = data.encode()
        return hashlib.sha256(data).digest()

    def _hash_pair(self, a, b): return hashlib.sha256(a + b).digest()

    # Build bottom-up: pair-hash each level until one root hash remains
    def root(self):
        level = list(self._leaves)
        while len(level) > 1:
            if len(level) % 2: level.append(level[-1])  # duplicate last if odd
            level = [self._hash_pair(level[i], level[i+1]) for i in range(0, len(level), 2)]
        return level[0]

    # Merkle proof: O(log n) sibling hashes needed to prove leaf at index
    def get_proof(self, idx):
        proof, level = [], list(self._leaves)
        while len(level) > 1:
            if len(level) % 2: level.append(level[-1])
            is_left = idx % 2 == 0  # is current node a left child?
            sib_idx = idx + 1 if is_left else idx - 1
            proof.append((level[sib_idx], not is_left))  # (sibling, sibling_on_right)
            level = [self._hash_pair(level[i], level[i+1]) for i in range(0, len(level), 2)]
            idx //= 2
        return proof

    # Verify: mobile wallet recomputes root from just O(log n) hashes
    def verify(self, tx, proof, expected_root):
        cur = self._hash(tx)
        for sib, sib_on_right in proof:
            cur = self._hash_pair(cur, sib) if sib_on_right else self._hash_pair(sib, cur)
        return cur == expected_root

Building the Merkle Tree (at block creation):
1. Hash each transaction with SHA-256 → the leaf hashes.
2. If the number of leaves is odd, duplicate the last one (Bitcoin's tie-breaking convention).
3. Pair-hash each level: parent = SHA-256(leftChild ∥ rightChild). Repeat until 1 root hash remains. Total work: O(n) hashing operations.

4. At each level, record the sibling hash and its position (left or right). These O(log n) hashes form the proof path from the leaf to the root.
5. For n=4096 transactions: 12 sibling hashes × 32 bytes = 384 bytes total proof.
Verification (mobile wallet, no block download needed):
6. Start with H(TxA). For each (sibling, position) in proof: combine in the correct order and hash. If final result == trusted block header Merkle root → transaction confirmed present.

// Problem 1: Merge overlapping meetings — collapses conflicts into busy blocks
public int[][] MergeIntervals(int[][] meetings)
{
    Array.Sort(meetings, (a, b) => a[0] - b[0]); // sort by start time — O(n log n)
    var result = new List<int[]> { meetings[0] };
    foreach (var m in meetings.Skip(1))
    {
        var last = result[^1];
        if (m[0] <= last[1])              // overlaps or touches → extend
            last[1] = Math.Max(last[1], m[1]);
        else
            result.Add(m);               // gap → start new block
    }
    return result.ToArray();
}

// Problem 2: Minimum rooms needed = max simultaneous meetings
public int MinRoomsRequired(int[][] meetings)
{
    Array.Sort(meetings, (a, b) => a[0] - b[0]);
    var endTimes = new PriorityQueue<int, int>(); // min-heap of active meeting end times
    foreach (var m in meetings)
    {
        if (endTimes.Count > 0 && endTimes.Peek() <= m[0])
            endTimes.Dequeue();        // reuse room whose meeting just ended
        endTimes.Enqueue(m[1], m[1]); // assign room, track its end time
    }
    return endTimes.Count; // rooms still occupied = peak rooms needed
}

public List<(int start, int end)> FindFreeSlots(

    var free = new List<(int, int)>();

    foreach (var m in merged)

        prev = Math.Max(prev, m[1]);

    if (dayEnd - prev >= minDuration) free.Add((prev, dayEnd)); // gap after last
    return free;

public class SlidingWindowRateLimiter
{
    private readonly int _limit;
    private readonly TimeSpan _window;
    private readonly Dictionary<string, Queue<DateTime>> _hits = new();

    public SlidingWindowRateLimiter(int limit, TimeSpan window)
    {
        _limit = limit;
        _window = window;
    }

    public bool Allow(string key, DateTime nowUtc)
    {
        if (!_hits.TryGetValue(key, out var q))
        {
            q = new Queue<DateTime>();
            _hits[key] = q;
        }

        var cutoff = nowUtc - _window;
        while (q.Count > 0 && q.Peek() <= cutoff)
            q.Dequeue();

        q.Enqueue(nowUtc);
        return true;
    }

using System.Collections.Generic;

public class LRUCache
{
    private readonly int _capacity;
    private readonly Dictionary<int, LinkedListNode<(int key, int value)>> _map = new();
    private readonly LinkedList<(int key, int value)> _list = new();

    public LRUCache(int capacity) => _capacity = capacity;

    public int Get(int key)
    {
        if (!_map.TryGetValue(key, out var node)) return -1;
        _list.Remove(node);
        _list.AddFirst(node);
        return node.Value.value;
    }

    public void Put(int key, int value)
    {
        if (_map.TryGetValue(key, out var node))
        {
            node.Value = (key, value);
            _list.Remove(node);
            _list.AddFirst(node);
            return;
        }

        if (_map.Count == _capacity)
        {
            var lru = _list.Last!;
            _map.Remove(lru.Value.key);
            _list.RemoveLast();
        }

        var fresh = new LinkedListNode<(int key, int value)>((key, value));
        _list.AddFirst(fresh);
        _map[key] = fresh;
    }
}

        self.capacity = capacity
        self.data = OrderedDict()

    def get(self, key: int) -> int:
        if key not in self.data:
            return -1
        self.data.move_to_end(key, last=False)  # move to MRU position
        return self.data[key]

    def put(self, key: int, value: int) -> None:
        if key in self.data:
            self.data[key] = value
            self.data.move_to_end(key, last=False)
            return

        if len(self.data) == self.capacity:
            self.data.popitem(last=True)  # evict LRU

        self.data[key] = value
        self.data.move_to_end(key, last=False)

using System.Collections.Generic;

public class Fenwick
{
    private readonly int[] _bit;
    public Fenwick(int n) => _bit = new int[n + 1];

    public void Add(int idx, int delta)
    {
        for (; idx < _bit.Length; idx += idx & -idx) _bit[idx] += delta;
    }

    public int Sum(int idx)
    {
        int res = 0;
        for (; idx > 0; idx -= idx & -idx) res += _bit[idx];
        return res;
    }
}

    }

    public void UpdateScore(string playerId, int newScore)
    {
        if (_playerScore.TryGetValue(playerId, out int oldScore))
            _fw.Add(oldScore + 1, -1);

        _playerScore[playerId] = newScore;
        _fw.Add(newScore + 1, +1);
    }

    public int GetRank(string playerId)
    {
        int score = _playerScore[playerId];
        int total = _fw.Sum(_maxScore + 1);
        int upToScore = _fw.Sum(score + 1);
        int higher = total - upToScore;
        return higher + 1;
    }

class Fenwick:
    def __init__(self, n: int):
        self.bit = [0] * (n + 1)

    def add(self, idx: int, delta: int) -> None:
        while idx < len(self.bit):
            self.bit[idx] += delta
            idx += idx & -idx

    def prefix_sum(self, idx: int) -> int:
        result = 0
        while idx > 0:
            result += self.bit[idx]
            idx -= idx & -idx
        return result


class Leaderboard:
    def __init__(self, max_score: int):
        self.max_score = max_score
        self.fw = Fenwick(max_score + 2)
        self.player_score = {}

    def update_score(self, player_id: str, new_score: int) -> None:
        old = self.player_score.get(player_id)
        if old is not None:
            self.fw.add(old + 1, -1)
        self.player_score[player_id] = new_score
        self.fw.add(new_score + 1, +1)

    def get_rank(self, player_id: str) -> int:
        score = self.player_score[player_id]
        total = self.fw.prefix_sum(self.max_score + 1)
        up_to_score = self.fw.prefix_sum(score + 1)
        higher = total - up_to_score
        return higher + 1
📖 Solution Walkthrough
We maintain a frequency table of scores and compress it into a Fenwick Tree for fast prefix sums. Updating a player means decrementing old score bucket and incrementing new score bucket. Rank is computed as 1 + count(scores strictly greater than mine), which is a suffix query derived from prefix sums. Time complexity is O(log M) per update/query, a massive win over re-sorting all players.
