Section 1 — Why Naive Pattern Matching Fails
NAIVE APPROACH: For each position i in text, check if pattern matches starting at i. Text T = "AAAAAAB" (length n=7) Pattern P = "AAAB" (length m=4) i=0: A A A A → mismatch at pos 3. Restart at i=1. i=1: A A A B → mismatch at pos 2 (A≠B). Restart at i=2. i=2: A A A B → mismatch. Restart. ... We wasted work! We already KNOW that positions 1-3 of T are 'A'. After failing at position 3, we shouldn't restart from scratch. WORST CASE: O(n × m) — e.g., T = "AAA...A" and P = "AAA...AB" Each position tries m characters before failing at the last one. For n=10^5 and m=10^5/2: up to 2.5×10^9 operations → too slow. SOLUTION: Pre-process the PATTERN to skip redundant comparisons. KMP and Z-function both achieve O(n + m).
Section 2 — KMP Algorithm

The Knuth-Morris-Pratt algorithm pre-processes the pattern into a failure function (also called the lps array — Longest Proper Prefix which is also Suffix). When a mismatch occurs, instead of restarting at position 0, we "fall back" to the longest prefix of the pattern that's also a suffix of what we've matched so far.

FAILURE FUNCTION (LPS array): lps[i] = length of longest proper prefix of P[0..i] that is also a suffix. Pattern: "ABABCABAB" Index: 0 1 2 3 4 5 6 7 8 lps: 0 0 1 2 0 1 2 3 4 Computed left-to-right in O(m): i=0: lps[0]=0 always. i=1: P[1]='B' vs P[0]='A' → mismatch → lps[1]=0 i=2: P[2]='A' vs P[0]='A' → match → lps[2]=1 i=3: P[3]='B' vs P[1]='B' → match → lps[3]=2 i=4: P[4]='C' vs P[2]='A' → mismatch; fall back: lps[1]=0, P[4] vs P[0]='A' → mismatch → lps[4]=0 i=5: P[5]='A' vs P[0]='A' → match → lps[5]=1 i=6: P[6]='B' vs P[1]='B' → match → lps[6]=2 i=7: P[7]='A' vs P[2]='A' → match → lps[7]=3 i=8: P[8]='B' vs P[3]='B' → match → lps[8]=4 SEARCH using LPS: i = position in text, j = position in pattern - If T[i] == P[j]: i++, j++. If j == m: found match at i-m; j = lps[j-1] - If mismatch and j>0: j = lps[j-1] (DON'T increment i) - If mismatch and j==0: i++
C# 
// Build LPS (failure function) array — O(m)
public int[] BuildLps(string pattern)
{
    int m = pattern.Length;
    int[] lps = new int[m]; // lps[0] = 0 always
    int len = 0, i = 1;
    while (i < m)
    {
        if (pattern[i] == pattern[len]) { lps[i++] = ++len; }
        else if (len > 0) { len = lps[len - 1]; }  // fallback (don't increment i)
        else { lps[i++] = 0; }
    }
    return lps;
}

// KMP Search — returns all start indices of pattern in text — O(n+m)
public List<int> KmpSearch(string text, string pattern)
{
    int[] lps = BuildLps(pattern);
    int n = text.Length, m = pattern.Length;
    var matches = new List<int>();
    int i = 0, j = 0;
    while (i < n)
    {
        if (text[i] == pattern[j]) { i++; j++; }
        if (j == m)
        {
            matches.Add(i - m);  // match found at index i-m
            j = lps[j - 1];        // continue searching
        }
        else if (i < n && text[i] != pattern[j])
        {
            if (j > 0) j = lps[j - 1];
            else i++;
        }
    }
    return matches;
}
// KmpSearch("ABABCABABABABCABAB", "ABABCABAB") → [0, 8]
Python 
def build_lps(pattern):
    m = len(pattern); lps = [0] * m; ln = 0; i = 1
    while i < m:
        if pattern[i] == pattern[ln]: lps[i] = ln + 1; ln += 1; i += 1
        elif ln > 0: ln = lps[ln-1]
        else: lps[i] = 0; i += 1
    return lps

def kmp_search(text, pattern):
    if not pattern: return []
    lps = build_lps(pattern); n = len(text); m = len(pattern)
    matches = []; i = j = 0
    while i < n:
        if text[i] == pattern[j]: i += 1; j += 1
        if j == m: matches.append(i-m); j = lps[j-1]
        elif i < n and text[i] != pattern[j]:
            if j > 0: j = lps[j-1]
            else: i += 1
    return matches
Section 3 — Z-Function

The Z-function is an alternative to KMP that many find cleaner. For string s, z[i] is the length of the longest substring starting from s[i] that is also a prefix of s. Setting z[0] = 0 by convention (the whole string is a prefix of itself but we exclude it).

Z-ARRAY DEFINITION: z[i] = length of longest s[i..i+z[i]-1] that equals s[0..z[i]-1] String: "aabxaa" z: [0, 1, 0, 0, 2, 1] z[1]=1: s[1..]="abxaa", longest prefix match = "a" (length 1) z[4]=2: s[4..]="aa", longest prefix match = "aa" (length 2) ✓ PATTERN SEARCH USING Z-FUNCTION: Concatenate: s = pattern + "$" + text ($ = separator not in either string) Compute z-array of s. For positions i >= m+1: if z[i] == m, then text has a match at i-(m+1). EXAMPLE: pattern = "ab", text = "ababab" s = "ab$ababab" (length = 2 + 1 + 6 = 9) z = [0,0,0,2,0,2,0,2,0] Positions i=3,5,7: z[i]=2=m → matches at text pos 0, 2, 4 ✓ Z-FUNCTION COMPUTATION in O(n): Maintain a window [l, r] = rightmost z-box (substring = prefix). For i < r: z[i] = min(r-i, z[i-l]). Then try to extend. Update [l, r] if i + z[i] > r.
C# 
public int[] ZFunction(string s)
{
    int n = s.Length;
    int[] z = new int[n];
    int l = 0, r = 0;
    for (int i = 1; i < n; i++)
    {
        if (i < r) z[i] = Math.Min(r - i, z[i - l]);
        while (i + z[i] < n && s[z[i]] == s[i + z[i]]) z[i]++;
        if (i + z[i] > r) { l = i; r = i + z[i]; }
    }
    return z;
}

public List<int> ZSearch(string text, string pattern)
{
    string s = pattern + "$" + text;
    int[] z = ZFunction(s);
    int m = pattern.Length;
    var matches = new List<int>();
    for (int i = m + 1; i < s.Length; i++)
        if (z[i] == m) matches.Add(i - m - 1);
    return matches;
}
Python 
def z_function(s):
    n = len(s); z = [0] * n; l = r = 0
    for i in range(1, n):
        if i < r: z[i] = min(r-i, z[i-l])
        while i+z[i] < n and s[z[i]] == s[i+z[i]]: z[i] += 1
        if i+z[i] > r: l, r = i, i+z[i]
    return z

def z_search(text, pattern):
    s = pattern + "$" + text; m = len(pattern)
    z = z_function(s)
    return [i-m-1 for i in range(m+1, len(s)) if z[i]==m]

# LeetCode #459 — Repeated Substring Pattern
# s has a repeated pattern iff z[i] + i == n and i divides n
def repeatedSubstringPattern(s):
    z = z_function(s); n = len(s)
    return any(z[i]+i==n and n%i==0 for i in range(1,n))
Section 4 — Rabin-Karp Rolling Hash
RABIN-KARP IDEA: Compute a hash of the pattern (length m). Slide a window of size m over the text, maintaining a ROLLING hash. If hashes match: verify with direct comparison (avoid hash collisions). ROLLING HASH: hash(s[i..i+m-1]) = s[i]*b^(m-1) + s[i+1]*b^(m-2) + ... + s[i+m-1]*b^0 When sliding right: hash(s[i+1..i+m]) = (hash(s[i..i+m-1]) - s[i]*b^(m-1)) * b + s[i+m] All operations mod a large prime (or two primes for double hashing). COMPLEXITY: Average: O(n + m) — few hash collisions in practice Worst: O(nm) — all hash collisions (adversarial input) COLLISION AVOIDANCE: Use double hashing: two different (base, mod) pairs. Probability of collision per position: ~1/MOD. With MOD=10^9+7 and n=10^5: expected 0 collisions. ADVANTAGES OVER KMP: - Easier to extend to 2D (2D grid pattern matching) - Easier to search for multiple patterns simultaneously - More naturally handles "find any occurrence" vs "all occurrences"
C# 
// Rabin-Karp — all matches of pattern in text
public List<int> RabinKarp(string text, string pattern)
{
    int n = text.Length, m = pattern.Length;
    if (m > n) return new List<int>();

    const long BASE = 31, MOD = 1_000_000_007;
    var matches = new List<int>();

    // Compute b^(m-1) mod MOD
    long power = 1;
    for (int i = 0; i < m - 1; i++) power = power * BASE % MOD;

    long H(char c) => c - 'a' + 1;

    // Compute hash of pattern and first window
    long patHash = 0, winHash = 0;
    for (int i = 0; i < m; i++)
    {
        patHash = (patHash * BASE + H(pattern[i])) % MOD;
        winHash = (winHash * BASE + H(text[i])) % MOD;
    }
    if (patHash == winHash && text[..m] == pattern) matches.Add(0);

    for (int i = 1; i <= n - m; i++)
    {
        winHash = (winHash - H(text[i - 1]) * power % MOD + MOD) % MOD;
        winHash = (winHash * BASE + H(text[i + m - 1])) % MOD;
        if (patHash == winHash && text.Substring(i, m) == pattern)
            matches.Add(i);
    }
    return matches;
}
Python 
def rabin_karp(text, pattern):
    n, m = len(text), len(pattern)
    if m > n: return []
    BASE, MOD = 31, 10**9+7
    h = lambda c: ord(c) - ord('a') + 1
    power = pow(BASE, m-1, MOD)

    pat_hash = win_hash = 0
    for i in range(m):
        pat_hash = (pat_hash * BASE + h(pattern[i])) % MOD
        win_hash = (win_hash * BASE + h(text[i])) % MOD

    matches = []
    if pat_hash == win_hash and text[:m] == pattern: matches.append(0)

    for i in range(1, n-m+1):
        win_hash = (win_hash - h(text[i-1]) * power) % MOD
        win_hash = (win_hash * BASE + h(text[i+m-1])) % MOD
        if pat_hash == win_hash and text[i:i+m] == pattern:
            matches.append(i)
    return matches

Section 6 — Q&A Review
Q1: What does the LPS (failure function) array represent, and how is it used during KMP search?
lps[i] = the length of the longest proper prefix of pattern[0..i] that is also a suffix. "Proper" means not the full string itself. During search, when a mismatch occurs at position j in the pattern: instead of restarting at j=0, we set j = lps[j-1]. This is correct because lps[j-1] tells us that the first lps[j-1] characters of the pattern already match the text at the current position (they're the suffix we've already seen that equals a prefix). We skip re-checking those characters. Example: pattern "ABABC" with lps=[0,0,1,2,0]. Mismatch at j=4 → j=lps[3]=2. We know "AB" already matches, so resume from j=2.
Q2: How would you solve LeetCode #459 "Repeated Substring Pattern" using the KMP failure function?
Compute the LPS array of s. Let n = len(s) and lp = lps[n-1]. If lp > 0 and n % (n-lp) == 0, then s has a repeated pattern of length n-lp. Intuition: lps[n-1] = longest prefix=suffix. The remaining length n-lp is a candidate period. If n is divisible by (n-lp), the string is made entirely of repetitions of s[0..n-lp-1]. Example: "abababab" → lps[-1]=6, n=8, n-lp=2, 8%2=0 → True. Period is "ab". Alternative: check if s in (s+s)[1:-1] — if yes, s has a repeated pattern. This works because s+s contains all rotations of s, and a rotation of a repeated-pattern string includes the string itself.
Q3: How does the Z-function differ from the KMP failure function? When would you prefer one over the other?
KMP's lps[i] looks at the string ending at position i: "what's the longest prefix matching the suffix ending at i?" Z[i] looks forward from position i: "how far does the prefix match starting at i?" Both are O(m) to compute and O(n+m) for search. Use Z-function when: the problem naturally involves "how much of the prefix matches starting at position i" (e.g., finding period, counting prefix occurrences). Use KMP when: the problem involves "what to do after a mismatch during linear scan" — the failure function directly encodes the fallback. In practice, many people prefer Z-function for its conceptual simplicity (z[i] has a clear geometric meaning), while KMP's lps is more directly tied to the search algorithm.
Q4: What is a "hash collision" in Rabin-Karp, and how do you handle it?
A hash collision occurs when two different strings produce the same hash value. In Rabin-Karp, when the window hash matches the pattern hash but the strings are actually different — this causes a false positive and requires O(m) verification. With a single hash of modulus ~10^9, collision probability per position is ~10^-9. For n=10^5 positions, expected ~10^-4 collisions — essentially zero. To eliminate risk in competitive programming: use double hashing (two independent (base, mod) pairs). A false positive requires collision in both simultaneously: probability ~10^-18. In C# and Python for LeetCode, single hashing with MOD=10^9+7 and BASE=31 is sufficient in practice (test cases aren't adversarial).
Q5: How do you use rolling hash for the binary-search-on-length pattern (e.g., LeetCode #1044)?
LeetCode #1044: Find the longest substring that appears at least twice. Approach: binary search on the length L. For each candidate L, check if any substring of length L appears twice using Rabin-Karp's rolling hash. Check function: slide a window of size L across the string, compute rolling hash for each window, store hashes in a set. If any hash is seen twice (verify to avoid collisions), return true. Binary search: if length L is achievable (duplicate found), try longer. If not, try shorter. Time: O(n log n) — O(log n) iterations of binary search, each O(n) for rolling hash. This is the classic "binary search on answer + hash check" pattern used in many hard LeetCode problems.
