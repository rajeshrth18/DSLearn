Section 1 — Bit Operation Fundamentals
BINARY REPRESENTATION & BASIC OPERATIONS: n = 13 = 1101₂ (bit positions: bit3=1, bit2=1, bit1=0, bit0=1) OPERATION SYMBOL EXAMPLE (5=0101, 6=0110) AND & 5 & 6 = 0100 = 4 (bits set in BOTH) OR | 5 | 6 = 0111 = 7 (bits set in EITHER) XOR ^ 5 ^ 6 = 0011 = 3 (bits set in EXACTLY ONE) NOT ~ ~5 = ...11111010 (flip all bits; -(n+1) in two's complement) Left shift << 5 << 2 = 10100 = 20 (multiply by 2^k) Right shift >> 20 >> 2 = 101 = 5 (divide by 2^k, floor) KEY IDENTITIES: n & (n-1) → clears the lowest set bit of n n & (-n) → isolates the lowest set bit of n (also: n & ~(n-1)) n | (n+1) → sets the lowest clear bit of n n ^ n → 0 (XOR with itself = 0) n ^ 0 → n (XOR with 0 = identity) TWO'S COMPLEMENT (how negative numbers work): -n = ~n + 1 So: n & (-n) = n & (~n + 1) isolates LSB ← used in Fenwick trees! Trick Collection

BIT TRICK REFERENCE: CHECK if bit k is set: (n >> k) & 1 SET bit k: n | (1 << k) CLEAR bit k: n & ~(1 << k) TOGGLE bit k: n ^ (1 << k) CHECK if n is power of 2: n > 0 && (n & (n-1)) == 0 CLEAR lowest set bit: n & (n-1) ← Brian Kernighan ISOLATE lowest set bit: n & (-n) ← Fenwick tree index ROUND DOWN to power of 2: n = 1 << (int)log2(n) ROUND UP to power of 2: n--, n|=n>>1, n|=n>>2, n|=n>>4, n|=n>>8, n|=n>>16, n++ BRIAN KERNIGHAN'S BIT COUNT (count set bits): count = 0 while n != 0: n &= (n - 1) # clear lowest set bit count += 1 # Each iteration removes exactly one set bit → O(number of set bits) vs. naive loop: O(log n) iterations but visits ALL bits Brian Kernighan: O(popcount(n)) — faster for sparse numbers
C# 
public class BitTricks
{
    // Count set bits using Brian Kernighan
    public static int PopCount(int n)
    {
        int count = 0;
        while (n != 0) { n &= n - 1; count++; }
        return count;
    }

    // OR: use built-in — C# BitOperations.PopCount(uint n)
    public static int PopCountBuiltIn(uint n)
        => System.Numerics.BitOperations.PopCount(n);

    // Check power of 2
    public static bool IsPow2(int n) => n > 0 && (n & (n - 1)) == 0;

    // Lowest set bit (used in BIT/Fenwick)
    public static int Lsb(int n) => n & (-n);

    // Reverse bits of a 32-bit integer
    public static uint ReverseBits(uint n)
    {
        uint res = 0;
        for (int i = 0; i < 32; i++) { res = (res << 1) | (n & 1); n >>= 1; }
        return res;
    }

    // Count bits set in all numbers 0..n (LeetCode #338)
    public static int[] CountBits(int n)
    {
        int[] dp = new int[n + 1];
        for (int i = 1; i <= n; i++)
            dp[i] = dp[i >> 1] + (i & 1);  // dp[i] = dp[i/2] + (i is odd)
        return dp;
    }
}

    res = 0

    return res
Section 3 — XOR Properties & Patterns
XOR KEY PROPERTIES: 1. n ^ n = 0 (XOR with itself cancels out) 2. n ^ 0 = n (identity) 3. Commutative: a ^ b = b ^ a 4. Associative: (a ^ b) ^ c = a ^ (b ^ c) 5. Self-inverse: (a ^ b) ^ b = a ← CRUCIAL for many tricks APPLICATIONS: Single Number (LeetCode #136): All nums appear twice except one. XOR all numbers. Pairs cancel (n^n=0). Result = the single number. [2,2,1] → 2^2^1 = 0^1 = 1 Two Single Numbers (LeetCode #260): XOR all → x ^ y (where x and y are the two singles). Find any set bit in x^y (e.g., LSB). Use it to split nums into 2 groups: Group 1: bit set, Group 2: bit not set. XOR each group independently → x and y. Missing Number (LeetCode #268): XOR all indices 0..n with all array values. Every paired value cancels. Missing value remains. expected ^ given = 0^1^...^n ^ a[0]^a[1]^...^a[n-1] = missing Swap without temp: a ^= b; b ^= a; a ^= b; ← classic but avoid (UB if a==b address) In Python: a, b = b, a (preferred) XOR IN COMPETITIVE PROGRAMMING: - Check if array has XOR equal to k - XOR prefix sums (XOR up to index i): xor_prefix[i] = a[0]^a[1]^...^a[i] - XOR prefix: a[l..r] = xor_prefix[r] ^ xor_prefix[l-1] ← range XOR query - Maximum XOR pair (uses Trie of binary representations)

// LeetCode #260 — Single Number III (two unique)
public int[] SingleNumberIII(int[] nums)
{
    int xorAll = nums.Aggregate(0, (a, n) => a ^ n);
    int bit = xorAll & (-xorAll);  // any differing bit between the two singles
    int a = 0, b = 0;
    foreach (int n in nums) {
        if ((n & bit) != 0) a ^= n; else b ^= n;
    }
    return new[] { a, b };
}

// LeetCode #421 — Maximum XOR of Two Numbers (Trie approach)
public int FindMaximumXOR(int[] nums)
{
    int maxVal = 0, mask = 0;
    // Bit-by-bit greedy from MSB
    for (int i = 31; i >= 0; i--) {
        mask |= (1 << i);
        var prefixes = new HashSet<int>(nums.Select(n => n & mask));
        int candidate = maxVal | (1 << i);
        if (prefixes.Any(p => prefixes.Contains(p ^ candidate)))
            maxVal = candidate;
    }
    return maxVal;
}

def singleNumberIII(nums):

    bit = xor_all & (-xor_all)
    a = b = 0
    for n in nums:
        if n & bit: a ^= n
        else: b ^= n

# Maximum XOR using prefix sets (greedy from MSB)
def findMaximumXOR(nums):
    ans = mask = 0
    for i in range(31, -1, -1):
        mask |= 1 << i
        prefixes = {n & mask for n in nums}
        candidate = ans | (1 << i)
        if any(candidate ^ p in prefixes for p in prefixes): ans = candidate
    return ans

// LeetCode #847 — Shortest Path Visiting All Nodes
// BFS with bitmask state: (node, visitedMask)
public int ShortestPathLength(int[][] graph)
{
    int n = graph.Length, full = (1 << n) - 1;
    var vis = new HashSet<(int, int)>();
    var q = new Queue<(int node, int mask, int dist)>();
    for (int i = 0; i < n; i++) {
        q.Enqueue((i, 1 << i, 0)); vis.Add((i, 1 << i));
    }
    while (q.Count > 0) {
        var (node, mask, dist) = q.Dequeue();
        if (mask == full) return dist;
        foreach (int next in graph[node]) {
            int nm = mask | (1 << next);
            if (vis.Add((next, nm))) q.Enqueue((next, nm, dist + 1));
        }
    }
    return -1;
}

# LeetCode #1986 Minimum Number of Work Sessions (Bitmask DP)
def minSessions(tasks, sessionTime):
    n = len(tasks); full = (1<<n)-1
    # dp[mask] = min sessions to complete tasks in mask
    dp = [float('inf')] * (1<<n)
    dp[0] = 0
    rem = [0] * (1<<n)  # remaining time in last session
    for mask in range(1<<n):
        for i in range(n):
            if mask & (1<<i): continue
            prev = mask; t = tasks[i]; nm = mask | (1<<i)
            if rem[prev] >= t:
                if dp[prev] < dp[nm] or (dp[prev]==dp[nm] and rem[prev]-t > rem[nm]):
                    dp[nm] = dp[prev]; rem[nm] = rem[prev] - t
            else:
                if dp[prev]+1 < dp[nm] or (dp[prev]+1==dp[nm] and sessionTime-t > rem[nm]):
                    dp[nm] = dp[prev]+1; rem[nm] = sessionTime - t
    return dp[full]

Q1: How does the expression n & (n-1) work, and what is it used for?
n-1 flips the lowest set bit of n to 0, and sets all bits below it to 1. When you AND with n, the bits below the lowest set bit (all 0s in n) AND with the 1s in n-1 → still 0. The lowest set bit in n AND with 0 in n-1 → 0. All higher bits unchanged. Net effect: the lowest set bit of n is cleared. Uses: (1) Brian Kernighan's popcount — repeatedly clear lowest set bit to count them; (2) Check if n is power of 2: n & (n-1) == 0 (power of 2 has exactly one set bit, so clearing it gives 0); (3) LeetCode #338 implicit in dp[i] = dp[i>>1] + (i&1).
Q2: Explain the "sum of two integers without using + operator" (LeetCode #371).
Binary addition: XOR gives the sum without carry (a ^ b). AND gives where carries occur (a & b), shifted left by 1 ((a & b) << 1). Repeat until carry is 0: while b != 0: a, b = a^b, (a&b)<<1. Why? At each position, the sum bit is 1 iff exactly one input is 1 (XOR). Carry into the next position is 1 iff both inputs are 1 (AND), shifted left. This terminates because carries always propagate to higher bits, so b eventually becomes 0. In Python, you need to handle two's complement manually for 32-bit: mask with 0xFFFFFFFF and handle sign.
Q3: What is the time complexity of enumerating all subsets of all subsets, and why is it O(3^n)?
For each of the 2^n possible masks, we enumerate all its subsets using the loop for sub = mask; sub > 0; sub = (sub-1) & mask. The total number of (mask, sub) pairs across all masks is: Σ_{mask=0}^{2^n-1} 2^{popcount(mask)}. Each bit independently contributes a factor of 3: for any given position, it can be (0,0) [not in mask], (1,0) [in mask but not submask], or (1,1) [in mask and submask]. So there are exactly 3^n total (mask, submask) pairs. This is the complexity of iterating all submasks, used in bitmask DP over subsets.
Q4: How do you find the single element appearing once when all others appear three times (LeetCode #137)?
Count bits modulo 3. For each bit position 0-31: sum all numbers' bits at position k. If sum % 3 != 0, the single number has bit k set. This works because each of the triplet numbers contributes 1 to the sum at each of their set bit positions, contributing 3 total per triplet → cancels mod 3. The single number's contribution remains. Implementation: use two variables `ones` and `twos` to track bits seen once and twice: ones = (ones ^ n) & ~twos; twos = (twos ^ n) & ~ones. After processing all numbers, `ones` contains the single number. This is O(n) time, O(1) space.
Q5: How does the maximum XOR of two numbers (LeetCode #421) use a greedy approach with prefix sets?
Greedy from the most significant bit (bit 31) down to bit 0. At each step, try to set the current bit in the answer. For bit i: compute all prefixes nums[j] & mask (mask = all bits from 31 down to i). If there exist two prefixes p1, p2 such that p1 ^ p2 == candidate (where candidate = current answer with bit i set), then we can achieve this bit → set it. The key insight: a ^ b = candidate iff b = a ^ candidate. So for each prefix a in the set, check if a ^ candidate is also in the set. This works greedily because once a higher bit is set in the XOR, no lower bits can change that, and we want to maximize.
