Section 1 — DFS Trees & Back Edges

When you run DFS on an undirected graph, the traversal implicitly defines a DFS tree: tree edges (edges you traverse to discover new nodes) and back edges (edges that connect a node back to one of its DFS ancestors). Understanding this structure is the foundation for almost all advanced graph algorithms.

GRAPH and its DFS TREE (starting from node 0): ORIGINAL GRAPH: 0 ── 1 ── 3 | | | 2 4 5 | 6 DFS TREE (tree edges = solid, back edges = dashed): 0 ├── 1 (tree edge) │ ├── 3 (tree edge) │ │ └── 5 (tree edge) │ └── 4 (tree edge) │ └── 6 (tree edge) └── 2 (tree edge) If edge 4──1 also existed, it would be a BACK EDGE (4 is a descendant of 1 and 4 connects up to 1). KEY INSIGHT: Tree edges = used by DFS to explore new nodes Back edges = connect descendants to ancestors (create CYCLES) In an undirected DFS: every non-tree edge IS a back edge (no forward/cross edges)
📝 Why DFS Trees Matter: A back edge from node u to ancestor v means there's an alternative path between all nodes on the path from v to u in the DFS tree. This alternative path is what prevents those edges and nodes from being critical. No back edge = single path = removing it disconnects the graph.
Section 2 — Discovery Time & Low Values

Tarjan's bridge/articulation point algorithm tracks two values at each node during DFS:

DISCOVERY TIME (disc[u]): The DFS clock time when node u was FIRST VISITED. Assigned in order: 0, 1, 2, 3, ... as DFS progresses. disc[u] is UNIQUE for each node and NEVER changes after assignment. LOW VALUE (low[u]): The EARLIEST discovery time reachable from the subtree rooted at u. This includes: - disc[u] itself - disc of all nodes reachable via ≥1 back edge from u or u's descendants low[u] = min( disc[u], ← itself low[child] for each DFS-tree child, ← through tree edges disc[w] for each back edge (u,w) ← direct back edge ) EXAMPLE: Graph 0-1-2-0 (triangle): DFS order: 0 → 1 → 2 → (back edge to 0) disc = [0, 1, 2] low = [0, 0, 0] ← all can reach node 0 via back edge Graph 0-1-2 (path, no cycle): disc = [0, 1, 2] low = [0, 1, 2] ← no back edges, low = disc KEY OBSERVATIONS: Edge (u, v) is a BRIDGE if low[v] > disc[u] Node u is ARTICULATION PT if any child v has low[v] >= disc[u] (with special rule for root)
Section 3 — Bridges (Cut Edges)

A bridge is an edge whose removal increases the number of connected components in the graph. Networks depend on bridges — they're single points of failure for connectivity.

BRIDGE DETECTION RULE: Edge (u → v) is a bridge if and only if: low[v] > disc[u] Meaning: the subtree rooted at v has NO back edge that reaches u or any ancestor of u. Therefore removing edge (u,v) would completely disconnect v's subtree. EXAMPLE: Graph: 0──1──2──3 | 4 DFS from 0: disc = [0,1,2,3,4] (in order of visit) Compute low: Node 3: no back edges → low[3] = disc[3] = 3 Node 2: low[3]=3 > disc[2]=2 → edge 2──3 IS A BRIDGE low[4]=4 > disc[2]=2 → edge 2──4 IS A BRIDGE low[2] = min(disc[2], low[3], low[4]) = 2 Node 1: low[2]=2 > disc[1]=1 → edge 1──2 IS A BRIDGE Node 0: low[1]=1 > disc[0]=0 → edge 0──1 IS A BRIDGE All edges in a path graph are bridges (no cycles = every edge is critical). COUNTER-EXAMPLE (cycle protects edges): Graph: 0──1──2──0 (triangle) No bridges because low[v] = 0 for all v (all can reach 0 via back edge).
Section 4 — Articulation Points (Cut Vertices)

An articulation point is a vertex whose removal increases the number of connected components. It's the vertex equivalent of a bridge.

ARTICULATION POINT DETECTION RULE: Node u is an articulation point if EITHER: CASE 1 — u is the ROOT of the DFS tree AND has ≥ 2 children. (Removing root disconnects its subtrees from each other.) CASE 2 — u is NOT the root AND has a child v with low[v] >= disc[u]. (The subtree at v cannot bypass u via a back edge → removing u disconnects v.) NOTE: "child" here means DFS-tree child, not just any neighbor. EXAMPLE: Graph: 1──2──3──4──5 | 6──7 DFS from 1: disc = [0,1,2,3,4,5,6] (for nodes 1..7 in order) Node 3 has children 4 and 6. low[4] = disc[4] = 3 >= disc[3] = 2 → 3 is an ARTICULATION POINT low[6] = disc[6] = 5 >= disc[3] = 2 → 3 is CONFIRMED articulation point Removing node 3 splits: {1,2} | {4,5} | {6,7} ROOT SPECIAL CASE: If start node has 2+ DFS children, removing it disconnects those subtrees. Example: star graph with center 0 → 0 is articulation point (root, 3 children).
Section 5 — Full Implementation
C# 
public class GraphAnalysis
{
    private int n;
    private List<int>[] adj;
    private int[] disc, low;
    private bool[] visited;
    private int timer = 0;

    public GraphAnalysis(int n)
    {
        this.n = n;
        adj  = Enumerable.Range(0, n).Select(_ => new List<int>()).ToArray();
        disc = new int[n]; low = new int[n]; visited = new bool[n];
    }

    public void AddEdge(int u, int v) { adj[u].Add(v); adj[v].Add(u); }

    // ── Find all Bridges ───────────────────────────────────
    public List<(int, int)> FindBridges()
    {
        Array.Fill(visited, false); timer = 0;
        var bridges = new List<(int, int)>();
        for (int i = 0; i < n; i++)
            if (!visited[i]) BridgeDfs(i, -1, bridges);
        return bridges;
    }

    private void BridgeDfs(int u, int parent, List<(int, int)> bridges)
    {
        visited[u] = true;
        disc[u] = low[u] = timer++;

        foreach (int v in adj[u])
        {
            if (!visited[v])
            {
                BridgeDfs(v, u, bridges);
                low[u] = Math.Min(low[u], low[v]);
                if (low[v] > disc[u])             // BRIDGE CONDITION
                    bridges.Add((u, v));
            }
            else if (v != parent)             // back edge (not parent edge)
                low[u] = Math.Min(low[u], disc[v]);
        }
    }

    // ── Find all Articulation Points ───────────────────────
    public HashSet<int> FindArticulationPoints()
    {
        Array.Fill(visited, false); timer = 0;
        var ap = new HashSet<int>();
        for (int i = 0; i < n; i++)
            if (!visited[i]) ApDfs(i, -1, ap);
        return ap;
    }

    private void ApDfs(int u, int parent, HashSet<int> ap)
    {
        visited[u] = true;
        disc[u] = low[u] = timer++;
        int childCount = 0;

        foreach (int v in adj[u])
        {
            if (!visited[v])
            {
                childCount++;
                ApDfs(v, u, ap);
                low[u] = Math.Min(low[u], low[v]);

                if (parent == -1 && childCount > 1) ap.Add(u);  // root case
                if (parent != -1 && low[v] >= disc[u])  ap.Add(u);  // non-root case
            }
            else if (v != parent)
                low[u] = Math.Min(low[u], disc[v]);
        }
    }
}

// ── Usage ───────────────────────────────────────────────
var g = new GraphAnalysis(6);
g.AddEdge(0, 1); g.AddEdge(1, 2); g.AddEdge(2, 0);  // triangle
g.AddEdge(1, 3); g.AddEdge(3, 4); g.AddEdge(4, 5); g.AddEdge(5, 3); // triangle
// Bridges: only edge 1──3 (connects two triangles)
// Articulation point: node 1 and node 3

Console.WriteLine($"Bridges: {string.Join(", ", bridges.Select(b => $"({b.Item1},{b.Item2})"))}");
Console.WriteLine($"Articulation Points: {string.Join(", ", aps)}");
Python 
from collections import defaultdict
import sys; sys.setrecursionlimit(10**5)

class GraphAnalysis:
    def __init__(self, n):
        self.n = n
        self.adj = defaultdict(list)
        self.disc = [0] * n; self.low = [0] * n
        self.visited = [False] * n
        self.timer = 0

    def add_edge(self, u, v):
        self.adj[u].append(v); self.adj[v].append(u)

    # ── Find all Bridges ──────────────────────────────────
    def find_bridges(self):
        self.visited = [False] * self.n; self.timer = 0
        bridges = []
        for i in range(self.n):
            if not self.visited[i]:
                self._bridge_dfs(i, -1, bridges)
        return bridges

    def _bridge_dfs(self, u, parent, bridges):
        self.visited[u] = True
        self.disc[u] = self.low[u] = self.timer; self.timer += 1
        for v in self.adj[u]:
            if not self.visited[v]:
                self._bridge_dfs(v, u, bridges)
                self.low[u] = min(self.low[u], self.low[v])
                if self.low[v] > self.disc[u]:      # BRIDGE CONDITION
                    bridges.append((u, v))
            elif v != parent:                      # back edge
                self.low[u] = min(self.low[u], self.disc[v])

    # ── Find all Articulation Points ──────────────────────
    def find_articulation_points(self):
        self.visited = [False] * self.n; self.timer = 0
        ap = set()
        for i in range(self.n):
            if not self.visited[i]:
                self._ap_dfs(i, -1, ap)
        return ap

    def _ap_dfs(self, u, parent, ap):
        self.visited[u] = True
        self.disc[u] = self.low[u] = self.timer; self.timer += 1
        child_count = 0
        for v in self.adj[u]:
            if not self.visited[v]:
                child_count += 1
                self._ap_dfs(v, u, ap)
                self.low[u] = min(self.low[u], self.low[v])
                if parent == -1 and child_count > 1: ap.add(u)  # root
                if parent != -1 and self.low[v] >= self.disc[u]: ap.add(u)
            elif v != parent:
                self.low[u] = min(self.low[u], self.disc[v])

# ── Usage ──────────────────────────────────────────────
g = GraphAnalysis(6)
g.add_edge(0,1); g.add_edge(1,2); g.add_edge(2,0)  # triangle
g.add_edge(1,3); g.add_edge(3,4); g.add_edge(4,5); g.add_edge(5,3)
print("Bridges:", g.find_bridges())               # [(1, 3)]
print("Articulation Points:", g.find_articulation_points()) # {1, 3}
⚠️ Parallel Edges (Multi-Graph): The basic algorithm above breaks when there are multiple edges between the same two nodes (multi-graph). The fix: track the edge index (not the parent node) to distinguish "going back on the same edge" from "a true back edge via a parallel edge." In LeetCode problems, check constraints — most guarantee simple graphs.
Section 6 — LeetCode: Critical Connections #1192

Problem: There are n servers numbered 0 to n-1. You are given the list of connections as undirected edges. A critical connection is a connection that, if removed, will make some servers unable to reach some other server. Return all critical connections.

📝 This IS the bridge problem. "Critical connection" = bridge. The solution is exactly the Tarjan bridge algorithm above, just applied to this specific graph. Time: O(V + E).
C# 
// LeetCode #1192 — Critical Connections in a Network
public IList<IList<int>> CriticalConnections(int n, IList<IList<int>> connections)
{
    var adj = Enumerable.Range(0, n).Select(_ => new List<int>()).ToArray();
    foreach (var c in connections) { adj[c[0]].Add(c[1]); adj[c[1]].Add(c[0]); }

    int[] disc = new int[n], low = new int[n];
    Array.Fill(disc, -1);
    int timer = 0;
    var result = new List<IList<int>>();

    void Dfs(int u, int parent)
    {
        disc[u] = low[u] = timer++;
        foreach (int v in adj[u])
        {
            if (disc[v] == -1)
            {
                Dfs(v, u);
                low[u] = Math.Min(low[u], low[v]);
                if (low[v] > disc[u]) result.Add(new[] { u, v });
            }
            else if (v != parent) low[u] = Math.Min(low[u], disc[v]);
        }
    }
    for (int i = 0; i < n; i++) if (disc[i] == -1) Dfs(i, -1);
    return result;
}
// n=4, connections=[[0,1],[1,2],[2,0],[1,3]] → [[1,3]]
Python 
def criticalConnections(n, connections):
    adj = defaultdict(list)
    for u, v in connections: adj[u].append(v); adj[v].append(u)

    disc = [-1] * n; low = [0] * n; timer = [0]; result = []

    def dfs(u, parent):
        disc[u] = low[u] = timer[0]; timer[0] += 1
        for v in adj[u]:
            if disc[v] == -1:
                dfs(v, u)
                low[u] = min(low[u], low[v])
                if low[v] > disc[u]: result.append([u, v])
            elif v != parent: low[u] = min(low[u], disc[v])

    for i in range(n):
        if disc[i] == -1: dfs(i, -1)
    return result

Section 7 — Q&A Review
Q1: What is the difference between a bridge and an articulation point?
A bridge is an edge — removing it increases connected components. An articulation point is a vertex — removing it (along with all its incident edges) increases connected components. They're related but distinct: a bridge's endpoints are often articulation points, but an articulation point doesn't necessarily have a bridge incident to it (e.g., the center of a "bowtie" graph — two triangles sharing one vertex — is an AP but the edges are not bridges). Both are detectable in O(V+E) using DFS.
Q2: What does low[v] represent, and how is it computed?
low[v] = the minimum discovery time reachable from the subtree rooted at v in the DFS tree. It's computed as: low[v] = min(disc[v], min(low[child]) for each DFS-tree child, min(disc[w]) for each back edge (v,w)). Intuitively, low[v] tells us "can this subtree escape upward in the tree via a back edge?" If low[v] > disc[u], the subtree at v is "trapped" below u — there's no back edge reaching u or above, meaning edge (u,v) is a bridge and u is a potential articulation point.
Q3: Why is the root node of the DFS tree treated as a special case for articulation points?
For non-root nodes, the condition low[child] >= disc[u] works because: if the subtree at child can't reach any ancestor of u via a back edge, removing u disconnects child from the rest. But for the ROOT, it has no parent, so the "can it reach above u" concept doesn't apply the same way. The root is an articulation point if and only if it has 2 or more DFS children. Removing the root disconnects those child subtrees from each other. With only 1 child, the root is just a leaf's perspective — removing it leaves one connected piece (the child's subtree), so it's not an AP.
Q4: How does the algorithm handle multi-graphs (parallel edges between the same two nodes)?
The naive approach (skip neighbor == parent) breaks with parallel edges because two separate edges between u and v form a "cycle" — neither should be a bridge. Fix: track edge indices instead of parent nodes. Each edge gets an index. When at node u coming from edge id, skip edge id (not all edges to parent). This way, a second parallel edge from u back to v is correctly treated as a back edge (alternative path), preventing false bridge detection. In Python: adj[u].append((v, edge_id)) and skip if eid == parent_edge: continue.
Q5: Can you find bridges iteratively (without recursion)? Why might you need to?
Yes — the recursive DFS will stack-overflow on large graphs (n = 10^5+). Iterative version: use an explicit stack of (node, parent, iterator_position). Maintain the iterator over each node's adjacency list. When you push a new node, save the adjacency list iterator for the parent. When you pop (finish a node), update parent's low value. Python's default recursion limit is 1000 — set sys.setrecursionlimit(10**6) or use iterative DFS for large inputs. C# default stack is ~1MB; for n=50k+ nodes, either increase stack size (new Thread(...) { ... }.Start() with StackSize) or convert to iterative.
