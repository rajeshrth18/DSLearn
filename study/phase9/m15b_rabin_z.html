Section 1 — Rabin-Karp: Rolling Hash

Rabin-Karp uses hashing to speed up pattern matching. Instead of comparing characters one by one, it computes a hash of the pattern and slides a rolling window of the same length across the text, comparing hashes. Only when hashes match do we verify character-by-character.

ROLLING HASH — Text: "abcabc", Pattern: "cab" (m=3) Hash("cab") = hash_value Window 1: "abc" → compute hash → compare with pattern hash Window 2: "bca" → ROLL: subtract 'a', shift, add 'a' (O(1) update!) Window 3: "cab" → ROLL: subtract 'b', shift, add 'b' → hash matches! → verify "cab" == "cab" → FOUND at index 2 Window 4: "abc" → roll... ROLLING FORMULA: hash(s[i+1..i+m]) = (hash(s[i..i+m-1]) - s[i] * base^(m-1)) * base + s[i+m] Using modular arithmetic to prevent overflow: hash = hash * base + char_value (when building) hash = (hash - leading_char * pow_mod) * base + new_char (when rolling)

1.1 — Why Hash? Why Rolling?

  • Average case O(n + m): Hash comparison is O(1), only do O(m) comparison on match
  • Worst case O(n·m): If all hash values match (hash collision), we verify each time
  • Ideal for multiple patterns: Hash all patterns, check each window against the set
  • Rolling: Update hash in O(1) by removing leading char and adding trailing char
C#
static List<int> RabinKarp(string text, string pattern)
{
    var result = new List<int>();
    int n = text.Length, m = pattern.Length;
    if (m > n) return result;

    const long BASE = 31, MOD = 1_000_000_007;

    // Compute BASE^(m-1) % MOD
    long pow = 1;
    for (int i = 0; i < m - 1; i++) pow = pow * BASE % MOD;

    // Hash function: map 'a'→1, 'b'→2, ... (avoid 0 for empty prefix)
    static long CharVal(char c) => c - 'a' + 1;

    long patHash = 0, winHash = 0;
    for (int i = 0; i < m; i++)
    {
        patHash = (patHash * BASE + CharVal(pattern[i])) % MOD;
        winHash = (winHash * BASE + CharVal(text[i])) % MOD;
    }

    for (int i = 0; i <= n - m; i++)
    {
        if (winHash == patHash)
        {
            // Verify to avoid false positive (hash collision)
            if (text.Substring(i, m) == pattern)
                result.Add(i);
        }
        if (i < n - m)
        {
            // Roll the window: remove leading, add trailing
            winHash = (winHash - CharVal(text[i]) * pow % MOD + MOD) % MOD;
            winHash = (winHash * BASE + CharVal(text[i + m])) % MOD;
        }
    }
    return result;
}

// Usage
var found = RabinKarp("abcabcabc", "abc");
// Output: [0, 3, 6]
Python
def rabin_karp(text: str, pattern: str) -> list[int]:
    n, m = len(text), len(pattern)
    if m > n:
        return []
    BASE, MOD = 31, 10**9 + 7
    char_val = lambda c: ord(c) - ord('a') + 1

    # Compute BASE^(m-1) mod MOD
    pow_val = pow(BASE, m - 1, MOD)

    pat_hash = win_hash = 0
    for i in range(m):
        pat_hash = (pat_hash * BASE + char_val(pattern[i])) % MOD
        win_hash = (win_hash * BASE + char_val(text[i])) % MOD

    result = []
    for i in range(n - m + 1):
        if win_hash == pat_hash and text[i:i+m] == pattern:
            result.append(i)
        if i < n - m:
            win_hash = (win_hash - char_val(text[i]) * pow_val) % MOD
            win_hash = (win_hash * BASE + char_val(text[i + m])) % MOD
    return result

print(rabin_karp("abcabcabc", "abc"))  # [0, 3, 6]
Section 2 — Z-Algorithm

The Z-algorithm computes the Z-array for a string s. Z[i] = length of the longest substring starting at s[i] that is also a prefix of s. By definition, Z[0] is undefined (or the full string length).

String: "aabxaab" Index: 0 1 2 3 4 5 6 Z[0] = 7 (by convention, entire string) Z[1] = 1 → "a" matches prefix "a" (next char 'b' ≠ 'a') Z[2] = 0 → "b" ≠ 'a' Z[3] = 0 → "x" ≠ 'a' Z[4] = 3 → "aab" matches prefix "aab" (s[4..6] = "aab" = s[0..2]) Z[5] = 1 → "ab" → 'a' matches, 'b' ≠ 'b'... wait "ab" vs prefix "aab" → 'a'='a', 'b'='a'? No → Z[5]=1 Z[6] = 0 → "b" ≠ 'a' Z = [7, 1, 0, 0, 3, 1, 0] Pattern matching trick: concat "pattern#text", compute Z. Any Z[i] == len(pattern) at position > len(pattern)+1 → match found!

2.1 — Building the Z-Array

The Z-algorithm maintains a window [l, r] — the rightmost Z-box (a substring that matches a prefix). This allows O(n) construction.

C#
static int[] BuildZArray(string s)
{
    int n = s.Length;
    int[] z = new int[n];
    int l = 0, r = 0;  // current Z-box boundaries [l, r]

    for (int i = 1; i < n; i++)
    {
        if (i < r)
            z[i] = Math.Min(r - i, z[i - l]);  // use known Z-box info

        while (i + z[i] < n && s[z[i]] == s[i + z[i]])
            z[i]++;   // extend match

        if (i + z[i] > r)
        {
            l = i;
            r = i + z[i];  // update rightmost Z-box
        }
    }
    return z;
}

static List<int> ZSearch(string text, string pattern)
{
    string combined = pattern + "$" + text;  // '$' won't appear in text
    int[] z = BuildZArray(combined);
    var result = new List<int>();
    int offset = pattern.Length + 1;

    for (int i = offset; i < combined.Length; i++)
        if (z[i] == pattern.Length)
            result.Add(i - offset);

    return result;
}

// Usage
var hits = ZSearch("aabcaabc", "aabc");
// Output: [0, 4]
Python
def build_z_array(s: str) -> list[int]:
    n = len(s)
    z = [0] * n
    l = r = 0
    for i in range(1, n):
        if i < r:
            z[i] = min(r - i, z[i - l])
        while i + z[i] < n and s[z[i]] == s[i + z[i]]:
            z[i] += 1
        if i + z[i] > r:
            l, r = i, i + z[i]
    return z

def z_search(text: str, pattern: str) -> list[int]:
    combined = pattern + "$" + text
    z = build_z_array(combined)
    m, offset = len(pattern), len(pattern) + 1
    return [i - offset for i in range(offset, len(combined)) if z[i] == m]

print(z_search("aabcaabc", "aabc"))  # [0, 4]

3.1 — Rabin-Karp for Multiple Patterns

Rabin-Karp shines when searching for multiple patterns simultaneously. Hash all patterns into a set, then

C#
public static Dictionary<string, List<int>> MultiPatternSearch(string text, string[] patterns)
{
    const long BASE = 31, MOD = 1_000_000_007;
    long CV(char c) => c - 'a' + 1;
    var byLen = patterns.GroupBy(p => p.Length).ToDictionary(
        g => g.Key,
        g => g.ToDictionary(
            p => { long h=0; foreach(var c in p) h=(h*BASE+CV(c))%MOD; return h; },
            p => p));
    var res = new Dictionary<string, List<int>>();
    int n = text.Length;
    foreach (var (m, ph) in byLen) {
        if (m > n) continue;
        long pw=1; for (int i=0; i<m-1; i++) pw=pw*BASE%MOD;
        long wh=0; for (int i=0; i<m; i++) wh=(wh*BASE+CV(text[i]))%MOD;
        for (int i=0; i<=n-m; i++) {
            if (ph.TryGetValue(wh, out var p) && text.Substring(i,m)==p) {
                if (!res.ContainsKey(p)) res[p]=new(); res[p].Add(i);
            }
            if (i<n-m) wh=((wh-CV(text[i])*pw%MOD+MOD)*BASE+CV(text[i+m]))%MOD;
        }
    }
    return res;
}
var r = MultiPatternSearch("abcabdabc", new[] { "abc", "abd" });
// {"abc": [0, 6], "abd": [3]}
Python
def multi_pattern_search(text: str, patterns: list[str]) -> dict:
    """Find all occurrences of each pattern in text using rolling hashes."""
    BASE, MOD = 31, 10**9 + 7
    char_val = lambda c: ord(c) - ord('a') + 1

    # Group patterns by length (only same-length windows share hash windows)
    from collections import defaultdict
    by_len = defaultdict(dict)
    for p in patterns:
        h = 0
        for c in p:
            h = (h * BASE + char_val(c)) % MOD
        by_len[len(p)][h] = p

    results = defaultdict(list)
    n = len(text)

    for m, pat_hashes in by_len.items():
        if m > n:
            continue
        pow_val = pow(BASE, m - 1, MOD)
        win_hash = 0
        for c in text[:m]:
            win_hash = (win_hash * BASE + char_val(c)) % MOD

        for i in range(n - m + 1):
            if win_hash in pat_hashes:
                p = pat_hashes[win_hash]
                if text[i:i+m] == p:
                    results[p].append(i)
            if i < n - m:
                win_hash = (win_hash - char_val(text[i]) * pow_val) % MOD
                win_hash = (win_hash * BASE + char_val(text[i+m])) % MOD
    return dict(results)

result = multi_pattern_search("abcabdabc", ["abc", "abd"])
# {'abc': [0, 6], 'abd': [3]}
📌 Key Takeaways:
  • Rabin-Karp: Best for multiple pattern search; O(n + Σm) average with a hash set
  • Z-Algorithm: Simple O(n) prefix matching; useful for "does pattern appear as prefix at each position"
  • Both use the pattern + separator + text concatenation trick
  • Choose MOD as a large prime and BASE as a prime to minimize collisions
