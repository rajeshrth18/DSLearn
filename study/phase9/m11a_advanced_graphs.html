1. Union-Find (Disjoint Set Union)

Union-Find tracks connected components in a graph. Two operations: Find (which component is this node in?) and Union (merge two components). With optimizations, both run in near-O(1) amortized time.

Initial state: each node is its own parent Nodes: [0] [1] [2] [3] [4] parent: [0, 1, 2, 3, 4] rank: [0, 0, 0, 0, 0] Union(0,1): merge sets containing 0 and 1 parent: [0, 0, 2, 3, 4] (1's root becomes 0) Union(2,3): merge 2 and 3 parent: [0, 0, 2, 2, 4] Union(0,2): merge {0,1} and {2,3} parent: [0, 0, 0, 2, 4] (root of 2's tree → 0) Path Compression: when finding root, flatten the tree Find(3): 3→2→0, then set parent[3]=0 directly parent: [0, 0, 0, 0, 4] ← flat!
public class UnionFind
{
    private int[] parent, rank;

    public UnionFind(int n)
    {
        parent = Enumerable.Range(0, n).ToArray(); // parent[i] = i initially
        rank   = new int[n];
    }

    // Find root with path compression (flatten tree)
    public int Find(int x)
    {
        if (parent[x] != x)
            parent[x] = Find(parent[x]); // Path compression: point directly to root
        return parent[x];
    }

    // Union by rank: attach smaller tree under larger tree's root
    public bool Union(int x, int y)
    {
        int rx = Find(x), ry = Find(y);
        if (rx == ry) return false; // Already in same set → would form cycle

        if (rank[rx] < rank[ry]) (rx, ry) = (ry, rx); // Make rx the taller root
        parent[ry] = rx;
        if (rank[rx] == rank[ry]) rank[rx]++;
        return true;
    }

    public bool Connected(int x, int y) => Find(x) == Find(y);
}
// Time: O(α(n)) per operation — α is the inverse Ackermann function, effectively O(1)
// Space: O(n)

Classic Use Case: Detect Cycle in Undirected Graph

public bool HasCycle(int n, int[][] edges)
{
    var uf = new UnionFind(n);
    foreach (var edge in edges)
    {
        // If both endpoints already share a root, adding this edge creates a cycle
        if (!uf.Union(edge[0], edge[1])) return true;
    }
    return false;
}

Number of Connected Components (LeetCode 323)

public int CountComponents(int n, int[][] edges)
{
    var uf = new UnionFind(n);
    int components = n;
    foreach (var e in edges)
        if (uf.Union(e[0], e[1])) components--;
    return components;
}

2. Minimum Spanning Tree — Kruskal's Algorithm

A Minimum Spanning Tree (MST) connects all n nodes with n-1 edges of minimum total weight. Kruskal's sorts edges by weight and adds them greedily if they don't form a cycle (detected via Union-Find).

Graph edges sorted by weight: (0-1,w=2),(1-2,w=3),(1-4,w=4),(2-5,w=5),(0-3,w=6),(3-4,w=7) Process: Add (0-1, w=2): ✓ no cycle MST edges: 1 Add (1-2, w=3): ✓ no cycle MST edges: 2 Add (1-4, w=4): ✓ no cycle MST edges: 3 Add (2-5, w=5): ✓ no cycle MST edges: 4 Add (0-3, w=6): ✓ no cycle MST edges: 5 = n-1 ← DONE Total weight: 2+3+4+5+6 = 20
public int MinCostConnectPoints(int[][] edges, int n)
{
    // edges = [u, v, weight]; sort by weight ascending
    Array.Sort(edges, (a, b) => a[2] - b[2]);

    var uf   = new UnionFind(n);
    int cost = 0, edgesUsed = 0;

    foreach (var e in edges)
    {
        if (!uf.Union(e[0], e[1])) continue; // Skip: would form cycle
        cost += e[2];
        if (++edgesUsed == n - 1) break;     // MST complete
    }
    return cost;
}
// Time: O(E log E) for sorting, O(E α(V)) for Union-Find → O(E log E)

3. Minimum Spanning Tree — Prim's Algorithm

Prim's grows the MST from a starting node by always picking the cheapest edge that connects the current MST to a new node. Uses a min-heap (priority queue) instead of sorting all edges upfront.

Key difference from Kruskal's: Kruskal's: global sort of all edges, union-find to avoid cycles Prim's: grow from one node, min-heap picks cheapest frontier edge Best for dense graphs (Prim's) vs sparse graphs (Kruskal's)
public int PrimMST(int n, List<(int to, int w)>[] adj)
{
    // Min-heap: (weight, node)
    var pq      = new PriorityQueue<(int w, int node), int>();
    bool[] inMST = new bool[n];
    int totalCost = 0, edgesUsed = 0;

    pq.Enqueue((0, 0), 0); // Start from node 0 with cost 0

    while (pq.Count > 0 && edgesUsed < n)
    {
        var (w, u) = pq.Dequeue();
        if (inMST[u]) continue; // Already in MST

        inMST[u] = true;
        totalCost += w;
        edgesUsed++;

        foreach (var (to, weight) in adj[u])
            if (!inMST[to]) pq.Enqueue((weight, to), weight);
    }
    return totalCost;
}
// Time: O((V + E) log V) with binary heap

4. Bellman-Ford Algorithm

Dijkstra's fails with negative edge weights. Bellman-Ford handles negative weights and can detect negative cycles (a cycle whose total weight is negative, making shortest path undefined).

Why Dijkstra fails with negative edges: A →(4)→ B →(-6)→ C A →(2)→ C Dijkstra greedily visits A→C first (cost 2), marks C as "done" But A→B→C costs 4+(-6) = -2 which is BETTER — Dijkstra missed it! Bellman-Ford: relax ALL edges V-1 times After each "round", guaranteed to find shortest paths using ≤ k edges After V-1 rounds, all shortest paths found (longest simple path = V-1 edges) Round V: if any edge still relaxes → NEGATIVE CYCLE detected
public int[] BellmanFord(int n, int[][] edges, int src)
{
    int[] dist = Enumerable.Repeat(int.MaxValue, n).ToArray();
    dist[src] = 0;

    // Relax all edges V-1 times
    for (int i = 0; i < n - 1; i++)
    {
        foreach (var e in edges) // e = [u, v, weight]
        {
            int u = e[0], v = e[1], w = e[2];
            if (dist[u] != int.MaxValue && dist[u] + w < dist[v])
                dist[v] = dist[u] + w;
        }
    }

    // V-th round: detect negative cycle
    foreach (var e in edges)
        if (dist[e[0]] != int.MaxValue && dist[e[0]] + e[2] < dist[e[1]])
            throw new Exception("Negative cycle detected");

    return dist;
}
// Time: O(V * E), Space: O(V)
// Use case: currency arbitrage detection, network routing with negative costs

Cheapest Flights Within K Stops (LeetCode 787)

// Bellman-Ford variant: at most k+1 edges (k stops = k+1 flights)
public int FindCheapestPrice(int n, int[][] flights, int src, int dst, int k)
{
    int[] dist = Enumerable.Repeat(int.MaxValue, n).ToArray();
    dist[src] = 0;

    for (int i = 0; i <= k; i++) // k+1 iterations = at most k+1 edges
    {
        int[] temp = (int[])dist.Clone(); // Snapshot: only use THIS round's distances
        foreach (var f in flights)
        {
            int u = f[0], v = f[1], w = f[2];
            if (dist[u] != int.MaxValue && dist[u] + w < temp[v])
                temp[v] = dist[u] + w;
        }
        dist = temp;
    }
    return dist[dst] == int.MaxValue ? -1 : dist[dst];
}
// Key: use temp array — prevents relaxing within same iteration (would allow > k stops)

5. Floyd-Warshall — All-Pairs Shortest Paths

Find shortest paths between every pair of nodes in O(V³). Handles negative weights (not negative cycles). Uses DP: can we improve path i→j by going through intermediate node k?

DP definition: dist[i][j] = shortest path from i to j Recurrence: for each intermediate node k: dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]) Process all k from 0..n-1. After considering k, dist[i][j] holds the shortest path using only nodes 0..k as intermediates.
public int[,] FloydWarshall(int n, int[][] edges)
{
    int[,] dist = new int[n, n];

    // Initialize: same node=0, direct edge=weight, else infinity
    for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        dist[i, j] = (i == j) ? 0 : int.MaxValue / 2; // /2 avoids overflow in addition

    foreach (var e in edges)
        dist[e[0], e[1]] = e[2];

    // Try every node k as an intermediate
    for (int k = 0; k < n; k++)
    for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        if (dist[i, k] + dist[k, j] < dist[i, j])
            dist[i, j] = dist[i, k] + dist[k, j];

    // Negative cycle check: if dist[i,i] < 0, negative cycle exists
    return dist;
}
// Time: O(V³), Space: O(V²)
// Use when: dense graph, all-pairs needed, small n (≤ ~400)
Algorithm Comparison — Shortest Path:
  • Dijkstra: single-source, non-negative weights, O((V+E) log V)
  • Bellman-Ford: single-source, negative weights ok, O(V*E)
  • Floyd-Warshall: all-pairs, negative weights ok, O(V³)

6. Topological Sort — Kahn's BFS Algorithm

Kahn's algorithm uses in-degree counts to find topological order iteratively. Start with all nodes of in-degree 0, process them, reduce neighbors' in-degrees, and enqueue any new zeros.

Course prerequisites: 0→1, 0→2, 1→3, 2→3 In-degrees: [0:0, 1:1, 2:1, 3:2] Queue: [0] (in-degree 0) Process 0: reduce in-degree of 1→0, 2→0. Queue: [1, 2] Process 1: reduce in-degree of 3→1. Queue: [2] Process 2: reduce in-degree of 3→0. Queue: [3] Process 3: done. Order: [0, 1, 2, 3] If processed count < n → CYCLE EXISTS (can't complete all courses)
public int[] TopoSortKahn(int n, int[][] prerequisites)
{
    var adj     = new List<int>[n].Select(_ => new List<int>()).ToArray();
    int[] inDeg = new int[n];

    foreach (var p in prerequisites)
    {
        adj[p[1]].Add(p[0]); // p[1] must come before p[0]
        inDeg[p[0]]++;
    }

    var queue  = new Queue<int>();
    var result = new List<int>();

    for (int i = 0; i < n; i++)
        if (inDeg[i] == 0) queue.Enqueue(i);

    while (queue.Count > 0)
    {
        int node = queue.Dequeue();
        result.Add(node);
        foreach (int next in adj[node])
            if (--inDeg[next] == 0) queue.Enqueue(next);
    }

    return result.Count == n ? result.ToArray() : Array.Empty<int>(); // Empty = cycle
}
// Time: O(V + E), Space: O(V + E)

7. Alien Dictionary (Topological Sort Application)

Given a sorted list of words in an alien language, determine the ordering of characters. Build a directed graph from character comparisons between adjacent words.

public string AlienOrder(string[] words)
{
    var adj    = new Dictionary<char, HashSet<char>>();
    var inDeg  = new Dictionary<char, int>();

    foreach (var w in words)
        foreach (char c in w)
        {
            adj[c]   = new HashSet<char>();
            inDeg[c] = 0;
        }

    // Compare adjacent words to extract ordering
    for (int i = 0; i < words.Length - 1; i++)
    {
        string a = words[i], b = words[i + 1];
        int len = Math.Min(a.Length, b.Length);

        if (a.Length > b.Length && a.StartsWith(b)) return ""; // Invalid: "abc" before "ab"

        for (int j = 0; j < len; j++)
        {
            if (a[j] == b[j]) continue;
            if (!adj[a[j]].Contains(b[j]))
            {
                adj[a[j]].Add(b[j]);
                inDeg[b[j]]++;
            }
            break;
        }
    }

    // Kahn's topological sort on character DAG
    var queue = new Queue<char>(inDeg.Where(kv => kv.Value == 0).Select(kv => kv.Key));
    var sb    = new StringBuilder();

    while (queue.Count > 0)
    {
        char c = queue.Dequeue(); sb.Append(c);
        foreach (char next in adj[c])
            if (--inDeg[next] == 0) queue.Enqueue(next);
    }
    return sb.Length == inDeg.Count ? sb.ToString() : ""; // "" = cycle
}

2. Minimum Spanning Tree — Kruskal's Algorithm

A Minimum Spanning Tree (MST) connects all n nodes with n-1 edges of minimum total weight, with no cycles. Kruskal's algorithm sorts edges by weight and greedily adds them if they don't form a cycle (using Union-Find).
