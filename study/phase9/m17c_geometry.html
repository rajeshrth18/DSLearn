Section 1 — Points & Vectors
FUNDAMENTAL CONCEPTS: Point P = (x, y) — coordinates on 2D plane Vector v = B - A = (B.x - A.x, B.y - A.y) — direction + magnitude DISTANCE: |AB| = sqrt((B.x-A.x)^2 + (B.y-A.y)^2) — Euclidean distance Manhattan: |x1-x2| + |y1-y2| Note: avoid sqrt when possible — compare squared distances to avoid float errors DOT PRODUCT: A · B = A.x * B.x + A.y * B.y = |A| * |B| * cos(θ) - Positive: angle between vectors < 90° - Zero: perpendicular (θ = 90°) - Negative: angle > 90° (obtuse) Test if angle AOB is acute/right/obtuse: sign of (B-O) · (A-O) ANGLE of vector (x, y): atan2(y, x) ← note: atan2(y, x) not atan2(x, y)! Returns angle in [-π, π] For polar sort by angle: compare atan2 values or use cross product sign ⚠ FLOATING POINT: Use long integer coordinates when possible to avoid precision errors. Add epsilon (1e-9) for comparisons: if abs(a - b) < EPS: treat as equal
C# — Point struct 
record struct P(long X, long Y)   // integer coordinates for exact arithmetic
{
    public static P operator -(P a, P b) => new(a.X - b.X, a.Y - b.Y);
    public static P operator +(P a, P b) => new(a.X + b.X, a.Y + b.Y);
    public long Dist2(P other) => (X - other.X) * (X - other.X) + (Y - other.Y) * (Y - other.Y);
    public long Dot(P other) => X * other.X + Y * other.Y;
    public long Cross(P other) => X * other.Y - Y * other.X;  // 2D cross product
    public double Angle() => Math.Atan2(Y, X);
}

Section 2 — Cross Product (The Core Tool)
2D CROSS PRODUCT of vectors A and B: cross(A, B) = A.x * B.y - A.y * B.x = |A| * |B| * sin(θ) > 0: B is to the LEFT of A (counterclockwise turn / left turn) = 0: A and B are COLLINEAR (parallel or antiparallel) < 0: B is to the RIGHT of A (clockwise turn / right turn) TURN DIRECTION at point B on path A→B→C: cross(B-A, C-A) — or equivalently cross(C-B, A-B) > 0: left turn (counterclockwise) = 0: collinear (no turn) < 0: right turn (clockwise) Example: A=(0,0), B=(1,0), C=(1,1) B-A = (1,0), C-A = (1,1) cross = 1*1 - 0*1 = 1 > 0 → left turn (CCW) AREA of triangle OAB (O at origin): Area = |cross(A, B)| / 2 AREA of polygon with vertices P[0..n-1]: 2 * Area = |Σ cross(P[i], P[(i+1) % n])| ← Shoelace formula (Works for convex and concave polygons, not self-intersecting) COLLINEARITY: Three points A, B, C are collinear iff cross(B-A, C-A) == 0
C# 
// Cross product of vectors OA and OB (all as P structs)
long Cross(P O, P A, P B)
    => (A - O).Cross(B - O);  // = (A.X-O.X)*(B.Y-O.Y) - (A.Y-O.Y)*(B.X-O.X)

// Area of polygon (×2, as long, to avoid float)
long Area2(P[] pts)
{
    long s = 0;
    int n = pts.Length;
    for (int i = 0; i < n; i++)
        s += new P(0, 0).Cross(pts[i], pts[(i + 1) % n]); // or just Shoelace
    return Math.Abs(s);
}

// Shoelace formula (Gauss area formula)
long Shoelace(P[] pts)
{
    long s = 0; int n = pts.Length;
    for (int i = 0; i < n; i++)
        s += pts[i].X * pts[(i + 1) % n].Y - pts[(i + 1) % n].X * pts[i].Y;
    return Math.Abs(s); // divide by 2 for actual area
}
Python 
def cross3(O, A, B):
    """Cross product of vectors OA and OB."""
    return (A[0]-O[0]) * (B[1]-O[1]) - (A[1]-O[1]) * (B[0]-O[0])

def shoelace(pts):
    n = len(pts)
    s = sum(pts[i][0]*pts[(i+1)%n][1] - pts[(i+1)%n][0]*pts[i][1] for i in range(n))
    return abs(s) / 2
Section 3 — Convex Hull (Graham Scan)
CONVEX HULL: Smallest convex polygon containing all given points. Think of: stretching a rubber band around all the points. GRAHAM SCAN ALGORITHM: O(n log n) 1. Find the lowest point (min y, then min x). Call it P0. This is always on the hull. 2. Sort all other points by polar angle relative to P0. (Use cross product to compare: if cross(P0→A, P0→B) > 0 → A comes before B) Ties: by distance (closer first or further first depending on application) 3. Process points in sorted order using a STACK: - For each point C: while the last two stack points A, B form a non-left turn (cross(A, B, C) ≤ 0), pop B from stack. - Push C. 4. Stack = convex hull in CCW order. MONOTONE CHAIN (Andrew's algorithm) — often preferred, no angle sort needed: 1. Sort points by (x, y). 2. Build LOWER hull: iterate L→R, keep CCW (pop if right turn or collinear). 3. Build UPPER hull: iterate R→L, same rule. 4. Combine (remove duplicate endpoints). Time: O(n log n) from sort, O(n) hull construction. COLLINEAR POINTS on hull edge: Standard: pop if cross ≤ 0 (excludes collinear). Include collinear on edges: pop if cross < 0 (strict right turn only).
C# — Monotone Chain 
public P[] ConvexHull(P[] pts)
{
    int n = pts.Length;
    if (n < 2) return pts;
    P[] sorted = pts.OrderBy(p => p.X).ThenBy(p => p.Y).ToArray();
    var hull = new P[2 * n]; int k = 0;
    
    // Lower hull
    for (int i = 0; i < n; i++) {
        while (k >= 2 && Cross(hull[k - 2], hull[k - 1], sorted[i]) <= 0) k--;
        hull[k++] = sorted[i];
    }
    // Upper hull
    for (int i = n - 2, t = k + 1; i >= 0; i--) {
        while (k >= t && Cross(hull[k - 2], hull[k - 1], sorted[i]) <= 0) k--;
        hull[k++] = sorted[i];
    }
    return hull[..(k - 1)];
}
Python — Monotone Chain 
def convex_hull(pts):
    pts = sorted(set(pts))
    if len(pts) <= 1: return pts
    def build(pts):
        h = []
        for p in pts:
            while len(h) >= 2 and cross3(h[-2], h[-1], p) <= 0: h.pop()
            h.append(p)
        return h
    return build(pts) + build(pts[::-1])  # lower + upper, first/last shared
Section 4 — Segment Intersection
DO SEGMENTS AB AND CD INTERSECT? GENERAL CASE (using orientation test): Segment AB intersects CD iff: 1. A and B are on OPPOSITE sides of line CD, AND 2. C and D are on OPPOSITE sides of line AB. "Opposite sides" = cross products have opposite signs. cross(C, D, A) * cross(C, D, B) < 0 AND cross(A, B, C) * cross(A, B, D) < 0 DEGENERATE CASES (collinear points): If cross = 0, the point is ON the line. Check if it's on the segment: Point P is on segment AB iff collinear AND min(A.x, B.x) ≤ P.x ≤ max(A.x, B.x) AND similar for y. FIND INTERSECTION POINT of lines AB and CD: Parametric: P = A + t*(B-A) Substitute into line CD, solve for t: t = cross(C-A, D-C) / cross(B-A, D-C) Intersection = A + t*(B-A) C# — Segment Intersection 
// Assumes Cross3(p,q,r) = (q.x-p.x)*(r.y-p.y)-(q.y-p.y)*(r.x-p.x)
static bool OnSegment((double x, double y) p,
    (double x, double y) a, (double x, double y) b)
    => Math.Min(a.x,b.x)<=p.x && p.x<=Math.Max(a.x,b.x) &&
       Math.Min(a.y,b.y)<=p.y && p.y<=Math.Max(a.y,b.y);

static bool SegmentsIntersect(
    (double x, double y) a, (double x, double y) b,
    (double x, double y) c, (double x, double y) d)
{
    double d1=Cross3(c,d,a), d2=Cross3(c,d,b),
           d3=Cross3(a,b,c), d4=Cross3(a,b,d);
    if (d1*d2<0 && d3*d4<0) return true;
    if (d1==0 && OnSegment(a,c,d)) return true;
    if (d2==0 && OnSegment(b,c,d)) return true;
    if (d3==0 && OnSegment(c,a,b)) return true;
    if (d4==0 && OnSegment(d,a,b)) return true;
    return false;
}
Python — Segment Intersection 
def on_segment(p, a, b):
    """Check if p lies on segment ab (assuming p is collinear with a, b)."""
    return (min(a[0],b[0]) <= p[0] <= max(a[0],b[0]) and
            min(a[1],b[1]) <= p[1] <= max(a[1],b[1]))

def segments_intersect(a, b, c, d):
    """Return True if segment AB intersects segment CD."""
    d1, d2 = cross3(c, d, a), cross3(c, d, b)
    d3, d4 = cross3(a, b, c), cross3(a, b, d)
    if d1*d2 < 0 and d3*d4 < 0: return True
    if d1 == 0 and on_segment(a, c, d): return True
    if d2 == 0 and on_segment(b, c, d): return True
    if d3 == 0 and on_segment(c, a, b): return True
    if d4 == 0 and on_segment(d, a, b): return True
    return False
Section 5 — Rotating Calipers & LeetCode Map
ROTATING CALIPERS — Maximum Distance Between Two Points on Convex Hull: The diameter of a point set = max distance between any two points. Brute force: O(n^2). Rotating calipers on convex hull: O(n). Algorithm: 1. Build convex hull (O(n log n)). 2. Find the antipodal point: for hull[0], find the farthest point. 3. Rotate one caliper along the hull; advance the antipodal pointer as long as it increases distance. 4. Track maximum distance throughout. The key: as we advance the "base" edge, the antipodal point can only move forward (never backward) — total O(n) pointer movement. MINIMUM BOUNDING RECTANGLE: Also O(n) with rotating calipers after convex hull. Rotate an edge of the hull as one side of the rectangle; find the support points (min/max projections on perpendicular axis). Track minimum area rectangle. POINT-IN-CONVEX-POLYGON TEST: O(log n) Binary search on the fan triangulation from a fixed pivot. Much faster than O(n) for general polygon when hull is sorted.

Section 6 — Q&A Review
Q1: Why is the cross product the fundamental tool in computational geometry?
The 2D cross product cross(A, B) = A.x*B.y - A.y*B.x encodes orientation information: its sign tells you which side B is on relative to A, without needing angles or trigonometry. This makes it exact with integer arithmetic (no floating-point errors) and O(1). Applications: (1) Turn direction (left/right/straight) when traversing a path — core of convex hull. (2) Collinearity test — three points are collinear iff cross = 0. (3) Half-plane test — which side of a line is a point on. (4) Area of triangle = |cross(A,B)|/2. (5) Segment intersection — check if endpoints are on opposite sides. Almost every geometric algorithm reduces to a sequence of cross product sign checks.
Q2: Trace through the monotone chain convex hull algorithm on points: (0,0),(1,1),(2,0),(1,2),(0,2).
Sorted by (x,y): (0,0),(0,2),(1,1),(1,2),(2,0). Lower hull (L→R): Add (0,0)→h=[(0,0)]. Add (0,2)→h=[(0,0),(0,2)]. Add (1,1): cross((0,0),(0,2),(1,1))=(0*1-2*1)=-2≤0, pop (0,2). h=[(0,0),(1,1)]. Add (1,2): cross((0,0),(1,1),(1,2))=1*2-1*1=1>0, keep. h=[(0,0),(1,1),(1,2)]. Add (2,0): cross((1,1),(1,2),(2,0))=(0*(-1)-1*1)=-1≤0 pop. cross((0,0),(1,1),(2,0))=(1*0-1*2)=-2≤0 pop. h=[(0,0),(2,0)]. Upper hull (R→L, t=k+1=3): starts from (1,2). Build remaining points: (1,2),(1,1),(0,2),(0,0). Final hull is the unique convex boundary enclosing all points.
Q3: How do you find the maximum number of points on a line (LeetCode #149) efficiently?
For each "anchor" point P[i], count how many other points share the same slope with P[i]. Represent slope as a reduced fraction (dy/gcd, dx/gcd) to avoid floating-point errors. Use a HashMap keyed by (dy, dx). Normalize: if dx=0 (vertical), use (1,0). If dy=0 (horizontal), use (0,1). Always make dx positive (multiply both by -1 if dx<0) so that (-1,-2) and (1,2) map to the same key. Handle duplicate points separately (collinear with everything). For each anchor, find the max count in the map + duplicates. Answer = max over all anchors. Time: O(n^2) time, O(n) space. Why not cross product? Cross product checks collinearity for triples — that's O(n^3). Slope hashing per anchor is O(n^2) total.
Q4: Why should you prefer integer arithmetic over floating point in geometry algorithms?
Floating-point arithmetic has rounding errors (IEEE 754 doubles have ~15-16 decimal digits of precision). In geometry, sign comparisons (is cross product positive, negative, or zero?) are extremely sensitive: a value that is mathematically 0 might compute as ±1e-15, causing incorrect turn direction, missed collinearities, or degenerate hull errors. Integer arithmetic is exact: if input coordinates are integers, cross products and dot products are exact integers too (as long as you use 64-bit: coordinates up to ~10^9 mean products up to ~10^18, which fits in long). Safe practices: (1) Use integer coords where possible. (2) Only convert to float for output (distances, angles). (3) When float is unavoidable, add epsilon: if abs(x) < 1e-9: treat as zero. (4) Use fractions for slope representations in #149.
Q5: What is the rotating calipers technique and what is its time complexity?
Rotating calipers uses two parallel supporting lines (like calipers) that "rotate" around a convex hull to efficiently compute properties like diameter (max distance), width, or minimum bounding rectangle. Key insight: for a convex polygon, as you advance one supporting line along an edge, the antipodal (opposite) supporting line can only advance forward in the same rotation direction — it never needs to go backward. This means both pointers together make at most 2n advances total, giving O(n) after the O(n log n) hull construction. Finding polygon diameter: start with the widest pair (antipodal points), advance the "bottom" caliper by one edge, advance the "top" until it's no longer increasing distance. Record max distance. Total: O(n log n) dominated by hull construction.
