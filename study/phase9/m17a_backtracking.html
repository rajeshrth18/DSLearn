Section 1 — The Backtracking Template

Backtracking is a systematic approach to explore all possible configurations of a solution by building candidates incrementally and abandoning (pruning) candidates that cannot lead to a valid solution.

Backtracking Template: solve(state): if isGoal(state): record/return solution return for each choice in candidates(state): if isValid(state, choice): apply(state, choice) ← make move solve(state) ← recurse undo(state, choice) ← BACKTRACK (undo move) Key insight: "Undo" restores state — backtracking explores a tree of partial solutions, pruning entire subtrees early.
💡 Pruning is the key to performance. Without pruning, backtracking is brute-force O(n!). Good constraint checking prunes orders of magnitude of the search space.
Section 2 — N-Queens (LC 51, 52)

Place N queens on an N×N chessboard so no two queens attack each other (no shared row, column, or diagonal).

Strategy: Place one queen per row. Track used columns and diagonals with sets for O(1) constraint checks.

4-Queens partial solution trace: Row 0: try col 0 → blocked, col 1 → ok → place Q Row 1: try col 0,1,2 → all attacked by diagonals → backtrack Row 0: try col 1 → blocked, col 2 → ok → place Q ... Two solutions exist for N=4: . Q . . . . Q . . . . Q Q . . . Q . . . . . . Q . . Q . . Q . .
C#
public IList<IList<string>> SolveNQueens(int n)
{
    var results = new List<IList<string>>();
    var cols  = new HashSet<int>();
    var diag1 = new HashSet<int>();  // row - col (top-left diagonals)
    var diag2 = new HashSet<int>();  // row + col (top-right diagonals)
    var board = new char[n][];
    for (int i = 0; i < n; i++) board[i] = new string('.', n).ToCharArray();

    void Backtrack(int row)
    {
        if (row == n) { results.Add(board.Select(r => new string(r)).ToList()); return; }
        for (int col = 0; col < n; col++)
        {
            if (cols.Contains(col) || diag1.Contains(row-col) || diag2.Contains(row+col)) continue;
            board[row][col] = 'Q';
            cols.Add(col); diag1.Add(row-col); diag2.Add(row+col);
            Backtrack(row + 1);
            board[row][col] = '.';
            cols.Remove(col); diag1.Remove(row-col); diag2.Remove(row+col);
        }
    }
    Backtrack(0);
    return results;
}

Python
def solve_n_queens(n: int) -> list[list[str]]:
    results = []
    cols = set()
    diag1 = set()   # row - col
    diag2 = set()   # row + col
    queens = []      # queens[row] = col

    def backtrack(row: int):
        if row == n:
            board = []
            for c in queens:
                row_str = '.' * c + 'Q' + '.' * (n - c - 1)
                board.append(row_str)
            results.append(board)
            return
        for col in range(n):
            if col in cols or row-col in diag1 or row+col in diag2:
                continue
            cols.add(col); diag1.add(row-col); diag2.add(row+col)
            queens.append(col)
            backtrack(row + 1)
            cols.discard(col); diag1.discard(row-col); diag2.discard(row+col)
            queens.pop()

    backtrack(0)
    return results

print(len(solve_n_queens(4)))  # 2 solutions
Section 3 — Sudoku Solver (LC 37)

Fill a 9×9 Sudoku grid so each row, column, and 3×3 box contains 1–9 exactly once.

Strategy: Find empty cells one by one. Try digits 1–9; check row/col/box constraints. If valid, recurse. If recursion fails, backtrack.

C#
public void SolveSudoku(char[][] board)
{
    Solve(board);
}

private bool Solve(char[][] board)
{
    for (int r = 0; r < 9; r++)
    for (int c = 0; c < 9; c++)
    {
        if (board[r][c] != '.') continue;
        for (char d = '1'; d <= '9'; d++)
        {
            if (!IsValid(board, r, c, d)) continue;
            board[r][c] = d;
            if (Solve(board)) return true;
            board[r][c] = '.';   // backtrack
        }
        return false;  // no digit worked → dead end
    }
    return true;  // all cells filled
}

private bool IsValid(char[][] board, int row, int col, char d)
{
    int boxRow = (row / 3) * 3, boxCol = (col / 3) * 3;
    for (int i = 0; i < 9; i++)
    {
        if (board[row][i] == d) return false;
        if (board[i][col] == d) return false;
        if (board[boxRow + i/3][boxCol + i%3] == d) return false;
    }
    return true;
}
Python
def solve_sudoku(board: list[list[str]]) -> None:

    def is_valid(r: int, c: int, d: str) -> bool:
        box_r, box_c = (r // 3) * 3, (c // 3) * 3
        for i in range(9):
            if board[r][i] == d: return False
            if board[i][c] == d: return False
            if board[box_r + i//3][box_c + i%3] == d: return False
        return True

    def solve() -> bool:
        for r in range(9):
            for c in range(9):
                if board[r][c] != '.': continue
                for d in '123456789':
                    if not is_valid(r, c, d): continue
                    board[r][c] = d
                    if solve(): return True
                    board[r][c] = '.'   # backtrack
                return False   # no digit worked
        return True   # all filled

    solve()
📌 Optimization: Instead of scanning the board linearly, collect all empty cells upfront. Further optimizations include constraint propagation (AC-3) and "minimum remaining values" heuristic — fill the cell with fewest valid digits first. For interviews, the simple recursive version is sufficient.
