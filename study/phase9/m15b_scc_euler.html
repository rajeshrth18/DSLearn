Section 1 — Strongly Connected Components

In a directed graph, a Strongly Connected Component (SCC) is a maximal set of vertices such that every vertex in the set is reachable from every other vertex in the set.

EXAMPLE DIRECTED GRAPH: 0 → 1 → 2 → 3 ↑ ↓ ↓ └── 4 5 → 6 ↑ 7 SCCs: {0, 1, 4} — 0→1→4→0 forms a cycle; all reachable from each other {2} — can reach 3,5,6 but nothing reaches back to 2 {3} — isolated in its reach {5, 7} — 5→7? No... only {5} and {7} if no cycle CORRECTED EXAMPLE (with cycles): 0 ⇄ 1 (bidirectional), 1→2→3→1 (triangle), 4 standalone SCC1 = {0, 1} (0↔1) SCC2 = {1, 2, 3} (1→2→3→1) CONDENSATION: Replace each SCC with a single node. The result is ALWAYS a DAG (directed acyclic graph). This is the "meta-graph" or "condensation graph." WHY SCCS MATTER: - 2-SAT problem solutions - Deadlock detection (circular dependencies) - Compiler optimization (code motion) - Web page ranking (PageRank groups SCCs)
Section 2 — Kosaraju's Algorithm
KOSARAJU'S ALGORITHM — Two DFS passes: PASS 1: Run DFS on ORIGINAL graph. Push nodes to stack in FINISH order. (The last node to finish = the "source" SCC in condensation.) PASS 2: Transpose the graph (reverse all edges). Pop nodes from stack. For each unvisited node, run DFS on TRANSPOSE graph. Each DFS tree in pass 2 = one SCC. WHY THIS WORKS: The finish-order in pass 1 ensures we visit "source" SCCs first. In the transposed graph, a DFS from a source SCC node can ONLY reach nodes within the same SCC (edges to other SCCs are reversed). EXAMPLE: Graph: 0→1→2→0, 1→3, 3→4→3 (3,4 form a cycle) Pass 1 finish order (DFS from 0): [2, 0, 4, 3, 1] (stack top = 1 = last finished) Wait — actual finish: depends on adjacency. Generally: DFS: 0→1→2→0(visited)→3→4→3(visited), finish 4,3, back to 1, finish 1, finish 0, done Stack bottom-to-top: [2, 4, 3, 1, 0] Pass 2 on TRANSPOSE (pop from stack = 0 first): DFS from 0: can reach 0,1,2 in transpose → SCC1 = {0,1,2} DFS from 3: can reach 3,4 → SCC2 = {3,4} COMPLEXITY: O(V + E) — two DFS passes
C# 
public class KosarajuSCC
{
    private int n;
    private List<int>[] adj, radj;

    public KosarajuSCC(int n)
    {
        this.n = n;
        adj  = Enumerable.Range(0, n).Select(_ => new List<int>()).ToArray();
        radj = Enumerable.Range(0, n).Select(_ => new List<int>()).ToArray();
    }

    public void AddEdge(int u, int v) { adj[u].Add(v); radj[v].Add(u); }

    public List<List<int>> FindSCCs()
    {
        // Pass 1: DFS on original, fill stack by finish time
        bool[] visited = new bool[n];
        var stack = new Stack<int>();
        for (int i = 0; i < n; i++)
            if (!visited[i]) Dfs1(i, visited, stack);

        // Pass 2: DFS on transposed graph in stack order
        Array.Fill(visited, false);
        var sccs = new List<List<int>>();
        while (stack.Count > 0)
        {
            int v = stack.Pop();
            if (!visited[v])
            {
                var scc = new List<int>();
                Dfs2(v, visited, scc);
                sccs.Add(scc);
            }
        }
        return sccs;
    }

    private void Dfs1(int u, bool[] vis, Stack<int> stack)
    {
        vis[u] = true;
        foreach (int v in adj[u]) if (!vis[v]) Dfs1(v, vis, stack);
        stack.Push(u);  // Push AFTER visiting all children
    }

    private void Dfs2(int u, bool[] vis, List<int> scc)
    {
        vis[u] = true; scc.Add(u);
        foreach (int v in radj[u]) if (!vis[v]) Dfs2(v, vis, scc);
    }
}

from collections import defaultdict
import sys; sys.setrecursionlimit(10**5)

def kosaraju_scc(n, edges):
    adj  = defaultdict(list)
    radj = defaultdict(list)
    for u, v in edges: adj[u].append(v); radj[v].append(u)

    visited = [False] * n; order = []

    def dfs1(u):
        visited[u] = True
        for v in adj[u]:
            if not visited[v]: dfs1(v)
        order.append(u)

    def dfs2(u, scc):
        visited[u] = True; scc.append(u)
        for v in radj[u]:
            if not visited[v]: dfs2(v, scc)

    for i in range(n):
        if not visited[i]: dfs1(i)

    visited = [False] * n; sccs = []
    for u in reversed(order):
        if not visited[u]:
            scc = []; dfs2(u, scc); sccs.append(scc)
    return sccs
Section 3 — Tarjan's SCC Algorithm (One-Pass)

Tarjan's SCC uses the same disc[] and low[] arrays as the bridge algorithm, plus an explicit stack of nodes. When we detect that a node is an SCC root (low[u] == disc[u]), we pop everything off the stack until (and including) u — that's one SCC.

TARJAN'S SCC KEY IDEA: A node u is an SCC ROOT if low[u] == disc[u]. Meaning: no back edge from u's subtree reaches ABOVE u. When u finishes and low[u] == disc[u], pop the stack until u → that's one SCC. Stack tracks "nodes potentially in the current SCC." onStack[] prevents counting a node in multiple SCCs via cross edges. low[u] = min( disc[u], low[v] for unvisited tree children v, disc[w] for w in stack (NOT just any back edge — only if w is onStack!) ) If w is NOT on stack → w belongs to a completed SCC → don't update low[u] from it.
C# 
public class TarjanSCC
{
    private int n, timer = 0;
    private List<int>[] adj;
    private int[] disc, low, sccId;
    private bool[] onStack;
    private Stack<int> stack;
    private List<List<int>> sccs;

    public TarjanSCC(int n) {
        this.n = n;
        adj  = Enumerable.Range(0, n).Select(_ => new List<int>()).ToArray();
        disc = new int[n]; low = new int[n]; sccId = new int[n];
        onStack = new bool[n]; stack = new Stack<int>();
        sccs = new List<List<int>>();
        Array.Fill(disc, -1);
    }
    public void AddEdge(int u, int v) => adj[u].Add(v);

    public (List<List<int>> SCCs, int[] SccId) FindSCCs()
    {
        for (int i = 0; i < n; i++) if (disc[i] == -1) Dfs(i);
        return (sccs, sccId);
    }

    private void Dfs(int u)
    {
        disc[u] = low[u] = timer++;
        stack.Push(u); onStack[u] = true;

        foreach (int v in adj[u])
        {
            if (disc[v] == -1)
            {
                Dfs(v);
                low[u] = Math.Min(low[u], low[v]);
            }
            else if (onStack[v])
                low[u] = Math.Min(low[u], disc[v]);
        }

        if (low[u] == disc[u])   // u is an SCC root
        {
            var scc = new List<int>();
            while (true)
            {
                int w = stack.Pop(); onStack[w] = false;
                scc.Add(w); sccId[w] = sccs.Count;
                if (w == u) break;
            }
            sccs.Add(scc);
        }
    }
}
Python 
def tarjan_scc(n, edges):
    adj = defaultdict(list)
    for u, v in edges: adj[u].append(v)

    disc = [-1] * n; low = [0] * n
    on_stack = [False] * n; stack = []; timer = [0]; sccs = []

    def dfs(u):
        disc[u] = low[u] = timer[0]; timer[0] += 1
        stack.append(u); on_stack[u] = True
        for v in adj[u]:
            if disc[v] == -1: dfs(v); low[u] = min(low[u], low[v])
            elif on_stack[v]: low[u] = min(low[u], disc[v])
        if low[u] == disc[u]:
            scc = []
            while True:
                w = stack.pop(); on_stack[w] = False; scc.append(w)
                if w == u: break
            sccs.append(scc)

    for i in range(n):
        if disc[i] == -1: dfs(i)
    return sccs
Section 4 — Euler Paths & Circuits
DEFINITIONS: Euler CIRCUIT: A closed walk that visits every EDGE exactly once (starts = ends). Euler PATH: A walk that visits every EDGE exactly once (may start ≠ end). EXISTENCE CONDITIONS: UNDIRECTED GRAPH: Euler Circuit exists iff: graph is connected AND all vertices have EVEN degree. Euler Path exists iff: graph is connected AND exactly 2 vertices have ODD degree. (Start at one odd-degree vertex, end at the other.) DIRECTED GRAPH: Euler Circuit exists iff: graph is strongly connected AND in-degree == out-degree for ALL vertices. Euler Path exists iff: graph is connected AND: - Exactly one vertex with (out-degree - in-degree = 1) [start] - Exactly one vertex with (in-degree - out-degree = 1) [end] - All other vertices have in-degree == out-degree FAMOUS EXAMPLE: Königsberg Bridge Problem (1736, Euler) 7 bridges over Pregel River. Can you cross each bridge exactly once? NO — 4 land masses all have odd degree. No Euler path exists. HIERHOLZER'S ALGORITHM: Find Euler circuit/path in O(E). 1. Start at a valid node (odd-degree for path, any for circuit). 2. Follow edges (using a stack), deleting used edges. 3. When stuck (no more edges from current node), output it. 4. Backtrack to last node with unused edges, continue. Result: reversed output = Euler path/circuit.
C# 
// Hierholzer's Algorithm — Directed graph Euler path
public List<string> FindItinerary(IList<IList<string>> tickets)
{
    // LeetCode #332 — tickets are edges, airports are nodes
    var adj = new Dictionary<string, SortedList<string, int>>();
    foreach (var t in tickets)
    {
        if (!adj.ContainsKey(t[0])) adj[t[0]] = new SortedList<string, int>(new DupKeyComparer());
        adj[t[0]].Add(t[1], 0);
    }
    var result = new LinkedList<string>();
    var stk = new Stack<string>();
    stk.Push("JFK");
    while (stk.Count > 0)
    {
        var top = stk.Peek();
        if (adj.ContainsKey(top) && adj[top].Count > 0)
        {
            string next = adj[top].Keys[0];
            adj[top].RemoveAt(0);
            stk.Push(next);
        }
        else
        {
            result.AddFirst(stk.Pop());  // prepend to result when stuck
        }
    }
    return result.ToList();
}

import heapq
# LeetCode #332 — Reconstruct Itinerary (Euler path in directed graph)
def findItinerary(tickets):
    adj = defaultdict(list)
    for u, v in sorted(tickets, reverse=True):  # sort reverse for pop from back
        adj[u].append(v)

    result = []; stack = ["JFK"]
    while stack:
        top = stack[-1]
        if adj[top]:
            stack.append(adj[top].pop())  # take next (smallest, was sorted reverse)
        else:
            result.append(stack.pop())    # stuck: add to result
    return result[::-1]                    # reverse = correct order

# Using heapq for guaranteed lexicographic order:
def findItinerary_heap(tickets):
    adj = defaultdict(list)
    for u, v in tickets: heapq.heappush(adj[u], v)
    result = []; stack = ["JFK"]
    while stack:
        while adj[stack[-1]]:
            stack.append(heapq.heappop(adj[stack[-1]]))
        result.append(stack.pop())
    return result[::-1]

Section 6 — Q&A Review
Q1: What is the condensation graph, and why is it always a DAG?
The condensation graph (or "meta-graph") is formed by replacing each SCC with a single supernode and drawing directed edges between supernodes when there's an edge between the SCCs in the original graph. It is always a DAG because: if two different SCCs had a cycle between them (SCC A → SCC B → SCC A), then all nodes in both SCCs would be in the same SCC by definition (every node can reach every other). Since SCCs are maximal, no two distinct SCCs can have a cycle between them. The condensation DAG is fundamental for topological sort of the original graph's strongly connected structure.
Q2: In Kosaraju's algorithm, why do we reverse all edges for the second DFS pass?
In the first DFS (original graph), we compute a finish-time ordering. The node that finishes LAST is in a "source SCC" — an SCC with no incoming edges from other SCCs in the condensation graph. In the second pass (reversed graph), starting a DFS from this source SCC node: in the reversed graph, the old "source" becomes a "sink" — edges that used to leave the SCC now point INTO the SCC in the reversed graph. So a DFS from the source node in the reversed graph stays trapped within the same SCC. This is why the second-pass DFS trees are exactly the SCCs.
Q3: In Tarjan's SCC algorithm, why do we only update low[u] with disc[v] (not low[v]) for neighbors already on the stack?
When v is already visited AND on the stack, the edge (u,v) is a back edge or cross edge within the current "open" DFS branch. We use disc[v] (not low[v]) because we want to know if u can reach v via this edge — v's discovery time tells us where v "is" in the DFS tree. Using low[v] here would be wrong: if v can reach some ancestor w via its own back edges, that doesn't mean u should inherit that reach (u might not be in the same SCC as v). The disc[v] update only propagates reachability along the actual edge (u→v), while the low[v] update (for unvisited v) propagates reachability through the DFS subtree.
Q4: How do you determine if an Euler path exists in a directed graph, and where does it start?
For a directed graph to have an Euler path: (1) The graph must be weakly connected (connected if you ignore edge directions). (2) Exactly one vertex has out-degree − in-degree = 1 (this is the start). (3) Exactly one vertex has in-degree − out-degree = 1 (this is the end). (4) All other vertices have equal in-degree and out-degree. If ALL vertices have equal in/out degree AND the graph is strongly connected, it has an Euler CIRCUIT. To find the start node for an Euler path: find the unique vertex with out-degree − in-degree = 1. If no such vertex (circuit case), start anywhere. LeetCode #2097 uses exactly this: find start, then run Hierholzer's.
Q5: Why does Hierholzer's algorithm use a stack and prepend to result (or reverse at end)?
The algorithm works by following edges until stuck, then "outputting" the stuck node. When you get stuck at node v (no more edges), v should be the LAST node in the path up to that point. But you might backtrack to an earlier node and find more edges, extending the path further. The trick: nodes are added to result when stuck (post-order). The result built this way is the Euler path in REVERSE. Reversing at the end gives the correct order. The stack holds the current DFS path; when we pop (stuck), we prepend to the result list. This is equivalent to a recursive DFS where you output the current node AFTER all its neighbors are visited — a topological post-order for Euler paths.
