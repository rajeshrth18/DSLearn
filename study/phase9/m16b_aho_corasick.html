Section 1 — The Multi-Pattern Problem

KMP and Z-function solve one pattern in O(n+m). But what if you have k patterns and one large text? Running KMP for each pattern costs O(k×n + Σm), which is too slow when k is large (e.g., antivirus with 10,000 signatures scanning a 100MB file).

PROBLEM: Find all occurrences of any pattern from set P = {p1, p2, ..., pk} in text T. NAIVE approach: Run KMP k times. Time: O(k×n + Σm) → O(10000 × 10^8) = 10^12 operations. Way too slow. BETTER: Build a combined automaton once from ALL patterns. Then scan the text ONCE to find all matches of all patterns. Time: O(Σm + n + output_size) ← scan text only once! This is the Aho-Corasick algorithm (1975): 1. Build a Trie from all patterns 2. Add "failure links" (like KMP failure function, but on the trie) 3. Optionally add "output links" to collect all matching patterns at each position 4. Scan text: follow trie transitions + failure links
Section 2 — Building the Aho-Corasick Automaton

Step 1: Build the Trie

Insert all patterns into a trie. Each path from root to a marked node represents a pattern.

Step 2: Compute Failure Links

The failure link of node v is the longest proper suffix of the string represented by v that is also a prefix of some pattern in the trie (i.e., is a node in the trie).

TRIE + FAILURE LINKS for patterns: {"he", "she", "his", "hers"} root / \ h s | | e* h | | r e* | s* (*) = end of pattern FAILURE LINKS (computed via BFS): Intuition: fail[v] = where to "fall back" if the next character doesn't match fail[root] = root (by definition) fail[h] = root ('h' has no shorter prefix that's a trie node) fail[s] = root fail[he] = root (suffix "e" is NOT a trie prefix) fail[sh] = h (suffix "h" IS a trie node!) fail[she] = he (suffix "he" IS a trie node → she's fail = he) fail[her] = root fail[his] = s (suffix "s" IS a trie node) fail[hers] = s SEARCH: Process text character by character. At each position, follow the trie transition. If no transition for current char: follow failure links until you find one (or reach root). At each node, collect patterns that end here (including via output/dict links).

Step 3: Compute Output Links (Dictionary Links)

An output link from node v points to the next node on the failure link chain that is a complete pattern. This lets us efficiently collect all patterns that match at any position (since a node may have multiple patterns matching through its failure chain).

C# 
public class AhoCorasick
{
    private int[][] next;      // trie transition table
    private int[] fail;        // failure links
    private int[] output;      // output link (nearest complete pattern ancestor)
    private List<int>[] patterns; // patterns that end at this node
    private int size = 1;

    public AhoCorasick(int maxNodes = 100000, int sigma = 26)
    {
        next = new int[maxNodes][];
        fail = new int[maxNodes];
        output = new int[maxNodes];
        patterns = new List<int>[maxNodes];
        for (int i = 0; i < maxNodes; i++) {
            next[i] = new int[sigma]; Array.Fill(next[i], -1);
            output[i] = -1;
            patterns[i] = new List<int>();
        }
    }

    public void Insert(string word, int id)
    {
        int cur = 0;
        foreach (char c in word) {
            int ch = c - 'a';
            if (next[cur][ch] == -1) next[cur][ch] = size++;
            cur = next[cur][ch];
        }
        patterns[cur].Add(id);
    }

    public void Build()
    {
        var q = new Queue<int>();
        // Initialize: root's children. If child exists → fail=root. Else → link to root.
        for (int c = 0; c < 26; c++) {
            if (next[0][c] != -1) { fail[next[0][c]] = 0; q.Enqueue(next[0][c]); }
            else next[0][c] = 0;  // shortcut: undefined transitions at root → root
        }
        while (q.Count > 0) {
            int u = q.Dequeue();
            // Merge output: patterns at fail[u] are also reachable from u
            if (patterns[fail[u]].Count > 0) output[u] = fail[u];
            else output[u] = output[fail[u]];

            for (int c = 0; c < 26; c++) {
                if (next[u][c] != -1) {
                    fail[next[u][c]] = next[fail[u]][c];
                    q.Enqueue(next[u][c]);
                } else {
                    next[u][c] = next[fail[u]][c];  // shortcut: propagate
                }
            }
        }
    }

    // Search returns list of (patternId, textPosition) for all matches
    public List<(int id, int pos)> Search(string text)
    {
        var results = new List<(int, int)>();
        int cur = 0;
        for (int i = 0; i < text.Length; i++) {
            cur = next[cur][text[i] - 'a'];
            int temp = cur;
            while (temp > 0) {
                foreach (int id in patterns[temp]) results.Add((id, i));
                temp = output[temp] == -1 ? 0 : output[temp];
                if (output[temp] == -1) break;
            }
        }
        return results;
    }
}
Python 
from collections import deque

class AhoCorasick:
    def __init__(self):
        self.goto = [{}]   # trie transitions: goto[node][char] = next_node
        self.fail = [0]
        self.output = [[]]  # list of pattern indices at each node

    def insert(self, word, idx):
        cur = 0
        for c in word:
            if c not in self.goto[cur]:
                self.goto[cur][c] = len(self.goto)
                self.goto.append({})
                self.fail.append(0)
                self.output.append([])
            cur = self.goto[cur][c]
        self.output[cur].append(idx)

    def build(self):
        q = deque()
        # Root's children: fail → root
        for c, child in self.goto[0].items():
            self.fail[child] = 0; q.append(child)
        while q:
            u = q.popleft()
            self.output[u] += self.output[self.fail[u]]  # inherit patterns via fail
            for c, child in self.goto[u].items():
                f = self.fail[u]
                while f != 0 and c not in self.goto[f]: f = self.fail[f]
                self.fail[child] = self.goto[f].get(c, 0) if f != child else 0
                q.append(child)

    def search(self, text):
        """Returns list of (pattern_idx, end_pos) for all matches."""
        cur = 0; results = []
        for i, c in enumerate(text):
            while cur != 0 and c not in self.goto[cur]: cur = self.fail[cur]
            cur = self.goto[cur].get(c, 0)
            for idx in self.output[cur]: results.append((idx, i))
        return results

Section 3 — Suffix Arrays

A suffix array SA is an array of all starting positions of suffixes of string s, sorted in lexicographic order. Combined with the LCP (Longest Common Prefix) array, suffix arrays can solve many string problems optimally.

SUFFIX ARRAY EXAMPLE: String s = "banana" Suffixes: 0: "banana" 1: "anana" 2: "nana" 3: "ana" 4: "na" 5: "a" Sorted lexicographically: "a" → index 5 "ana" → index 3 "anana" → index 1 "banana" → index 0 "na" → index 4 "nana" → index 2 SA = [5, 3, 1, 0, 4, 2] LCP ARRAY (lcp[i] = LCP of SA[i] and SA[i-1]): lcp[0] = 0 (undefined, set to 0) lcp[1] = 1 ("a" and "ana" share prefix "a") lcp[2] = 3 ("ana" and "anana" share "ana") lcp[3] = 0 ("anana" and "banana" share nothing) lcp[4] = 0 ("banana" and "na") lcp[5] = 2 ("na" and "nana" share "na") LCP = [0, 1, 3, 0, 0, 2] APPLICATIONS: - Count distinct substrings: n*(n+1)/2 - Σlcp[i] - Longest repeated substring: max(lcp) - Longest common substring of two strings: concat s+'#'+t, find max lcp between suffixes from different strings - Pattern matching: binary search in SA in O(m log n)

O(n log n) Suffix Array Construction

The standard competitive programming approach uses doubling: C# 

// O(n log² n) Suffix Array — doubling sort
public static int[] BuildSuffixArray(string s) {
    int n = s.Length;
    int[] sa=Enumerable.Range(0,n).ToArray(), rank=s.Select(c=>(int)c).ToArray(), tmp=new int[n];
    for (int k=1; k<n; k<<=1) {
        int kk=k;
        Array.Sort(sa,(a,b)=>{
            if(rank[a]!=rank[b]) return rank[a]-rank[b];
            return (a+kk<n?rank[a+kk]:-1)-(b+kk<n?rank[b+kk]:-1);});
        tmp[sa[0]]=0;
        for (int i=1; i<n; i++) {
            bool same=rank[sa[i]]==rank[sa[i-1]]&&
                (sa[i]+k<n?rank[sa[i]+k]:-1)==(sa[i-1]+k<n?rank[sa[i-1]+k]:-1);
            tmp[sa[i]]=tmp[sa[i-1]]+(same?0:1);
        }
        Array.Copy(tmp,rank,n);
        if(rank[sa[n-1]]==n-1) break;
    }
    return sa;
}
// Count distinct substrings = n*(n+1)/2 - sum(lcp)
public static long CountDistinct(string s) {
    int n=s.Length; var sa=BuildSuffixArray(s); var lcp=BuildLcp(s,sa);

Python 
def build_suffix_array(s):
    """O(n log^2 n) suffix array construction."""
    n = len(s)
    sa = list(range(n))
    rank = [ord(c) for c in s]
    tmp = [0] * n
    k = 1
    while k < n:
        def cmp_key(i): return (rank[i], rank[i+k] if i+k < n else -1)
        sa.sort(key=cmp_key)
        tmp[sa[0]] = 0
        for i in range(1, n):
            tmp[sa[i]] = tmp[sa[i-1]] + (1 if cmp_key(sa[i]) != cmp_key(sa[i-1]) else 0)
        rank = tmp[:]
        if rank[sa[-1]] == n-1: break
        k *= 2
    return sa

def build_lcp_array(s, sa):
    """Kasai's algorithm: O(n) LCP array from suffix array."""
    n = len(s); rank = [0] * n; lcp = [0] * n
    for i, v in enumerate(sa): rank[v] = i
    h = 0
    for i in range(n):
        if rank[i] > 0:
            j = sa[rank[i]-1]
            while i+h < n and j+h < n and s[i+h] == s[j+h]: h += 1
            lcp[rank[i]] = h
            if h > 0: h -= 1
    return lcp

# Count distinct substrings
def count_distinct(s):
    n = len(s); sa = build_suffix_array(s); lcp = build_lcp_array(s, sa)
    return n*(n+1)//2 - sum(lcp)
C# 
// Kasai's LCP algorithm — O(n)
public int[] BuildLcp(string s, int[] sa)
{
    int n = s.Length;
    int[] rank = new int[n], lcp = new int[n];
    for (int i = 0; i < n; i++) rank[sa[i]] = i;
    int h = 0;
    for (int i = 0; i < n; i++) {
        if (rank[i] > 0) {
            int j = sa[rank[i] - 1];
            while (i + h < n && j + h < n && s[i + h] == s[j + h]) h++;
            lcp[rank[i]] = h;
            if (h > 0) h--;
        }
    }
    return lcp;
}

Pattern Search Using Suffix Array

Binary search on the sorted SA: since SA is in lexicographic order, we binary search for the first and last suffix with the given pattern as prefix. Time: O(m log n).

Python 
import bisect
def sa_search(s, sa, pattern):
    """Find all occurrences of pattern in s using binary search on SA."""
    n, m = len(s), len(pattern)
    # Find leftmost position where suffix >= pattern
    lo = bisect.bisect_left(sa, 0, key=lambda i: s[i:i+m] >= pattern)
    hi = bisect.bisect_left(sa, 0, key=lambda i: s[i:i+m] > pattern)
    return sorted(sa[lo:hi])  # O(m log n + output)

Section 5 — Q&A Review
Q1: How does the Aho-Corasick failure link differ from the KMP failure function?
KMP's lps[j] applies to a single pattern: "the longest proper prefix of pattern[0..j] that's also a suffix." Aho-Corasick's failure link at trie node v points to: "the longest proper suffix of the string represented by v that is a valid PREFIX of ANY pattern" (i.e., that corresponds to some node in the trie). KMP has O(m) failure states (one per character). Aho-Corasick has O(Σm) states (one per character across all patterns), but each state has the same "fall back when stuck" semantics. The key conceptual difference: KMP fails within one pattern; Aho-Corasick fails across the entire pattern trie. Both guarantee linear total work via the same amortized argument.
Q2: Why is the goto transition table filled in with shortcuts during BFS in Aho-Corasick?
In the original trie, many transitions are undefined (e.g., from node "he", if we encounter 'x', there's no 'x' child). Without shortcuts, during search we'd need to follow failure links in a loop until we find a state with the desired transition. By filling undefined transitions during BFS (making them point to wherever we'd end up after following failure links), we create a complete DFA: every state has exactly one transition for every character. This eliminates the while loop during search, making the search phase O(n) with zero overhead — just follow one precomputed transition per character. The BFS order ensures parent shortcuts are computed before children.
Q3: What is the LCP array, how is it computed in O(n), and what can you do with it?
LCP[i] = length of the longest common prefix between suffix SA[i] and suffix SA[i-1] (adjacent in the sorted order). Kasai's algorithm computes it in O(n) using a key property: if the LCP of suffixes starting at positions i and j is h, then the LCP of suffixes starting at i+1 and j+1 is at least h-1. We process suffixes in order of their starting position (not SA order), maintaining a running h that only ever increments by 1 or decreases by at most 1, giving O(n) total. With the LCP array you can: (1) count distinct substrings in O(n), (2) find the longest repeated substring in O(n), (3) find the longest common substring of two strings by concatenating them with a separator and finding max LCP between a suffix from each string.
Q4: What is the Burrows-Wheeler Transform and how does it relate to suffix arrays?
The Burrows-Wheeler Transform (BWT) rearranges string characters to cluster similar characters together, enabling better compression. Given string s with sentinel '$' appended, BWT[i] = s[SA[i] - 1] (the character just before the i-th sorted suffix). Since SA gives suffixes in sorted order, BWT can be computed in O(n) from the suffix array. The reverse BWT (BWT inversion) can reconstruct the original string. The FM-index (used in genome aligners like BWA and Bowtie) combines BWT + suffix array to achieve compressed full-text indexing: pattern search uses only O(m) memory-accesses into O(n) space (vs O(n) space for raw SA), enabling alignment of billions of short reads against human genome.
Q5: How would you use Aho-Corasick to solve LeetCode #212 "Word Search II"?
Build an Aho-Corasick automaton (or Trie, which is essentially Aho-Corasick without failure links for this problem) from all words. Then DFS/backtrack through the grid, maintaining the current trie node. At each cell, follow the trie transition for the current character. If the current node represents a complete word, add it to results. Prune if the current trie node has no children (no words extend through this path). Why Aho-Corasick over simple Trie? If multiple words share prefixes (e.g., "ab", "abc"), the automaton merges their prefix states. Why Aho-Corasick over running KMP for each word? The grid DFS builds one path that can match multiple words simultaneously — running all words through the same automaton is O(words × path_length), whereas separate KMPs would be O(|words| × grid_size).
