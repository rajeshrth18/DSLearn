Section 1 — Modular Arithmetic Review
MODULAR ARITHMETIC RULES (all mod p): (a + b) mod p = ((a mod p) + (b mod p)) mod p (a - b) mod p = ((a mod p) - (b mod p) + p) mod p ? ADD p to avoid negative (a * b) mod p = ((a mod p) * (b mod p)) mod p (a / b) mod p = (a * b^(-1)) mod p ? division requires modular inverse! Power: a^b mod p ? use binary exponentiation (O(log b)) BINARY EXPONENTIATION (fast pow): pow(a, b, p): result = 1 a = a % p while b > 0: if b & 1: result = result * a % p a = a * a % p b >>= 1 return result Python: pow(a, b, p) ? built-in, very fast C#: no built-in for long; implement manually IMPORTANT: For LeetCode problems with "return answer mod 10^9+7": MOD = 1_000_000_007 (prime) Every intermediate result: apply % MOD after each multiplication/addition Never let values accumulate past ~10^18 before taking mod (use long in C#)
C# Binary Exponentiation 
public long ModPow(long baseVal, long exp, long mod)
{
    long result = 1; baseVal %= mod;
    while (exp > 0) {
        if ((exp & 1) == 1) result = result * baseVal % mod;
        baseVal = baseVal * baseVal % mod;
        exp >>= 1;
    }
    return result;
}
Section 2 — Modular Inverse
MODULAR INVERSE of a mod p: the value a^(-1) such that a * a^(-1) = 1 (mod p) Required when dividing under modulus: a/b mod p = a * b^(-1) mod p METHOD 1: Fermat's Little Theorem (p must be prime, a ? 0 mod p) a^(p-1) = 1 (mod p) ? a^(-1) = a^(p-2) (mod p) inv(a, p) = pow(a, p-2, p) Time: O(log p) — simple, works for all a in [1, p-1] METHOD 2: Extended Euclidean Algorithm (works even when p is not prime) Bezout's identity: gcd(a, p) = a*x + p*y If gcd(a, p) = 1, then x is the modular inverse of a mod p. Time: O(log min(a,p)) METHOD 3: Precomputed inverse table in O(n) linear time inv[1] = 1 inv[i] = -(p / i) * inv[p % i] mod p ? avoid the division loop Use this when you need inv[1..n] for many values
C# 
const long MOD = 1_000_000_007;

// Fermat's: inv(a) = a^(MOD-2) mod MOD (MOD must be prime)
long ModInv(long a) => ModPow(a, MOD - 2, MOD);

// Extended GCD — returns gcd and sets x, y such that a*x + b*y = gcd
long ExtGcd(long a, long b, out long x, out long y)
{
    if (b == 0) { x = 1; y = 0; return a; }
    long g = ExtGcd(b, a % b, out long x1, out long y1);
    x = y1; y = x1 - (a / b) * y1;
    return g;
}
long ModInvExtGcd(long a, long m)
{
    ExtGcd(a, m, out long x, out long _);
    return ((x % m) + m) % m;
}

// Precompute factorials and inverse factorials for C(n,k) mod MOD
long[] fact, invFact;
void Precompute(int n)
{
    fact = new long[n + 1]; invFact = new long[n + 1];
    fact[0] = 1;
    for (int i = 1; i <= n; i++) fact[i] = fact[i - 1] * i % MOD;
    invFact[n] = ModInv(fact[n]);
    for (int i = n - 1; i >= 0; i--) invFact[i] = invFact[i + 1] * (i + 1) % MOD;
}
long C(int n, int k)
{
    if (k < 0 || k > n) return 0;
    return fact[n] * invFact[k] % MOD * invFact[n - k] % MOD;
}
Python 
MOD = 10**9 + 7

def mod_inv(a, p=MOD): return pow(a, p-2, p)  # Fermat

def precompute(n, p=MOD):
    fact = [1] * (n+1)
    for i in range(1, n+1): fact[i] = fact[i-1] * i % p
    inv_fact = [1] * (n+1)
    inv_fact[n] = pow(fact[n], p-2, p)
    for i in range(n-1, -1, -1): inv_fact[i] = inv_fact[i+1] * (i+1) % p
    return fact, inv_fact

def comb(n, k, fact, inv_fact, p=MOD):
    if k < 0 or k > n: return 0
    return fact[n] * inv_fact[k] % p * inv_fact[n-k] % p
Section 3 — Chinese Remainder Theorem
CHINESE REMAINDER THEOREM (CRT): Given system of congruences: x = r1 (mod m1) x = r2 (mod m2) ... x = rk (mod mk) If all mi are pairwise coprime, there exists a UNIQUE solution mod M = m1*m2*...*mk. ALGORITHM (for 2 congruences): x = a (mod m) x = b (mod n) Use Extended GCD: find p,q such that m*p + n*q = gcd(m,n) If gcd(m,n) doesn't divide (b-a): NO solution. Otherwise: x = a + m * p * (b-a)/gcd(m,n) [mod lcm(m,n)] SIMPLE CASE (m and n coprime): x = a * n * inv(n, m) + b * m * inv(m, n) mod (m*n) EXAMPLE: x = 2 (mod 3) ? x could be 2, 5, 8, 11, ... x = 3 (mod 5) ? x could be 3, 8, 13, 18, ... Solution: x = 8 (mod 15) APPLICATIONS: - LeetCode #1390 (Four Divisors), construction problems - Number theory proofs - Calendar computations - Breaking ciphers (RSA small exponent)
C# 
// CRT for two congruences: x = a (mod m), x = b (mod n)
// Returns (solution, lcm) or (-1, -1) if no solution
(long sol, long lcm) CRT(long a, long m, long b, long n)
{
    long g = ExtGcd(m, n, out long p, out long _);
    if ((b - a) % g != 0) return (-1, -1);
    long lcm = m / g * n;
    long x = (a + m * ((b - a) / g % (n / g)) * p % (n / g)) % lcm;
    return ((x + lcm) % lcm, lcm);
}
Python 
def crt(a, m, b, n):
    """Solve x = a (mod m) and x = b (mod n). Returns (sol, lcm) or (-1,-1)."""
    from math import gcd
    g = gcd(m, n)
    if (b - a) % g != 0: return -1, -1
    lcm = m // g * n
    # p = modular inverse of m//g mod n//g
    p = pow(m//g, -1, n//g)  # Python 3.8+ supports pow(a, -1, m) for modular inverse
    x = (a + m * ((b - a) // g * p % (n // g))) % lcm
    return x, lcm
Section 4 — Catalan Numbers
CATALAN NUMBERS: C_n = (2n choose n) / (n+1) = C(2n,n) / (n+1) C_0=1, C_1=1, C_2=2, C_3=5, C_4=14, C_5=42, C_6=132, ... RECURRENCE: C_n = S C_i * C_(n-1-i) for i in [0, n-1] CLOSED FORM: C_n = C(2n, n) - C(2n, n+1) = C(2n, n) / (n+1) WHAT CATALAN NUMBERS COUNT (all equivalent!): 1. Number of valid sequences of n pairs of balanced parentheses n=2: ()() and (()) ? 2 n=3: ()()(), (())(), ()(()), (()()), ((())) ? 5 2. Number of distinct BSTs with n nodes (Each BST corresponds to a parenthesization) 3. Number of ways to triangulate a convex polygon with n+2 sides 4. Number of monotonic lattice paths from (0,0) to (n,n) that do not go above the main diagonal 5. Number of full binary trees with n+1 leaves RECURRENCE for DP: dp[0] = 1, dp[1] = 1 dp[n] = sum(dp[i] * dp[n-1-i] for i in range(n))
C# 
// Catalan number C_n mod MOD using precomputed factorials
long Catalan(int n)
{
    // C_n = C(2n, n) * inv(n+1) mod MOD
    return C(2 * n, n) * ModInv(n + 1) % MOD;
}

// DP recurrence version (for small n, useful to see structure)
long[] CatalanDP(int maxN)
{
    long[] dp = new long[maxN + 1]; dp[0] = dp[1] = 1;
    for (int n = 2; n <= maxN; n++)
        for (int i = 0; i < n; i++)
            dp[n] = (dp[n] + dp[i] * dp[n - 1 - i]) % MOD;
    return dp;
}
Python 
def catalan(n, MOD=10**9+7):
    fact, inv_fact = precompute(2*n)
    return comb(2*n, n, fact, inv_fact) * pow(n+1, MOD-2, MOD) % MOD

# Number of unique BSTs with n nodes (LeetCode #96)
def numTrees(n):
    dp = [0] * (n+1); dp[0] = dp[1] = 1
    for i in range(2, n+1):
        for j in range(i): dp[i] += dp[j] * dp[i-1-j]
    return dp[n]  # = Catalan(n)
Section 5 — Lucas' Theorem & LeetCode Map
LUCAS' THEOREM: For prime p and non-negative integers n, k: C(n, k) mod p = product of C(n_i, k_i) mod p where n_i, k_i are the digits of n and k in base p. More concisely: C(n, k) = C(n mod p, k mod p) * C(n/p, k/p) (mod p) If any k_i > n_i, then C(n, k) = 0 (mod p). USE CASE: When n can be very large (up to 10^18) but p is small prime. Normal precomputed factorial table won't work (table would be huge). Lucas reduces it to C(small, small) computations recursively. EXAMPLE: C(1000000000, 500000000) mod 7 Represent in base 7 and apply Lucas recursively. PYTHON SHORTCUT for moderate n: import math; math.comb(n, k) % MOD But this fails for n > ~10^5 with mod — use precomputed inv_fact instead.
Python — Lucas' Theorem 
def lucas(n, k, p):
    """C(n, k) mod p where p is prime, using Lucas' theorem."""
    if k == 0: return 1
    ni, ki = n % p, k % p
    if ki > ni: return 0
    # C(ni, ki) computed directly (ni < p, so small factorial table)
    fact_p = [1] * p
    for i in range(1, p): fact_p[i] = fact_p[i-1] * i % p
    def small_comb(a, b): return fact_p[a] * pow(fact_p[b], p-2, p) % p * pow(fact_p[a-b], p-2, p) % p
    return small_comb(ni, ki) * lucas(n//p, k//p, p) % p

Section 6 — Q&A Review
Q1: Why does division mod p require the modular inverse, and when does it not exist?
In modular arithmetic, a/b mod p means multiplying by b's multiplicative inverse: a * b^(-1) mod p. b^(-1) exists iff gcd(b, p) = 1 (b and p are coprime). If p is prime and 0 < b < p, then gcd(b,p) = 1 always, so every non-zero element has an inverse. When p is not prime, some elements have no inverse (e.g., 2 has no inverse mod 4: 2x = 1 mod 4 has no integer solution). For competitive programming with MOD = 10^9+7 (prime): every value in [1, MOD-1] has a unique inverse. Fermat's Little Theorem gives b^(-1) = b^(p-2) mod p in O(log p) time.
Q2: How do you efficiently compute C(n, k) mod p for many different k values (all with the same n)?
Precompute factorial and inverse factorial tables once in O(n). Then each C(n, k) = fact[n] * inv_fact[k] * inv_fact[n-k] mod p is O(1). Build tables: fact[0]=1, fact[i] = fact[i-1]*i % p. inv_fact[n] = pow(fact[n], p-2, p). inv_fact[i] = inv_fact[i+1] * (i+1) % p (fill backwards). This leverages the identity inv_fact[i] = inv_fact[i+1] * (i+1) because inv_fact[i] = (i! )^(-1) = (i+1)! ^(-1) * (i+1) = inv_fact[i+1] * (i+1). Total preprocessing: O(n log p) for one inverse at the end + O(n) fill.
Q3: Explain why the number of unique BSTs with n nodes equals the n-th Catalan number.
For a BST with keys 1..n, choose the root r (1 = r = n). The left subtree uses keys 1..r-1 (exactly r-1 keys), and the right subtree uses keys r+1..n (exactly n-r keys). The number of BSTs is independent of actual key values — only the count matters. So: dp[n] = S dp[r-1] * dp[n-r] for r in 1..n = S dp[i] * dp[n-1-i] for i in 0..n-1. This is exactly the Catalan recurrence. Base case dp[0] = dp[1] = 1. The first few values: dp[2]=2, dp[3]=5, dp[4]=14 match C_2, C_3, C_4.
Q4: What is the Chinese Remainder Theorem and how does it apply to LeetCode #1201 (Ugly Number III)?
CRT states: if m and n are coprime, the system x = a (mod m), x = b (mod n) has a unique solution mod mn. #1201 asks for the n-th ugly number divisible by a, b, or c. Use binary search on the answer X. Count of ugly numbers = X: by inclusion-exclusion = X/a + X/b + X/c - X/lcm(a,b) - X/lcm(b,c) - X/lcm(a,c) + X/lcm(a,b,c). Binary search: find smallest X where count = n. The lcm computations use gcd. CRT isn't directly needed here but the underlying modular structure (lcm = product/gcd) is the same mathematical machinery.
Q5: When would you use Lucas' Theorem over the standard precomputed factorial approach?
Use Lucas' Theorem when: (1) n is very large (up to 10^18) — building a factorial table of size n is impossible, (2) the modulus p is a small prime. Lucas reduces C(n,k) mod p to a product of C(n_i, k_i) mod p where n_i, k_i are the base-p digits of n and k. Since each n_i < p, we only need a factorial table of size p (tiny). Standard precomputed approach works when n = ~10^6 and p is large prime (like 10^9+7). If n = 10^6 and p = 10^9+7: precomputed table. If n = 10^18 and p is small prime like 7 or 13: Lucas. If n = 10^18 and p = 10^9+7: need different techniques (Andrew Granville's generalization for composite moduli).
