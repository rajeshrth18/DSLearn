Section 1 — Why Learn O(n²) Sorts?

You might wonder: if there are O(n log n) algorithms, why learn O(n²)? Three reasons:

  • Interviews: Every coding interview expects you to know these
  • Small data: For n ≤ 50, O(n²) algorithms are fast enough and simpler
  • Insertion sort advantage: For nearly-sorted data, insertion sort is O(n) — even faster than merge sort!

Section 2 — Bubble Sort

2.1 The Analogy — Bubbles Rising

Imagine a glass of carbonated water. Heavy bubbles stay at the bottom; light bubbles rise to the top. In bubble sort, in each pass through the array, the largest unsorted element "bubbles up" to its final position at the end — just like bubbles rising.

2.2 How It Works

Compare adjacent pairs. If the left is bigger than the right, swap them. After one full pass, the largest element is at the end. Repeat for the remaining unsorted portion.

Array: [64, 34, 25, 12, 22, 11, 90] Pass 1: Compare and swap adjacent pairs: [64,34] → swap → [34,64,25,12,22,11,90] [64,25] → swap → [34,25,64,12,22,11,90] [64,12] → swap → [34,25,12,64,22,11,90] [64,22] → swap → [34,25,12,22,64,11,90] [64,11] → swap → [34,25,12,22,11,64,90] [64,90] → ok → [34,25,12,22,11,64,90] ← 90 bubbled to end ✓ Pass 2: [25,12,22,11,34,64,|90] ← 64 settles ✓ Pass 3: [12,22,11,25,34,|64,90] ... continue until fully sorted
using System;

// ─── Bubble Sort — Complete Implementation ────────────────────────────────────

class BubbleSort
{
    /// Sorts array in ascending order using bubble sort.
    /// Time: O(n²) worst/avg, O(n) best (already sorted with optimization)
    /// Space: O(1) — in-place
    /// Stable: Yes — equal elements maintain their relative order
    static void Sort(int[] arr)
    {
        int n = arr.Length;

        for (int i = 0; i < n - 1; i++)
        {
            bool swapped = false;  // optimization: detect early termination

            // After pass i, the last i elements are already in place
            for (int j = 0; j < n - 1 - i; j++)
            {
                // Compare adjacent elements
                if (arr[j] > arr[j + 1])
                {
                    // Swap them
                    (int tmp, arr[j], arr[j+1]) = (arr[j], arr[j+1], arr[j]);
                    swapped = true;
                }
            }

            // If no swaps occurred, array is already sorted
            if (!swapped) break;
        }
    }

    static void Main()
    {
        int[] arr1 = { 64, 34, 25, 12, 22, 11, 90 };
        Sort(arr1);
        Console.WriteLine("Sorted: " + string.Join(", ", arr1));
        // Sorted: 11, 12, 22, 25, 34, 64, 90

        // Test: already sorted (best case — should be O(n))
        int[] arr2 = { 1, 2, 3, 4, 5 };
        Sort(arr2);
        Console.WriteLine("Already sorted: " + string.Join(", ", arr2));

        // Test: reverse sorted (worst case)
        int[] arr3 = { 5, 4, 3, 2, 1 };
        Sort(arr3);
        Console.WriteLine("Reversed: " + string.Join(", ", arr3));
    }
}

/* ── Output ──────────────────────────────────────────────────────────────────
   Sorted: 11, 12, 22, 25, 34, 64, 90
   Already sorted: 1, 2, 3, 4, 5
   Reversed: 1, 2, 3, 4, 5
   ─────────────────────────────────────────────────────────────────────────── */
# ─── Bubble Sort — Complete Implementation ────────────────────────────────────
from typing import List

def bubble_sort(arr: List[int]) -> List[int]:
    """Sorts array in ascending order using bubble sort.
    Modifies in-place and returns the array.
    Time: O(n²) worst, O(n) best | Space: O(1) | Stable: Yes
    """
    n = len(arr)
    for i in range(n - 1):
        swapped = False   # early termination optimisation

        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]  # swap
                swapped = True

        if not swapped:
            break   # already sorted

    return arr


if __name__ == "__main__":
    print(bubble_sort([64,34,25,12,22,11,90]))  # [11,12,22,25,34,64,90]
    print(bubble_sort([1,2,3,4,5]))             # already sorted: [1,2,3,4,5]
    print(bubble_sort([5,4,3,2,1]))             # reversed: [1,2,3,4,5]

Section 3 — Selection Sort

3.1 The Analogy — Pick the Smallest Card

Imagine sorting a hand of playing cards. You look through all your cards, find the smallest one, and move it to the far left. Then look through the remaining cards, find the smallest, move it second from left. Repeat until done. That's selection sort.

3.2 How It Works

Divide the array into sorted (left) and unsorted (right) portions. In each pass, find the minimum element in the unsorted portion and swap it into the sorted portion.

Array: [29, 10, 14, 37, 13] Pass 1: Find min in [29,10,14,37,13] → 10 at index 1. Swap with index 0. [10 | 29, 14, 37, 13] ← 10 sorted Pass 2: Find min in [29,14,37,13] → 13 at index 4. Swap with index 1. [10, 13 | 29, 37, 14] ← 13 sorted Pass 3: Find min in [29,37,14] → 14 at index 4. Swap with index 2. [10, 13, 14 | 37, 29] Pass 4: Find min in [37,29] → 29. Swap. [10, 13, 14, 29, 37] ← fully sorted! 
using System;

// ─── Selection Sort — Complete Implementation ─────────────────────────────────

class SelectionSort
{
    /// Sorts array using selection sort.
    /// Time: O(n²) always — makes exactly n*(n-1)/2 comparisons regardless
    /// Space: O(1) — in-place
    /// Stable: No — a swap can change relative order of equal elements
    /// Key advantage: makes at most n-1 SWAPS (minimizes writes — good for SSDs)
    static void Sort(int[] arr)
    {
        int n = arr.Length;

        for (int i = 0; i < n - 1; i++)
        {
            // Find the minimum element in arr[i..n-1]
            int minIndex = i;
            for (int j = i + 1; j < n; j++)
            {
                if (arr[j] < arr[minIndex])
                    minIndex = j;
            }

            // Swap minimum element with the first unsorted element
            if (minIndex != i)
                (arr[i], arr[minIndex]) = (arr[minIndex], arr[i]);
        }
    }

    static void Main()
    {
        int[] arr = { 29, 10, 14, 37, 13 };
        Sort(arr);
        Console.WriteLine(string.Join(", ", arr));  // 10, 13, 14, 29, 37

        int[] arr2 = { 5,5,5,1,2,3 };
        Sort(arr2);
        Console.WriteLine(string.Join(", ", arr2));  // 1, 2, 3, 5, 5, 5
    }
}
from typing import List

def selection_sort(arr: List[int]) -> List[int]:
    """Selection sort: find minimum, place in sorted section.
    Time: O(n²) always | Space: O(1) | Stable: No
    Advantage: minimises number of swaps (at most n-1)
    """
    n = len(arr)
    for i in range(n - 1):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr

if __name__ == "__main__":
    print(selection_sort([29,10,14,37,13]))  # [10,13,14,29,37]
    print(selection_sort([5,5,5,1,2,3]))    # [1,2,3,5,5,5]

Section 4 — Insertion Sort

4.1 The Analogy — Sorting Playing Cards in Your Hand

Think of picking up playing cards one at a time. As you pick each new card, you slide it into the correct position among the cards already in your hand. You don't reshuffle everything — you just find where the new card belongs and insert it there. That's exactly insertion sort.

4.2 How It Works

Start with one element (sorted). For each new element, shift all larger sorted elements one position right to make space, then insert the new element in the gap.

Array: [12, 11, 13, 5, 6] Step 1: [12] — single element, sorted Step 2: key=11. Compare with 12: 11<12, shift 12 right. Insert 11. [11, 12, 13, 5, 6] Step 3: key=13. Compare with 12: 13>12, no shift. Insert in place. [11, 12, 13, 5, 6] Step 4: key=5. Shift 13,12,11 right. Insert 5 at front. [5, 11, 12, 13, 6] Step 5: key=6. Shift 13,12,11 right. Insert 6. [5, 6, 11, 12, 13] ← sorted! 
using System;

// ─── Insertion Sort — Complete Implementation ─────────────────────────────────

class InsertionSort
{
    /// Time: O(n²) worst, O(n) best (nearly sorted!)
    /// Space: O(1) — in-place
    /// Stable: Yes
    /// BEST USE CASE: nearly-sorted data — real world this is common!
    static void Sort(int[] arr)
    {
        int n = arr.Length;

        for (int i = 1; i < n; i++)
        {
            int key = arr[i];     // element to insert
            int j = i - 1;       // start comparing from left of key

            // Shift elements greater than key one position to the right
            while (j >= 0 && arr[j] > key)
            {
                arr[j + 1] = arr[j];  // shift right
                j--;
            }

            // Insert key in its correct position
            arr[j + 1] = key;
        }
    }

    static void Main()
    {
        int[] arr1 = { 12, 11, 13, 5, 6 };
        Sort(arr1);
        Console.WriteLine(string.Join(", ", arr1));  // 5, 6, 11, 12, 13

        // Nearly sorted: insertion sort shines!
        int[] arr2 = { 1, 2, 3, 5, 4 };  // only 1 swap needed
        Sort(arr2);
        Console.WriteLine(string.Join(", ", arr2));  // 1, 2, 3, 4, 5
    }
}
from typing import List

def insertion_sort(arr: List[int]) -> List[int]:
    """Insertion sort: insert each element into sorted position.
    Time: O(n²) worst, O(n) best | Space: O(1) | Stable: Yes
    Best for: nearly-sorted data
    """
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]  # shift right
            j -= 1
        arr[j + 1] = key         # insert
    return arr

if __name__ == "__main__":
    print(insertion_sort([12,11,13,5,6]))  # [5,6,11,12,13]
    print(insertion_sort([1,2,3,5,4]))   # [1,2,3,4,5]

Section 5 — Algorithm Properties Explained

5.1 Stability

A sorting algorithm is stable if equal elements maintain their original relative order after sorting. This matters when sorting objects with multiple fields (e.g., sort students by grade, keeping same-grade students in alphabetical order).

📝 Stable vs UnstableBubble Sort: ✅ Stable — equal elements never swap (only strictly greater triggers swap). Insertion Sort: ✅ Stable — shifts only when strictly greater. Selection Sort: ❌ Unstable — a distant swap can displace equal elements.

5.2 In-place

An algorithm is in-place if it uses O(1) extra memory (besides the input array). All three O(n²) sorts are in-place. Merge Sort (next module) is NOT in-place — it needs O(n) extra space.

5.3 Adaptive

An algorithm is adaptive if it performs better on partially sorted data. Insertion sort is adaptive — on nearly sorted input, it approaches O(n). Bubble sort with the swapped flag is also adaptive. Selection sort is NOT adaptive — always O(n²).

Section 7 — Practice Questions

❓ Q1: How many comparisons does bubble sort make for an array of n=5 elements in the worst case?
✅ n*(n-1)/2 = 5*4/2 = 10 comparisons

Pass 1: 4 comparisons. Pass 2: 3. Pass 3: 2. Pass 4: 1. Total: 4+3+2+1 = 10. This is O(n²).

❓ Q2: Trace insertion sort on [5, 3, 1, 4, 2]. Show the array after each pass.
✅ Step-by-step trace:

Start: [5, 3, 1, 4, 2]
i=1, key=3: shift 5 right → [3, 5, 1, 4, 2]
i=2, key=1: shift 5,3 right → [1, 3, 5, 4, 2]
i=3, key=4: shift 5 right → [1, 3, 4, 5, 2]
i=4, key=2: shift 5,4,3 right → [1, 2, 3, 4, 5]

❓ Q3: Which O(n²) sort is NOT stable and why does that matter?
✅ Selection Sort is not stable

Selection sort swaps the current element with the minimum element found, which could be far away. This can change the relative order of equal elements. Example: [3a, 3b, 1] → after pass 1, 1 and 3a swap → [1, 3b, 3a]. The two 3s changed order. This matters when sorting objects where equal keys should maintain original order (e.g., sorting by last name then by first name).

❓ Q4: Write a function to sort an array using insertion sort in DESCENDING order.
✅ Change the comparison from > to <
static void InsertionSortDesc(int[] arr) {
    for (int i=1; i<arr.Length; i++) {
        int key=arr[i], j=i-1;
        while (j>=0 && arr[j] < key) {  // changed > to < for descending
            arr[j+1]=arr[j]; j--;
        }
        arr[j+1]=key;
    }
}
// Result: [13, 12, 11, 6, 5] for input [12,11,13,5,6]
def insertion_sort_desc(arr):
    for i in range(1, len(arr)):
        key, j = arr[i], i-1
        while j >= 0 and arr[j] < key:  # changed > to < for descending
            arr[j+1] = arr[j]; j -= 1
        arr[j+1] = key
    return arr
