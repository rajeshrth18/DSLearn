Section 1 Ś Divide and Conquer

Both merge sort and quick sort use the divide and conquer strategy:

  1. Divide: Split the problem into smaller sub-problems
  2. Conquer: Solve each sub-problem recursively
  3. Combine: Merge the solutions

Why does this give O(n log n)? Because we split n elements into log2(n) levels, and each level does O(n) total work.

n=8 elements split into log2(8)=3 levels: Level 0: [38,27,43,3,9,82,10,1] ? n=8 work Level 1: [38,27,43,3] [9,82,10,1] ? n=8 total work Level 2: [38,27][43,3][9,82][10,1] ? n=8 total work Level 3: individual elements ? base cases Total: 3 levels ū n work each = n log n = 8 ū 3 = 24 operations Compare to O(n▓): 8▓ = 64 operations

Section 2 Ś Merge Sort Deep Dive

2.1 The Analogy Ś Merging Two Sorted Stacks of Papers

Imagine you have two sorted stacks of numbered papers. To merge them into one sorted stack: look at the top of each stack, take the smaller one, place it on the result stack. Repeat until both stacks are empty. This merge operation is O(n).

Merge sort works by recursively splitting the array until each piece has 1 element (trivially sorted), then merging sorted pieces back together.

Merge Sort Trace: [38, 27, 43, 3] Split: [38, 27, 43, 3] +-- [38, 27] ” +-- [38] (sorted) ” +-- [27] (sorted) +-- [43, 3] +-- [43] (sorted) +-- [3] (sorted) Merge: [27] + [38] ? merge ? [27, 38] [3] + [43] ? merge ? [3, 43] [27,38] + [3,43] ? merge ? [3, 27, 38, 43] ?

2.2 Complete Merge Sort Implementation

using System;

// --- Merge Sort Ś Complete Implementation -------------------------------------
// Time:  O(n log n) Ś all cases (best/avg/worst identical)
// Space: O(n) Ś needs auxiliary array for merging
// Stable: Yes Ś equal elements keep their relative order

class MergeSort
{
    /// Main sort function Ś divides array in half recursively
    static void Sort(int[] arr, int left, int right)
    {
        // BASE CASE: subarray of 0 or 1 element is already sorted
        if (left >= right) return;

        int mid = left + (right - left) / 2;  // avoid int overflow

        Sort(arr, left, mid);        // sort left half
        Sort(arr, mid + 1, right);  // sort right half
        Merge(arr, left, mid, right); // merge both halves
    }

    /// Merges arr[left..mid] and arr[mid+1..right] (both sorted)
    static void Merge(int[] arr, int left, int mid, int right)
    {
        int n1 = mid - left + 1;
        int n2 = right - mid;

        // Create temp arrays for left and right halves
        int[] L = new int[n1];
        int[] R = new int[n2];

        // Copy data to temp arrays
        for (int i = 0; i < n1; i++) L[i] = arr[left + i];
        for (int j = 0; j < n2; j++) R[j] = arr[mid + 1 + j];

        int ii = 0, jj = 0, k = left;

        // Merge L and R back into arr in sorted order
        while (ii < n1 && jj < n2)
        {
            if (L[ii] <= R[jj])  // <= makes it stable
                arr[k++] = L[ii++];
            else
                arr[k++] = R[jj++];
        }

        // Copy remaining elements
        while (ii < n1) arr[k++] = L[ii++];
        while (jj < n2) arr[k++] = R[jj++];
    }

    static void Main()
    {
        int[] arr = { 38, 27, 43, 3, 9, 82, 10 };
        Console.WriteLine("Before: " + string.Join(", ", arr));
        Sort(arr, 0, arr.Length - 1);
        Console.WriteLine("After:  " + string.Join(", ", arr));
        // After: 3, 9, 10, 27, 38, 43, 82
    }
}
from typing import List

# --- Merge Sort Ś Complete Implementation -------------------------------------

def merge_sort(arr: List[int]) -> List[int]:
    """Merge sort: split, sort halves, merge.
    Time: O(n log n) all cases | Space: O(n) | Stable: Yes
    """
    if len(arr) <= 1:
        return arr    # base case: already sorted

    mid = len(arr) // 2
    left  = merge_sort(arr[:mid])    # sort left half
    right = merge_sort(arr[mid:])    # sort right half
    return merge(left, right)         # merge and return


def merge(L: List[int], R: List[int]) -> List[int]:
    """Merges two sorted lists into one sorted list."""
    result, i, j = [], 0, 0

    while i < len(L) and j < len(R):
        if L[i] <= R[j]:   # <= for stability
            result.append(L[i]); i += 1
        else:
            result.append(R[j]); j += 1

    result.extend(L[i:])
    result.extend(R[j:])
    return result


if __name__ == "__main__":
    arr = [38, 27, 43, 3, 9, 82, 10]
    print("Before:", arr)
    sorted_arr = merge_sort(arr)
    print("After: ", sorted_arr)  # [3, 9, 10, 27, 38, 43, 82]

Section 3 Ś Merge Sort Counting Inversions (Bonus)

An inversion is a pair (i,j) where i < j but arr[i] > arr[j]. Counting inversions measures "how unsorted" an array is. Merge sort can count inversions in O(n log n) Ś a beautiful application!

using System;

// --- Count Inversions using Modified Merge Sort -------------------------------
// An inversion is a pair where arr[i] > arr[j] but i < j
// Example: [3,1,2] has 2 inversions: (3,1) and (3,2)

class InversionCount
{
    static long MergeSortCount(int[] arr, int left, int right)
    {
        if (left >= right) return 0;
        int mid = (left + right) / 2;
        long count = 0;
        count += MergeSortCount(arr, left, mid);
        count += MergeSortCount(arr, mid+1, right);
        count += MergeAndCount(arr, left, mid, right);
        return count;
    }

    static long MergeAndCount(int[] arr, int left, int mid, int right)
    {
        int n1=mid-left+1, n2=right-mid;
        int[] L=new int[n1], R=new int[n2];
        for(int i=0;i<n1;i++) L[i]=arr[left+i];
        for(int j=0;j<n2;j++) R[j]=arr[mid+1+j];
        int ii=0,jj=0,k=left; long inv=0;
        while(ii<n1 && jj<n2) {
            if(L[ii]<=R[jj]) arr[k++]=L[ii++];
            else { inv+=n1-ii; arr[k++]=R[jj++]; } // all remaining L elements form inversions!
        }
        while(ii<n1) arr[k++]=L[ii++];
        while(jj<n2) arr[k++]=R[jj++];
        return inv;
    }

    static void Main() {
        int[] arr = { 3, 1, 2 };
        Console.WriteLine("Inversions: " + MergeSortCount(arr, 0, arr.Length-1)); // 2
        int[] arr2 = { 7,5,6,1,3 };
        Console.WriteLine("Inversions: " + MergeSortCount(arr2, 0, arr2.Length-1)); // 7
    }
}
from typing import List, Tuple

def count_inversions(arr: List[int]) -> Tuple[List[int], int]:
    if len(arr) <= 1: return arr, 0
    mid = len(arr) // 2
    left, left_inv  = count_inversions(arr[:mid])
    right, right_inv = count_inversions(arr[mid:])
    merged, split_inv = merge_count(left, right)
    return merged, left_inv + right_inv + split_inv

def merge_count(L, R):
    result, i, j, inv = [], 0, 0, 0
    while i < len(L) and j < len(R):
        if L[i] <= R[j]: result.append(L[i]); i+=1
        else: inv += len(L)-i; result.append(R[j]); j+=1
    result.extend(L[i:]); result.extend(R[j:])
    return result, inv

_, inv = count_inversions([3,1,2])
print(f"Inversions: {inv}")  # 2

Section 4 Ś Quick Sort Deep Dive

4.1 The Analogy Ś Sorting a Line of People by Height

Pick one person (the pivot). Ask everyone shorter to stand on the left and everyone taller to stand on the right. Now the pivot is in the perfect final position! Repeat this process for the left group and right group separately. This is quick sort.

4.2 The Partition Step

The key operation in quick sort is partition: rearrange the array so all elements less than the pivot come before it, and all greater elements come after it. After partitioning, the pivot is in its final sorted position.

Quick Sort Trace: [10, 80, 30, 90, 40, 50, 70] Pivot: 70 (last element) Partition step (Lomuto scheme): i=-1, scan left to right: 10 < 70? Yes ? i=0, arr=[10, 80, 30, 90, 40, 50, 70] 80 < 70? No ? skip 30 < 70? Yes ? i=1, swap 30?80 ? [10, 30, 80, 90, 40, 50, 70] 90 < 70? No ? skip 40 < 70? Yes ? i=2, swap 40?80 ? [10, 30, 40, 90, 80, 50, 70] 50 < 70? Yes ? i=3, swap 50?90 ? [10, 30, 40, 50, 80, 90, 70] Place pivot: swap arr[i+1] ? pivot ? [10, 30, 40, 50, 70, 90, 80] ? 70 is in FINAL POSITION! Recurse on [10,30,40,50] and [90,80] 
using System;

// --- Quick Sort Ś Complete Implementation (Lomuto Partition) ------------------
// Time:  O(n log n) avg, O(n▓) worst (when array already sorted + bad pivot)
// Space: O(log n) avg Ś call stack
// Stable: No
// In-place: Yes

class QuickSort
{
    static Random _rng = new Random();

    /// Main sort function
    static void Sort(int[] arr, int low, int high)
    {
        if (low < high)
        {
            // Partition: place pivot in correct position
            int pivotIndex = Partition(arr, low, high);

            Sort(arr, low, pivotIndex - 1);  // sort elements before pivot
            Sort(arr, pivotIndex + 1, high); // sort elements after pivot
        }
    }

    /// Lomuto partition: uses last element as pivot
    /// Returns the final position of the pivot
    static int Partition(int[] arr, int low, int high)
    {
        // Randomise pivot to avoid worst case on sorted arrays
        int randomIdx = _rng.Next(low, high + 1);
        (arr[randomIdx], arr[high]) = (arr[high], arr[randomIdx]); // swap random with last

        int pivot = arr[high];
        int i = low - 1;  // index of last element smaller than pivot

        for (int j = low; j < high; j++)
        {
            if (arr[j] <= pivot)
            {
                i++;
                (arr[i], arr[j]) = (arr[j], arr[i]);  // swap
            }
        }

        // Place pivot in its final position
        (arr[i + 1], arr[high]) = (arr[high], arr[i + 1]);
        return i + 1;
    }

    static void Main()
    {
        int[] arr = { 10, 80, 30, 90, 40, 50, 70 };
        Console.WriteLine("Before: " + string.Join(", ", arr));
        Sort(arr, 0, arr.Length - 1);
        Console.WriteLine("After:  " + string.Join(", ", arr));
        // After: 10, 30, 40, 50, 70, 80, 90

        // Test with duplicates
        int[] arr2 = { 3, 6, 8, 10, 1, 2, 1 };
        Sort(arr2, 0, arr2.Length - 1);
        Console.WriteLine("Duplicates: " + string.Join(", ", arr2));
        // Duplicates: 1, 1, 2, 3, 6, 8, 10
    }
}
import random
from typing import List

# --- Quick Sort Ś Complete Implementation -------------------------------------

def quick_sort(arr: List[int], low: int, high: int) -> None:
    """Quick sort in-place. Call with quick_sort(arr, 0, len(arr)-1)
    Time: O(n log n) avg, O(n▓) worst | Space: O(log n) | Stable: No
    """
    if low < high:
        pivot_idx = partition(arr, low, high)
        quick_sort(arr, low, pivot_idx - 1)
        quick_sort(arr, pivot_idx + 1, high)


def partition(arr: List[int], low: int, high: int) -> int:
    # Randomise pivot to avoid worst case
    rand_idx = random.randint(low, high)
    arr[rand_idx], arr[high] = arr[high], arr[rand_idx]

    pivot = arr[high]
    i = low - 1

    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]

    arr[i+1], arr[high] = arr[high], arr[i+1]
    return i + 1


if __name__ == "__main__":
    arr = [10, 80, 30, 90, 40, 50, 70]
    print("Before:", arr)
    quick_sort(arr, 0, len(arr)-1)
    print("After: ", arr)  # [10,30,40,50,70,80,90]

Section 5 Ś 3-Way Quick Sort (Dutch National Flag)

When the array has many duplicate elements, standard quick sort degrades because all duplicates end up in one partition. The 3-way partition creates three regions: <pivot, =pivot, >pivot. Elements equal to the pivot are already in their final positions.

using System;

// --- 3-Way Quick Sort (Dutch National Flag) -----------------------------------
// Handles arrays with many duplicates efficiently: O(n) for all-equal arrays

class ThreeWayQS
{
    static void Sort(int[] arr, int lo, int hi)
    {
        if (lo >= hi) return;

        int pivot = arr[lo];
        int lt = lo;    // arr[lo..lt-1] < pivot
        int gt = hi;    // arr[gt+1..hi] > pivot
        int i  = lo;    // current element

        while (i <= gt)
        {
            if      (arr[i] < pivot) (arr[lt++], arr[i++]) = (arr[i], arr[lt]);
            else if (arr[i] > pivot) (arr[i], arr[gt--]) = (arr[gt], arr[i]);
            else                     i++;
        }
        // arr[lo..lt-1] < pivot, arr[lt..gt] == pivot, arr[gt+1..hi] > pivot

        Sort(arr, lo, lt - 1);  // sort less-than part
        Sort(arr, gt + 1, hi);  // sort greater-than part
        // skip equal part Ś already in final positions!
    }

    static void Main()
    {
        int[] arr = { 5,3,5,1,5,3,2,5,4 };
        Sort(arr, 0, arr.Length-1);
        Console.WriteLine(string.Join(", ", arr)); // 1,2,3,3,4,5,5,5,5
    }
}
from typing import List

def three_way_qs(arr: List[int], lo: int, hi: int) -> None:
    if lo >= hi: return
    pivot = arr[lo]
    lt, i, gt = lo, lo, hi
    while i <= gt:
        if   arr[i] < pivot: arr[lt], arr[i] = arr[i], arr[lt]; lt+=1; i+=1
        elif arr[i] > pivot: arr[i], arr[gt] = arr[gt], arr[i]; gt-=1
        else: i+=1
    three_way_qs(arr, lo, lt-1)
    three_way_qs(arr, gt+1, hi)

arr = [5,3,5,1,5,3,2,5,4]
three_way_qs(arr, 0, len(arr)-1)
print(arr)  # [1,2,3,3,4,5,5,5,5]

Section 6 Ś Merge Sort vs Quick Sort

PropertyMerge SortQuick Sort
Best TimeO(n log n)O(n log n)
Average TimeO(n log n)O(n log n)
Worst TimeO(n log n)O(n▓) without random pivot
SpaceO(n) Ś extra array neededO(log n) Ś just call stack
Stable? Yes? No
Cache performanceFairExcellent Ś sequential access
Best forLinked lists, stable neededArrays in memory, fastest avg
?? Real-World ChoiceQuick sort is typically faster in practice for in-memory arrays (better cache locality). Merge sort is preferred when: sorting linked lists (no random access), need stable sort, sorting external data (disk/database). Python's sorted() uses TimSort (merge + insertion). Java's Arrays.sort() for objects uses TimSort; for primitives uses dual-pivot quick sort.
?? Quick Sort Worst CaseIf you always pick the first or last element as pivot, and the array is already sorted, quick sort degrades to O(n▓). Solution: use random pivot or median-of-three (middle element of arr[first], arr[mid], arr[last]).

Section 7 Ś Practice Questions

? Q1: What is the space complexity of recursive merge sort? Why?
? O(n) space total

Two reasons: (1) The merge step creates temporary arrays totaling O(n) at each level. (2) The call stack goes O(log n) deep. The dominant term is O(n) from the temp arrays. This is the main disadvantage of merge sort over quick sort.

? Q2: After one partition step with pivot=5 on [3,8,1,5,9,2,7], what does the array look like?
? Elements <5 on left, 5 in middle, elements >5 on right

One possible result: [3, 1, 2, 5, 9, 8, 7]. The exact arrangement of elements within each group may vary by implementation, but 5 is always at its final sorted position (index 3 for this array). 5 never moves again.

? Q3: Merge sort on [2, 4, 1, 6, 3]: trace the merge steps showing how the sorted array is built bottom-up.
? Bottom-up merge trace:

Split: [2,4,1,6,3] ? [2,4] [1,6,3]
[2,4] ? [2] [4] ? merge ? [2,4]
[1,6,3] ? [1] [6,3] ? [6] [3] ? merge ? [3,6] ? merge [1]+[3,6] ? [1,3,6]
Final merge: [2,4] + [1,3,6] ? [1,2,3,4,6]
