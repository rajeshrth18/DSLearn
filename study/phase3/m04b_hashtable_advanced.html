Section 1 — The Two-Sum Pattern

The Two-Sum pattern is the most fundamental hash table technique in competitive programming and interviews. It transforms an O(n²) brute-force search into an O(n) solution by using the dictionary to "remember what we've seen" and look up the complement in O(1).

🌍 Real-World Example: A supermarket checkout system needs to find any two items whose prices total exactly $50. Brute force checks every pair (O(n²)). Hash map approach: for each item, check if ($50 - item.price) is already in the map — O(n).

1.1 — Classic Two Sum (return indices)

TWO SUM TRACE — nums=[2,7,11,15], target=9 One-pass hash map: store {value → index} as we scan i=0: value=2, complement=9-2=7. Map={}. 7 not in map. Add {2→0} i=1: value=7, complement=9-7=2. Map={2:0}. 2 IS in map at index 0! → return [0, 1] ✓ Map never grows beyond current index → O(1) lookup per element → O(n) total
// Two Sum — Classic LeetCode #1
public int[] TwoSum(int[] nums, int target)
{
    var map = new Dictionary<int, int>(); // value → index
    for (int i = 0; i < nums.Length; i++)
    {
        int complement = target - nums[i];
        if (map.TryGetValue(complement, out int j))
            return new [] { j, i };
        map[nums[i]] = i;
    }
    return Array.Empty<int>();
}
// [2,7,11,15], target=9  → [0, 1]
// [3,2,4],     target=6  → [1, 2]
// [3,3],       target=6  → [0, 1]
// Time: O(n), Space: O(n)

1.2 — Two Sum Variants

// Variant: Does ANY pair sum to target? (return bool)
public bool HasPairWithSum(int[] nums, int target)
{
    var seen = new HashSet<int>();
    foreach (int n in nums)
    {
        if (seen.Contains(target - n)) return true;
        seen.Add(n);
    }
    return false;
}

// Variant: Count ALL pairs that sum to target
public int CountPairs(int[] nums, int target)
{
    var freq = new Dictionary<int, int>();
    int count = 0;
    foreach (int n in nums)
    {
        count += freq.GetValueOrDefault(target - n);
        freq[n] = freq.GetValueOrDefault(n) + 1;
    }
    return count;
}
// [1,5,3,3,1,5,1], target=6 → 3 pairs: (1,5),(3,3),(1,5)

// Variant: Four Sum Count (LeetCode 454)
// Count tuples (i,j,k,l) such that A[i]+B[j]+C[k]+D[l] == 0
// Key insight: split into two halves → O(n²) instead of O(n⁴)
public int FourSumCount(int[] A, int[] B, int[] C, int[] D)
{
    var abSums = new Dictionary<int, int>();
    foreach (int a in A)
        foreach (int b in B)
            abSums[a + b] = abSums.GetValueOrDefault(a + b) + 1;
    int count = 0;
    foreach (int c in C)
        foreach (int d in D)
            count += abSums.GetValueOrDefault(-(c + d));
    return count;
}
// Time: O(n²) — far better than O(n⁴) brute force
Section 2 — Frequency Maps & Counting

Counting how many times each element appears is one of the most common operations in data processing, text analysis, and algorithm optimization. A frequency map (Dictionary<T,int>) answers "how many times does X appear?" in O(1) after O(n) build time.

2.1 — Building a Frequency Map

// ── Word frequency in a sentence ──────────────────────────
string text = "the cat sat on the mat the cat";
string[] wordsArr = text.Split(' ');

var freq = new Dictionary<string, int>();
foreach (string w in wordsArr)
    freq[w] = freq.GetValueOrDefault(w) + 1;
// freq: { "the":3, "cat":2, "sat":1, "on":1, "mat":1 }

// LINQ alternative (concise but slightly slower)
var freqLinq = wordsArr.GroupBy(w => w)
                        .ToDictionary(g => g.Key, g => g.Count());

// ── Character frequency (useful for anagrams) ─────────────
int[] CharFreq(string s)
{
    int[] count = new int[26];
    foreach (char c in s) count[c - 'a']++;
    return count;
}
// Use array[26] instead of Dictionary when keys are only 'a'-'z' — faster!

2.2 — Top K Frequent Elements

TOP K FREQUENT — nums=[1,1,1,2,2,3], k=2 → [1,2] Step 1: Build frequency map {1:3, 2:2, 3:1} Step 2: Use MinHeap of size k to track top-k - Push (freq=3, val=1) → heap: [(3,1)] - Push (freq=2, val=2) → heap: [(2,2),(3,1)] - Push (freq=1, val=3) → heap size = k+1 = 3 → Pop minimum (1,3): heap: [(2,2),(3,1)] Result: extract remaining → [1, 2] ✓ Alternative: Bucket Sort — O(n) time, O(n) space - Create buckets[0..n], where bucket[i] = all elements with frequency i - Scan buckets from high to low, collect k elements
// Top K Frequent — MinHeap approach (general, O(n log k))
public int[] TopKFrequent(int[] nums, int k)
{
    var freq = new Dictionary<int, int>();
    foreach (int n in nums) freq[n] = freq.GetValueOrDefault(n) + 1;

    // Min-heap keyed by frequency (smallest freq at top)
    var heap = new PriorityQueue<int, int>();
    foreach (var (val, cnt) in freq)
    {
        heap.Enqueue(val, cnt);
        if (heap.Count > k) heap.Dequeue(); // evict least frequent
    }
    var result = new int[k];
    for (int i = k - 1; i >= 0; i--) result[i] = heap.Dequeue();
    return result;
}

// Top K Frequent — Bucket Sort approach (O(n) time)
public int[] TopKFrequentBucket(int[] nums, int k)
{
    var freq = new Dictionary<int, int>();
    foreach (int n in nums) freq[n] = freq.GetValueOrDefault(n) + 1;

    // Buckets: index = frequency, value = list of numbers with that frequency
    var buckets = new List<int>[nums.Length + 1];
    foreach (var (val, cnt) in freq)
    {
        buckets[cnt] ??= new List<int>();
        buckets[cnt].Add(val);
    }

    var result = new List<int>();
    for (int i = buckets.Length - 1; i >= 1 && result.Count < k; i--)
        if (buckets[i] != null) result.AddRange(buckets[i]);
    return result.Take(k).ToArray();
}
// [1,1,1,2,2,3], k=2 → [1, 2]
Section 3 — Grouping & Anagram Detection

Anagram detection is a classic hash map pattern. Two strings are anagrams if they contain the same characters with the same frequencies. The key insight: represent each string by a canonical key that all its anagrams share.

3.1 — Is Anagram? (single pair)

// Method 1: Sort both strings — O(n log n)
public bool IsAnagramSort(string s, string t)
{
    if (s.Length != t.Length) return false;
    char[] a = s.ToCharArray(), b = t.ToCharArray();
    Array.Sort(a); Array.Sort(b);
    return new string(a) == new string(b);
}

// Method 2: Character frequency array — O(n) time, O(1) space
public bool IsAnagram(string s, string t)
{
    if (s.Length != t.Length) return false;
    int[] count = new int[26];
    foreach (char c in s) count[c - 'a']++;
    foreach (char c in t) count[c - 'a']--;
    foreach (int n in count) if (n != 0) return false;
    return true;
}
// "anagram","nagaram" → true
// "rat","car"         → false

3.2 — Group Anagrams (classic interview problem)

GROUP ANAGRAMS — ["eat","tea","tan","ate","nat","bat"] Canonical key: sorted string of the word "eat" → sorted → "aet" "tea" → sorted → "aet" SAME KEY as "eat" "tan" → sorted → "ant" "ate" → sorted → "aet" SAME KEY "nat" → sorted → "ant" SAME KEY as "tan" "bat" → sorted → "abt" Dictionary: "aet" → ["eat","tea","ate"] "ant" → ["tan","nat"] "abt" → ["bat"] Result: [["eat","tea","ate"],["tan","nat"],["bat"]] ✓
public IList<IList<string>> GroupAnagrams(string[] strs)
{
    var groups = new Dictionary<string, List<string>>();
    foreach (string s in strs)
    {
        char[] chars = s.ToCharArray();
        Array.Sort(chars);
        string key = new string(chars);  // canonical key

        if (!groups.ContainsKey(key))
            groups[key] = new List<string>();
        groups[key].Add(s);
    }
    return new List<IList<string>>(groups.Values);
}

// Optimized key: encode character counts as "#1#2#0#..." instead of sorting
public IList<IList<string>> GroupAnagramsFast(string[] strs)
{
    var groups = new Dictionary<string, List<string>>();
    foreach (string s in strs)
    {
        int[] cnt = new int[26];
        foreach (char c in s) cnt[c - 'a']++;
        string key = string.Join(",", cnt);  // "1,0,0,0,1,0,...,1,0,0" — O(1) since 26 chars

        if (!groups.ContainsKey(key)) groups[key] = new List<string>();
        groups[key].Add(s);
    }
    return new List<IList<string>>(groups.Values);
}
// O(n·k) time where k = average string length. Better than O(n·k log k) with sorting.
Section 4 — Prefix Sum + Hash Map

Combining prefix sums with a hash map unlocks O(n) solutions for subarray problems that would otherwise require O(n²) nested loops. The key idea: store previously seen prefix sums in a map, so you can check in O(1) whether a subarray ending here has the required sum.

🌍 Analogy: You're tracking your cumulative spending each day. You want to know if there's any period where you spent exactly $500. You store each day's cumulative total in a map. If today's total minus $500 appears in the map, that period existed.

4.1 — Subarray Sum Equals K

SUBARRAY SUM = K — nums=[3,4,7,2,-3,1,4,2], k=7 Prefix sums: [0, 3, 7, 14, 16, 13, 14, 18, 20] ↑ prefix[0]=0 is always in map (empty subarray) Key insight: subarray[i..j] sum = prefix[j+1] - prefix[i] If prefix[j+1] - prefix[i] = k, then prefix[i] = prefix[j+1] - k For each position, check: does (current_prefix - k) exist in map? i=0: prefix=3. Need 3-7=-4. Map={0:1}. Not found. Add {3:1} i=1: prefix=7. Need 7-7=0. Map={0:1,3:1}. FOUND! count+=1 i=2: prefix=14. Need 14-7=7. Map has 7:1. count+=1 ... Final count = 4 subarrays summing to 7
public int SubarraySum(int[] nums, int k)
{
    var prefixCount = new Dictionary<int, int> { [0] = 1 }; // empty subarray
    int prefix = 0, count = 0;
    foreach (int n in nums)
    {
        prefix += n;
        count += prefixCount.GetValueOrDefault(prefix - k);
        prefixCount[prefix] = prefixCount.GetValueOrDefault(prefix) + 1;
    }
    return count;
}
// [1,1,1], k=2 → 2  ([1,1] at positions [0,1] and [1,2])
// [1,2,3], k=3 → 2  ([3] and [1,2])
// Time: O(n), Space: O(n)

// Variant: Longest subarray with sum = 0
public int MaxLenSubarrayZeroSum(int[] nums)
{
    var firstSeen = new Dictionary<int, int> { [0] = -1 }; // prefix → first index
    int prefix = 0, maxLen = 0;
    for (int i = 0; i < nums.Length; i++)
    {
        prefix += nums[i];
        if (firstSeen.TryGetValue(prefix, out int j))
            maxLen = Math.Max(maxLen, i - j);
        else
            firstSeen[prefix] = i;
    }
    return maxLen;
}
// [15,-2,2,-8,1,7,10,23] → 5  (subarray [-2,2,-8,1,7])
Section 5 — LRU Cache

An LRU (Least Recently Used) Cache evicts the least-recently-accessed item when the cache is full. It is one of the most commonly asked system design and coding interview problems because it combines a hash map (O(1) lookup) with a doubly linked list (O(1) reordering).

🌍 Real-World Usage:
  • Browser cache: Recently visited pages are kept; old ones evicted when memory fills
  • CPU L1/L2 cache: Recently used memory lines stay; cold lines are evicted
  • ASP.NET MemoryCache: Sliding expiry is an LRU variant
  • Database buffer pool: Recently read pages stay in RAM; cold pages are flushed
LRU CACHE DESIGN — capacity=3 Data structure: - Dictionary: key → node (O(1) lookup) - Doubly Linked List: MRU ←→ ... ←→ LRU - head (dummy) ↔ [most recent] ↔ ... ↔ [least recent] ↔ tail (dummy) - When accessed → move node to front (most recent) - When full and inserting → remove tail (least recently used) Operations trace (capacity=3): put(1,1): List: [1] Map: {1→node1} put(2,2): List: [2,1] Map: {1,2} put(3,3): List: [3,2,1] Map: {1,2,3} get(1): List: [1,3,2] Map: {1,2,3} → move 1 to front, return 1 put(4,4): FULL! evict tail (2) List: [4,1,3] Map: {1,3,4} → 2 evicted get(2): returns -1 (evicted!) get(3): List: [3,4,1] returns 3 get(4): List: [4,3,1] returns 4
public class LRUCache
{
    private class Node
    {
        public int Key, Val;
        public Node? Prev, Next;
        public Node(int k, int v) { Key = k; Val = v; }
    }

    private readonly int _cap;
    private readonly Dictionary<int, Node> _map;
    private readonly Node _head, _tail; // dummy sentinels

    public LRUCache(int capacity)
    {
        _cap  = capacity;
        _map  = new(capacity);
        _head = new(- 1, -1); _tail = new(-1, -1);
        _head.Next = _tail; _tail.Prev = _head;
    }

    public int Get(int key)
    {
        if (!_map.TryGetValue(key, out var node)) return -1;
        MoveToFront(node);
        return node.Val;
    }

    public void Put(int key, int value)
    {
        if (_map.TryGetValue(key, out var node))
        {
            node.Val = value;
            MoveToFront(node);
            return;
        }
        if (_map.Count == _cap)
        {
            var lru = _tail.Prev!;
            Unlink(lru);
            _map.Remove(lru.Key);
        }
        var newNode = new Node(key, value);
        InsertFront(newNode);
        _map[key] = newNode;
    }

    private void MoveToFront(Node n) { Unlink(n); InsertFront(n); }

    private void Unlink(Node n)
    {
        n.Prev!.Next = n.Next;
        n.Next!.Prev = n.Prev;
    }

    private void InsertFront(Node n)
    {
        n.Next = _head.Next;
        n.Prev = _head;
        _head.Next!.Prev = n;
        _head.Next = n;
    }
}

// Usage:
var cache = new LRUCache(3);
cache.Put(1, 10); cache.Put(2, 20); cache.Put(3, 30);
Console.WriteLine(cache.Get(1));     // 10 — moves 1 to front
cache.Put(4, 40);                    // evicts 2 (LRU)
Console.WriteLine(cache.Get(2));     // -1 (evicted)
// All operations: O(1) time — dictionary for lookup, DLL for order
Section 6 — Rolling Hash & Rabin-Karp String Matching

The Rolling Hash technique lets you compute the hash of a sliding window in O(1) per step (rather than O(k) per step). Combined with a hash map, it enables the Rabin-Karp algorithm for finding all occurrences of a pattern in a text in O(n+m).

ROLLING HASH — Sliding window of size k=3 over "abcde" Naive: hash("abc") → hash("bcd") → hash("cde") Each hash = O(k) time → total O(n·k) — too slow Rolling: remove old character, add new character in O(1) hash("abc") = a*31² + b*31 + c hash("bcd") = hash("abc") - a*31² ... * 31 + d = (hash("abc") - a*p^(k-1)) * 31 + d This slide happens in O(1) — no re-reading old characters! Use case: Find all positions where pattern P occurs in text T - Compute hash(P) - Slide window of size |P| across T - When hash matches: verify exact equality (to handle hash collisions) - O(n + m) average, O(nm) worst (many false hash matches)
// Rabin-Karp — find all start indices where pattern occurs in text
public List<int> RabinKarp(string text, string pattern)
{
    var result = new List<int>();
    int n = text.Length, m = pattern.Length;
    if (m > n) return result;

    const long base_ = 31, mod = 1_000_000_007;

    // Compute base^(m-1) mod p (for removing leftmost character)
    long power = 1;
    for (int i = 0; i < m - 1; i++) power = power * base_ % mod;

    // Compute hash of pattern and first window
    long patHash = 0, winHash = 0;
    for (int i = 0; i < m; i++)
    {
        patHash = (patHash * base_ + pattern[i]) % mod;
        winHash = (winHash * base_ + text[i])    % mod;
    }

    for (int i = 0; i <= n - m; i++)
    {
        if (winHash == patHash && text.Substring(i, m) == pattern)
            result.Add(i);

        // Roll the window: remove text[i], add text[i+m]
        if (i < n - m)
        {
            winHash = (winHash - text[i] * power % mod + mod) % mod;
            winHash = (winHash * base_ + text[i + m]) % mod;
        }
    }
    return result;
}
// RabinKarp("abcababc", "abc") → [0, 5]
// RabinKarp("aaaa", "aa")     → [0, 1, 2]

// Practical application: Find all anagrams in a string (LeetCode 438)
// Use rolling hash on sorted windows OR rolling character frequency array

// Find all anagram start positions of p in s — O(n) with rolling frequency array
public IList<int> FindAnagrams(string s, string p)
{
    var result = new List<int>();
    if (s.Length < p.Length) return result;
    int[] need = new int[26], window = new int[26];
    foreach (char c in p) need[c - 'a']++;
    int k = p.Length;
    for (int i = 0; i < s.Length; i++)
    {
        window[s[i] - 'a']++;
        if (i >= k) window[s[i - k] - 'a']--;   // slide out left
        if (i >= k - 1 && window.SequenceEqual(need)) result.Add(i - k + 1);
    }
    return result;
}
// FindAnagrams("cbaebabacd","abc") → [0, 6]
// FindAnagrams("abab","ab")        → [0, 1, 2]

💡 Interview Heuristic:
Whenever you see an O(n²) brute-force solution involving nested loops or repeated scanning, ask: "Can I use a hash map to remember something and check it in O(1)?"

Common transformations:
  • "Find a pair" → O(n²) → hash complement → O(n)
  • "Count occurrences" → O(n²) → frequency map → O(n)
  • "Find subarray with sum" → O(n²) → prefix sum + map → O(n)
  • "Group similar items" → O(n²) → canonical key + dict → O(n·k)
