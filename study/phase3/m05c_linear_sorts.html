Section 1 — Breaking the O(n log n) Barrier

1.1 Why Comparison-Based Sorts Have a Lower Bound

Merge sort, quick sort — all comparison-based sorts — have a proven lower bound of Ω(n log n). The proof: any comparison-based sort can be modeled as a decision tree. The tree must have n! leaves (all permutations). A binary tree with n! leaves needs at least log₂(n!) ≈ n log n height. So we need at least n log n comparisons.

📝 Can We Do Better?Yes! But only if we use information beyond comparisons. Linear sorts exploit the structure of the data (e.g., values are integers in a range). They can run in O(n+k) where k is the value range. They do NOT compare elements — they use counting or positional placement.

1.2 The Trade-off

Linear sorts trade generality for speed. They work on specific data types (integers, strings with bounded alphabet), require extra memory, and need a known value range. Use them when these conditions are met and performance matters.

Section 2 — Counting Sort

2.1 The Analogy — Counting Votes

Imagine you're counting votes for 5 candidates (0–4). Instead of sorting the ballot papers, you just count how many votes each candidate got. Then you reconstruct the sorted order: candidate 0 gets their count of papers, then candidate 1, etc. That's counting sort.

2.2 How It Works — Three Phases

Input: [4, 2, 2, 8, 3, 3, 1] Range: 0-8 (k=9) PHASE 1 — Count occurrences: count[] = [0, 1, 2, 2, 1, 0, 0, 0, 1] ↑ ↑ ↑ ↑ ↑ idx=0 idx=2 3 4 8 PHASE 2 — Cumulative sum (position of each element in output): count[] = [0, 1, 3, 5, 6, 6, 6, 6, 7] (each count[i] = where value i ENDS in output) PHASE 3 — Build output (scan input right to left for stability): Scan [4,2,2,8,3,3,1] backwards: 1 → output[count[1]-1=0] = 1, count[1]-- 3 → output[count[3]-1=4] = 3, count[3]-- 3 → output[count[3]-1=3] = 3, count[3]-- 8 → output[count[8]-1=6] = 8, count[8]-- 2 → output[count[2]-1=2] = 2, count[2]-- 2 → output[count[2]-1=1] = 2, count[2]-- 4 → output[count[4]-1=5] = 4, count[4]-- Output: [1, 2, 2, 3, 3, 4, 8] ✓ sorted!
using System;

// ─── Counting Sort — Complete Implementation ──────────────────────────────────
// Time:  O(n + k) where k = range of values (max - min + 1)
// Space: O(n + k) — count array + output array
// Stable: Yes (scanning right to left in phase 3)
// Constraint: integer values only, k must be reasonably small

class CountingSort
{
    /// Sorts array of non-negative integers with values in [0, maxVal]
    static int[] Sort(int[] arr)
    {
        if (arr.Length == 0) return arr;

        // Find range
        int max = int.MinValue, min = int.MaxValue;
        foreach (int x in arr) { if(x>max)max=x; if(x<min)min=x; }

        int k = max - min + 1;
        int[] count  = new int[k];
        int[] output = new int[arr.Length];

        // Phase 1: Count occurrences (offset by min to handle negative values)
        foreach (int x in arr) count[x - min]++;

        // Phase 2: Cumulative sum
        for (int i = 1; i < k; i++) count[i] += count[i - 1];

        // Phase 3: Build output (right to left for stability)
        for (int i = arr.Length - 1; i >= 0; i--)
        {
            int idx = arr[i] - min;
            output[count[idx] - 1] = arr[i];
            count[idx]--;
        }

        return output;
    }

    static void Main()
    {
        int[] arr1 = { 4, 2, 2, 8, 3, 3, 1 };
        int[] sorted = Sort(arr1);
        Console.WriteLine(string.Join(", ", sorted)); // 1, 2, 2, 3, 3, 4, 8

        // Works with negative numbers too (offset by min)
        int[] arr2 = { -3, 1, -5, 0, 4, -2 };
        Console.WriteLine(string.Join(", ", Sort(arr2))); // -5, -3, -2, 0, 1, 4

        // Sorting characters (a=97...z=122)
        char[] chars = "banana".ToCharArray();
        int[] charNums = Array.ConvertAll(chars, c => (int)c);
        int[] sortedNums = Sort(charNums);
        Console.WriteLine(new string(Array.ConvertAll(sortedNums, n=>(char)n))); // aaabnn
    }
}
from typing import List

def counting_sort(arr: List[int]) -> List[int]:
    """Counting sort for integers.
    Time: O(n + k) | Space: O(n + k) | Stable: Yes
    k = range of values (max - min + 1)
    """
    if not arr: return []

    lo, hi = min(arr), max(arr)
    k = hi - lo + 1
    count = [0] * k

    # Phase 1: count occurrences
    for x in arr:
        count[x - lo] += 1

    # Phase 2: cumulative sum
    for i in range(1, k):
        count[i] += count[i - 1]

    # Phase 3: build output right to left
    output = [0] * len(arr)
    for x in reversed(arr):
        idx = x - lo
        output[count[idx] - 1] = x
        count[idx] -= 1

    return output


if __name__ == "__main__":
    print(counting_sort([4,2,2,8,3,3,1]))  # [1,2,2,3,3,4,8]
    print(counting_sort([-3,1,-5,0,4,-2])) # [-5,-3,-2,0,1,4]
    chars = [ord(c) for c in "banana"]
    print("".join(chr(c) for c in counting_sort(chars)))  # aaabnn
⚠️ When NOT to Use Counting SortIf k (the value range) is much larger than n (the input size), counting sort wastes huge amounts of memory. Example: n=10 items but values range from 0 to 1,000,000 → creates 1M count array entries. Rule of thumb: only use when k is O(n).

Section 3 — Radix Sort

3.1 The Analogy — Sorting Punched Cards

Old IBM computers sorted punched cards by running them through a machine that sorted by one column at a time, starting from the rightmost column. After sorting all columns, the cards were in order. That's radix sort — sort by least significant digit first, then next, and so on.

3.2 How It Works

Sort the numbers digit by digit from least significant to most significant, using a stable sort (counting sort) at each digit position. "Stable" is crucial — it preserves the order established by previous digits.

Input: [170, 45, 75, 90, 802, 24, 2, 66] Pass 1: Sort by ones digit (LSD): Groups: 0→[170,90], 2→[802,2], 4→[24], 5→[45,75], 6→[66] Result: [170, 90, 802, 2, 24, 45, 75, 66] Pass 2: Sort by tens digit: Groups: 0→[802,2], 2→[24], 4→[45], 6→[66], 7→[170,75], 9→[90] Result: [802, 2, 24, 45, 66, 170, 75, 90] Pass 3: Sort by hundreds digit: Groups: 0→[2,24,45,66,75,90], 1→[170], 8→[802] Result: [2, 24, 45, 66, 75, 90, 170, 802] ✓ Sorted!
using System;

// ─── Radix Sort — Complete Implementation ─────────────────────────────────────
// Time:  O(d * (n + b)) where d=digit count, b=base (usually 10)
//        ≈ O(n) when d is constant (most integers have ≤10 digits)
// Space: O(n + b)
// Stable: Yes (uses stable counting sort on each digit)

class RadixSort
{
    /// Stable counting sort by a specific digit position (exp = 1, 10, 100, ...)
    static void CountSortByDigit(int[] arr, int exp)
    {
        int n = arr.Length;
        int[] output = new int[n];
        int[] count  = new int[10];  // digits 0-9

        // Count occurrences of each digit
        foreach (int x in arr) count[(x / exp) % 10]++;

        // Cumulative sum
        for (int i = 1; i < 10; i++) count[i] += count[i-1];

        // Build output right to left (stable)
        for (int i = n - 1; i >= 0; i--)
        {
            int digit = (arr[i] / exp) % 10;
            output[--count[digit]] = arr[i];
        }

        // Copy back to original array
        for (int i = 0; i < n; i++) arr[i] = output[i];
    }

    /// Main radix sort — processes each digit from LSD to MSD
    static void Sort(int[] arr)
    {
        int max = arr[0];
        foreach (int x in arr) if (x > max) max = x;

        // Process each digit position: 1s, 10s, 100s, ...
        for (int exp = 1; max / exp > 0; exp *= 10)
            CountSortByDigit(arr, exp);
    }

    static void Main()
    {
        int[] arr = { 170, 45, 75, 90, 802, 24, 2, 66 };
        Console.WriteLine("Before: " + string.Join(", ", arr));
        Sort(arr);
        Console.WriteLine("After:  " + string.Join(", ", arr));
        // After: 2, 24, 45, 66, 75, 90, 170, 802

        // Sort strings lexicographically using radix on character codes
        int[] phoneCodes = { 5551234, 5559999, 5550001, 5554567 };
        Sort(phoneCodes);
        Console.WriteLine(string.Join(", ", phoneCodes));
        // 5550001, 5551234, 5554567, 5559999
    }
}
from typing import List

def counting_sort_by_digit(arr: List[int], exp: int) -> None:
    """Stable counting sort on a single digit position."""
    n = len(arr)
    output = [0] * n
    count  = [0] * 10

    for x in arr: count[(x // exp) % 10] += 1
    for i in range(1, 10): count[i] += count[i-1]

    for x in reversed(arr):
        d = (x // exp) % 10
        count[d] -= 1
        output[count[d]] = x

    for i, v in enumerate(output): arr[i] = v


def radix_sort(arr: List[int]) -> List[int]:
    """LSD radix sort for non-negative integers.
    Time: O(d*(n+10)) ≈ O(n) | Space: O(n)
    """
    if not arr: return arr
    max_val = max(arr)
    exp = 1
    while max_val // exp > 0:
        counting_sort_by_digit(arr, exp)
        exp *= 10
    return arr


if __name__ == "__main__":
    arr = [170, 45, 75, 90, 802, 24, 2, 66]
    print(radix_sort(arr))  # [2, 24, 45, 66, 75, 90, 170, 802]

Section 4 — Bucket Sort

4.1 The Analogy — Sorting Names into Letter Bins

Imagine sorting 100 names alphabetically. You place each name into a bin based on its first letter (A bin, B bin, ... Z bin). Then sort each small bin individually. This is bucket sort — distribute items into buckets, sort each bucket, concatenate.

4.2 How It Works

Bucket sort works best when the input is uniformly distributed over a range. Create n buckets covering the range, scatter elements into buckets based on value, sort each bucket, then concatenate.

Input (floats in [0,1)): [0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12] n = 8 buckets spanning [0,1): Scatter into buckets (bucket i covers [i/n, (i+1)/n)): Bucket 0 [0.0, 0.125): [0.12] Bucket 1 [0.125, 0.25): [0.17, 0.21] Bucket 2 [0.25, 0.375): [0.26] Bucket 3 [0.375, 0.5): [0.39] Bucket 6 [0.75, 0.875): [0.78, 0.72] Bucket 7 [0.875, 1.0): [0.94] Sort each bucket (insertion sort — small buckets): Bucket 1: [0.17, 0.21] Bucket 6: [0.72, 0.78] Concatenate: [0.12, 0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94] ✓
using System;
using System.Collections.Generic;

// ─── Bucket Sort — Complete Implementation ────────────────────────────────────
// Time:  O(n + k) average with uniform distribution, O(n²) worst case
// Space: O(n + k)
// Best for: floating-point numbers uniformly distributed in [0,1)

class BucketSort
{
    /// Sorts array of floats in [0.0, 1.0)
    static void Sort(double[] arr)
    {
        int n = arr.Length;
        if (n <= 1) return;

        // Create n empty buckets
        List<double>[] buckets = new List<double>[n];
        for (int i = 0; i < n; i++) buckets[i] = new List<double>();

        // Scatter elements into buckets
        foreach (double x in arr)
        {
            int bucketIdx = (int)(x * n);
            if (bucketIdx == n) bucketIdx = n - 1;  // edge case: x == 1.0
            buckets[bucketIdx].Add(x);
        }

        // Sort each bucket and concatenate
        int idx = 0;
        foreach (var bucket in buckets)
        {
            bucket.Sort();  // uses insertion sort internally for small lists
            foreach (double x in bucket) arr[idx++] = x;
        }
    }

    /// Bucket sort for arbitrary range [min, max]
    static void SortRange(double[] arr)
    {
        if (arr.Length <= 1) return;
        double min=arr[0], max=arr[0];
        foreach(var x in arr){if(x<min)min=x;if(x>max)max=x;}

        int n = arr.Length;
        double range = max - min + 1e-9;  // small epsilon to avoid edge case
        List<double>[] buckets = new List<double>[n];
        for(int i=0;i<n;i++) buckets[i] = new List<double>();

        foreach(var x in arr) {
            int bi = (int)((x - min) / range * n);
            if(bi==n) bi=n-1;
            buckets[bi].Add(x);
        }

        int idx=0;
        foreach(var b in buckets){ b.Sort(); foreach(var x in b) arr[idx++]=x; }
    }

    static void Main()
    {
        double[] arr = { 0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12 };
        Sort(arr);
        Console.WriteLine(string.Join(", ", arr));
        // 0.12, 0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94
    }
}
from typing import List

def bucket_sort(arr: List[float]) -> List[float]:
    """Bucket sort for floats (handles any range automatically).
    Time: O(n) average with uniform distribution | Space: O(n)
    """
    if len(arr) <= 1: return arr

    lo, hi = min(arr), max(arr)
    n = len(arr)
    span = (hi - lo) + 1e-9   # tiny epsilon to prevent index out of range

    # Create n empty buckets
    buckets: List[List[float]] = [[] for _ in range(n)]

    # Scatter
    for x in arr:
        bi = int((x - lo) / span * n)
        buckets[min(bi, n-1)].append(x)

    # Sort each bucket and gather
    result = []
    for b in buckets:
        b.sort()   # Python list.sort() uses TimSort
        result.extend(b)

    return result


if __name__ == "__main__":
    arr = [0.78,0.17,0.39,0.26,0.72,0.94,0.21,0.12]
    print(bucket_sort(arr))
    # [0.12, 0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94]

5.1 Decision Guide

Your data looks like...Use this sort
Integers in small range [0..k] where k ≈ nCounting Sort
Large integers (phone numbers, zip codes, IDs)Radix Sort
Floats uniformly in [0,1) or any rangeBucket Sort
Arbitrary objects, need stable sortMerge Sort
General purpose, in-memory, fastQuick Sort (random pivot)
Small arrays (< 20 elements)Insertion Sort
🌍 Real-World Applications Counting sort: Sorting exam scores (0-100), histogram computation in image processing. Radix sort: IP address sorting in network routers, sorting phone numbers in databases, suffix array construction. Bucket sort: External sorting when data is known to be uniformly distributed, sorting floating-point sensor readings.

Section 6 — Practice Questions

❓ Q1: Can counting sort handle negative numbers? If so, how?
✅ Yes — use an offset of min value

Shift all values by subtracting the minimum. If values range from -5 to 10, subtract -5 (add 5) to shift range to [0..15]. After sorting, the positions are correct — you never need to shift back since we're reconstructing the original values.

❓ Q2: Why must the inner sort in radix sort be STABLE?
✅ Stability preserves previous digit orderings

Example: After sorting by ones digit, we have [20, 50, 30] (all end in 0, in their original relative order). When we sort by tens digit, 20, 50, and 30 all have different tens digits, so they sort correctly. But if two numbers have the SAME tens digit (e.g., 23 and 43 → both have 3 in ones digit), a stable sort ensures the one sorted correctly by the ones digit comes first. Using an unstable sort would scramble the work done by previous passes.

❓ Q3: What happens to bucket sort's performance when all elements fall into the same bucket?
✅ Degrades to O(n²) — the worst case

If all elements are equal or cluster tightly, they all go into one bucket. The inner sort (often insertion sort) then runs on all n elements → O(n²). This is why bucket sort works best only with uniformly distributed data. For skewed distributions, use a different algorithm.
