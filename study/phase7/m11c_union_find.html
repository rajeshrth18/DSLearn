Section 1 — The Problem: Dynamic Connectivity

The Disjoint Set Union (DSU), also called Union-Find, solves the dynamic connectivity problem: you have n nodes and you want to:

  1. Union(a, b): Connect node a and node b (merge their sets)
  2. Find(a): Return the "representative" (root) of a's set
  3. Connected(a, b): Return Find(a) == Find(b)
?? Real-World Applications:
  • Kruskal's MST: Use DSU to check if adding an edge creates a cycle — O(E log E) overall
  • Network connectivity: "Are these two servers in the same connected component?"
  • Percolation: Physics simulations of fluid flow through grids
  • Image processing: Connected component labeling — group adjacent pixels of same color
  • Social networks: "Are these two users in the same friend group?"

Where a(n) is the inverse Ackermann function — for all practical values of n (up to the number of atoms in the universe), a(n) = 4. So it's effectively constant time.

Section 2 — Naive DSU: parent[] Array

Each node points to a parent. A node is a root if parent[i] == i. All nodes in the same set eventually point (via chain) to the same root.

INITIAL STATE: 5 nodes, all separate parent: [0, 1, 2, 3, 4] (each is its own root) Union(0, 2): parent[Find(0)] = Find(2) ? parent[0] = 2 parent: [2, 1, 2, 3, 4] component: {0,2} Union(1, 3): parent[1] = 3 parent: [2, 3, 2, 3, 4] components: {0,2}, {1,3}, {4} Union(2, 3): parent[Find(2)] = Find(3) ? parent[2] = 3 parent: [2, 3, 3, 3, 4] components: {0,1,2,3}, {4} Find(0): 0?2?3 (chain of length 2)
C# 
// Naive DSU — no optimizations
int[] parent;

void Init(int n) {
    parent = new int[n];
    for (int i=0; i<n; i++) parent[i] = i;
}

int Find(int x) {
    while (parent[x] != x) x = parent[x];   // follow chain to root
    return x;
}

void Union(int a, int b) => parent[Find(a)] = Find(b);

bool Connected(int a, int b) => Find(a) == Find(b);

    return x

def union(a, b): parent[find(a)] = find(b)

Section 3 — Path Compression

During a Find(x) call, after traversing the chain to the root, set every visited node's parent directly to the root. Future Find calls on these nodes will be O(1).

PATH COMPRESSION EXAMPLE Before (chain): 0 ? 2 ? 5 ? 9 (root) Find(0): Step 1: Follow chain: 0?2?5?9 (root = 9) Step 2: Compress: parent[0] = 9 parent[2] = 9 parent[5] = 9 (already 9's child, no change) After: 0 ? 9, 2 ? 9, 5 ? 9 (all point directly to root!)
C# 
// Path compression — recursive (one-pass halving)
int Find(int x) {
    if (parent[x] != x) parent[x] = Find(parent[x]);   // recursion + compress
    return parent[x];
}

// Path compression — iterative (two-pass)
int FindIter(int x) {
    int root = x;
    while (parent[root] != root) root = parent[root];  // find root
    while (parent[x] != root) {                         // compress path
        int next = parent[x]; parent[x] = root; x = next;
    }
    return root;
}
Python 
# Path compression — recursive (one-pass)
def find(x):
    if parent[x] != x: parent[x] = find(parent[x])   # recursion + compress
    return parent[x]

# Path compression — iterative (two-pass)
def find_iter(x):
    root = x
    while parent[root] != root: root = parent[root]   # find root
    while parent[x] != root:                           # compress path
        parent[x], x = root, parent[x]
    return root
Section 4 — Union by Rank / Size

When merging two trees, always attach the smaller tree under the root of the larger tree. This keeps the tree height at most O(log n).

UNION BY RANK (tree height bound) rank[i] = upper bound on height of tree rooted at i Union(a, b): ra = Find(a), rb = Find(b) if ra == rb: return (already connected) if rank[ra] < rank[rb]: swap ra, rb (always attach rb under ra) parent[rb] = ra if rank[ra] == rank[rb]: rank[ra]++ (only grows when equal heights merge) UNION BY SIZE (count of elements): size[i] = number of nodes in tree rooted at i Union(a, b): ra = Find(a), rb = Find(b) if ra == rb: return if size[ra] < size[rb]: swap ra, rb parent[rb] = ra size[ra] += size[rb] (update size of new root)
?? Rank vs Size: Both achieve O(log n) height bound. "Union by Size" is often preferred in practice because the size field is useful (you often want to know the component size). "Union by Rank" has a slightly tighter theoretical bound.
Section 5 — The Inverse-Ackermann Complexity

With both path compression AND union by rank/size, the amortized cost per operation is O(a(n)), the inverse Ackermann function.

ACKERMANN FUNCTION grows incredibly fast: A(1,1) = 2 A(2,2) = 7 A(3,3) = 61 A(4,4) = 2^2^2^2^... (a power tower of 65536 twos) ˜ 8 for practical purposes INVERSE ACKERMANN a(n) grows incredibly slowly: a(n) = 1 for n = 1 a(n) = 2 for n = 3 a(n) = 3 for n = 7 a(n) = 4 for n = A(4,4) ˜ 10^19728 So for ALL practical n (even 10^18): a(n) = 4 This means: with both optimizations, DSU is effectively O(1) per operation for any real-world input. The proof requires amortized analysis — tracking a "potential function" that accounts for the cost of future path compressions amortized over past operations.
Section 6 — Full C# & Python Implementations
C# 
public class DSU
{
    private readonly int[] parent, size;
    public int Components { get; private set; }

    public DSU(int n)
    {
        parent = Enumerable.Range(0, n).ToArray();
        size   = Enumerable.Repeat(1, n).ToArray();
        Components = n;
    }

    /// Find root with path compression — O(a(n))
    public int Find(int x)
    {
        if (parent[x] != x) parent[x] = Find(parent[x]);
        return parent[x];
    }

    /// Union by size — O(a(n)). Returns false if already connected.
    public bool Union(int a, int b)
    {
        int ra = Find(a), rb = Find(b);
        if (ra == rb) return false;
        if (size[ra] < size[rb]) (ra, rb) = (rb, ra);   // ra = larger
        parent[rb] = ra;
        size[ra]  += size[rb];
        Components--;
        return true;
    }

    /// Are a and b in the same component?
    public bool Connected(int a, int b) => Find(a) == Find(b);

    /// Size of the component containing x
    public int GetSize(int x) => size[Find(x)];
}

// Usage
var dsu = new DSU(5);
dsu.Union(0, 1);
dsu.Union(1, 2);
Console.WriteLine(dsu.Connected(0, 2));    // True
Console.WriteLine(dsu.Connected(0, 3));    // False
Console.WriteLine(dsu.Components);          // 3  ({0,1,2}, {3}, {4})
Console.WriteLine(dsu.GetSize(0));         // 3
Python 
class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n
        self.components = n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # path compression
        return self.parent[x]

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb: return False
        if self.size[ra] < self.size[rb]: ra, rb = rb, ra  # ra = larger
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]
        self.components -= 1
        return True

    def connected(self, a, b): return self.find(a) == self.find(b)
    def get_size(self, x): return self.size[self.find(x)]
Section 7 — Weighted DSU

A Weighted DSU stores a relative weight from each node to its parent. During path compression, the weight is updated to reflect the weight relative to the root directly.

USE CASE: "Is there a consistent assignment of values?" Each edge (a,b) in a graph has a label: val[b] - val[a] = w We want to know if the system of equations is consistent (no contradiction). weight[i] = w(i, root) — value difference from node i to its component root Find(x): If x is root: return (x, 0) (root, w) = Find(parent[x]) weight[x] += w ? update weight relative to root (during path compression) parent[x] = root return (root, weight[x]) Union(a, b, w_ab): ? w_ab = val[b] - val[a] (ra, wa) = Find(a), (rb, wb) = Find(b) If ra == rb: check wa - wb == w_ab (consistency) parent[ra] = rb weight[ra] = wb - wa + w_ab ? derived from the equation system
C# 
public class WeightedDSU
{
    private int[] parent, size;
    private double[] weight;  // weight[i] = val[i] / val[root]  (for ratio problems)

    public WeightedDSU(int n) {
        parent = Enumerable.Range(0,n).ToArray();
        size   = new int[n]; Array.Fill(size, 1);
        weight = new double[n]; Array.Fill(weight, 1.0);
    }

    public (int root, double w) Find(int x) {
        if (parent[x] == x) return (x, 1.0);
        var (root, pw) = Find(parent[x]);
        weight[x] *= pw;
        parent[x] = root;
        return (root, weight[x]);
    }

    public bool Union(int a, int b, double ratio)  // val[a] / val[b] = ratio
    {
        var (ra, wa) = Find(a);
        var (rb, wb) = Find(b);
        if (ra == rb) return true;
        if (size[ra] < size[rb]) { (ra,rb)=(rb,ra); (wa,wb)=(wb,wa); ratio=1.0/ratio; }
        parent[rb] = ra;
        weight[rb] = wa / wb / ratio;
        size[ra] += size[rb];
        return true;
    }
}

Python 
class WeightedDSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size   = [1] * n
        self.weight = [1.0] * n  # weight[i] = val[i] / val[root] (ratio problems)

    def find(self, x):
        if self.parent[x] == x: return (x, 1.0)
        root, pw = self.find(self.parent[x])
        self.weight[x] *= pw
        self.parent[x]  = root
        return (root, self.weight[x])

    def union(self, a, b, ratio):   # val[a] / val[b] = ratio
        ra, wa = self.find(a)
        rb, wb = self.find(b)
        if ra == rb: return True
        if self.size[ra] < self.size[rb]:
            ra, rb, wa, wb, ratio = rb, ra, wb, wa, 1.0 / ratio
        self.parent[rb] = ra
        self.weight[rb] = wa / wb / ratio
        self.size[ra]  += self.size[rb]
        return True

LeetCode #684 — Redundant Connection (Medium)

Given a tree with one extra edge, find the redundant edge (the one that forms a cycle).

C# 
public int[] FindRedundantConnection(int[][] edges)
{
    var dsu = new DSU(edges.Length + 1);
    foreach (var e in edges)
        if (!dsu.Union(e[0], e[1]))   // Union returns false if already connected
            return e;               // this edge creates a cycle
    return null;
}
// Time: O(n a(n)) ˜ O(n). Classic DSU problem!
Python 
def find_redundant_connection(edges: list) -> list:
    dsu = DSU(len(edges) + 1)
    for e in edges:
        if not dsu.union(e[0], e[1]):  # union returns False if already connected
            return e                    # this edge creates a cycle
    return []

LeetCode #547 — Number of Provinces (Medium)

Given an adjacency matrix, count the number of connected components.

C# 
public int FindCircleNum(int[][] isConnected)
{
    int n = isConnected.Length;
    var dsu = new DSU(n);
    for (int i=0; i<n; i++)
        for (int j=i+1; j<n; j++)
            if (isConnected[i][j] == 1)
                dsu.Union(i, j);
    return dsu.Components;   // DSU tracks component count automatically!
}
Python 
def find_circle_num(is_connected: list) -> int:
    n   = len(is_connected)
    dsu = DSU(n)
    for i in range(n):
        for j in range(i + 1, n):
            if is_connected[i][j] == 1:
                dsu.union(i, j)
    return dsu.components   # DSU tracks component count automatically!

LeetCode #990 — Satisfiability of Equality Equations (Medium)

Given equations like "a==b" and "a!=b", check if all can be satisfied simultaneously.

C# 
public bool EquationsPossible(string[] equations)
{
    var dsu = new DSU(26);
    // Pass 1: process all == equations
    foreach (var eq in equations)
        if (eq[1] == '=')
            dsu.Union(eq[0]-'a', eq[3]-'a');
    // Pass 2: check all != equations for contradictions
    foreach (var eq in equations)
        if (eq[1] == '!' && dsu.Connected(eq[0]-'a', eq[3]-'a'))
            return false;
    return true;
}
// Key insight: process == first to build groups, then check != against groups.
// Two-pass approach is the DSU pattern for contradiction detection.
Python 
def equations_possible(equations: list) -> bool:
    dsu = DSU(26)
    # Pass 1: process all == equations
    for eq in equations:
        if eq[1] == '=':
            dsu.union(ord(eq[0]) - ord('a'), ord(eq[3]) - ord('a'))
    # Pass 2: check all != equations for contradictions
    for eq in equations:
        if (eq[1] == '!' and
                dsu.connected(ord(eq[0]) - ord('a'), ord(eq[3]) - ord('a'))):
            return False
    return True

Section 9 — Q&A Review
Q1: What are the two essential optimizations for DSU, and what complexity does each one alone give?
Path Compression: During Find(x), set every visited node's parent directly to the root. Alone: amortized O(log n) per operation. Union by Rank/Size: When merging, always attach the smaller tree under the larger. Alone: O(log n) worst case per operation (bounded tree height). Together: O(a(n)) amortized — inverse Ackermann, effectively constant. The synergy is crucial: neither alone achieves the O(a(n)) bound, but together they do (proved by Tarjan 1975).
Q2: DSU supports Union and Find. How would you add a "split" (undo) operation?
Standard DSU with path compression is irreversible — once you flatten paths, you can't undo it easily. The solution is "Rollback DSU" or "DSU with Undo": don't use path compression (use only union by rank to keep O(log n)), and maintain a stack of (node, old_parent, old_rank) entries for each Union. To undo, pop from the stack and restore the old values. This is used in "offline" divide-and-conquer algorithms where you need to process queries with temporal union/split patterns. The complexity becomes O(log n) per operation instead of O(a(n)).
Q3: In Kruskal's MST algorithm, where exactly does DSU get used?
Kruskal's algorithm: (1) Sort all edges by weight O(E log E). (2) Iterate edges from smallest to largest. (3) For each edge (u, v): if Find(u) != Find(v) — they're in different components — add this edge to MST and Union(u, v). Otherwise, skip it (adding it would create a cycle). DSU makes step 3 O(a(n)) per edge, giving total O(E log E + E a(V)) ˜ O(E log E) for the entire algorithm. Without DSU, detecting cycles would require DFS/BFS: O(V) per edge, giving O(EV) total.
Q4: Can you use DSU for directed graphs? What about for bipartite checking?
Standard DSU is for undirected connectivity — it's symmetric (Union(a,b) = Union(b,a)). For directed graphs, you'd need Kosaraju's or Tarjan's SCC algorithm instead. For bipartite checking: yes! Use a "colored DSU" where each node has a color bit. For each edge (u,v), u and v must be in different colors. You can use 2n nodes: node i and node i+n represent "node i in color 0" and "node i in color 1". Union(u, v+n) and Union(u+n, v). If Find(u) == Find(v), the graph is not bipartite.
Q5: What is the "virtual node" technique in DSU and when is it useful?
A "virtual node" is an extra DSU node that doesn't correspond to any input element, used as a common connector. Example use case: "Find the number of components, plus detect if any component connects to the boundary." Add a virtual node "boundary" and union it with every boundary cell. Now "Is cell X connected to the boundary?" = Connected(X, boundary_node). Another example: "All isolated nodes should be connected to a virtual sink." This avoids special-casing and lets you express complex connectivity conditions as simple DSU operations. LeetCode #130 (Surrounded Regions) uses this: union border O's to a virtual "safe" node, then everything NOT connected to "safe" and still 'O' gets flipped.
