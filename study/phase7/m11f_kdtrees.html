Section 1 — The Spatial Query Problem

Given a set of points in k-dimensional space, answer queries like:

  • Nearest Neighbor (NN): Find the point closest to a query point
  • Range Search: Find all points within a rectangular (or spherical) region
  • K-Nearest Neighbors (KNN): Find the k closest points

Section 2 — K-D Tree Structure

A K-D Tree is a binary tree where each internal node splits the space along one axis. At depth d, the split is along axis d mod k. Each node stores one point.

2D K-D TREE EXAMPLE — Points: (7,2),(5,4),(9,6),(4,7),(8,1),(2,3),(1,5) Step 1: Sort by x, pick median ? (7,2) is root. Split: x=7. Left subtree: all points with x < 7: {(5,4),(4,7),(2,3),(1,5)} Right subtree: all points with x = 7: {(9,6),(8,1)} Step 2 (depth 1, split by y): Left: sort by y ? (2,3) is median. Split: y=3. Left: {(1,5)}... Wait, y<3: {(5,4)? No, 4>3} Actually: {(5,4),(4,7),(2,3),(1,5)} sorted by y: (2,3),(5,4),(1,5),(4,7) Median = (5,4) or (1,5). Choose (5,4). Left: y < 4: {(2,3),(1,5)... wait 5>4}. y<4: {(2,3)} Right: y = 4: {(1,5),(4,7)} TREE STRUCTURE: (7,2) x-split / \ (5,4) y-split (9,6) y-split / \ / (2,3) (4,7) (8,1) \ (1,5) VISUAL SPACE PARTITION: Y 8 | (4,7) 7 | 6 | (9,6) 5 | (1,5) 4 | (5,4) 3 | (2,3) 2 | (7,2) 1 | (8,1) +--2--4--6--8--10 X Root x-split at x=7: LEFT=blue region, RIGHT=red region (5,4) y-split at y=4: divides left region (9,6) y-split at y=6: divides right region
Section 3 — Build: O(n log n)

At each level, choose the median point along the current axis. Using nth_element (partial sort) this is O(n) per level × O(log n) levels = O(n log n) total.

C# 
public class KDNode
{
    public double[] point;
    public KDNode left, right;
    public KDNode(double[] p) => point = p;
}

public class KDTree
{
    private readonly int k;
    private KDNode root;

    public KDTree(double[][] points)
    {
        k = points[0].Length;
        root = Build(points.ToList(), 0);
    }

    private KDNode Build(List<double[]> pts, int depth)
    {
        if (pts.Count == 0) return null;
        int axis = depth % k;
        pts.Sort((a, b) => a[axis].CompareTo(b[axis]));
        int mid = pts.Count / 2;
        return new KDNode(pts[mid])
        {
            left  = Build(pts.GetRange(0, mid), depth + 1),
            right = Build(pts.GetRange(mid + 1, pts.Count - mid - 1), depth + 1)
        };
    }
}
Python 
class KDNode:
    def __init__(self, point): self.point = point; self.left = self.right = None

class KDTree:
    def __init__(self, points):
        self.k    = len(points[0])
        self.root = self._build(points[:], 0)

    def _build(self, pts, depth):
        if not pts: return None
        axis = depth % self.k
        pts.sort(key=lambda p: p[axis])
        mid  = len(pts) // 2
        node = KDNode(pts[mid])
        node.left  = self._build(pts[:mid],    depth + 1)
        node.right = self._build(pts[mid+1:], depth + 1)
        return node
Section 4 — Nearest Neighbor Search

The NN search uses branch-and-bound pruning: when the closest possible point in a subtree (the perpendicular distance to the splitting hyperplane) is farther than the current best, skip that entire subtree.

NEAREST NEIGHBOR of query point (3, 5) Start at root (7,2): Distance = sqrt((7-3)²+(2-5)²) = sqrt(16+9) = sqrt(25) = 5.0 Best so far: 5.0 at (7,2) Query[x=0]=3 < node[x=0]=7 ? go LEFT first (closer side) At (5,4): Distance = sqrt((5-3)²+(4-5)²) = sqrt(4+1) = sqrt(5) ˜ 2.24 Best: 2.24 at (5,4) Query[y=1]=5 = node[y=1]=4 ? go RIGHT first At (4,7): Distance = sqrt(1+4) = sqrt(5) ˜ 2.24 tie, keep (5,4) as best Back at (5,4), try LEFT subtree: (2,3) |query[y] - split[y]| = |5-4| = 1 < best=2.24 ? MUST explore left At (2,3): Distance = sqrt(1+4) = sqrt(5) ˜ 2.24 still tie, keep (5,4) At (1,5): Distance = sqrt(4+0) = 2.0 Best: 2.0 at (1,5)! Back at root (7,2): try RIGHT subtree |query[x] - split[x]| = |3-7| = 4 > best=2.0 ? PRUNE right subtree Result: nearest neighbor is (1,5) at distance 2.0 ?
C# 
public (double[] point, double dist) NearestNeighbor(double[] query)
{
    double[] best = null;
    double bestDist = double.MaxValue;
    NNSearch(root, query, 0, ref best, ref bestDist);
    return (best, Math.Sqrt(bestDist));
}

private void NNSearch(KDNode node, double[] query, int depth,
    ref double[] best, ref double bestDist)
{
    if (node == null) return;
    double d = DistSq(node.point, query);
    if (d < bestDist) { bestDist = d; best = node.point; }

    int axis = depth % k;
    double diff = query[axis] - node.point[axis];
    // Search the closer side first
    KDNode near = diff <= 0 ? node.left  : node.right;
    KDNode far  = diff <= 0 ? node.right : node.left;
    NNSearch(near, query, depth + 1, ref best, ref bestDist);
    // Only search far side if it could contain a closer point
    if (diff * diff < bestDist)
        NNSearch(far, query, depth + 1, ref best, ref bestDist);
}

private double DistSq(double[] a, double[] b)
{
    double s = 0;
    for (int i = 0; i < a.Length; i++) s += (a[i]-b[i]) * (a[i]-b[i]);
    return s;
}
Python 
def dist_sq(a, b): return sum((x-y)**2 for x, y in zip(a, b))

def nn_search(node, query, depth, k, state):
    if node is None: return
    d = dist_sq(node.point, query)
    if d < state['best_dist']:
        state['best_dist'] = d; state['best'] = node.point
    axis = depth % k
    diff = query[axis] - node.point[axis]
    near, far = (node.left, node.right) if diff <= 0 else (node.right, node.left)
    nn_search(near, query, depth+1, k, state)
    if diff**2 < state['best_dist']:   # explore far side only if needed
        nn_search(far, query, depth+1, k, state)

def nearest_neighbor(tree, query):
    state = {'best': None, 'best_dist': float('inf')}
    nn_search(tree.root, query, 0, tree.k, state)
    return state['best'], state['best_dist']**0.5
Section 5 — Range Search

Find all points within a rectangular region [xmin, xmax] × [ymin, ymax]. At each node: if the node's split axis range doesn't overlap the query range, prune. If the point lies within the range, include it.

C# 
public List<double[]> RangeSearch(double[] lo, double[] hi)
{
    var result = new List<double[]>();
    RangeHelper(root, lo, hi, 0, result);
    return result;
}

private void RangeHelper(KDNode node, double[] lo, double[] hi,
    int depth, List<double[]> result)
{
    if (node == null) return;

    // Check if current point is in range
    bool inRange = true;
    for (int i = 0; i < k; i++)
        if (node.point[i] < lo[i] || node.point[i] > hi[i]) { inRange = false; break; }
    if (inRange) result.Add(node.point);

    int axis = depth % k;
    // Prune: if splitting plane is outside query range, skip that side
    if (lo[axis] <= node.point[axis])
        RangeHelper(node.left,  lo, hi, depth + 1, result);
    if (hi[axis] >= node.point[axis])
        RangeHelper(node.right, lo, hi, depth + 1, result);
}
Python 
def range_helper(node, lo, hi, depth, k, result):
    if node is None: return
    # Check if current point is in range
    if all(lo[i] <= node.point[i] <= hi[i] for i in range(k)):
        result.append(node.point)
    axis = depth % k
    if lo[axis] <= node.point[axis]:
        range_helper(node.left,  lo, hi, depth+1, k, result)
    if hi[axis] >= node.point[axis]:
        range_helper(node.right, lo, hi, depth+1, k, result)

def range_search(tree, lo, hi):
    result = []
    range_helper(tree.root, lo, hi, 0, tree.k, result)
    return result

C# 
// Complete 2D K-D Tree
public class KDTree2D
{
    private record Node(int X, int Y, Node Left, Node Right);

    private readonly Node root;

    public KDTree2D(int[][] pts) => root = Build(pts.ToList(), 0);

    private Node Build(List<int[]> pts, int d)
    {
        if (!pts.Any()) return null;
        pts.Sort((a,b) => d==0 ? a[0]-b[0] : a[1]-b[1]);
        int m = pts.Count/2;
        return new Node(pts[m][0], pts[m][1],
            Build(pts[..(m)], 1-d),
            Build(pts[(m+1)..], 1-d));
    }

    /// Find nearest neighbor — returns distance squared
    public int NearestDistSq(int qx, int qy)
    {
        int best = int.MaxValue;
        NN(root, qx, qy, 0, ref best);
        return best;
    }

    private void NN(Node n, int qx, int qy, int d, ref int best)
    {
        if (n == null) return;
        int cur = (qx-n.X)*(qx-n.X) + (qy-n.Y)*(qy-n.Y);
        if (cur < best) best = cur;
        int diff = d==0 ? qx-n.X : qy-n.Y;
        (Node near, Node far) = diff<=0 ? (n.Left,n.Right) : (n.Right,n.Left);
        NN(near, qx, qy, 1-d, ref best);
        if ((long)diff*diff < best) NN(far, qx, qy, 1-d, ref best);
    }
}
Python 
import math

class KDTree:
    def __init__(self, points):
        self.k = len(points[0])
        self.root = self._build(points, 0)

    def _build(self, pts, depth):
        if not pts: return None
        axis = depth % self.k
        pts = sorted(pts, key=lambda p: p[axis])
        mid = len(pts) // 2
        return {
            'point': pts[mid],
            'left':  self._build(pts[:mid], depth+1),
            'right': self._build(pts[mid+1:], depth+1)
        }

    def nearest(self, query):
        self.best = (float('inf'), None)
        self._nn(self.root, query, 0)
        return self.best[1], math.sqrt(self.best[0])

    def _nn(self, node, q, depth):
        if node is None: return
        p = node['point']
        d2 = sum((a-b)**2 for a, b in zip(p, q))
        if d2 < self.best[0]: self.best = (d2, p)
        axis = depth % self.k
        diff = q[axis] - p[axis]
        near, far = (node['left'], node['right']) if diff <= 0 \
                    else (node['right'], node['left'])
        self._nn(near, q, depth+1)
        if diff*diff < self.best[0]: self._nn(far, q, depth+1)

Section 7 — K-D Tree Limitations

The Curse of Dimensionality

K-D Trees work well for low dimensions (k = 10-20), but degrade badly in high dimensions:

WHY HIGH DIMENSIONS BREAK K-D TREES In d dimensions, a hypersphere of radius r has volume ~ r^d. The ratio of hypersphere volume to hypercube volume ? 0 as d grows. For nearest neighbor search: - In 2D: the NN sphere intersects few partition boxes ? prune most - In d=20+: the NN sphere may intersect ALL 2^20 = 1M partition boxes ? no pruning, degrades to O(n) linear scan ROUGH RULE: K-D Trees give speedup over brute force when n >> 2^k k=2: speedup when n >> 4 ? k=10: speedup when n >> 1024 ? (for n=10^6) k=20: speedup when n >> 10^6 marginal k=50: nearly always slower than brute force ALTERNATIVES FOR HIGH DIMENSIONS: - Approximate NN (FAISS, Annoy, HNSW) - Ball Trees (better than KD for curved manifolds) - Random Projection Trees - Locality Sensitive Hashing (LSH)
?? Interview Note: In interviews, K-D Trees come up for 2D/3D spatial problems. For high-dimensional ML tasks (embeddings, vector search), mention that approximate methods like HNSW or FAISS are industry-standard — K-D Trees don't scale.

LeetCode #973 — K Closest Points to Origin (Medium)

Given an array of points, return the k closest to the origin.

?? K-D Tree approach: Build K-D Tree in O(n log n), then query k-nearest-neighbors using a max-heap of size k. However, for this specific problem, the optimal solution is a max-heap of size k in O(n log k) time — simpler and faster. Alternatively, QuickSelect gives O(n) expected time. K-D Tree shines when the same point set is queried many times.
C# 
// Optimal: max-heap of size k — O(n log k)
public int[][] KClosest(int[][] points, int k)
{
    // Max-heap by distance squared
    var pq = new PriorityQueue<int[], int>(
        Comparer<int>.Create((a,b) => b-a));  // max-heap
    foreach (var p in points)
    {
        int d = p[0]*p[0] + p[1]*p[1];
        pq.Enqueue(p, d);
        if (pq.Count > k) pq.Dequeue();
    }
    return pq.UnorderedItems.Select(x => x.Element).ToArray();
}
// K-D Tree is better when n is fixed and you query many different (qx,qy,k) requests.
Python 
import heapq

def k_closest(points: list, k: int) -> list:
    # Max-heap by distance squared — O(n log k)
    max_heap = []
    for p in points:
        d = p[0]*p[0] + p[1]*p[1]
        heapq.heappush(max_heap, (-d, p))  # negate for max-heap
        if len(max_heap) > k:
            heapq.heappop(max_heap)
    return [item[1] for item in max_heap]

Section 9 — Q&A Review
Q1: Why do we choose the median point (not the mean) as the splitting plane in K-D tree construction?
The median ensures a balanced tree — exactly half the points go to each subtree, guaranteeing tree height O(log n). Using the mean would depend on data distribution: if points are skewed, one subtree could get 90% of the points while the other gets 10%, leading to a highly unbalanced tree and O(n) queries in the worst case. The median is the exact center of the sorted list, so it always splits into two halves of size ?n/2? and ?n/2?. Note: finding the exact median takes O(n) via QuickSelect, giving overall O(n log n) build time.
Q2: In the NN search, why must we check the "far side" of a split sometimes? What's the condition?
The nearest neighbor might be on the "far side" of a splitting plane. Example: query = (3,5), split at x=7. The near side is x < 7. Even though most close points are on the near side, a point like (2,4) is only distance ~1.4 away but still on the near side. However, if the current best distance is less than the perpendicular distance to the splitting plane |query[axis] - node[axis]|, then no point on the far side can be closer — prune it. Condition to check far side: diff² < bestDistSq where diff = query[axis] - node[axis]. Without this check, the algorithm degenerates to O(n).
Q3: What is the expected time complexity of K-D tree NN search, and under what conditions is it O(log n)?
For uniformly distributed points in k dimensions, NN search takes O(2^k + log n) expected time. For k=2: O(log n) ˜ O(log n). For k=3: O(8 + log n) = O(log n). For k=20: O(10^6 + log n) ˜ O(n) — the tree is useless. The O(log n) analysis assumes the tree is balanced (median splits) AND the pruning condition eliminates the majority of the tree. In the best case (2D, random data), only O(vn) nodes need to be visited for range queries (the O(vn + k) bound for range search), and O(log n) for NN. These bounds break down when dimensions increase because the hyperplane distance becomes small relative to NN distance, defeating pruning.
Q4: How does a K-D tree generalize to k dimensions? What changes between 2D and kD?
The algorithm is identical — only the axis selection changes. At depth d, split along axis d mod k (cycling through all k dimensions). The splitting condition in search/insert/delete uses point[axis] where axis cycles. The pruning condition in NN search uses diff = query[axis] - node[axis] for the current axis only (perpendicular distance to hyperplane). Distance computation uses all k coordinates. The tree structure, build algorithm, and query algorithms are otherwise unchanged. This is why K-D trees are elegant: the same code handles 2D, 3D, and 100D by just changing the dimension parameter k.
Q5: What is a "ball tree" and how does it compare to a K-D tree?
A ball tree (also called metric tree or BallTree) partitions points into nested hyperspheres ("balls") rather than axis-aligned hyperplanes. Each internal node represents a ball containing all its descendants. For NN search: if the distance from the query to the ball's surface (ball_center_dist - ball_radius) is greater than the current best, prune the entire ball. Advantages over K-D tree: (1) Better performance for high-dimensional or non-uniformly distributed data. (2) Not restricted to axis-aligned splits — handles data on curved manifolds better. (3) In practice, ball trees outperform K-D trees for d > 10. Disadvantages: more complex to build, higher constant factors. Python scikit-learn supports both: KNeighborsClassifier(algorithm='kd_tree') vs algorithm='ball_tree'.
