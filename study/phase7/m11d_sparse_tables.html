Section 1 — The RMQ Problem

The Range Minimum Query (RMQ) problem: given an array arr of n elements and Q queries, each asking for the minimum (or maximum) in a range [L, R].

ApproachBuildQuerySpaceUpdates?
Brute force (iterate)O(1)O(n)O(1)Yes
Segment TreeO(n)O(log n)O(n)Yes
Sparse TableO(n log n)O(1)O(n log n)No
Fischer-Heun (<± 1 arrays)O(n)O(1)O(n)No
?? When to use Sparse Table: The array is static (no updates) and you need many O(1) range minimum/maximum queries. Common in problems with heavy query load and no modifications — e.g., finding the minimum in a sliding window, LCA in trees, or answering millions of RMQ queries on a fixed array.
Section 2 — The Sparse Table Idea

The key insight: any range [L, R] can be covered by two overlapping intervals of a power-of-2 length — and for minimum/maximum queries, overlapping is fine (an element counted twice in a min-query doesn't change the answer).

SPARSE TABLE — Array [3, 1, 4, 1, 5, 9, 2, 6] (0-indexed) st[k][i] = minimum of arr[i .. i + 2^k - 1] (interval of length 2^k starting at i) k=0 (intervals of length 1 = individual elements): st[0]: [3, 1, 4, 1, 5, 9, 2, 6] k=1 (intervals of length 2): st[1][0] = min(arr[0..1]) = min(3,1) = 1 st[1][1] = min(arr[1..2]) = min(1,4) = 1 st[1][2] = min(arr[2..3]) = min(4,1) = 1 st[1][3] = min(arr[3..4]) = min(1,5) = 1 st[1][4] = min(arr[4..5]) = min(5,9) = 5 st[1][5] = min(arr[5..6]) = min(9,2) = 2 st[1][6] = min(arr[6..7]) = min(2,6) = 2 st[1]: [1, 1, 1, 1, 5, 2, 2, _] k=2 (intervals of length 4): st[2][0] = min(st[1][0], st[1][2]) = min(1, 1) = 1 st[2][1] = min(st[1][1], st[1][3]) = min(1, 1) = 1 st[2][2] = min(st[1][2], st[1][4]) = min(1, 5) = 1 st[2][3] = min(st[1][3], st[1][5]) = min(1, 2) = 1 st[2][4] = min(st[1][4], st[1][6]) = min(5, 2) = 2 st[2]: [1, 1, 1, 1, 2, _, _, _] k=3 (intervals of length 8): st[3][0] = min(st[2][0], st[2][4]) = min(1, 2) = 1 st[3]: [1, _, _, _, _, _, _, _]
Section 3 — Build: O(n log n)

Fill the table bottom-up: st[k][i] = combine(st[k-1][i], st[k-1][i + 2^(k-1)])

RECURRENCE: st[k][i] = min(st[k-1][i], st[k-1][i + 2^(k-1)]) Explanation: "The minimum of the interval [i, i+2^k-1] = min of first half [i, i+2^(k-1)-1] and min of second half [i+2^(k-1), i+2^k-1]" = min(st[k-1][i], st[k-1][i + 2^(k-1)]) Number of levels: K = floor(log2(n)) + 1 (about 17 for n=10^5) Total cells: n * K = O(n log n)
C# 
// Precompute log2 table for O(1) log lookup during queries
int[] log2 = new int[n + 1];
log2[1] = 0;
for (int i = 2; i <= n; i++) log2[i] = log2[i/2] + 1;

int K = log2[n] + 1;
int[,] st = new int[K, n];

// Base case: k=0
for (int i = 0; i < n; i++) st[0, i] = arr[i];

// Fill table: each level uses the previous level
for (int k = 1; k < K; k++)
    for (int i = 0; i + (1 << k) <= n; i++)
        st[k, i] = Math.Min(st[k-1, i], st[k-1, i + (1 << (k-1))]);
// Time: O(n log n)   Space: O(n log n)

for k in range(1, K):
    for i in range(n - (1 << k) + 1):
        st[k][i] = min(st[k-1][i], st[k-1][i + (1 << (k-1))])
# Time: O(n log n)   Space: O(n log n)
Section 4 — Query: O(1)

For range [L, R]: find the largest k such that 2^k = (R - L + 1). Then:

RMQ(L, R) = min(st[k][L], st[k][R - 2^k + 1])

Two intervals both of length 2^k: one from L, one ending at R. They overlap, but that's OK for min/max!

QUERY min of [2, 6] (length = 5) k = floor(log2(5)) = 2 (2^2 = 4 = 5) Interval 1: [2, 2+4-1] = [2, 5] ? st[2][2] = 1 Interval 2: [6-4+1, 6] = [3, 6] ? st[2][3] = 1 Answer = min(1, 1) = 1 ? (arr[3]=1 and arr[2-6]={4,1,5,9,2}, min=1) Verification: arr[2..6] = [4, 1, 5, 9, 2], min = 1 ? The overlap ([3,5] is counted in both) doesn't matter for min/max queries because min(x, x) = x (idempotent property).
?? Idempotent operations only! The overlap trick only works for operations where f(x, x) = x. This includes min, max, GCD, AND, OR. It does NOT work for sum (since sum(x, x) = 2x ? x). For sum queries with O(1), use a prefix sum array instead.
C# 
// Query minimum of arr[l..r] — O(1)
int RangeMin(int l, int r)
{
    int k = log2[r - l + 1];
    return Math.Min(st[k, l], st[k, r - (1 << k) + 1]);
}

Section 5 — Full C# & Python Implementations
C# 
public class SparseTable
{
    private readonly int[,] st;
    private readonly int[] log2;
    private readonly int n;

    public SparseTable(int[] arr)
    {
        n = arr.Length;
        int K = (int)Math.Log2(n) + 2;
        st   = new int[K, n];
        log2 = new int[n + 1];

        // Precompute log table
        log2[1] = 0;
        for (int i = 2; i <= n; i++) log2[i] = log2[i/2] + 1;

        // Base case
        for (int i = 0; i < n; i++) st[0, i] = arr[i];

        // Build: O(n log n)
        for (int k = 1; k < K; k++)
            for (int i = 0; i + (1 << k) <= n; i++)
                st[k, i] = Math.Min(st[k-1, i], st[k-1, i + (1 << (k-1))]);
    }

    /// Range minimum of arr[l..r] — O(1)
    public int RangeMin(int l, int r)
    {
        int k = log2[r - l + 1];
        return Math.Min(st[k, l], st[k, r - (1 << k) + 1]);
    }
}

// Usage
var sp = new SparseTable(new int[] {3,1,4,1,5,9,2,6});
Console.WriteLine(sp.RangeMin(0, 7));   // 1
Console.WriteLine(sp.RangeMin(4, 7));   // 2
Console.WriteLine(sp.RangeMin(2, 5));   // 1
Python 
import math

class SparseTable:
    def __init__(self, arr):
        n = len(arr)
        K = int(math.log2(n)) + 2 if n > 0 else 1
        self.st = [[float('inf')] * n for _ in range(K)]
        self.log2 = [0] * (n + 1)
        for i in range(2, n + 1): self.log2[i] = self.log2[i // 2] + 1

        self.st[0] = arr[:]
        for k in range(1, K):
            for i in range(n - (1 << k) + 1):
                self.st[k][i] = min(self.st[k-1][i], self.st[k-1][i + (1 << (k-1))])

    def range_min(self, l, r):
        k = self.log2[r - l + 1]
        return min(self.st[k][l], self.st[k][r - (1 << k) + 1])
Section 6 — Applications: LCA & Cartesian Trees

6.1 — Lowest Common Ancestor (LCA) via Sparse Table

The LCA of two nodes in a tree can be reduced to RMQ in O(n) preprocessing + O(n) space, then answered in O(1):

  1. Perform an Euler tour (DFS, recording every visit including return visits)
  2. The Euler tour has 2n-1 nodes. Record the depth of each visited node
  3. LCA(u, v) = the node with minimum depth between the first occurrences of u and v in the tour
  4. Use sparse table on the depth array for O(1) RMQ
EXAMPLE TREE: 1 / \ 2 3 / \ 4 5 EULER TOUR (record node at each DFS step): Visit 1 (d=0), Visit 2 (d=1), Visit 4 (d=2), Back to 2 (d=1), Visit 5 (d=2), Back to 2 (d=1), Back to 1 (d=0), Visit 3 (d=1), Back to 1 (d=0) Euler tour: [1, 2, 4, 2, 5, 2, 1, 3, 1] Depths: [0, 1, 2, 1, 2, 1, 0, 1, 0] First occ: node1?0, node2?1, node3?7, node4?2, node5?4 LCA(4, 5): first_occ[4]=2, first_occ[5]=4 RMQ on depths[2..4] = min(2,1,2) = 1 at index 3 euler_tour[3] = 2 ? LCA = 2 ? LCA(4, 3): first_occ[4]=2, first_occ[3]=7 RMQ on depths[2..7] = min(2,1,2,1,0,1) = 0 at index 6 euler_tour[6] = 1 ? LCA = 1 ?

6.2 — Cartesian Trees

A Cartesian Tree is a binary tree built from an array such that:

  • The root is the minimum (or maximum) element
  • The left subtree is a Cartesian Tree of the subarray to the left of the minimum
  • The right subtree is a Cartesian Tree of the subarray to the right

Cartesian Trees can be built in O(n) using a monotonic stack, and they directly encode the RMQ structure of the array. They're the basis of treaps (tree + heap).

Section 7 — LeetCode Walkthroughs

LeetCode #239 — Sliding Window Maximum (Hard)

Given an array and window size k, find the maximum in each sliding window of size k.

?? Two approaches:

Monotonic Deque (O(n)): Maintain a deque of indices in decreasing order of values. For each new element, pop from back while it's smaller. Pop from front if outside window. Front is always the window maximum.

Sparse Table (O(n log n) build, O(1) per query): Build a sparse table for range max, then for each i from k-1 to n-1, answer RangeMax(i-k+1, i) in O(1). Total: O(n log n + n) = O(n log n). Simpler code than deque for this specific use case.
C# 
// Sparse Table approach for sliding window maximum
public int[] MaxSlidingWindow(int[] nums, int k)
{
    int n = nums.Length;
    // Build max sparse table
    int K = (int)Math.Log2(n) + 2;
    int[,] st = new int[K, n];
    int[] lg = new int[n + 1];
    for (int i=2; i<=n; i++) lg[i] = lg[i/2] + 1;
    for (int i=0; i<n; i++) st[0,i] = nums[i];
    for (int j=1; j<K; j++)
        for (int i=0; i+(1<<j)<=n; i++)
            st[j,i] = Math.Max(st[j-1,i], st[j-1,i+(1<<(j-1))]);

    int RangeMax(int l, int r) {
        int p = lg[r-l+1];
        return Math.Max(st[p,l], st[p,r-(1<<p)+1]);
    }

    var res = new int[n - k + 1];
    for (int i = k-1; i < n; i++)
        res[i-k+1] = RangeMax(i-k+1, i);
    return res;
}
Python 
import math

def max_sliding_window(nums: list, k: int) -> list:
    n  = len(nums)
    # Build max sparse table
    K  = int(math.log2(n)) + 2 if n > 0 else 1
    st = [[0] * n for _ in range(K)]
    lg = [0] * (n + 1)
    for i in range(2, n + 1): lg[i] = lg[i // 2] + 1
    for i in range(n): st[0][i] = nums[i]
    for j in range(1, K):
        for i in range(n - (1 << j) + 1):
            st[j][i] = max(st[j-1][i], st[j-1][i + (1 << (j-1))])

    def range_max(l, r):
        p = lg[r - l + 1]
        return max(st[p][l], st[p][r - (1 << p) + 1])

    res = []
    for i in range(k - 1, n):
        res.append(range_max(i - k + 1, i))
    return res

LeetCode #2104 — Sum of Subarray Ranges (Medium)

For every subarray, sum up (max - min). Return the total.

?? Approaches: O(n²) brute force or O(n) using monotonic stacks to count the contribution of each element as a range maximum or minimum. The sparse table approach (O(n² log n)) is not optimal here, but understanding which tools fit which problem is key: sparse table excels when you have many independent range queries, not when you need to aggregate over ALL subarrays (which benefits more from monotonic stack techniques).
Section 8 — Q&A Review
Q1: Why can't a Sparse Table answer range sum queries with O(1) like it does for range min/max?
The O(1) query trick uses two overlapping intervals. For min/max, overlapping is fine because min(x, x) = x — the same element counted twice doesn't change the answer (idempotent property). But for sum, counting an element twice DOUBLES its contribution: sum(x, x) = 2x ? x. The two-interval approach would overcount elements in the overlap region. For O(1) range sum queries on static arrays, use a prefix sum array instead: rangeSum(L,R) = prefix[R] - prefix[L-1].
Q2: The sparse table build is O(n log n) time and space. For n=10^5, is this a problem?
For n=10^5: log2(n) ˜ 17, so the table is 10^5 × 17 = 1.7 million integers ˜ 6.8 MB. This is acceptable. For n=10^6: ~20 million entries ˜ 80 MB — near the typical 256 MB memory limit. For n=10^7+: memory becomes a concern. In practice, for most competitive programming problems (n = 10^5 or 10^6), sparse table is fine. The build time O(n log n) is also usually acceptable since it's a one-time cost, and O(1) queries make it worthwhile when Q (number of queries) is large.
Q3: How does the Euler Tour + Sparse Table achieve O(1) LCA queries after O(n) preprocessing?
The Euler Tour of a tree visits each edge twice (once down, once up), giving 2n-1 visits total. LCA(u,v) = the shallowest (minimum depth) node visited between u and v in the tour. This reduces LCA to range minimum query on the depth array. Build sparse table on the 2n-1 depth values: O(n log n) time/space. Then LCA in O(1): find first occurrence of u (index fu) and first occurrence of v (fv), swap if fu > fv, then answer = euler_tour[RMQ(fu, fv)]. The classic reference is the Bender-Farach-Colton algorithm which uses blocks to achieve O(n) preprocessing.
Q4: What operations besides min and max work with the sparse table O(1) trick?
Any idempotent binary operation f(x,y) where f(x,x)=x works: min (min(x,x)=x ?), max (max(x,x)=x ?), GCD (gcd(x,x)=x ?), bitwise AND (x AND x = x ?), bitwise OR (x OR x = x ?). Does NOT work: sum (x+x=2x), product (x*x=x² ? x for most x), XOR (x XOR x = 0 ? x), subtraction. For non-idempotent operations with O(1) queries, you need more complex structures like sqrt decomposition or offline algorithms.
Q5: Compare sparse table to segment tree for a read-heavy, no-update scenario with 10^6 queries on an array of size 10^5.
Segment tree: O(n) build, O(log n) per query = O(n + Q log n) ˜ 10^5 + 10^6 × 17 = ~17 million operations. Sparse table: O(n log n) build, O(1) per query = O(n log n + Q) ˜ 1.7M + 1M = ~2.7M operations. Sparse table is ~6× faster for this workload. Additionally, sparse table queries have excellent cache behavior (two direct array accesses), while segment tree queries involve O(log n) pointer-chasing through the tree. Conclusion: for static arrays with heavy query loads, sparse table clearly wins. The main downside is the 6.8 MB vs 400 KB memory usage.
