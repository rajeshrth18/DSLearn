Section 1 — Motivation: The Sorted List Problem

A sorted array supports O(log n) search (binary search) but O(n) insert/delete (shifting). A sorted linked list supports O(1) insert/delete once you're at the position, but O(n) search (no random access). Can we get O(log n) for ALL three?

StructureSearchInsertDeleteSorted order
Sorted arrayO(log n)O(n)O(n)Yes
Sorted linked listO(n)O(1)*O(1)*Yes
BST (balanced)O(log n)O(log n)O(log n)Yes
Skip ListO(log n) expO(log n) expO(log n) expYes

* Requires a pointer to the predecessor node; finding it is O(n)

🌍 Real-World Uses:
  • Redis — Sorted sets (ZADD, ZRANK) are implemented with skip lists
  • LevelDB / RocksDB — MemTable uses skip list for in-memory sorted storage
  • Java ConcurrentSkipListMap — Lock-free sorted map in concurrent environments
  • Database indices — Alternative to B-Trees for some workloads
Section 2 — Skip List Structure

A skip list is a hierarchy of sorted linked lists. The bottom level (L0) contains all elements. Each higher level is a randomly chosen subset of the level below. Think of it as an express lane system on a highway — some "express" nodes span large distances at higher levels.

SKIP LIST with elements {1, 3, 5, 7, 9, 11, 13} Level 3: HEAD ─────────────────────────────── 9 ──────── TAIL Level 2: HEAD ──────── 3 ──────── 7 ──────── 9 ──────── TAIL Level 1: HEAD ──── 1 ── 3 ── 5 ── 7 ──────── 9 ── 11 ── TAIL Level 0: HEAD ── 1 ── 3 ── 5 ── 7 ── 9 ── 11 ── 13 ── TAIL Each node has forward[] array pointing to the next node at each level. Node 9: forward[0]=11, forward[1]=TAIL, forward[2]=TAIL, forward[3]=TAIL SEARCH for 7: Start at HEAD, level 3 L3: HEAD.next=9 > 7, drop down to L2 L2: HEAD.next=3 ≤ 7, go right to 3 L2: 3.next=7 = 7, go right to 7 FOUND!
📐 Structure Details:
  • A sentinel HEAD node with value -∞ points to the first node at each level
  • A sentinel TAIL node with value +∞ marks the end at each level
  • Each node stores: value + forward[] array of pointers (one per level it exists at)
  • Maximum level: typically ceil(log₂ n) + some buffer (e.g., MAX_LEVEL = 32)
  • Probability p = 0.5: each node is promoted to the next level with probability 50%
Section 3 — Search: O(log n) Expected

Start at the highest level, HEAD. At each step:

  • If current.forward[level].value <= target: move RIGHT
  • Otherwise: move DOWN one level
  • When level = 0 and you can't move right: check current.forward[0]
SEARCH for 11 in the skip list above Current=HEAD, level=3 HEAD.forward[3] = 9 ≤ 11 → move RIGHT to 9 9.forward[3] = TAIL > 11 → move DOWN to level 2 9.forward[2] = TAIL > 11 → move DOWN to level 1 9.forward[1] = 11 ≤ 11 → move RIGHT to 11 11.forward[1] = TAIL > 11 → move DOWN to level 0 11.forward[0] = 13 > 11 → can't move right Level = 0 and can't go right: CHECK current.forward[0] = 13? No. Wait — we're at 11 (we moved to it). forward[0] is checked. We landed on 11 when we moved right. Check: value == target? YES → FOUND 11 ✓ Total steps: 5 (vs O(n)=7 linear, O(log n)≈3 ideal)

Section 4 — Insert: O(log n) Expected

During search, track the update[] array: update[level] = the rightmost node at level that is less than the target. These are the nodes whose forward[level] pointers must be updated when the new node is inserted.

INSERT 6 into the skip list Search phase — track update[]: Level 3: HEAD.f[3]=9 > 6 → update[3] = HEAD Level 2: HEAD.f[2]=3 ≤ 6 → advance to 3; 3.f[2]=7 > 6 → update[2] = 3 Level 1: 3.f[1]=5 ≤ 6 → advance to 5; 5.f[1]=7 > 6 → update[1] = 5 Level 0: 5.f[0]=7 > 6 → update[0] = 5 Generate random level for new node (e.g., coin flips: H,H,T → level 2) Insert new node (val=6, height=2): Level 0: newNode.f[0] = update[0].f[0] = 7; update[0].f[0] = newNode Level 1: newNode.f[1] = update[1].f[1] = 7; update[1].f[1] = newNode (Level 2 and 3 unchanged — new node doesn't reach those levels) Result: Level 2: HEAD ── 3 ── 7 ── 9 ── TAIL (unchanged) Level 1: HEAD ── 1 ── 3 ── 5 ── 6 ── 7 ── 9 ── 11 ── TAIL (6 inserted) Level 0: HEAD ── 1 ── 3 ── 5 ── 6 ── 7 ── 9 ── 11 ── 13 ── TAIL (6 inserted)

public void Insert(int val)
{
    var update = new SkipNode[MAX_LEVEL];
    var cur = head;
    for (int i = maxLevel - 1; i >= 0; i--)
    {
        while (cur.forward[i] != null && cur.forward[i].val < val)
            cur = cur.forward[i];
        update[i] = cur;
    }

    int newLevel = RandomLevel();
    if (newLevel > maxLevel)
    {
        for (int i = maxLevel; i < newLevel; i++) update[i] = head;
        maxLevel = newLevel;

    for (int i = 0; i < newLevel; i++)

        node.forward[i]    = update[i].forward[i];
        update[i].forward[i] = node;
    }
}

            cur = cur.forward[i]
        update[i] = cur

    for i in range(new_level):
        node.forward[i]      = update[i].forward[i]
        update[i].forward[i] = node
Section 5 — Delete: O(log n) Expected

Same as insert: find update[], then splice out the node at each level it exists.

C# 
public bool Delete(int val)
{
    var update = new SkipNode[MAX_LEVEL];
    var cur = head;
    for (int i = maxLevel - 1; i >= 0; i--)
    {
        while (cur.forward[i] != null && cur.forward[i].val < val)
            cur = cur.forward[i];
        update[i] = cur;
    }
    var target = cur.forward[0];
    if (target == null || target.val != val) return false;

    for (int i = 0; i < maxLevel; i++)
    {
        if (update[i].forward[i] != target) break;
        update[i].forward[i] = target.forward[i];  // splice out at level i
    }
    // Reduce maxLevel if top levels are now empty
    while (maxLevel > 1 && head.forward[maxLevel-1] == null) maxLevel--;
    return true;
}

def delete(self, val):

    cur = self.head
    for i in range(self.max_level - 1, -1, -1):
        while cur.forward[i] and cur.forward[i].val < val:
            cur = cur.forward[i]

    if not target or target.val != val: return False
    for i in range(self.max_level):
        if update[i].forward[i] != target: break
        update[i].forward[i] = target.forward[i]   # splice out at level i
    while self.max_level > 1 and not self.head.forward[self.max_level-1]:
        self.max_level -= 1   # shrink if top levels are empty
    return True
Section 6 — Probabilistic Analysis
WHY O(log n) EXPECTED? With p=0.5 promotion probability: Expected nodes at level 0: n Expected nodes at level 1: n/2 Expected nodes at level 2: n/4 ... Expected nodes at level k: n / 2^k Expected height of skip list: O(log n) (when n/2^k ≈ 1 → k ≈ log₂ n) SEARCH COST ANALYSIS (backward analysis): Consider the search path reversed: starting from the found node, walking backward. At each step, we either: - Go UP (with prob p = 0.5): we climbed to the next level - Go LEFT (with prob 1-p = 0.5): we moved to the previous node This is a classic geometric random walk. Expected number of steps to reach height h is O(h/p) = O(log n / 0.5) = O(2 log n) = O(log n). WORST CASE: O(n) — all random coin flips go tails (nodes stay at level 1). But this has probability (1/2)^n ≈ 0 for large n. For n=1000: probability of O(n) worst case ≈ 1/2^1000 ≈ 10^(-301) — astronomically rare. SPACE: Expected O(n) — each node's expected height is O(1) (geometric series: 1 + 1/2 + 1/4 + ... = 2).
Section 7 — Full C# & Python Implementations
C# 
public class SkipList
{
    private const int MAX_LEVEL = 16;
    private const double P = 0.5;

    private class Node
    {
        public int val;
        public Node[] forward;
        public Node(int val, int level)
        {
            this.val = val;
            forward = new Node[level];
        }
    }

    private readonly Node head = new Node(int.MinValue, MAX_LEVEL);
    private int currentLevel = 1;
    private readonly Random rng = new Random();

    private int RandomLevel() {
        int lvl = 1;
        while (rng.NextDouble() < P && lvl < MAX_LEVEL) lvl++;
        return lvl;
    }

    public bool Search(int target) {
        var cur = head;
        for (int i = currentLevel-1; i >= 0; i--)
            while (cur.forward[i] != null && cur.forward[i].val < target)
                cur = cur.forward[i];
        cur = cur.forward[0];
        return cur != null && cur.val == target;
    }

    public void Insert(int val) {
        var update = new Node[MAX_LEVEL];
        var cur = head;
        for (int i = currentLevel-1; i >= 0; i--) {
            while (cur.forward[i] != null && cur.forward[i].val < val)
                cur = cur.forward[i];
            update[i] = cur;
        }
        int newLvl = RandomLevel();
        if (newLvl > currentLevel) {
            for (int i = currentLevel; i < newLvl; i++) update[i] = head;
            currentLevel = newLvl;
        }
        var node = new Node(val, newLvl);
        for (int i = 0; i < newLvl; i++) {
            node.forward[i] = update[i].forward[i];
            update[i].forward[i] = node;
        }
    }

    public bool Delete(int val) {
        var update = new Node[MAX_LEVEL];
        var cur = head;
        for (int i = currentLevel-1; i >= 0; i--) {
            while (cur.forward[i] != null && cur.forward[i].val < val)
                cur = cur.forward[i];
            update[i] = cur;
        }
        var tgt = cur.forward[0];
        if (tgt == null || tgt.val != val) return false;
        for (int i = 0; i < currentLevel; i++) {
            if (update[i].forward[i] != tgt) break;
            update[i].forward[i] = tgt.forward[i];
        }
        while (currentLevel > 1 && head.forward[currentLevel-1] == null) currentLevel--;
        return true;
    }
}

Python 
import random

class SkipList:
    MAX_LEVEL = 16
    P = 0.5

    class Node:
        def __init__(self, val, level):
            self.val = val
            self.forward = [None] * level

    def __init__(self):
        self.head = self.Node(float('-inf'), self.MAX_LEVEL)
        self.level = 1

    def _random_level(self):
        lvl = 1
        while random.random() < self.P and lvl < self.MAX_LEVEL: lvl += 1
        return lvl

    def search(self, target):
        cur = self.head
        for i in range(self.level - 1, -1, -1):
            while cur.forward[i] and cur.forward[i].val < target: cur = cur.forward[i]
        cur = cur.forward[0]
        return cur is not None and cur.val == target

    def insert(self, val):
        update = [self.head] * self.MAX_LEVEL
        cur = self.head
        for i in range(self.level - 1, -1, -1):
            while cur.forward[i] and cur.forward[i].val < val: cur = cur.forward[i]
            update[i] = cur
        new_lvl = self._random_level()
        if new_lvl > self.level:
            for i in range(self.level, new_lvl): update[i] = self.head
            self.level = new_lvl
        node = self.Node(val, new_lvl)
        for i in range(new_lvl):
            node.forward[i] = update[i].forward[i]
            update[i].forward[i] = node

    def delete(self, val):
        update = [self.head] * self.MAX_LEVEL
        cur = self.head
        for i in range(self.level - 1, -1, -1):
            while cur.forward[i] and cur.forward[i].val < val: cur = cur.forward[i]
            update[i] = cur
        tgt = cur.forward[0]
        if not tgt or tgt.val != val: return False
        for i in range(self.level):
            if update[i].forward[i] != tgt: break
            update[i].forward[i] = tgt.forward[i]
        while self.level > 1 and not self.head.forward[self.level-1]: self.level -= 1
        return True

Section 9 — Q&A Review
Q1: What is the purpose of the update[] array during skip list insert and delete?
update[i] stores the rightmost node at level i whose value is strictly less than the target. During the search phase, we fill update[] as we drop down through levels. When inserting a new node, for each level i up to the new node's height, we splice the new node between update[i] and update[i].forward[i]. Similarly for deletion: update[i].forward[i] = target.forward[i] removes the target at each level. Without update[], we'd need to traverse again for each level — O(n log n) total instead of O(log n).
Q2: What does "expected O(log n)" mean versus "worst-case O(log n)"? When would skip list perform poorly?
"Expected O(log n)" means the average over all random choices is O(log n), but individual operations CAN be slower. Worst case is O(n) if all coin flips produce heads and all nodes end up at level 1 (a plain linked list). However, this probability is (0.5)^n — negligible for any real n. "Worst-case O(log n)" (as in balanced BSTs) means EVERY operation is guaranteed O(log n) regardless. Skip lists are fine in practice; their "worst case" is so rare that they're considered equivalent to deterministic O(log n) structures for practical purposes.
Q3: Why does Redis use skip lists for its sorted set (ZADD) implementation?
Redis chose skip lists over balanced BSTs for several reasons: (1) Simpler implementation — skip lists are much easier to code, debug, and maintain than AVL or red-black trees, which require complex rotation logic. (2) Range operations are natural — iterating in sorted order (ZRANGE) just follows the level-0 linked list, while BST range queries require inorder traversal. (3) Comparable performance — same asymptotic complexity, with similar cache performance in practice. (4) Ease of lock-free concurrent versions — skip lists are more amenable to lock-free concurrent implementations. The Redis source code comment literally says "Skip lists are implemented in a simpler way than balanced trees."
Q4: How would you implement a "find Kth smallest element" operation on a skip list?
Two approaches: (1) Simple — O(n): Walk the bottom level (level 0) sequentially, counting k steps. O(k) time. (2) Augmented skip list — O(log n): Each pointer stores the "span" (how many level-0 nodes it skips over). During traversal, subtract span from k when moving right. When k=0, you've found the Kth element. This is the same idea used in order-statistic trees (which support rank and select). The span values are maintained during insert/delete in O(log n) per operation. This augmented structure is used in Redis's sorted set ZRANK command.
Q5: What promotion probability p gives the best performance, and why?
The optimal probability is p = 1/e ≈ 0.368, but p = 1/2 = 0.5 is commonly used because it's simple. With p = 1/2: expected levels = log₂(n), space = 2n pointers on average. With p = 1/4: slightly lower expected search time (fewer levels scanned), space ≈ 1.33n pointers. With higher p (closer to 1): more levels, higher space, slower search. The tradeoff: lower p → fewer total pointers (less space) but MORE horizontal steps per level; higher p → more levels but fewer horizontal steps. The optimal p balances these. In practice, p=1/4 is used when memory is critical (like in some database implementations), while p=1/2 is used for simplicity.
