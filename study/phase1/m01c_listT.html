Section 7.1 — What Is List<T> and Why It Exists

Arrays are powerful, but they have one critical limitation: fixed size. You must declare the size when you create an array, and that size never changes. In real applications, you almost never know the exact number of items you'll have upfront. List<T> solves this by wrapping a regular array and automatically resizing it when needed.

?? Real-World Analogy — The Expandable Folder:
A regular array is like a filing cabinet with exactly 10 slots — fixed forever.
List<T> is like an accordion folder — it starts small, and when you run out of space, you get a bigger folder, copy everything in, and keep going. The user never sees the swap.

Why Not Just Always Use List<T>?

Arrays still win in specific scenarios:

  • Known fixed size: A 12-month budget, a 3×3 matrix — size is baked into the domain. Arrays communicate intent clearly.
  • Multi-dimensional data: int[,] is more expressive than a list of lists.
  • Performance-critical inner loops: Arrays are slightly faster because there's no bounds-check overhead from the abstraction layer.
  • Interop with legacy/unsafe code: C APIs expect raw arrays, not managed List objects.
C# 
using System.Collections.Generic; // Required namespace

// -- Creating a List ----------------------------------------------
List<int>    nums    = new List<int>();            // empty, starts with cap=0
List<string> names   = new List<string>();         // empty
List<double> prices  = new List<double>(100);      // pre-allocate 100 slots

// Initialize with values (collection initializer):
List<int> scores = new List<int> { 85, 92, 78, 96, 88 };

// C# 9+ — simplified with 'new()' when type is clear from context:
List<string> cities = new() { "London", "Paris", "Tokyo" };

// Convert from existing array or other IEnumerable:
int[] arr      = { 1, 2, 3, 4, 5 };
List<int> list = new List<int>(arr);             // from array
List<int> l2   = arr.ToList();                   // LINQ ToList()

Section 7.2 — How Capacity Doubling Works

This is the most important concept for understanding List performance. The secret is: List<T> maintains an internal array (the backing store) whose Capacity is always >= Count. When the array fills up, it doubles the capacity.

LIST INTERNAL GROWTH — Adding items one by one State: Count=0, Capacity=0, InternalArray=[] Add(10): Count becomes 1 ? array is empty ? allocate Capacity=4 InternalArray: [ 10 | _ | _ | _ ] Count=1, Cap=4 Add(20): Count=2, fits in capacity InternalArray: [ 10 | 20 | _ | _ ] Count=2, Cap=4 Add(30): Count=3, fits InternalArray: [ 10 | 20 | 30 | _ ] Count=3, Cap=4 Add(40): Count=4, fits (exactly fills) InternalArray: [ 10 | 20 | 30 | 40 ] Count=4, Cap=4 Add(50): Count would become 5, Cap=4 is FULL ? DOUBLE to Cap=8 ? Allocate new 8-element array ? Copy 4 elements into new array ? O(n) cost here ? Place 50 at index 4 InternalArray: [ 10 | 20 | 30 | 40 | 50 | _ | _ | _ ] Cap=8 Growth sequence: 0 ? 4 ? 8 ? 16 ? 32 ? 64 ? 128 ? ... AMORTIZED O(1): A resize happens after EVERY n/2 adds. Total copy work for n adds = 2 + 4 + 8 + ... + n = 2n - 2 = O(n) Spread across n operations: O(n) / n = O(1) per add ?
C# 
// Observing Count vs Capacity
List<int> list = new List<int>();
Console.WriteLine($"Count={list.Count}, Capacity={list.Capacity}"); // 0, 0

list.Add(1);
Console.WriteLine($"Count={list.Count}, Capacity={list.Capacity}"); // 1, 4

for(int i=2; i<=5; i++) list.Add(i);
Console.WriteLine($"Count={list.Count}, Capacity={list.Capacity}"); // 5, 8

for(int i=6; i<=9; i++) list.Add(i);
Console.WriteLine($"Count={list.Count}, Capacity={list.Capacity}"); // 9, 16

// -- Pre-allocate if you know approximate size --------------------
List<int> optimized = new List<int>(1000); // no resizes for first 1000 adds

// -- TrimExcess — free unused capacity (saves memory, prevents future add) --
list.TrimExcess(); // shrinks Capacity to match Count exactly
Console.WriteLine($"Count={list.Count}, Capacity={list.Capacity}"); // 9, 9
Python 
# Python uses len() for count; capacity is internal but visible via sys.getsizeof
lst = []
print(f"len={len(lst)}")   # 0

lst.append(1)
print(f"len={len(lst)}")   # 1

for i in range(2, 6): lst.append(i)
print(f"len={len(lst)}")   # 5

for i in range(6, 10): lst.append(i)
print(f"len={len(lst)}")   # 9

# Memory usage grows in steps (Python doubles internally):
import sys
lst2 = []
for i in range(10):
    lst2.append(i)
    print(f"len={len(lst2)}, bytes={sys.getsizeof(lst2)}")
# bytes grow in steps: 0→56, 1→88, 5→120, 9→184 (capacity doublings)

# —— len() vs capacity: len() is logical size — always use it to iterate ——
?? Count vs Capacity — Don't Confuse Them:
  • list.Count — How many elements you've actually added (the logical size)
  • list.Capacity — How many elements fit in the current internal array (physical size)
  • Capacity is always = Count. You iterate up to Count — never Capacity.
Section 7.3 — All List<T> Operations

7.3.1 — Adding Elements

C# 
List<string> fruits = new List<string> { "Apple", "Banana", "Cherry" };

// Add to end — O(1) amortized
fruits.Add("Date");                         // [Apple,Banana,Cherry,Date]

// Insert at index — O(n) due to shifting
fruits.Insert(1, "Avocado");               // [Apple,Avocado,Banana,Cherry,Date]

// Add multiple items at once
fruits.AddRange(new[] { "Elderberry", "Fig" }); // adds 2 more to end
fruits.InsertRange(0, new[] { "Açaí" });      // inserts at beginning
Python 
fruits = ["Apple", "Banana", "Cherry"]

# Add to end — O(1) amortized
fruits.append("Date")                             # [Apple,Banana,Cherry,Date]

# Insert at index — O(n) due to shifting
fruits.insert(1, "Avocado")                      # [Apple,Avocado,Banana,Cherry,Date]

# Add multiple items at once
fruits.extend(["Elderberry", "Fig"])             # adds 2 more to end
fruits[0:0] = ["Acaí"]                        # inserts at beginning (slice assignment)

7.3.2 — Reading & Searching

C# 
List<int> nums = new List<int> { 10, 20, 30, 20, 40, 30 };

// Index access — O(1)
int first = nums[0];           // 10
int last  = nums[^1];          // 30  (C# 8+ index-from-end)
int count = nums.Count;         // 6

// Existence check — O(n)
bool has30 = nums.Contains(30); // true

// Position search — O(n)
int idx      = nums.IndexOf(20);      // 1  — first occurrence
int lastIdx  = nums.LastIndexOf(20); // 3  — last occurrence
int notFound = nums.IndexOf(99);     // -1

// Find with predicate — O(n)
int gt25     = nums.Find(x => x > 25);         // 30  — first match
int findIdx  = nums.FindIndex(x => x > 25);    // 2   — index of first match
int findLast = nums.FindLastIndex(x => x == 30);// 5   — last match index
List<int> allGt25 = nums.FindAll(x => x > 25); // [30,40,30]
Python 
nums = [10, 20, 30, 20, 40, 30]

# Index access — O(1)
first  = nums[0]           # 10
last   = nums[-1]           # 30  (Python negative indexing)
count  = len(nums)          # 6

# Existence check — O(n)
has30  = 30 in nums         # True

# Position search — O(n)
idx    = nums.index(20)     # 1  — first occurrence (raises ValueError if absent)
safe   = nums.index(20) if 20 in nums else -1  # safe version

# Find with condition — O(n)
gt25     = next(x for x in nums if x > 25)                     # 30  — first match
gt25_idx = next(i for i, x in enumerate(nums) if x > 25)       # 2   — index
all_gt25 = [x for x in nums if x > 25]                          # [30, 40, 30]

7.3.3 — Removing Elements

C# 
List<string> colors = new List<string> { "Red", "Green", "Blue", "Red", "Yellow" };

// Remove first matching value — O(n)
colors.Remove("Red");            // removes FIRST "Red": [Green,Blue,Red,Yellow]

// Remove by index — O(n) due to shifting
colors.RemoveAt(0);              // removes index 0 "Green": [Blue,Red,Yellow]

// Remove range — O(n)
colors.RemoveRange(0, 2);       // removes 2 items from index 0: [Yellow]

// Remove all matching predicate — O(n)
List<int> data = new List<int> { 1, 2, 3, 4, 5, 6, 7, 8 };
int removed = data.RemoveAll(x => x % 2 == 0); // removes evens: 4 removed
// data = [1,3,5,7]

// Remove last element cheaply (no shifting!) — O(1)
data.RemoveAt(data.Count - 1);   // removes 7 — no shifting!

// Clear all — O(1) with GC (references cleared, no resize)
data.Clear();                     // data = []
Python 
colors = ["Red", "Green", "Blue", "Red", "Yellow"]

# Remove first matching value — O(n)
colors.remove("Red")         # removes FIRST "Red": [Green,Blue,Red,Yellow]

# Remove by index — O(n) due to shifting
colors.pop(0)                 # removes index 0 "Green": [Blue,Red,Yellow]

# Remove range — O(n)
del colors[0:2]               # removes indices 0-1: [Yellow]

# Remove all matching predicate — O(n)
data = [1, 2, 3, 4, 5, 6, 7, 8]
data[:] = [x for x in data if x % 2 != 0]  # keep odds: [1,3,5,7]

# Remove last element cheaply (no shifting!) — O(1)
data.pop()     # removes last element: [1,3,5]

# Clear all — O(n) (GC must dereference each object)
data.clear()   # data = []
?? Performance Trick — Remove from End:
RemoveAt(list.Count - 1) removes the last element in O(1) — no elements need to shift. If you find yourself removing from the end frequently (like a stack), List<T> is fine. If you're removing from the front constantly, consider Queue<T> or LinkedList<T>.

7.3.4 — Sorting & Ordering

C# 
List<int> nums = new List<int> { 64, 25, 12, 22, 11 };

// Sort ascending (in-place) — O(n log n)
nums.Sort();
Console.WriteLine(string.Join(", ", nums)); // 11, 12, 22, 25, 64

// Sort descending — using Comparison<T> delegate
nums.Sort((a, b) => b.CompareTo(a));
Console.WriteLine(string.Join(", ", nums)); // 64, 25, 22, 12, 11

// Sort objects by a property:
var students = new List<(string Name, int Age)>
{
    ("Charlie", 22), ("Alice", 20), ("Bob", 19)
};
students.Sort((a, b) => a.Age.CompareTo(b.Age)); // sort by age ascending
// [Bob=19, Alice=20, Charlie=22]

// Reverse the list — O(n)
nums.Reverse();
Console.WriteLine(string.Join(", ", nums)); // 11, 12, 22, 25, 64

// Binary search (must be sorted first!) — O(log n)
int position = nums.BinarySearch(22); // 2
Python 
nums = [64, 25, 12, 22, 11]

# Sort ascending (in-place) — O(n log n)
nums.sort()
print(", ".join(map(str, nums)))  # 11, 12, 22, 25, 64

# Sort descending
nums.sort(reverse=True)
print(", ".join(map(str, nums)))  # 64, 25, 22, 12, 11

# Sort objects by a property:
students = [("Charlie", 22), ("Alice", 20), ("Bob", 19)]
students.sort(key=lambda s: s[1])  # sort by age ascending
# [(Bob,19), (Alice,20), (Charlie,22)]

# Reverse the list — O(n)
nums.reverse()
print(", ".join(map(str, nums)))  # 11, 12, 22, 25, 64

# Binary search (must be sorted first!) — O(log n)
import bisect
position = bisect.bisect_left(nums, 22)  # 2 (if found)

7.3.5 — Converting & Copying

C# 
List<int> list = new List<int> { 1, 2, 3, 4, 5 };

// List ? Array
int[] arr = list.ToArray(); // creates a copy — O(n)

// List ? string (for display)
string display = string.Join(", ", list); // "1, 2, 3, 4, 5"

// Shallow copy
List<int> copy = new List<int>(list);      // copies all elements
List<int> sub  = list.GetRange(1, 3);    // [2,3,4] — 3 items from index 1

// ConvertAll — transforms each element into a new type
List<string> asStrings = list.ConvertAll(x => x.ToString()); // ["1","2","3","4","5"]
List<double> doubled  = list.ConvertAll(x => (kw)x * 1.5); // [1.5,3.0,4.5,6.0,7.5]

// Exists / TrueForAll — boolean checks
bool anyNeg = list.Exists(x => x < 0);        // false
bool allPos = list.TrueForAll(x => x > 0);    // true

// ForEach — executes action on each item
list.ForEach(x => Console.Write(x + " ")); // 1 2 3 4 5
Python 
lst = [1, 2, 3, 4, 5]

# List → tuple (immutable)
tpl = tuple(lst)            # (1, 2, 3, 4, 5)

# List → string (for display)
display = ", ".join(map(str, lst))  # "1, 2, 3, 4, 5"

# Shallow copy
copy = lst[:]               # copies all elements
sub  = lst[1:4]             # [2, 3, 4] — 3 items from index 1
copy2 = list(lst)           # another way to copy

# Transform each element (equivalent to ConvertAll)
as_strings = [str(x) for x in lst]          # ["1","2","3","4","5"]
doubled    = [x * 1.5 for x in lst]         # [1.5,3.0,4.5,6.0,7.5]

# Exists / TrueForAll (equivalent to Exists/TrueForAll)
any_neg = any(x < 0 for x in lst)           # False
all_pos = all(x > 0 for x in lst)           # True

# ForEach — execute action on each item
for x in lst: print(x, end=" ")             # 1 2 3 4 5
?? Real-World Use — Shopping Cart: 
var cart = new List<(string Item, decimal Price, int Qty)>();
cart.Add(("Laptop", 999.99m, 1));
cart.Add(("Mouse",  29.99m,  2));
cart.Add(("HDMI Cable", 12.50m, 3));

// Sort by price descending
cart.Sort((a, b) => b.Price.CompareTo(a.Price));

// Remove all items under $15
cart.RemoveAll(item => item.Price < 15);

// Total cost
decimal total = 0;
cart.ForEach(item => total += item.Price * item.Qty);
Section 7.4 — LINQ on List<T>

LINQ (Language Integrated Query) adds powerful query methods to any collection. When used on List<T>, LINQ operations iterate the list — so most are O(n) unless documented otherwise. They return lazy IEnumerable<T> — they don't execute until you iterate (or call ToList()/ToArray()).

C# 
using System.Linq; // Required namespace

List<int> nums = new() { 3, 1, 4, 1, 5, 9, 2, 6, 5, 3 };

// -- FILTERING — Where --------------------------------------------
List<int> evens = nums.Where(x => x % 2 == 0).ToList(); // [4,2,6]
List<int> big   = nums.Where(x => x > 4).ToList();       // [5,9,6,5]

// -- TRANSFORMATION — Select --------------------------------------
List<int>    squared = nums.Select(x => x * x).ToList();       // [9,1,16,1,25,81,4,36,25,9]
List<string> strs   = nums.Select(x => $"#{x}").ToList();     // ["#3","#1","#4",...]

// -- ORDERING — OrderBy, OrderByDescending ------------------------
List<int> sorted  = nums.OrderBy(x => x).ToList();             // [1,1,2,3,3,4,5,5,6,9]
List<int> sortedD = nums.OrderByDescending(x => x).ToList(); // [9,6,5,5,4,3,3,2,1,1]

// -- AGGREGATION --------------------------------------------------
int    total  = nums.Sum();              // 39
double avg    = nums.Average();         // 3.9
int    max    = nums.Max();              // 9
int    min    = nums.Min();              // 1
int    cnt    = nums.Count();            // 10 — .Count() method = O(1) on List
int    cntEvn = nums.Count(x => x % 2 == 0); // 3  — count matching predicate

// -- ELEMENT ACCESS -----------------------------------------------
int f1  = nums.First();                   // 3  — throws if empty
int f2  = nums.First(x => x > 7);        // 9  — first matching
int f3  = nums.FirstOrDefault(x => x > 50); // 0  — 0 (default int) if not found
int s1  = nums.Single(x => x == 9);      // 9  — throws if not exactly 1 match
int l1  = nums.Last();                    // 3  — last element

// -- EXISTENCE CHECKS ---------------------------------------------
bool any  = nums.Any(x => x > 8);    // true  — stops at first match
bool all  = nums.All(x => x > 0);    // true  — checks all

// -- DEDUPLICATION — Distinct -------------------------------------
List<int> unique = nums.Distinct().ToList(); // [3,1,4,5,9,2,6]

// -- SKIP / TAKE — Pagination -------------------------------------
List<int> page2 = nums.Skip(4).Take(4).ToList(); // skip 4, take 4: [5,9,2,6]

// -- CHAINING MULTIPLE OPERATIONS ---------------------------------
List<string> result = nums
    .Where(x => x > 3)            // filter: [4,5,9,6,5]
    .Distinct()                    // dedupe: [4,5,9,6]
    .OrderByDescending(x => x)   // sort:   [9,6,5,4]
    .Select(x => $"#{x}")         // map:    ["#9","#6","#5","#4"]
    .ToList();
Python 
from collections import OrderedDict
nums = [3, 1, 4, 1, 5, 9, 2, 6, 5, 3]

# —— FILTERING — list comprehension (equivalent to Where) ——————————
evens = [x for x in nums if x % 2 == 0]    # [4, 2, 6]
big   = [x for x in nums if x > 4]          # [5, 9, 6, 5]

# —— TRANSFORMATION — list comprehension (equivalent to Select) ———————
squared = [x * x for x in nums]              # [9,1,16,1,25,81,4,36,25,9]
strs    = [f"#{x}" for x in nums]             # ["#3","#1",...]

# —— ORDERING — sorted() (equivalent to OrderBy) ——————————————
srt    = sorted(nums)                          # [1,1,2,3,3,4,5,5,6,9]
srt_d  = sorted(nums, reverse=True)           # [9,6,5,5,4,3,3,2,1,1]

# —— AGGREGATION ——————————————————————————————————————
total   = sum(nums)                            # 39
avg     = sum(nums) / len(nums)               # 3.9
mx      = max(nums)                            # 9
mn      = min(nums)                            # 1
cnt     = len(nums)                            # 10
cnt_evn = sum(1 for x in nums if x % 2 == 0) # 3

# —— ELEMENT ACCESS —————————————————————————————————————
f1 = nums[0]                                   # 3  — first
f2 = next(x for x in nums if x > 7)           # 9  — first matching
f3 = next((x for x in nums if x > 50), 0)    # 0  — default if not found

# —— EXISTENCE CHECKS ————————————————————————————————————
any_big = any(x > 8 for x in nums)           # True
all_pos = all(x > 0 for x in nums)           # True

# —— DEDUPLICATION — preserve order (equivalent to Distinct) ————————
unique = list(OrderedDict.fromkeys(nums))     # [3,1,4,5,9,2,6]

# —— SKIP / TAKE — slicing (equivalent to Skip/Take) ——————————————
page2 = nums[4:8]                              # skip 4, take 4: [5,9,2,6]

# —— CHAINING MULTIPLE OPERATIONS —————————————————————————————
result = [f"#{x}" for x in
          sorted(set(x for x in nums if x > 3), reverse=True)]
# ["#9","#6","#5","#4"]
?? LINQ is Lazy (Deferred Execution):
.Where(), .Select(), .OrderBy() do NOT execute immediately — they build a query pipeline.
Execution happens when you call ToList(), ToArray(), foreach, or an aggregator like Count() / Sum().
This means you can chain 5 operations and the list is iterated only once.
Section 7.5 — Collection Comparison Guide

ArrayList vs List<T>

ArrayList is an old .NET 1.x collection (before generics existed). You should never use ArrayList in modern C#. It stores object references, causing boxing/unboxing overhead and zero compile-time type safety.

FeatureArrayList (legacy)List<T> (modern)
Type safetyNone — stores objectFull — generic T
Value type performanceBoxing/unboxingNo boxing
Compile-time errorsRuntime onlyCompile-time
LINQ supportAwkward (Cast required)Full native support
SyntaxVerboseClean and readable
Modern C# useNeverAlways prefer

?? Golden Rule of Collection Choice:
Start with List<T>. Switch to a specialized collection only when a specific operation is a bottleneck. Profile first — premature optimization is the root of all evil.

Key Takeaways:
• List<T> internally wraps a regular array and doubles capacity when full
• Add() is O(1) amortized — occasional O(n) resize spread over many adds
• Count = logical size. Capacity = physical buffer size. Always iterate up to Count
• LINQ is lazy — chains build a pipeline, executed only when enumerated
• Remove from end is O(1). Remove from middle/start is O(n) due to shifting
