3 — Pattern Recognition Guide

The key to acing technical interviews is recognizing which pattern applies. Use these trigger keywords:

Two Pointers

Keywords: "sorted array", "pair sum", "in-place", "no extra space"
? Opposite ends, fast/slow pointers, Floyd's cycle

Sliding Window

Keywords: "subarray", "substring", "contiguous", "at most k distinct"
? Fixed size: deque; Variable: expand+shrink with condition

Binary Search

Keywords: "sorted", "rotated", "minimum", "feasible", "log n"
? Classic, rotated array, search on answer space

BFS

Keywords: "shortest path", "minimum steps", "level order", "nearest"
? Queue + visited set; multi-source BFS from all seeds

DFS / Backtracking

Keywords: "all combinations", "all paths", "explore", "permutations"
? Recursive DFS; backtrack by undoing choice before returning

Dynamic Programming

Keywords: "count ways", "max/min value", "optimal", "can we achieve"
? 1D: Fibonacci-like; 2D: LCS/Edit Distance; define state carefully

Heap / Priority Queue

Keywords: "top K", "K closest", "median", "merge K sorted"
? Min-heap for top-K largest; max-heap for top-K smallest

Hash Map / Set

Keywords: "two sum", "frequency", "duplicate", "group", "anagram"
? O(1) lookup; count chars with Counter; group by key

Intervals

Keywords: "merge intervals", "overlap", "meeting rooms", "schedule"
? Sort by start; merge if overlap; heap of end times for rooms

Tree DP

Keywords: "tree", "nodes", "subtree", "path sum", "diameter"
? Post-order DFS; return value for parent; track global max

Monotonic Stack

Keywords: "next greater/smaller", "largest rectangle", "temperatures"
? Maintain increasing/decreasing stack; pop when invariant breaks

Union-Find

Keywords: "connected components", "groups", "redundant connection"
? Path compression + union by rank ? near O(1) operations

Trie

Keywords: "prefix", "dictionary words", "autocomplete", "word search II"
? Build trie; DFS/BFS on trie nodes matching chars

Cyclic Sort

Keywords: "1 to n", "missing numbers", "find duplicate", "numbers in range"
? Place each number at its index; scan for mismatches

Topological Sort

Keywords: "dependencies", "course schedule", "build order", "prerequisites"
? Kahn's BFS (in-degree 0) or DFS post-order on DAG

Greedy

Keywords: "minimum cost", "maximum meetings", "activity selection"
? Local optimal choice ? global optimal; prove no counter-example

Space Complexity Targets
  • O(1) — No extra space (two pointers, in-place operations)
  • O(log n) — Recursion stack for divide and conquer / binary search
  • O(h) — Tree recursion stack (h = height; O(log n) balanced, O(n) worst)
  • O(n) — HashMap, visited set, result array
  • O(n²) — 2D DP table (optimize to O(n) with rolling array when possible)
5 — Dynamic Programming Decision Guide
Is it DP?
  • Optimal substructure: optimal solution contains optimal solutions to subproblems
  • Overlapping subproblems: same subproblems solved multiple times in naive recursion
  • Asking for: max/min value, count of ways, true/false feasibility
DP Design Steps
  1. Define state: what does dp[i] or dp[i][j] represent?
  2. Transition: how do we go from smaller subproblems to larger?
  3. Base cases: what are the smallest subproblems with known values?
  4. Order of computation: ensure subproblems computed before they're needed
  5. Optimize space: can we reduce to O(n) or O(1) by observing dependencies?

6 — Interview Strategy & Playbook
The 5-Step Problem-Solving Framework
  1. Understand (2–3 min): Clarify constraints, examples, edge cases. Ask: "What are the input/output types?" "Can values be negative?" "How large is n?"
  2. Examples (1–2 min): Walk through provided examples. Create your own edge cases (empty input, single element, all same values, sorted/reverse sorted).
  3. Approach (3–5 min): Explain brute force first, then optimize. Name the pattern: "This looks like a sliding window problem because…". State time/space complexity before coding.
  4. Code (15–20 min): Write clean code. Use meaningful variable names. Talk through your logic as you code. Handle edge cases explicitly.
  5. Test (3–5 min): Trace through your code with examples. Check edge cases. Verify complexity.
Communication Tips
  • Think out loud — interviewers want to see your reasoning process
  • Say "I'm considering a [pattern] approach because [reason]"
  • When stuck: "Let me try a simpler subproblem first" or "What if the input were size 1?"
  • Before coding: "Let me explain my approach and get your feedback"
  • After coding: "Let me trace through this with the example to verify"
Common Interview Mistakes
  • Jumping to code immediately — always clarify first, think second, code third
  • Not handling edge cases — null/empty input, single element, duplicates
  • Off-by-one errors — double-check loop bounds and array indices
  • Ignoring the follow-up — "Can you optimize space?" is often the real question
  • Silent coding — explain what you're doing at each step

8 — Must-Know Code Snippets
Union-Find with Path Compression

Essential for connected components, Kruskal's MST, number of islands variants.

class UnionFind:
    def __init__(self, n): self.parent = list(range(n)); self.rank = [0]*n
    def find(self, x):
        if self.parent[x] != x: self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py: return False
        if self.rank[px] < self.rank[py]: px, py = py, px
        self.parent[py] = px
        if self.rank[px] == self.rank[py]: self.rank[px] += 1
        return True
Binary Search Template

Universal template that handles all variants. Adjust the condition and what lo/hi represent.

# Find leftmost position where condition(mid) is True
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = lo + (hi - lo) // 2  # avoids overflow
        if arr[mid] == target: return mid
        elif arr[mid] < target: lo = mid + 1
        else: hi = mid - 1
    return -1

# Binary search on answer space (find min feasible)
def search_on_answer(lo, hi, feasible):
    while lo < hi:
        mid = (lo + hi) // 2
        if feasible(mid): hi = mid
        else: lo = mid + 1
    return lo
Monotonic Stack
# Next Greater Element
def nextGreater(nums):
    res = [-1] * len(nums); stack = []
    for i, n in enumerate(nums):
        while stack and nums[stack[-1]] < n:
            res[stack.pop()] = n
        stack.append(i)
    return res
Graph BFS with Shortest Path
from collections import deque
def bfs_shortest(graph, src, dst):
    q = deque([(src, 0)]); visited = {src}
    while q:
        node, dist = q.popleft()
        if node == dst: return dist
        for nb in graph[node]:
            if nb not in visited: visited.add(nb); q.append((nb, dist+1))
    return -1
Backtracking Template
def backtrack(candidates, target):
    result = []
    def bt(start, path, remaining):
        if remaining == 0: result.append(path[:]); return
        if remaining < 0: return  # pruning
        for i in range(start, len(candidates)):
            # Optional dedup: if i > start and candidates[i] == candidates[i-1]: continue
            path.append(candidates[i])
            bt(i+1, path, remaining-candidates[i])  # i or i+1 depending on reuse
            path.pop()
    bt(0, [], target)
    return result
