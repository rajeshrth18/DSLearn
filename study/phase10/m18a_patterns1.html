Section 1 — Two Pointers: Opposite Ends
OPPOSITE-ENDS PATTERN: Two pointers start at the two ends of the array. Move inward based on a condition. Stop when they meet. Requires: SORTED array (usually). Typical problems: - Two Sum II (sorted array, find pair summing to target) - Container With Most Water (maximize area) - 3Sum (reduce to 2-pointer after fixing first element) - Palindrome check VISUAL — Two Sum II on [2, 7, 11, 15], target=9: L=0, R=3 ? 2+15=17 > 9 ? R-- L=0, R=2 ? 2+11=13 > 9 ? R-- L=0, R=1 ? 2+7=9 == 9 ? FOUND [0, 1] TEMPLATE: L, R = 0, n-1 while L < R: if condition_met(arr[L], arr[R]): return result elif need_bigger: L++ else: R--
C# — Two Sum II + Container With Most Water 
// #167 Two Sum II — sorted array, O(n) time, O(1) space
public int[] TwoSum(int[] numbers, int target)
{
    int L = 0, R = numbers.Length - 1;
    while (L < R) {
        int s = numbers[L] + numbers[R];
        if (s == target) return new[] { L + 1, R + 1 };  // 1-indexed
        if (s < target) L++; else R--;
    }
    return Array.Empty<int>();
}

// #11 Container With Most Water — O(n)
public int MaxArea(int[] height)
{
    int L = 0, R = height.Length - 1, best = 0;
    while (L < R) {
        best = Math.Max(best, Math.Min(height[L], height[R]) * (R - L));
        if (height[L] < height[R]) L++; else R--;  // always move shorter side
    }
    return best;
}

// #15 3Sum — O(n˛), sorted, no duplicates in answer
public IList<IList<int>> ThreeSum(int[] nums)
{
    Array.Sort(nums);
    var res = new List<IList<int>>();
    for (int i = 0; i < nums.Length - 2; i++) {
        if (i > 0 && nums[i] == nums[i - 1]) continue;  // skip dup
        int L = i + 1, R = nums.Length - 1;
        while (L < R) {
            int s = nums[i] + nums[L] + nums[R];
            if (s == 0) {
                res.Add(new[] { nums[i], nums[L], nums[R] });
                while (L < R && nums[L] == nums[L + 1]) L++;
                while (L < R && nums[R] == nums[R - 1]) R--;
                L++; R--;
            } else if (s < 0) L++; else R--;
        }
    }
    return res;
}

Section 2 — Two Pointers: Same Direction (Fast/Slow)
SAME-DIRECTION PATTERNS: 1. FAST/SLOW (Floyd's — cycle detection): slow = slow.next (1 step) fast = fast.next.next (2 steps) If fast meets slow ? cycle exists. To find cycle start: after meeting, put one pointer at head, advance both by 1 until they meet ? cycle entry point. 2. IN-PLACE ARRAY EDITING (overwrite mode): slow = write position (how many valid elements kept so far) fast = read position (iterates through all elements) Write arr[slow] only when arr[fast] meets the keep condition. Example: Remove duplicates from sorted array [1,1,2,3,3] ? slow writes unique values as fast scans Result: [1,2,3,_,_], return slow (= 3) 3. REMOVE ELEMENT: slow = write index fast scans, writes arr[fast] to arr[slow] only if arr[fast] != val
C# 
// #26 Remove Duplicates from Sorted Array — O(n), O(1)
public int RemoveDuplicates(int[] nums)
{
    int slow = 1;
    for (int fast = 1; fast < nums.Length; fast++)
        if (nums[fast] != nums[slow - 1]) nums[slow++] = nums[fast];
    return slow;
}

// #141 Linked List Cycle — Floyd's algorithm
public bool HasCycle(ListNode? head)
{
    ListNode? slow = head, fast = head;
    while (fast?.Next != null) {
        slow = slow!.Next;
        fast = fast.Next.Next;
        if (slow == fast) return true;
    }
    return false;
}

// #142 Linked List Cycle II — Find cycle start
public ListNode? DetectCycle(ListNode? head)
{
    ListNode? slow = head, fast = head;
    while (fast?.Next != null) {
        slow = slow!.Next; fast = fast.Next.Next;
        if (slow == fast) {
            slow = head;  // reset one pointer to head
            while (slow != fast) { slow = slow!.Next; fast = fast!.Next; }
            return slow;  // cycle entry point
        }
    }
    return null;
}

Section 3 — Fixed-Size Sliding Window
FIXED WINDOW (size = k): Maintain a window of exactly k elements. Slide it one step at a time: add rightmost, remove leftmost. Key: reuse computation from previous window instead of re-scanning. Examples: - Max sum of subarray of size k: O(n) vs O(n*k) brute force - Max of each window of size k (#239 Sliding Window Maximum) — use deque - Find all anagrams (#438) — character frequency window TEMPLATE: Initialize window (first k elements) for i in range(k, n): add arr[i] # enter right remove arr[i-k] # leave left update answer DEQUE FOR WINDOW MAX: O(n) total Maintain a deque of indices in DECREASING order of arr[idx]. - Remove from front if out of window (idx <= i - k) - Remove from back while arr[back] <= arr[i] (useless smaller elements) - Front of deque = max of current window
C# 
// #239 Sliding Window Maximum — O(n) using deque of indices
public int[] MaxSlidingWindow(int[] nums, int k)
{
    int n = nums.Length;
    int[] res = new int[n - k + 1];
    var dq = new LinkedList<int>();  // stores indices, decreasing value order
    for (int i = 0; i < n; i++) {
        while (dq.Count > 0 && dq.First!.Value <= i - k) dq.RemoveFirst();
        while (dq.Count > 0 && nums[dq.Last!.Value] <= nums[i]) dq.RemoveLast();
        dq.AddLast(i);
        if (i >= k - 1) res[i - k + 1] = nums[dq.First!.Value];
    }
    return res;
}

// Max sum subarray of fixed size k
public int MaxSumFixed(int[] arr, int k)
{
    int sum = arr[..k].Sum(), best = sum;
    for (int i = k; i < arr.Length; i++) {
        sum += arr[i] - arr[i - k];
        best = Math.Max(best, sum);
    }
    return best;
}
Python 
from collections import deque

def maxSlidingWindow(nums, k):
    dq, res = deque(), []
    for i, v in enumerate(nums):
        while dq and dq[0] <= i-k: dq.popleft()
        while dq and nums[dq[-1]] <= v: dq.pop()
        dq.append(i)
        if i >= k-1: res.append(nums[dq[0]])
    return res
Section 4 — Variable Sliding Window
VARIABLE WINDOW — Expand right freely, shrink left on violation: PATTERN: L = 0 for R in range(n): add arr[R] to window while window_invalid: remove arr[L] from window L += 1 update answer with (R - L + 1) ? or (R - L + 1 for max) / L for min When to use: - Longest substring without repeating characters (#3) - Minimum window substring (#76) - Longest subarray with at most k distinct values (#340) - Longest substring with at most 2 distinct (#159) INVARIANT: After the while loop, window [L..R] always satisfies the constraint. For maximum window: update answer AFTER while loop (window is valid). For minimum window: update answer INSIDE while loop (window is valid and shrinking). TRICK for "at most k" ? "exactly k": f(exactly k) = f(at most k) - f(at most k-1) ? powerful decomposition
C# 
// #3 Longest Substring Without Repeating Characters
public int LengthOfLongestSubstring(string s)
{
    var last = new Dictionary<char, int>();
    int L = 0, ans = 0;
    for (int R = 0; R < s.Length; R++) {
        if (last.TryGetValue(s[R], out int prev) && prev >= L)
            L = prev + 1;  // jump L past last occurrence
        last[s[R]] = R;
        ans = Math.Max(ans, R - L + 1);
    }
    return ans;
}

// #76 Minimum Window Substring — O(n+m)
public string MinWindow(string s, string t)
{
    var need = new Dictionary<char, int>();
    foreach (char c in t) need[c] = need.GetValueOrDefault(c) + 1;
    int formed = 0, required = need.Count, L = 0, minLen = int.MaxValue, minL = 0;
    var window = new Dictionary<char, int>();
    for (int R = 0; R < s.Length; R++) {
        window[s[R]] = window.GetValueOrDefault(s[R]) + 1;
        if (need.ContainsKey(s[R]) && window[s[R]] == need[s[R]]) formed++;
        while (formed == required) {
            if (R - L + 1 < minLen) { minLen = R - L + 1; minL = L; }
            window[s[L]]--;
            if (need.ContainsKey(s[L]) && window[s[L]] < need[s[L]]) formed--;
            L++;
        }
    }
    return minLen == int.MaxValue ? "" : s.Substring(minL, minLen);
}

from collections import Counter

def lengthOfLongestSubstring(s):
    last = {}; L = ans = 0
    for R, c in enumerate(s):
        if c in last and last[c] >= L: L = last[c] + 1
        last[c] = R; ans = max(ans, R-L+1)
    return ans

def minWindow(s, t):
    need = Counter(t); window = {}
    formed = required = len(need)
    L = ans_l = ans_r = 0; best = float('inf')
    for R, c in enumerate(s):
        window[c] = window.get(c, 0) + 1
        if c in need and window[c] == need[c]: formed += 1
        while formed == required:
            if R-L+1 < best: best = R-L+1; ans_l, ans_r = L, R+1
            window[s[L]] -= 1
            if s[L] in need and window[s[L]] < need[s[L]]: formed -= 1
            L += 1
    return s[ans_l:ans_r] if best < float('inf') else ''

Q1: How do you decide whether to use two pointers vs. a hash map for "Two Sum"?
If the array is sorted (or you can sort it): two pointers is O(n) time and O(1) space — optimal. If unsorted and sorting would break index requirements (LeetCode #1 wants original indices): use a hash map for O(n) time and O(n) space. If sorted and you need original indices after sorting — two pointers but track original indices. Rule of thumb: sorted + pair sum ? two pointers. Unsorted + any pair sum ? hash map. The two-pointer works because of monotonicity: moving L right increases the sum, moving R left decreases it — a binary-search-like logical structure.
Q2: What's the key insight that makes the variable sliding window O(n) instead of O(n˛)?
The L pointer never moves backward. Each element is added once (when R includes it) and removed once (when L passes it). Total pointer movement: O(2n) = O(n). By contrast, a naive approach fixes L and scans all R from L to n, giving O(n˛). The critical design: instead of re-scanning the window from scratch each time, we maintain an incrementally updated data structure (counter, sum, set) so adding/removing an element is O(1). This makes each step O(1) and the total O(n). This amortized analysis is the core of all sliding window efficiency.
Q3: When should you use a monotone deque in a sliding window problem?
Use a monotone deque when you need the maximum (or minimum) of a sliding window efficiently. A plain max-tracking variable fails because when the current max leaves the window (gets removed from the left), you don't know what the new max is without scanning. A sorted structure would work but sorting per window is O(n log k). A monotone deque maintains indices in strictly decreasing (for max) or increasing (for min) value order. Key operations: (1) Remove expired indices from front (left of window). (2) Remove useless elements from back (smaller than incoming element, since they'll never be the max). The front is always the current window's max. Each element is pushed/popped at most once ? O(n) total.
Q4: Explain the "at most k distinct ? exactly k distinct" transformation.
Count(exactly k) = Count(at most k) - Count(at most k-1). Why? "At most k" includes windows with 0, 1, 2, ..., k distinct values. Subtracting "at most k-1" removes those with 0 to k-1 distinct, leaving exactly those with k distinct. This is powerful because the variable window template naturally handles "at most k" (expand right, shrink left when distinct > k). The "exactly k" variant is hard to code directly (window boundaries are not clean) but trivially decomposable. Example: LeetCode #904 (fruit into baskets) asks "longest subarray with at most 2 distinct" — directly a variable window. #992 asks "subarrays with exactly k odd numbers" — use the decomposition.
Q5: Why does the two-pointer approach for Container With Most Water always move the shorter side?
Area = min(height[L], height[R]) × (R - L). The width decreases by 1 with each move. To potentially increase area, we need to increase the height component. The height is determined by the shorter side. If we move the taller side inward: min(height[L], height[R]) can only stay the same or decrease (the limiting factor is still the short side), while width decreases — area can only decrease or stay. If we move the shorter side inward: the new min might be larger (if we find a taller bar), potentially increasing area despite the smaller width. Keeping the taller side is always the greedy choice because it leaves the "potential" open. Moving the shorter side is the only move that could possibly improve the answer.
