Section 1 — Rotated Sorted Array
ROTATED SORTED ARRAY: Original: [1,2,3,4,5,6,7] Rotated: [4,5,6,7,1,2,3] (rotated at index 3) Property: exactly ONE sorted half exists at any mid-point. Check which half is sorted: compare arr[lo] to arr[mid]. FIND MINIMUM (#153): The minimum is at the "inflection point" — where the array drops. Binary search: if arr[mid] > arr[hi] → min is in right half. Else → min is in left half (including mid). SEARCH TARGET (#33): Determine which half is sorted. If target is in the sorted half → search there. Else → search the other (rotated) half. INVARIANT: Keep the half that COULD contain the target. EDGE CASE (#81 — with duplicates): If arr[lo] == arr[mid] == arr[hi] → can't determine which half is sorted. Shrink both ends: lo++, hi--. Worst case O(n).
C# 
// #153 Find Minimum in Rotated Sorted Array — O(log n)
public int FindMin(int[] nums)
{
    int lo = 0, hi = nums.Length - 1;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] > nums[hi]) lo = mid + 1;  // min in right half
        else hi = mid;                             // min in left half (incl mid)
    }
    return nums[lo];
}

// #33 Search in Rotated Sorted Array — O(log n)
public int Search(int[] nums, int target)
{
    int lo = 0, hi = nums.Length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target) return mid;
        if (nums[lo] <= nums[mid]) {            // left half sorted
            if (nums[lo] <= target && target < nums[mid]) hi = mid - 1;
            else lo = mid + 1;
        } else {                                // right half sorted
            if (nums[mid] < target && target <= nums[hi]) lo = mid + 1;
            else hi = mid - 1;
        }
    }
    return -1;
}

Section 2 — Binary Search on Answer
BINARY SEARCH ON ANSWER — Most powerful binary search variant: Instead of searching for a VALUE in an array, search for the ANSWER in the answer space. TEMPLATE: Define: feasible(x) → True if answer x is achievable The feasible function is MONOTONE: once True, always True (or vice versa). Binary search on the answer space [lo, hi]. lo, hi = answer_space_bounds while lo < hi: mid = (lo + hi) // 2 if feasible(mid): hi = mid ← find minimum feasible else: lo = mid + 1 return lo ← minimum x where feasible(x) is True EXAMPLES: - Koko Eating Bananas (#875): find minimum speed k such that Koko can eat all piles in H hours. feasible(k): sum(ceil(pile/k) for pile in piles) <= H - Capacity To Ship Packages (#1011): minimum capacity such that all packages shipped in D days. - Split Array Largest Sum (#410): minimize the maximum subarray sum when splitting into m parts. - Minimum Days to Bloom (#1482): minimum days until m bouquets of k consecutive flowers bloom.
C# 
// #875 Koko Eating Bananas — Binary search on speed
public int MinEatingSpeed(int[] piles, int h)
{
    int lo = 1, hi = piles.Max();
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        long hours = piles.Sum(p => (p + mid - 1) / mid);  // ceil(p/mid)
        if (hours <= h) hi = mid;   // feasible → try smaller
        else lo = mid + 1;          // too slow → need faster
    }
    return lo;
}

// #1011 Capacity to Ship Packages — Binary search on capacity
public int ShipWithinDays(int[] weights, int days)
{
    int lo = weights.Max(), hi = weights.Sum();
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2, daysNeeded = 1, curr = 0;
        foreach (int w in weights) {
            if (curr + w > mid) { daysNeeded++; curr = 0; }
            curr += w;
        }
        if (daysNeeded <= days) hi = mid; else lo = mid + 1;
    }
    return lo;
}
Python

Section 3 — Matrix Binary Search
TWO TYPES OF SORTED MATRIX: TYPE 1 — Fully sorted (#74): Each row left-to-right, last of row < first of next row. Treat as 1D array: index i → row=i//cols, col=i%cols Standard binary search on virtual index 0..m*n-1. TYPE 2 — Row & column sorted (#240): Each row sorted, each column sorted. BUT last of row NOT necessarily < first of next row. Cannot flatten to 1D — different approach needed. STAIRCASE SEARCH for Type 2 — O(m+n): Start at TOP-RIGHT corner: row=0, col=n-1 if arr[r][c] == target → found if arr[r][c] > target → col-- (eliminate current column) if arr[r][c] < target → row++ (eliminate current row) WHY? Top-right has a "searchable" property: everything to its left is smaller, everything below is larger. Each step eliminates an entire row or column → O(m+n).
C# 
// #74 Search a 2D Matrix (fully sorted) — O(log(m*n))
public bool SearchMatrix(int[][] matrix, int target)
{
    int m = matrix.Length, n = matrix[0].Length, lo = 0, hi = m * n - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        int v = matrix[mid / n][mid % n];
        if (v == target) return true;
        if (v < target) lo = mid + 1; else hi = mid - 1;
    }
    return false;
}

// #240 Search a 2D Matrix II (row+col sorted) — O(m+n) staircase
public bool SearchMatrixII(int[][] matrix, int target)
{
    int r = 0, c = matrix[0].Length - 1;
    while (r < matrix.Length && c >= 0) {
        int v = matrix[r][c];
        if (v == target) return true;
        if (v > target) c--;
        else r++;
    }
    return false;
}

Section 4 — Generating All Subsets
SUBSETS (#78) — 2^n total subsets for n elements: BITMASK APPROACH — Iterative, clean: For each number i from 0 to 2^n - 1: If bit j of i is set → include nums[j] in this subset. O(n * 2^n) time and space. BACKTRACKING APPROACH — Recursive: At each index, make two choices: INCLUDE or SKIP. Collect result when recursion finishes (or at every node = include partial sets). VISUAL for nums=[1,2,3]: 000 → [] 001 → [1] 010 → [2] 011 → [1,2] 100 → [3] 101 → [1,3] 110 → [2,3] 111 → [1,2,3] SUBSETS WITH DUPLICATES (#90): Sort first. Skip duplicate elements at the same level. In backtracking: if i > start AND nums[i] == nums[i-1] → skip.
C# 
// #78 Subsets — backtracking O(n·2^n)
public IList<IList<int>> Subsets(int[] nums)
{
    var res = new List<IList<int>>();
    var curr = new List<int>();
    void Bt(int start) {
        res.Add(new List<int>(curr));  // snapshot at every node
        for (int i = start; i < nums.Length; i++) {
            curr.Add(nums[i]);
            Bt(i + 1);
            curr.RemoveAt(curr.Count - 1);
        }
    }
    Bt(0);
    return res;
}

// #90 Subsets II — with duplicates
public IList<IList<int>> SubsetsWithDup(int[] nums)
{
    Array.Sort(nums);
    var res = new List<IList<int>>();
    var curr = new List<int>();
    void Bt(int start) {
        res.Add(new List<int>(curr));
        for (int i = start; i < nums.Length; i++) {
            if (i > start && nums[i] == nums[i - 1]) continue;  // skip dup
            curr.Add(nums[i]);
            Bt(i + 1);
            curr.RemoveAt(curr.Count - 1);
        }
    }
    Bt(0);
    return res;
}

Section 5 — Permutations & Combinations
C# 
// #46 Permutations — O(n!) — swap-based backtracking
public IList<IList<int>> Permute(int[] nums)
{
    var res = new List<IList<int>>();
    void Bt(int start) {
        if (start == nums.Length) { res.Add(nums.ToList()); return; }
        for (int i = start; i < nums.Length; i++) {
            (nums[start], nums[i]) = (nums[i], nums[start]);  // swap in
            Bt(start + 1);
            (nums[start], nums[i]) = (nums[i], nums[start]);  // swap back
        }
    }
    Bt(0);
    return res;
}

// #77 Combinations — choose k from n — O(k * C(n,k))
public IList<IList<int>> Combine(int n, int k)
{
    var res = new List<IList<int>>();
    var curr = new List<int>();
    void Bt(int start) {
        if (curr.Count == k) { res.Add(new List<int>(curr)); return; }
        int remain = k - curr.Count;
        for (int i = start; i <= n - remain + 1; i++) {  // pruning: can't get k
            curr.Add(i); Bt(i + 1); curr.RemoveAt(curr.Count - 1);
        }
    }
    Bt(1);
    return res;
}
Python 
def permute(nums):
    res = []
    def bt(start):
        if start == len(nums): res.append(nums[:]); return
        for i in range(start, len(nums)):
            nums[start], nums[i] = nums[i], nums[start]
            bt(start+1)
            nums[start], nums[i] = nums[i], nums[start]
    bt(0); return res

Q1: What invariant must hold for binary search on answer to work?
The feasibility predicate must be monotone: if feasible(x) is True for some x, then feasible(x') is True for all x' ≥ x (for "find minimum feasible") or all x' ≤ x (for "find maximum feasible"). This monotonicity creates the binary search property — we can eliminate half the search space at each step. Examples: in Koko, "can eat all bananas at speed k in H hours" — once True for speed k, it's True for all speeds ≥ k. The answer space must also be bounded. If neither condition holds (predicate is non-monotone), binary search on answer doesn't apply.
Q2: How do you handle the case where the rotated array has duplicate elements?
When arr[lo] == arr[mid] == arr[hi], we can't determine which half is sorted. We can't tell if [lo..mid] is monotonically increasing (sorted) or if there's a rotation point there. Solution: shrink both ends by 1 (lo++, hi--) and continue. This degrades worst-case to O(n) when all elements are identical, but maintains O(log n) average. For purely finding the minimum with duplicates, this shrinkage approach still works. For searching a target with duplicates (#81), the same trick applies. The key insight: when you can't eliminate half the search space, you can only safely eliminate one element at a time from each end.
Q3: Why does the staircase search work for a row-and-column sorted matrix?
Starting at the top-right corner: all elements in the same row to the left are smaller (row sorted), all elements below in the same column are larger (column sorted). This gives us a "decision tree": if current > target, we know target must be in a column to the left (current's entire column downward is too large) → move left. If current < target, we know target must be in a row below (current's entire row to the left is too small) → move down. Each step eliminates an entire row or column, so worst case is O(m+n) steps. Starting at bottom-left also works by the symmetric argument. Top-left and bottom-right don't work (both neighbors are larger or both smaller, so you can't decide direction).
Q4: What's the time complexity of generating all subsets and why?
O(n · 2^n). There are exactly 2^n subsets (each element is either in or out). For each subset, we spend O(n) time copying it to the result. So total is O(n · 2^n). In terms of the recursion tree: there are 2^n leaf nodes, and the depth is n, but the total nodes in the tree is 2^(n+1) - 1 = O(2^n). The backtracking approach visits each node once and does O(size_of_subset) work at leaf nodes. Average subset size is n/2, so average work per leaf is n/2. Total: 2^n · n/2 = O(n · 2^n).
Q5: How do you avoid duplicate subsets when there are duplicate elements in the input?
Sort the array first. In the backtracking loop, skip element at index i if i > start AND nums[i] == nums[i-1]. The condition i > start is critical — it means we're skipping duplicates at the SAME recursion level, not across levels. Why? If we're at the same level and nums[i] == nums[i-1], including nums[i] would produce the same subtree as including nums[i-1] (since they're equal). But at different levels (i == start), we must still process the first occurrence. Without the i > start guard, we'd skip too aggressively and miss valid subsets. The same trick applies to permutations with duplicates (#47) — sort, use used[], skip if nums[i]==nums[i-1] and !used[i-1].
