Section 1 — Dictionary<TKey,TValue> — The Workhorse

Dictionary<TKey,TValue> is .NET's built-in hash table. It is the most-used collection in C# after arrays. Under the hood it uses open addressing with linear probing (since .NET 6+, changed from chaining for better cache performance).

C# 
using System;
using System.Collections.Generic;
using System.Linq;

class Program
{
    static void Main()
    {
        // ── Creation ─────────────────────────────────────
        var scores = new Dictionary<string, int>();
        var prebuilt = new Dictionary<string, int>
        {
            ["Alice"] = 95, ["Bob"] = 87, ["Carol"] = 91
        };

        // ── Add / Update ─────────────────────────────────
        scores["Dave"] = 88;          // Add new key
        scores["Dave"] = 92;          // Update existing key
        scores.Add("Eve", 79);         // Add (throws if key exists)
        scores.TryAdd("Eve", 80);      // Add only if not present (safe)

        // ── Safe Lookup ──────────────────────────────────
        if (scores.TryGetValue("Dave", out int val))
            Console.WriteLine($"Dave: {val}");  // Dave: 92

        // ── GetValueOrDefault (no throw) ─────────────────
        int frank = scores.GetValueOrDefault("Frank", 0);  // 0 (not found)

        // ── Increment safely (count pattern) ────────────
        string[] words = { "the", "fox", "the", "cat", "the" };
        var freq = new Dictionary<string, int>();
        foreach (var w in words)
            freq[w] = freq.GetValueOrDefault(w) + 1;
        // freq = { "the":3, "fox":1, "cat":1 }

        // ── Remove & ContainsKey ─────────────────────────
        scores.Remove("Eve");
        bool hasAlice = prebuilt.ContainsKey("Alice");  // true

        // ── Iteration ────────────────────────────────────
        foreach (var kvp in prebuilt)
            Console.WriteLine($"{kvp.Key}: {kvp.Value}");

        foreach (var (key, score) in prebuilt)  // Deconstruct
            Console.WriteLine($"{key} scored {score}");

        // ── LINQ on Dictionary ───────────────────────────
        var topScorer = prebuilt.MaxBy(kvp => kvp.Value);
        Console.WriteLine($"Top: {topScorer.Key} = {topScorer.Value}");

        var highScorers = prebuilt
            .Where(kvp => kvp.Value >= 90)
            .Select(kvp => kvp.Key)
            .ToList();
        // highScorers = ["Alice", "Carol"]

        /* ── Output ──
        Dave: 92
        Alice: 95
        Bob: 87
        Carol: 91
        Alice scored 95
        Bob scored 87
        Carol scored 91
        Top: Alice = 95
        */
    }
}

Section 2 — HashSet<T> — Uniqueness & Set Operations

HashSet<T> is a hash table that stores only keys — no values. Its primary use: membership testing in O(1), and set algebra (union, intersection, difference). Every element is unique; duplicates are silently ignored.

C# 
using System.Collections.Generic;

class Program
{
    static void Main()
    {
        var setA = new HashSet<int> { 1, 2, 3, 4, 5 };
        var setB = new HashSet<int> { 3, 4, 5, 6, 7 };

        // ── Membership (O(1)) ────────────────────────────
        bool has3 = setA.Contains(3);   // true
        setA.Add(3);                     // false — already exists

        // ── Set Algebra (modifies setA in-place) ─────────
        var union  = new HashSet<int>(setA); union.UnionWith(setB);
        // union = {1,2,3,4,5,6,7}

        var inter  = new HashSet<int>(setA); inter.IntersectWith(setB);
        // inter = {3,4,5}

        var diff   = new HashSet<int>(setA); diff.ExceptWith(setB);
        // diff  = {1,2}  (elements in A but not B)

        var symDiff = new HashSet<int>(setA); symDiff.SymmetricExceptWith(setB);
        // symDiff = {1,2,6,7}  (in A or B, but not both)

        // ── Subset / Superset Checks ─────────────────────
        var small = new HashSet<int> { 3, 4 };
        bool isSub = small.IsSubsetOf(setA);       // true (3,4 are in setA)
        bool isSuper = setA.IsSupersetOf(small);   // true
        bool disjoint = setA.Overlaps(new HashSet<int> { 8, 9 });  // false

        // ── Classic pattern: deduplicate an array ────────
        int[] nums = { 1, 2, 2, 3, 3, 4 };
        var unique = new HashSet<int>(nums);  // {1,2,3,4}

        // ── Classic pattern: check duplicates O(n) ──────
        static bool HasDuplicate(int[] arr)
        {
            var seen = new HashSet<int>();
            foreach (var x in arr)
                if (!seen.Add(x)) return true;  // Add returns false if dup
            return false;
        }

        /* ── Output ──
        (No direct output — demonstrates operations above)
        */
    }
}

Section 3 — SortedDictionary vs SortedList

When you need keys in sorted order, .NET offers two choices — both using Red-Black Trees internally, not hash tables:

FeatureSortedDictionary<K,V>SortedList<K,V>
Underlying structureRed-Black TreeTwo sorted arrays
Lookup by keyO(log n)O(log n) binary search
Insert / DeleteO(log n)O(n) — array shift
Index access (by position)Not supportedO(1)
MemoryHigher (tree nodes)Lower (arrays)
Best forFrequent inserts/deletesMostly read, sorted index access
💡 When to choose: If you frequently add/remove keys from a sorted collection, use SortedDictionary. If the collection is built once and mostly read (and you need positional access), use SortedList. Neither is a hash table — they live in O(log n) land, not O(1).
Section 4 — ConcurrentDictionary — Thread Safety

Dictionary<TKey,TValue> is not thread-safe. Concurrent reads are safe, but concurrent writes corrupt the internal state. For multi-threaded code, use ConcurrentDictionary<TKey,TValue>, which provides atomic operations without requiring external locks.

C# 
using System.Collections.Concurrent;

var cache = new ConcurrentDictionary<string, int>();

// GetOrAdd — atomic "get if present, compute and add if absent"
int v1 = cache.GetOrAdd("x", key => 42);  // adds and returns 42
int v2 = cache.GetOrAdd("x", key => 99);  // returns 42 (already existed)

// AddOrUpdate — atomic "add if absent, update if present"
cache.AddOrUpdate(
    key:          "counter",
    addValue:     1,                    // initial value if new
    updateValueFactory: (k, old) => old + 1  // increment if exists
);

// TryUpdate — atomic compare-and-swap
cache.TryUpdate("x", newValue: 100, comparisonValue: 42);
// Only updates if current value == 42 (prevents lost updates)

// Thread-safe word frequency counter (parallel)
var freq = new ConcurrentDictionary<string, int>();
string[] words = { "a", "b", "a", "c", "a", "b" };
Parallel.ForEach(words, w =>
    freq.AddOrUpdate(w, 1, (_, old) => old + 1)
);
// Safe even with multiple threads writing simultaneously
Python 
from threading import Lock
from collections import defaultdict
import threading

# Python dict is NOT thread-safe for read-modify-write. Use Lock.
cache = {}
lock  = Lock()

# GetOrAdd equivalent — atomic "get if present, add if absent"
def get_or_add(d, key, factory):
    with lock:
        if key not in d: d[key] = factory(key)
        return d[key]

v1 = get_or_add(cache, "x", lambda k: 42)  # adds 42
v2 = get_or_add(cache, "x", lambda k: 99)  # returns 42 (already exists)

# AddOrUpdate equivalent — atomic increment
freq = defaultdict(int)
def increment(key):
    with lock: freq[key] += 1

# Thread-safe word frequency (parallel)
words = ["a", "b", "a", "c", "a", "b"]
ts = [threading.Thread(target=increment, args=(w,)) for w in words]
for t in ts: t.start()
for t in ts: t.join()
# freq = {"a": 3, "b": 2, "c": 1}
Section 5 — Python dict & set

Python's dict is a hash table (open addressing with pseudorandom probing). Since Python 3.7, it also preserves insertion order. Python's set is the equivalent of HashSet<T>.

C# 
// C# equivalents of Python's advanced collection helpers
using System;
using System.Collections.Generic;
using System.Linq;
using System.Collections.Immutable;

// defaultdict equivalent — GetValueOrDefault + auto-init
var freq = new Dictionary<string, int>();
foreach (var w in new[] { "the", "fox", "the", "cat", "the" })
    freq[w] = freq.GetValueOrDefault(w) + 1;
// freq = { "the":3, "fox":1, "cat":1 }

// Counter equivalent — LINQ GroupBy
var words = new[] { "the", "fox", "the", "cat", "the" };
var counter = words.GroupBy(w => w).ToDictionary(g => g.Key, g => g.Count());

// most_common(2) equivalent
var top2 = counter.OrderByDescending(kv => kv.Value).Take(2).ToList();
// [("the",3), ("fox",1)]

// OrderedDict — Dictionary preserves insertion order since .NET 5+
var ordered = new Dictionary<string, int>();
ordered["first"]  = 1; ordered["second"] = 2; ordered["third"]  = 3;
// Iterates in insertion order: first, second, third

// frozenset equivalent — ImmutableHashSet (System.Collections.Immutable)
var frozen = ImmutableHashSet.Create(1, 2, 3);
bool has2  = frozen.Contains(2);   // true — immutable, safe for dict keys

Section 6 — The Two-Sum Pattern

The most fundamental hash table interview pattern: replace an O(n²) nested-loop search with O(n) by storing "what we've seen" in a hash map. For each element, check if its complement already exists in the map.

C# 
// Two Sum — return indices of two numbers that add to target
// O(n) time, O(n) space
static int[] TwoSum(int[] nums, int target)
{
    var seen = new Dictionary<int, int>();  // value → index
    for (int i = 0; i < nums.Length; i++)
    {
        int complement = target - nums[i];
        if (seen.TryGetValue(complement, out int j))
            return new[] { j, i };  // found! complement was at index j
        seen[nums[i]] = i;
    }
    return Array.Empty<int>();
}

// Two Sum II — sorted array variant (two pointers, O(1) space)
static int[] TwoSumSorted(int[] nums, int target)
{
    int lo = 0, hi = nums.Length - 1;
    while (lo < hi)
    {
        int sum = nums[lo] + nums[hi];
        if      (sum == target) return new[] { lo + 1, hi + 1 };
        else if (sum <  target) lo++;
        else                    hi--;
    }
    return Array.Empty<int>();
}

class Program
{
    static void Main()
    {
        var result = TwoSum(new[] { 2, 7, 11, 15 }, 9);
        Console.WriteLine($"[{result[0]}, {result[1]}]");  // [0, 1]

        /* ── Output ──
        [0, 1]
        */
    }
}
Python 
# Two Sum — O(n) time, O(n) space
def two_sum(nums, target):
    seen = {}                       # value → index
    for i, x in enumerate(nums):
        complement = target - x
        if complement in seen:
            return [seen[complement], i]
        seen[x] = i
    return []

# Two Sum II — sorted array variant (two pointers, O(1) space)
def two_sum_sorted(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo < hi:
        s = nums[lo] + nums[hi]
        if   s == target: return [lo + 1, hi + 1]
        elif s <  target: lo += 1
        else:             hi -= 1
    return []

if __name__ == "__main__":
    print(two_sum([2, 7, 11, 15], 9))   # [0, 1]
    # Output:
    # [0, 1]
Section 7 — The Frequency Count Pattern

Count occurrences of each element to solve grouping and top-k problems. This covers a huge class of interview questions: Group Anagrams, Top K Frequent Elements, Valid Anagram, and more.

C# 
using System;
using System.Collections.Generic;
using System.Linq;

class Program
{
    // Group Anagrams — ["eat","tea","tan","ate","nat","bat"]
    // → [["eat","tea","ate"], ["tan","nat"], ["bat"]]
    static IList<IList<string>> GroupAnagrams(string[] strs)
    {
        var groups = new Dictionary<string, List<string>>();
        foreach (var s in strs)
        {
            // Key = sorted characters (anagrams share the same sorted form)
            char[] chars = s.ToCharArray();
            Array.Sort(chars);
            string key = new string(chars);

            if (!groups.TryGetValue(key, out var list))
                groups[key] = list = new List<string>();
            list.Add(s);
        }
        return groups.Values.Cast<IList<string>>().ToList();
    }

    // Top K Frequent Elements
    static int[] TopKFrequent(int[] nums, int k)
    {
        var freq = new Dictionary<int, int>();
        foreach (var n in nums)
            freq[n] = freq.GetValueOrDefault(n) + 1;

        return freq
            .OrderByDescending(kvp => kvp.Value)
            .Take(k)
            .Select(kvp => kvp.Key)
            .ToArray();
    }

    static void Main()
    {
        var groups = GroupAnagrams(new[] { "eat", "tea", "tan", "ate", "nat", "bat" });
        foreach (var g in groups)
            Console.WriteLine(string.Join(",", g));

        var top = TopKFrequent(new[] { 1, 1, 1, 2, 2, 3 }, 2);
        Console.WriteLine(string.Join(",", top));

        /* ── Output ──
        eat,tea,ate
        tan,nat
        bat
        1,2
        */
    }
}
Python 
from collections import Counter, defaultdict

# Group Anagrams
def group_anagrams(strs):
    groups = defaultdict(list)
    for s in strs:
        key = "".join(sorted(s))   # sorted chars = canonical anagram key
        groups[key].append(s)
    return list(groups.values())

# Top K Frequent Elements — Counter.most_common is O(n log k)
def top_k_frequent(nums, k):
    return [x for x, _ in Counter(nums).most_common(k)]

if __name__ == "__main__":
    groups = group_anagrams(["eat", "tea", "tan", "ate", "nat", "bat"])
    for g in groups:
        print(",".join(g))
    print(",".join(map(str, top_k_frequent([1, 1, 1, 2, 2, 3], 2))))
    # Output:
    # eat,tea,ate
    # tan,nat
    # bat
    # 1,2
Section 8 — LRU Cache: LinkedList + Dictionary

The LRU (Least Recently Used) Cache is the most famous hash table interview problem. It combines a Dictionary for O(1) lookup with a LinkedList for O(1) eviction order maintenance. Both languages below implement the same O(1) get and O(1) put.

C# 
using System.Collections.Generic;

public class LRUCache
{
    private readonly int _cap;
    private readonly Dictionary<int, LinkedListNode<(int key, int val)>> _map;
    private readonly LinkedList<(int key, int val)> _list;

    public LRUCache(int capacity)
    {
        _cap  = capacity;
        _map  = new();
        _list = new();
    }

    public int Get(int key)
    {
        if (!_map.TryGetValue(key, out var node)) return -1;
        _list.Remove(node);       // O(1) — have the node reference
        _list.AddFirst(node);     // Move to front (most recently used)
        return node.Value.val;
    }

    public void Put(int key, int value)
    {
        if (_map.TryGetValue(key, out var existing))
        {
            _list.Remove(existing);   // Remove old node
            _map.Remove(key);
        }
        else if (_list.Count == _cap)
        {
            var lru = _list.Last!;         // Evict least recently used
            _map.Remove(lru.Value.key);
            _list.RemoveLast();
        }
        var node = _list.AddFirst((key, value));
        _map[key] = node;
    }
}

class Program
{
    static void Main()
    {
        var lru = new LRUCache(3);
        lru.Put(1, 10); lru.Put(2, 20); lru.Put(3, 30);
        Console.WriteLine(lru.Get(1));   // 10 — moves key 1 to front
        lru.Put(4, 40);                   // evicts key 2 (LRU)
        Console.WriteLine(lru.Get(2));   // -1 (evicted)
        Console.WriteLine(lru.Get(3));   // 30
        Console.WriteLine(lru.Get(4));   // 40

        /* ── Output ──
        10
        -1
        30
        40
        */
    }
}
Python 
from collections import OrderedDict

class LRUCache:
    """Python LRU Cache using OrderedDict (move_to_end = O(1))"""
    def __init__(self, capacity: int):
        self._cap  = capacity
        self._cache = OrderedDict()  # maintains insertion order

    def get(self, key: int) -> int:
        if key not in self._cache:
            return -1
        self._cache.move_to_end(key)  # O(1) — mark as most recently used
        return self._cache[key]

    def put(self, key: int, value: int) -> None:
        if key in self._cache:
            self._cache.move_to_end(key)
        self._cache[key] = value
        if len(self._cache) > self._cap:
            self._cache.popitem(last=False)  # evict oldest (front)


if __name__ == "__main__":
    lru = LRUCache(3)
    lru.put(1, 10); lru.put(2, 20); lru.put(3, 30)
    print(lru.get(1))  # 10
    lru.put(4, 40)     # evicts key 2
    print(lru.get(2))  # -1 (evicted)
    print(lru.get(4))  # 40

    # Output:
    # 10
    # -1
    # 40

Section 9 — Check Your Understanding
Q1. You call dict[key] = value when the key already exists. What happens, and how does it differ from dict.Add(key, value)?
Hint: One throws, one doesn't.
The indexer dict[key] = value is an upsert — it adds the key if absent, or silently updates the value if the key already exists. dict.Add(key, value) throws ArgumentException if the key already exists. Use Add when you want to ensure you're only inserting new keys (fail-fast). Use the indexer when you want to set a value regardless.
Q2. Why does HashSet<T>.Add() return a bool? How is this useful?
Hint: It returns false when the element already exists.
Add() returns true if the element was newly inserted, false if it was already present. This lets you detect duplicates in a single operation: if (!seen.Add(x)) return true; // duplicate found. It replaces the two-step pattern of Contains() + Add() with a single O(1) call, both checking membership and inserting atomically.
Q3. In the LRU cache implementation, why do we store the full (key, value) tuple in the LinkedList nodes instead of just the value?
Hint: Think about what happens during eviction.
During eviction, we remove the last node from the LinkedList (the LRU item). We then need to remove that same key from the Dictionary. If the node only stored the value, we would not know which key to delete from the Dictionary — we'd have to search for it (O(n)). By storing the key in the node, we can do map.Remove(lruNode.Value.key) in O(1) without any search.
Q4. GroupAnagrams uses a sorted string as the dictionary key. What is the time complexity of the full algorithm for n strings each of length k?
Hint: Count the sorting cost per string.
For each of the n strings, we sort its k characters: O(k log k) per sort. Total: O(n · k log k). The dictionary operations are O(k) per string (to hash a key of length k), adding O(nk) which is dominated by the sort. Space: O(nk) for all keys + values.
Q5. When should you choose ConcurrentDictionary over Dictionary with a lock?
Hint: Think about fine-grained vs. coarse-grained locking.
ConcurrentDictionary uses stripe-based locking — it divides the internal array into segments, each with its own lock. Multiple threads can write to different segments simultaneously. A manual lock around a Dictionary uses a single, coarse-grained lock that serialises ALL threads. Choose ConcurrentDictionary when: (1) you have high concurrency with many threads, (2) writes are common, and (3) you need atomic compound operations like GetOrAdd. Use a manual lock when you need atomicity across multiple dictionary operations as a unit (e.g., read-then-write logic that must be atomic).
