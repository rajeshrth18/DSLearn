Easy Problems (1—4)
#1 Two Sum
Easy HashMap Array

Problem: Given an array of integers nums and an integer target, return the indices of the two numbers that add up to target. Each input has exactly one solution. You may not use the same element twice.

Example: nums = [2,7,11,15], target = 9 → [0, 1]

🧠 Approach: Single-pass HashMap. For each element, compute complement = target - nums[i]. If the complement is already in the map, return its stored index and i. Otherwise store nums[i]?i. O(n) time, O(n) space.
Time: O(n) — single HashMap pass; each element stored/looked up once &nbsp;|&nbsp; Space: O(n) — HashMap stores up to n elements
C# 
static int[] TwoSum(int[] nums, int target)
{
    var seen = new Dictionary<int, int>();  // value ? index
    for (int i = 0; i < nums.Length; i++)
    {
        int comp = target - nums[i];
        if (seen.TryGetValue(comp, out int j))
            return new[] { j, i };
        seen[nums[i]] = i;
    }
    return Array.Empty<int>();
}
// TwoSum([2,7,11,15], 9) ? [0, 1]
Python 
def two_sum(nums: list[int], target: int) -> list[int]:
    seen = {}  # value ? index
    for i, n in enumerate(nums):
        comp = target - n
        if comp in seen:
            return [seen[comp], i]
        seen[n] = i
    return []

# two_sum([2,7,11,15], 9) ? [0, 1]
#2 Valid Anagram
Easy HashMap String

Problem: Given two strings s and t, return true if t is an anagram of s, and false otherwise.

Example: s = "anagram", t = "nagaram" → true  |  s = "rat", t = "car" → false

🧠 Approach: Count character frequencies with a dictionary (or a 26-element array for lowercase ASCII). Increment for every char in s, decrement for every char in t. If all counts are zero at the end, it's an anagram. O(n) time, O(1) space (fixed 26-letter alphabet).
Time: O(n) — two passes, one per string &nbsp;|&nbsp; Space: O(1) — 26-element fixed-size frequency array
C# 
static bool IsAnagram(string s, string t)
{
    if (s.Length != t.Length) return false;
    var count = new int[26];
    for (int i = 0; i < s.Length; i++)
    {
        count[s[i] - 'a']++;
        count[t[i] - 'a']--;
    }
    return count.All(c => c == 0);
}
// IsAnagram("anagram","nagaram") ? true
Python 
from collections import Counter

def is_anagram(s: str, t: str) -> bool:
    return Counter(s) == Counter(t)
    # Or: return sorted(s) == sorted(t)  — O(n log n)

# is_anagram("anagram", "nagaram") ? True
#3 Contains Duplicate
Easy HashSet Array

Problem: Given an integer array nums, return true if any value appears at least twice in the array, and false if every element is distinct.

Example: [1,2,3,1] → true  |  [1,2,3,4] → false

🧠 Approach: Use a HashSet. For each element, try to add it. Add() returns false if the element already exists — that's a duplicate. O(n) time, O(n) space.
Time: O(n) — single pass adding each element to HashSet &nbsp;|&nbsp; Space: O(n) — HashSet stores up to n elements
C# 
static bool ContainsDuplicate(int[] nums)
{
    var seen = new HashSet<int>();
    foreach (var n in nums)
        if (!seen.Add(n)) return true;  // Add returns false = duplicate
    return false;
}
Python 
def contains_duplicate(nums: list[int]) -> bool:
    return len(nums) != len(set(nums))
    # Or explicit: seen = set(); return any(x in seen or seen.add(x) for x in nums)
#4 First Non-Repeating Character
Easy HashMap String

Problem: Given a string s, find the first non-repeating character and return its index. If it does not exist, return -1.

Example: "leetcode" → 0  |  "aabb" → -1

🧠 Approach: Two-pass. Pass 1: count frequency of each character. Pass 2: iterate the string again and return the index of the first character with count == 1. O(n) time, O(1) space (26 letters).
Time: O(n) — two passes over the string &nbsp;|&nbsp; Space: O(1) — 26-element fixed-size frequency array
C# 
static int FirstUniqChar(string s)
{
    var freq = new int[26];
    foreach (var c in s) freq[c - 'a']++;
    for (int i = 0; i < s.Length; i++)
        if (freq[s[i] - 'a'] == 1) return i;
    return -1;
}
// FirstUniqChar("leetcode") ? 0
Python 
from collections import Counter

def first_uniq_char(s: str) -> int:
    freq = Counter(s)
    for i, c in enumerate(s):
        if freq[c] == 1:
            return i
    return -1

# first_uniq_char("leetcode") ? 0
Medium Problems (5—9)
#5 Group Anagrams
Medium HashMap String Sorting

Problem: Given an array of strings, group the anagrams together. You can return the answer in any order.

Example: ["eat","tea","tan","ate","nat","bat"] → [["bat"],["nat","tan"],["ate","eat","tea"]]

🧠 Approach: Use a sorted string as the dictionary key — anagrams share the same sorted form. Group all strings with the same sorted key together. O(n—k—log k) where k = max string length.
Time: O(n—k log k) — sort each string of length k; n strings total &nbsp;|&nbsp; Space: O(n—k) — HashMap stores n entries, each key/value up to length k
C# 
static IList<IList<string>> GroupAnagrams(string[] strs)
{
    var groups = new Dictionary<string, List<string>>();
    foreach (var s in strs)
    {
        char[] ch = s.ToCharArray(); Array.Sort(ch);
        string key = new string(ch);
        if (!groups.TryGetValue(key, out var lst))
            groups[key] = lst = new();
        lst.Add(s);
    }
    return groups.Values.Cast<IList<string>>().ToList();
}
Python 
from collections import defaultdict

def group_anagrams(strs: list[str]) -> list[list[str]]:
    groups = defaultdict(list)
    for s in strs:
        groups[tuple(sorted(s))].append(s)
    return list(groups.values())
#6 Subarray Sum Equals K
Medium HashMap Prefix Sum

Problem: Given an array of integers nums and an integer k, return the total number of subarrays whose sum equals k.

Example: nums = [1,1,1], k = 2 → 2

🧠 Approach: Prefix sum + HashMap. Keep a running prefix sum. For each position, check how many earlier prefix sums equal currentSum - k (because sum[l..r] = prefixSum[r] - prefixSum[l-1]). Store prefix sum frequencies. O(n) time.
Time: O(n) — single pass with O(1) HashMap lookup per step &nbsp;|&nbsp; Space: O(n) — prefix sum frequency map
C# 
static int SubarraySum(int[] nums, int k)
{
    var prefixCount = new Dictionary<int, int> { [0] = 1 };
    int sum = 0, result = 0;
    foreach (var n in nums)
    {
        sum += n;
        result += prefixCount.GetValueOrDefault(sum - k);
        prefixCount[sum] = prefixCount.GetValueOrDefault(sum) + 1;
    }
    return result;
}
// SubarraySum([1,1,1], 2) ? 2
Python 
from collections import defaultdict

def subarray_sum(nums: list[int], k: int) -> int:
    prefix_count = defaultdict(int, {0: 1})
    total = s = 0
    for n in nums:
        s += n
        total += prefix_count[s - k]
        prefix_count[s] += 1
    return total

# subarray_sum([1,1,1], 2) ? 2
#7 Longest Consecutive Sequence
Medium HashSet Array

Problem: Given an unsorted array of integers, find the length of the longest consecutive elements sequence. Must run in O(n).

Example: [100,4,200,1,3,2] → 4 (the sequence 1,2,3,4)

🧠 Approach: Add all numbers to a HashSet. For each number n that has no left neighbour (n-1 not in set), it's the start of a sequence. Extend the sequence by checking n+1, n+2, ... and tracking the length. Only start sequences at their beginning to avoid O(n—) recounts.
Time: O(n) — each number visited at most twice (start check + extend) &nbsp;|&nbsp; Space: O(n) — HashSet of all n numbers
C# 
static int LongestConsecutive(int[] nums)
{
    var numSet = new HashSet<int>(nums);
    int best = 0;
    foreach (var n in numSet)
    {
        if (numSet.Contains(n - 1)) continue;  // not start of sequence
        int cur = n, len = 1;
        while (numSet.Contains(++cur)) len++;
        best = Math.Max(best, len);
    }
    return best;
}
// LongestConsecutive([100,4,200,1,3,2]) ? 4
Python 
def longest_consecutive(nums: list[int]) -> int:
    num_set = set(nums)
    best = 0
    for n in num_set:
        if n - 1 in num_set:
            continue  # not the start
        length = 1
        while n + length in num_set:
            length += 1
        best = max(best, length)
    return best

# longest_consecutive([100,4,200,1,3,2]) ? 4
#8 Top K Frequent Elements
Medium HashMap Heap Bucket Sort

Problem: Given an integer array nums and an integer k, return the k most frequent elements. The answer may be returned in any order.

Example: nums = [1,1,1,2,2,3], k = 2 → [1,2]

🧠 Approach: Count frequencies with a HashMap. Use bucket sort: create an array of size n+1 where index = frequency, bucket[i] contains all numbers with that frequency. Collect from highest bucket down until k elements. O(n) time.
Time: O(n) — frequency count + bucket sort &nbsp;|&nbsp; Space: O(n) — frequency HashMap + n+1 bucket array
C# 
static int[] TopKFrequent(int[] nums, int k)
{
    var freq = new Dictionary<int, int>();
    foreach (var n in nums)
        freq[n] = freq.GetValueOrDefault(n) + 1;

    // Bucket sort: bucket[i] = list of numbers with frequency i
    var buckets = new List<int>[nums.Length + 1];
    foreach (var (num, cnt) in freq)
    {
        buckets[cnt] ??= new();
        buckets[cnt].Add(num);
    }

    var result = new List<int>();
    for (int i = buckets.Length - 1; i >= 0 && result.Count < k; i--)
        if (buckets[i] != null) result.AddRange(buckets[i]);

    return result.Take(k).ToArray();
}
// TopKFrequent([1,1,1,2,2,3], 2) ? [1,2]
Python 
from collections import Counter

def top_k_frequent(nums: list[int], k: int) -> list[int]:
    # Counter.most_common is O(n log k) using a heap internally
    return [x for x, _ in Counter(nums).most_common(k)]

# top_k_frequent([1,1,1,2,2,3], 2) ? [1, 2]
#9 LRU Cache
Medium HashMap LinkedList Design

Problem: Design a data structure that follows the constraints of a Least Recently Used (LRU) cache. Implement the LRUCache class with get(key) (returns -1 if not found) and put(key, value) (evicts least recently used if over capacity). Both operations must run in O(1) average time.

🧠 Approach: Combine a Dictionary (key ? LinkedList node) for O(1) lookup with a doubly LinkedList for O(1) order maintenance. Most recently used = head. Least recently used = tail. On every access, move node to head.
Time: O(1) &mdash; get and put use HashMap + doubly linked list; O(1) per op &nbsp;|&nbsp; Space: O(capacity) &mdash; HashMap + LinkedList bounded by cache size
📚 Full implementation covered in Module 8B, Section 8. The solution below is a compact recap.
C# 
public class LRUCache
{
    private readonly int _cap;
    private readonly Dictionary<int, LinkedListNode<(int k, int v)>> _map;
    private readonly LinkedList<(int k, int v)> _list;

    public LRUCache(int cap) { _cap = cap; _map = new(); _list = new(); }

    public int Get(int key)
    {
        if (!_map.TryGetValue(key, out var node)) return -1;
        _list.Remove(node); _list.AddFirst(node);
        return node.Value.v;
    }

    public void Put(int key, int val)
    {
        if (_map.TryGetValue(key, out var ex)) { _list.Remove(ex); _map.Remove(key); }
        else if (_list.Count == _cap) { _map.Remove(_list.Last!.Value.k); _list.RemoveLast(); }
        _map[key] = _list.AddFirst((key, val));
    }
}
Python 
from collections import OrderedDict

class LRUCache:
    def __init__(self, cap):
        self.cap = cap; self.cache = OrderedDict()

    def get(self, key):
        if key not in self.cache: return -1
        self.cache.move_to_end(key); return self.cache[key]

    def put(self, key, val):
        if key in self.cache: self.cache.move_to_end(key)
        self.cache[key] = val
        if len(self.cache) > self.cap: self.cache.popitem(last=False)
Hard Problems (10—12)
#10 Minimum Window Substring
Hard HashMap Sliding Window Two Pointers

Problem: Given strings s and t, return the minimum window in s which contains all the characters in t. If no such window exists, return empty string.

Example: s = "ADOBECODEBANC", t = "ABC" → "BANC"

🧠 Approach: Sliding window with two hash maps. Track how many characters from t are "satisfied". Expand the right pointer until all characters are covered, then shrink the left pointer to find the minimum window. O(|s| + |t|) time.
Time: O(|s| + |t|) — each char of s visited at most twice (expand + shrink) &nbsp;|&nbsp; Space: O(|s| + |t|) — two frequency maps
C# 
static string MinWindow(string s, string t)
{
    if (s.Length == 0 || t.Length == 0) return "";

    var need = new Dictionary<char, int>();
    foreach (var c in t) need[c] = need.GetValueOrDefault(c) + 1;

    var window = new Dictionary<char, int>();
    int formed = 0, required = need.Count;
    int lo = 0, bestLen = int.MaxValue, bestLo = 0;

    for (int hi = 0; hi < s.Length; hi++)
    {
        char c = s[hi];
        window[c] = window.GetValueOrDefault(c) + 1;
        if (need.ContainsKey(c) && window[c] == need[c]) formed++;

        while (formed == required)
        {
            if (hi - lo + 1 < bestLen) { bestLen = hi - lo + 1; bestLo = lo; }
            char lc = s[lo++];
            window[lc]--;
            if (need.ContainsKey(lc) && window[lc] < need[lc]) formed--;
        }
    }
    return bestLen == int.MaxValue ? "" : s.Substring(bestLo, bestLen);
}
// MinWindow("ADOBECODEBANC", "ABC") ? "BANC"
Python 
from collections import Counter

def min_window(s: str, t: str) -> str:
    if not t: return ""
    need = Counter(t)
    window = Counter()
    formed = required = len(need)
    lo = best_lo = 0
    best_len = float("inf")

    for hi, c in enumerate(s):
        window[c] += 1
        if need[c] > 0 and window[c] == need[c]:
            formed -= 1

        while formed == 0:
            if hi - lo + 1 < best_len:
                best_len, best_lo = hi - lo + 1, lo
            lc = s[lo]; lo += 1
            window[lc] -= 1
            if need[lc] > 0 and window[lc] < need[lc]:
                formed += 1

    return "" if best_len == float("inf") else s[best_lo: best_lo + best_len]

# min_window("ADOBECODEBANC", "ABC") ? "BANC"
#11 Subarrays with K Different Integers
Hard HashMap Sliding Window

Problem: Given an integer array nums and an integer k, return the number of good subarrays. A good subarray is a subarray with exactly k different integers.

Example: nums = [1,2,1,2,3], k = 2 → 7

🧠 Approach: Key insight: exactly(k) = atMost(k) - atMost(k-1). The atMost(k) helper counts subarrays with at most k distinct integers using a sliding window with a frequency map. O(n) time.
Time: O(n) — two sliding window passes over the array &nbsp;|&nbsp; Space: O(n) — frequency map in each window
C# 
static int SubarraysWithKDistinct(int[] nums, int k)
    => AtMost(nums, k) - AtMost(nums, k - 1);

static int AtMost(int[] nums, int k)
{
    var freq = new Dictionary<int, int>();
    int lo = 0, result = 0;
    for (int hi = 0; hi < nums.Length; hi++)
    {
        freq[nums[hi]] = freq.GetValueOrDefault(nums[hi]) + 1;
        while (freq.Count > k)
        {
            freq[nums[lo]]--;
            if (freq[nums[lo]] == 0) freq.Remove(nums[lo]);
            lo++;
        }
        result += hi - lo + 1;  // All subarrays ending at hi with =k distinct
    }
    return result;
}
// SubarraysWithKDistinct([1,2,1,2,3], 2) ? 7
Python 
from collections import defaultdict

def subarrays_with_k_distinct(nums: list[int], k: int) -> int:
    def at_most(k):
        freq = defaultdict(int); lo = total = 0
        for hi, n in enumerate(nums):
            freq[n] += 1
            while len(freq) > k:
                freq[nums[lo]] -= 1
                if freq[nums[lo]] == 0: del freq[nums[lo]]
                lo += 1
            total += hi - lo + 1
        return total
    return at_most(k) - at_most(k - 1)

# subarrays_with_k_distinct([1,2,1,2,3], 2) ? 7
#12 Design a Bloom Filter
Hard Bloom Filter Bit Manipulation Design

Problem: Design a Bloom filter that supports Add(item) and MightContain(item). Given an expected capacity and a target false positive rate, choose optimal parameters. False negatives must be impossible.

🧠 Approach: Use m = -n—ln(p) / (ln 2)— bits and k = (m/n)—ln 2 hash functions. Use double hashing (two base hashes + linear combination) to approximate k independent hash functions cheaply. Store bits in a BitArray (C#) or bytearray (Python).
Time: O(k) per insert/query &mdash; k hash function evaluations &nbsp;|&nbsp; Space: O(m) &mdash; m-bit array where m = -n&times;ln(p)/(ln 2)&sup2;
📚 Full implementation covered in Module 8C, Sections 6 & 7. Below is a self-contained sketch of the key design decisions.
C# 
// Key design decisions for a Bloom Filter:

// 1. Compute optimal parameters
double p = 0.01;   // target FP rate
int n = 10000;     // expected insertions
int m = (int)(-n * Math.Log(p) / (Math.Log(2) * Math.Log(2)));
int k = (int)((double)m / n * Math.Log(2));

// 2. Use double hashing to simulate k independent hash functions
//    h_i(x) = (h1(x) + i * h2(x)) % m
//    Only two actual hash computations per item — O(1) setup

// 3. Store state in BitArray (m/8 bytes, very compact)
var bits = new BitArray(m);

// 4. Add: set k bits
// 5. MightContain: check k bits — return false if ANY bit is 0
// 6. NEVER clear bits ? no false negatives possible

// Memory comparison (n=10000, p=1%):
// Bloom filter: m — 95,851 bits — 12 KB
// HashSet<string> (avg 10 chars): 10000 — ~40 bytes — 400 KB  (33— larger)
Python 
import math, hashlib

class BloomFilter:
    def __init__(self, n=10000, p=0.01):
        self.m = int(-n * math.log(p) / math.log(2)**2)
        self.k = int(self.m / n * math.log(2))
        self.bits = bytearray((self.m + 7) // 8)

    def _indices(self, item):
        d = item.encode()
        h1 = int(hashlib.md5(d).hexdigest(), 16)
        h2 = int(hashlib.sha1(d).hexdigest(), 16)
        return [abs(h1 + i*h2) % self.m for i in range(self.k)]

    def add(self, item):
        for i in self._indices(item):
            self.bits[i//8] |= (1 << (i%8))

    def might_contain(self, item):
        return all(self.bits[i//8] & (1 << (i%8)) for i in self._indices(item))
