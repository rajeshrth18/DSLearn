Part 1 — Bellman-Ford Algorithm
Section 1 — What is Bellman-Ford?

Dijkstra's algorithm is fast, but it breaks on graphs with negative edge weights. Bellman-Ford, developed independently by Richard Bellman (1958) and Lester Ford Jr. (1956), handles negative edges and can also detect negative weight cycles — cycles where the total weight is negative, making "shortest paths" undefined (you can keep looping forever to reduce cost).

The key insight: instead of greedily picking the closest vertex like Dijkstra, Bellman-Ford takes a simpler brute-force approach — it relaxes all edges V-1 times. By the V-1th pass, the shortest path to every reachable vertex is guaranteed to be found (since the longest possible simple path in a V-vertex graph has V-1 edges).

?? Why V-1 passes? The shortest path between any two vertices in a graph with V vertices can use at most V-1 edges (otherwise it would revisit a vertex, creating a cycle — but if all cycles had positive weight, the path without the cycle would be shorter). Each relaxation pass adds one more "hop" of confidence. After k passes, we know the true shortest paths using at most k edges.

Section 2 — Negative Cycle Detection

After V-1 passes, if we do one more (Vth) pass and any distance can still be reduced, the graph contains a negative weight cycle. In such a graph, shortest paths are undefined — you can loop the cycle infinitely to decrease cost forever.

NEGATIVE CYCLE DETECTION — Extra Pass Graph: A?B(1), B?C(2), C?A(-10) Cycle: A?B?C?A, weight = 1+2+(-10) = -7 ? NEGATIVE CYCLE! After V-1 passes, distances have been stabilized (for a non-negative-cycle graph). Doing pass V: A?B: can dist[B] be reduced? B = A + 1. C?A: can dist[A] be reduced? Yes! dist[A] = dist[C] + (-10) = smaller value. If any dist[v] changes in the Vth pass ? NEGATIVE CYCLE DETECTED. This is Bellman-Ford's killer feature — Dijkstra cannot detect negative cycles at all.
Section 3 — C# Implementation
C# — Bellman-Ford with Negative Cycle Detection 
using System;
using System.Collections.Generic;

public class BellmanFord
{
    // Edge list representation — (from, to, weight)
    private readonly List<(int u, int v, int w)> edges = new();
    private readonly int n;  // number of vertices

    public BellmanFord(int vertices) => n = vertices;

    public void AddEdge(int u, int v, int w) => edges.Add((u, v, w));

    // --- Run Bellman-Ford --------------------------------------------
    // Returns (dist array, hasNegativeCycle)
    // dist[v] = shortest distance from source to v
    // dist[v] = int.MinValue means v is on or reachable from a negative cycle
    public (int[] dist, bool hasNegCycle) Run(int source)
    {
        const int INF = int.MaxValue / 2;  // use /2 to avoid overflow on addition
        int[] dist = new int[n];
        Array.Fill(dist, INF);
        dist[source] = 0;

        // -- Phase 1: Relax all edges V-1 times --
        for (int pass = 0; pass < n - 1; pass++)
        {
            bool updated = false;
            foreach (var (u, v, w) in edges)
            {
                if (dist[u] == INF) continue;  // u not yet reachable
                if (dist[u] + w < dist[v])
                {
                    dist[v] = dist[u] + w;
                    updated = true;
                }
            }
            if (!updated) break;  // early exit — no more relaxations possible
        }

        // -- Phase 2: Check for negative cycles (Vth pass) --
        bool hasNegCycle = false;
        foreach (var (u, v, w) in edges)
        {
            if (dist[u] == INF) continue;
            if (dist[u] + w < dist[v])
            {
                dist[v] = int.MinValue;  // mark as affected by neg cycle
                hasNegCycle = true;
            }
        }
        return (dist, hasNegCycle);
    }

    // --- Path Reconstruction ----------------------------------------
    public List<int> GetPath(int source, int target)
    {
        const int INF = int.MaxValue / 2;
        int[] dist = new int[n];  Array.Fill(dist, INF);
        int[] prev = new int[n];  Array.Fill(prev, -1);
        dist[source] = 0;

        for (int pass = 0; pass < n - 1; pass++)
            foreach (var (u, v, w) in edges)
                if (dist[u] != INF && dist[u] + w < dist[v])
                {
                    dist[v] = dist[u] + w;
                    prev[v] = u;  // track predecessor
                }

        var path = new List<int>();
        for (int cur = target; cur != -1; cur = prev[cur])
            path.Insert(0, cur);
        return dist[target] == INF ? new List<int>() : path;
    }

    public static void Main()
    {
        var bf = new BellmanFord(5);  // 5 vertices: 0,1,2,3,4
        bf.AddEdge(0, 1, 6); bf.AddEdge(0, 3, 7);
        bf.AddEdge(1, 2, 5); bf.AddEdge(1, 4, 4); bf.AddEdge(1, 3, -4);
        bf.AddEdge(2, 1, -2);
        bf.AddEdge(3, 2, -3); bf.AddEdge(3, 4, 9);
        bf.AddEdge(4, 0, 2); bf.AddEdge(4, 2, 7);

        var (dist, hasCycle) = bf.Run(0);
        Console.WriteLine($"Negative cycle: {hasCycle}");  // False
        for (int i = 0; i < dist.Length; i++)
            Console.WriteLine($"0?{i}: {dist[i]}");
        // 0?0:0  0?1:2  0?2:4  0?3:-2  0?4:2  (CLRS example)
    }
}
Python — Bellman-Ford 
from typing import List, Tuple

class BellmanFord:
    def __init__(self, n: int):
        self.n = n
        self.edges: List[Tuple[int, int, int]] = []  # (u, v, w)

    def add_edge(self, u: int, v: int, w: int):
        self.edges.append((u, v, w))

    def run(self, source: int) -> Tuple[List[float], bool]:
        INF  = float('inf')
        dist = [INF] * self.n
        dist[source] = 0

        # Phase 1: V-1 relaxation passes
        for _ in range(self.n - 1):
            updated = False
            for u, v, w in self.edges:
                if dist[u] != INF and dist[u] + w < dist[v]:
                    dist[v] = dist[u] + w
                    updated = True
            if not updated: break  # early exit

        # Phase 2: Check for negative cycles (Vth pass)
        has_neg_cycle = False
        for u, v, w in self.edges:
            if dist[u] != INF and dist[u] + w < dist[v]:
                dist[v] = -INF   # reachable via negative cycle
                has_neg_cycle = True

        return dist, has_neg_cycle


if __name__ == "__main__":

    bf.add_edge(0,1,6); bf.add_edge(0,3,7)
    bf.add_edge(1,2,5); bf.add_edge(1,4,4); bf.add_edge(1,3,-4)
    bf.add_edge(2,1,-2)
    bf.add_edge(3,2,-3); bf.add_edge(3,4,9)
    bf.add_edge(4,0,2); bf.add_edge(4,2,7)
    dist, neg = bf.run(0)
    print(f"Negative cycle: {neg}")   # False
    for i, d in enumerate(dist):
        print(f"0?{i}: {d}")
    # 0?0:0  0?1:2  0?2:4  0?3:-2  0?4:2

Part 2 — Floyd-Warshall Algorithm
Section 4 — What is Floyd-Warshall?

Both Dijkstra and Bellman-Ford are single-source shortest path algorithms: they find shortest distances from one source to all other vertices. But what if you need the shortest distance between every pair of vertices? Running Dijkstra V times gives O(V × (V+E)logV) — acceptable for sparse graphs, but there's a cleaner alternative.

Floyd-Warshall (Robert Floyd 1962, Stephen Warshall 1962) solves all-pairs shortest paths in O(Vł) with a beautifully simple dynamic programming approach. It works on graphs with negative edges (but not negative cycles) and is often the simplest implementation for dense graphs.

?? The DP Intuition: Define dist[i][j] = shortest distance from i to j using only vertices {0, 1, ..., k} as intermediate nodes. The recurrence is: dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]) — "is going through vertex k shorter than the current best?" Iterate k from 0 to V-1, updating the matrix. After all V iterations, dist[i][j] = true shortest path.

(continue updating) FINAL matrix after all 4 iterations: 0 1 2 3 0 [ 0 3 5 6 ] 1 [ 5 0 2 3 ] 2 [ 3 6 0 1 ] 3 [ 2 5 7 0 ] Example: 0?3 shortest path = 6 (0?1?2?3: 3+2+1=6, better than direct 0?3=7)
Section 5 — C# Implementation
C# — Floyd-Warshall with Path Reconstruction 
using System;

public class FloydWarshall
{
    private readonly int n;
    private readonly int[,] dist;    // dist[i,j] = shortest distance i?j
    private readonly int[,] next;    // next[i,j] = next hop from i on path to j
    private const int INF = 1_000_000_000;  // use large but not MaxValue (overflow risk)

    public FloydWarshall(int vertices)
    {
        n    = vertices;
        dist = new int[n, n];
        next = new int[n, n];
        // Initialize: no edge between different vertices, 0 on diagonal
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++)
            {
                dist[i, j] = (i == j) ? 0 : INF;
                next[i, j] = -1;
            }
    }

    public void AddEdge(int u, int v, int w)
    {
        if (w < dist[u, v])  // handle multiple edges between same pair
        {
            dist[u, v] = w;
            next[u, v] = v;  // next hop from u to v is directly v
        }
    }

    public void Compute()
    {
        // Classic Floyd-Warshall triple loop
        for (int k = 0; k < n; k++)          // intermediate vertex
            for (int i = 0; i < n; i++)      // source
                for (int j = 0; j < n; j++)  // destination
                {
                    if (dist[i, k] == INF || dist[k, j] == INF) continue;
                    int through = dist[i, k] + dist[k, j];
                    if (through < dist[i, j])
                    {
                        dist[i, j] = through;
                        next[i, j] = next[i, k];  // update path: go to k first
                    }
                }
    }

    public bool HasNegativeCycle()
    {
        // After Compute(), if any dist[i,i] < 0, there's a negative cycle
        for (int i = 0; i < n; i++)
            if (dist[i, i] < 0) return true;
        return false;
    }

    public int Distance(int u, int v) => dist[u, v] == INF ? -1 : dist[u, v];

    public int[] GetPath(int u, int v)
    {
        if (next[u, v] == -1) return Array.Empty<int>();  // no path
        var path = new System.Collections.Generic.List<int>();
        for (int cur = u; cur != v; cur = next[cur, v])
        {
            path.Add(cur);
            if (path.Count > n) break;  // cycle guard
        }
        path.Add(v);
        return path.ToArray();
    }

    public static void Main()
    {
        var fw = new FloydWarshall(4);
        fw.AddEdge(0,1,3); fw.AddEdge(0,3,7);
        fw.AddEdge(1,0,8); fw.AddEdge(1,2,2);
        fw.AddEdge(2,0,5); fw.AddEdge(2,3,1);
        fw.AddEdge(3,0,2);
        fw.Compute();

        Console.WriteLine(fw.Distance(0, 3));           // 6  (0?1?2?3)
        Console.WriteLine(string.Join("?", fw.GetPath(0, 3)));  // 0?1?2?3
        Console.WriteLine(fw.HasNegativeCycle());          // False
    }
}
Python — Floyd-Warshall 
from typing import List, Optional

class FloydWarshall:
    def __init__(self, n: int):
        INF = float('inf')
        self.n    = n
        self.dist = [[INF]*n for _ in range(n)]
        self.nxt  = [[-1]*n  for _ in range(n)]
        for i in range(n):
            self.dist[i][i] = 0

    def add_edge(self, u: int, v: int, w: float):
        if w < self.dist[u][v]:
            self.dist[u][v] = w
            self.nxt[u][v]  = v

    def compute(self):
        n = self.n
        for k in range(n):            # intermediate
            for i in range(n):        # source
                for j in range(n):    # destination
                    if (self.dist[i][k] == float('inf') or
                        self.dist[k][j] == float('inf')): continue
                    through = self.dist[i][k] + self.dist[k][j]
                    if through < self.dist[i][j]:
                        self.dist[i][j] = through
                        self.nxt[i][j]  = self.nxt[i][k]

    def has_negative_cycle(self) -> bool:
        return any(self.dist[i][i] < 0 for i in range(self.n))

    def get_path(self, u: int, v: int) -> List[int]:
        if self.nxt[u][v] == -1: return []
        path = []
        while u != v:
            path.append(u); u = self.nxt[u][v]
        path.append(v)
        return path


if __name__ == "__main__":
    fw = FloydWarshall(4)
    fw.add_edge(0,1,3); fw.add_edge(0,3,7)
    fw.add_edge(1,0,8); fw.add_edge(1,2,2)
    fw.add_edge(2,0,5); fw.add_edge(2,3,1)
    fw.add_edge(3,0,2)
    fw.compute()
    print(fw.dist[0][3])           # 6
    print(fw.get_path(0, 3))        # [0, 1, 2, 3]
    print(fw.has_negative_cycle())  # False

Section 7 — Real-World Applications
?? Bellman-Ford — Distance Vector Routing (RIP Protocol)
The Routing Information Protocol (RIP), one of the oldest internet routing protocols, uses a distributed version of Bellman-Ford. Each router knows the cost to its direct neighbors and exchanges distance vectors with neighbors iteratively. Bellman-Ford's ability to handle cost changes (including "negative" in the sense of cost decreases) makes it suitable. However, it's slow to converge — modern networks prefer OSPF (Dijkstra-based). The "count to infinity" problem in RIP is essentially Bellman-Ford's convergence on negative feedback.
?? Bellman-Ford — Arbitrage Detection in Finance
In currency exchange, if you can convert USD?EUR?GBP?USD and end up with more USD than you started, that's arbitrage. Model it as a graph: currencies = vertices, exchange rates = edges with weight = -log(rate). A negative cycle in this graph = arbitrage opportunity! Bellman-Ford is specifically used by financial systems to detect (and exploit) these cycles. Banks and hedge funds run this constantly on real-time exchange rate data.
?? Floyd-Warshall — Network Analysis (All-Pairs Connectivity)
Floyd-Warshall is used to compute the transitive closure of a graph (which vertices can reach which others) and to find network diameter (maximum shortest path). Applications include: social network analysis (degrees of separation for all user pairs), transportation hub analysis (which airports have good connectivity to all others), and database query optimization (finding all transitive dependencies).
Section 8 — Common Mistakes
?? Floyd-Warshall: Wrong loop order — k must be the OUTER loop: The three loops must be ordered k (intermediate) ? i (source) ? j (destination). If you put k as the inner loop, the algorithm produces incorrect results because you'd be using partially-updated values for intermediate vertices that haven't been considered yet.
?? Bellman-Ford: Not handling unreachable vertices: When dist[u] = INF, the expression dist[u] + w can overflow in languages with fixed-size integers. Always guard with if (dist[u] == INF) continue; before relaxing. In Python, float('inf') + any_number is still inf, so this is less of an issue.
?? Floyd-Warshall O(V˛) space — not suitable for large sparse graphs: A graph with 100,000 vertices would require a 100,000 × 100,000 matrix — 40 billion bytes! For large sparse graphs, always use Dijkstra (or Bellman-Ford) per source vertex instead.
?? Bellman-Ford early termination — don't skip the negative cycle check: Even if you add the early-exit optimization (stop when no relaxations happen in a pass), you must still perform the separate Vth-pass negative cycle check. The early exit only skips unnecessary passes when the graph has no negative cycles — it does not detect them.
Section 9 — Check Your Understanding
Q1 — Bellman-Ford runs V-1 relaxation passes. Why exactly V-1 and not V or V+1?
Answer: In a graph with V vertices, the longest possible simple path (a path that doesn't revisit any vertex) has at most V-1 edges. After k relaxation passes, Bellman-Ford guarantees that all shortest paths using at most k edges have been found. Therefore, after V-1 passes, all shortest paths are finalized — no simple path can be longer. Running more passes would only be needed to detect negative cycles (the Vth pass does this). Running fewer passes might miss paths with many hops.
Key insight: If a shortest path requires more than V-1 edges, it must revisit a vertex — forming a cycle. If that cycle has positive weight, removing it gives a shorter path. If it has zero weight, the path is equivalent. Only negative-weight cycles could "help" — but those make shortest paths undefined.
Q2 — In Floyd-Warshall, why do we check dist[i][i] < 0 after running to detect negative cycles?
Answer: dist[i][i] represents the shortest cycle starting and ending at vertex i. Initially, dist[i][i] = 0 (trivial zero-length cycle). After Floyd-Warshall runs, if a negative-weight cycle exists that passes through vertex i, then going around that cycle and returning to i gives a negative total cost — setting dist[i][i] < 0. Any vertex on a negative cycle will have a negative self-distance, making this check reliable.
Implication: If dist[i][j] is computed using a path that goes through a negative cycle, that distance is meaningless (could be -8). So always check for negative cycles before trusting Floyd-Warshall's output.
Q3 — When would you choose Bellman-Ford over Floyd-Warshall for all-pairs shortest paths?
When the graph is sparse. Floyd-Warshall always runs in O(Vł) and requires O(V˛) space regardless of how many edges exist. For a sparse graph with E « V˛, running Bellman-Ford V times gives O(V˛ × E) « O(Vł) time and O(V + E) space — much better. For example, a road network with V=10,000 cities and E=50,000 roads: Floyd-Warshall = 10,000ł = 10ą˛ operations, matrix size = 10,000˛ × 4 bytes = 400MB. Running Bellman-Ford V times = 10,000 × 10,000 × 50,000 = 5×10ą˛ — actually worse here! Better: use Dijkstra V times (no negative edges in road networks).
Rule of thumb: Floyd-Warshall wins for dense graphs with V = 500 and when you need all-pairs. For larger graphs, run Dijkstra (or Bellman-Ford if negative edges) from each source.
Q4 — A graph has exchange rates: USD?EUR: 0.85, EUR?GBP: 0.78, GBP?USD: 1.52. Is there an arbitrage opportunity? How would you detect it with Bellman-Ford?
Check: 1 USD ? 0.85 EUR ? 0.663 GBP ? 1.008 USD. Yes! You end with 1.008 USD — a 0.8% profit per cycle. This is arbitrage. To detect with Bellman-Ford: transform edge weights using w_new = -log(rate). A negative cycle in this transformed graph = a cycle where the product of exchange rates > 1 = arbitrage. The negative sum of -log(rates) corresponds to log(product of rates) > 0, i.e., product > 1. Run Bellman-Ford and check for a negative cycle using the Vth relaxation pass.
Transformation: Edge A?B with rate r gets weight -log(r). Bellman-Ford finds the "shortest" path, which corresponds to the most profitable exchange sequence. Negative cycle = arbitrage.
Q5 — What is the "count-to-infinity" problem in distributed Bellman-Ford routing, and how is it mitigated?
The problem: In distributed routing (like RIP), when a link fails, the cost to an unreachable destination slowly counts up by 1 each round as routers update each other with outdated information. For example, if A?B fails, B tells C "I can reach A with cost 8", but C tells B "I can reach A with cost 2" (through B itself — a loop!), so B updates to cost 3, then C to 4, then B to 5... counting up to infinity. This wastes time and bandwidth before routers realize the route is gone. Mitigations: (1) Split Horizon: don't advertise a route back to the neighbor you learned it from. (2) Poison Reverse: advertise the route back but with infinite cost. (3) Maximum hop count: RIP uses 15 as infinity, treating any cost = 16 as unreachable. (4) Use OSPF (Dijkstra-based) instead, which converges faster.
Root cause: Distributed Bellman-Ford has no global view — nodes only know their neighbors' distances, which can create routing loops during failures before convergence.
