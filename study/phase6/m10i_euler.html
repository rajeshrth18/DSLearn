Section 1 — Euler Path vs. Euler Circuit

Named after Leonhard Euler who solved the Königsberg Bridge Problem in 1736 — the origin of graph theory itself — an Euler path and circuit ask: can we traverse every edge in a graph exactly once?

  • Euler Circuit (Closed Trail): A path that visits every edge exactly once and returns to the starting vertex.
  • Euler Path (Open Trail): A path that visits every edge exactly once but starts and ends at different vertices.
Königsberg Bridge Problem (1736): Land A /| |\ B1 B2 B3 B4 (7 bridges) \| |/ Land B ─── Land C | Land D Question: Can you walk through the city crossing each bridge exactly once? Answer: NO — Euler proved it impossible because all 4 landmasses have odd degree. Euler's key insight → degree parity determines Euler path/circuit existence!
🌍 Real-World Applications:
  • Route optimization: Postman/delivery routes that traverse every street exactly once
  • Circuit design: Single-stroke drawing of circuit diagrams
  • DNA sequencing: Eulerian paths in de Bruijn graphs reconstruct DNA sequences
  • Puzzle problems: "Draw this figure without lifting your pen" (LeetCode #332, #753)

Section 3 — Hierholzer's Algorithm O(E)

Hierholzer's algorithm constructs an Euler circuit (or path) in O(E) time using a stack-based DFS with backtracking. The naive DFS approach would be O(E²); Hierholzer avoids this by using a stack to defer adding nodes to the result.

Algorithm Steps

  1. Check existence conditions (degree parity).
  2. Choose the start vertex (any vertex with edges; for Euler path, start at the odd-degree vertex).
  3. Use a stack + DFS: push the current vertex, follow any unused edge, repeat.
  4. When stuck (no unused edges), pop the vertex into the result list.
  5. Reverse the result — it's the Euler path/circuit.
C#
public class EulerPath
{
    // Returns Euler circuit/path, or empty if none exists.
    // For undirected graphs. Edges are passed as list of (u,v) pairs.
    public static List<int> Hierholzer(int n, List<(int,int)> edges)
    {
        var adj = new List<int>[n];
        int[] degree = new int[n];
        for (int i = 0; i < n; i++) adj[i] = new();
        foreach (var (u, v) in edges)
        {
            adj[u].Add(v); adj[v].Add(u);
            degree[u]++; degree[v]++;
        }

        // Count odd-degree vertices
        int oddCount = 0, start = 0;
        for (int i = 0; i < n; i++)
        {
            if (degree[i] % 2 != 0) { oddCount++; start = i; }
            else if (degree[i] > 0 && oddCount == 0) start = i;
        }
        if (oddCount != 0 && oddCount != 2) return new(); // no Euler path/circuit

        // Track index into each adjacency list (to mark "used" edges efficiently)
        int[] idx = new int[n];
        var stack  = new Stack<int>();
        var result = new List<int>();
        stack.Push(start);

        while (stack.Count > 0)
        {
            int u = stack.Peek();
            if (idx[u] < adj[u].Count)
            {
                int v = adj[u][idx[u]++];
                stack.Push(v);
            }
            else
            {
                result.Add(stack.Pop());
            }
        }
        result.Reverse();
        return result; // empty if graph not fully traversed
    }
}

// Example: Euler circuit on a triangle
var path = EulerPath.Hierholzer(3, new List<(int,int)> { (0,1),(1,2),(2,0) });
Console.WriteLine(string.Join(" → ", path)); // e.g. 0 → 1 → 2 → 0
Python
from collections import defaultdict

def hierholzer(n: int, edges: list[tuple]) -> list[int]:
    adj = defaultdict(list)
    degree = [0] * n
    for u, v in edges:
        adj[u].append(v); adj[v].append(u)
        degree[u] += 1; degree[v] += 1

    # Determine start: odd-degree vertex for Euler path, any for circuit
    odd = [i for i in range(n) if degree[i] % 2 == 1]
    if len(odd) not in (0, 2):
        return []  # no Euler path/circuit

    start = odd[0] if odd else next((i for i in range(n) if degree[i] > 0), 0)

    # Use index pointers instead of removing from adj (for O(E) instead of O(E^2))
    idx = [0] * n
    stack, result = [start], []

    while stack:
        u = stack[-1]
        if idx[u] < len(adj[u]):
            v = adj[u][idx[u]]
            idx[u] += 1
            stack.append(v)
        else:
            result.append(stack.pop())

    return result[::-1]

# Euler circuit on 0-1-2-0
print(hierholzer(3, [(0,1),(1,2),(2,0)]))   # [0, 2, 1, 0] or similar

# Euler path: 0-1-2-3 chain
print(hierholzer(4, [(0,1),(1,2),(2,3)]))   # [0, 1, 2, 3]
Section 4 — Directed Graphs & LeetCode Patterns

For directed graphs, the conditions change: use in-degree and out-degree per vertex. A common LeetCode application is "reconstruct itinerary"

C# — Directed Euler Path (Reconstruct Itinerary)
using System.Collections.Generic;
using System.Linq;

// LeetCode 332 — Reconstruct Itinerary (start "JFK", lex smallest)
public static IList<string> FindItinerary(IList<IList<string>> tickets)
{
    var adj = new Dictionary<string, LinkedList<string>>();
    foreach (var t in tickets.OrderBy(t => t[1])) {
        if (!adj.ContainsKey(t[0])) adj[t[0]] = new();
        adj[t[0]].AddLast(t[1]);
    }
    var result = new LinkedList<string>();
    void Dfs(string ap) {
        while (adj.TryGetValue(ap, out var nb) && nb.Count > 0)
            { var nx = nb.First!.Value; nb.RemoveFirst(); Dfs(nx); }
        result.AddFirst(ap);
    }
    Dfs("JFK"); return result.ToList();
}
var tix = new List<IList<string>> {
    new[]{ "JFK","MUC" }, new[]{ "MUC","LHR" },
    new[]{ "LHR","SFO" }, new[]{ "SFO","SJC" }};
Console.WriteLine(string.Join(" → ", FindItinerary(tix)));
// JFK → MUC → LHR → SFO → SJC
Python — Directed Euler Path (Reconstruct Itinerary)
def find_itinerary(tickets: list[list[str]]) -> list[str]:
    """LeetCode 332: Reconstruct Itinerary
    tickets = [["MUC","LHR"],["JFK","MUC"],["SFO","SJC"],["LHR","SFO"]]
    Start at "JFK", use all tickets exactly once, lexicographically smallest."""
    from collections import defaultdict
    adj = defaultdict(list)
    for src, dst in sorted(tickets, reverse=True):  # sort descending for lex order
        adj[src].append(dst)

    result = []
    def dfs(airport):
        while adj[airport]:
            dfs(adj[airport].pop())
        result.append(airport)

    dfs("JFK")
    return result[::-1]

tickets = [["JFK","MUC"],["MUC","LHR"],["LHR","SFO"],["SFO","SJC"]]
print(find_itinerary(tickets))
# ["JFK", "MUC", "LHR", "SFO", "SJC"]
📌 Summary:
  • Euler Circuit (undirected): All vertices even degree + connected → O(V+E) check
  • Euler Path (undirected): Exactly 2 odd-degree vertices + connected
  • Hierholzer's: O(E) — use adjacency list with index pointers, not edge deletion
  • Directed Euler path: Track in-degree and out-degree; one source (out>in), one sink (in>out)
  • LeetCode pattern: "Reconstruct itinerary", "Valid Arrangement of Pairs" (#2097)
