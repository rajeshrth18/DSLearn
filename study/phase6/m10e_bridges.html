Section 1 — What Are Articulation Points & Bridges?

In a connected graph, some nodes and edges are more "critical" than others — removing them would disconnect the graph into multiple components. Identifying these is crucial for network reliability analysis.

  • Articulation Point (Cut Vertex): A node whose removal disconnects the graph.
  • Bridge (Cut Edge): An edge whose removal disconnects the graph.
Example graph — find articulation points and bridges: 1 -- 2 -- 3 | | 4 5 -- 6 -- 7 | 8 Articulation points: 2 (removing splits {1,4} from {3,5,6,7,8}), 5 (removing splits {8} from rest), 6 (removing splits {7} from rest) Bridges: edge (2,3), edge (5,8), edge (6,7) Real-world meaning: In a network, bridges are single points of failure. A server ON an articulation point is a critical infrastructure node.
?? Real-World Applications:
  • Network engineering: Find links/routers that, if failed, partition the network
  • Social networks: Identify "connector" individuals who bridge communities
  • Biology: Critical proteins in metabolic networks — knockouts cause system failure
  • Transportation: Critical roads/bridges in a city road network
Section 2 — Tarjan's DFS: Discovery Time & Low Value

Tarjan's algorithm finds both articulation points and bridges in a single O(V+E) DFS pass using two values per node:

  • disc[u] — Discovery time: the DFS visit order (timestamp when first visited)
  • low[u] — Low value: the earliest discovery time reachable from the subtree rooted at u via back edges (excluding the parent edge)
DFS Tree with back edges: 1(disc=0) / \ 2 3 / \ 4 5 --back edge--? 1 disc[4]=2, low[4]=2 (no back edges from subtree of 4) disc[5]=3, low[5]=0 (back edge to node 1, disc=0) Key rules: Bridge rule: edge(u,v) is a bridge iff low[v] > disc[u] Artic. point: u is an AP (non-root) iff low[v] >= disc[u] for some child v Root is AP: root is an AP iff it has 2+ children in DFS tree
Section 3 — Finding Bridges

An edge (u, v) is a bridge if and only if low[v] > disc[u] — meaning there is no back edge from the subtree rooted at v that reaches u or above. Removing this edge would disconnect v's component.

C#
public class BridgeFinder
{
    private List<int>[] adj;
    private int[] disc, low;
    private bool[] visited;
    private int timer = 0;
    private List<(int, int)> bridges = new();

    public List<(int, int)> FindBridges(int n, List<(int, int)> edges)
    {
        adj = new List<int>[n];
        disc = new int[n]; low = new int[n]; visited = new bool[n];
        for (int i = 0; i < n; i++) adj[i] = new();
        foreach (var (u, v) in edges) { adj[u].Add(v); adj[v].Add(u); }
        for (int i = 0; i < n; i++) if (!visited[i]) DFS(i, -1);
        return bridges;
    }

    private void DFS(int u, int parent)
    {
        visited[u] = true;
        disc[u] = low[u] = timer++;

        foreach (int v in adj[u])
        {
            if (!visited[v])
            {
                DFS(v, u);
                low[u] = Math.Min(low[u], low[v]);
                if (low[v] > disc[u])         // ? Bridge condition
                    bridges.Add((u, v));
            }
            else if (v != parent)              // Back edge (not parent edge)
                low[u] = Math.Min(low[u], disc[v]);
        }
    }
}

// Usage
var finder = new BridgeFinder();
var edges  = new List<(int,int)> { (0,1),(1,2),(2,0),(1,3),(3,4) };
var bridges = finder.FindBridges(5, edges);
// Bridges: (1,3), (3,4)
Python
from collections import defaultdict

def find_bridges(n: int, edges: list[tuple]) -> list[tuple]:
    adj = defaultdict(list)
    for u, v in edges:
        adj[u].append(v); adj[v].append(u)

    disc = [-1] * n
    low  = [-1] * n
    timer = [0]
    bridges = []

    def dfs(u: int, parent: int):
        disc[u] = low[u] = timer[0]
        timer[0] += 1
        for v in adj[u]:
            if disc[v] == -1:          # unvisited
                dfs(v, u)
                low[u] = min(low[u], low[v])
                if low[v] > disc[u]:    # bridge condition
                    bridges.append((u, v))
            elif v != parent:
                low[u] = min(low[u], disc[v])

    for i in range(n):
        if disc[i] == -1: dfs(i, -1)
    return bridges

print(find_bridges(5, [(0,1),(1,2),(2,0),(1,3),(3,4)]))
# [(1, 3), (3, 4)]
Section 4 — Finding Articulation Points

A vertex u is an articulation point if any of these hold:

  1. Root with 2+ children: The DFS root is an AP if it has 2 or more children in the DFS tree.
  2. Non-root with low[v] ≥ disc[u]: For a non-root vertex u with a DFS child v, u is an AP if low[v] ≥ disc[u] — meaning no back edge from the subtree of v goes above u.
C#
public HashSet<int> FindArticulationPoints(int n, List<(int,int)> edges)
{
    var adj = new List<int>[n];
    var disc = new int[n]; var low = new int[n];
    var visited = new bool[n];
    var aps = new HashSet<int>();
    int timer = 0;
    for (int i = 0; i < n; i++) adj[i] = new();
    foreach (var (u, v) in edges) { adj[u].Add(v); adj[v].Add(u); }

    void DFS(int u, int parent)
    {
        visited[u] = true;
        disc[u] = low[u] = timer++;
        int children = 0;
        foreach (int v in adj[u])
        {
            if (!visited[v])
            {
                children++;
                DFS(v, u);
                low[u] = Math.Min(low[u], low[v]);
                // Non-root AP: low[v] >= disc[u]
                if (parent != -1 && low[v] >= disc[u]) aps.Add(u);
                // Root AP: has 2+ DFS children
                if (parent == -1 && children > 1) aps.Add(u);
            }
            else if (v != parent)
                low[u] = Math.Min(low[u], disc[v]);
        }
    }
    for (int i = 0; i < n; i++) if (!visited[i]) DFS(i, -1);
    return aps;
}
Python
def find_articulation_points(n: int, edges: list[tuple]) -> set[int]:
    adj = defaultdict(list)
    for u, v in edges:
        adj[u].append(v); adj[v].append(u)
    disc = [-1] * n; low = [0] * n
    timer = [0]; aps = set()

    def dfs(u: int, parent: int):
        disc[u] = low[u] = timer[0]
        timer[0] += 1
        children = 0
        for v in adj[u]:
            if disc[v] == -1:
                children += 1
                dfs(v, u)
                low[u] = min(low[u], low[v])
                if parent != -1 and low[v] >= disc[u]: aps.add(u)
                if parent == -1 and children > 1:      aps.add(u)
            elif v != parent:
                low[u] = min(low[u], disc[v])

    for i in range(n):
        if disc[i] == -1: dfs(i, -1)
    return aps

# Graph: 0-1-2-0 triangle, plus 1-3-4 chain
print(find_articulation_points(5, [(0,1),(1,2),(2,0),(1,3),(3,4)]))
# {1, 3}
?? Complexity: O(V + E) time and space — single DFS pass. Both bridges and articulation points can be found in the same DFS run. For multigraphs (multiple edges between same pair), track edge IDs instead of parent vertex to avoid treating a parallel edge as a back edge.
